## v3.4 可继承事实

- 小区域 L2/L3 `AgentProposal -> WorldCore post-check -> 玩家可见候选表达` 已进入 runtime。
- v3.4 没有新增 store slice、save field、migration、`SAVE_FORMAT_VERSION` bump 或 `runFingerprint`。
- v3.4 没有调用 live DeepSeek，没有扩 DeepSeek 权限，没有 MiroFish、backend 或 external framework。
- v3.4 PA 10/10 说明玩家能理解候选不是事实。

## v3.4 暴露的下一步摩擦

- 生命周期还偏二元：可见候选 / 阻断项，缺少 `expired` 与 `needs_user_decision` 的一致显示。
- rejection 可读性需要强化，避免玩家把“阻断探针”理解成游戏已经发生。
- deterministic variation 只有 30 轮，需要提升到 60 轮来确认 same-start 变化不依赖持久随机种子。
- PA 需要从 10 轮提升到 20 轮，专门检查候选、阻断、过期、待决策四类状态。

## v3.5 范围冻结

v3.5 只做：

- lifecycle v2：`candidate / rejected / expired / needs_user_decision`。
- candidate-vs-fact 与 rejection copy 强化。
- focused hardening first cut。
- 60 轮 deterministic drift。
- 20 轮 Player Advocate。

## 不扩权项

v3.5 不扩大 runtime agent 到 persistent、save、live DeepSeek、MiroFish、backend、external framework、L4/L5 或正式世界事实裁决。

## 结论

a0 通过。v3.5 可进入 a1 lifecycle v2 设计门禁。
