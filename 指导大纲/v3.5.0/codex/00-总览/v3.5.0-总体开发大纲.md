# v3.5.0 总体开发大纲

日期：2026-05-25
状态：startup；pending user approval

## 主线

`transient AgentProposal hardening-first`

v3.5 不追求“更多 agent 权限”，而追求“v3.4 首刀足够稳、玩家足够懂、未来能继续扩”。

## 阶段目标

- a0：v3.4 首刀复盘与 v3.5 hardening 范围冻结。
- a1：AgentProposal lifecycle v2 设计门禁。
- a2：candidate-vs-fact / UI 误读 / rejection copy 二次门禁。
- b1：实现或检查 v350 hardening 第一刀。
- b2：same-start deterministic variation 与 no-`runFingerprint` 复核。
- b3：20 轮 Player Advocate + 60 轮 deterministic drift。
- process-1：前置审批制度第十一轮复核。
- process-2：长线漂移与知识边界复核。
- rc：质量收束，决定 v3.6 是否微扩或继续硬化。

## 设计原则

- WorldCore final authority 不变。
- Agent 只给候选，不给事实。
- UI 只呈现候选表达、拒绝原因、机会窗口和下一步，不呈现正式结果。
- 同开局差异只能来自 deterministic candidate deck 和公开状态组合，不写随机种子或 `runFingerprint`。
- 失败样本进入测试矩阵，不靠文案绕过。

## 版本完成标准

v3.5 完成时必须证明：

- 玩家能理解候选不是事实。
- rejected proposal 不污染状态。
- old-save 打开不需要 migration。
- rollback 不留存档残留。
- P0/P1 boundary false negative 为 0。
- 没有新增 save / DeepSeek / MiroFish / backend / external framework 权限。
