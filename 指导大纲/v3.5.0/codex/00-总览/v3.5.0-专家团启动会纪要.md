# v3.5.0 专家团启动会纪要

日期：2026-05-25
状态：startup；pending user approval
主题：v3.4 首个 transient runtime first cut 后，v3.5 是继续硬化还是扩大试点

## 启动前证据

- v3.4 已完成最小 `transient proposal-only` L2/L3 runtime first cut。
- v3.4 新增 `v340-transient-agent-proposal`、世界面板 `意图` tab、WorldCore post-check、focused unit/e2e、10 轮 Player Advocate 和 30 轮 deterministic 证据。
- v3.4 不写 store slice，不写 save，不 bump `SAVE_FORMAT_VERSION`，不调用 live DeepSeek，不用 MiroFish，不引入 backend / external framework。
- v3.4 CI 已通过，当前分支 `codex/v340-startup-runtime-route-decision` 已闭环。

## 路线 A：hardening-first（推荐）

### 内容

继续使用 v3.4 的窄口，只做：

- AgentProposal lifecycle 硬化：candidate / rejected / expired / needs_user_decision。
- WorldCore post-check 证据链可读性。
- UI 中“候选不是事实”的二次验证。
- deterministic same-start variation，不引入 `runFingerprint`。
- rejection / rollback / old-save / no-save-write 复核。
- Player Advocate 强度从 10 轮提升到 20 轮，deterministic 从 30 轮提升到 60 轮。

### 收益

- 最符合长期路线：v3.4-v3.6 是最小 L2/L3 小区域试点窗口，但每一步必须窄。
- 能把 v3.4 首刀从“能跑”推进到“可被长期维护”。
- 风险低，不触发 save、DeepSeek、MiroFish、backend、external framework。
- 对玩家体验有真实收益：减少候选误读、重复、空转和目标遗失。

### 风险

- 玩法鲜活度提升有限，仍不是完整 NPC agent。
- 如果过度保守，v3.5 看起来会像“抛光”而不是“新能力”。
- 需要较细的测试矩阵，否则硬化容易变成只写文档。

## 路线 B：小样本扩展（可选但不默认）

### 内容

仍然 `transient proposal-only`，但在 synthetic / RebornG-owned 范围内扩到：

- 2-3 个 L2/L3 非命名 NPC archetype。
- 1 个抽象小势力 pressure source。
- 多个同开局候选表达 deck。

### 收益

- 能更明显验证同开局差异和 NPC 意图多样性。
- 为 v3.6 的 broader hardening 提供材料。

### 风险

- 更容易滑向“社会系统”“正式关系”“势力状态”。
- UI 更容易把候选表达误读成事实。
- 若样本触及命名 NPC、真实势力或 hidden-adjacent，MiroFish 必须升级 blocking。

## 路线 C：暂停 runtime，回到 report-only

### 收益

- 零新增 runtime 风险。
- 可以把精力转向 v4.0 高阶战斗 / 天道宿命预备。

### 风险

- 会浪费 v3.4 刚打开的最小 runtime 经验窗口。
- 长期路线会重新回到“报告漂亮但缺少玩家摩擦”的问题。

## 专家团意见

### Project Lead

v3.5 应走路线 A。v3.4 的第一刀刚进入 runtime，下一步应先把它变成稳定制度，而不是立刻扩大 agent 权限。

### Systems Architect

不建议新增存档或 agent memory。v3.5 的关键指标应是 no-save-write、rollback、old-save 和 deterministic same-start variation 都稳。

### AI Pipeline Architect

v3.5 不建议 live DeepSeek。先不要把模型变量放进 agent runtime 接入变量里。若未来要 live，必须单独审批模型、样本、轮次、成本和 P0/P1 停机标准。

### Lore Guard

v3.5 继续保持 synthetic / RebornG-owned 泛化样本。真实原著地点、势力、命名 NPC、方源证据、hidden-adjacent 或正式 lore 结论会立刻触发 MiroFish blocking。

### Player Advocate

v3.5 要重点看玩家是否能分清：NPC 候选意图、机会窗口、压力提示都不是既成事实。建议 20 轮 PA，专门记录误读点。

### QA / Release

建议新增 v350 focused unit/e2e 或 checker；full unit、typecheck、build、e2e、long e2e、production-preview 仍作为 rc 收束门。

### Git / Rollback Steward

已切分支 `codex/v350-startup-transient-agent-hardening`。startup 提交只放文档与入口同步；实现必须另起明确提交边界。

## 专家团推荐结论

推荐 v3.5 主线：

`transient AgentProposal hardening-first`

也就是：不扩大权限，只把 v3.4 的最小 runtime first cut 做成更稳、更可测、更不容易被玩家误读的工程机制。
