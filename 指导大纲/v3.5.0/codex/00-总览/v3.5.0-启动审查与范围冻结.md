# v3.5.0 启动审查与范围冻结

日期：2026-05-25
状态：completed

## 上一版本输入

v3.4 已完成并通过远端 CI。可作为 v3.5 输入的事实只有：

- 小区域 L2/L3 transient AgentProposal 可以进入 runtime 表达层。
- WorldCore post-check 是最终裁决门。
- 玩家可见 UI 必须持续强调 candidate / rejected / no-save / no-fact。
- 当前没有持久 agent 状态、没有 agent memory、没有 live DeepSeek、没有 MiroFish、没有 backend、没有外部框架。

## v3.5 范围

本版本建议冻结为：

`transient AgentProposal hardening-first`

允许讨论或在获批后实现：

- v3.4 proposal lifecycle 硬化。
- candidate / rejected / expired / needs_user_decision 分类可读性。
- deterministic variation deck，不写 `runFingerprint`。
- UI candidate-vs-fact 二次复核。
- no-save-write / rollback / old-save 复核。
- 20 轮 Player Advocate 和 60 轮 deterministic 漂移检查。

## 明确不在范围

- persistent runtime agent。
- agent memory save。
- store slice / save field / migration / `SAVE_FORMAT_VERSION` bump / `runFingerprint`。
- live DeepSeek、prompt/context/model/authority。
- MiroFish topic-slice、真实原著事实、命名 NPC、真实势力、hidden-adjacent、正式 lore。
- backend/BFF/service。
- external framework PoC/dependency/subagent/read-only scan/patch artifact。
- L4/L5、天道/宿命 runtime、原著关键人物 runtime agent。
- formal location/faction/identity/reward/NPC life-death。

## 启动判断

v3.5 已获得 D-350 授权并按 hardening-first 范围完成。未触发任何 future gate。
