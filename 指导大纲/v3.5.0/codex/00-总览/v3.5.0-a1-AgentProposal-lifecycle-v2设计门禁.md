# v3.5.0-a1 AgentProposal lifecycle v2 设计门禁

日期：2026-05-25
状态：completed
冻结版本：`v350_a1_agentproposal_lifecycle_v2`

## Lifecycle v2

v3.5 将 runtime 可见 AgentProposal 统一分为四类：

| 状态 | 含义 | 可见策略 | 是否写事实 |
|---|---|---|---|
| `candidate` | 通过 WorldCore post-check 的可见表达候选 | 显示候选表达和安全下一步 | 否 |
| `rejected` | 触发 P0/P1 阻断的越权候选 | 只显示安全原因，不显示被拒正文 | 否 |
| `expired` | 旧窗口或重排后不再适用的候选 | 只提示重新取样 | 否 |
| `needs_user_decision` | 触发 future gate 的候选 | 只显示待决策原因 | 否 |

## Hard invariants

- `factsCommitted = false`
- `saveWritten = false`
- `promptWritten = false`
- `canonPromoted = false`
- `persistenceMode = transient_proposal_only`
- `WorldCore post-check` 仍是最终裁决。

## Runtime 设计

新增纯函数层：

- `src/engine/v350-transient-agent-proposal-hardening.ts`

它复用 v3.4 的 `buildV340TransientAgentProposal()` 与 `postCheckV340AgentProposal()`，只包装 lifecycle、copy guard、deterministic 60 轮证据，不新增 store 或 save。

## 结论

a1 通过。lifecycle v2 可以进入 UI/copy 门禁。
