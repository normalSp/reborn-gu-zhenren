# v1.5.0 MiroFish 资料需求与交付协议

状态：completed
日期：2026-05-21

## 初始 need level

`preferred`

## 为什么不是 blocking

v1.5 第一刀建议是 projection-first 冲突后果解释层，可以先复用：

- v0.17 历史战斗、杀招、小队与阵法深化资料。
- 当前 runtime canon / engine / combat trace。
- v1.1-v1.4 已有 route/survival/social/region evidence。

这些足以支撑 a1/a2 设计门禁和 b1 projection-only 第一刀。

## 什么时候升级 blocking

出现以下任一情况，必须先做 MiroFish topic-slice request 和 RebornG intake review：

- 正式命名 NPC 追杀、截杀、死亡、捕获、背叛。
- 原著角色参与战斗结论。
- 原著杀招传承、特定杀招归属、杀招效果 canon 晋升。
- 隐藏事实邻近的冲突、方源私密因果、重生/春秋蝉相关诱导。
- 正式势力通缉、围剿、招揽、封锁、敌对关系。
- 任何会进入 runtime canon、DeepSeek visible context、玩家可见事实或测试硬门的原著边界材料。

## 全书基础包规则

`指导大纲/vMiroFish/基础包/` 仍是 archive/source-pointer inventory。

禁止：

- 整包导入知识库。
- 整包喂给 DeepSeek。
- 整包当作 runtime canon。
- 直接把隐藏/private body 写入 UI、prompt 或测试可见文本。

允许：

- 查 source pointer 覆盖缺口。
- 生成 topic-slice request 草案。
- 作为 a2 intake review 的候选来源。

v1.5 没有请求新 MiroFish 包。a2 已完成 topic-slice / historical source review，结论是：
