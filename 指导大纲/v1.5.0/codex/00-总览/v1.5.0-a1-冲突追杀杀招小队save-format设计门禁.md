## 1. 本门禁要回答什么

v1.5 的核心风险不是“能否显示战斗信息”，而是冲突系统会不会无意中变成正式奖励、正式追杀、正式 NPC 生死或 DeepSeek 战斗裁判。

a1 必须回答：

1. b1 是否需要把 `SAVE_FORMAT_VERSION` 从 24 bump 到 25。
2. 是否现在新增 `conflictConsequenceState`、`pursuitState`、`combatAftermathState` 或同类持久字段。
3. 现有 combat trace、candidate queue、action ledger、`routeLocationState`、`survivalEconomyState`、`livingWorldState` 和 v1.4 region projection 是否足够承载 b1。
4. v0.17 战斗深化资料能否直接作为当前 runtime authority。
5. DeepSeek、MiroFish、UI 动效在战斗后果里各自能做什么，不能做什么。

门禁结论先给出：专家团建议 **v1.5-b1 不 bump v25、不新增冲突后果持久字段，先做 projection-first 冲突后果解释层**。如果 b1/b2 证明长期冲突后果 ledger 必要，再单独进入 `SAVE_FORMAT_VERSION = 25` 小门禁。

## 3. 权威边界表

| 对象 | v1.5-a1 结论 | 说明 |
|---|---|---|
| 战斗伤害、命中、胜负、步骤 | 本地 combat engine / BattleResolutionStep | UI 与 DeepSeek 不得私算。 |
| 冲突风险来源 | 可 projection-only | 可从路线、生存、社会、区域和 combat candidate 推导风险。 |
| 正式追杀/通缉/敌对结论 | 不开放 | 只能显示风险、压力、前置和降险建议。 |
| NPC 生死、捕获、背叛、永久伤势 | 不开放 | 需单独 canon/IF/user decision。 |
| 掉落、稀有蛊、仙蛊、完整杀招传承 | 不开放 | 不允许由战斗深化顺手产生。 |
| `livingWorldState` | 可作为证据来源 | 只读证据，不扩写语义为正式冲突后果系统。 |
| `routeLocationState` | 可作为路线风险来源 | 不写正式新地点/路线成功。 |
| `survivalEconomyState` | 可作为补给/维护压力来源 | 不结算真实消耗、价格、掉落。 |
| v0.17 战斗深化资料 | 可作为历史/候选规则参考 | 需 a2 复核，不能自动成为 v1.5 runtime authority。 |
| DeepSeek | 只写叙事、传闻、压力、战前/战后候选文本 | 不写伤害、掉落、生死、正式追杀、阵营结论。 |
| MiroFish | source pointer / candidate / test seed | 不直接进 runtime/canon/DeepSeek visible。 |
| reborn-combat-motion | 表现与 reduced-motion 规则 | 不能表现未由 engine/canon 提供的战斗事实。 |

## 4. 方案比较

| 路线 | 做法 | 收益 | 风险 | 专家团建议 |
|---|---|---|---|---|
| A：保持 v24 projection-first | 不 bump save；新增纯 engine projection helper；UI 只读展示冲突来源、追杀风险、杀招/小队前置和禁止项 | 最稳；旧档零迁移；能快速验证玩家是否看懂冲突后果 | 不保存长期冲突后果；后续若需要 ledger 要再开 v25 | 推荐 b1 使用 |
| B：v25 + `conflictConsequenceState` | 新增单一聚合对象，记录战斗后果、追杀压力、反制缺口、source refs | 后续长线冲突 ledger 更清楚；可做 replay/audit | 迁移和旧档成本高；容易误开正式死亡/掉落/通缉结论 | 暂缓，等 b1/b2 证明必要 |
| C：把冲突后果塞进 `livingWorldState` | 不 bump save，扩写现有字段语义 | 表面省事 | 语义污染；把记忆/压力证据误当战斗后果权威 | 不推荐 |
| D：直接复用 v0.17 规则作为 v1.5 authority | 读取 v0.17 rules，直接驱动新 UI/逻辑 | 快 | 旧版本范围与 v1.5 新地基不同；可能绕过 v1.5 a2 intake | 不推荐直接使用，需 a2 复核 |
| E：DeepSeek 叙事裁决冲突后果 | 让 DeepSeek 写追杀、掉落、生死、战后变化 | 自由感强 | P0 越权；不可测试，不可回滚 | 拒绝 |

## 5. b1 推荐 authority 模型

b1 如按推荐路线执行，应新增只读 projection 类型或内部返回对象，不进入 save：

```ts
type V150ConflictAftermathProjection = {
  scopeId: 'low_rank_conflict_outer_sample';
  savePolicy: 'no_new_persistence_v24';
  authority: 'local_projection_only';
  conflictPosture:
    | 'route_ambush_risk'
    | 'pursuit_attention_window'
    | 'countermeasure_gap'
    | 'squad_formation_readiness';
  routeSignals: string[];
  survivalSignals: string[];
  socialSignals: string[];
  regionSignals: string[];
  combatSignals: string[];
  counterHints: string[];
  squadHints: string[];
  nextStepCandidates: string[];
  forbiddenWrites: string[];
  sourceRefs: string[];
};
```

允许读取：

- `routeLocationState`
- `survivalEconomyState`
- `livingWorldState.knownFacts`
- `livingWorldState.npcMemories`
- `livingWorldState.factionPressure`
- `livingWorldState.actionConsequences`
- scene-local action ledger
- existing combat candidates / battle trace / `BattleResolutionStep`
- v0.17 reviewed data after a2 复核
- v1.3 social projection 与 v1.4 region sample projection 的公开输出

- 正式势力通缉、围剿、招揽、封锁、敌对关系。
