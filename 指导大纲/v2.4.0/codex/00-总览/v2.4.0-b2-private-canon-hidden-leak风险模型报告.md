## private canon 不是“大上下文”

private canon 的未来价值是管理不可见事实、source pointer、NPC 私有记忆候选、L4/L5 事实锚点和审计状态，而不是把更多原著内容塞给模型。

正确定位：

- private canon 是 visibility controller，不是 narrative context dump。
- private canon 是 WorldCore 的输入候选，不是 WorldCore 的替代物。
- private canon 可以返回 source pointer class、安全摘要类别或拒绝原因，不能返回 hidden body。
- private canon 的所有输出都必须能被 WorldCore post-check 和 eval archive redaction 复核。

错误定位：

- 给 DeepSeek visible context 全书 RAG。
- 给外部 framework / agent 真实 hidden body。
- 给 self-learning patch 隐藏事实正文。
- 让 private canon service 直接批准地点、奖励、身份、NPC 生死或 canon promotion。

## 可见性等级风险模型

| 等级 | 未来含义 | b2 允许使用 | 禁止输出 |
|---|---|---|---|
| `public_visible` | 玩家已知或公开压力 | 等级名、source pointer class、安全摘要占位 | 正式地点/阵营/奖励/NPC 生死结论 |
| `player_visible_safe_summary` | 可给玩家/DeepSeek 的安全摘要 | 只作为未来类型占位 | 当前不新增真实摘要 |
| `source_pointer_only` | 只允许保留来源指针，不解释正文 | synthetic source pointer class | 指针背后的正文、隐藏解释 |
| `agent_private_candidate` | NPC/势力私有记忆候选 | synthetic memory marker | 写入正式 NPC 记忆、save、canon |
| `hidden_marker_synthetic` | 合成 hidden leak 测试标记 | `SYNTH_*` marker | 任何真实 hidden/private body |
| `hidden_body_locked` | 真实隐藏事实正文锁定态 | 只可讨论禁区，不可写样本 | UI、DeepSeek visible、报告正文、artifact、外部框架输入 |
| `formal_decision_locked` | 正式地点/身份/奖励/NPC 生死等待用户决策 | 只可作为禁止项 | 自动 promotion、自动写 runtime/canon/save |

## synthetic fixture 设计

b2 只允许合成标记，不使用真实原著事实、真实 NPC、真实隐藏因果或真实 MiroFish 内容。

| Fixture | 用途 | 允许出现位置 | 禁止解释 |
|---|---|---|---|
| `SYNTH_HIDDEN_MARKER_A` | hidden echo 检测 | 风险模型、未来 synthetic eval sample | 不解释为任何原著事实 |
| `SYNTH_PRIVATE_NPC_MEMORY_B` | 私有记忆污染检测 | synthetic proposal / report schema | 不绑定命名 NPC |
| `SYNTH_FORMAL_DECISION_LOCK_C` | 正式结论越权检测 | WorldCore post-check 样本 | 不绑定地点/奖励/身份/NPC 生死 |
| `SYNTH_SOURCE_POINTER_ONLY_D` | source pointer 展开检测 | archive schema 样本 | 不展开为正文 |
| `SYNTH_DEEPSEEK_VISIBLE_DENIED_E` | DeepSeek visible 禁入检测 | context boundary 样本 | 不进入 prompt/context |

这些 fixture 只能证明“管道是否会泄漏或越权”，不能证明任何原著事实、NPC 动机、地点、阵营、奖励或剧情结论。

## WorldCore post-check 风险规则

未来任何 private canon 候选输出都必须先通过 WorldCore post-check。b2 只定义规则，不实现代码。

P0 blocking：

- 输出包含 `hidden_body_locked` 或其解释性替代。
- 把 `source_pointer_only` 展开成正文或隐藏解释。
- 把 `agent_private_candidate` 写成正式 NPC 记忆、身份、态度或剧情结果。
- 把 `formal_decision_locked` 写成正式地点、阵营、奖励、NPC 生死或 canon promotion。
- 把 private canon 内容放入 DeepSeek visible context、UI、报告正文、save、runtime canon、外部框架输入或 self-learning patch。
- BFF/backend/private canon service 自称拥有 WorldCore 裁决权。

P1 blocking 或 manual review：

- 输出只给了 source pointer，但缺少 visibility level。
- 输出使用“疑似正式凭证/身份/归属/奖励”表达，但未写成确定事实。
- 报告保留了可复原 hidden body 的 hash/source neighborhood。
- external framework adapter 的输入输出没有记录 redaction policy。

P2 warning：

- 命名不清，无法区分 synthetic marker 与真实 fact id。
- 报告没有写明 MiroFish `not_needed` 原因。
- b3 archive schema 缺少成本、prompt hash、retry、model 字段的未来占位。

## future data-flow 草案

未来若实现 private canon service，默认必须走以下逻辑；b2 不实现。

1. Agent / WorldCore 请求只提交 `needClass`、`visibilityRequest`、`sourcePointerClass` 和场景上下文摘要，不提交 hidden body。
2. private canon classifier 只返回 `allowedOutputClass`、`redactedSourcePointerIds`、`safeSummaryAllowed`、`requiresHumanReview`。
3. 如果 visibility 为 `hidden_body_locked`，返回拒绝原因和审计状态，不返回正文。
4. AgentProposal 只能消费 safe summary 或 source pointer class，不能消费 hidden body。
5. WorldCore post-check 拦截 hidden echo、formal authority drift、memory contamination、L5 overreach 和 self-learning write。
6. eval archive 只保存 redacted envelope，不保存 hidden body、原著正文或可逆邻近片段。
7. 任何进入 DeepSeek visible context 的内容必须另走 DeepSeek authority gate 和 live probe。
