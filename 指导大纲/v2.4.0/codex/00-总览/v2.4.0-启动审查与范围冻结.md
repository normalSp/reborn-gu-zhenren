# v2.4.0 启动审查与范围冻结

日期：2026-05-23
状态：startup；待 D-240 审批。

## 版本目标

v2.4 的目标是把未来基础设施边界设计清楚，而不是实现基础设施。

核心问题：

- 薄 BFF 是否必要，边界在哪里。
- private canon / hidden body 如何服务未来 agent，但不泄漏。
- eval archive / replay / cost / prompt hash 如何成为工程制度。
- 外部 agent framework 如何只作为 adapter 参考，不成为世界裁决权。
- self-learning 如何只生成候选，不自裁决。

## 明确不做

- 不改 runtime。
- 不新增后端/BFF。
- 不调用 live DeepSeek。
- 不改 DeepSeek prompt/context/model/authority。
- 不新增 save 字段，不 bump `SAVE_FORMAT_VERSION`。
- 不新增 `runFingerprint`。
- 不启用 subagents。
- 不新增第三方 dependency / vendored subset。
- 不做 Hermes 或其他框架 PoC。
- 不允许外部 SDK/agent 读写 RebornG 项目文件。
- 不允许外部 SDK/agent 执行 shell/package/project 命令。
- 不允许外部 SDK/agent 生成 patch artifact 或操作 git。
- 不新增 MiroFish export。
- 不新增知识库正文、runtime canon 或 DeepSeek visible lore/RAG。
- 不开放正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。
- 不自动部署 EdgeOne。

## 触发即停

以下事项需要新的用户决策：

- 任何后端/BFF 实现。
- 任何外部 framework PoC。
- 任何依赖安装、vendored subset、隔离 PoC 目录。
- 任何外部工具读取项目文件、写项目文件、执行命令、生成 patch、操作 git。
- 任何 DeepSeek live probe 或 prompt/context 权限变化。
- 任何 private canon 下发机制进入 runtime。
- 任何 save-format 变化。
- 任何正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。

v2.4 可以启动，但必须先让用户审批 D-240。审批前只允许继续讨论和修正文档，不进入 a1 实作。
