## schema 定位

eval archive 是证据治理层，不是运行时功能。

正确定位：

- 统一记录 eval / replay / Player Advocate / live probe / risk review 的脱敏证据信封。
- 让未来报告具备可比性：模型、成本、prompt hash、retry、cache、acceptance、P0/P1/P2、WorldCore post-check。
- 把 b2 的 hidden leak 风险模型落成 archive 禁区：报告体和 artifact 不能保存 hidden/private body。
- 支撑 v2.5/v2.6 的 private canon / job queue / replay archive 评估。

错误定位：

- 把 eval 通过当作 canon promotion。
- 把 archive service 当作已批准后端。
- 把 schema 当作 runner 实现。
- 把 prompt hash 当作完整 prompt 或 hidden context。
- 把 source pointer 展开成原著正文或 hidden body。

## 字段解释与禁止误用

| 字段 | 用途 | 禁止误用 |
|---|---|---|
| `reportId` | 稳定引用报告 | 不能作为 runtime id |
| `targetVersion` / `phase` | 归属版本和阶段 | 不能暗示 scope 自动获批 |
| `sourceScope` | 说明证据来源 | 不能把 docs-only 写成 live 验证 |
| `visibilityLevel` | 记录可见性等级 | 不能反推出 hidden body |
| `redactedSourcePointerIds` | 保留脱敏来源指针 | 不能展开正文、邻近段落或原著原文 |
| `containsHiddenBody: false` | archive 硬断言 | 不能被 nullable 或 optional 替代 |
| `containsPromptBody: false` | 不归档 prompt 正文 | prompt hash 不等于 prompt body |
| `leakCheck` | 记录泄漏检测摘要 | 不能包含泄漏样本文本 |
| `worldCorePostCheck` | 记录裁决后检查摘要 | 不能替代 WorldCore 裁决 |
| `replayMetadata` | 记录 replay 规模和 seed 政策 | 不新增 `runFingerprint` |
| `modelMetadata` | 记录模型名、hash、liveCalled | 不调用 live，不归档 prompt body |
| `cost` | 记录成本来源和总量 | 不保存 `.env`、API key、账单私密数据 |
| `retryStats` | 记录 retry / repair 摘要 | 不保存原始失败 hidden 文本 |
| `boundaryAssertions` | 记录阶段边界 | 不能绕过用户审批 |

## Redaction policy

硬规则：

- `hidden_body_locked` 永远不进 archive body。
- 原著正文、MiroFish 原始输出、hidden/private body、完整 prompt、`.env`、API key、外部框架运行日志都不进 archive body。
- source pointer 只能 `id_only`，不得保存邻近正文或可逆上下文。
- prompt hash 只保存 hash，不保存 prompt body。
- model/cost/retry 只保存汇总，不保存可泄漏私有内容的 raw payload。
- eval 通过不等于 canon promotion。

若未来 archive service 被批准，服务日志也必须遵守同一 redaction policy。

## Replay / cost / prompt hash 口径

### Replay

- `deterministic`：本地可复现 replay。
- `live_replay`：live 输出后的 replay / rescore。
- `mixed`：deterministic + live 或 replay 混合证据。
- `runFingerprintPolicy` 默认 `absent` 或 `forbidden`，除非未来用户单独批准。

### Cost

- b3 只定义字段，不调用 API。
- 成本来源只允许 `api_report`、`manual_estimate`、`future` 或 `none`。
- 不保存 API key、`.env`、账单账号、请求 raw payload。

### Prompt hash

- `promptHash` 是 prompt/body/config 的不可逆摘要引用。
- `promptBodyArchived` 必须为 `false`。
- prompt hash 不能被用来恢复 prompt，也不能证明 hidden facts 被允许进入 visible context。

### Retry

- `retryAttemptCount` 记录重试次数。
- `recoveredRoundCount` 记录修复成功轮次。
- `unrecoveredRoundCount` 记录仍失败轮次。
- retry stats 不能保存原始 hidden leak 文本。

## MiroFish 与 lore 判定

v2.4-b3 保持 MiroFish `not_needed`。

原因：

- b3 只写 eval archive schema 草案。
- 不新增真实原著事实。
- 不引入命名 NPC、hidden-adjacent、方源公开证据或正式 lore 结论。
- 不新增知识库正文、runtime canon、DeepSeek visible lore/RAG。
- 不新增 artifact，不迁移历史报告，不读取 MiroFish output。

升级为 `blocking` 的条件：

- b3/rc 需要真实原著事实样本。
- archive schema 不再只用 synthetic / placeholder，而触及真实 hidden-adjacent body。
- external framework PoC 需要 lore fixture。
- 引入命名 NPC、L4/L5 fact anchor、方源公开证据、正式地点/阵营/身份/奖励/NPC 生死或 canon promotion。
- DeepSeek visible lore/RAG、private canon service implementation 或 eval archive service implementation 被提议进入实现。
