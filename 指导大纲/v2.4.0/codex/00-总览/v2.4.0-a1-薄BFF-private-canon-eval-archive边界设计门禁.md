## 一句话结论

v2.4-a1 冻结 `v240_a1_infrastructure_boundary_model`：薄 BFF、private canon、eval archive 都只能作为未来基础设施边界和 report-only/schema-only 设计输入，不能在本阶段变成后端、runtime agent、DeepSeek 可见知识、正式存档字段或外部框架 PoC。

本门禁只做文档与架构边界：

- 不改 runtime/source/UI/store/prompt/save。
- 不新增 save fields，不 bump `SAVE_FORMAT_VERSION`。
- 不实现后端/BFF/API proxy/private canon service/job queue/cloud save。
- 不调用 live DeepSeek，不改 DeepSeek prompt/context/model/authority。
- 不启用 subagents。
- 不做外部 framework PoC，不新增 dependency，不做 vendored subset。
- 不允许外部 SDK/agent 读写 RebornG 项目文件、执行命令、生成 patch artifact 或操作 git。
- 不新增 MiroFish export/intake。
- 不新增知识库正文、runtime canon 或 DeepSeek visible lore/RAG。
- 不开放正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。
- 不新增 `runFingerprint`。

## BFF role boundary

薄 BFF 可以被研究为未来基础设施，但不能接管游戏世界。

### 未来可评估职责

| 职责 | 允许研究的边界 | 进入实现前必须满足 |
|---|---|---|
| DeepSeek API proxy | 保护 key、集中观测模型、token、cache、retry、cost | 用户批准后端专项；不得改 DeepSeek authority |
| private canon delivery | 按 visibility 返回 source pointer 或安全摘要 | private canon schema、hidden leak gate、用户批准 |
| agent job queue | 调度离线 AgentProposal / eval jobs | report-only runner 稳定；不能写正式 save |
| eval archive | 归档 replay/live/eval/player advocate 报告 | schema 审批；不得存 hidden/private body |
| prompt hash / cost telemetry | 记录 prompt hash、模型、成本、缓存命中和 retry | 不归档原始 hidden prompt；不暴露 `.env` |
| cloud save pre-study | 只评估同步边界和安全需求 | 单独 save/cloud 决策；WorldCore 仍负责 migration |

### 明确禁止职责

| 禁止项 | 原因 |
|---|---|
| BFF 直接写 save / store / ledger | 会绕过 WorldCore 和本地 migration/defaults |
| BFF 批准奖励、地点、身份、阵营、NPC 生死、canon | 会把基础设施变成世界裁判 |
| BFF 把 hidden/private body 喂给 DeepSeek visible context | 会破坏 lore / hidden fact 边界 |
| BFF 直接调用外部 agent framework 修改项目 | 会绕过 license/SBOM/adapter/人工复核 |
| BFF 直接执行经济、战斗、掉落、价格、库存结算 | 会制造 UI/服务私有公式 |
| BFF 作为 public release / EdgeOne 自动部署触发器 | 部署仍需用户/独立发布决策 |

## private canon visibility boundary

private canon 不是“给模型更多上下文”，而是未来把原著事实、隐藏事实、NPC 私有记忆和 source pointer 管起来的可见性制度。

### 设计层级

| visibility | 含义 | 可进入 DeepSeek visible | 可进入 UI | 可进入 WorldCore | 备注 |
|---|---|---|---|---|---|
| `public_visible` | 玩家已知、公开安全事实摘要 | 是 | 是 | 是 | 需是 RebornG-owned summary |
| `player_known_safe` | 玩家当前档内已知且安全的私有化摘要 | 是，需裁剪 | 是，需按档 | 是 | 不能含 hidden body |
| `source_pointer_only` | 只有来源指针和分类，不含正文 | 否 | 可显示 ref id 的安全解释 | 是 | 默认用于 MiroFish/base-pack 后勤 |
| `private_ref_only` | 系统知道存在，但不泄露内容 | 否 | 否 | 是 | 可用于 blocker / risk check |
| `hidden_body_locked` | 隐藏事实正文或不可见因果 | 否 | 否 | 仅本地 gated check | 不能进入报告正文 |
| `agent_private_candidate` | Agent/NPC 候选记忆或反思 | 否，除非安全摘要通过门禁 | 否 | 可由 WorldCore 复核 | 不是 canon/save truth |
| `synthetic_eval_only` | 合成测试 marker | 否 | 否 | eval only | 用于 hidden leak/failure fixtures |
| `rejected` | 已拒绝或污染材料 | 否 | 否 | 否 | 只留审计原因 |

### 晋升规则

1. `source_pointer_only` 不能自动晋升为 `public_visible`。
2. `agent_private_candidate` 不能自动晋升为事实、canon、save 或 DeepSeek visible context。
3. `hidden_body_locked` 永远不能进入 UI、DeepSeek visible、Player Advocate 正文、eval report 正文或公开文案。
4. MiroFish output 必须经过 topic-slice intake 和 RebornG-owned rewrite，才能成为候选摘要或测试样本。
5. 任何 private canon service 实现前，必须先有 schema、fixture、hidden leak evaluator 和用户批准。

## eval archive / replay / cost / prompt hash boundary

eval archive 是质量制度，不是运行时权限。

### 最小 report envelope 草案

```ts
type V240EvalArchiveReport = {
  reportId: string
  version: "v2.4.0" | string
  phase: string
  runnerKind:
    | "deterministic_replay"
    | "live_probe"
    | "player_advocate"
    | "agent_lab_offline"
    | "schema_report"
  model?: string
  promptHash?: string
  sampleCount?: number
  roundCount?: number
  acceptedCount?: number
  severityCounts?: { p0: number; p1: number; p2: number }
  retryStats?: {
    retryAttemptCount?: number
    recoveredRoundCount?: number
    unrecoveredRoundCount?: number
  }
  cost?: {
    totalUsd?: number
    inputTokens?: number
    outputTokens?: number
    cacheHitRatio?: number
  }
  sourceRefs: string[]
  artifactPath: string
  boundaryAssertions: {
    wroteRuntime: false
    wroteSave: false
    exposedHiddenBody: false
    promotedCanon: false
    expandedDeepSeekAuthority: false
  }
  approvalStatus: "report_only" | "candidate" | "approved_for_gate" | "rejected"
}
```

这是设计草案，不是当前代码字段表。

### 归档禁止项

- 不归档 `.env`、API key、完整 prompt hidden/private body。
- 不归档原著正文或 MiroFish raw output。
- 不把 live probe 通过写成 runtime agent 通过。
- 不把 eval archive 作为自动 canon promotion。
- 不把 retry repair 作为 DeepSeek 获得事实裁决权。

## prompt hash / cost / retry / cache 观测口径

| 观测项 | 记录目的 | 禁止误用 |
|---|---|---|
| promptHash | 证明同一批 eval 使用同一 prompt 版本 | 不能反推或公开 hidden prompt |
| model | 记录 `deepseek-v4-flash` 等实际模型 | 不能默认切模型 |
| token / cache / cost | 评估长期成本和缓存稳定性 | 成本低不等于自动扩大 live 范围 |
| retryAttemptCount | 观察修复次数 | retry 不能掩盖 P0/P1 漏判 |
| recoveredRoundCount | 观察可修复漂移 | recovered 仍需保留原始风险分类 |
| artifactPath | 让报告可追溯 | artifact 不能保存 hidden body |

## MiroFish 与 lore 判定

v2.4-a1 保持 MiroFish `not_needed`。

原因：

- a1 只讨论 visibility 层级、BFF 职责、eval archive schema 草案。
- 不引入命名 NPC。
- 不引入 hidden-adjacent 正文或方源公开证据。
- 不产出正式 lore 结论。
- 不新增知识库正文或 runtime canon。

升级为 `blocking` 的条件：

- private canon 设计需要真实原著事实样本。
- 引入命名 NPC、原著关键人物、大势力核心或 L4/L5 事实锚点。
- hidden-adjacent、方源公开证据、正式地点/身份/奖励/NPC 生死或 canon promotion 进入 scope。
- DeepSeek visible lore/RAG 被提议进入实现。

## 通过意义

- v2.4 的 BFF / private canon / eval archive 边界已经制度化。
- 后续 a2/b1/b2/b3 有了明确的准入和禁止项。

- Hermes 或其他框架可以 PoC、安装依赖、读取项目或写补丁。
- DeepSeek 可以看到全书知识、hidden/private body 或获得事实裁决权。
