## 风险收益摘要

### 收益

- 把后端、私有事实、eval archive、外部框架、自学习拆成可审批单元，避免一口气做成高风险重构。
- 让 v2.5/v2.6 的讨论有清晰输入，而不是从愿景直接跳实现。
- 把 Hermes/LangGraph/Mastra 等强框架能力吸收为 RebornG-owned adapter 思想，保留世界内核裁决权。
- 给 hidden/private body、DeepSeek visible context、外部框架权限建立共同禁区。

### 风险

- report-only 会显得慢，但这是避免后端和 agent 过早耦合的代价。
- 如果后续 b2/b3 继续只写文档，可能推迟实际工具收益；因此每一刀都要输出明确 go/no-go。
- 外部框架的诱惑很大，尤其是 self-learning 和 autonomous tool use，必须防止它们绕过 WorldCore 和人工门禁。
- private canon 一旦设计不严，会直接破坏原著 hidden fact 和玩家探索体验。

## MiroFish 与 lore 判定

v2.4-b1 保持 MiroFish `not_needed`。

原因：

- b1 只汇总 a1/a2 report-only 边界。
- 不新增真实原著事实。
- 不引入命名 NPC、hidden-adjacent、方源公开证据或正式 lore 结论。
- 不新增知识库正文、runtime canon、DeepSeek visible lore/RAG。
- 不向外部 framework、PoC、self-learning 或 eval archive 提供 hidden/private body。

升级为 `blocking` 的条件：

- b2 需要真实原著事实样本。
- private canon 风险模型不再使用 synthetic hidden marker，而触及真实 hidden-adjacent body。
- external framework PoC 需要 lore fixture。
- 引入命名 NPC、L4/L5 fact anchor、方源公开证据、正式地点/阵营/身份/奖励/NPC 生死或 canon promotion。
- DeepSeek visible lore/RAG 或 private canon service implementation 被提议进入实现。
