- 不做 Hermes、LangGraph、Mastra、LlamaIndex、Dify、Flowise、AutoGPT、Agno、Browser-use 或其他外部 framework PoC。
- 不安装 dependency，不引入 vendored subset，不读取外部项目源码到 RebornG 运行链。
- 不允许外部 SDK/agent 读写 RebornG 项目文件、执行 shell/package/project 命令、生成 patch artifact 或操作 git。
- 不新增 MiroFish export/intake。
- 不新增知识库正文、runtime canon 或 DeepSeek visible lore/RAG。
- 不开放正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。
- 不自动部署 EdgeOne。

## External Framework Adapter Boundary

外部框架不是 RebornG 的世界内核。它们只能通过 RebornG-owned adapter 输出候选、报告或评估材料。

### 分层吸收定位

| 层级 | 框架/项目 | 当前定位 | 允许输出 | 当前禁止 |
|---|---|---|---|---|
| P0 | Hermes Agent | 架构参考；P1 隔离 PoC 候选 | 自学习循环拆解、权限边界、候选 patch 流程建议 | 直接复用源码、运行 PoC、读写项目、执行命令、写 patch |
| P0/P1 | LangGraph | 状态图、长期任务、human-in-loop 参考 | AgentProposal graph mapping、WorldCore post-check 节点建议 | 作为 runtime orchestrator |
| P0/P1 | Mastra | TS-first workflow / memory / evals / MCP 参考 | TypeScript adapter idea、eval archive schema 建议 | 安装依赖或接入 runtime |
| P1 | LlamaIndex | 知识索引和 retrieval governance 参考 | visibility / source-pointer / retrieval risk report | 全书 RAG、DeepSeek visible knowledge |
| P1 | Letta | 长期记忆参考 | memory quarantine / contamination risk report | 记忆自进化写 canon/save |
| P1/P2 | PydanticAI / OpenAI Agents SDK / Google ADK | schema、guardrail、workflow、trace 参考 | schema validator、guardrail matrix、handoff pattern | 直接替换自有 AgentProposal |
| P2 | AutoGen / CrewAI | 多 agent 协作参考 | role-message separation、orchestration risk report | NPC 聊天群式世界裁决 |
| P2 | Dify / Flowise / AutoGPT / Agno / Browser-use / OpenHands | workflow、工具权限、UI/automation 参考 | low-code/workflow/sandbox 风险报告 | 接管 WorldCore、浏览器自动玩法、外部 agent 写项目 |

### Adapter 可以输出

- `AgentProposal` 草案。
- WorldCore post-check mapping。
- failure taxonomy / eval suggestion。
- report-only framework comparison。
- license/SBOM intake report。
- self-learning candidate patch。
- future sample pool 条目。

### Adapter 不能输出

- runtime facts。
- save writes。
- canon promotion。
- formal location / faction / identity / reward / NPC life-death decisions。
- DeepSeek visible hidden/private body。
- project file writes。
- shell/package/project commands。
- patch artifact。
- git operations。
- dependency or vendored subset。

## License / SBOM Intake Checklist

此清单是未来复用源码或 SDK 前的必过入口，不等于当前批准 dependency。

| 字段 | 必填内容 | 阻断条件 |
|---|---|---|
| projectName | 项目名、repo URL、上游维护者 | 来源不明或镜像污染 |
| exactVersion | commit hash、tag、release、下载来源 | 只给 floating branch |
| license | 根 license、子目录 license、文档 license | GPL/AGPL 或不明许可证需单独法务/架构评审 |
| dependencySBOM | transitive dependencies、runtime deps、dev deps | 依赖链不可解析或含高风险许可 |
| modelProviderDeps | 是否绑定模型供应商、API key、遥测 | 默认外发数据或隐藏 key |
| dataBehavior | 输入/输出/日志/缓存/训练数据声明 | 默认上传项目文件或训练数据 |
| networkBehavior | 是否联网、访问哪些 host、是否有遥测 | 无法关闭的外网/遥测 |
| secretHandling | `.env`、token、用户存档、hidden/private body 处理 | 可读取或外传 secrets |
| sandbox | 文件、命令、浏览器、git、package manager 权限 | 默认可写项目或执行命令 |
| removalPlan | 如何卸载、隔离、回滚 | 无法清理或污染 repo |
| architectureFit | 是否能映射到 AgentProposal / WorldCore / eval archive | 只能作为黑盒 agent |

当前结论：license/SBOM intake 只能作为报告或未来门禁输入；不能据此安装依赖、复制源码或运行 PoC。

### 自学习候选晋升链

1. 外部框架或自有 runner 只能生成 candidate patch。
2. candidate patch 先进入人工 review。
3. 触及 lore、hidden、正式地点/阵营/身份/奖励/NPC 生死时升级用户决策。
4. 触及 MiroFish topic-slice 时升级 `blocking`。
5. 触及代码实现、依赖、PoC、后端、DeepSeek live 时另开版本门禁。
6. 通过后也只能进入对应 surface，例如 test matrix、future sample pool 或文档，不自动进入 runtime。

## MiroFish 与 lore 判定

v2.4-a2 保持 MiroFish `not_needed`。

原因：

- a2 只设计外部框架 adapter、license/SBOM、PoC 申请模板和 self-learning candidate schema。
- 不新增真实原著事实。
- 不引入命名 NPC、hidden-adjacent、方源公开证据或正式 lore 结论。
- 不新增知识库正文、runtime canon、DeepSeek visible lore/RAG。
- self-learning candidate patch 不允许保存 hidden/private body。

升级为 `blocking` 的条件：

- 外部框架评估需要真实原著事实样本。
- 引入命名 NPC、L4/L5 事实锚点、hidden-adjacent、方源公开证据。
- formal location/faction/identity/reward/NPC life-death/canon promotion 进入 scope。
- DeepSeek visible lore/RAG 或 private canon service implementation 被提议进入实现。

## 通过意义

v2.4-a2 通过代表：

- 外部 agent framework 进入 RebornG 前的 adapter/合规/权限边界已制度化。
- Hermes/self-learning 的强能力被收进候选链，而不是直接进入 runtime。
- 后续如要 PoC、依赖、read-only scan、patch artifact、private canon 或 BFF，都有明确申请入口。

v2.4-a2 不代表：

- Hermes 或任何框架可以安装、运行或接入 runtime。
- 自学习可以改 canon、runtime、save、prompt、知识库正文或 DeepSeek visible context。
- license/SBOM 通过就能复用源码。
- isolated PoC 已获批准。
- b1/backend/private canon/eval archive 已自动获批。
