# v3.2.0-a1 synthetic world snapshot 与 rehearsal envelope schema 门禁

日期：2026-05-24
状态：completed
冻结模型：`v320_a1_synthetic_world_snapshot_rehearsal_envelope_v1`

## schema 要点

每个 rehearsal sample 必须能声明：

- `syntheticWorldSnapshot=true`。
- `rehearsalEnvelopeVersion=v320_a1_synthetic_world_snapshot_rehearsal_envelope_v1`。
- `visibilityClass=synthetic_redacted`。
- `proposalOnly=true`。
- `expressionOnly=true`。
- `worldCoreFinalAuthority=true`。
- `requiresWorldCorePostCheck=true`。
- `persistenceMode=transient_candidate_only`。
- `noSaveFieldAdded=true`。
- Player Advocate / live metadata 明确 `liveDeepSeekCalled=false`。

## 禁止进入 schema 的内容

- 真实命名 NPC。
- 真实原著隐藏事实。
- hidden/private body。
- formal location / faction / identity / reward / NPC life-death。
- DeepSeek visible lore/RAG。
- external framework output。
- runtime write / save write。

## 验证落点

样本 `V320-OK-SNAPSHOT-ENVELOPE-001` 覆盖 snapshot/envelope/visible context。样本 `V320-REJ-REAL-SNAPSHOT-026` 验证真实事实/MiroFish 进入 snapshot 时会被拒绝。

## 结论

a1 完成。schema 只允许 synthetic/redacted/report-only envelope。
