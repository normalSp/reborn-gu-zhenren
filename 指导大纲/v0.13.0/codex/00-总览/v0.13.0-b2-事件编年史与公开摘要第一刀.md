# v0.13.0-b2 事件编年史与公开行动摘要第一刀

## 目标

b2 第一刀只回答：

哪些玩家公开行动和公开后果，可以安全压缩为玩家可见、DeepSeek 可读的公开摘要？

## 本次交付

新增：

- `src/canon/qingmao-public-event-chronicle-rules.json`
- `src/engine/v013-qingmao-public-event-chronicle.ts`
- `src/engine/v013-qingmao-public-event-chronicle.test.ts`

## 资料吸收方式

本刀使用已通过 intake 的 `qingmao_public_event_chronicle_pack`，但只吸收：

- MiroFish item id。
- source pointer id。
- category。
- candidate-only gate。

所有摘要规则均改写为 RebornG-owned 本地公开摘要，不复制原文，不使用 hidden body。

## 输出

`buildQingmaoPublicEventChronicle()` 输出：

- `events`
- `promptSafePublicSummary`
- `matchedRuleIds`
- `visibleSourceRefs`
- `forbiddenWrites`
- `intakeReviewRef`
- `statePatchApplied: false`

每条 event 都包含：

- 公开范围。
- prompt-safe 公开摘要。
- 可见来源 refs。
- 后续候选。
- 禁止升级项。
- `hiddenRefsRedacted: true`
- `canPatch: false`

## 边界

本刀不做：

- 持久化 `publicEventChronicle`。
- 新增存档字段。
- DeepSeek 新写入权。
- hidden fact body。
- 正式声望、通缉、招揽、任务、奖励、地点、阵营或 NPC 生死。
- 追击成功/失败。
- canon anchor mutation。

## 测试覆盖

已通过：

`npm test -- src/engine/v013-qingmao-npc-memory.test.ts src/engine/v013-qingmao-faction-stance.test.ts src/engine/v013-qingmao-public-event-chronicle.test.ts --reporter=dot`

- 输出不含春秋蝉、重生、隐藏因果已揭示、投靠成功、通缉生效、任务奖励已发放等越界文本。
