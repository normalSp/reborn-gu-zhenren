# v0.13.0-a2 NPC 记忆投影引擎第一刀

日期：2026-05-16
状态：第一刀完成
定位：把 a1 社会记忆协议落成一个只读 NPC 记忆投影引擎。

## 目标

第一刀只回答一个问题：

玩家的公开行动和公开后果，可能被哪些青茅主体记住、怀疑、关注或解释？

## 本次交付

新增：

- `src/canon/qingmao-npc-memory-rules.json`
- `src/engine/v013-qingmao-npc-memory.ts`
- `src/engine/v013-qingmao-npc-memory.test.ts`

## 资料吸收方式

本刀使用已通过 intake 的 `qingmao_npc_memory_motive_pack`，但只吸收：

- MiroFish item id。
- source pointer id。
- category。
- candidate-only gate。

所有 runtime 规则文本均改写为 RebornG-owned 本地摘要和规则草案。

## 运行时边界

本刀是 read-only projection：

- 不写 `npcMemories`。
- 不写 `factionPressure`。
- 不写 `actionConsequences`。
- 不写 `localActionLedger`。
- 不新增存档字段。
- 不提升 `SAVE_FORMAT_VERSION`。
- 不开放命名 NPC runtime 规则。
- 不结算关系分数、声望、招揽、通缉、任务、奖励、地点或 NPC 生死。

## 主体策略

第一刀只允许非命名主体：

- 族学同辈视线。
- 族学记录视线。
- 白家外围试探视线。
- 方源公开谨慎阴影。
- 古月内务流程视线。
- 青茅商队补给视线。
- 古月任务小组视线。
- 前山巡查视线。

`runtimeNamedNpc` 全部为 `false`。以后如果要把某人升级为命名 NPC runtime rule，必须停下来让用户决策。

## 投影输出

`buildQingmaoNpcMemoryProjection()` 输出：

- `projections`
- `matchedRuleIds`
- `visibleSourceRefs`
- `forbiddenWrites`
- `intakeReviewRef`
- `statePatchApplied: false`

每条 projection 都包含：

- 主体引用。
- 记忆轴：attention / suspicion / avoidance / interest / record_trace 等。
- 公开原因。
- 风险级别。
- 后续行动候选。
- 禁止升级项。
- `canPatch: false`。

## 测试覆盖

已通过：

`npm test -- src/engine/v013-qingmao-npc-memory.test.ts --reporter=dot`

覆盖：

- 规则和 subject allowlist 克隆，避免外部突变。
- 只有 hidden refs 而无公开证据时阻断。
- 公开状态可生成 capped read-only NPC 记忆候选。
- 方源隐藏因果 ref 不进入输出。
- 输出不包含春秋蝉、重生、投靠成功、通缉生效等越界文本。
- 单次 projection 上限可控。
