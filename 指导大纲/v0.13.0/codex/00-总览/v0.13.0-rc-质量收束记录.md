## v0.13 已关闭范围

### a1 社会记忆协议

已明确社会记忆、势力压力、公开事件摘要和后续候选的字段边界与写入权限。v0.13 第一阶段复用 v22 既有字段，不新增持久化社会账本。

### a2 NPC 记忆投影

已从公开行动、公开后果和可见压力中生成 NPC 记忆候选。输出是 read-only projection，不写 store，不开放命名 NPC runtime 规则，不泄露隐藏因果。

### b1 势力态度 / 压力投影

已让古月、白家、熊家、商队、族学 / 内务等主体对公开行动给出不同压力和机会解释。仍不开放正式声望、通缉、招揽、任务或奖励。

### b2 公开事件编年史

已建立公开行动摘要与 prompt-safe summary。它只压缩玩家可见事实和公开后果，不把 hidden body 或隐藏因果交给 DeepSeek。

### b3 后续行动候选

已把 NPC 记忆、势力态度和公开摘要转成解释、递话、遮掩、利用机会、避开视线等候选。候选不等于任务开放，不发奖励，不改变阵营。

### b4 Player Advocate UI

已在 FreeGoalPanel 中展示“社会影响 / 局势后续”：谁记住了你、势力态度、公开行动摘要、后续机会 / 风险。UI 只读消费 engine 输出，不私算关系分数。

### process-0/1/2 Git 与 CI

已建立 v0.13 Git 提交与推送计划、回滚守门职责和脏项隔离规则。远端 Actions 已能在 `codex/**` 分支跑通确定性质量门。历史工程基线已同步到当前分支。

## 剩余风险

- v0.13 仍是社会反应骨架，不是完整 NPC 人生模拟或正式势力系统。
- 命名 NPC、正式通缉、招揽、任务、声望数值、长期关系账本仍需后续版本单独决策。
- 目前社会影响 UI 是可读性第一刀，后续还需要更多玩家口感走查，确认玩家能清楚理解“影响了谁、为什么、下一步能做什么”。
- `edgeone.json` 仍固定 Node `22.11.0`；公共部署前应单独确认 EdgeOne 构建环境是否需要与 GitHub Actions 的 `22.12.0` 对齐。
- 旧 key 形状字符串已经从 HEAD 移除；如果它是真实可用 key，应在 DeepSeek 控制台轮换。

## 下一步建议

进入下一版本前需要专家团启动审查和用户方向决策。建议候选方向：

1. `v0.14.0` 推进青茅社会反应第二层：命名 NPC allowlist、关系证据更细分、正式接触前置条件。
2. 或优先做青茅主线压力链：族学、资源、三寨、狼潮前置压力与可见事件节奏。
3. 或先做部署 / 观测专项：EdgeOne Node 对齐、公开预览 smoke、错误上报、发布回滚清单。

无论选择哪条，都不应直接跳到完整通缉、正式阵营变化、NPC 生死或大世界开放。
