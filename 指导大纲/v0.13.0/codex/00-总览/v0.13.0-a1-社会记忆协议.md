# v0.13.0-a1 社会记忆协议

日期：2026-05-16
状态：第一版
定位：在写 runtime 前，冻结 NPC/势力社会记忆的流转协议、边界和门禁。

## 目标

v0.13 的社会记忆不是“关系数值系统”的第一版，而是一个可审计的解释层：

`公开行动 -> 主体解释 -> 记忆/压力候选 -> 可见摘要 -> 后续行动候选`

它要回答：

- 谁可能注意到玩家。
- 这个主体为什么会这样解释玩家行为。
- 这会形成记忆、怀疑、压力、机会还是公开摘要。
- 哪些后续行动可以让玩家处理误会、递话、遮掩、合作或避险。

## 非目标

a1 不做：

- runtime 代码。
- 新增持久化字段。
- 提升 `SAVE_FORMAT_VERSION`。
- 正式声望数值。
- 正式投靠、叛逃、通缉、招揽、任务、奖励或地点解锁。
- NPC 生死、方源轨迹、正史锚点或隐藏事实改写。
- DeepSeek 社会关系写入权。

## 资料输入

已通过 intake 的 MiroFish 包：

- `qingmao_npc_memory_motive_pack`：NPC/小组/人物动机、记忆触发、怀疑与利益轴候选。
- `qingmao_faction_reputation_pressure_pack`：势力压力、封锁、招揽、通缉、任务来源候选。
- `qingmao_public_event_chronicle_pack`：公开事件、公开行动摘要、事件可见性候选。

这些包只能作为候选材料。RebornG 必须把它们重写为本地规则草案、测试样本或阶段候选，不能把 MiroFish 原始 item 当 runtime 真相。

## 核心流程

### 1. 行动来源

社会记忆只消费 RebornG 本地已经结算的公开行动和公开后果：

- `livingWorldState.knownFacts`
- `livingWorldState.actionConsequences`
- `livingWorldState.localActionLedger`
- `livingWorldState.factionPressure`
- 已过滤的 `lastWorldActionReturnContext`

不得消费：

- DeepSeek 未验证正文。
- MiroFish 原始包的 hidden body。
- 玩家未经历、未公开、未可见的原著事实。
- UI 临时推断。

### 2. 主体解释

每个主体必须先过主体门禁：

- subject 是否在当前阶段允许。
- subject 是否可能知道该行动。
- subject 是否处于当前时间线和区域压力范围内。
- subject 是否只需要匿名聚合表达，而不是命名 NPC。

主体可以是：

- 命名 NPC 候选。
- 小组或身份层，如族学同窗、内务执事、巡山队。
- 势力，如古月、白家、熊家、商队。

a1 只定义协议；a2/b1 再决定哪些 subject 进入 runtime rule draft。

### 3. 候选输出

社会记忆输出分三类：

- `npc_memory_candidate`：某主体可能记住、怀疑、感激、避让或试探玩家。
- `faction_pressure_candidate`：某势力可能产生压力、机会、封锁、观察或递话窗口。
- `public_event_summary_candidate`：某公开行动可被压缩为公开摘要，供玩家回看或 DeepSeek 叙事使用。

候选不等于正式结论。候选必须带原因、来源、可见性、风险和推荐下一步。

### 4. 可见摘要

玩家可见摘要只能表达公开层：

- “族学有人记住了你的缺席。”
- “白家方向出现可试探的递话窗口。”
- “内务对你的山道行动留下记录。”

不得表达：

- “某隐藏事实已被触发。”
- “方源因为重生所以改变行动。”
- “白家正式招揽成功。”
- “古月山寨已正式通缉你。”

### 5. 后续行动

后续行动候选只提供选择入口：

- 解释误会。
- 找中间人递话。
- 避开视线。
- 补交公开理由。
- 利用临时窗口。

候选不发奖励，不变阵营，不开放新地点，不写 NPC 生死。

## 强制门禁

| 门禁 | 作用 | 不通过时 |
|---|---|---|
| Source gate | 输入必须来自本地已结算公开状态或 intake 后候选 | 丢弃 |
| Visibility gate | 玩家/主体必须可能知道该事实 | 降级为 hidden ref 或丢弃 |
| Timeline gate | 不允许未来章节或未到阶段事件影响当前 | 延期 |
| Subject allowlist gate | 只允许本阶段主体进入投影 | 匿名化或延期 |
| Player-action gate | 必须由玩家行为或公开局势触发 | 不生成个人化记忆 |
| Hidden-fact gate | hidden body 不得进入 UI/DeepSeek | 只保留 hidden ref |
| Escalation gate | 通缉、招揽、任务、阵营变更需单独决策 | 只输出压力/机会候选 |
| Duplicate gate | 重复行为不得无限堆记忆 | 合并或刷新 |

## DeepSeek 边界

DeepSeek 可消费：

- 本地过滤后的公开摘要。
- 本地生成的候选后续行动描述。
- 不含 hidden body 的压力提示。

DeepSeek 不可：

- 读取 MiroFish 原始包。
- 写 `npcMemories`、`factionPressure`、`actionConsequences`。
- 判定声望、阵营、任务、奖励、地点、NPC 生死。
- 暴露隐藏事实或方源私密因果。
