## Git 真相源规则

Git 状态以本地命令输出为准：

- `git status -sb`
- `git status --short`
- `git log --oneline --decorate --all -n 12`
- `git remote -v`

Git 制度以以下文件为准：

- `指导大纲/v0.11.0/codex/00-总览/v0.11.0-Git提交与推送制度.md`
- `指导大纲/v0.13.0/codex/00-总览/v0.13.0-Git提交与推送计划.md`
- `指导大纲/v0.13.0/codex/00-总览/v0.13.0-process-0-Git安全收束.md`

若聊天记忆、旧 handoff 或 GitHub 页面与本地状态冲突，以本地 git 命令和上述制度文件为准。

## DeepSeek 规则

DeepSeek 可：

- 解释本地社会记忆结果。
- 生成候选对话、传闻、压力表达。
- 总结公开行动摘要。
- 用本地返回的公开事实 ids 和 summary 写叙事。

DeepSeek 不可：

- 直接改变 NPC 关系。
- 直接改变势力声望。
- 直接开放任务、奖励或地点。
- 直接判定通缉、追杀、抓捕或 NPC 生死。
- 暴露 hidden fact body。

## 玩家可见规则

玩家可见摘要必须避免：

- hidden fact id/body。
- 方源隐藏因果、春秋蝉、重生/回溯。
- “正式通缉已生效”“投靠成功”“阵营已变更”等未开放结论。
- UI 自行推断的关系数值。

推荐表达：

- “某人对你的缺席有了印象。”
- “白家方向出现试探窗口。”
- “内务/族学对你近期行为保留记录。”
- “商队渠道可能接受递话，但需要公开理由或代价。”
