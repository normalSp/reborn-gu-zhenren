# v0.13.0-process-1 GitHub Actions 远端门禁复核

日期：2026-05-16
状态：第一刀完成，远端 CI 已触发但失败
MiroFish：`not_needed`，本阶段只做工程流程，不吸收 canon/IF/NPC/route/hidden-fact 数据。

## 目标

把 v0.12 已建立的 GitHub/CI 门禁真正纳入当前 v0.13 分支，并验证远端 GitHub Actions 能跑起来。

## 本轮发现

执行 `gh run list --branch codex/v013-npc-faction-reaction --limit 5` 时没有返回记录。

原因：

- `.github/` 仍在本地未跟踪状态，尚未进入当前分支。
- `ci.yml` 原先只监听 `main` / `master` push；当前开发分支推送不会触发 CI。

这不是运行时代码问题，但会削弱“每个小版本可回滚、可远端验证”的流程目标。

## 本次交付

纳入并复核 `.github` 门禁文件：

- `.github/workflows/ci.yml`
- `.github/pull_request_template.md`
- `.github/ISSUE_TEMPLATE/phase_task.yml`
- `.github/ISSUE_TEMPLATE/bug_report.yml`
- `.github/ISSUE_TEMPLATE/config.yml`

调整：

- `ci.yml` 的 push 分支增加 `codex/**`，用于当前 Codex 分支远端质量门触发。
- PR 模板补充 NPC/faction/social-memory 边界。
- PR 模板补充 `npm run check:player-visible-copy` 验证项。
- Phase issue 模板补充 `NPC/faction/social memory` 与 `Git/rollback` 门禁。

## CI 默认门

远端默认质量门仍只跑确定性检查：

- `npm ci`
- `npx tsc --noEmit --pretty false`
- `npm test -- --reporter=dot`
- `npm run build`
- `npm run check:runtime-assets`
- `npm run check:qingmao-assets`
- `npm run check:player-visible-copy`

Playwright smoke 仍为手动：

- `workflow_dispatch run_e2e=true`

## 边界

本刀不做：

- branch protection。
- auto merge。
- EdgeOne 自动部署。
- PR 默认强制全量 Playwright / long e2e。
- runtime 依赖变更。
- save format 变更。
- DeepSeek 权限变更。
- MiroFish 数据吸收。

以上内容如要启用，必须用户另行决策。
