## 背景

`v0.13.0-process-1` 已经把 `.github` 门禁纳入当前分支，并成功触发 GitHub Actions。

远端 run `25962959025` 失败在 `Type check`，但本地同阶段的 `tsc`、unit、build 和 v0.13 e2e 已通过。判断不是 v0.13 社会反应 UI/engine 的逻辑失败，而是远端分支缺少本地长期历史成果：依赖、类型声明、TS 配置、旧阶段源码、测试存档、运行时资产与文档基线没有完整进入 Git。

因此本专项目标是把“本地已验证的当前工程基线”按可审查边界纳入当前 v0.13 分支，让远端 CI 重新拥有与本地一致的输入。
