## 提交前硬检查

每次分批提交前：

1. 运行 `git status --short`。
2. 列出本次 stage 文件清单。
3. 确认没有密钥、`.env`、runtime state、SQLite/log、插件缓存、大体积 zip。
4. 确认提交边界对应一个阶段，不跨多个大版本混写。
5. 确认必要验证已经完成，或明确这是本地 checkpoint 而非阶段完成。
6. 更新当前版本项目仪表盘。
7. 更新上下文交接中的 commit / push 状态。
