# v0.13.0 专家团启动会纪要

日期：2026-05-16
状态：主线、第一刀和前两项 MiroFish 请求已获用户批准
会议主题：`NPC 与势力反应系统`

## 会议结论

v0.13 的核心不是“多加几个 NPC 文案反应”，而是建立社会记忆系统的第一层地基。

专家团建议把本版本收束为：

`行动后果 -> 社会解释 -> 记忆/压力入账 -> 玩家可见摘要 -> 后续行动候选`

这能把 v0.12 的事实卡、IF 矩阵、路线压力、势力反应桥，推进成更像活世界的长期反馈。

## 角色意见

### 项目总策划

v0.13 要做“世界里有人在看你”。玩家不必马上看到完整声望条，但应该看到：

- 谁注意到了我。
- 为什么他这样反应。
- 这件事以后还会不会影响我。
- 我能做什么修复、利用或扩大这个影响。

### 世界观审校

青茅阶段的反应必须低阶、地方化、公开可见。
不能把后期大势力、仙蛊、高阶追杀、尊者级因果提前塞进低阶山寨。

隐藏事实继续保持：

- hidden fact 只能本地 ref。
- 方源隐藏因果、春秋蝉、重生/回溯不得进入玩家可见摘要。
- NPC 的怀疑可以存在，但不能写成“知道隐藏事实”。

### 系统架构师

建议先复用 v22 已有字段：

- `livingWorldState.npcMemories`
- `livingWorldState.factionPressure`
- `livingWorldState.actionConsequences`
- `livingWorldState.localActionLedger`
- `livingWorldState.playerGoals`

第一阶段不急着新增 `relationshipLedger` 或 `factionStandingLedger`。
如果后续必须新增稳定关系账本，再单独提升 `SAVE_FORMAT_VERSION` 并补迁移、默认值和长测。

### AI Pipeline 专家

DeepSeek 在 v0.13 只能：

- 解释本地社会记忆结果。
- 写候选对话、传闻、压力表达。
- 总结玩家公开行动摘要。

DeepSeek 不得：

- 改声望数值。
- 直接招揽玩家入势力。
- 发任务奖励。
- 判定通缉、抓捕、NPC 生死。
- 暴露隐藏事实。

### Player Advocate

v0.13 必须有“玩家看懂自己影响了什么”的走查。
只在日志里写 `factionPressure +1` 不够，玩家需要看到：

- 影响对象。
- 原因。
- 严重程度。
- 后续可行动方向。

### 前端与动效负责人

v0.13 先做可读社交反馈 UI，不做大场面。
推荐界面：

- `谁记住了你`
- `势力态度`
- `公开行动摘要`
- `后续机会 / 风险`

动效只做轻量状态变化，不做奖励闪光、通缉演出或阵营归属大动画。

### QA / 测试工程专家

测试重点：

- 同一玩家行为让不同 NPC/势力产生不同解释。
- 隐藏事实不泄露。
- 记忆不会重复刷。
- 长档里旧行为仍能影响后续推演。
- UI 展示的原因来自 engine，不是 UI 私算。

## 专家团推荐小版本

1. `v0.13.0-a0` 启动审查与范围冻结。
2. `v0.13.0-a1` 社会记忆协议、字段表、测试矩阵。
3. `v0.13.0-a2` NPC 记忆投影引擎第一刀。
4. `v0.13.0-b1` 势力态度 / 压力投影第一刀。
5. `v0.13.0-b2` 事件编年史与公开行动摘要。
6. `v0.13.0-b3` 后续行动候选：修复关系、利用机会、规避风险。
7. `v0.13.0-b4` Player Advocate 可读性走查 UI。
8. `v0.13.0-process-1` GitHub Actions 远端首次运行与门禁复核。
9. `v0.13.0-rc` 质量收束。

## 用户已决策

1. 批准 v0.13 主线为 `NPC 与势力反应系统`。
2. 批准第一刀先做设计门禁，不先写 runtime。
3. 批准第一阶段尽量复用 v22 字段，新增持久化关系账本另行决策。
4. 批准先请求 NPC 动机/记忆包与势力压力/声望包。
