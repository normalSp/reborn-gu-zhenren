# v0.13.0 总体开发大纲

日期：2026-05-16
状态：主线与第一刀已获用户批准
主线：`NPC 与势力反应系统`

## 版本目标

v0.13 的目标是让青茅世界开始具备“社会记忆”。

玩家公开行动不再只是写一条结果日志，而应被不同 NPC/势力按自身立场解释：

- 有人觉得你可用。
- 有人觉得你可疑。
- 有人觉得你越界。
- 有人因为利益而试探你。
- 有势力因为公开行为给你机会、限制或压力。

## 核心问题

本版本要回答：

- NPC 记住什么，不记住什么。
- 势力如何把玩家行为归类为机会、怀疑、风险、资源争夺或任务痕迹。
- 同一行为为什么会让不同主体产生不同反应。
- 玩家如何看懂自己影响了什么。
- DeepSeek 如何解释这些影响，而不是硬记全部历史。

## 核心交付

1. 社会记忆协议：NPC memory、faction pressure、action consequence、public event summary 的字段边界。
2. NPC 记忆投影引擎：从公开行动、已知事实和后果中生成记忆候选。
3. 势力态度 / 压力投影引擎：从行动账本和压力源生成机会、怀疑、封锁、招揽、通缉、任务来源候选。
4. 事件编年史和公开行动摘要：替代 DeepSeek 长上下文硬记。
5. 后续行动候选：修复关系、利用机会、规避风险、递话、解释、遮掩。
6. Player Advocate 走查 UI：玩家能看懂谁记住了什么、为什么。
7. 测试矩阵：重复行为、防刷、隐藏事实泄露、不同主体差异反应、长测回归。

## 专家团收束

v0.13 不直接等于完整 NPC 社会模拟。

本版本只允许“候选和压力”先行：

- 招揽可以是 `recruitment_opportunity_candidate`，不是加入成功。
- 通缉可以是 `wanted_pressure_candidate`，不是正式通缉生效。
- 任务来源可以是 `task_source_hint`，不是奖励任务开放。
- 声望可以是 `stance/pressure projection`，不急着公开成正式数值。

正式阵营身份、任务奖励、NPC 生死、追捕结果和公开声望数值，需要单独用户决策。

## 非目标

- 不做完整声望条和声望商店。
- 不做正式通缉/追杀结算。
- 不做正式阵营身份变化。
- 不做完整任务网络。
- 不开放新地域。
- 不发新奖励、蛊虫、蛊方或材料池。
- 不让 DeepSeek 直接写 NPC 关系、势力声望、任务、地点或 NPC 生死。
- 不引入后端。

## 完成标志

- 同一玩家行为至少能被两个不同 NPC/势力解释成不同反应。
- 玩家能看到“谁记住了你、为什么、后果是什么、下一步能做什么”。
- 长期撒谎、偷窃、投靠、救援、背叛等行为进入可回放账本或可推导摘要。
- Hidden fact 不进入 UI 或 DeepSeek visible context。
- 记忆和压力写入必须通过受控 patch。
- 测试覆盖重复刷、隐藏泄露、不同主体差异和长档稳定。

## 风险池

- 过早公开数值声望会让系统像普通 RPG 条形表，而不像活世界。
- 命名 NPC 反应如果没有原著依据，容易跑偏。
- 通缉/招揽/任务来源如果直接开放，会牵涉阵营身份、奖励和剧情分支。
- 新增持久化关系账本会影响存档版本，必须单独决策。
- UI 如果只展示内部字段，会让玩家看不懂自己影响了世界。
