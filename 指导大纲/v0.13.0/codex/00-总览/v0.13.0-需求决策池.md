# v0.13.0 需求决策池

日期：2026-05-16
状态：主线、第一刀、v22 字段策略、Git 安全收束和前两项 MiroFish 请求已获用户批准

## 候选需求

### D-013-001 v0.13 主线：NPC 与势力反应系统

状态：已批准。

内容：

- 建立 NPC/势力社会记忆地基。
- 同一玩家行为可被不同主体不同解释。
- 玩家能看到自己影响了谁、为什么、后续可能怎样。

建议：批准。

### D-013-002 第一刀先做社会记忆协议，不先写 runtime

状态：已批准。

内容：

- `v0.13.0-a1` 先做字段表、权限表、测试矩阵、MiroFish 门禁。
- 避免一上来写出难迁移、难测试、难解释的反应系统。

建议：批准。

### D-013-003 优先复用 v22 字段

状态：已批准。

内容：

- 第一阶段复用 `npcMemories`、`factionPressure`、`actionConsequences`、`localActionLedger`。
- 不急着新增 `relationshipLedger`、`factionStandingLedger`、`eventChronicleState`。

正面影响：

- 降低存档迁移风险。
- 更快验证社会记忆口感。

风险：

- 字段可能不够表达复杂长期关系。
- 如果 b2/b3 证明不足，需要升级 D-013-008。

建议：批准。

建议：批准为优先包。进入命名 NPC runtime 前应视为 blocking。

### D-013-010 GitHub Actions 远端首次运行与门禁复核

状态：作为 process sidecar 保留；执行远端 CI/branch policy 前复核。

内容：

边界：

- 不启用 branch protection。
- 不自动发布。

### D-013-011 Git 安全收束与每大版本提交计划

状态：已批准。

内容：

- 每个大版本必须建立本版本 Git 提交与推送计划。
- 每个小版本完成后必须更新仪表盘、交接和 commit / push 状态。
- `v0.13.0-process-0` 先建立 Git 安全收束，不直接做 runtime。
- 新增 `Git / 回滚守门人` 轻量职责，由 QA / Release 与 Codex Safety 共同承担。

边界：

- 不新增会写代码的独立专家。
- 不自动启用 branch protection。
- 不自动部署 EdgeOne。
- 不重写历史。
- 不使用 `git add -A` 混提交历史脏项。

建议：批准，作为 v0.13 进入 a1 前的前置流程刀。
