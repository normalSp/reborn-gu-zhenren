## a0 启动审查与范围冻结

目标：

- 确定 v0.13 主题和非目标。
- 列出 MiroFish 包需求。
- 明确哪些内容必须停下来让用户决策。

交付：

- 当前目录全套规划文档。
- MiroFish 请求草案。

## a1 社会记忆协议、字段表、测试矩阵

目标：

- 先定义社会记忆怎么流动，不先写 runtime。

交付：

- NPC memory 字段边界。
- Faction pressure / stance 字段边界。
- Public event summary 字段边界。
- Controlled patch 写入权限表。
- 测试矩阵：重复行为、防刷、隐藏事实、差异反应、长档稳定。

MiroFish：

- `preferred`。可以先做设计，但 runtime 命名 NPC 规则前需要包。

## process-0 Git 安全收束

目标：

- 在进入 v0.13 runtime 或继续堆工作前，先恢复可回滚、可审查、可推送的工程节奏。

交付：

- `v0.13.0-Git提交与推送计划.md`。
- `v0.13.0-process-0-Git安全收束.md`。
- `Git / 回滚守门人` 作为专家团轻量职责写入 Skill / AGENTS / PROJECT-STATE。
- 项目仪表盘和上下文交接记录当前 commit / push 状态。

边界：

- 不改 runtime。
- 不启用 branch protection。
- 不自动部署 EdgeOne。
- 不用 `git add -A` 一口气提交历史脏项。

## a2 NPC 记忆投影引擎第一刀

目标：

- 从公开行动和后果中推导 NPC 记忆候选。

交付：

- 纯 TS read-only projection engine。
- 输入：`knownFacts`、`actionConsequences`、`localActionLedger`、`factionPressure`。
- 输出：memory candidates、reason、visibility、risk、recommended follow-up。
- 不写 store，不加字段，不发奖励。

MiroFish：

- `qingmao_npc_memory_motive_pack`。
- 如果只做无名主体/通用规则可 `preferred`。
- 如果做命名 NPC 动机和反应，升级为 `blocking`。

## b1 势力态度 / 压力投影第一刀

目标：

- 让不同势力对同一公开行为有不同态度和压力。

交付：

- 古月山寨、白家、熊家、商队、族学/内务等主体的 pressure / opportunity projection。
- 输出只作为压力、机会、阻断、任务来源候选。
- 不开放正式阵营、声望数值、通缉结算或任务奖励。

MiroFish：

- `qingmao_faction_reputation_pressure_pack`。
- 若写正式 rule draft，建议 `blocking`。

## b2 事件编年史与公开行动摘要

目标：

- 让系统能压缩玩家历史，不依赖 DeepSeek 长上下文硬记。

交付：

- Public event chronicle read helper。
- Action summary builder。
- DeepSeek prompt-safe summary：只含公开事实、可见后果、压力摘要，不含 hidden body。

MiroFish：

- `qingmao_public_event_chronicle_pack` preferred。

## b3 后续行动候选

目标：

- 把记忆/压力转成可玩的下一步。

候选：

- 解释误会。
- 递话试探。
- 遮掩痕迹。
- 利用商队窗口。
- 修复小组信任。
- 避开某势力视线。

边界：

- 候选不等于任务开放。
- 候选不发奖励。
- 候选不直接改变阵营或声望。

## b4 Player Advocate 可读性走查 UI

目标：

- 玩家看得懂自己影响了什么。

交付：

- `谁记住了你`。
- `势力态度`。
- `公开行动摘要`。
- `后续机会 / 风险`。

边界：

- UI 只消费 engine 输出。
- UI 不私算关系分数。
- UI 不显示 hidden fact id/body。

## process-1 GitHub Actions 远端首次运行与门禁复核

目标：

- 验证 v0.12 建立的 GitHub Actions 在远端真实可跑。
- 复核 PR 模板是否覆盖 v0.13 的 NPC/势力风险。

是否需要用户决策：

- 只记录远端运行结果：不需要。
- 启用 branch protection、自动发布、强制 PR 必跑全量 Playwright：需要用户决策。

## rc 质量收束

最低验证：

- `npm test -- --reporter=dot`
- `npx tsc --noEmit --pretty false`
- `npm run build`
- `npm run check:runtime-assets`
- `npm run check:qingmao-assets`
- `npm run check:player-visible-copy`
- `npm run check:production-preview`
- FreeGoalPanel / v0.13 社会记忆相关 e2e
- 长测 e2e

## 必停决策点

- 新增持久化关系账本。
- 正式声望数值公开。
- 正式阵营身份变化。
- 正式通缉/追杀/抓捕。
- 任务来源变成正式任务和奖励。
- NPC 生死或关键原著轨迹变化。
- DeepSeek 新写入权。
- 后端/BFF、外部 runtime 依赖、branch protection、自动发布。
