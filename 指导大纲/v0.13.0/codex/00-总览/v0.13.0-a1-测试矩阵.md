# v0.13.0-a1 测试矩阵

日期：2026-05-16
状态：第一版
定位：给 v0.13 社会记忆、NPC/势力投影、公开摘要建立可工程化回归标准。

## 测试目标

v0.13 测试不只测“有没有输出”，还要测：

- 输出是否来自本地公开状态。
- hidden facts 是否保持不可见。
- MiroFish 候选是否只作为候选。
- 同一行为是否会被不同主体不同解释。
- 重复行为是否防刷。
- DeepSeek 是否仍无写入权。

## 定量门槛

| 指标 | 门槛 |
|---|---|
| MiroFish 原始包进入 runtime | 0 |
| hidden body 进入 UI/DeepSeek | 0 |
| 未过 visibility/timeline gate 的候选输出 | 0 |
| a1/a2 第一刀新增 save field | 0 |
| `SAVE_FORMAT_VERSION` 变化 | 0 |
| 正式声望/通缉/招揽/任务/奖励结论 | 0 |

| 重复同类行为合并率 | 必须可测，不得无限叠加 |

## 测试层级

### 单元测试

适合 a2/b1/b2：

- social memory candidate builder。
- subject allowlist。
- visibility/timeline gate。
- duplicate merge。
- hidden-fact filter。
- DeepSeek prompt-safe summary filter。

### Store 测试

仅在后续受控写入时启用：

- 所有写入必须走 `applyLivingWorldPatch()` 或批准 action。
- UI/DeepSeek/narrative text 不能直写。
- 存档迁移和 hydration 保持稳定。

### E2E 测试

适合 b4/rc：

- 玩家能看到“谁记住了你”。
- 玩家看不到 hidden facts。
- 长描述不遮挡按钮。
- 移动端可读。
- reduced motion 不影响理解。

### 长测

适合 rc：

- 多轮公开行动后社会记忆不爆炸增长。
- 旧档迁移不丢 livingWorldState。
- DeepSeek prompt 只包含公开摘要。

## 回归绑定

每次修复下列问题，必须更新 `.learnings/ERRORS.md` 或本矩阵：

- hidden fact 泄露。
- MiroFish 候选被误当真相。
- UI 自己计算关系/声望。
- 重复行为刷出无限记忆。
- DeepSeek 文案被当作状态写入。
- 玩家无法理解 NPC/势力为什么反应。
