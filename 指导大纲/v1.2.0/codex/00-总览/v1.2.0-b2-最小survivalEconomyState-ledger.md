## 结论先行

用户已批准 b2 执行 `SAVE_FORMAT_VERSION = 24`、新增单一聚合对象 `survivalEconomyState`、并落地最小 pressure ledger。

本阶段只把 b1 的低阶生存经济压力投影登记成可迁移、可归一化、可回滚的压力账本。它不开放正式库存、正式货币、价格表、交易结算、食料真实消耗、维护周期、炼蛊成功/失败、黑市、委托、稳定套利、地点开放、阵营变化、NPC 生死或 DeepSeek 新权限。

## 字段表

| 字段 | 类型 | 说明 |
|---|---|---|
| `schemaVersion` | `1` | survival economy 子结构版本 |
| `status` | `not_started / pressure_tracked / blocked` | 当前 ledger 状态 |
| `authority` | `migration_default / survival_economy_engine` | 仅本地 engine 可写 |
| `pressureScore` | `0-100` | 压力强度摘要，非数值资源 |
| `ledger[]` | array | 最多 24 条压力记录 |
| `ledger[].category` | route/feeding/refinement/market/gray/anti-farm | 压力类别 |
| `ledger[].blockedWrites` | string[] | 本阶段禁止写入项 |
| `evidenceRefs` | string[] | 可见证据引用 |
| `sourceRefs` | string[] | 规则/文档/样本来源 |
| `lastUpdatedAtTurn` | number/null | 最后登记回合 |
| `migrationNote` | string optional | 旧档或异常归一化说明 |
