状态：已按用户批准执行复核与门禁加固

## 结论先行

用户批准根目录 BGM 音频做一次正式 runtime 晋升专项。本次复核确认：运行时只引用 `public/audio/...`，根目录 `bgm/` 仍是本地 fan-pack 暂存层，不作为 runtime 路径。

现有 `src/canon/audio-source-manifest.json` 与 `src/canon/character-bgm-manifest.json` 已把用户提供曲目映射到 `public/audio/bgm/...`；本专项把 audio manifest 引用纳入 `npm run check:runtime-assets`，并同步音频说明文档，避免后续出现“manifest 说启用但 public 文件缺失”的隐性坏档。

## 运行时边界

1. runtime 音频路径必须是 `public/audio/...`。
2. `src/canon/audio-source-manifest.json` 是源码真相源。
3. `public/audio/audio-source-manifest.json` 是发布查看镜像。
4. 根目录 `bgm/v0.7/*.mp3` 是用户本地来源暂存，不直接进 runtime。
5. 不自动抓取 B 站、网易云或其他平台音频。
6. AI/FFmpeg/synthetic BGM 占位继续禁止。
