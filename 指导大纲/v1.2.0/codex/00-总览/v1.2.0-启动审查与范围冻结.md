## 用户输入分类

- `方向输入`：开始开 v1.2 专家团启动会。
- `边界输入`：继续遵守 v1.1 后的制度化开发流程；不要自动部署 EdgeOne；不要擅自加存档字段、DeepSeek 权限、正式地点/阵营/奖励/NPC 生死。
- `优先级输入`：围绕长期愿景和专家团讨论推进，不急着堆功能。
- `质量输入`：长线叙事漂移、世界观漂移、MiroFish/知识库治理和测试制度必须成为开发流程的一部分。

## 版本定位

`v1.2.0` 是 `v1.x` 的第二块地基：在 `v1.1.0` 建立路线/地点/区域状态后，让低阶蛊师的行动开始受到更稳定的资源、生存、维护和交易窗口约束。

冻结主线：

`玩家目标 -> 路线/地点状态 -> 资源与食料缺口 -> 本地 engine 判断可做准备 -> UI 显示缺口和下一步 -> DeepSeek 只写压力与叙事 -> 反刷与漂移测试`

2. bump `SAVE_FORMAT_VERSION = 24`。
3. 新增 `survivalEconomyState`。
4. 开正式材料库存、货币账本、价格表、商店库存、买卖结算。

8. 让 DeepSeek 写材料、元石、奖励、蛊虫、蛊方、炼成/失败、价格、交易结果。
9. 让 MiroFish 或全书基础包直接成为 canon/runtime/DeepSeek authority。
10. 自动部署 EdgeOne 或写对外发布承诺。
