# v1.2.0 MiroFish 资料需求与交付协议

状态：已同步至 b1
日期：2026-05-20

## 当前结论

v1.2 启动阶段不需要新的 blocking MiroFish 包。启动阶段分类为：

`preferred`

b1 projection-only 阶段分类为：

`not_needed`

理由：

- v0.15 三包已经覆盖低阶经济、炼养用、商队市场窗口和灰色交易边界。
- 全书基础包已经位于 `指导大纲/vMiroFish/基础包/`，但只作为档案层和 source-pointer inventory。
- v1.2 a1/a2 已把历史候选和测试样本整理为 RebornG-owned 规则草案。
- b1 不做正式价格、库存、交易、黑市、委托、商家城、高阶资源或 runtime canon 晋升，因此不需要新包。

## 可复用来源

| 来源 | v1.2 用途 | 限制 |
|---|---|---|
| v0.15 low-rank economy/refinement/feeding pack | 食料、蛊材、残方、炼养用压力 | 不直接进 runtime |
| v0.15 caravan/trade pack | 商队窗口、询价、担保、公开理由 | 不开放成交/价格表 |
| v0.15 black-market/commission boundary pack | 黑市、委托、代售风险样本 | 只做边界和 future sample |
| 全书基础包 | source pointer、主题切片候选 | 不整包吸收 |

## 需要 topic-slice intake 的情况

以下情况必须先做 v1.2 topic-slice intake review：

1. 将某个全书基础包条目转写为知识库索引。
2. 将某个条目晋升为测试样本。
3. 将某个条目晋升为 RebornG-owned rule draft。
4. 将某个条目提供给 DeepSeek visible context。
5. 将某个条目进入 `src/canon/*.json`。

第 4 和第 5 项还需要当前版本 gate 和用户决策。

## 升级为 blocking 的条件

如果 v1.2 要推进以下任何事项，MiroFish 资料需求升级为 `blocking`：

- 正式库存或货币账本。
- 正式价格表。
- 正式交易结算。
- 商队库存或商店 UI。
- 黑市买卖。
- 委托、代售或稳定收益。
- 商家城或完整南疆市场。
- 高阶蛊材、宝黄天、仙道资源。
- 任何原著隐藏事实可见化。

## forbidden uses

MiroFish output 和全书基础包不得：

1. 直接进入 DeepSeek visible context。
2. 直接作为 canon truth。
3. 直接作为 runtime authority。
4. 直接给玩家展示 hidden/private 内容。
5. 直接产生地点、阵营、奖励、NPC 生死、交易、价格、材料、蛊虫或蛊方结算。
