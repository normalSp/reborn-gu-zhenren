# v1.2.0 专家团启动会纪要

状态：已召开启动会；D-120/D-121 已批准；a1/a2 过门；b1 projection-only 第一刀已完成

## 会议目标

确定 `v1.2.0` 是否可以启动、启动后先做什么、哪些事情必须停下来让用户决策。

## 输入来源

- `v1.1.0` 已完成本地开发里程碑：`SAVE_FORMAT_VERSION = 23`、`routeLocationState`、T0 deterministic soak、180 轮 Player Advocate、D-025 C27 后小规模 live drift 阻塞门通过。
- `v0.15.0` 已完成低阶经济、补给、喂养、炼养用和商队/灰色交易边界样本，但没有开放完整库存、市场、价格、委托、交易结算或 DeepSeek 经济权限。
- `v1.0.0` 已完成 public release prep，EdgeOne 部署仍由用户/CodeBuddy 手动负责。
- `v1.1.0-process-2` 已建立长线叙事漂移与全书知识库治理制度。
- 全书 MiroFish 基础包已入档案层，只能按主题切片 intake，不能整包吸收。

## 专家意见

### Project Lead

v1.2 可以进入启动审查，但不能把“经济正式化”理解为一次性做完整经济系统。v1.2 应先把低阶生存压力做稳，把玩家为什么缺资源、为什么需要维护蛊虫、为什么不能随便买卖高阶资源解释清楚。

### Lore & World Designer

低阶蛊师经济必须符合蛊真人世界的残酷资源压力。食料、蛊材、残方、商队、身份担保和灰色渠道可以作为压力来源，但不能让玩家通过一句话绕到宝黄天、高阶资源、商家城正式市场、黑市稳定套利或隐藏因果。

### Systems Architect

最大风险是 save schema 和经济漏洞。正式库存、货币、价格表、交易结算、消耗、炼蛊失败代价和维护周期都是强状态系统。a1 必须先做 authority table 和 save-format gate，不能在 b1 顺手加字段。

### AI Pipeline Architect

D-025 的 clean-final 只证明 C27 后小规模 live drift 阻塞门通过，不证明长线 live narrative quality 已全量验证。v1.2 经济叙事必须继承 hidden-name echo、奖励越权、价格越权、交易越权、正式地点越权等样本。live probe 仍需另批。

### Player Advocate

玩家需要看到“缺口”和“下一步”，而不是只看到被系统拒绝。v1.2 第一刀应让补给、食料、维护和交易窗口成为可读压力：告诉玩家能做什么准备、缺什么、为什么不能直接成交。

### QA/Test Engineering Guardian

v1.2 是高风险 player-facing/runtime 版本。建议 runtime 小版本至少 20 轮 Player Advocate；rc 默认 60 轮，若新增持久经济字段或真实交易/消耗，建议提升到 80 轮或更高，并追加 deterministic drift soak。

### Git/rollback Steward

v1.2 必须继续窄提交边界。a0/a1/a2 只提交文档和测试矩阵；runtime 阶段再按 canon -> schema -> engine -> store -> UI -> tests/docs 分段提交。不要 `git add -A`。

## 会议共识

1. v1.2 主线建议定为：低阶蛊师生存与经济正式化第一阶段。
2. v1.2 第一优先级不是完整市场，而是低阶生存压力、补给缺口、蛊虫喂养/维护、炼养用边界和交易窗口解释。
3. a0/a1/a2 必须在 runtime 前完成。
4. a1 后必须由用户拍板 save-format、字段、第一刀 runtime 范围和测试强度。
5. v1.2 不默认开放正式库存、正式价格、正式买卖、正式黑市、正式委托、商家城、稳定套利或任何 DeepSeek 经济结算权限。
6. v1.2 需要继承 v1.1 D-025 的 live drift 样本，尤其是 hidden-name echo、术语边界、DeepSeek 越权和长线叙事漂移。
