3. b1 不 bump `SAVE_FORMAT_VERSION = 24`。
4. `survivalEconomyState` 在 b1 仅作为 b2 可能方案；b2 已批准落地为最小 pressure ledger。
5. b1 不允许真实库存、食料消耗、维护周期、炼蛊失败代价。

7. v1.2 live DeepSeek drift probe 当前仍不执行；后续需 b1/b2 稳定后另批小规模 probe。
