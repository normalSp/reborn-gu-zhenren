# v1.2.0 总体开发大纲

状态：已同步至 b2；D-120/D-121/D-122 已批准，b1 完成，b2 最小 ledger 已完成本地实现与验证
日期：2026-05-20

## 目标

把低阶蛊师生存压力从候选/边界样本推进到更稳定的玩家体验：

- 玩家能理解资源为什么紧张。
- 玩家能看见蛊虫喂养、维护、炼养用准备和路线补给之间的关系。
- 玩家能通过本地 engine 做有限准备，而不是靠 DeepSeek 送资源。
- 商队、交易窗口、灰色渠道保持为有风险的窗口，不是完整市场或套利系统。

## 不是目标

v1.2 不负责：

- 完整库存系统。
- 完整货币系统。
- 价格表。
- 商店 UI 和商店库存。
- 正式买卖结算。
- 黑市/委托/代售收益。
- 商家城市场。
- 宝黄天、高阶蛊材、仙蛊、仙道资源。
- DeepSeek 经济结算权限。

如果上述任何一项需要推进，必须从需求池单独升级并由用户拍板。

## 权威原则

| 事项 | 权威 |
|---|---|
| 正式材料、元石、蛊虫、蛊方、食料、消耗、交易结果 | 本地 canon/engine/store |
| 缺口、风险、候选窗口、传闻、请求、环境压力 | DeepSeek 可叙事，但不能结算 |
| 低阶经济规则草案、原著边界、source pointer | MiroFish/知识库 candidate，仅经 intake 后使用 |
| 玩家可见下一步、缺口解释、不可做原因 | UI + Player Advocate gate |
| 漂移、越权、反刷、长线质量 | 测试矩阵 + 长线叙事漂移制度 |

## 建议小版本结构

1. `a0`：启动审查与范围冻结。
2. `a1`：低阶生存经济 save-format / authority / scope 设计门禁。
3. `a2`：主题切片、知识库、测试矩阵、反刷样本冻结。
4. `b1`：补给/喂养/维护缺口第一刀，默认 projection-only。
5. `b2`：最小 ledger 或只读资源视图，取决于 a1 用户决策。
6. `b3`：炼养用准备、失败代价和维护周期第一刀，默认不结算成功/失败。
7. `b4`：商队/交易窗口边界第一刀，默认询价/担保/风险，不成交。
8. `process-1`：经济反刷、save 兼容、回滚和测试矩阵复核。
9. `process-2`：长线叙事漂移、DeepSeek 越权和知识库使用复核。
10. `rc`：质量收束、Player Advocate、deterministic drift soak，live probe 另批。

## 成功标准

- 玩家看得懂低阶生存压力和下一步。
- 已批准的 b2 仅新增单一 `survivalEconomyState` pressure ledger；除此之外没有新增未批准 save 字段。
- 没有未批准经济结算。
- 没有 DeepSeek 奖励、材料、价格、交易或地点/阵营越权。
- 没有用全书包直接污染 runtime/canon/DeepSeek visible context。
- 新增样本全部进入测试矩阵演进流程。
- 若 runtime 触碰经济或 DeepSeek 叙事，必须执行或记录豁免长线叙事漂移 gate。
