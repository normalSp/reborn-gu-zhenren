## MiroFish 判断

当前 b1 MiroFish 需求：`not_needed`。

理由：

- v0.15 三包已经覆盖 b1 的补给、喂养、炼养用、市场窗口和灰色交易边界。
- b1 不做正式价格、库存、黑市、委托、商家城、高阶资源或完整经济系统。
- 全书基础包只作为档案层，当前没有新增原著/IF 边界需要阻塞提取。

升级为 `blocking` 的触发条件：

1. 准备做正式价格、库存、交易或黑市。
2. 准备把某个蛊、蛊材、蛊方、商队、势力或地点写成 runtime canon。
3. 准备让 DeepSeek 看到新的知识库摘要。
4. 准备改变原著边界、隐藏事实可见性或 IF 晋升状态。
