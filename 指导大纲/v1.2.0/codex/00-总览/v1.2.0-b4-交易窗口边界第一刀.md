# v1.2.0-b4 交易窗口边界第一刀

状态：已完成本地实现与验证
日期：2026-05-20

## 结论先行

b4 不开放市场系统，只把商队接触、问价、公开理由、身份担保和风险提示作为候选窗口。玩家可以理解“怎么试探”，但不会得到成交、价格表、商店库存、元石变化或商队加入结果。

## 用户批准

| ID | 决策 | 执行 |
|---|---|---|
| D-126 | b4 交易窗口第一刀 | 只做询价、担保、身份风险、拒绝理由、候选窗口 |
| D-127 | b4 禁止项 | 不成交、不写价格表、不开放库存、不加入商队、不开放黑市/委托收益 |

## Runtime 改动

| 文件 | 作用 |
|---|---|
| `src/components/game/LowRankSurvivalEconomyPanel.tsx` | 新增 b4 操作区，可从 survival economy 面板执行“试探窗口” |
| `src/engine/v120-low-rank-survival-economy-projection.ts` | v1.2 ledger 来源链加入 D-126/D-127，边界说明明确 b4 不成交 |
| `src/engine/v120-survival-economy-state.ts` | ledger sourceRefs 记录 b4 决策来源 |
| `tests/e2e/v120-low-rank-survival-economy.spec.ts` | 覆盖 survival economy 面板执行 b4 后仍无价格、库存、交易 |

底层执行仍由 `resolveQingmaoMarketWindowAction()` 负责；它已经在 v0.15 中验证过不成交、不写价格、不开放库存、不加入商队。

## 禁止误读

1. b4 没有开放正式市场。
2. b4 没有写价格表或商店库存。
3. b4 没有产生买卖、元石变化或材料变化。
4. b4 没有开放黑市、委托、代售收益或稳定套利。
5. b4 没有开放完整南疆市场或商家城市场。
