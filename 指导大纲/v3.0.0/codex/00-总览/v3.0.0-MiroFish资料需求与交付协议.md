# v3.0.0 MiroFish 资料需求与交付协议

日期：2026-05-24
状态：draft

## 当前分类

v3.0 startup MiroFish need level：`not_needed`。

理由：

- 当前只开专家团会和设计门禁。
- 不新增真实原著事实。
- 不新增命名 NPC。
- 不新增正式地点、势力、奖励或 NPC 生死。
- 不写知识库正文或 runtime canon。
- 不把原著资料放进 DeepSeek visible context。

## blocking 触发器

只要 v3.0 后续触及以下任一项，必须停止并升级为 MiroFish blocking：

- 命名 NPC。
- 真实原著地点、势力、组织。
- hidden-adjacent。
- 方源公开证据。
- L4/L5 原著锚点。
- 正式 lore 结论。
- 原著事件正式因果结论。
- 知识库正文或 runtime canon promotion。
- DeepSeek visible lore/RAG。

## blocking 流水线

若触发 blocking，必须按双仓制度：

1. RebornG 写 request。
2. MiroFish 输出 quote-redacted export / coverage / handoff。
3. RebornG 做 intake review。
4. RebornG 只晋升 rewritten summary / rule draft / test sample。
5. 仍不得直接写 runtime canon、hidden body、DeepSeek visible context。

## v3.0 禁止项

- 不读取全书基础包正文来直接写游戏结论。
- 不把 MiroFish 原始输出复制进 RebornG 知识库正文。
- 不把 source pointer 当作 canon。
- 不让 MiroFish 结果绕过 WorldCore、测试矩阵或用户决策。
