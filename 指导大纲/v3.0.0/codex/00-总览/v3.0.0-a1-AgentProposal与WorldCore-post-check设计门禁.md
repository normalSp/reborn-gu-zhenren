# v3.0.0-a1 AgentProposal 与 WorldCore post-check 设计门禁

日期：2026-05-24
状态：completed
冻结模型：`v300_a1_agentproposal_worldcore_postcheck_contract_v1`

## 设计目标

a1 的目标是把未来 L2/L3 agent 的输出锁成“候选协议”，避免 agent 直接变成世界事实、存档写入、奖励、地点、身份或 NPC 生死裁决者。

## AgentProposal 草案

最小字段：

- `proposalId`：候选 id，未来必须可追踪。
- `agentLayer`：只能是 `L2` 或 `L3`。
- `scope`：必须是 small-area。
- `visibleContextRefs`：只允许引用 redacted / player-visible / local-state-safe 上下文。
- `proposalType`：`memory_candidate`、`intent_hint`、`dialogue_candidate`、`pressure_hint`、`safe_next_step`。
- `claimFamilies`：候选可能触及的边界族。
- `persistenceMode`：v3.0 设计门禁固定为 `transient_candidate_only`。
- `requiresWorldCorePostCheck`：必须为 true。

禁止字段：

- 正式地点、正式阵营、正式身份、正式奖励、NPC 生死结果。
- hidden/private body、MiroFish 原文、prompt body、DeepSeek visible lore/RAG。
- save write、runtime write、canon promotion、knowledge-index body。

## Visible Context Envelope

可见上下文只允许：

- 当前玩家已知状态。
- 已通过 runtime local core 的公开事实。
- 已 redacted 的 synthetic test context。
- report-only/offline 评估样本。

不可见上下文：

- MiroFish hidden/private。
- 原著隐藏因果链。
- 未审查知识库正文。
- L4/L5 关键人物私有动机。
- 天道/宿命宏观裁决文本。

## WorldCore post-check result

最小结果分类：

- `approved_expression`：只可用于未来表达层候选。
- `rejected_violation`：越过 F-300 或违反 contract。
- `needs_user_decision`：触发 save、DeepSeek、MiroFish、backend、外部框架、正式地点/奖励/NPC 生死等 future gate。
- `manual_review_note`：可进入报告，不进入 runtime。
