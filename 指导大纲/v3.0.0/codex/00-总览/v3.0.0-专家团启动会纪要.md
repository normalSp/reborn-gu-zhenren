# v3.0.0 专家团启动会纪要

日期：2026-05-24
状态：completed；D-300 已批准，F-300 保持 `future_gate_required`
主题：`有限 L2/L3 runtime agent 试点设计门禁`

## 启动背景

用户要求开 v3.0 专家团会。v2.9 已完成 v3.0 前安全收束，并明确：

- 可以开 v3.0 专家团/设计门禁。
- 没有授权 runtime agent 实现。
- v3.0 必须先给 D-300/F-300 决策包。

## 专家团共同结论

v3.0 可以启动，但启动语义必须精确：

`有限 L2/L3 runtime agent 试点设计门禁`

它不是完整 agent society，不是所有 NPC live DeepSeek，不是 L4/L5 原著关键人物 runtime，不是后端化，也不是 DeepSeek authority 扩权。

专家团建议 v3.0 先做三层门：

1. 设计门：agent 看什么、输出什么、不能输出什么。
2. 裁决门：WorldCore 如何拒绝、降级、转写或接受候选。
3. 质量门：live probe / Player Advocate / drift / rollback / old-save 如何证明它能进 runtime。

## Project Lead

v3.0 的价值不是“终于能接 agent”，而是把 agent 放进笼子里试运行。

建议的第一阶段目标：

- 一个小区域。
- 只涉及 L2/L3。
- agent 输出 `AgentProposal`。
- WorldCore 拥有最终事实与存档裁决权。
- 玩家看到的是经 post-check 的表达，不是 agent 原始脑补。

不建议 v3.0 一上来就做完整 runtime implementation。先 a0/a1/a2 过门禁，再决定 b1 是否可以做最小 runtime 第一刀。

## Systems Architect

v3.0 的核心 contract 应该是：

`WorldSnapshot + VisibleContext -> AgentProposal -> WorldCorePostCheck -> ApprovedExpression / RejectedFinding / NeedsUserDecision`

必须默认拒绝：

- agent 直接写 save。
- agent 写正式事实。
- agent 决定奖励、地点、身份、NPC 生死。
- agent 读 hidden/private body。
- agent 绕过 local engine / WorldCore。

建议 v3.0-a1 专门冻结 schema 和 post-check rules，b1 之前不动 runtime。

## Gameplay / Numerics Architect

v3.0 不应让 agent 参与数值裁决。

可讨论能力：

- L2 小队成员“下一步想做什么”的候选。
- L2 小势力“公开行为后如何施压”的候选。
- L3 当前场景关键 NPC 的语言、情绪、短期意图候选。

不可讨论能力：

- 价格、奖励、掉落、蛊虫获得。
- 战斗伤亡、永久伤、NPC 生死。
- 正式身份、阵营关系、通缉、招揽、封锁。

所有这些仍由现有 engine / future WorldCore 处理。

## Lore Guard

当前 v3.0 若只做小区域、合成 L2/L3、非命名 NPC、无真实原著锚点，MiroFish 可以暂为 `not_needed`。

一旦出现以下任一项，必须升级为 `blocking`：

- 命名 NPC。
- 真实原著地点/势力。
- hidden-adjacent。
- 方源公开证据。
- L4/L5 原著锚点。
- 正式 lore 结论。
- 知识库正文或 DeepSeek visible lore/RAG。

## AI Pipeline Architect

DeepSeek 可以成为 v3.0 的表达和候选生成器，但不能成为裁决者。

建议：

- a0/a1/a2 不调用 live DeepSeek。
- b1 若要接 runtime，先写 prompt/context diff 和 authority delta。
- live probe 必须单独批准样本、轮次、成本、接受标准。
- 输出必须结构化，禁止 freeform 直接进 store。

## Long Narrative Drift Reviewer

v3.0 的新风险是“短期好看，长期污染”：

- NPC 记忆把候选当事实。
- DeepSeek 把玩家猜测当 canon。
- 同一事件被不同 NPC 反复改写。
- 小势力压力突然变正式追杀。
- agent 语言暗示 hidden/private。

因此 v3.0 必须保留 deterministic replay、short live probe、Player Advocate 和 drift checkpoint 四件套。

## QA / Test Engineering Guardian

建议 D-300 获批后分阶段：

- a0：复盘 v2.9，冻结小区域/L2-L3/非持久候选范围。
- a1：AgentProposal contract 与 WorldCore post-check 门禁。
- a2：live/PA/drift/rollback/old-save 门禁。
- b1：若设计门通过，做最小 runtime 第一刀或继续 report-only，由用户拍板。
- rc：必须有 live probe、Player Advocate、rollback/old-save、no-authority-expansion 证据。

## Player Advocate

玩家体验目标不是“看到 AI 很聪明”，而是：

- NPC 有自己的短期目标。
- 玩家选择会改变后续态度、表达和压力。
- 每次同开局有差异，但差异不污染事实。
- 玩家能分辨“某人这么想/这么说”和“世界事实已经改变”。

v3.0 的 Player Advocate 记录必须写清是否调用 live DeepSeek，若调用则记录模型、样本、轮次、成本和报告路径。

## Git / Rollback Steward

建议 v3.0 使用新分支：

`codex/v300-startup-runtime-agent-design-gate`

startup 只提交文档与入口同步，不提交 runtime 实现。若后续批准 b1 runtime 第一刀，再用单独 commit 边界承载代码。
