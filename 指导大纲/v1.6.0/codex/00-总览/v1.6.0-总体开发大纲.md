# v1.6.0 总体开发大纲

状态：draft
日期：2026-05-21
主题候选：内容生产、canon schema 与长测工厂

## 目标

把 RebornG 的长期内容生产和知识治理从“人读文档记规则”推进成“机械检查 + 人工门禁 + 可追溯引用链”的工程系统。

## 阶段拆分

| 阶段 | 名称 | 目标 |
|---|---|---|
| a0 | 治理补丁与范围冻结 | 建立 v1.6 入口、启动会、决策池、Git 计划、Skill 审计 |
| a1 | 内容/知识库/canon schema 设计门禁 | 定义知识条目、canon 草案、测试样本、promotion chain 的 schema |

## 第一刀建议

v1.6 第一刀仍然不碰 runtime：

- 先写 schema 和 report-only checker。
- 只读检查 `指导大纲/vMiroFish/基础包/`、`指导大纲/知识库/蛊真人/`、`指导大纲/vMiroFish/intake-reviews/`、当前版本测试矩阵。
- 输出报告，不自动修改知识库、不生成 runtime canon。

## 进入 runtime 前硬门

v1.6 默认没有 runtime。若后续出现 runtime 需求，必须重新经过：

1. 当前阶段设计门禁。
2. MiroFish topic-slice intake review。
3. 用户批准 runtime/canon/DeepSeek 可见边界。
4. focused tests + Player Advocate / drift gate。
5. `SAVE_FORMAT_VERSION` 影响评估。
