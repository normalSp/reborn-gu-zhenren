## 1. 本门禁要回答什么

a2 只定义 `check:mirofish-base-pack-inventory` 未来脚本的检查边界：

1. 它能读取哪些基础包文件。
2. 它检查哪些结构事实。
3. 它输出什么报告。
4. 哪些问题是 P0/P1/P2/Info。
5. 它不能自动修什么、不能晋升什么。

门禁结论先给出：专家团建议 **a2 只冻结 inventory checker 设计，不实现脚本，不扫描并吸收章节内容，不创建知识索引条目，不生成 runtime canon**。

## 3. 未来脚本输入边界

未来 `check:mirofish-base-pack-inventory` 只能读取：

| 输入 | 是否允许 | 用途 |
|---|---|---|
| `指导大纲/vMiroFish/基础包/manifest.json` | 允许 | 检查导出策略、权限字段、redaction 摘要 |
| `指导大纲/vMiroFish/基础包/coverage_report.json` | 允许 | 检查 totals、continuity、source pointer、forbidden-key 摘要 |
| `指导大纲/vMiroFish/基础包/ri_lw_ch_*.json` | 允许 | 结构扫描、JSON parse、章节连续性、forbidden-key 扫描 |
| `指导大纲/vMiroFish/基础包/README.md` | 可选 | 仅用于报告链接 |
| MiroFish 原始 `sourceDir` | 禁止 | 不读取外部原始基底，避免把未导出内容混入 RebornG |
| 原著正文文件 | 禁止 | inventory checker 不读取原文 |
| `src/canon/*.json` | 禁止 | a2 不做 runtime canon 晋升 |
| `指导大纲/知识库/蛊真人/` | 禁止 | a2 不做知识索引边界检查，留给 b1 |

脚本只能做结构检查；不得生成摘要、不得改写内容、不得创建知识库条目。

## 4. 检查项

### manifest 检查

必须检查：

- `chapterPackages === 2340`
- `runtimeAuthority === "candidate_only"`
- `canonAuthority === "human_review_required"`
- `runtimeVisible === false`
- `deepSeekVisible === false`
- `quoteRedacted === true`
- `redaction.chaptersWritten === 2340`
- `redaction.removedForbiddenFieldCounts` 包含 `quote`、`originalText`、`excerpt`、`verbatim`、`rawText`、`sourceText`

### coverage 检查

必须检查：

- `totals.packages === 2340`
- `continuity.expected === 2340`
- `continuity.missing.length === 0`
- `continuity.extras.length === 0`
- `continuity.duplicates.length === 0`
- `totals.sourcePointersBad === 0`
- `totals.sourcePointers === totals.sourcePointersComplete`
- `totals.forbiddenKeyHitsInRebornGExport === 0`
- `reborngExport.runtimeAuthority === "candidate_only"`
- `reborngExport.canonAuthority === "human_review_required"`

### chapter 文件检查

必须检查：

- 文件名符合 `ri_lw_ch_0001.json` 至 `ri_lw_ch_2340.json`。
- 章节文件数量为 2340。
- 无缺号、重号、越界号。
- 每个章节文件可以 JSON parse。
- 章节 JSON key 中不出现 forbidden text keys：
  - `quote`
  - `originalText`
  - `excerpt`
  - `verbatim`
  - `rawText`
  - `sourceText`
- 章节包不得声明：
  - `runtimeAuthority = runtime`
  - `canonAuthority = accepted`
  - `runtimeVisible = true`
  - `deepSeekVisible = true`

## 5. 严重级别

| 级别 | 含义 | 示例 | 处理 |
|---|---|---|---|
| P0 | 会导致基础包越权或泄漏 | 章节包出现原文键；`deepSeekVisible=true`；`runtimeAuthority` 不是 candidate-only | 阻断进入任何吸收/工具后续；只报告，不自动修 |
| P1 | 破坏 inventory 可信度 | 缺章节、重复章节、JSON parse 失败、sourcePointersBad > 0 | 阻断工具链后续；需要人工确认 |
| P2 | 审计质量不足 | coverage 计数与 manifest 不一致、review/status 信息缺失 | 可继续讨论，但不得吸收 |
| Info | 正常风险提示 | hiddenFacts 数量、needs_review 状态、local base 曾移除 quote 数 | 记录为 inventory 背景 |

任何 P0/P1 都不得由脚本自动修复。

## 7. report-only 行为边界

未来脚本允许：

- 读取基础包结构。
- 统计文件、字段和风险。
- 输出 report。
- 用非零 exit code 标记 P0/P1。

未来脚本禁止：

- 自动删除、重写、移动基础包文件。
- 自动创建 intake review。
- 自动创建知识库条目。
- 自动创建测试样本。
- 自动修改 `src/canon/*.json`。
- 自动修改 DeepSeek prompt/context。
- 自动把报告结果加入 CI hard gate。
