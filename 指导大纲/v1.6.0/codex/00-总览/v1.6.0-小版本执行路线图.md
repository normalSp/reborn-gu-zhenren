## 默认验证强度建议

| 场景 | 建议门禁 |
|---|---|
| a0 文档启动 | Markdown/入口一致性人工复核、旧口径扫描 |
| a1 schema | schema 样本 + no-runtime/no-DeepSeek checklist |
| b1/b2 checker | fixture 或现有文档 report-only 输出，不自动改文件 |
| a2 inventory gate | 只定义输入、检查项、严重级别和报告 schema，不实现脚本 |
| b3 长测工厂 | deterministic sample + replay report；live probe 另批 |
| b4 stale checker | report-only；不自动删除/重写旧文档 |
| rc | `npm test` / `tsc` / build 可按脚本触及面决定，至少跑新增脚本 focused tests 和文档一致性扫描 |
