# v1.6.0-process-1 流程减法与冗余复核

状态：completed
日期：2026-05-21
性质：流程治理复核；不删除历史证据；不破坏当前门禁引用

## 复核结论

v1.6 不做大规模文档删减。当前最稳妥的减法是：

- 用 `historical-index.md` 和项目仪表盘压缩旧版本入口展示。
- 让 stale-entrypoint checker 先发现过期当前口径。
- 把“要不要删除/归档/改名”的操作推迟到检查结果稳定之后。

## 已做的正向减法

- 当前入口明确指向 v1.6 complete，旧 a1/a2 分支不再作为当前入口。
- v1.6 决策池从“待用户批准”切换为“已批准/已落地/无待批”。
- 工具脚本全部 report-only，不把制度直接变成不可逆改写器。

## 暂不做的减法

- 不删除 v1.6 a0/a1/a2 设计门禁，它们是 D-160/D-161/D-162 的审计证据。
- 不删除 MiroFish 基础包或 intake review。
- 不删除旧版本证据文档。
- 不把 report-only 脚本直接加入 CI hard gate。

## 后续可优化点

后续若 stale checker 连续多个版本绿，可以考虑：

- 将“当前入口状态”检查加入轻量 CI。
- 给旧版本 README 增加统一 historical banner。
- 将项目仪表盘的旧版本长表继续压缩到 historical-index。

这些都需要单独评估，不能为了整洁牺牲可追溯性。
