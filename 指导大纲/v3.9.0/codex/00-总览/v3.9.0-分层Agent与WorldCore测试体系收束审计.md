# v3.9.0 分层 Agent 与 WorldCore 测试体系收束审计

状态：completed locally。

## 当前判断

v2.1-v3.8 的测试体系已经从离线 Agent Lab 逐步演进到玩家可见 transient AgentProposal：

- v2.1-v2.3：Agent Lab、expanded offline、eval farm。
- v2.6-v2.8：private canon / runtime admission / boundary checkers。
- v2.9-v3.3：pre-runtime go/no-go、design gate、rehearsal、F-300 reopen。
- v3.4-v3.8：transient proposal-only UI、lifecycle hardening、micro-expansion、proposal graph、long-horizon stability。

这条线整体顺利：每一步都保留了 WorldCore final authority，没有把 candidate 写成事实，没有新增 save field，没有扩大 DeepSeek 或 MiroFish 权限。

## v3.9 必须复核的问题

| 维度 | 复核点 |
|---|---|
| AgentProposal | proposal 是否仍是候选，而不是正式意图/事实 |
| WorldCore | post-check 是否仍拥有最终通过/拒绝/降级/needs_user_decision 权 |
| Player Advocate | 玩家是否理解候选、拒绝、过期、需决策、事实之间的差异 |
| Deterministic | 同开局差异是否来自候选差异，而不是不可追踪随机污染 |
| Memory contamination | 谣言、候选、rejected 是否污染事实或后续候选 |
| Save safety | old-save/no-save/rollback 是否仍证明无隐式持久状态 |
| DeepSeek | 是否仍没有 prompt/context/model/authority 扩张 |
| MiroFish | synthetic/generic 是否仍足够；触发真实事实时是否会 blocking |
| External frameworks | 是否仍只作为 architecture reference |
| Git/Mainline | 完成分支是否有归宿判断，是否避免主线长期漂移 |
