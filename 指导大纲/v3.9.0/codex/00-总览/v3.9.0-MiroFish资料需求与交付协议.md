# v3.9.0 MiroFish 资料需求与交付协议

状态：completed locally；MiroFish need level `not_needed`。

## 当前 need level

`not_needed`。

理由：

- v3.9 启动包只做 v4.0 前安全收束规划。
- 当前建议的后续实现也是 report-only / synthetic / generic checker。
- 不触及命名 NPC、真实势力、真实地点、hidden-adjacent、方源公开证据、L4/L5 fact anchor 或 formal lore conclusion。

## 升级为 blocking 的触发器

出现以下任一情况，MiroFish 立即升级为 `blocking`，并停止自动推进：

- 命名 NPC。
- 真实原著势力。
- 真实地点或正式地域结论。
- hidden-adjacent material。
- 方源公开证据。
- L4/L5 fact anchor。
- HeavenWill/Fate 真实原著锚点。
- formal lore conclusion。
- knowledge-index body。
- DeepSeek visible lore/RAG。
- runtime canon promotion。

## 交付链

若未来需要 MiroFish：

`RebornG request -> MiroFish export -> RebornG intake review -> rule/test/candidate material`

不得把 MiroFish 基础包、hidden body 或原著正文直接导入 runtime、UI、prompt、DeepSeek visible context 或 report body。

## v4.0 提醒

v4.0 若正式讨论高阶战斗、天道/宿命、时代锚点、L5 宏观压力或原著关键人物，MiroFish 很可能从 `not_needed` 升级为 `blocking`。v3.9 只能准备触发规则，不能预先消耗或吸收真实原著材料。
