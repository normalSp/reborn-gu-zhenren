# v3.9.0 总体开发大纲

状态：completed locally。

## 一句话目标

把 v3.4-v3.8 的 transient AgentProposal 证据链收束成 v4.0 前 go/no-go 基线，并为高阶战斗与天道/宿命双预备建立准入框架。

## 阶段目标

| 阶段 | 目标 | 产物 |
|---|---|---|
| startup | 专家团启动会与前置授权包 | README、启动会、范围冻结、授权包、停机清单 |
| a0 | v3.4-v3.8 证据链复盘 | v3.x transient runtime 证据、缺口、风险表 |
| a1 | agent / WorldCore / future gate 统一审计 | F-300/F-380 状态、WorldCore authority、save/DeepSeek/MiroFish/external boundary |
| a2 | v4.0 高阶战斗 + 天道/宿命准入框架 | v4.0 readiness criteria、禁止项、用户未来决策包草案 |
| b1 | report-only v390 go/no-go checker 第一刀 | 自有 checker / samples / report，不写 runtime |
| b2 | 180 轮 deterministic pre-v4 stability | same-start / proposal / memory-contamination / drift evidence |
| b3 | Player Advocate 与 old-save/no-save/rollback 复核 | 60 轮 PA，live DeepSeek：否，rollback/no-save 证据 |
| rc | 质量收束 | local verification、skill sync、dashboard/AGENTS/PROJECT-STATE、commit/push/CI |

## 设计原则

- `v390` 若实现，只能是 report-only / checker / docs / evidence。
- 玩家 runtime 可见面不扩权；如需 UI 改动，必须保持候选不是事实。
- WorldCore final authority 不变。
- MiroFish 默认 `not_needed`，真实原著或 hidden-adjacent 触发 blocking。
- live DeepSeek 默认 exempted；若要跑，必须另批档位。
- v4.0 准入框架不等于 v4.0 runtime 授权。

## 预期交付价值

1. 让 v3.x 不只是“做了很多小检查”，而是形成可解释的 v4.0 准入基线。
2. 把 high-rank combat / HeavenWill / Fate / L5 这些巨大主题拆成可测试、可停机、可被 WorldCore 裁决的门禁。
3. 给用户下一步 v4.0 专家团会一个清晰选择：开高阶战斗预备、开天道/宿命预备、继续加固 runtime agent，或先做 mainline merge。
