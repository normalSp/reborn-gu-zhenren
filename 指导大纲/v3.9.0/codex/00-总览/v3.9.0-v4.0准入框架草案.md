# v3.9.0 v4.0 准入框架草案

状态：completed locally；已被 a2 readiness 门禁吸收。

## 目标

v3.9 要给 v4.0 一个清晰答案：哪些方向可以进入“预备设计”，哪些仍必须保持 future gate，哪些必须先补证据。

v4.0 默认不是完整蛊仙开放，也不是天道/宿命 runtime 裁决。它只应做两条预备线：

1. 高阶战斗表现与结算架构预备。
2. HeavenWill / Fate / L5 macro pressure 架构预备。

## 高阶战斗准入标准

可以进入 v4.0 预备设计的内容：

- theater map 概念模型。
- killer-move stack 概念模型。
- domain / formation / terrain pressure 概念模型。
- Immortal Gu House state 的边界模型。
- environment damage 的 evidence chain：只显示 WorldCore / combat engine 已裁决事实。

v4.0 前仍禁止：

- 实现高阶战斗 runtime。
- 让表现层裁决命中、伤害、资源消耗、NPC 生死、奖励、地点破坏。
- 开放所有仙蛊、杀招、仙蛊屋。
- 让 DeepSeek 把战斗旁白直接写成事实结算。

## HeavenWill / Fate 准入标准

可以进入 v4.0 预备设计的内容：

- Fate state 边界：`intact`、`fractured`、`destroyed`。
- HeavenWill pressure：压力、修正、排斥、机会偏置。
- era anchor graph：时代、正史锚点、区域大势、尊者布局的 source-pointer 图。
- causality debt：偏离正史后的压力模型。
- L5 macro director：只输出 pressure / constraint / risk。

v4.0 前仍禁止：

- 让天道/宿命直接裁决玩家结局。
- 让 HeavenWill 成为万能失败按钮。
- 开放方源等原著关键人物自由 live agent。
- 暴露隐藏因果链给 UI 或 DeepSeek visible context。
- 未经 MiroFish blocking intake 就写真实原著锚点或 formal lore conclusion。

## Go / No-Go 维度

| 维度 | Go 条件 | No-Go 条件 |
|---|---|---|
| WorldCore authority | 所有事实、save、奖励、生死、地点、阵营仍由 WorldCore 裁决 | agent、DeepSeek、UI、表现层任一方能直接裁决事实 |
| Save safety | 无新字段、无 migration、old-save/no-save/rollback 证据完整 | 需要新增字段或存在隐式持久状态 |
| Lore boundary | 只用 synthetic/generic 或已 intake 的 rewritten summary | 触及真实原著、hidden、L4/L5 但无 blocking intake |
| Player clarity | PA 能理解候选不是事实 | 玩家把候选/压力/传闻当成已发生事实 |
| External framework | 仍只是 architecture reference | 出现 PoC、dependency、subagent、project write |
| DeepSeek | 无 prompt/context/model/authority 扩张 | DeepSeek visible lore/RAG 或 live probe 未获批 |

## v4.0 候选启动问题

v3.9 完成后，v4.0 专家团至少需要用户拍板：

1. v4.0 是否优先高阶战斗预备。
2. v4.0 是否优先 HeavenWill / Fate / L5 macro pressure 预备。
3. 是否允许 MiroFish blocking topic-slice 为 v4.0 提供真实 source pointer。
4. 是否继续保持 report-only / design-gate，还是打开最小 prototype。
5. 是否先处理 mainline merge，再进入 v4.0。
