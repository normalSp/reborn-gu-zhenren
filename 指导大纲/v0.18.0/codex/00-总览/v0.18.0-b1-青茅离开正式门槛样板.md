# v0.18.0-b1 青茅离开正式门槛样板

日期：2026-05-19
状态：已完成
性质：runtime 第一刀，existing v22 字段，无 save-format bump

## 目标

把 `v0.14` 的路线准备/遮掩/山路候选和 `v0.15` 的补给/喂养缺口合并成“离开青茅山承诺前门槛”。玩家能看懂：自己已经不是随口说要走，而是有一组本地前置被系统承认；但这仍不是正式离开青茅山。

## 实现

- 新增 `src/canon/v018-route-multi-region-rules.json`，保存 RebornG 重写后的路线门槛、区域事实、压力、入口边界规则。
- 新增 `src/engine/v018-qingmao-route-multi-region.ts`。
- `resolveV018QingmaoRouteEntryThresholdAction()` 写入：
  - `knownFacts.v018_qingmao_route_entry_threshold_commitment`
  - 轻量 `factionPressure`
  - 公开 `npcMemories`
  - `playerGoals` 的下一步提示
  - `actionConsequences`
  - `localActionLedger` 与 `lastWorldActionReturnContext`
- `FreeGoalPanel` 增加 `v0.18 南疆路线承接` 面板和 `确认门槛` 按钮。

## 权威边界

本阶段明确不写：

- `route_entered`
- `location_unlock`
- `region_unlock`
- `faction_transfer`
- 正式奖励、材料、元石、蛊虫、蛊方
- NPC 生死、捕获、追索成败
- DeepSeek route/location/reward/faction/NPC-life 权限

## 验收

- focused unit 覆盖门槛 ready、阻断、禁止写入、quarantined item 隔离。
- e2e 走通目标、路线准备、遮掩、山路候选、补给、门槛确认。
- Player Advocate 20 轮记录：`v0.18.0-b1-Player-Advocate-20轮走查记录.md`。

b1 通过。玩家能看到“门槛已确认”，但 UI 和账本都明确这不是正式地点进入。
