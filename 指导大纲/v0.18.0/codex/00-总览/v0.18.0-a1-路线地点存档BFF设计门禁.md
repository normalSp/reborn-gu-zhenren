# v0.18.0-a1 路线、地点、存档、BFF 设计门禁

日期：2026-05-19
状态：第一版完成
性质：文档/设计门禁，不改 runtime

## 目标

v0.18 不能把“离开青茅”做成一段文本跳转，也不能一口气开放完整南疆。a1 的目标是先定义：

1. 路线状态和地点状态由谁决定。
2. 什么叫候选、承诺、正式进入。
3. 哪些字段可复用 v22，哪些必须等 `SAVE_FORMAT_VERSION` 决策。
4. BFF/后端什么时候才是必要条件。
5. DeepSeek、MiroFish、UI、engine 的权力边界。

## 状态分层

| 层级 | 含义 | 可见性 | 写入权 |
|---|---|---|---|
| `candidate` | 玩家看到某条路线可能可走 | 玩家可见 | read-only projection 或候选行动 |
| `commitment` | 玩家做了准备并选择朝某路线推进 | 玩家可见 | 统一行动协议可写 existing v22 ledger/goal/return context |
| `route_entered` | 本地 engine 判定已正式进入路线状态 | 玩家可见 | 仅在用户批准 route/location 持久策略后可写 |
| `region_arrived` | 到达新区域/据点/外缘 | 玩家可见 | v0.18 默认不开放；需要独立用户决策 |
| `hidden_route_pressure` | 追索、隐秘因果、私密事实压力 | 玩家不可见或只见公开压力 | 本地 hidden refs；不得进 DeepSeek 可见上下文 |

|---|---|---|

| 路线前置 | RebornG local engine | 允许 |
| 补给/追索/身份压力 | RebornG local engine 读取 existing ledger | 允许 |

| MiroFish | quote-redacted 候选材料 | 需 intake 后重写 |

## v22 可复用字段

不新增持久字段时，b1 可以继续复用：

- `livingWorldState.playerGoals`
- `livingWorldState.knownFacts`
- `livingWorldState.hiddenFactRefs`
- `livingWorldState.actionConsequences`
- `livingWorldState.factionPressure`
- `livingWorldState.npcMemories`
- `livingWorldState.localActionLedger`
- `livingWorldState.lastWorldActionReturnContext`
- `worldClock.lastActionId`

复用原则：

- 可写“准备、遮掩、补给、追索压力、公开后果、下一步提示”。
- 不写“正式地点、正式路线状态、正式阵营身份、正式奖励”。
- 不靠 DeepSeek 说了什么来改变 route/location。

## 候选 v23 字段

如果 b2 前用户批准正式 route/location state，才考虑 `SAVE_FORMAT_VERSION = 23`。

候选字段只作为设计草案：

| 字段 | 类型草案 | 用途 | 风险 |
|---|---|---|---|
| `livingWorldState.routeState.status` | `none/candidate/committed/entered/blocked/returned` | 区分是否正式离开 | 需要旧档默认 |
| `livingWorldState.routeState.routeId` | string/null | 当前路线 | 需要 route id truth source |
| `livingWorldState.routeState.stage` | string/null | 山路、商队接触、外缘落脚等阶段 | 需要迁移与 UI 文案 |
| `livingWorldState.routeState.originRegionId` | string | 原区域，如 Qingmao | 需要 region id 表 |
| `livingWorldState.routeState.targetRegionHint` | string/null | 目标区域提示，不等于到达 | 容易被误读为地点解锁 |
| `livingWorldState.routeState.milestoneIds` | string[] | 已完成公开里程碑 | 需要防重复 |
| `livingWorldState.routeState.pressureIds` | string[] | 补给/追索/身份压力引用 | 需要清理和排序 |
| `livingWorldState.routeState.lastRouteActionId` | string/null | 最近路线行动 | 需要账本一致性 |

任何 v23 字段都必须与迁移、默认值、旧档测试、e2e、Player Advocate 一起提交。

## BFF/后端判断

v0.18 不启动 BFF。只记录触发条件：

- 需要保护 DeepSeek API key。
- 需要私有 canon/hidden fact 服务端隔离。
- 需要云存档/跨端同步。
- 需要线上 eval/archive/成本观测。
- 需要公开测试安全边界。

在这些触发条件出现前，继续 frontend-first：本地 deterministic TypeScript core + static canon + Zustand save + DeepSeek 叙事 API。

## Stop Gates

以下任何事项出现，必须停下来让用户决策：

- 新增 `routeState`、`locationState` 或任何持久字段。
- bump `SAVE_FORMAT_VERSION`。
- 正式进入新地域或新地点。
- 正式路线奖励、材料、蛊虫、蛊方、元石。
- 正式阵营转移、商队加入、通缉、招揽、长期声望。
- NPC 生死、捕获、追杀成败。
- DeepSeek 获得 route/location/reward/faction/NPC-life 写入权。
- BFF/后端、云存档、自动部署。
- MiroFish hidden/quarantined 项进入 UI 或 DeepSeek。

## a1 通过结论

a1 通过：

- v0.18 可以进入 b1 的“正式门槛样板”，但 b1 仍只写 existing v22 ledger/goal/return context。
- b2 若要写 `route_entered` 或 route/location 持久状态，必须再次停下来让用户批准 save-format 方案。
- BFF 只进入长期触发条件，不进入 v0.18 runtime。
