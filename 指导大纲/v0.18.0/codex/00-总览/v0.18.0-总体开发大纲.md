# v0.18.0 总体开发大纲

日期：2026-05-19
状态：草案，待用户批准
主题：南疆路线与多区域承接

## 背景

v0.11 到 v0.17 已经把活世界的核心地基铺到可扩展状态：

- `livingWorldState` 与自由目标让玩家能表达长期行动。
- 青茅事实卡、IF 矩阵、路线准备、NPC/势力反应让世界开始有本地边界。
- 低阶经济、补给、炼养用让“离开青茅”不只是剧情跳转。
- UI 减法让行动、蛊道、世界、记录各有心智位置。
- 战斗深化让路上冲突具备可解释的准备、trace 和回流。

v0.18 的任务是把这些系统第一次串成“青茅之后的人生路线”。它不是做大地图，也不是做通用跑团随机迁移，而是做低阶、可解释、可测试、可回滚的路线承接。

## 版本目标

1. 玩家能理解自己为什么可以或不能离开青茅。
2. 玩家能看到路线进入所需的身份、补给、遮掩、追索和社会压力。
3. 前期账本影响后续路线条件，而不是被剧情跳转抹掉。
4. 南疆低阶区域开始有公开事实/隐藏事实分离。
5. 如果开放正式路线状态，必须有字段、迁移、测试和用户批准。
6. 商队、散修、商家城外缘只开放第一层公开入口，不开放完整城市或阵营系统。

## 范围切分

### a1：路线/地点/存档/BFF 设计门禁

- 盘点 v0.11-v0.17 已有 route/region/location/faction/supply/combat 字段和权威。
- 决定 v0.18 是否需要新增持久字段，例如 `livingWorldState.routeState` 或 `currentRoute/currentRegion`。
- 如果需要新增字段，先写字段表、迁移策略、旧档兼容和测试计划，不直接改 runtime。
- 评估 hidden fact、API key、云存档、公开测试是否触发 BFF；默认不做后端。

### a2：MiroFish intake、字段表、测试矩阵

- 对 v0.18 所需 MiroFish 包做 request/intake。
- 将 v0.14 已通过的 `southern_border_low_rank_route_pack`、`shang_clan_city_public_entry_pack` 作为 carryover 候选，但不直接当 runtime truth。
- 冻结路线状态、区域事实卡、后续反应的字段表和测试矩阵。
- 建立 high-risk 样本：提前去商家城、跳过补给、直接转阵营、绕过追索、携带隐藏事实越界等。

### b1：青茅离开路线正式门槛样板

- 把 v0.14 的山路候选、遮掩痕迹、补给准备、追索压力组合成“离开承诺前最后门槛”。
- 如果用户尚未批准新增 route state，只能保持 `candidate/commitment preview`，不写 `route_entered`。
- 如果用户批准 route state，则 b1 只做前置校验，不直接开大地图。

### b2：路线状态第一刀或候选承接第一刀

- 推荐做最小正式路线状态：只记录路线阶段、公开位置语义、风险、来源 action，不记录完整地图。
- 若不批准存档变更，则做候选承接第一刀，并明确 v0.18 不达成“正式进入下一阶段”。
- 任何 `route_entered`、`currentRoute`、`currentRegion`、`location change` 都必须由本地 engine 写入。

### b3：南疆低阶区域事实卡与路线面板

- 建立低阶南疆区域事实卡：商路、山路、野外、客栈/坊市、商家城外缘等。
- 区分公开事实、玩家可见事实、隐藏事实引用。
- UI 只显示玩家可见区域语义和下一步，不暴露隐藏事实或完整地图。

### b4：前期账本影响后续补给、追索、身份压力

- 读取青茅行动账本、NPC/势力记忆、补给、战斗后果，影响路线风险。
- 展示追索、身份、商队担保、补给缺口等后续压力。
- 不结算正式通缉、招揽、阵营身份或 NPC 生死。

### b5：商队/散修/商家城外缘公开入口边界

- 做公开入口条件展示：商队同行、散修落脚、商家城外缘公共入口。
- 不开放完整商家城，不开放商家城正式任务、拍卖、演武场、商会系统。
- 若玩家目标过大，降级为长期目标与前置条件。

### rc：质量收束

- 全量本地测试、e2e、long e2e、production-preview smoke。
- 如果 b2 写正式 route/location state，rc Player Advocate 建议 80 轮；否则 60 轮。
- 更新仪表盘、handoff、Git 状态和下一版本建议。

## DeepSeek 边界

DeepSeek 允许：

- 描写路线气氛、候选线索、传闻、玩家可见区域压力。
- 根据本地 engine 输出写路线叙事和回流文本。

DeepSeek 不允许：

- 决定路线进入、地点解锁、阵营转移、奖励、NPC 生死或追索成败。
- 直接写 `route_entered`、`currentRoute`、`currentRegion` 或新存档字段。
- 暴露隐藏事实、方源私密因果、重生、春秋蝉或高阶世界信息。

## 完成标志

v0.18 完成时，玩家应能感到：

- 自己不是被文本瞬移，而是带着前期账本进入青茅之后的低阶路线。
- 路线进入、补给、身份、追索和社会压力都有本地解释。
- 南疆早期有可见区域感，但没有失控成完整大地图。
- 系统仍保持本地裁决、测试可回归、DeepSeek 不越权。
