# v0.18.0-b2 路线候选承接第一刀

日期：2026-05-19
状态：已完成
性质：runtime，候选承接，不新增 route/location 持久字段

## 决策

本阶段不写 `routeState`、不 bump `SAVE_FORMAT_VERSION = 23`。因此 b2 走“候选承接第一刀”，不是正式 `route_entered`。

## 目标

把 b1 的承诺前门槛承接成南疆低阶路线候选视图，让玩家看到公开区域事实、入口边界和下一步，但仍不改变地点。

## 实现

- `resolveV018QingmaoCandidateContinuationAction()` 要求先存在 `v018_qingmao_route_entry_threshold_commitment`。
- 写入 `knownFacts.v018_qingmao_route_candidate_continuation_view`。
- 写入 `actionConsequences` 的 `v018:entry:caravan_outer`、`v018:entry:rogue_survival`、`v018:entry:shang_outer` 后续引用。
- `FreeGoalPanel` 的 `候选承接` 按钮展示南疆低阶公开事实与入口边界。

## 非目标

- 不写 `route_entered`。
- 不写当前区域/地点变化。
- 不开放完整南疆地图或完整商家城。
- 不创建正式路线失败/成功状态。

## 验收

- focused unit 覆盖缺门槛时阻断、门槛存在时承接、区域事实只作为公开候选。
- e2e 验证承接后出现南疆低阶事实、商家城外缘、入口边界。
- Player Advocate 20 轮记录：`v0.18.0-b2-Player-Advocate-20轮走查记录.md`。

## 结论

b2 通过。v0.18 保持 candidate-only 路线策略，避免在未批准存档字段前偷写正式路线状态。
