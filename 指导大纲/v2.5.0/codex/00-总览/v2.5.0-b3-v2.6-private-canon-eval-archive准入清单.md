## Manual Review Chain 草案

未来任何 private canon / eval archive / self-learning 候选必须经过：

1. Producer classification：来源、visibility、是否 synthetic、是否 MiroFish、是否 original text。
2. Redaction check：`containsHiddenBody=false`、`containsOriginalText=false`、`containsPromptBody=false`。
3. WorldCore post-check：hidden leak、formal authority drift、memory contamination、L5 overreach。
4. Skill sync audit：expert/game-dev/lore，必要时 combat/mirofish。
5. Test matrix triage：current_matrix / future_sample_pool / discarded。
6. User decision gate：触发 F 项时停机。
7. Git/CI evidence：明确提交边界和 CI 结果。

## Dependency / PoC / Subagent Future Gate Checklist

若未来申请外部框架、PoC、依赖、vendored subset、read-only scan、patch artifact 或 subagents，必须先提供：

- license / SBOM / security posture。
- 架构适配说明：它服务 WorldCore，还是试图替代 WorldCore。
- 输入数据清单：是否触及 hidden/private body、`.env`、玩家存档、MiroFish raw output、知识库正文。
- 输出权限清单：是否会写文件、执行命令、操作 git、生成 patch、修改 runtime。
- sandbox / isolation plan。
- rollback plan。
- 人工复核链路。
- 明确不授权 runtime/canon/save/prompt/DeepSeek visible 直接写入。

当前 v2.5 不批准任何上述事项。
