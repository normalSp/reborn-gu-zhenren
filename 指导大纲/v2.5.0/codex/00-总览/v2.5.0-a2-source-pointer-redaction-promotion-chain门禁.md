v2.5-a2 冻结 `v250_a2_source_pointer_redaction_promotion_chain_v1`：source pointer 只能做审计指针，redaction 是硬边界，promotion chain 必须逐级晋升，任何从档案层直接跳到 runtime/canon/DeepSeek visible 的路径都是禁止路径。

## Redaction Policy

v2.5 固定 redaction policy：

- 原著正文：禁止。
- MiroFish raw output：禁止。
- hidden/private body：禁止。
- prompt body：禁止。
- `.env` / API key / 私密账单：禁止。
- source pointer：只允许 id / class / review status，不允许邻近正文。
- safe summary：当前只定义类别，不写正文。
- report body：只能保存分类、计数、拒绝原因、future gate id。

## Promotion Chain

允许的未来晋升链：

```text
archive/source-pointer
  -> intake review
  -> RebornG-owned candidate summary
  -> rule draft or test sample
  -> user decision gate
  -> canon/schema gate
  -> runtime canon subset
```

v2.5 当前只冻结链路，不执行晋升。

## Forbidden Promotion Paths

以下路径全部禁止：

- `MiroFish base pack -> knowledge-index body`
- `MiroFish base pack -> DeepSeek visible context`
- `MiroFish base pack -> src/canon`
- `source_pointer_only -> player-visible hidden explanation`
- `agent_private_candidate -> save field`
- `hidden_body_locked -> eval archive body`
- `formal_decision_locked -> runtime fact`
- `self-learning candidate -> canon/runtime/save/prompt/knowledge-index body`
- `external framework output -> RebornG patch`

## MiroFish Upgrade-To-Blocking Triggers

当前：`not_needed`。

必须升级为 `blocking` 的条件：

- 需要真实原著事实样本。
- 需要命名 NPC、原著关键人物、大势力核心或 L4/L5 fact anchor。
- 需要 hidden-adjacent、方源公开证据、宿命/天道正式结论。
- 需要正式地点、阵营、身份、奖励、NPC 生死、canon promotion。
- 需要知识库正文或 runtime canon。
- 需要 DeepSeek visible lore/RAG。
- 外部 framework PoC 需要 lore fixture。
- eval archive / private canon service 需要真实 hidden/private body 或 source-neighborhood。

触发 blocking 后，必须停止当前 `/goal` 自动继续并让用户决策。
