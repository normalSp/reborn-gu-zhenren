日期：2026-05-23

## 主线

`private canon / knowledge visibility 试验设计`

## 版本目标

v2.5 的目标是把未来 private canon、knowledge index、DeepSeek visible context、AgentProposal、eval archive、MiroFish intake 的知识流转边界设计清楚。

本版本不是 private canon implementation，也不是后端版本。

## 预期产物

- v2.5 前置授权包。
- private canon / knowledge visibility schema 门禁。
- source-pointer / redaction / promotion chain 门禁。
- visibility boundary report。
- hidden leak / memory contamination / AgentProposal 风险模型。
- v2.6 private-canon/eval-archive 准入清单。
- 前置审批制度试运行复核。
- rc 质量收束记录。

以上产物已全部落地。v2.5 不包含 runtime/backend/service/runner 实现。

## 设计原则

- WorldCore 拥有最终裁决权。
- private canon 是可见性治理，不是大上下文。
- DeepSeek 不读取 hidden/private body。
- source pointer 是审计指针，不是 runtime fact。
- MiroFish 输出只能经 topic-specific intake 进入候选材料。
- AgentProposal 可以引用 redacted class，不可写事实。
- self-learning 只能生成候选，不可改 canon/runtime/save。

## 技术边界

v2.5 默认不实现：

- private canon service。
- backend/BFF。
- eval archive service。
- report runner。
- live DeepSeek probe。
- DeepSeek visible lore/RAG。
- runtime agent。
- knowledge-index body。

## 与 v2.6/v3.0 的关系

- v2.5：设计可见性与拒绝规则。
- v2.6：如果用户批准，才可能进入 agent job queue / replay archive / private-canon 工程预备。

- v3.0：有限 L2/L3 runtime agent 接入仍必须通过 WorldCore，不能直接读取 hidden/private body。

## 成功标准

v2.5 成功不以“功能上线”为标准，而以以下条件为标准：

- 知识可见性边界清楚。
- 未来 service/runner/backend 的准入条件清楚。
- DeepSeek visible 与 hidden/private 的拒绝规则清楚。
- MiroFish 升级 blocking 的条件清楚。
- 后续 `/goal` 执行不会频繁向用户索要重复审批。
