## SAVE_FORMAT_VERSION 结论

`v0.10.0-a1` 不提升 `SAVE_FORMAT_VERSION`。

原因：

- 本阶段只新增静态 canon JSON 和验证测试。
- 未新增 `qingmaoRegionState`、`regionPressureLedger`、区域线索持久化账本或迁移默认值。
- 首批 `clan_school_training` 和 `mountain_patrol` 可以先复用现有 `trainingGroundState`、场景行动账本和 `field-action` metadata。

后续提升条件：

- 新增持久化区域主线状态。
- 新增三寨压力账本并写入存档。
- 新增区域线索池并需要跨读档保留。
- 新增正式三寨委托状态、冷却、失败记录或关系变化历史。

一旦触发上述条件，先设计默认迁移值，再把 `SAVE_FORMAT_VERSION` 从 `21` 提升到下一版本。

## a2 入口

`v0.10.0-a2` 应基于 `qingmao-region-board.json` 做区域线索池与行动入口。

优先顺序：

1. 先读取 `identityScope`，确认玩家当前起点身份。
2. 再读取 `clueSources`，筛出当前身份可收到的候选线索。
3. DeepSeek 只提交候选线索或传闻，本地系统决定是否进入正式行动。
4. UI 显示 AP、地点、风险、阻断原因、身份边界和 DeepSeek 不可越权提示。
5. 如果 a2 需要持久化区域线索池，先回到本文件修改 `SAVE_FORMAT_VERSION` 结论。
