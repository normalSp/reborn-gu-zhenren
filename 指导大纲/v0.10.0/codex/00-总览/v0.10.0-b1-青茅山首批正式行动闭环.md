# v0.10.0-b1 青茅山首批正式行动闭环

日期：2026-05-13
状态：最小闭环完成，已接入 engine、store、UI 与桌面/移动端 e2e；下一步进入 b2 低阶凡蛊与战斗内容包。

## 目标

把 a2 的青茅区域行动候选推进为首批可本地结算的正式闭环：

- 族学/道场磨练：走现有 `v090-training-ground-clue-engine`。
- 山道巡查：走现有 `field-action`。
- 三寨委托/传闻核验：因需要持久化区域状态，本轮继续阻断，不开放正式出发。

## 新增能力

`src/engine/v010-qingmao-region-engine.ts` 新增 `resolveQingmaoRegionAction`。

它会先复用 a2 的 `validateQingmaoRegionCandidate`，再按行动槽进入本地结算：

### clan_school_training

- 自动登记 `tg_nanjiang_refine` 道场线索。
- 使用 `resolveTrainingGroundAction` 完成本地道场结算。
- 返回 `WorldActionCandidate`、`WorldActionDeparture`、`WorldActionResolution`、`LocalActionLedgerEntry`、`NarrativeReturnContext`。
- 不直接由 UI 计算道痕、元石或奖励。

### mountain_patrol

- 使用 `resolveFieldAction({ kind: 'scout' })` 完成本地巡查结算。
- 使用 `buildFieldActionWorldActionBridge` 投影为统一世界行动闭环。
- 只登记风险、巡查事实或本地辅助标记，不直接解锁正式地点、蛊虫、仙材或奖励。

### three_clan_commission

- 保持阻断。
- `saveFormatImpact = requires_persistent_region_state`。
- 不扣 AP，不写正式奖励，不把传闻变成正式任务。

## Store 最小接入

第二刀新增 `src/store/slices/qingmaoRegionSlice.ts`，只做区域入口路由，不新增持久化区域状态：

- `listQingmaoRegionActionEntriesAction()`：从当前 store 身份、AP 和场景状态列出青茅区域行动入口。
- `resolveQingmaoRegionActionAction(input)`：先调用青茅区域引擎做门禁，再按行动槽委托现有 store action。
- `clan_school_training` 委托 `recordTrainingGroundCandidateAction` + `resolveTrainingGroundAction('tg_nanjiang_refine')`。
- `mountain_patrol` 委托 `performFieldAction('scout', 'field')`。
- `three_clan_commission` 继续阻断，并把 `saveFormatImpact = requires_persistent_region_state` 透出给 UI/日志。

该 slice 只暴露运行时方法，没有新增 `qingmaoRegionState`，也没有写入新的专属持久化字段。

## UI 最小入口

第三刀在 `src/components/game/ActionPanel.tsx` 中新增“青茅山区域行动”小节：

- 读取 `listQingmaoRegionActionEntriesAction()` 返回的行动入口。
- 显示线索来源、行动槽、AP、风险、身份范围、阻断原因和警告。
- 点击可出发行动时调用 `resolveQingmaoRegionActionAction()`。
- UI 不计算奖励、伤害、地点解锁、关系变化或世界观事实。
- 移动端与 reduced-motion 下只保留可读卡片，不做额外强演出；视觉深化留给 b4。

配套 e2e 新增 `tests/e2e/v010-qingmao-region-actions.spec.ts`，并在 `src/e2e/installE2eHarness.ts` 增加 `startQingmaoRegionActionDemo()`。

## SAVE_FORMAT_VERSION 结论

`v0.10.0-b1` 最小闭环仍不提升 `SAVE_FORMAT_VERSION`。

原因：

- 道场闭环复用现有 `trainingGroundState`。
- 山道巡查复用现有场景行动账本和 field-action bridge。
- 三寨委托没有开放持久化状态。
- store 接入只新增运行时 action，并在 persist `partialize` 中明确排除这些方法。
- UI 接入只读现有 store/engine 结果，不新增持久化字段。

后续如果要开放三寨委托、三寨压力历史、区域线索跨读档保留，必须先设计 `qingmaoRegionState` 默认迁移值并提升 `SAVE_FORMAT_VERSION`。
