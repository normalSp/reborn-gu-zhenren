# v0.10.0-a2 区域线索池与行动入口

日期：2026-05-13
状态：第一刀完成，等待进入 `v0.10.0-b1` 青茅山首批正式行动闭环。

## 目标

把 `v0.10.0-a1` 的青茅区域主线板转成可运行的本地筛选与候选生成层：

- 根据当前 `StartProfile` 解析青茅区域身份。
- 根据身份筛出可承接的线索来源。
- 根据线索来源和行动槽生成 `WorldActionCandidate`。
- 在候选层解释 AP、地点、风险、身份边界和持久化阻断。
- 继续保证 DeepSeek 只写候选线索、传闻、压力提示和回流文本。

## 新增源码

- `src/engine/v010-qingmao-region-engine.ts`
- `src/engine/v010-qingmao-region-engine.test.ts`

## 核心能力

### 身份解析

`resolveQingmaoRegionIdentity` 读取：

- 显式 `startProfileId`
- `store.flags._start_profile`
- `store.selectedStartProfileId`
- `store.startProfileId`
- `store.profile.startProfileId`

如果无法匹配青茅身份，会降级到散修/外来身份，并返回 issues。这样可避免把未知身份强行写成古月族人。

### 线索来源筛选

`listQingmaoRegionSourceEntries` 读取 `qingmao-region-board.json` 的 `clueSources`：

- 古月/熊家/白家可看到族学告示和山道巡查。
- 商队/武家/铁家/散修按身份看到商队传闻、山道巡查或外部调查入口。
- 身份不允许的来源不会进入正式行动入口列表。

### 行动候选生成

`buildQingmaoRegionActionEntries` 会生成候选行动入口：

- `clan_school_notice -> clan_school_training`
- `mountain_path_patrol -> mountain_patrol`
- `caravan_rumor -> mountain_patrol`
- `caravan_rumor -> three_clan_commission` 仅保留为持久化阻断预览，不开放正式出发。

每个入口生成一个 `WorldActionCandidate`，字段包括：

- `domain`
- `source`
- `sceneId`
- `locationId`
- `risk`
- `apCost`
- `blockers`
- `warnings`
- `metadata.regionId`
- `metadata.identityRole`
- `metadata.actionSlotId`

### AI 候选验证

`validateQingmaoRegionCandidate` 会阻断：

- 非古月身份被写成古月族人或古月族学弟子。
- 文本出现仙蛊、十转、永生蛊、宿命蛊归属、凡人宝黄天交易等禁区。
- 传闻来源直接生成正式委托或原著事实。

### Prompt 上下文

`formatQingmaoRegionContextForPrompt` 会输出当前身份、身份边界、可承接入口、AP/风险/阻断原因和 DeepSeek 权限边界。

## SAVE_FORMAT_VERSION 结论

`v0.10.0-a2` 仍不提升 `SAVE_FORMAT_VERSION`。

原因：

- a2 只新增纯函数引擎和测试。
- 未新增 `qingmaoRegionState` 或区域线索持久化账本。
- 候选行动通过 `WorldActionCandidate` 临时生成，正式持久化要等 b1/b 后续阶段确认。

## 剩余风险

- `three_clan_commission` 需要持久化区域状态，本轮只做阻断预览。进入 b1 前若要正式开放，必须先决定是否提升 `SAVE_FORMAT_VERSION`。
- 当前 a2 还没有 UI 面板；b1 可以先接最小入口，也可以继续只做引擎闭环。若进入 UI，需要遵守“不让 UI 私算行动事实”的规则。
- 当前候选来源数量很少，这是刻意控制范围。扩展来源前先更新 `qingmao-region-board.json` 并补测试。
