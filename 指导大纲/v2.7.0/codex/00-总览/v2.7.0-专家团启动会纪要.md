## 专家团意见

### Project Lead

v2.7 不应该直接进入 runtime agent。更合理的下一步是把 v2.2 的 20 NPC 离线模拟推进到更贴近游戏体验的多 NPC 交互样板：商队短工、小队成员、散修聚落、小势力压力和路人聚合反应。

### Systems Architect

v2.7 应继续坚持 AgentProposal -> WorldCore post-check -> report-only archive 的结构。Agent 可以提出记忆、动机、压力和下一步候选，但不能写事实、身份、奖励、NPC 命运或存档。

建议冻结一个 synthetic 场景族：

`outer_edge_multi_npc_small_faction_synthetic_v1`

它只描述抽象边界，不使用真实原著命名 NPC、真实小势力结论或隐藏事实。

### AI Pipeline Architect

v2.7 默认不调用 live DeepSeek。若后续要测试 live NPC 语言表达，应另开 D 项，因为 live 语言可能触及 prompt/context、模型成本、hidden leak、formal authority drift 和 Player Advocate。

v2.7 更适合先验证：同一事件如何被 L1 路人聚合、L2 小队成员记忆、L3 当前关键 NPC 候选和 L5 宏观压力共同解释，同时仍被 WorldCore 裁决。

### Canon / Knowledge Curator

MiroFish need level 建议保持 `not_needed`。原因是 v2.7 的第一轮应该使用 synthetic archetype，不碰真实原著事实和命名人物。

升级为 blocking 的触发点必须写清：命名 NPC、真实势力、hidden-adjacent、方源公开证据、正式 lore 结论、L4/L5 原著锚点、知识库正文、runtime canon 或 DeepSeek visible lore/RAG。

### Lore Guard

小势力和商队可以作为低阶外缘生活压力的抽象样板，但不能被写成正式阵营身份或正式地点进入。小队成员可形成 candidate memory，但不得自动确认死亡、背叛、永久伤残、收编或正式关系数值。

L5 天道/宿命仍只能作为压力和约束，不是普通 NPC agent，不能裁决玩家结局。

### QA / Eval Guardian

v2.7 应扩大负样本，重点盯：

- multi-agent 互相污染记忆。
- L1 聚合反应越权成正式势力结论。
- L2/L3 把 candidate memory 写成 canon。
- 小队牺牲/救援被误判为 NPC 生死结论。
- 小势力压力漂成正式通缉、招揽或封锁。
- self-learning 候选绕过人工复核。
