## Memory Visibility

| visibility | 用途 | 限制 |
|---|---|---|
| `public_pressure` | L1 聚合反应、公开传闻、场景压力 | 不变成 canon，不写玩家存档 |
| `agent_private_candidate` | L2/L3 候选记忆、意图、反思 | 必须可撤销、可审计、可拒绝 |
| `ref_only` | 仅供 report 解释样本结构 | 不进入 runtime/prompt/context |
| `gated_report_only` | 需要用户决策或高风险候选 | 不自动晋升 |

## Candidate Lifetime

候选记忆默认只存在于离线 report 中。v2.7 不定义正式 TTL、不写 save、不写 knowledge-index body、不写 runtime canon。后续若 v2.8/v3.0 要接入 runtime，必须重新设计持久化、清理、回滚、审计和玩家可见解释。

## Pressure Aggregation

L1 可以把多个小反应聚合成 public pressure；L2 可以把 public pressure 当成候选上下文；L3 可以提出表达/意图候选。任一层都不能把压力转换成正式通行证、身份、封锁、招揽、奖励、地点开放、NPC 生死或阵营关系。

## Manual Review

以下情况必须 `needs_user_decision` 或 `rejected_violation`：

- self-learning candidate patch 可能触碰规则或 skill。
- L5 输出接近结局裁决或正史锚点。
- 候选记忆含正式关系、正式证据、隐藏事实邻近内容。
- 小势力 pressure 试图变成正式 faction state。
