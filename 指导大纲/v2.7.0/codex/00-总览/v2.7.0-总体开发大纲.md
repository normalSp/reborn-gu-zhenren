`低阶多 NPC / 小势力 Agent Lab 样板扩展`

## 设计原则

- WorldCore 拥有最终事实和裁决权。
- Agent 只产出 proposal，不直接写 runtime、save、canon 或知识库正文。
- L1 可聚合反应，不产生个人深层记忆。
- L2 可产生 candidate memory，但必须可撤销、可审计、可拒绝。
- L3 可产生当前场景意图候选和表达候选，不写正式结果。
- L5 只产生宏观压力，不是普通 NPC。
- self-learning 只能生成候选，不能自裁决。
- 小势力和商队样板是 synthetic pressure source，不是正式阵营。

## 工程边界

允许：

- 文档、schema、scenario model、test matrix、truth-source index。
- 自有 zero-dependency / dry-run / report-only runner，已按 D-271-007 执行。
- synthetic fixtures。

不允许：

- runtime/source/UI/store/prompt/save 变更。
- backend/BFF/service。
- live DeepSeek。
- MiroFish export/intake。
- 外部依赖、PoC、subagents、patch artifact。
- 正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。

## 与 v3.0 的关系

v2.7 是 v3.0 runtime agent 的准入证据之一。它只能回答：

- 多 NPC 记忆和压力是否能离线稳定。
- AgentProposal 是否能被 WorldCore post-check 拒绝。
- 小势力压力是否会漂成正式游戏事实。
- 哪些样本应该进入 v2.8 runtime 接入准入门禁。

v2.7 不能回答：

- runtime agent 是否已经批准。
- live DeepSeek NPC 是否可上线。
- 后端或 job queue 是否应实现。
- 正式原著人物/势力是否可被 agent 控制。
