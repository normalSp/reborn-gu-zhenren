## 升级为 blocking 的条件

以下任一条件触发，必须停止自动推进，写 RebornG request，并走 MiroFish dual-repo topic-slice pipeline：

- 需要命名 NPC 或真实 NPC 性格/动机证据。
- 需要真实势力、真实地点、正式阵营压力结论。
- 需要 hidden-adjacent、方源公开证据、L4/L5 原著锚点。
- 需要正式 lore 结论。
- 需要把材料写入知识库正文、runtime canon、DeepSeek visible context 或正式测试事实。
- 需要将小势力 pressure 从 abstract source 升级为具体原著势力。

## 若升级后的流水线

必须按项目制度：

`RebornG request -> MiroFish export -> RebornG intake review -> test matrix / rule draft`

边界：

- MiroFish 只产出 quote-redacted export、coverage report、handoff。
- RebornG 只吸收 rewritten candidate/rule/test material。
- 不直接复制原著正文、hidden body、MiroFish raw output 或 export body 到 runtime、canon、knowledge-index body、DeepSeek context。
