## 专项目标

把 RebornG 从“高速功能扩展”过渡到“能长期稳态推进”的工程秩序：

1. 降低存档迁移和持久化不一致风险。
2. 提升 DeepSeek/response pipeline 的错误可观测性。
3. 减少项目记忆重复和 Skill 更新遗漏。
4. 建立轻量专家团门禁，补上 AI Pipeline、Player Advocate、Content Designer 视角。

### A1-2 存档迁移与持久化归一化

建议优先级：最高。第一刀已完成，见 `v0.11.0-a1-存档归一化第一刀.md`。

当前源码事实：

- `migrateSave()`、`persist.migrate()`、`persist.merge()` 都在处理历史字段或归一化。
- 三处逻辑存在重复维护，后续新增字段时容易漏改。
- `SAVE_FORMAT_VERSION = 21`，但 v0.10-a1 至 b3 未新增持久化字段，不应为了专项本身盲目提升版本号。

已完成：

- 提取共享的 `normalizePersistedGameState()` 纯函数。
- 让 `migrateSave()`、`persist.migrate()`、`persist.merge()` 和手动读档落地路径复用同一归一化路径。
- 补 `src/store/save-normalization.test.ts` 覆盖默认值、子状态归一化和 `formatVersion 21` 不变路径。
- 未新增持久化字段，`SAVE_FORMAT_VERSION` 仍为 `21`。

后续可加强：

- 对 localStorage hydration 做更接近浏览器环境的回归样本。
- 后续新增活世界字段时，在同一个测试文件补迁移/default/merge 案例。

不做：

- 不在第一刀重构整个 store。
- 不在没有新增持久化字段时提升 `SAVE_FORMAT_VERSION`。

### A1-3 response pipeline 静默异常可观测

建议优先级：高。第一刀已完成，见 `v0.11.0-a1-response-pipeline可观测第一刀.md`。

当前源码事实：

- `src/engine/response-pipeline.ts` 仍有多处 `catch { /* skip */ }`。
- 这些异常不一定是 bug，但会让 DeepSeek 问题、成就/目标/遭遇系统缺口难以定位。

已完成：

- 建立低噪声 `pipelineWarn(scope, error, meta)` 与 `buildPipelineWarning()`。
- 对非关键可跳过路径记录 scope、turn、stage、失败原因，不改变运行时行为。
- 测试验证不会抛出二次异常，不泄露 API key、token、prompt、system/user/messages。

不做：

- 不把所有可选系统失败升级为阻断。
- 不把完整 DeepSeek prompt 写入日志。

### A1-4 localStorage key 统一管理

## 暂缓项

以下建议有价值，但不适合作为 a1 第一刀：

- 拆分 `src/types/index.ts`。
- 重命名 `v070/v080/v090/v010` 前缀文件。
- 重组 `src/engine/` 目录。
- 提取 `advanceTurn` 为完整 turn orchestrator。
- 为全部 canon JSON 补 Zod schema。
- 批量迁移全局 Skill 到项目 `.codex/skills/`。
- 创建完整 Sub-Agent 团队或接入 Symphony。

暂缓原因：

- 文件 churn 大，容易打断当前玩法主线。
- 当前 `tsc`、测试和构建已经能通过，先处理更接近玩家数据安全和调试稳定性的部分。
- Codex 桌面当前虽有子代理工具，但使用规则要求用户明确授权；不能假设 TOML 自动等价于可用专家团。

## 验收建议

- `npx tsc --noEmit --pretty false`

- `npm test`
- `npm run test:e2e:long`
