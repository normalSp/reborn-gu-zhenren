# v0.11.0-b2 轻量复盘与升级决策

日期：2026-05-15
状态：已完成复盘，等待用户决策
输入类型：优先级 + 边界 + 口感

## 本轮完成

`v0.11.0-b2` 已把 `a2 活世界状态`、`a3 自由意图裁决` 和 `b1 原著事实卡` 接成青茅山最小活世界样本。

已完成切口：

- `b2` 可见范围调查：玩家调查白家可写入可见事实与行动后果。
- `b2-2` 白家接触压力样本：白家调查可写入 `baijia_zhai` 的 `opportunity` pressure，但不改阵营。
- `b2-3` 方源隐藏保护失败样本：跟踪/打听方源只写 hidden ref、公开失败后果和谨慎 `npcMemories`，不泄露隐藏因果。
- `b2-4` 调查结果后续提示候选：从活世界账本派生白家接触窗口、方源旁证避险和资源线公开旁证提示；提示不是正式行动。

## 专家团结论

### Project Lead

b2 已完成“青茅山活世界最小闭环”的第一层：玩家自由调查可以被本地裁决、回写账本、触发势力/NPC/提示层反馈。

继续推进前必须让用户决策下一条正式行动方向，因为从“提示”到“行动”会改变玩家可做的事情，不应由 Codex 直接替用户开口子。

### Lore & World Designer

当前世界观边界稳定：

- 隐藏事实没有以正文暴露给 UI 或 DeepSeek。
- 方源私有因果、春秋蝉、回溯、重生等仍未向玩家泄露。
- 白家压力只是可见接触机会，不是投靠成功。
- 灵泉/资源内情仍是 hidden ref，不是公开事实。

风险在下一步：一旦开放正式行动，就要明确 `canon-near` 和 `IF` 的差别，特别是白家接触、跟踪方源、逃离青茅山都可能碰到原著锚点。

### Systems Architect

当前工程地基稳：

- `livingWorldState` 已有持久化根。
- 写入仍走 `applyLivingWorldPatch({ source: 'living_world_engine' })`。
- b2-4 提示层是纯派生，不会污染存档。

下一步如果做正式行动，必须决定是否复用现有 `WorldActionCandidate` / `field_action` / `qingmaoRegionSlice`，不能再开第三套行动协议。

### AI Pipeline Architect

DeepSeek 权限没有扩大，这是正确的。

后续建议：

- DeepSeek 只能拿到 `visibleFactIds`、玩家已知事实和本地结算后的可见后果。
- 不能把 hidden ref id、hidden source pointer、NPC privateRefId 传给 DeepSeek。
- 正式行动升级时，需要新增一组 DeepSeek 越权样本：模型试图把提示写成成功投靠、方源被发现、灵泉秘密被揭示时，本地必须覆盖。

### Player Advocate

b2 的玩家体验已经从“我输入一句话，系统给我一句裁决”提升为：

`我调查 -> 世界记住 -> 后续提示出现`

这非常重要，但第一版仍偏保守。玩家会感觉“系统知道我做了什么”，但还没有完整“我能顺着这个提示继续行动”的爽感。

所以下一步最有价值的是挑一条提示升级成正式行动，而不是继续只堆提示。

### QA/Test Engineering Guardian

b2 测试跟上了：

- focused unit 覆盖裁决、事实卡、可见调查、后续提示。
- store test 覆盖 patch 门禁和非法写入。
- e2e 覆盖自由目标面板、调查执行、隐藏事实不泄露和无奖励。
- build 通过。

后续正式行动升级前，需要先把样本矩阵扩到“行动执行后果”层，至少新增：

- 行动可执行但无奖励。
- 行动失败写后果。
- 重复执行不刷状态。
- DeepSeek 越权叙事被本地覆盖。
- UI 不把候选行动写成已成功。

### Frontend & Motion Lead

当前 b2-4 UI 只是文本提示，没有触发新的战斗/动效要求。后续如果做正式行动按钮，必须保证：

- “提示”和“可执行行动”视觉层级清楚。
- 按钮文案不能让玩家误以为已经成功投靠、追踪成功或揭示秘密。
- 移动端面板不能因为提示和行动卡叠在一起而降低可读性。

### Dev Environment Steward

当前不建议提交/推送：

- 本地分支名仍是旧阶段 `codex/v090-b1-world-action-protocol`。
- 工作区存在大量历史脏项和未跟踪文件。
- 若提交，应先清理分支策略，只 stage v0.11 相关文件。

## 正式行动升级候选

### 候选 A：白家接触窗口正式行动

建议：优先批准。

定位：

- 从 `followup_baijia_visible_contact_probe` 升级为正式行动候选。
- 行动含义是“公开渠道试探/递话/核对接触窗口”，不是投靠白家。

正面影响：

- 玩家最容易理解，能马上看到“调查白家 -> 可以尝试接触白家”的活世界反馈。
- 风险比方源低，因为白家公开事实和三寨格局已是可见事实。
- 可以练习势力压力、失败代价和后续行动候选。

限制：

- 不改 `currentFaction`。
- 不加正式声望。
- 不开放白家内部地点。
- 不触发白凝冰重大 IF。
- 不给奖励。

建议第一刀完成定义：

- 新增正式候选但只做 `contact_probe`。
- 成功/失败都只写 `actionConsequences/factionPressure`。
- 结果可作为 DeepSeek 可见事实回流，但 DeepSeek 不得写入成功投靠。

### 候选 B：逃离青茅山路线准备链

建议：第二优先。

定位：

- 从自由目标“逃离青茅山”升级为路线准备行动链。
- 第一刀只调查路线、补给、追踪风险，不开放新地图。

正面影响：

- 非常贴近用户想要的自由感：玩家不是只能跟原著主线走。
- 可以建立长期目标拆行动链的地基。

限制：

- 不直接离开青茅山。
- 不开放后续地域。
- 不规避原著/区域压力。
- 不让 DeepSeek 生成新地点成功抵达。

风险：

- 需要更清晰的路线/补给/追踪数据结构。
- 可能触发新的 `routeCandidate` 或区域状态字段，若要持久化必须另行决策。

### 候选 C：方源旁证调查正式行动

建议：延期。

定位：

- 从 `followup_fang_yuan_visible_scope_caution` 升级为低风险公开调查。

正面影响：

- 题材吸引力强，玩家会感觉自己真的在原著世界旁边行动。

风险：

- 极易泄露隐藏事实。
- 极易变成“跟踪主角”的高风险 IF。
- 需要更多方源公开行动事实卡和 NPC 行为边界。

建议延期条件：

- 先补 3-5 张方源公开活动事实卡。
- 明确哪些行为是玩家角色可见，哪些永远不可见。
- 再开放旁证调查。

### 候选 D：b2 收束进入 rc

建议：可选，但不作为首推。

定位：

- 不继续 b2 runtime，转入 rc 质量收束。

正面影响：

- 稳。
- 文档、测试、门禁可以收干净。

负面影响：

- 玩家还不能顺着提示继续做一条正式行动。
- “活世界”体验会停在第一层账本反馈。

## 专家团建议

综合建议：

1. 优先升级候选 A：白家接触窗口正式行动。
2. 候选 B：逃离青茅山路线准备链作为下一条。
3. 候选 C：方源旁证调查延期。
4. 候选 D：如果用户更想稳，可以先进入 rc。

我的推荐是：`A -> B -> rc`，C 放到后面事实卡更稳后再做。
