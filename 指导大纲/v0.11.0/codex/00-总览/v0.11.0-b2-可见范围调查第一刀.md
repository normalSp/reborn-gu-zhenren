# v0.11.0-b2 可见范围调查第一刀

日期：2026-05-15
状态：已完成第一刀。
结论：青茅活世界最小闭环开始从“可记录长期目标”推进到“可执行可见范围调查”，但仍不开放完整主线、自由 NPC 反应、奖励、地点或正史锚点改写。

## 本刀目标

建立最小链路：

```text
玩家调查意图 -> World Intent Engine 本地裁决 -> 青茅事实卡可见/隐藏分流
-> living_world_engine 受控 patch -> knownFacts/actionConsequences/hiddenFactRefs
-> 自由目标面板可读回显
```

## 已实现

- 新增 `src/engine/v011-qingmao-visible-investigation.ts`。
- 新增 `resolveQingmaoVisibleInvestigation()`：
  - 只接受 `investigate` 意图。
  - 只把可见事实卡写成 `PlayerKnownFact`。
  - 隐藏事实只转成 `HiddenFactRefState`，不生成玩家可见摘要。
  - 输出 `deepSeekVisibleFactIds`，且不包含隐藏事实卡。
  - 输出 `actionConsequences`，记录本地行动后果。
- 更新 `src/store/slices/livingWorldSlice.ts`：
  - 新增 `resolveVisibleInvestigationAction()`。
  - 写入只能走 `applyLivingWorldPatch({ source: 'living_world_engine' })`。
  - 不写 `inventory/materialBag/currency/locationUnlocks/npcDeath/canonAnchorStatus/ending`。
- 更新 `src/components/game/FreeGoalPanel.tsx`：
  - 新增 `执行调查` 按钮。
  - 调查类可行动意图显示 `knownFacts` 写入范围。
  - 只展示可见事实 id，不展示隐藏事实 id 或隐藏摘要。
- 更新 e2e harness：
  - `getStateSummary().livingWorld` 增加 `knownFactIds/hiddenFactRefIds/actionConsequenceCount`，便于浏览器回归断言。

## 当前可玩行为

### 调查白家

- 输入：`我要调查白家`
- 裁决：`investigate / available_with_cost`
- 写入：
  - `knownFacts.qingmao_three_clans_layout`
  - `knownFacts.baijia_bai_ning_bing_public_talent`
  - 1 条 `actionConsequences`
- 不写奖励、背包、材料、地点、势力归属。

### 调查灵泉

- 输入：`我要调查灵泉`
- 裁决：调查意图允许尝试，但可见范围结算被隐藏事实保护拦下。
- 写入：
  - `hiddenFactRefs.guyue_spirit_spring_resource_basis`
- 不写可见事实。
- 不展示“元石根基”“枯竭”等隐藏摘要。

### 跟踪方源

- 输入：`我要跟踪方源`
- 裁决：可尝试调查。
- 当前 b2 第一刀结果：因只命中隐藏事实保护，执行调查不会产生玩家可见事实。
- 不泄露春秋蝉、回溯、重生等隐藏因果。
