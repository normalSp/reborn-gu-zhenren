# v0.11.0 总体开发大纲

日期：2026-05-15
状态：主线已批准并进入执行；a2 活世界状态协议已获准启动。

## 版本定位

`v0.11.0` 建议定为：

`活世界地基与自由意图闸门`

`v0.9.0` 解决“行动如何统一”。
`v0.10.0` 解决“青茅山低阶区域如何形成可玩样板”。
`v0.11.0` 要解决“世界如何记住玩家、判断玩家自由目标、并稳定保存后果”。

## 设计教训

前面开发有一个节奏问题：蛊师低阶生活、区域状态、NPC 反应和正史/IF 还没有做稳，就已经触碰蛊仙、福地、宝黄天、尊者级远景。

v0.11 的任务不是继续铺大，而是收住步子：

- 先把蛊师低阶世界做得可信。
- 先让青茅山能记住和反馈。
- 先让玩家自由意图有本地裁决。
- 先把旧版本承诺和玩家可见占位清干净。

## 核心目标

### 1. 世界能记住

建立可持久化的活世界状态：

- 世界时间。
- 当前区域状态。
- 玩家已知事实。
- 隐藏事实。
- NPC 记忆。
- 势力压力。
- 玩家长期目标。
- 行动后果账本。
- IF 偏离账本。

### 2. 世界能判断

建立自由意图裁决引擎：

- 玩家说出目标。
- DeepSeek 解析意图候选。
- 本地 engine 按境界、资源、地点、关系、世界观、正史/IF 进行裁决。
- 输出可执行、需前置条件、只能传闻、长期目标、硬阻断或重大 IF 偏离。

### 3. 世界能反馈

让玩家行动产生长期后果：

- NPC 更信任或更警惕。
- 势力开放或封锁。
- 线索质量变化。
- 资源入口变化。
- 被记录、被骗、被追查、被招揽或被排斥。

### 4. 原著事实有来源

青茅山正史和 IF 不能靠聊天记忆。v0.11 只做事实抽取试点，后续 v0.12 再深入做完整青茅锚点。

## 非目标

v0.11 不做：

- 完整青茅山原著主线。
- 五域开放世界。
- 盗天魔尊时期等大时代开局。
- 完整蛊仙期经营。
- 宝黄天正式凡人入口。
- 尊者级剧情参与。
- 大规模类型拆分和 engine 目录重组。
- Sub-Agent TOML 自动团队。
- 全量 canon schema sweep。

## DeepSeek 边界

DeepSeek 可以：

- 理解自由意图。
- 生成结构化候选。
- 写叙事和解释。
- 给替代行动建议。

DeepSeek 不可以：

- 直接决定成功。
- 直接发奖励、蛊虫、材料、蛊方、仙蛊。
- 直接改变 NPC 生死、势力归属、正史结果、结局。
- 直接泄露隐藏事实。

## 完成定义

v0.11 完成时至少满足：

- 玩家可见旧债文案清理完成。
- 存档迁移/merge/hydration 归一化，不因新增状态破坏旧档。
- 活世界状态协议至少覆盖青茅山最小样本。
- 自由意图裁决引擎能处理 5-7 个极端样本。
- 原著事实抽取试点完成，不复制大段原文。
- 青茅山活世界最小闭环能保存并回读玩家行为后果。
- `npm test`、`tsc`、`build`、相关 e2e 和生产预览烟测通过。
