# v0.11.0-a1 response pipeline 可观测第一刀

日期：2026-05-15
状态：已完成

## 目标

把 response pipeline 中“可选系统失败但静默跳过”的路径改成低噪声 warning，方便后续定位 DeepSeek 回流、章节目标、成就、遭遇、战斗候选和死亡/起源钩子的真实问题。

## 代码变更

- `src/engine/response-pipeline.ts`
  - 新增 `buildPipelineWarning()` 与 `pipelineWarn()`。
  - warning 记录 `scope`、失败摘要和经过脱敏/截断的 `meta`。
  - `apiKey`、`token`、`secret`、`prompt`、`system`、`user`、`messages` 等敏感键统一脱敏。
  - `console.warn` 或游戏日志写入失败时不再抛出二次异常。
  - 将以下可选路径的静默 catch 改为 warning：
    - L4/L3 拒绝日志写入。
    - 奖励真相源校验日志。
    - 叙事一致性校验日志。
    - 章节目标检测。
    - 成就检测。
    - 叙事回复日志。
    - 章节推进日志。
    - combat-router 候选检测。
    - 活跃遭遇消费。
    - 随机遭遇检测。
    - 死亡日志。
    - 起源解锁日志和检测。
- `src/engine/response-pipeline-token-usage.test.ts`
  - 增加 warning 脱敏测试。
  - 增加 warning 非阻塞测试。

## 行为边界

- 不改变 DeepSeek prompt、模型、temperature、retry 策略或 token 统计。
- 不把可选系统失败升级成玩家可见阻断。
- 不记录完整 prompt、API key、原始 user/system messages。
- 仍保留 `PipeResult` 原有结构；warning 当前进入 console 和 game log，不新增存档字段。
