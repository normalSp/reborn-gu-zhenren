# v0.11.0-a2 设计门禁输出

日期：2026-05-15
状态：已获用户批准，作为 `v0.11.0-a2` 第一刀输出。

## 结论

`v0.11.0-a2` 可以进入“活世界状态协议”开发，但第一刀只冻结设计门禁输出、字段表和测试矩阵，不先写运行时代码。

本轮通过的门禁结论：

- `v0.11.0-a2` 的核心不是自由输入裁决，而是让世界状态可以被保存、回读、审计。
- 持久化字段必须集中进入一个根字段：`livingWorldState`。
- 如果运行时代码新增 `livingWorldState`，则 `SAVE_FORMAT_VERSION` 从 `21` 提升到 `22`，并在同一改动内补默认值、迁移、归一化和旧档测试。
- DeepSeek 仍只写叙事、候选、线索、传闻、压力描述，不直接写存档字段、奖励、战斗事实、地点、原著硬事实或结局。
- 原著隐藏事实不能以正文形式进入玩家可见上下文；a2 只允许保存隐藏事实引用和揭示规则引用。
- UI 只展示玩家已知事实、行动后果摘要、目标状态和风险说明，不展示隐藏事实正文。

## 范围冻结

a2 做：

- 活世界状态根字段设计。
- 世界时间、区域状态、已知事实、隐藏事实引用、NPC 记忆、势力压力、玩家目标、行动后果、IF 偏离账本的第一版字段。
- 读写权矩阵。
- DeepSeek 输入输出契约。
- 保存、迁移、归一化、读档和 UI 泄密测试矩阵。

a2 不做：

- 完整自由输入解析器。
- 完整青茅山原著主线。
- 蛊仙期、宝黄天正式交易、仙蛊、十转、永生、尊者时代开局。
- 大规模类型拆分、engine 目录重组、版本前缀重命名。
- Sub-Agent TOML。
- DeepSeek 模型切换评估。

## 读写权矩阵

| 子系统 | 可读取 | 可写入 | 禁止写入 | a2 契约 |
|---|---|---|---|---|
| DeepSeek runtime | 玩家可见事实、当前行动上下文、已裁决候选摘要、可见压力 | 叙事文本、候选、线索、传闻、非正式压力描述 | 奖励、数值、正式地点、战斗结果、原著硬事实、隐藏事实正文、存档字段 | DeepSeek 输出必须先经过本地 schema 和 engine 才能影响状态 |
| World/Living State engine | canon、store 当前状态、行动结果、本地裁决结果 | `livingWorldState` 的受控 patch | 直接发放奖励、改战斗胜负、绕过 action protocol | 只负责“记忆与世界反应”，不替代其他 engine |
| Action protocol | 玩家行动、候选动作、成本、目标 | 行动结果、action ledger 输入 | 未裁决的自由文本直接落地 | a3 后只能接收 `IntentRuling` 认可的候选 |
| Qingmao region engine | 青茅区域 canon、行动候选、局部状态摘要 | 青茅行动结果、可被 a2 账本记录的摘要 | 直接写长期世界记忆 | 通过活世界状态 engine 记录长期后果 |
| Combat engine | 本地战斗候选、单位、技能、棋盘 | 战斗事实、轨迹、结果 | DeepSeek 战斗胜负、UI 私算伤害 | a2 只记录战斗结果摘要，不重算战斗 |
| Resource/refine engine | 资源、配方、材料缺口 | 正式材料、配方进度、资源结果 | DeepSeek 奖励、传闻奖励落地 | a2 只能记录“玩家已知线索”，不能发资源 |
| Story/canon/IF | 原著摘要事实、来源指针、IF 偏离规则 | 事实卡、偏离等级、可见/隐藏分类 | 复制大段原文、把讨论样例当硬锚点 | b1 负责事实抽取，a2 只预留引用字段 |
| Store/save | 已批准字段、默认值、迁移结果 | `livingWorldState`、save version、归一化默认值 | 未冻结字段、散落状态、UI 派生缓存 | a2 runtime 若新增字段必须升到 `SAVE_FORMAT_VERSION = 22` |
| UI | 玩家已知事实、目标状态、后果摘要、风险标签 | 用户选择、显示状态 | 隐藏事实正文、正式奖励、剧情真相 | UI 解释世界反应，不拥有世界事实 |
| Tests/logs | mock 状态、mock DeepSeek usage、存档快照 | 测试断言、脱敏日志 | 泄露密钥、把测试 fixture 当 canon | 每个新增字段必须有默认、迁移、读写边界测试 |

## DeepSeek 输入输出契约

输入给 DeepSeek：

- 当前角色可见状态。
- 当前地点和行动上下文。
- 玩家已知事实摘要。
- 已裁决候选或待裁决候选的可见部分。
- 可见风险、传闻和压力。
- 不含隐藏事实正文的引用标识，例如 `hidden_fact_ref:qmg_spirit_spring_pressure`。

禁止输入给 DeepSeek：

- 未向玩家揭示的原著硬事实正文。
- 可以直接推导隐藏剧情真相的完整因果链。
- 存档内部实现字段。
- API key、token、secret、system prompt 全文。

DeepSeek 可以输出：

- 叙事文本。
- 2 到 4 个候选行动或后续方向。
- 传闻、线索、压力感、NPC 反应草稿。
- 面向玩家的失败解释草稿。

DeepSeek 不可以输出为事实写入：

- 正式奖励。
- 正式坐标或传送结果。
- NPC 生死、战斗胜负、势力归属。
- 仙蛊、十转、永生、宝黄天交易等未开放事实。
- 原著隐藏事实的直接揭示。

## 原著与 IF 边界

- 用户讨论过的狼潮、灵泉、古月一代、白凝冰、方源回溯、后续地域选择等，只是方向样例，不是已冻结剧情锚点。
- b1 前不得把这些样例直接写成硬事实。
- 原著事实必须来自本地原著文本的摘要抽取，只保存摘要、来源指针和设计解释，不复制大段原文。
- 玩家可以尝试极端目标，但系统必须把目标转成“阻断、传闻、长期目标、前置条件或重大 IF 偏离”，不能直接满足。
- IF 偏离必须记录偏离等级和原因，不能默默改写世界。

## UI 展示规则

UI 可以展示：

- 玩家已知事实。
- 当前目标状态。
- 行动后果摘要。
- 区域压力和势力态度的玩家可见版本。
- “需要前置条件”“只能作为长期目标”“违反当前世界规则”等解释。

UI 不展示：

- 隐藏事实正文。
- 未揭示的原著因果。
- 引擎内部裁决字段。
- DeepSeek 未通过本地验证的候选。
- 任何暗示玩家已获得仙蛊、十转、永生或宝黄天正式交易资格的特效和文案。

## 后续 runtime 解锁条件

进入 a2 runtime 前必须同时满足：

- 字段表已冻结：见 `v0.11.0-a2-活世界状态协议字段表.md`。
- 测试矩阵已冻结：见 `v0.11.0-a2-测试矩阵.md`。
- `SAVE_FORMAT_VERSION` 决策明确：新增持久化字段时从 `21` 升到 `22`。
- 实现顺序遵守 `canon -> types/schema -> engine -> store -> UI -> tests/docs`。
- 不创建未批准的大型新系统，不让 DeepSeek 直接写存档。

## 专家团审查记录

- 项目总策划：a2 可以做，但必须把“活世界状态”与“自由意图裁决”拆开，先状态协议再裁决器。
- 蛊真人世界观审校：隐藏事实只能做引用，不存玩家可见正文；原著锚点必须等 b1 摘要抽取。
- 玩法/数值架构师：a2 不发奖励，不改战斗和经济，只记录已验证结果。
- DeepSeek/模型调优专家：不切换模型；缓存、token 和失败原因继续沿用 v0.9/v0.10 的观测方向。
- QA/发布稳定专家：新增存档字段必须同改动补迁移、归一化和旧档测试。
- Codex 操作安全官：本阶段不使用子代理；阶段完成后写交接，防止长任务偏航。
