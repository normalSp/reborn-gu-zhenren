# v0.11.0-process-1 外部模式借鉴映射

日期：2026-05-15
状态：第一版已建立。
用途：把外部开发模式和 agent 研究转成 RebornG 可执行规则。

## 原则

- 借鉴思想，不照搬组织结构。
- 优先吸收能提升稳定性、连续性、测试能力和少跑偏能力的模式。
- 不为了“像大厂”增加流程负担。
- 不为了“像多 agent 公司”启用未经验证的可写多代理。

## GitHub Flow / GitHub Actions

可取思想：

- 小批量变更。
- PR/issue 模板。
- 自动 CI 门禁。
- 每次合入前有可重复验证。

RebornG 落地：

- 后续建立 `.github` workflow。
- PR 模板包含：阶段、用户决策、触发专家、测试命令、文档同步。
- 先本地执行，后续同步到 GitHub。

暂不做：

- 现在不强行把本地开发全部搬到 PR 流程。

## DORA / Continuous Delivery

可取思想：

- 小步快跑。
- 自动化验证。
- 频繁集成。
- 减少大爆炸式改动。

RebornG 落地：

- 小版本冻结范围。
- 每个阶段有完成定义。
- `npm test`、`tsc`、build、e2e、资产扫描和生产预览烟测按风险触发。

暂不做：

- 不为了指标牺牲世界观质量和内容密度。

## Google SRE

可取思想：

- 无责复盘。
- 事故记录。
- 以系统改进防止复发。

RebornG 落地：

- 黑屏、布局遮挡、DeepSeek 越权、存档迁移失败都要进 `.learnings/ERRORS.md` 或完整复盘。
- 每个严重问题必须绑定防复发测试或门禁。

暂不做：

- v1.0 前不建立重型线上值班体系。

## Microsoft SDL

可取思想：

- 威胁建模。
- 安全边界前置。
- 把风险当作设计问题，而不是上线后问题。

RebornG 落地：

- AI 越权。
- prompt 注入。
- 隐藏事实泄露。
- 存档污染。
- 玩家输入绕过本地裁决。
- DeepSeek 获得非法写入权。

这些都进入阶段门禁和测试矩阵。

暂不做：

- 不做企业级安全流程全套表格。

## Voyager

可取思想：

- 自动课程。
- 技能库。
- 自我验证。
- 从环境反馈中积累能力。

RebornG 落地：

- 专家团主动提 3-5 个候选需求。
- skill 随项目阶段更新。
- `.learnings` 记录经验。
- 测试矩阵随版本增长。

暂不做：

- 不让 agent 自主无限探索并写代码。

## Generative Agents

可取思想：

- 观察。
- 记忆。
- 反思。
- 计划。

RebornG 落地：

- 后续 NPC/势力记忆设计可以参考这个结构。
- 记忆必须本地结构化、可审计、可迁移。
- LLM 只能写候选叙事，不能自由写长期世界事实。

暂不做：

- 不让 DeepSeek 自己维护不可验证的 NPC 记忆。

## Reflexion

可取思想：

- 失败经验反哺下一次行动。
- 语言经验变成策略。

RebornG 落地：

- `.learnings/ERRORS.md` 不是日志垃圾桶，而是防复发规则库。
- 每次 bug 修复必须问：哪个测试或门禁阻止它下次复发？

暂不做：

- 不写没有测试落点的空泛总结。

## Agent-as-a-Judge

可取思想：

- 用 agent 审查 agent 输出。
- 对复杂输出做二次判断。

RebornG 落地：

- 后续可试点只读审查代理。
- 首批候选：QA/Test Engineering Guardian、Lore & World Designer。
- 只读，不写文件，不处理关键路径。

暂不做：

- 不启用可写多代理。
- 不让子代理做当前关键路径。

## Claude-Code-Game-Studios

可取思想：

- 角色分工。
- 游戏开发 SOP。
- 用组织结构帮助 AI 思考。

RebornG 落地：

- 保留专家团角色。
- 用阶段门禁和模板替代复杂 agent 公司结构。
- 借鉴提案和审查思想，不复制全套组织。

暂不做：

- 不批量创建多 agent。
- 不让流程文档多于真实游戏推进。

## RebornG 自己的原则

外部模式最终必须服务三件事：

1. 让蛊真人世界更稳。
2. 让玩家自由度更真实。
3. 让开发不因复杂度上升而失控。

不能服务这三件事的流程，不引入。
