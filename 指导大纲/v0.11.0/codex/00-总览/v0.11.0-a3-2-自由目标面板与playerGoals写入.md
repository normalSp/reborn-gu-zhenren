# v0.11.0-a3-2 自由目标面板与 playerGoals 写入

日期：2026-05-15
状态：第一刀已完成。

## 用户决策

用户已批准：

- 入口采用单独“自由目标”面板。
- 玩家确认后的长期目标可以持久化。
- 持久化范围只允许 `livingWorldState.playerGoals`。
- 不写奖励、地点、势力归属、NPC 生死、正史锚点、结局或 DeepSeek 新权限。

## 本刀目标

把 a3 的本地 `WorldIntentEngine` 接到玩家可见 UI/store，但保持它仍是“裁决与目标管理”，不是完整自由行动执行系统。

本刀完成后玩家可以：

1. 输入自由目标。
2. 看到本地裁决、风险、前置/代价和路由提示。
3. 对有目标草案的裁决执行确认。
4. 确认后只把目标写入 `livingWorldState.playerGoals`。

## 新增运行时代码

### Store

`src/store/slices/livingWorldSlice.ts`

新增：

- `previewWorldIntentAction(rawText)`
- `confirmWorldIntentGoalAction(adjudication)`

规则：

- `previewWorldIntentAction()` 调用 `adjudicateWorldIntent()`，不写状态。
- `confirmWorldIntentGoalAction()` 只接受 `candidate.source = player_input`。
- 只有存在 `suggestedPlayerGoal` 的裁决可以确认。
- 写入必须走 `applyLivingWorldPatch({ source: living_world_engine })`。
- `ui`、`deepseek`、`narrative_text` 仍由 patch API 拒绝。

## 非目标

本刀不做：

- 不把自由文本直接转成正式行动结算。
- 不接 DeepSeek 运行时生成新目标。
- 不做原著事实卡。
- 不做青茅活世界 NPC/势力长期反应。
- 不新增存档字段，不提升 `SAVE_FORMAT_VERSION`。
- 不解锁九转蛊、春秋蝉、宝黄天凡人交易、白凝冰死亡或青茅外域传送。
