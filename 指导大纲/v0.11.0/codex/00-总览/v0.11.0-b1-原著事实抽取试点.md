# v0.11.0-b1 原著事实抽取试点

日期：2026-05-15
状态：第一刀已完成。

## 目的

b1 的目标不是做完整青茅山主线，而是先把“原著事实 -> 项目事实卡 -> 后续活世界裁决”的格式铺稳。

本阶段解决三个问题：

1. 不再靠记忆讨论青茅山锚点。
2. 不复制原著正文，只保存摘要、来源指针和设计边界。
3. 把硬事实、玩家可见事实、隐藏事实和 IF 边界分清。

## 新增真相源

`src/canon/qingmao-canon-fact-cards.json`

关键字段：

- `id`：稳定事实卡 ID。
- `scope`：`region`、`faction`、`npc`、`item` 等。
- `classification`：`hard_fact`、`player_visible_fact`、`hidden_fact` 等。
- `visibility`：`player_visible`、`rumor`、`discovered_or_rumor`、`hidden`。
- `summary`：开发摘要，不复制原文。
- `playerVisibleSummary`：可进入 UI/DeepSeek 可见上下文的短摘要；隐藏事实不得有该字段。
- `sourcePointers`：本地原著文本路径、行号定位和检索关键词。
- `runtimeExposure`：隐藏事实必须是 `hidden_ref_only`。
- `revealPolicyId`：隐藏事实揭示策略引用。
- `designBoundary`：后续系统如何使用、不能越过什么边界。

## 版权与内容边界

事实卡不得保存：

- 原著正文摘录。
- 长段引文。
- 可直接复制进 UI/prompt 的隐藏事实正文。

允许保存：

- 摘要。
- 本地文件路径。
- 行号定位。
- 检索关键词。
- 设计边界。

## 新增测试

`src/canon/qingmao-canon-fact-cards.test.ts`

覆盖：

- `copyrightPolicy = summary_and_locator_only`。
- 每张事实卡必须有 source pointer。
- source pointer 格式必须是 `L数字` 或 `L数字-L数字`。
- 不允许 `originalText`、`excerpt`、`quote` 字段。
- 摘要和设计边界有长度上限。
- 隐藏事实必须是 `hidden_ref_only`，不能有 `playerVisibleSummary`。
- 首批 b2 所需青茅域、族学、灵泉、三寨、白凝冰、方源隐藏因果卡均存在。
