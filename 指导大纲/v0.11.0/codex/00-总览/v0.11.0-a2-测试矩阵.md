## 测试目标

a2 的测试不追求“大世界已经活了”，只追求四件事：

- 新增活世界字段不会破坏旧档。
- DeepSeek 不能越权写事实。
- UI 不泄露隐藏事实。
- 行动、战斗、资源、原著/IF 子系统的权责不被活世界状态协议绕开。

## a2 runtime 必测用例

### 旧档迁移

- 输入：`SAVE_FORMAT_VERSION = 21`，缺失 `livingWorldState`。
- 期望：迁移到 `22` 后补默认 `livingWorldState`。
- 期望：其他 v0.10 字段不丢失。

### 默认归一化

- 输入：`livingWorldState.regions = null`。
- 期望：归一化为 `{}`。
- 输入：`livingWorldState.npcMemories = {}`。
- 期望：归一化为 `[]`。
- 输入：`worldClock.turn = -1`。
- 期望：归一化为 `0` 或拒绝畸形值，具体以实现的 parser 为准。

### 隐藏事实不泄露

- 输入：`hiddenFactRefs.qingmao_secret` 存在，带 `sourcePointer` 和 `revealPolicyId`。
- 期望：DeepSeek prompt 不包含隐藏事实正文。
- 期望：UI 不显示 `sourcePointer` 的内部路径。
- 期望：玩家只看到可见阻断说明。

### DeepSeek 不能写正式奖励

- 输入：mock DeepSeek 输出“获得九转蛊”或“获得仙蛊”。
- 期望：不写入 `gu`、`inventory`、`livingWorldState.knownFacts.confirmed`。
- 期望：只可能变成 `playerGoals` 的 `blocked/deferred` 或被 schema 拒绝。

### 行动后果只记录已验证结果

- 输入：本地 action protocol 生成成功结果。
- 期望：living-world engine 可以追加 `actionConsequences`。
- 输入：未通过本地验证的候选。
- 期望：不追加 `actionConsequences`。

### 青茅回归

- 进入青茅区域行动。
- 执行资源/战斗/普通行动。
- 期望：原本 v0.10 的候选、奖励、战斗事实不被 a2 状态协议改写。

## a3/b1/b2 延伸测试预留

a3 自由意图裁决：

- “我要拿九转蛊” -> `world_rule_blocked` 或 `long_term_goal`。
- “我要获得春秋蝉” -> `hidden_fact_blocked` 或 `world_rule_blocked`，不泄露原著隐藏事实。
- “我要去宝黄天交易” -> mortal 阶段 `world_rule_blocked`。
- “我要投靠白家” -> `requires_prerequisite` 或 `available_with_cost`，不得直接改阵营。
- “我要跟踪方源” -> 风险高，隐藏事实保护，不暴露回溯。
- “我要逃离青茅山” -> 可转为路线目标或区域行动，不强行成功。

b1 原著事实抽取：

- fact card 只保存摘要和来源指针。
- 不复制长段原文。
- discussion sample 不自动进入硬锚点。

b2 青茅活世界闭环：

- 玩家行动影响区域压力或势力压力。
- 保存后重载仍保留玩家已知事实和后果账本。
- 下次 DeepSeek 可引用玩家已知事实，但不能引用隐藏事实正文。
