# v0.11.0-b2-5 白家接触窗口正式行动

日期：2026-05-15
状态：已完成
输入类型：优先级 + 边界

## 目标

把 b2-4 的 `followup_baijia_visible_contact_probe` 从“只读提示”升级成一个正式但受限的本地行动。

行动含义：

- 公开渠道试探。
- 递话或核对接触窗口。
- 记录机会压力和身份暴露风险。

明确不做：

- 不投靠成功。
- 不改 `currentFaction`。
- 不加正式声望。
- 不开放白家内部地点。
- 不触发白凝冰重大 IF。
- 不给奖励或材料。
- 不新增存档字段。

## 行动协议

该行动复用 `v090-world-action-protocol.ts`：

- `WorldActionCandidate.id = qingmao_baijia_contact_window_probe`
- `domain = other`
- `source = player_choice`
- `rewardPolicy = none`
- `chargeAp = false`

执行后写入：

- `factionPressure`
  - `faction_pressure_qingmao_baijia_contact_window_opportunity`
  - `faction_pressure_qingmao_baijia_contact_window_<当前势力>_suspicion`
- `actionConsequences`
  - `consequence_qingmao_baijia_contact_window_probe`
- `sceneSessionState.localActionLedger`
- `flags.lastWorldActionReturnContext`

不写入：

- `inventory`
- `materialBag`
- `currency`
- `currentFaction`
- `locationUnlocks`
- `npcDeath`
- `ifDeviations`

## UI

`FreeGoalPanel` 的白家后续提示现在显示为“可执行试探”，并提供 `试探接触` 按钮。

按钮只来自已有可见调查账本，不直接暴露隐藏事实，也不让提示绕过行动协议。

## 测试

已验证：

```powershell
npm test -- src/engine/v011-qingmao-bai-contact-window.test.ts src/engine/v011-qingmao-investigation-followups.test.ts src/store/slices/livingWorldSlice.test.ts
npx tsc --noEmit --pretty false
npm run test:e2e -- tests/e2e/v011-free-goal-panel.spec.ts
```

关键断言：

- 可见白家调查后才能执行。
- `rewardPolicy = none`。
- 重复执行不刷 `factionPressure/actionConsequences`。
- UI 不显示隐藏事实、春秋蝉、回溯。
- 文案不把结果写成入族、归附、声望或地点开放。

## 结论

b2-5 是 RebornG 第一条从“调查账本 -> 后续提示 -> 正式行动协议 -> 活世界回流”的小闭环。它提升自由感，但仍保持本地裁决、世界观边界和无奖励原则。
