# v0.11.0-b2 启动检查：可见范围调查

日期：2026-05-15
状态：已获用户批准，进入开发。
用户决策：b2 第一刀选择 `可见范围调查`。

## 目标

建立第一个青茅活世界最小闭环：

`自由调查意图 -> 本地裁决 -> 可见事实读取 -> 本地行动结果 -> livingWorldState 回流`

玩家应该能感到：

- 自己提出的调查目标不是装饰。
- 系统会判断这个目标是不是当前可见范围内能做。
- 成功后世界会记住玩家知道了什么。
- 隐藏事实不会被直接揭露。

## 非目标

本刀不做：

- 不做完整青茅山主线。
- 不开放投靠白家完整闭环。
- 不开放跟踪方源完整闭环。
- 不开放逃离青茅山完整路线。
- 不新增奖励、地点、NPC 生死、正史锚点变化。
- 不新增存档字段，不提升 `SAVE_FORMAT_VERSION`。
- 不给 DeepSeek 新写入权。

## 触发专家角色

- Project Lead：冻结 b2 第一刀范围。
- Lore & World Designer：确认调查只使用可见事实卡，不泄露隐藏事实。
- Systems Architect：确认回流只写 `knownFacts/actionConsequences` 或必要 hidden ref。
- AI Pipeline Architect：确认 DeepSeek 只拿 visible facts。
- Player Advocate：确认玩家能理解调查目标、结果和下一步。
- QA/Test Engineering Guardian：补样本矩阵和 focused tests。
- Dev Environment Steward：更新仪表盘、Git 制度、交接。

## 建议第一刀实现

新增一个本地纯引擎函数，优先命名为：

- `resolveQingmaoVisibleInvestigation()`

输入：

- `WorldIntentAdjudication`
- 当前 `LivingWorldState`
- 当前 turn/region/startProfile

输出：

- 是否可结算。
- 玩家可见调查摘要。
- 可写入 `knownFacts` 的事实。
- 可写入 `actionConsequences` 的行动后果。
- 被保护的 hidden fact refs。
- DeepSeek 回流边界。

写入路径：

- store action 必须通过 `applyLivingWorldPatch({ source: 'living_world_engine' })`。

## 测试计划

Focused tests：

- 可见调查白家/三寨：写入 visible known fact，不写 hidden summary。
- 调查灵泉：不能输出隐藏摘要；最多写 hidden ref 或阻断理由。
- 调查方源：不能泄露隐藏因果；当前不做完整跟踪闭环。
- DeepSeek candidate 越权：本地裁决覆盖。
- 重复调查：不应无限堆重复事实。

命令：

```powershell
npm test -- src/engine/v011-qingmao-visible-investigation.test.ts src/store/slices/livingWorldSlice.test.ts src/engine/v011-world-intent-engine.test.ts
npx tsc --noEmit --pretty false
```

阶段收束视改动范围再决定是否跑：

```powershell
npm test
npm run build
```
