# v0.11.0-b2-4 调查结果后续提示候选

日期：2026-05-15
状态：已完成
范围：本地派生提示候选，不是正式行动协议。

## 目标

把 b2 已经写入的 `knownFacts`、`hiddenFactRefs`、`factionPressure`、`npcMemories` 和 `actionConsequences` 转成玩家可读的“下一步提示”。

这一步解决的是：玩家执行调查后，系统不要只留下账本，还要告诉玩家“下一步大概该往哪里想”。

## 非目标

- 不新增 `WorldActionCandidate`。
- 不接统一行动协议。
- 不新增 store action。
- 不新增存档字段，`SAVE_FORMAT_VERSION` 仍为 `22`。
- 不开放正式投靠白家、逃离青茅山路线、NPC 生死、地点开放、正史锚点变化。
- 不让 DeepSeek 获得隐藏事实、正式行动、奖励或状态写入权限。

## 当前提示样本

### 白家可见接触提示

触发条件：

- 已知 `qingmao_three_clans_layout`
- 或已知 `baijia_bai_ning_bing_public_talent`
- 或已有 `baijia_zhai` 的 `opportunity` pressure

输出：

- `followup_baijia_visible_contact_probe`
- 标题：`谨慎核对白家接触窗口`

边界：

- 这只是提示，不等于白家行动。
- 不改阵营，不加声望，不开放地点，不给奖励，不改正史锚点。

### 方源可见范围避险提示

触发条件：

- 已记录 `fang_yuan_private_causality_hidden_anchor` hidden ref
- 或已有 `fang_yuan` 的隐藏事实保护型 `npcMemories`

输出：

- `followup_fang_yuan_visible_scope_caution`
- 标题：`暂缓深追方源，改走旁证调查`

边界：

- UI 只显示公开失败与旁证调查，不显示 hidden ref id。
- 不出现春秋蝉、回溯、重生或方源私有因果。
- 不改方源状态，不写 NPC 生死，不改正史锚点。

### 资源线公开旁证提示

触发条件：

- 已记录受保护资源内情 hidden ref

输出：

- `followup_qingmao_resource_public_inquiry`
- 标题：`资源线暂走公开渠道`

边界：

- UI 不显示 hidden ref id。
- 不显示隐藏事实正文。
- 不给资源奖励，不揭示势力秘密，不让 DeepSeek 获得隐藏上下文。

## UI 呈现

`FreeGoalPanel` 新增 `后续提示` 区块：

- 显示标题、提示类型、可见理由、下一步提示。
- 显示 `提示，不是正式行动`。
- 显示来源数量，不显示隐藏事实 id。
- 显示禁止升级项，避免玩家和后续开发误解为正式行动。

## 测试

已通过：

```powershell
npm test -- src/engine/v011-qingmao-investigation-followups.test.ts src/engine/v011-qingmao-visible-investigation.test.ts src/store/slices/livingWorldSlice.test.ts
npx tsc --noEmit --pretty false
npm run test:e2e -- tests/e2e/v011-free-goal-panel.spec.ts
```

当前测试覆盖：

- 白家可见事实/pressure 能派生接触提示。
- 方源保护失败能派生旁证避险提示。
- 受保护资源内情能派生公开旁证提示。
- 空状态不派生提示。
- UI 不显示春秋蝉、回溯、方源隐藏 fact id。
- UI 提示不授予奖励、不改材料、不写长期目标。

## 下一步决策

b2-4 之后，下一步如果要把某个提示升级为正式行动，就必须让用户拍板。

可选方向：

1. 升级白家接触窗口为正式行动候选。
2. 升级方源旁证调查为低风险公开调查行动。
3. 升级逃离青茅山为路线准备链。
4. 暂停 b2 runtime 扩展，做 b2 轻量复盘并进入 rc/后续版本规划。

任何升级都必须继续遵守：

- 不直接换阵营。
- 不直接开放地点。
- 不直接判定 NPC 生死。
- 不直接揭示隐藏事实。
- 不直接发奖励。
- 不新增持久字段，除非另行批准并同步迁移测试。
