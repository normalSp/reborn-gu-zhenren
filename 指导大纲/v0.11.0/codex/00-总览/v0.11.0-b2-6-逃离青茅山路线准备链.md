# v0.11.0-b2-6 逃离青茅山路线准备链

日期：2026-05-15
状态：已完成
输入类型：优先级 + 边界 + 口感

## 目标

把自由目标“逃离青茅山”升级成第一条长期目标拆行动链样板。

玩家流程：

1. 输入“我要逃离青茅山”。
2. 本地裁决为 `travel / requires_prerequisite`。
3. 用户确认后写入 `livingWorldState.playerGoals`。
4. 目标账本出现 `准备路线` 按钮。
5. 执行后只记录路线、补给、身份遮掩和追踪风险准备。

明确不做：

- 不直接离开青茅山。
- 不开放青茅山外新地域。
- 不传送。
- 不判定逃离成功。
- 不消除追踪风险。
- 不新增路线持久化字段。
- 不给奖励或材料。

## 行动协议

该行动复用 `v090-world-action-protocol.ts`：

- `WorldActionCandidate.id = qingmao_escape_route_preparation_probe`
- `domain = field_action`
- `source = player_choice`
- `rewardPolicy = none`
- `chargeAp = false`

执行前置：

- 必须已有 `playerGoals` 中的 `targetRef = region:outside_qingmao`。

执行后写入：

- `knownFacts.qingmao_escape_route_preparation_baseline`
- `factionPressure.faction_pressure_qingmao_escape_route_<当前势力>_pursuit_risk`
- 原 `playerGoals` 同 id 更新为仍需前置的 `deferred` 目标
- `actionConsequences.consequence_qingmao_escape_route_preparation_probe`
- `sceneSessionState.localActionLedger`
- `flags.lastWorldActionReturnContext`

不写入：

- 地点开放。
- 新 region。
- 逃离成功状态。
- 奖励、材料、声望。
- NPC 生死或正史锚点。

## UI

`FreeGoalPanel` 的目标账本中，`region:outside_qingmao` 目标会显示 `准备路线` 按钮。

按钮文案强调准备动作，不把目标描述成已完成。

## 测试

已验证：

```powershell
npm test -- src/engine/v011-qingmao-escape-route-prep.test.ts src/store/slices/livingWorldSlice.test.ts
npx tsc --noEmit --pretty false
npm run test:e2e -- tests/e2e/v011-free-goal-panel.spec.ts
```

关键断言：

- 没有确认逃离目标时阻断。
- 有目标后可执行路线准备。
- `knownFacts/factionPressure/playerGoals/actionConsequences` 通过 `action_protocol` patch 回流。
- 重复执行不刷状态。
- `currentDomain` 不变。
- `materialBag` 不变。
- UI 不显示逃离成功或新地域开放。

## 结论

b2-6 证明长期自由目标可以被拆成可执行的小行动链，而不需要一次性打开大世界。后续路线系统、补给系统、身份遮掩和追踪风险可以在 v0.12+ 继续深化。
