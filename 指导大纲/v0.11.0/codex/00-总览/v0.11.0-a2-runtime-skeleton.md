# v0.11.0-a2 活世界状态协议 runtime skeleton

日期：2026-05-15
状态：第一轮完成。

## 目标

把 a2 文档门禁中冻结的 `livingWorldState` 变成最小可持久化运行时骨架。

本轮只做：

- 类型。
- 默认值。
- 畸形字段归一化。
- store 根字段。
- `SAVE_FORMAT_VERSION 21 -> 22`。
- 旧档迁移和公开测试存档同步。

本轮不做：

- 自由意图裁决器。
- UI 展示。
- DeepSeek prompt 注入。
- NPC/势力真实反应。
- 原著事实卡抽取。
- 青茅山活世界闭环。

## 代码改动

- `src/types/index.ts`
  - 新增 `LivingWorldState` 及子字段类型。
  - 预留 a3 使用的 `IntentCandidate` 和 `IntentRuling` 类型。
- `src/store/defaultLivingWorldState.ts`
  - 新增 `createInitialLivingWorldState()`。
  - 新增 `normalizeLivingWorldState()`。
  - 对 region pressure、alertLevel、hiddenFactRefs、NPC memory、faction pressure、player goals、action consequences、IF deviation 做保守归一化。
  - 隐藏事实只保留 `sourcePointer`、`revealPolicyId`、`guard` 等引用字段，不保留隐藏事实正文。
- `src/store/slices/livingWorldSlice.ts`
  - 新增 `livingWorldState` 根字段。
- `src/store/initialState.ts`
  - 初始状态加入 `livingWorldState`。
  - `SAVE_FORMAT_VERSION` 从 `21` 提升到 `22`。
- `src/store/index.ts`
  - store 初始化接入 `createLivingWorldSlice()`。
  - `normalizePersistedGameState()` 接入 `normalizeLivingWorldState()`。
  - `migrateSave()`、`persist.migrate()`、`persist.merge()`、手动读档路径继续共享归一化。
- `src/store/defaultLivingWorldState.test.ts`
  - 覆盖默认值。
  - 覆盖畸形字段归一化和隐藏事实正文剥离。
- `src/store/save-normalization.test.ts`
  - 覆盖 v21 旧档迁移到 v22。
  - 覆盖 `livingWorldState` 默认补齐和畸形归一化。
- `public/test-saves/` 与 `测试存档/v0.7.0/`
  - 84 个测试存档同步到 `formatVersion 22`。
  - 补空 `livingWorldState` 默认根。

## 边界

- `livingWorldState` 现在只是可持久化状态地基，不代表世界已经会主动反应。
- DeepSeek 没有获得任何新写权限。
- UI 没有读取或展示隐藏事实。
- `IntentRuling` 只是 a3 类型预留，不直接持久化。
- `playerGoals` 目前为空账本，只有后续 a3/b2 引擎裁决后才能写入。
- 原著讨论样例仍不是 canon anchor。

## 下一步

继续 `v0.11.0-a2` 第二刀，建议做受控 patch API：

- `buildLivingWorldPatch()` 或同等纯函数。
- 只允许从已验证 action result 生成 `actionConsequences`。
- 只允许通过本地 engine 追加 `knownFacts`、`npcMemories`、`factionPressure`。
- DeepSeek 输出只能变成候选说明，不能直接调用 patch。

如果第二刀需要新增写入 action 名称或 UI 入口，再由专家团判断是否需要用户拍板。
