# v0.11.0-a0 玩家可见旧债清理

日期：2026-05-15
状态：已完成第一刀

## 目标

清理玩家可能看到的旧版本工程承诺，避免游戏界面出现 `待 v0.x`、`留到 v0.x`、`以后版本补充` 这类出戏文案。

本阶段不新增玩法、不改数值、不新增存档字段、不改变 DeepSeek 模型策略。

## 改动

### 白家寨开局旧债

更新 `src/canon/start-profiles.json`：

- 移除 `白家寨一转保底蛊待 v0.8 深线补充`。
- 改成当前真实口径：白家寨当前提供地域身份、后山溪瀑资源倾向和三寨竞争压力；专属起步资源需先通过区域状态和原著事实审查。

更新 `src/canon/faction-data.json`：

- 移除 `专属长线剧情留到v0.8深化`。
- 改成当前真实口径：白家寨作为原著本地势力入口，提供身份、地势与三寨竞争压力。

### 创建页效果标签去工程化

更新 `src/components/game/TimelineConfigScreen.tsx`：

- `运行时生效` -> `已生效`
- `已登记未消费` -> `已登记`
- `待系统` -> `需前置`
- `叙事效果` -> `叙事倾向`
- `需降级` -> `降阶处理`

保留原有 `data-testid`，避免破坏既有 e2e。

### 战斗卡开发词清理

更新 `src/engine/v080-battlefield-ui-model.ts`：

- `占位/士气` -> `阵位/士气`
- `a2 只展示阵位与阵纹` -> `当前只展示阵位与阵纹`
- `后续阶段` -> `阵法未成`
- `未接入` -> `未开放`

这些改动只影响展示文案，不改变战斗结算。

### 防复发检查

新增 `scripts/check-player-visible-copy.mjs` 和 package script：

- `npm run check:player-visible-copy`

当前检查范围：

- `src/canon`
- `src/components`
- `src/engine`

当前阻断短语：

- `待 v0.x`
- `待v0.x`
- `留到 v0.x`
- `留给 v0.x`
- `以后版本补充`
- `后续版本补充`

说明：`placeholder` 作为 HTML 输入占位属性、音频资源审查语境和测试语境不在本轮一刀切禁止范围；本阶段优先拦截明确旧版本承诺。
