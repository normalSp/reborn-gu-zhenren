# v0.11.0-a1 专家团门禁清单

日期：2026-05-15
状态：已完成

## 目的

让后续 `v0.11.0-a2/a3/b2` 不靠临场记忆推进，而是每次改运行时代码前都先过一张可执行清单。专家团不是形式审查，必须阻止以下风险：

- 新存档字段没迁移。
- World Intent Engine 绕过现有 action/combat/resource/refine/story/canon/store 权限。
- DeepSeek 被误放权写奖励、地点、命运、结局或原著硬事实。
- 讨论例子被误当成原著锚点。
- UI 或文案泄露隐藏事实。
- 没有极端样本测试就接入自由意图。

## 阶段入口门禁

进入任意新小版本前必须确认：

- `PROJECT-STATE.md`、当前版本 README、路线图、最新 `codex上下文信息/` 状态一致。
- 触发了哪些专家角色：
  - Project Lead：范围、优先级、是否需要用户拍板。
  - Lore & World Designer：原著事实、IF 边界、术语、隐藏事实。
  - Systems Architect：存档、经济、战斗、奖励、套利。
  - AI Pipeline Architect：DeepSeek 权限、schema、缓存、warning。
  - Player Advocate：玩家是否看得懂选择与后果。
  - Frontend & Motion Lead：可读性、移动端、reduced motion。
  - QA & Release Guardian：测试、构建、生产预览、旧档。
  - Content Designer：剧情/事件候选是否有足够差异。
  - Dev Environment Steward：Codex 稳定、上下文交接、子代理限制。
- 若范围未冻结，先给用户 3-5 条候选需求；用户批准后再动工。

## 运行时代码门禁

任何 runtime 改动默认按以下顺序检查：

1. Canon：是否已有真相源，是否需要 provenance。
2. Types/schema：是否需要类型或 Zod 协议。
3. Engine：是否能做成纯函数，是否可复现。
4. Store：是否通过 action 写入，是否需要迁移。
5. UI：是否只展示 engine/store 结果，是否避免私算。
6. Tests/docs：是否有最小测试和交接文档。

禁止项：

- UI 直接写复杂业务状态。
- DeepSeek 直接写数值奖励或原著硬事实。
- 在没有保存/迁移设计时新增持久化字段。
- 把讨论示例直接固化为青茅山原著锚点。

## 存档字段门禁

新增任何持久化字段前必须写清：

- 字段名、类型、默认值、所有者 slice/engine。
- 是否可从旧 flags 镜像恢复。
- 是否进入 `normalizePersistedGameState()`。
- 是否需要 `SAVE_FORMAT_VERSION` 从 `21` 升级。
- 旧档迁移测试样本。
- `persist.migrate()`、`persist.merge()`、手动文件导入是否同路。
- 是否需要 localStorage key，若需要必须先登记到 `src/store/storageKeys.ts`。

不满足以上条件时，字段只能先做静态 canon、派生 UI 或临时运行时状态。

## DeepSeek 权限门禁

每次新增 AI 参与链路时必须确认：

- DeepSeek 只能写叙事、候选、线索、传闻、请求、压力。
- 正式奖励、地点、战斗结果、命运、结局、原著锚点由本地 canon/engine 决定。
- prompt 不写完整隐藏事实给玩家可见输出。
- `state_update` 经过 schema、L3/L4、奖励真相源、一致性校验。
- warning 不泄露 API key、prompt、system/user/messages。
- 若涉及成本/缓存，记录模型、tokens、cache hit/miss、promptPrefixHash。

## 原著与 IF 门禁

涉及青茅山正史、方源、白凝冰、古月一代、狼潮、灵泉等内容时：

- 先从本地原著文本抽取摘要事实卡。
- 只保存摘要、章节/位置指针和设计决策，不复制大段原文。
- 分类为：
  - 原著硬事实。
  - 玩家可见事实。
  - 隐藏事实。
  - 可 IF 偏离点。
  - 讨论示例/未确认。
- 玩家不知道的事实不能被 UI 或 DeepSeek 提前泄露。
- IF 偏离必须有代价、压力、失败可能和后续追踪。

## 世界意图专项硬门禁

进入 `v0.11.0-a2/a3/b2` 前必须先读并满足：

- `v0.11.0-世界意图裁决引擎-设计门禁.md`
- `指导大纲/长期路线/世界意图裁决引擎-设计总纲.md`

最低交付：

- 读写权矩阵。
- 存档字段表。
- 极端意图样本表。
- DeepSeek 边界表。
- 测试矩阵。
- 用户决策点。

## 测试门禁

阶段完成前至少运行与改动相关的检查：

- 存档/store：focused tests + `npx tsc --noEmit --pretty false`。
- DeepSeek/pipeline：pipeline tests + token/warning 脱敏测试。
- UI/布局：Playwright e2e 或生产预览截图/烟测。
- 发布/启动：`npm run build` 与必要时 `npm run check:production-preview`。
- 玩家可见旧债：`npm run check:player-visible-copy`。

大阶段收束时运行：

- `npm test`
- `npm run build`
- 相关 e2e 或 long e2e

## 完成门禁

阶段完成后必须：

- 更新版本 README、路线图、阶段专项文档。
- 更新最新 `codex上下文信息/`。
- 必要时更新 `.learnings/LEARNINGS.md` 或 `.learnings/ERRORS.md`。
- 检查全局 skill 是否出现过期阶段事实。
- 若下一阶段会新增持久化字段、原著锚点或自由意图引擎运行时代码，停下来让用户拍板。
