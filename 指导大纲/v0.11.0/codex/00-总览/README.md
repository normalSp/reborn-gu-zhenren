# RebornG v0.11.0 Codex 当前入口

更新时间：2026-05-15
状态：`v0.10.0` 已完成；`v0.11.0` 本地开发里程碑已完成。当前主线 `活世界地基与自由意图闸门` 已交付：a0/a1/a2/a3/process-1/a3-2/b1/b1-2/b2/b2-2/b2-3/b2-4/b2 轻量复盘/b2-5/b2-6/rc 均已完成。下一步需要用户决策 `v0.12.0` 范围。

## 定位

`v0.11.0` 暂定为 `活世界地基与自由意图闸门`。

它不替代 `v0.10.0` 的青茅山可玩化成果，而是在其上补齐“世界能记住、能判断、能反馈”的地基。

用户已在 2026-05-15 决定：

- 批准进入 `v0.11.0` 开发，先做 `v0.11.0-a0` 玩家可见旧债清理。
- 批准 `v0.11.0-a1` 作为“架构与存档加固专项”，第一刀先做存档归一化。
- 批准进入 `v0.11.0-a2` 活世界状态协议。
- 批准 a2 第一刀先做“设计门禁输出文档 + 字段表 + 测试矩阵”。
- 批准 a2 runtime 新增 `livingWorldState` 持久化字段，`SAVE_FORMAT_VERSION` 已从 `21` 提升到 `22`，并同步补默认值、迁移和测试。
- 批准 `v0.11.0-a3` 第一刀：只做自由意图裁决入口；首批意图限定为 `obtain_item`、`join_faction`、`investigate`、`travel`、`long_term_goal`；DeepSeek 只做候选表达，本地 engine 最终裁决；极端样本必须进入测试。
- 批准 `v0.11.0-process-1`：先把开发流程、用户输入协议、专家团门禁、ADR/复盘模板和外部模式借鉴制度化，再进入 a3-2 UI/store 接入。
- `v0.10.0` 已完成。
- 暂缓大规模类型拆分、engine 目录重组、版本前缀重命名。
- 暂不创建 Sub-Agent TOML，继续用专家团清单门禁。

## 与 v0.10.0 的关系

- `v0.10.0` 已完成并锁定为本地开发里程碑。
- v0.11 不再向 v0.10 回塞新玩法、视觉资产、存档字段或模型策略。
- v0.11 允许修正文档事实和玩家可见旧债，但运行时新增必须进入 v0.11 阶段。

## 权威性

`指导大纲/大方向` 是外部评估参考，不是运行时真相源。后续 v0.11 工作以本目录、当前源码、测试和用户决策为准。
