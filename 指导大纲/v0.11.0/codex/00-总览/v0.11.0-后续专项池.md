### V11-A2 Skill 渐进式披露与事实去重

来源：`04-专业评审与Skill改进建议.md`、`09-Codex能力强化方案.md`、`11-Codex配置落地执行指南.md`

可取点：

- 四个 RebornG 全局 Skill 目前仍有长版本历史，容易重复更新。
- `PROJECT-STATE.md` 已经建立，可以逐步把历史流水账移出 Skill。
- 让 Skill 更像操作手册，而不是版本日志。

建议范围：

- 每次只重构一个 Skill。
- 保留硬边界和触发规则。
- 详细参考移到 `references/`。
- 不删除版本文档里的历史事实。

暂不做原因：

- 误删硬边界的风险比想象中高。
- 需要在 v0.10/v0.11-a1 稳定后慢慢做。

### V11-A3 Prompt / Context Budget 可观测

来源：`05-AI叙事管道技术设计-技术分享文章.md`、`08-专家团设计评审与强化方案.md`

可取点：

- RebornG 的核心优势是“AI 写提案，引擎做裁判”的四层管道。
- 随系统增加，Prompt 体积、动态上下文、缓存命中率都会影响成本和质量。
- AI Pipeline Architect 视角应该长期存在。

建议范围：

- 先观测：记录 system/static/dynamic prompt 估算 token、prompt hash、cache hit/miss、retry 次数。
- 再压缩：只有数据证明低命中或高成本时才压缩规则。

暂不做原因：

- v0.9 已经完成 DeepSeek Flash 切换与基础 telemetry。
- 现在不重新评估模型，先避免把“成本治理”变成“模型折腾”。

### V11-A4 Player Advocate 前 10 轮走查制度

来源：`08-专家团设计评审与强化方案.md`

状态：已升级为项目级流程。

可取点：

- 当前专家团偏工程、世界观和数值，玩家体验视角需要固定化。
- 文字游戏最怕“系统很强，但玩家不知道自己在做什么”。

建议范围：

- 建立手动走查表：每轮玩家是否理解目标、风险、状态变化、下一步。
- 重点看青茅山前 10 轮，而不是全游戏。
- 走查结果进入需求池，不直接改代码。

暂不做原因：

- 需要在 v0.10 可玩区域进一步完整后才有代表性。

### V11-A5 ADR / 决策记录模板轻量化

来源：`07-Claude-Code-Game-Studios优势迁移分析.md`

可取点：

- 我们已经有决策记录，但格式主要按版本推进。
- 对“为什么不做某件事”可以加轻量 ADR，避免后续重复争论。

建议范围：

- 只为重大技术选择写 ADR：存档策略、AI 管道、战斗路线、视觉资产路线。
- 模板保持 5 段：背景、决策、备选、后果、相关文件。

暂不做原因：

- 现有 `指导大纲/v0.9.0` 和 `v0.10.0` 决策记录已够用。
- 不能把每个小决定都写成 ADR。

### V11-A6 只读子代理试点

来源：`06-Sub-Agent映射分析-Claude-Code到Codex.md`、`09-Codex能力强化方案.md`、`11-Codex配置落地执行指南.md`

可取点：

- 独立审查对 QA、发布门禁、世界观审校可能有帮助。
- 当前 Codex feature flags 显示 `multi_agent = true`，桌面会话也有子代理工具。

建议范围：

- 只读试点，不写文件。
- 首个候选角色：QA & Release Guardian。
- 只在用户明确要求“使用子代理/并行审查”时启动。
- 任务必须具体、有边界、可独立完成。

暂不做原因：

- `.codex/agents/*.toml` 自动发现路径尚未在当前桌面 Codex 流程中验证。
- 用户担心稳定性；多代理可能增加上下文、进程、磁盘和冲突风险。
- 现阶段专家团清单门禁已经能覆盖大部分价值。

### V11-A7 Canon Schema 试点

来源：`01-架构审计报告.md`、`03-立即行动项.md`

可取点：

- 大型 canon JSON 数据多，字段漂移风险真实存在。
- Zod schema 对 `gu-database`、`npcs`、`chapters` 等核心数据有长期价值。

建议范围：

- 只选一个小而关键的文件试点。
- 先 test-only 校验，不改运行时加载路径。
- 发现数据不一致时先记录，不立即大修。

暂不做原因：

- 全量 schema sweep 工程量大。
- 当前更高优先级是存档/持久化归一化。

### V11-Future 大时代开局

来源：用户远期愿景。

可取点：

- 当蛊真人世界真正活过来后，玩家可以选择出生时间点，例如盗天魔尊时期、尊者时代、远古/上古/中古等。
- 这能强化“穿越过去开启另一段人生”的最终体验。

建议范围：

- 当前只进入 `指导大纲/长期路线/大时代开局远期扩展池.md`。
- v1.0 前不作为硬承诺。
- v2.0 或更后期再考虑第一个时代开局试点。

暂不做原因：

- 蛊师低阶世界尚未完全活。
- 大时代需要时代年表、尊者因果保护、时代势力版图和高阶世界门槛。
- 现在做会变成换皮开局，风险极高。

## 专家团结论

`指导大纲/大方向` 的价值在于提醒我们补齐治理、可观测和玩家体验视角；它的问题在于工具假设偏激进，且低估了 RebornG 现有体系已经完成的真实工程闭环。

后续采用原则：

- 能提升稳定性、连续性、少跑偏的，收进专项池。
- 会制造大规模 churn 或未验证工具依赖的，暂缓。
- 每个专项单独冻结范围、单独验收、单独让用户决策。
