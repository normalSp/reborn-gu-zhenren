# v0.11.0-a2 受控活世界 patch API

日期：2026-05-15
状态：第二刀完成。

## 目标

在 `livingWorldState` 已经可持久化之后，建立一扇统一写入门。

后续 action protocol、story/canon/IF、faction engine、living-world engine 想写长期世界记忆时，应先形成 patch draft，再交给 `applyLivingWorldPatch()` 归一化和审查。

## 代码改动

- `src/engine/v011-living-world-patch.ts`
  - 新增 `LivingWorldPatchDraft`。
  - 新增 `LivingWorldPatchResult`。
  - 新增 `applyLivingWorldPatch()`。
  - 允许来源：`action_protocol`、`living_world_engine`、`story_canon_engine`、`faction_engine`、`test_fixture`。
  - 拒绝来源：`deepseek`、`ui`、`narrative_text`。
- `src/engine/v011-living-world-patch.test.ts`
  - 覆盖 DeepSeek/UI 直写拒绝。
  - 覆盖本地 action result 追加 `knownFacts`、`actionConsequences` 和 `hiddenFactRefs`。
  - 覆盖隐藏事实正文剥离。
  - 覆盖 `playerGoals` 按 id upsert，避免重复账本。

## 写入契约

允许：

- 本地 action protocol 写已验证行动后果。
- story/canon/IF 写已审查事实或隐藏事实引用。
- faction engine 写压力账本。
- living-world engine 写区域状态和长期目标。

拒绝：

- DeepSeek 直接写世界状态。
- UI 按钮直接写世界状态。
- 叙事正文直接写世界状态。

## 当前边界

- patch API 还没有接到 store action。
- 还没有任何玩家行动会自动写 `livingWorldState`。
- 还没有 UI 展示 `livingWorldState`。
- 还没有 DeepSeek prompt 使用 `livingWorldState`。
