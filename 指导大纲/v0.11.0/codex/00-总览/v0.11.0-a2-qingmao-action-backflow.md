# v0.11.0-a2 青茅行动最小回流样本

日期：2026-05-15
状态：第三刀完成。

## 目标

验证 `livingWorldState` 不只是空字段，而是能从一个已验证本地行动结果写入最小世界记忆。

本轮只接一个样本：

- 青茅山 `mountain_patrol` 山道巡查。

不接：

- 族学训练。当前 store 委托 `trainingGround` 路径，青茅 region engine 侧没有稳定 `worldActionResolution` 可直接回流，暂不硬写。
- 三寨委托。仍需要更完整青茅区域状态机。
- 战斗、资源、NPC 生死、势力归属和奖励。

## 代码改动

- `src/store/slices/qingmaoRegionSlice.ts`
  - 新增 `commitQingmaoLivingWorldBackflow()`。
  - 当 `mountain_patrol` 经本地 `field-action` 成功后，通过 `applyLivingWorldPatch()` 写入：
    - `livingWorldState.worldClock`
    - `livingWorldState.knownFacts`
    - `livingWorldState.actionConsequences`
    - `flags.lastLivingWorldPatch` 诊断摘要
  - 不写资源、奖励、地点、战斗、NPC 生死或势力归属。
- `src/engine/v010-qingmao-region-engine.ts`
  - 更新三寨委托阻断文案，不再说“当前阶段不提升 SAVE_FORMAT_VERSION”，改为“需要更完整青茅区域状态机”。
- `src/store/slices/qingmaoRegionSlice.test.ts`
  - 覆盖 `mountain_patrol` 成功后写入活世界最小账本。
  - 覆盖 `clan_school_training` 不写假回流。
  - 覆盖 `three_clan_commission` 仍不写活世界账本。
- `src/engine/v010-qingmao-region-engine.test.ts`
  - 同步三寨委托阻断断言。

## 写入示例

`mountain_patrol` 成功后会写入：

- 一条 `knownFacts`：本地 engine 认可的青茅山巡查事实摘要。
- 一条 `actionConsequences`：本次行动后果摘要和本地 fact 引用。
- `worldClock.lastActionId`：本地 world action departure id。

这不是：

- 新奖励。
- 新地点正式开放。
- 新 NPC 记忆。
- 势力关系变化。
- 原著事实锚点。
