# v0.11.0-a3 第一刀：自由意图本地裁决引擎

日期：2026-05-15
状态：已完成第一刀。

## 用户批准范围

用户已批准进入 `v0.11.0-a3`，第一刀范围冻结为：

1. 只做自由意图裁决入口，不做完整大世界。
2. 首批意图类型限定为 `obtain_item`、`join_faction`、`investigate`、`travel`、`long_term_goal`。
3. DeepSeek 只生成候选解释/表达方式，本地引擎最终裁决可行性、门槛、阻断理由和回流字段。
4. 极端意图样本必须进入测试：一转拿九转蛊、凡人进宝黄天交易、跟踪方源、逃离青茅山等。

## 本刀目标

建立最小 `WorldIntentEngine`，把玩家自由输入先变成结构化 `IntentCandidate`，再由本地引擎输出 `IntentRuling`。

本刀只做纯函数裁决：

- 不接 UI 输入框。
- 不新增 store action。
- 不写 `livingWorldState`。
- 不新增持久化字段。
- 不提升 `SAVE_FORMAT_VERSION`。

## 新增真相源

### `src/canon/v011-world-intent-rules.json`

用途：

- 冻结 a3 首批意图类型。
- 冻结裁决分类。
- 冻结 DeepSeek 输出契约。
- 固化首批极端样本。

关键边界：

- DeepSeek 不得直接发奖励、改势力、开放地点、判定 NPC 死亡、泄露隐藏事实、改正史锚点或生成正式结局。
- UI 只能展示裁决和收集确认，不得私算成功、奖励、隐藏事实或势力归属。
- World Intent Engine 只是路由和裁决入口，不替代行动协议、战斗、资源、炼蛊、story/canon 或 store 权限。

## 新增运行时代码

### `src/engine/v011-world-intent-engine.ts`

导出：

- `createIntentCandidate()`
- `adjudicateIntentCandidate()`
- `adjudicateWorldIntent()`
- `buildWorldIntentContextFromStore()`

裁决输出：

- `candidate: IntentCandidate`
- `ruling: IntentRuling`
- `route: WorldIntentRouteSuggestion`
- `deepSeekContract`
- `suggestedPlayerGoal`
- `statePatchApplied: false`

`suggestedPlayerGoal` 只是草案。只有后续 UI/store 切口让玩家确认后，才允许通过 `applyLivingWorldPatch()` 写入 `livingWorldState.playerGoals`。

## 首批样本裁决

| 玩家输入 | 意图类型 | 裁决 | 是否可立即执行 | 说明 |
|---|---|---|---|---|
| 我要拿九转蛊 | `obtain_item` | `long_term_goal` | 否 | 只能作为远期野心、传闻追索或骗局风险 |
| 我要获得春秋蝉 | `obtain_item` | `world_rule_blocked` | 否 | 高阶特殊事实，不成为普通奖励 |
| 我要去宝黄天交易 | `travel` | `world_rule_blocked` | 否 | 凡人阶段不能正式交易 |
| 我要投靠白家 | `join_faction` | 视身份而定 | 古月/熊家否，白家是，散修/外来可尝试接触 | 不一键改阵营 |
| 我要跟踪方源 | `investigate` | `available_with_cost` | 是 | 只做可见行踪调查，不泄露隐藏因果 |
| 我要逃离青茅山 | `travel` | `requires_prerequisite` | 否 | 需要路线、补给、身份遮掩和追踪风险 |
| 我要杀死白凝冰 | `long_term_goal` | `major_if_deviation` | 否 | 重大 IF 偏离，不直接判定 NPC 死亡 |

## DeepSeek 契约

DeepSeek 可以：

- 生成候选措辞。
- 根据本地 ruling 写可见解释。
- 给传闻式表述。
- 给下一步提示文案。
- 给风险提示文案。

DeepSeek 不可以：

- 直接发奖励。
- 直接改势力归属。
- 直接开放地点。
- 直接判定 NPC 死亡。
- 写隐藏事实正文。
- 改正史锚点。
- 生成正式结局。
