# v0.11.0-a2 活世界状态协议字段表

日期：2026-05-15
状态：a2 第一刀冻结草案；runtime skeleton 第一轮已按本表落地。

## 存档决策

运行时代码一旦新增 `livingWorldState` 持久化根字段：

- `SAVE_FORMAT_VERSION` 从 `21` 提升到 `22`。
- 旧档迁移默认补齐 `livingWorldState`。
- `normalizePersistedGameState()`、`persist.migrate()`、`persist.merge()` 和手动读档路径必须共享同一默认值。
- 同一改动必须补测试，不允许先升版本后补测试。

后续 runtime skeleton 已按本表实现最小字段，当前 `SAVE_FORMAT_VERSION = 22`。

## 根字段

建议新增唯一持久化根字段：

```ts
livingWorldState: LivingWorldState
```

默认值：

```ts
{
  schemaVersion: 1,
  worldClock: {
    turn: 0,
    day: 1,
    phase: "start",
    lastActionId: null
  },
  regions: {},
  knownFacts: {},
  hiddenFactRefs: {},
  npcMemories: [],
  factionPressure: [],
  playerGoals: [],
  actionConsequences: [],
  ifDeviations: []
}
```

## 类型草案

```ts
export type LivingWorldPhase = "start" | "morning" | "afternoon" | "night";

export type LivingRegionStatus = "stable" | "tense" | "restricted" | "collapsing";

export type LivingRegionAccess = "known" | "rumored" | "restricted" | "blocked";

export type LivingGoalStatus = "active" | "blocked" | "deferred" | "completed" | "failed";

export type LivingIfDeviationLevel = "none" | "minor" | "major" | "critical";

export interface LivingWorldState {
  schemaVersion: 1;
  worldClock: LivingWorldClock;
  regions: Record<string, LivingRegionState>;
  knownFacts: Record<string, PlayerKnownFact>;
  hiddenFactRefs: Record<string, HiddenFactRefState>;
  npcMemories: LivingNpcMemoryEntry[];
  factionPressure: LivingFactionPressureEntry[];
  playerGoals: LivingPlayerGoalEntry[];
  actionConsequences: LivingActionConsequenceEntry[];
  ifDeviations: LivingIfDeviationEntry[];
}
```

## 子字段冻结

### `LivingRegionState`

```ts
export interface LivingRegionState {
  regionId: string;
  status: LivingRegionStatus;
  pressure: number;
  alertLevel: 0 | 1 | 2 | 3;
  access: LivingRegionAccess;
  knownEventIds: string[];
  hiddenFactRefIds: string[];
  factionPressureIds: string[];
  lastUpdatedTurn: number;
}
```

约束：

- `pressure` 范围为 `0..100`。
- `alertLevel` 只表示玩家可感知或本地系统认可的戒备等级。
- `hiddenFactRefIds` 只存引用，不存隐藏事实说明。

### `PlayerKnownFact`

```ts
export interface PlayerKnownFact {
  id: string;
  scope: "world" | "region" | "npc" | "faction" | "item" | "quest";
  source: "canon_summary" | "engine_result" | "deepseek_clue" | "player_observation";
  summary: string;
  learnedTurn: number;
  confidence: "confirmed" | "rumor" | "misleading";
  tags: string[];
}
```

约束：

- `summary` 必须是项目自写摘要，不复制原著长段文本。
- `deepseek_clue` 只代表线索或传闻，不能自动升为 `confirmed`。

### `HiddenFactRefState`

```ts
export interface HiddenFactRefState {
  id: string;
  scope: "world" | "region" | "npc" | "faction" | "item" | "quest";
  sourcePointer: string;
  revealPolicyId: string;
  guard: "hidden";
  lastCheckedTurn: number | null;
}
```

约束：

- 只存 `sourcePointer` 和 `revealPolicyId`。
- 不存隐藏事实正文，不给 DeepSeek 未揭示内容。

### `LivingNpcMemoryEntry`

```ts
export interface LivingNpcMemoryEntry {
  id: string;
  npcId: string;
  turn: number;
  regionId: string | null;
  actionId: string | null;
  publicSummary: string;
  privateRefId: string | null;
  attitudeDelta: number;
  weight: number;
  tags: string[];
  expiresTurn: number | null;
}
```

约束：

- `publicSummary` 是玩家可见或可被 DeepSeek 使用的摘要。
- 私密信息只存 `privateRefId`，不存正文。

### `LivingFactionPressureEntry`

```ts
export interface LivingFactionPressureEntry {
  id: string;
  factionId: string;
  pressureType: "suspicion" | "favor" | "hostility" | "opportunity";
  delta: number;
  reason: string;
  turn: number;
  visibility: "player_visible" | "system_hidden";
}
```

约束：

- `delta` 是压力账本变化，不直接等同正式关系数值。
- `system_hidden` 不进入玩家可见 UI 和 DeepSeek正文。

### `LivingPlayerGoalEntry`

```ts
export interface LivingPlayerGoalEntry {
  id: string;
  intentType: "obtain_item" | "join_faction" | "investigate" | "travel" | "long_term_goal";
  targetRef: string;
  status: LivingGoalStatus;
  createdTurn: number;
  lastUpdatedTurn: number;
  rationale: string;
  nextStepHints: string[];
  blockedByRefIds: string[];
}
```

约束：

- 极端目标如九转蛊、春秋蝉、宝黄天正式交易，默认只能进入 `blocked` 或 `deferred`，不能直接成功。
- `blockedByRefIds` 可以引用隐藏规则，但不能暴露正文。

### `LivingActionConsequenceEntry`

```ts
export interface LivingActionConsequenceEntry {
  id: string;
  actionId: string;
  turn: number;
  scope: "world" | "region" | "npc" | "faction" | "combat" | "resource";
  publicSummary: string;
  effectRefs: string[];
  followUpRefs: string[];
}
```

约束：

- 只记录已由本地 engine 验证的行动后果。
- 不在这里生成奖励、战斗结果或新地点。

### `LivingIfDeviationEntry`

```ts
export interface LivingIfDeviationEntry {
  id: string;
  anchorRefId: string;
  level: LivingIfDeviationLevel;
  reason: string;
  turn: number;
  status: "recorded" | "resolved" | "rejected";
  visibleToPlayer: boolean;
}
```

约束：

- `major` 和 `critical` 级偏离必须由后续 story/canon/IF engine 明确裁决。
- a2 只存账本，不能自己改原著锚点。

## a3 临时类型预留

a2 不实现自由意图裁决器，但要给 a3 留出字段口径。

```ts
export interface IntentCandidate {
  id: string;
  rawText: string;
  intentType: "obtain_item" | "join_faction" | "investigate" | "travel" | "long_term_goal";
  actorId: string;
  targetRef: string;
  regionId: string | null;
  source: "player_input" | "deepseek_candidate" | "engine_suggestion";
  evidenceRefIds: string[];
}

export interface IntentRuling {
  candidateId: string;
  category:
    | "available"
    | "available_with_cost"
    | "requires_prerequisite"
    | "rumor_only"
    | "long_term_goal"
    | "world_rule_blocked"
    | "hidden_fact_blocked"
    | "major_if_deviation";
  allowed: boolean;
  reasons: string[];
  prerequisiteRefs: string[];
  costRefs: string[];
  riskLevel: 0 | 1 | 2 | 3;
  visibleExplanation: string;
}
```

`IntentRuling` 默认不直接持久化。只有当玩家把裁决结果确认为长期目标时，才写入 `livingWorldState.playerGoals`。
