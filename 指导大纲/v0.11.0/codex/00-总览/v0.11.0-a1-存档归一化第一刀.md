# v0.11.0-a1 存档归一化第一刀

日期：2026-05-15
状态：已完成

## 目标

把旧文件导入、Zustand `persist.migrate()`、`persist.merge()` 和读档落地后的默认值补齐收束到同一条共享路径，降低后续新增活世界持久化字段时漏改的风险。

## 代码变更

- `src/store/index.ts`
  - 新增 `normalizePersistedGameState()`。
  - 统一补齐 `dynamicNPCs`、`npcContacts`、`targetedGuEffects`、炼养用折扣、日志、传闻地点、材料货架、势力经济 ledger、小队统计等默认值。
  - 统一归一化 `partyState`、`squadDispatchState`、`cultivationState`、`storyAnchorState`、`endingState`、`sceneSessionState`、`inheritanceLandState`、`trainingGroundState`。
  - 统一回写 story anchor / training ground 的兼容 flags 镜像。
  - `migrateSave()`、`persist.migrate()`、`persist.merge()` 和手动读档落地路径复用共享归一化函数。
- `src/store/save-normalization.test.ts`
  - 新增共享归一化测试。
  - 验证当前 `formatVersion 21` 文件导入也会走同一归一化路径。

## 存档版本

`SAVE_FORMAT_VERSION` 仍为 `21`。

本刀没有新增持久化字段，只是把已有旧档默认值与兼容镜像集中到同一处，因此不升版本。

## 验证

- `npm test -- src/store/save-normalization.test.ts src/store/slices/cultivationSlice.test.ts src/store/slices/trainingGroundSlice.test.ts`
- `npx tsc --noEmit --pretty false`
- `npm run check:player-visible-copy`
- `npm test`
- `npm run build`
- `npm run check:production-preview`

结果：全部通过。生产预览可渲染标题页，未复现 EdgeOne 类启动黑屏。

## 风险与边界

- 本刀不重构整个 store，不拆目录，不改变 localStorage key，不改 DeepSeek 行为。
- `normalizePersistedGameState()` 会比旧路径更积极地归一化当前版本档案中的 engine 子状态；这属于存档防腐目标，但后续新增字段时必须继续补测试。
- `persist.merge()` 返回值做了类型收束，避免 Zustand 泛型推断被共享 helper 带偏。

## 下一步

继续 `v0.11.0-a1` 的下一刀：response pipeline 静默异常可观测，或 localStorage key 统一管理。优先建议先做 response pipeline，因为它能更快帮助后续 DeepSeek/活世界调试定位。
