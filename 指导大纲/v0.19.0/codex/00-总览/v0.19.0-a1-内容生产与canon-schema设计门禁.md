# v0.19.0-a1 内容生产与 canon schema 设计门禁

日期：2026-05-19
状态：已完成
阶段性质：docs + tooling schema，不改 gameplay runtime

## 目标

把 v1.0 前新增内容的最低进入门槛制度化，避免事实卡、路线、压力、战斗、经济和素材继续靠聊天记忆散落推进。

## 已建立的内容模板

模板源：`src/canon/v019-content-governance-rules.json`

| 模板 | 用途 | 最低门槛 | 禁止 |
|---|---|---|---|
| `fact_card` | 原著/公开/隐藏事实卡 | summary、visibility、source pointer、runtime exposure、test sample | 原文摘录、quote、hidden body player-visible |
| `route_rule` | 路线/地点候选规则 | route id、requirements、blocked upgrades、source item、test sample | `route_entered`、`currentRoute`、location/reward |
| `pressure_rule` | NPC/势力/路线压力规则 | subject、pressure type、ledger inputs、blocked escalations | formal standing、warrant、recruitment、NPC life |
| `combat_candidate` | 战斗候选与杀招/小队样板 | entry gate、combat template、reward policy、failure reason | Gu/material drop、UI damage formula、NPC death |
| `economy_rule` | 经济/补给/炼养用规则 | resource kind、source gate、limit policy、blocked escalations | complete recipe、formal shop、commission profit、宝黄天交易 |
| `asset_manifest` | v1.0 hero/release art/runtime art 边界 | role、status、source、public path、usage、boundary | canon outcome、Immortal Gu implication、public commitment |

## 统一权威口径

- MiroFish 只能提供 quote-redacted candidate material。
- DeepSeek 只能写 narrative、candidate、clue、rumor、pressure、caption draft。
- local canon / local engine / tests 决定正式事实、奖励、地点、阵营、NPC 命运、战斗结果和公开口径。
- v0.19 保持 `SAVE_FORMAT_VERSION = 22`。

## 设计硬停

以下事项仍必须停下来让用户决策：

- 新增 route/location 持久字段。
- `SAVE_FORMAT_VERSION = 23`。
- 正式 route/location/faction/reward/NPC-life。
- BFF/backend implementation。
- EdgeOne 自动部署。
- public release commitment。
- DeepSeek authority expansion。
- 大规模新图生成或替换已选 hero。
