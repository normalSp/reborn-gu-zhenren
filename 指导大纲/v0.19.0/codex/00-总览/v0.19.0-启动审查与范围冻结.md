## 版本定位

`v0.19.0` 是 v1.0 前的 readiness 版本。它服务“公开前能不能稳定生产内容、长测、检查、展示和回滚”，不是继续开正式大地图或高阶玩法。

## 范围冻结建议

批准 v0.19 作为：

`内容生产 + canon schema + 长测 + public safety + release art pack + EdgeOne/rollback checklist`

## 非目标冻结

v0.19 不做：

- 正式路线状态字段。
- `SAVE_FORMAT_VERSION = 23`。
- 完整南疆地图或完整商家城。
- 正式阵营转移、任务、奖励、NPC 生死。
- 蛊仙期、高阶传承、仙材、仙蛊、宝黄天正式交易。
- 大规模后端/BFF、云存档、账号。
- 自动部署 EdgeOne。
- 新模型评估或模型切换。
- 对外发布承诺。

## 必须停手让用户决策

以下任何项出现，必须停止：

1. 新增/修改存档字段。
2. 开放正式 route/location/faction/reward/NPC-life。
3. 扩大 DeepSeek 权限或启动 live probe 作为 release 依据。
4. 把 MiroFish 输出升级为 runtime truth。
5. 新增 BFF/backend/云服务/外部运行时依赖。
6. 公开发布文案、公告标题、截图/短录屏对外承诺。
7. 新增大规模生成图或替换 hero 三件套。

## 专家团结论

- Project Lead：建议批准 v0.19 主线，但把“正式新玩法”全部挡在 v1.0 或后续版本之外。
- Lore & World Designer：需要 MiroFish preferred 包辅助 public copy / path anchor / art caption 边界，但不阻塞 a1。
- Systems Architect：不新增 save fields；内容校验和长测工具优先。
- AI Pipeline Architect：不切模型；如做 live narrative probe，只做固定样本评估。
- Player Advocate：v0.19 应提高走查轮次，建议 rc 100 轮。
- Frontend & Motion Lead：release art pack 做 manifest 和检查，不做大规模新图。
- QA/Test Engineering Guardian：v0.19 的主产物是测试和 preflight。
- Git/rollback Steward：每个小版本 commit/push，远端 CI 记录进仪表盘。
