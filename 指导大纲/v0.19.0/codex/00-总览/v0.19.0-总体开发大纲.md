# v0.19.0 总体开发大纲

日期：2026-05-19
状态：草案，待用户批准
主题：内容生产、长测与 v1.0 发布工具

## 背景

v0.11 到 v0.18 已经把活世界的核心闭环铺到低阶蛊师阶段：

- 自由目标与本地裁决。
- 青茅事实卡、正史锚点、IF 矩阵和隐藏事实门禁。
- NPC/势力反应、事件编年史和社会影响。
- 路线准备、候选承接、南疆低阶事实和入口边界。
- 低阶经济、炼养用、战斗、UI 减法和权威归并。

v0.19 的任务不是继续开新世界，而是把 v1.0 前最容易失控的东西工具化：内容怎么进、怎么测、怎么防泄密、怎么对外展示、怎么回滚。

## 版本目标

1. 新增事实卡、NPC、势力、路线、蛊虫、事件和素材必须进入模板与校验，不再散落在聊天里。
2. 长测从“跑 e2e”升级为代表性玩家路径、极端意图和旧档迁移的可回放矩阵。
3. 公开前检查覆盖 hidden fact、DeepSeek 权限、player-visible-copy、存档、资源/奖励/地点/阵营/NPC 生死越权。
4. v1.0 release art pack 有 manifest、用途、版权/原著边界、移动端/暗色 UI 可读性和回滚方案。
5. EdgeOne 预览和发布回滚有清单，但不做自动部署。

## 小版本范围

### a1：内容生产与 canon schema 设计门禁

- 盘点现有 canon JSON、fact card、route rules、NPC/faction rules、economy/combat rules 的 schema 状态。
- 建立 v1.0 前内容模板清单：fact card、route rule、pressure rule、combat candidate、economy rule、asset manifest。
- 冻结“内容进入 runtime”的最低门槛：source pointer、visibility、authority、test sample、DeepSeek boundary。

### a2：长测与代表性玩家路径矩阵

- 定义 v1.0 前 golden playthroughs。
- 定义 extreme intent、hidden fact probe、reward/location/faction/NPC-life overreach 样本。
- 定义旧档/公开档/移动端/reduced-motion 覆盖策略。
- 先不承诺 live DeepSeek 全量长测，只定义是否需要和何时需要。

### b1：canon/content 校验工具第一刀

- 建立或扩展本地脚本，检查内容包字段、隐藏事实、source pointer、player-visible copy 和 forbidden authority。
- 优先校验 v0.11-v0.18 已有内容，不先做大型编辑器。

### b2：长测 runner 与 Player Advocate 大走查第一刀

- 将代表性路径转成可复跑脚本或半自动走查记录。
- 建议小版本 20-30 轮，rc 100 轮，v1.0 rc 再提高。
- 若加入 live narrative probe，必须固定样本、固定本地事实输入、记录 tokens/cache/prompt hash，不允许 DeepSeek 写正式事实。

### b3：公开前 safety preflight

- 检查 player-visible-copy 中旧版本、隐藏事实、越权承诺、未开放地点/奖励/阵营/NPC 生死。
- 检查 DeepSeek 权限、prompt context、cache/usage telemetry、失败回退。
- 检查存档迁移、旧档载入、生产预览、runtime assets。

### b4：v1.0 release art pack 与公开素材候选

- 固化 hero 三件套、OG、宣传截图、短录屏、发布页素材的 manifest。
- 所有素材必须标注用途、状态、来源、是否 runtime、是否 review-only、是否 public-candidate。
- 不新增大规模生成图；如需新图或换图，单独让用户批准。

### process-1：EdgeOne 预览 smoke 与回滚清单

- 只做流程和检查清单。
- 不自动部署 EdgeOne，不改 CodeBuddy/EdgeOne 集成。
- 明确发布前、发布后、回滚、黑屏排查和 CI 证据。

### rc：v1.0 readiness review

- 汇总测试、长测、Player Advocate、MiroFish intake、release art、EdgeOne smoke、Git/CI。
- 输出 v1.0 是否可进入 rc 的建议和阻塞清单。

## DeepSeek 边界

DeepSeek 可以：

- 参与固定样本的 narrative probe。
- 生成候选文案、叙事回流、玩家可见压力描述。
- 提供 release copy 草稿，但必须过本地 player-visible-copy/safety gate。

DeepSeek 不可以：

- 决定 canon fact、hidden fact、route entered、location unlock、faction change、reward、NPC life。
- 生成未注册 Gu、材料、完整蛊方、宝黄天交易、高阶传承。
- 因 release copy 写出对外承诺。

## 完成标志

v0.19 完成时，应具备：

- v1.0 内容生产模板与校验门禁。
- 代表性玩家路径和长测矩阵。
- 公开前 safety preflight。
- release art pack manifest 和素材候选。
- EdgeOne 预览/回滚清单。
- v1.0 readiness review。
