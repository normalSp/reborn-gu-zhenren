## Same-start replayability rule

同一开局重复运行时，v1.8 允许差异来自：

- 抽到不同身份候选：临时帮工、散修短工、护送/护卫、采集跑腿、情报跑腿。
- 同一身份候选使用不同公开压力表达：盘问、短工、压价、收留、路障、传话。
- 同一稳定事实下，下一步候选顺序和文字呈现不同。
- DeepSeek 对本地给出的公开压力作叙事表达差异。

禁止差异来自：

- 正式身份、地点、阵营、奖励、NPC 生死变化。
- hidden/private 泄露。
- 未经 runtime gate 的 MiroFish item 直接进入叙事。
- 新增持久随机字段导致旧档或回滚不可解释。
- 同一存档内前后事实矛盾。

## Formal prop-word hardening rule

以下词必须按 P2/P1 风险处理，除非后续门禁单独批准正式身份系统：

- 木牌、令牌、腰牌、名册、登记、报到。
- 负责人点头、管事安排、跟队、临时帐篷。
- 商队成员、护卫身份、散修落脚点、情报人。

允许写法：

- 临时凭据。
- 需要复核的前置条件。
- 能降低盘问强度的公开理由。
- 可被拒绝或被安排杂务的候选窗口。

禁止写法：

- 正式加入商队/势力。
- 获得稳定职位、工资、身份牌或任务奖励。
- 解锁商家城或正式地点。
- 绑定命名 NPC 权威或 NPC 命运。

## b1 projection hint

若 b1 获批 runtime 第一刀，建议只实现纯 projection：

`routeLocationState + survivalEconomyState + livingWorldState + v1.3/v1.4/v1.5/v1.7 projection + localActionLedger + v1.8-a2 reviewed source pointers -> identity/replay pressure candidates`

输出只显示：

- 当前可解释的身份候选。
- 候选对应的公开压力。
- 可见前置条件。
- 拒绝/失败后的下一步。
- 同开局差异的公开来源。
- 禁止写入说明。

不写：

- 新 save 字段。
- 正式身份、职业、组织。
- 工资、价格、库存、任务奖励。
- 商家城入口、正式地点、阵营。
- NPC 生死或 hidden/private。
- DeepSeek visible MiroFish summary。
