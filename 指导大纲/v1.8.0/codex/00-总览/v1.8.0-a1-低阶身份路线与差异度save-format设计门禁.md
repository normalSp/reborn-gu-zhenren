## save-format 评估

| 方案 | 内容 | 收益 | 风险 | 专家团建议 |
|---|---|---|---|---|
| A | 保持 `SAVE_FORMAT_VERSION = 24`，b1 projection-first | 旧档零迁移；身份不误升格；能快速验证玩家理解和差异度 | 不保存长期身份历史；后续若要持久化需再开 v25 | 推荐 |
| B | 新增 `identityRouteState` / `professionState`，bump v25 | 可记录长期身份线和阶段 | 需要 migration/defaults/tests；容易把候选身份误写成正式职业/阵营 | 暂缓 |
| C | 新增 `runFingerprint` / `worldSeed` | 可审计同开局差异来源 | 需要证明收益大于 schema 风险；还要处理旧档、回放和重置语义 | 暂缓到 b2/process-2 评估 |
| D | 新增 `regionalEventLedger` | 为 v2.0 区域活世界铺路 | 过早，接近区域长期事件账本，blast radius 大 | 放入 v1.9/v2.0 future_sample_pool |
| E | 依赖 DeepSeek 自由随机制造差异 | 表面快 | 最容易导致身份、地点、奖励、hidden 和原著边界漂移 | 禁止 |

## 同开局差异度设计

定义：

同一开局、同一初始公开事实下，多次展开时应出现不同的低阶压力组合、候选顺序和叙事质感；但存档事实、身份边界、地点/阵营/奖励/NPC 生死、hidden/private 和原著边界必须稳定。

允许差异：

- 公开压力权重。
- 候选身份路线的出现顺序。
- 临时帮工、零活、护送、采集、传话等压力组合。
- 被压价、被盘问、欠人情、绕路、避祸、失败后继续推进的表达。
- DeepSeek 对本地给定压力的叙事措辞。

禁止差异：

- 玩家正式身份漂移。
- DeepSeek 自行发放正式凭据、工资、任务、奖励或地点。
- 随机揭示 hidden/private。
- 同一存档内前后矛盾。
- 未经 intake 的 MiroFish 内容进入可见叙事。

## MiroFish need level

a1 初始级别：`preferred`。

a2 已升级为 `blocking`：

`southern_border_low_rank_identity_routes_outer_edge_slice`

blocking 的含义：

- b1 runtime 前必须有 RebornG request、MiroFish quote-redacted export、RebornG intake review、测试矩阵/规则草案。
- MiroFish 输出只能进入 `candidate_pool`、`rule_draft`、`test_sample`、`deferred`、`quarantined` 或 `rejected`。
- 不得直接进入 runtime canon、DeepSeek visible context、知识库正文、UI、正式身份或正式地点/阵营/奖励/NPC 生死。
