# v1.8.0 专家团启动会纪要

状态：已召开；D-180 已获用户批准，a1 已建立
日期：2026-05-21
主题候选：低阶身份路线与同开局差异度地基

## 开会背景

v1.7 已经完成 `低阶区域活世界 projection-first 第一阶段`。它证明了：路线、生存、社会、区域、冲突和 MiroFish topic-slice 可以合流成一个小范围、可测试、可 live probe 的区域生活态势。

但 v1.7 也留下两个清晰的下一步：

1. 长期路线原本把 v1.8 定为“低阶人生分支职业与身份路线”。
2. v1.7 a1/rc 又把“同开局可重玩差异度”、`runFingerprint`、`regionalEventLedger` 放进 future_sample_pool。

专家团认为这不是冲突，而是应该合并成 v1.8 的主题：身份路线提供差异的意义，差异度地基保证同开局不会只靠 DeepSeek 即兴乱飘。

## 专家团总建议

推荐 v1.8 主线为：

`低阶身份路线与同开局差异度地基`

一句话目标：

> 让玩家在同一个低阶区域外缘里，不只是“人在路上”，而是开始拥有可解释的谋生身份、约束、风险和不同展开；同时保证稳定事实、原著边界、hidden、奖励、地点、NPC 生死不漂移。

## 角色意见

| 角色 | 意见 |
|---|---|
| Project Lead | v1.8 不宜直接进入 v2.0 区域状态，也不宜只加职业名。先把身份路线、差异度、save-format 和 MiroFish need 一次性设计清楚。 |
| Lore & World Designer | 低阶身份应是原创/边缘身份，不可写成正式原著身份、正式势力成员或命名 NPC 关系。身份差异只能来自公开压力、资源、关系、路线和风险，不来自 hidden 事实。 |
| Systems Architect | 默认继续 `SAVE_FORMAT_VERSION = 24`。`identityRouteState`、`professionState`、`runFingerprint`、`regionalEventLedger` 都要在 a1 严肃评估，不能因“长期需要”直接加字段。 |
| AI Pipeline Architect | 同开局差异不能交给 DeepSeek 自由发明。推荐先用本地 pressure deck / variant seed / projection audit 驱动，DeepSeek 只负责表达差异。v1.8 不开放 RAG、可见全书摘要或新 authority。 |
| Canon/Knowledge Curator | v1.8 如进入身份路线，MiroFish topic-slice 至少应为 `preferred`，在正式身份规则、命名地点/势力/NPC、hidden-adjacent、canon 晋升前升级 `blocking`。 |
| QA/Test Engineering Guardian | v1.8 的测试核心应是“同一开局三条展开”：稳定事实必须一致，候选压力/叙事表达应不同，正式地点/奖励/阵营/NPC 生死不得出现。 |
| Long Narrative Drift Reviewer | v1.8 应补“目标保持 + 身份约束保持”的 checkpoint。玩家选择了临时商队帮工，几十轮后不能突然变成正式商队成员或无身份散修。 |
| Player Advocate | 玩家要能看懂：我当前靠什么活、这条身份线能带来什么机会、我为什么被拒绝、下一步怎么改善，而不是只看到一堆限制。 |
| Frontend & Motion Lead | 若后续进 runtime，优先在 `活世` 或世界面板中做紧凑身份态势，不做新的大英雄页或视觉承诺。 |
| Git/rollback Steward | 启动会只做 docs/a0，分支为 `codex/v180-a0-expert-council-startup`。进入 a1 后已按制度切到 `codex/v180-a1-identity-route-save-gate`。 |
| Dev Environment Steward | v1.7 首个双仓样板已稳定，但 v1.8 仍不默认启用子代理。可在 process-1 单独给出只读/分析型子代理风险收益。 |
| Skill Sync Steward | v1.8 启动触发 expert/game-dev/lore/mirofish-export 同步；combat-motion 当前 no_update_needed，除非后续做身份 UI 动效或战斗表现。 |

## 集思广益：好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| 同开局三展开样本 | 直接回应“同样开局也要不同体验” | 只比较压力、候选、表达，不允许事实漂移 |
| 身份约束卡 | 玩家一眼看懂靠什么生存、谁会接纳、谁会压价 | 不写正式职业/阵营身份 |
| 低阶谋生牌组 | 商队短工、散修零活、采集跑腿、护送候选、传话观察 | 先 projection-first，不开收益结算 |
| 失败也推进 | 被拒绝、被盘问、低价雇佣、绕路都能产生下一步 | 不生成正式通缉、惩罚、奖励或 NPC 结果 |
| P2 正式道具词硬化 | 修复 v1.7 live probe 中木牌/令牌/登记/成员身份风险 | 进入 prompt/eval hardening，不必加 canon |
| 最小 variant seed 草案 | 给未来 `runFingerprint` 铺路 | a1 评估即可，不默认持久化 |
| 身份路线 topic-slice | 从基础包找低阶谋生、商队边缘、散修压力 source pointers | 只做 quote-redacted request/intake，不进 runtime canon |
| 只读子代理评估 | v1.7 样板稳定后，可让只读分析提速 coverage/样本审查 | 必须单独给用户风险收益，不给写权限 |

## 不推荐现在做

- 直接新增 `identityRouteState`、`professionState`、`runFingerprint` 或 `regionalEventLedger`。
- 正式职业系统、正式商队身份、正式散修势力身份。
- 完整商家城、完整南疆、完整区域事件账本。
- 正式价格、工资、库存、任务奖励。
- NPC 生死、招揽成功、通缉/封锁结论。
- DeepSeek 全书 RAG、可见知识摘要、hidden/private 可见化。
- 公开测试/BFF/backend 重构。
- 可写子代理或自动多代理生产内容。

## 专家团结论

v1.8 最稳的路线是：D-180 批准后进入 a1 设计门禁。a1 必须同时回答：

1. v1.8 是否以身份路线为主线。
2. 是否保持 `SAVE_FORMAT_VERSION = 24`。
3. 是否继续 projection-first。
4. 同开局差异度近期靠什么实现、远期哪些字段要进 v1.9/v2.0。
5. MiroFish identity-route topic-slice 是否在 a2 升级为 blocking。
6. live probe 和 Player Advocate 强度如何定。
7. 只读/分析型子代理是否值得进入单独评估。
