# RebornG v1.1 至 v2.0 长期路线重整草案

日期：2026-05-19
状态：草案；用于用户与专家团讨论 v1.1 之后的大方向，不等同于全部 runtime 承诺

## 为什么现在重整

`v1.0.0` 已完成为“活世界早期正式版”本地开发里程碑。它证明了 RebornG 的方向是可行的：

- 玩家可以从青茅山走向南疆早期路线的候选连续体验。
- 玩家自由意图可以被本地系统裁决、降级、拒绝或转为前置条件。
- 低阶修行、补给、炼养用、战斗、NPC/势力反应和公开事件能形成回流。
- DeepSeek 没有获得正式奖励、地点、阵营、NPC 生死或结局权威。

但 `v1.0.0` 仍然没有做三件大事：

- 没有正式 `route/location/currentRegion` 持久字段。
- 没有完整南疆、完整商家城、完整低阶区域社会。
- 没有任意时代、任意身份、任意地域的“大蛊界”开局。

因此 `v1.1` 之后不能继续只按局部功能推进，必须先把长期路线重整为“目标、能力层、版本节奏、硬停门禁”。

## 长期愿景

最终目标不是普通 AI 跑团，也不是只会聊天的剧情生成器。

RebornG 的长期目标是：

> 把蛊真人世界转化为一个可被玩家进入、行动、受限、被记住、被反应的活世界。玩家可以选择一个身份，在原著世界观、资源限制、境界差距、势力压力和正史因果下开启另一段人生。

这个愿景包含四层含义：

1. 自由表达：玩家能说“逃离青茅山”“投靠白家”“跟踪方源”“去商家城”“寻找盗天传承”。
2. 本地裁决：系统判断信息、境界、资源、身份、因果和世界规则，而不是让 DeepSeek 随口满足。
3. 长期后果：玩家行为进入存档、账本、NPC 记忆、势力压力、路线状态和公开编年史。
4. 原著稳定：正史事实、隐藏事实、IF 偏离和未开放高阶内容不会被 UI、DeepSeek 或玩家输入随意改写。

## 不能做成什么

以下方向不符合 RebornG 的核心路线：

- 纯 AI 长上下文人生模拟：容易剧情崩坏、事实漂移、长期记忆污染。
- 纯文本随机跑团：自由感高但世界规则和数值不可验证。
- 大量 agent 自动写内容：短期产量高，但 canon、存档、测试和 UI 权威容易散。
- 过早开放尊者时代、蛊仙期、宝黄天或任意人物附身：会绕过低阶地基，导致世界观和系统失控。

## v1.x 的主轴

v1.x 不追求“完整蛊界”，而是把“青茅之后的低阶人生”做成可信的活世界样板。

主轴如下：

| 轴 | 目标 | 说明 |
|---|---|---|
| 路线与地点 | 让世界承认玩家在哪里、从哪里来、要去哪 | `v1.1` 的核心 |
| 低阶生存 | 修行、补给、蛊虫维护、炼养用、交易压力形成闭环 | `v1.2` 起逐步正式化 |
| 社会关系 | NPC、势力、公开事件长期记住玩家 | `v1.3` 起深化 |
| 区域活性 | 南疆早期路线、商队、散修、城外缘、势力接触 | `v1.4` 起扩展 |
| 冲突闭环 | 战斗、杀招、追杀、小队、阵法和后果可解释 | `v1.5` 起深化 |
| 内容生产 | MiroFish intake、canon schema、长测、样本矩阵增长 | 贯穿 v1.x |
| 发布与架构 | 公开测试、安全、观测、可能的薄 BFF | 只在触发门禁时推进 |

## 建议大版本路线

### v1.1：路线、地点与区域状态地基

目标：让世界正式承认“玩家离开青茅山后处于哪里、走在什么路上、面对什么区域压力”。

核心问题：

- 是否引入 `SAVE_FORMAT_VERSION = 23`。
- 是否新增正式 `route/location/currentRegion` 或等价字段。
- 如何迁移旧档。
- 如何避免“路线候选”直接变成“正式地点解锁”。

完成标志：

- 玩家不再只看到“准备离开青茅山”的候选说明，而能看到正式的路线/地点/区域状态第一刀。
- 旧档可迁移，未离开青茅山的旧档不会被错误传送。
- DeepSeek 仍不能写正式地点、路线成功、阵营、奖励或 NPC 生死。

### v1.2：低阶蛊师生存与经济正式化第一阶段

目标：把 v1.0 的低阶 life loop 从验收/候选态推进到更正式的低阶生存循环。

可能交付：

- 小型正式补给/库存字段评审。
- 蛊虫喂养、真元消耗、资源缺口、简单交易窗口的正式边界。
- 市场/黑市/委托仍不做完整系统，先做小范围可验证样板。

硬停：

- 正式货币、库存、市场、价格表、委托收益都可能涉及存档和经济套利，必须单独门禁。

### v1.3：NPC 与势力长期关系第二层

目标：让“人”和“组织”更像活着，而不是只读提示。

可能交付：

- 命名 NPC allowlist。
- 关系证据、怀疑、信任、利益、恐惧、交易倾向。
- 势力封锁、招揽、通缉、接触窗口的前置条件。
- 行动后的公开摘要进入编年史。

硬停：

- 不直接开放正式阵营转移、通缉结论、招募成功、NPC 生死。

### v1.4：南疆早期低阶区域样板

目标：玩家离开青茅后，能在南疆低阶区域中看到多个可行动、可受限、可反馈的区域格。

可能交付：

- 商队路线、山路、散修聚点、城外缘、低阶资源点。
- 区域公开事实卡。
- 区域危险、补给、追踪、身份检查。
- 第一批区域事件模板。

硬停：

- 不做完整南疆，不做完整商家城，不做全地图。

### v1.5：冲突、追杀、杀招与小队后果深化

目标：冲突不只是战斗 UI，而是和路线、身份、势力、资源绑定。

可能交付：

- 低阶追杀/截杀样板。
- 杀招反制和失败代价样板。
- 小队/阵法候选进入更正式的前置条件。
- 战斗结果写回公开事件、补给、伤势、关系压力。

硬停：

- 不开放掉落池、稀有蛊、仙蛊、正式 NPC 生死，除非单独批准。

### v1.6：内容生产、canon schema 与长测工厂

目标：随着世界变大，内容不会靠临时手写和聊天记忆维持。

可能交付：

- MiroFish 包分类、intake、promote、quarantine 的工具化。
- 全书基础包库存审计与主题切片工具化，例如检查 `指导大纲/vMiroFish/基础包/` 文件数、manifest、coverage、quote-redacted 状态和 intake 引用链。
- 原著事实卡 schema 扩展。
- 长线 Player Advocate 自动样本。
- DeepSeek 固定 eval / live eval / replay 归档增强。
- 内容候选从 `candidate_pool` 到 `rule_draft` 到 `test_sample` 的标准流。
- 过期入口自动检查脚本，例如 `scripts/check-doc-entrypoints.mjs` / `check:doc-entrypoints`，检查当前入口是否错误引用历史流程文件。

硬停：

- MiroFish 仍不是 runtime 权威，不直接进入 DeepSeek visible context。
- 全书基础包不得整包导入知识索引、prompt 或 runtime；只能按主题切片进入 intake。
- 过期入口检查脚本在规则稳定前不加入 CI 硬门；是否进入 GitHub Actions 需用户另行批准。

### v1.7：公开测试与薄后端/BFF 门禁

目标：如果项目进入更公开的测试阶段，开始严肃处理安全、密钥、隐藏事实和云存档。

可能交付：

- API key 保护评审。
- hidden/private canon 是否应离开公开前端包。
- 云存档和账号是否必要。
- EdgeOne/边缘函数/BFF 可行性评审。

硬停：

- 不因为“后端看起来专业”而重构。只有公开安全、私有事实、云存档、成本观测成为硬需求时才做。

### v1.8：低阶人生分支职业与身份路线

目标：让玩家不是只有“逃亡者/散修”一种低阶人生。

可能交付：

- 商队学徒、散修、家族边缘人、低阶护卫、采集者、情报贩子等身份路线。
- 每类身份有不同资源、风险、关系和目标。

硬停：

- 不做任意人物附身；身份仍是项目允许的原创/边缘身份。

### v1.9：v2.0 区域活世界预备

目标：为 v2.0 的第一个“区域活世界”做收束。

可能交付：

- 选定第一个 v2.0 核心区域。
- 整理区域事件、NPC、势力、资源、路线、战斗、经济、公开事实和隐藏事实。
- 做 v2.0 门禁和迁移预案。

### v2.0：第一个区域活世界

目标：不是完整蛊界，而是第一个真正能长期运行的区域活世界。

完成标志：

- 一个低阶区域可以支持玩家连续游玩较长周期。
- 玩家行为被路线、NPC、势力、经济、战斗、公开事件和隐藏事实共同承接。
- 玩家能感觉自己不是在刷面板，而是在蛊世界中生活。
- DeepSeek 叙事增强世界质感，但不掌控世界事实。

## 任意时代开局的远期定位

用户提出的终极愿景包括：玩家可以选择任意时代、任意身份，甚至盗天魔尊时期、元始仙尊时期、路边凡人或蛊仙身份。

专家团判断：

- 这是正确的远期愿景，但不是 v1.x 的主线。
- v1.x 只能为它铺“时代锚点、身份边界、隐藏事实保护、自由意图裁决、后端私有事实服务”的地基。
- 真正的大时代开局至少应排到 v2.0 之后。

远期进入条件：

1. 低阶区域活世界跑通。
2. 大时代原著事实抽取完成。
3. 尊者核心因果保护完成。
4. hidden/private canon 不再完全暴露在公开前端。
5. 玩家身份边界和不可替代尊者因果的裁决样本成熟。

## v1.1 的位置

`v1.1` 是 v1.x 的第一块地基。

它不负责让世界变大，而负责让世界开始承认：

- 玩家在哪里。
- 玩家从哪里来。
- 玩家走的是哪条路线。
- 玩家当前区域压力是什么。
- 旧账本如何影响路线和地点。

如果这一步不稳，后面的经济、NPC、区域、战斗和身份路线都会悬空。

## 当前需要用户继续讨论的问题

1. v1.1 是否只允许设计门禁评估 `SAVE_FORMAT_VERSION = 23`，还是允许在 b1 后实际进入字段实现。
2. `route/location/currentRegion` 是否必须是 v1.1 的正式字段，还是先用更保守的 `routeState` 聚合字段。
3. v1.1 是否只处理青茅 -> 南疆早期，不碰完整南疆和商家城。
4. v1.2 是否优先做正式补给/库存，还是先继续只读/候选态经济。
5. v2.0 第一个区域活世界应选“南疆低阶区域”还是另一个更小、更稳的区域样板。

## 外部参考

外部活世界参考不直接进入 runtime。其设计思想已单独整理到：

`指导大纲/长期路线/RebornG-外部活世界参考映射.md`

当前结论：

- v1.1 主要吸收 ReAct、Reflexion、Voyager，用于意图裁决链路、测试矩阵回流和专家团能力成长。
- v1.3 之后重点吸收 Generative Agents、Concordia、AI Town，用于 NPC/势力社会层和玩家可见行为。
- v1.6 之后再评估 AutoGen、MetaGPT 的只读审查/流程分工思想，不急着上可写多代理。
