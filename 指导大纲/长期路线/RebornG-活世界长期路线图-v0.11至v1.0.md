# RebornG 活世界长期路线图：v0.11 至 v1.0

日期：2026-05-19
状态：长期路线；已同步到 v0.18 完成事实和 v0.19 启动审查草案，后续每个版本仍需单独冻结范围。

## 总目标

RebornG 要从“AI 叙事 + 本地系统样板”进化成“活着的蛊真人世界”。

活世界的最低定义：

- 世界有时间流逝。
- 区域、NPC、势力能记住玩家行为。
- 玩家能表达自由目标。
- 系统能判断目标是否可行、是否越界、能否降级为当前阶段行动。
- DeepSeek 不直接创造事实，只负责候选和叙事。
- 原著事实来自项目真相源，不靠聊天记忆。
- 玩家行动会留下长期后果，而不是每一轮都重开一段孤立文本。

## 架构演进原则

RebornG 不为了未来想象提前重构成重后端。v1.0 前默认以纯前端、本地确定性 TypeScript 引擎、静态 canon、Zustand 存档和 DeepSeek API 为主。

但从 v0.12 开始，所有 core 模块必须具备未来迁移能力：

- engine/canon helper 保持纯 TypeScript、纯函数优先、无 DOM、无浏览器 API 强依赖。
- UI 只消费 engine 输出，不拥有私有公式和结算权。
- public canon 与 hidden/private canon 概念分离。
- hidden fact body 不应成为公开前端长期必需数据。
- DeepSeek 代理、API key 保护、模型成本观测、云存档和私有事实服务未来可进入薄后端。
- 后端/BFF 不接管游戏裁决权，只承载密钥、观测、云存档和私有内容下发。

详细架构路线见 `RebornG-长期架构演进路线图-纯前端到薄后端.md`。

## 长线叙事防崩坏路线

RebornG 不把外部 AI 人生/跑团项目中的“0 岁到多年人生模拟”作为近期目标。当前的长线压力是：玩家在青茅山连续行动几十到几百轮后，正史事实、隐藏事实、NPC/势力反应、资源压力、路线准备和 IF 后果不能互相打架。

系统提示、Zod Schema、语义规则、金丝雀断言和双轮裁判只能拦住格式错误、典型越权和常见崩坏，不能单独保证长篇人生连续性。真正的防崩坏地基必须由本地结构化层承担：

- `v0.12`：正史锚点表、青茅低阶 IF 矩阵、route/supply/pursuit 事件链、轻量 reaction bridge。
- `v0.13`：正式 NPC 记忆和势力反应引擎。
- `v0.14`：地域路线承接，让青茅后续路线带着前期账本进入南疆。
- `v0.15`：蛊师经济/炼养用深循环，避免长期行动只剩文本。
- `v0.16`：系统收束、UI 减法与权威归并，保证后续系统不继续堆成工具台。
- `v0.17`：战斗/杀招/小队/阵法深化，保证冲突后果可解释。
- `v0.18`：南疆路线与多区域承接，先把青茅后续低阶人生做连续。
- `v0.19`：内容生产、canon schema、长测、公开测试工具和 v1.0 发布素材。
- `v1.0`：青茅+南疆主路线可连续体验，玩家自由意图、NPC/势力记忆、蛊师闭环和正史/IF 共存。

详细补充见 `RebornG-长线叙事防崩坏与青茅IF矩阵落位.md`。

## 大版本路线

### v0.11：活世界内核与自由意图闸门

目标：世界开始能记住、能判断、能拒绝、能反馈。

核心交付：

- 玩家可见旧债清理。
- 存档与持久化加固。
- 活世界状态协议。
- 自由意图裁决引擎。
- 原著事实抽取试点。
- 青茅山活世界最小闭环。

完成标志：

- 玩家提出“我要投靠白家 / 跟踪方源 / 逃离青茅山 / 获取九转蛊”这类目标时，系统能给出符合境界、信息、资源和世界观的判断。
- 不可行目标能被降级为传闻、长期目标、风险行动或明确阻断。
- 关键状态能进存档，旧存档能迁移。

### v0.12：青茅山正史锚点与 IF 框架

目标：青茅山不再只是区域样板，而是有原著事实来源、正史压力和 IF 偏离空间。

核心交付：

- 原著青茅山事实卡抽取。
- 正史锚点表。
- 玩家可见事实/隐藏事实分离。
- 青茅低阶 IF 矩阵、偏离规则和代价。
- 青茅山关键压力链：族学、三寨、狼潮、资源、战后路线等，以原著抽取为准。

完成标志：

- 系统能区分“正史硬事实”“玩家可见事实”“隐藏事实”“可 IF 偏离点”。
- 玩家可以侧翼参与青茅局势，但不能随口改写正史核心因果。
- 旧 v0.8 的 `IfBranchVector` 思想被低阶化为 `npc_attention`、`faction_pressure`、`resource_control`、`route_escape`、`hidden_fact_probe`、`local_survival`、`canon_anchor_pressure` 等青茅轴。

架构职责：

- 不引入后端。
- public fact / hidden fact / IF deviation point 分类必须具备未来分包能力。
- hidden fact body 不应成为公开前端长期必需数据。
- 正史锚点表和事实卡必须有 schema/test。

### v0.13：NPC 与势力反应系统

目标：世界开始像有人在里面生活。

核心交付：

- NPC 记忆、印象、关系、怀疑、利益驱动。
- 势力声望、封锁、招揽、通缉、任务来源。
- 行动后果对 NPC/势力的长期影响。
- 事件编年史和公开行动摘要，不依赖 DeepSeek 长上下文硬记。
- 早期 Player Advocate 走查：玩家是否看得懂自己影响了什么。

完成标志：

- 同一玩家行为能让不同 NPC/势力给出不同反应。
- 玩家长期撒谎、偷窃、投靠、救援、背叛会进入关系账本。

架构职责：

- NPC / faction reaction engine 保持纯 TS core，不绑 UI。
- NPC memory / faction pressure 写入必须通过受控 patch。
- 为未来云存档保留稳定 action ledger 和 migration 口径。

v0.13 落地切分建议：

- `a1`：社会记忆协议、字段表、测试矩阵，不先写 runtime。
- `a2`：NPC 记忆投影引擎第一刀。
- `b1`：势力态度 / 压力投影第一刀。
- `b2`：事件编年史与公开行动摘要。
- `b3`：后续行动候选。
- `b4`：Player Advocate 可读性走查 UI。
- `process-1`：GitHub Actions 远端首次运行与门禁复核。
- `rc`：质量收束。

边界：

- v0.13 第一阶段优先复用 v22 已有字段，不急着新增关系账本。
- 正式阵营变化、正式通缉、任务奖励、NPC 生死和公开声望数值必须单独决策。

### v0.14：青茅后续路线承接

状态：已完成。

目标：青茅山后续路线不再只是文本愿望，而能被本地条件、账本、前置和风险解释承接。

核心交付：

- 路线条件表和账本读取。
- 候选后续升级为正式前置行动样板。
- 青茅离开路线第一刀。
- 阵营目标前置条件展示。
- FreeGoalPanel 优先摘要。
- 高风险阶段 20 轮 Player Advocate，rc 60 轮。

完成标志：

- 玩家可以看到逃离、投靠、商队、商家城、身份过渡等长期目标的前置条件。
- 系统仍不直接开放正式路线状态、地点切换、阵营转移、奖励或 NPC 生死结果。

### v0.15：低阶蛊师经济、补给、炼养用深循环

状态：已完成。

目标：蛊师阶段先完整，不急着跳蛊仙。

核心交付：

- 低阶经济候选规则池与只读 helper。
- 补给/喂养缺口行动样板。
- 残方/炼蛊失败代价边界。
- 商队/市场候选窗口。
- 灰色交易/委托边界。
- 美术资源盘点和 v1.0 hero 三件套候选治理。

完成标志：

- 玩家成长不只靠剧情奖励，也开始面对补给、食料、炼养用、交易窗口和灰色边界。
- 仍不开放正式 inventory/currency 消耗、价格表、市场库存、黑市、委托收益、身份洗白、正式奖励或 DeepSeek 新权限。

### v0.16：系统收束、UI 减法与权威归并

目标：让现有功能更像一个游戏，而不是工具台。

核心交付：

- 主导航减法：目标为 `地图 / 行动 / 角色 / 蛊道 / 世界 / 记录`。
- `行动 + 自由目标` 合并设计与第一刀。
- `青茅 / 凡战 / 群像 / 大阵` 迁出正式底栏。
- `蛊虫 / 杀招 / 炼蛊 / 蛊材` 归并为 `蛊道` 工作台。
- `传承 / 宿命 / 终局 / 商会` 按境界、剧情、场景化显示。
- 旧 NPC/势力系统与 `livingWorldState` 权威归并表。
- 旧 material/shop/refine/economy 系统与 v0.15 低阶经济权威归并表。
- UI 减法后的 Player Advocate 走查。

完成标志：

- 玩家底栏不再暴露大量系统按钮。
- 玩家想“做事”时优先进入统一 `行动` 工作台。
- 自由目标、线索后续、路线候选、市场窗口、社会机会在统一行动语义下出现。
- 调试入口不混入正式玩家导航。
- 后续正式 NPC/势力和经济系统开工前，权威边界清楚。

架构职责：

- 不借减法扩大 DeepSeek 权限。
- 不借减法新增存档字段，除非用户单独批准。
- 不做大规模 engine 目录重组。
- 若触及玩家 UI，必须有 e2e 和 Player Advocate gate。

详细草案见 `RebornG-v0.16至v1.0减法专项与路线调整草案.md`。

### v0.17：战斗、杀招、小队与阵法深化

目标：冲突有策略，失败有代价。

核心交付：

- 5x3/可变棋盘继续深化。
- 多单位、小队、追击、撤退、重伤、士气。
- 杀招组合、反制、风险、消耗。
- 阵法/地形/遮蔽/观察的本地规则。

完成标志：

- 战斗不只是“成功/失败”文本，而是能解释位置、目标、代价、反制和后果。

架构职责：

- 战斗 core 与 UI/动效继续分离。
- battle trace 可重放。
- GSAP/Motion 只能展示 engine 事实，不能改变结算。

### v0.18：南疆路线与多区域承接

目标：把青茅山之后的人生路线打开，但仍保持正史/IF 边界。

核心交付：

- 正式路线状态设计评审。
- 青茅离开后的路线进入：商队、散修、商家城外缘、三寨残余等。
- 南疆低阶区域事实卡与公开/隐藏事实分离。
- 前期行动账本影响后续路线、身份、补给、追索和 NPC/势力反应。
- 如需新增 route/location/save 字段，必须单独停下让用户批准。

完成标志：

- 玩家可以从青茅山后续路线进入下一阶段，但不是纯文本跳转。
- 前期账本会影响路线条件和风险。
- 不开放完整南疆大地图，不开放高阶路线，不开放蛊仙期。

架构职责：

- 开始评估薄 BFF，但不默认开工。
- 评估触发条件：云存档、跨设备、hidden fact 保护、DeepSeek key 保护。
- 路线/地域状态设计要兼容未来云端同步。

### v0.19：内容生产、长测与 v1.0 发布工具

目标：让 v1.0 前的内容扩展、测试、资产和公开准备可控。

核心交付：

- 长测脚本。

- 公开前 player-visible-copy、hidden fact、DeepSeek 权限、存档迁移总检查。

完成标志：

- 新增区域、NPC、蛊虫、事件可以按模板进入审查，而不是散落在聊天里。
- v1.0 有可回放、可审查、可回滚的发布准备。

架构职责：

- GitHub Actions / CI。
- canon schema。
- 长测、旧档迁移、production preview。
- public/private canon 分包检查。
- 技术文档和发布前风险审查。

当前 v0.19 草案建议：

- a1/a2 先做内容生产 schema 和代表性长测矩阵，不动 runtime。
- b1/b2/b3 逐步做 canon/content 校验工具、长测 runner、公开前 safety preflight。
- b4 整理 v1.0 release art pack，但不做大规模新图生成。
- process-1 只做 EdgeOne 预览 smoke 与回滚清单，不自动部署。
- rc 做 v1.0 readiness review。

### v1.x 或 v2.0 之后：蛊仙期和高阶世界门槛

目标：等蛊师阶段稳定后，再把蛊仙期作为真正可玩系统开放。

延期原因：

- 当前 v1.0 的最低目标是青茅山 + 南疆早期路线 + 低阶蛊师闭环。
- 仙窍、宝黄天、仙材、仙蛊、高阶隐藏事实需要更强 private canon / BFF / 权限边界。
- 过早开放蛊仙期会让低阶体验和 v1.0 发布目标失焦。

高阶开放前必须重新评审：

- 宝黄天门槛。
- 仙材、仙蛊、仙蛊方、杀招传承的高阶门禁。
- 蛊仙经济与凡人经济隔离。

### v1.0：活世界早期正式版

- 正史锚点和 IF 偏离能共存。
- UI 不再暴露大量未开放、调试或高阶入口。
- 存档稳定，旧档迁移可测。

架构最低定义：

- API key 不直接暴露在公开前端包。
- hidden fact 不以玩家可直接阅读的完整形式下发。
- 关键 action ledger 可导出、回放或云端备份。
- 发布版有最小错误观测和版本回滚预案。
