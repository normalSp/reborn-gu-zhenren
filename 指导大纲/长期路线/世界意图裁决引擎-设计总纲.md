# 世界意图裁决引擎设计总纲

日期：2026-05-16
状态：长期系统设计；v0.11 起步。

## 一句话定义

世界意图裁决引擎是 RebornG 处理玩家自由目标的本地系统。

玩家可以说“我要投靠白家”“我要跟踪方源”“我要逃离青茅山”“我要拿九转蛊”，但系统必须判断：

- 当前能不能做。
- 为什么不能做。
- 能不能降级为当前阶段行动。
- 是否只是传闻/长期目标。
- 是否触发风险、被骗、暴露、结仇或 IF 偏离。

## 权限分工

### DeepSeek 的职责

DeepSeek 可以：

- 理解玩家自然语言。
- 把玩家目标改写成结构化意图候选。
- 给出叙事解释。
- 生成 3-5 个候选行动或替代路径。
- 根据本地裁决结果写回流文本。

DeepSeek 不可以：

- 直接决定行动成功。
- 直接发放蛊虫、蛊方、材料、仙蛊、奖励、地点权限。
- 直接改变 NPC 生死、正史锚点、结局、境界、势力归属。
- 直接把隐藏事实告诉玩家。

### 本地引擎的职责

本地引擎负责：

- 意图分类。
- 世界观门槛判断。
- 境界/资源/地点/关系/时间/身份校验。
- 正史与 IF 边界判断。
- 可执行行动生成或阻断。
- 状态写入和后果结算。

### canon 数据的职责

canon 数据负责：

- 蛊虫、材料、地点、势力、NPC、路径、正史锚点的真相源。
- 原著事实来源和 fan-made 边界。
- 玩家可见事实与隐藏事实的分离。
- 禁止项、门槛和可降级路径。

### store/save 的职责

store/save 负责持久化：

- 玩家已知事实。
- 世界隐藏事实状态。
- 区域压力。
- NPC 记忆。
- 势力关系。
- 玩家长期目标。
- IF 偏离账本。
- 行动后果账本。

## 部署与迁移边界

当前阶段，世界意图裁决引擎运行在前端本地。长期要求是：即使将来引入薄后端/BFF，裁决核心仍然保持同一套 TypeScript core 和测试语义。

### 当前前端职责

- 本地解析已结构化意图。
- 调用 canon/engine 校验门槛。
- 生成可执行、阻断、降级或长期目标结果。
- 通过受控 patch 写入 store/save。

### 未来后端/BFF 可承担

- DeepSeek / LLM API 代理，保护 key。
- hidden/private canon 按玩家可见范围下发。
- prompt hash、token、cache hit/miss、retry cost 和失败原因集中观测。
- live eval / replay eval 归档。
- 云存档和 action ledger 备份。

### 未来后端/BFF 不可承担

- 绕过本地 canon/engine 直接发奖励。
- 直接改变 NPC 生死、正史锚点、结局、境界、势力归属。
- 直接把 hidden fact body 暴露给玩家或 DeepSeek。
- 让 UI 或模型拥有最终裁决权。

### 迁移要求

- 裁决函数应尽量保持纯函数、输入输出明确。
- 不直接依赖 DOM、localStorage、React hook 或浏览器全局对象。
- UI 层只消费裁决结果，不写私有规则。
- 所有新增裁决规则必须有测试样本。

## 意图分类

第一阶段建议支持这些意图：

- `investigate`：调查事实、地点、人物、传闻。
- `travel`：前往地点、逃离区域、寻找路线。
- `join_faction`：投靠、结盟、求庇护。
- `betray_or_attack`：刺杀、抢夺、背叛、伏击。
- `obtain_item`：获得蛊虫、蛊方、蛊材、传承、线索。
- `train_or_cultivate`：修炼、突破、训练、炼蛊、喂养。
- `trade`：购买、出售、交换、委托。
- `social`：结交、试探、威胁、求助。
- `long_term_goal`：立道途目标、远期野心。
- `unknown_or_invalid`：无法解析或越界妄想。

## 裁决结果

本地裁决输出建议分为：

- `executable_now`：当前可执行。
- `executable_with_cost`：可执行，但有资源、时间、关系或风险代价。
- `requires_precondition`：需要先满足条件。
- `rumor_only`：只能打听或记录为传闻。
- `long_term_goal_only`：只能作为长期目标。
- `blocked_by_lore`：世界观硬阻断。
- `blocked_by_visibility`：玩家不该知道隐藏事实。
- `major_if_deviation`：重大 IF 偏离，需要特殊条件和后果账本。
- `dangerous_attention`：目标本身会引来骗子、势力、追查或天意压力。

## 示例

### 玩家：我要拿九转蛊

解析：

- intent：`obtain_item`
- targetTier：九转/仙蛊级高阶目标
- desire：直接获得

裁决：

- 当前一转，无法接触。
- 无仙窍、仙元、蛊仙交易资格。
- 不允许直接获得。
- 可降级为传闻调查、长期目标、被骗风险、低阶认知学习。

候选：

- 去酒肆打听高阶传闻。
- 查族学残卷，了解“仙蛊”概念。
- 将其记为长期道途目标。
- 向可疑散修打听，承担被骗和暴露风险。

### 玩家：我要投靠白家

裁决需要看：

- 玩家身份。
- 当前地点。
- 白家关系。
- 是否有引荐人。
- 当前三寨压力。
- 是否背叛原势力。

结果可能是：

- 古月族人：可尝试接触，但高风险，可能触发叛逃嫌疑。
- 白家出身：可走本家任务。
- 散修：可尝试求庇护，但需要资源/情报/功绩。
- 商队出身：可通过贸易关系侧面接触。

### 玩家：我要跟踪方源

裁决需要看：

- 玩家是否知道方源。
- 方源是否在玩家可见范围。
- 玩家是否有跟踪能力。
- 当前正史/IF 模式。
- 是否会泄露原著隐藏事实。

结果可能是：

- 可尝试观察古月族学弟子中的异常人物。
- 可接收“目标警觉、风险高”的警告。
- 不能直接告诉玩家“方源是重生者”。

## v0.11 起步范围

v0.11 不做全能力意图系统，只做青茅山低阶最小闭环：

- `obtain_item`
- `join_faction`
- `investigate`
- `travel`
- `long_term_goal`

首批极端样本：

- 我要拿九转蛊。
- 我要获得春秋蝉。
- 我要去宝黄天交易。
- 我要投靠白家。
- 我要跟踪方源。
- 我要逃离青茅山。
- 我要杀死白凝冰。

这些样本用于测试世界观闸门，不表示 v0.11 会开放对应剧情。

## 验收标准

- DeepSeek 输出越界候选时，本地引擎能拦截。
- 玩家看见的是清晰的阻断原因和替代路径。
- 不可行目标不会进入背包、奖励、地点、NPC 生死或正史结果。
- 重大 IF 偏离只进入候选或压力账本，不直接改世界。
- 所有裁决有测试样本。
