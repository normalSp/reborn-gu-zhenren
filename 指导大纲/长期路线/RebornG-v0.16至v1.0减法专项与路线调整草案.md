## 1. 结论

减法确实会影响后续开发，但不应把它理解成“额外插入一个庞大重构”。更准确的定位是：`v0.16` 应该变成一个轻量但硬的“系统收束与 UI 减法专项”，先把现有功能的入口、权威和玩家心智模型理顺，再继续推进战斗、南疆路线、多区域锚点和 v1.0 发布准备。

我的判断：

1. 需要调整后续版本路线。
2. 不需要推翻 v1.0 总目标。
3. 不建议把减法拖到 v1.0 前夕再做。
4. 不建议把减法混在战斗深化里顺手做完。
5. 建议 `v0.16` 主线改为 `系统收束、UI 减法与权威归并`。
6. 原长期路线中的战斗深化顺延到 `v0.17`。
7. 南疆路线/多区域承接应放在战斗和 UI 收束之后，建议 `v0.18`。
8. 蛊仙期和高阶世界门槛不应放在 v1.0 前，建议移到 v1.x 或 v2.0 之后。

这不是拖慢，而是防止后面每新增一个系统都继续往底栏加按钮、继续增加旧系统与新系统的权威重叠。

## 3. 为什么不能只做一个小补丁

如果只是把几个按钮藏起来，会出现三个问题：

1. UI 看起来少了，但系统权威仍旧重复。
2. 玩家入口少了，但底层 `ActionPanel`、`FreeGoalPanel`、旧 `uiSlice.activeTab`、旧 NPC/经济状态仍可能互相打架。
3. 后续战斗、路线、市场、NPC 关系继续新增时，入口会再次膨胀。

所以减法至少要覆盖三层：

- 玩家入口层：哪些按钮出现，哪些收起，哪些按境界/场景出现。
- 系统权威层：同一事实到底由哪个 engine/store/canon 决定。
- 测试与走查层：玩家是否真的看得懂下一步，而不是只看到按钮变少。

但这也不等于大规模重构。`v0.16` 应该避免以下事项：

- 不新增存档字段，除非用户单独批准。
- 不重组整个 `src/engine` 目录。
- 不改写所有旧系统。
- 不做完整新 UI 皮肤。
- 不开放正式路线、地点、阵营、NPC 生死或奖励。
- 不扩大 DeepSeek 权限。

## 4. 调整后的大版本路线

### v0.16：系统收束、UI 减法与权威归并

目标：让现有功能更像一个游戏，而不是工具台。

核心交付：

- 主导航减法：目标为 `地图 / 行动 / 角色 / 蛊道 / 世界 / 记录`。
- `行动 + 自由目标` 合并设计与第一刀。
- 调试/演示入口迁出正式底栏。
- `蛊虫 / 杀招 / 炼蛊 / 蛊材` 归并为 `蛊道` 工作台。
- `传承 / 宿命 / 终局 / 商会` 境界、剧情、场景化显示规则。
- 旧 NPC/势力系统与 `livingWorldState` 权威归并表。
- 旧 material/shop/refine/economy 系统与 v0.15 低阶经济权威归并表。
- UI 减法后的 Player Advocate 走查。

完成标志：

- 玩家底栏不再暴露大量系统按钮。
- 玩家想“做事”时优先进入统一 `行动` 工作台。
- 自由目标、线索后续、路线候选、市场窗口、社会机会能在同一行动语义下出现。
- 蛊虫、杀招、炼蛊、蛊材不再割裂为多个主入口。
- 调试入口不混入正式玩家导航。
- 后续正式 NPC/势力和经济系统开工前，权威边界清楚。

不做：

- 不做正式地点切换。
- 不做正式阵营转移。
- 不做 NPC 生死/捕获结论。
- 不新增正式奖励。
- 不新增 DeepSeek 权限。
- 不把候选/只读系统伪装成已发生事实。

建议切分：

| 小版本 | 主题 | 说明 |
|---|---|---|
| v0.16.0-a1 | 信息架构与权威表设计门禁 | 写主导航、面板归并、旧系统权威表、测试矩阵 |
| v0.16.0-a2 | 仪表盘与流程入口收束 | 根目录 `项目仪表盘.md` 成为唯一当前仪表盘入口 |
| v0.16.0-b1 | `行动 + 自由目标` 合并第一刀 | 玩家操作上合并，底层权威不扩大 |
| v0.16.0-b2 | `蛊道` 工作台第一刀 | 合并蛊虫、杀招、炼蛊、蛊材入口 |
| v0.16.0-b3 | 高阶/调试入口场景化 | 青茅/凡战/群像/大阵移入开发抽屉，传承/宿命/终局/商会按条件显示 |
| v0.16.0-b4 | NPC/势力与经济权威归并验证 | 不一定改大量代码，重点是 authority map 和防重复测试 |
| v0.16.0-rc | UI 减法质量收束 | e2e、Player Advocate、视觉可读性、Git/CI 收束 |

### v0.17：战斗、杀招、小队与阵法深化

目标：在 UI 与权威收束后，让冲突有策略，失败有代价。

核心交付：

- 低阶战斗正式化，不再只是竖切和候选入口。
- 蛊虫、杀招、资源消耗、反制、地形和行动账本连接。
- 战斗结果回流 NPC/势力、路线风险、经济消耗。
- 小队、撤退、追击、重伤、士气第一层。
- 战斗 UI 从 `行动` 与 `蛊道` 的新结构中进入，不再靠演示按钮。

完成标志：

- 玩家能理解每场战斗为什么发生、能做什么、损失什么、影响谁。
- 战斗不是纯文本成功/失败，也不是单独演示场。

### v0.18：南疆路线与多区域承接

目标：把青茅山之后的人生路线打开，但仍保持正史/IF 边界。

核心交付：

- 正式路线状态设计评审。
- 青茅离开后的路线进入：商队、散修、商家城外缘、三寨残余等。
- 南疆低阶区域事实卡与公开/隐藏事实分离。
- 前期行动账本影响后续路线、身份、补给、追索和 NPC/势力反应。
- 如需新增 route/location/save 字段，必须单独停下让用户批准。

完成标志：

- 玩家可以从青茅山后续路线进入下一阶段，但不是纯文本跳转。
- 前期账本会影响路线条件和风险。
- 不开放完整南疆大地图，不开放高阶路线，不开放蛊仙期。

### v0.19：内容生产、长测与 v1.0 发布工具

目标：让 v1.0 前的内容扩展、测试、资产和公开准备可控。

核心交付：

- canon schema 与事实卡模板强化。
- 长测脚本和代表性玩家路径。
- v1.0 Player Advocate 大走查。
- EdgeOne 预览 smoke 与发布回滚清单。
- 美术资产 release manifest、hero 三件套落地、OG/宣传素材候选。
- 公开前 player-visible-copy、hidden fact、DeepSeek 权限、存档迁移总检查。

完成标志：

- 新增区域、NPC、蛊虫、路线、事件不靠聊天散落。
- v1.0 有可回放、可审查、可回滚的发布准备。

当前状态：

- 启动审查、总体大纲、小版本路线图、测试矩阵、真相源索引、Git 计划和 MiroFish 请求草案已建立。
- 建议先获得用户对 D-001 至 D-008 的批准，再进入 a1/a2。
- v0.19 三个 MiroFish 包为 preferred，不阻塞 a1，但建议在 b2/b3/b4 前完成 intake。

### v1.0：活世界早期正式版

目标：让玩家真正感到自己在蛊真人世界开启另一段人生。

最低完成定义建议调整为：

- 青茅山主线样板 + 青茅后续南疆早期路线可连续体验。
- 玩家自由意图能被解析、判断、降级、拒绝、反馈。
- NPC/势力能以受控账本记住玩家行为。
- 低阶蛊师经济、修行、炼养用、战斗闭环相对完整。
- 正史锚点和 IF 偏离能共存。
- UI 不再暴露大量未开放或高阶入口。
- 存档稳定，旧档迁移可测。
- 公开版不泄露 hidden fact，不让 DeepSeek 越权发奖励、地点、阵营或 NPC 生死。

## 5. 被推迟或拆分的内容

### 蛊仙期应推迟到 v1.0 之后

原长期路线中 `v0.18` 曾写蛊仙期和高阶世界门槛。现在我建议移到 v1.x 或 v2.0 之后。

原因：

- 当前低阶蛊师经济、战斗、路线、NPC/势力反应还未完整闭环。
- 仙窍、宝黄天、仙材、仙蛊、高阶隐藏事实需要更强的 private canon / BFF / 权限边界。
- 过早做蛊仙会把 v1.0 的目标拉散。

### 多区域正史锚点网络要先低阶化

原 `v0.17` 多区域大事件锚点网络应拆分：

- `v0.18` 先做南疆早期路线需要的低阶事实卡和区域锚点。
- 更宏大的商家城、三王山、义天山、宿命战等应作为 v1.0 后路线。

### 完整内容生产工具不要过早重工程化

`v0.19` 应该做足 v1.0 发布所需的 schema、长测、asset manifest、公开检查，不必现在做完整编辑器或大型内容平台。

## 6. 美术资产专项时机

不建议等到 v1.0 发布前才集中处理美术资产。那样会遇到两个问题：

1. 玩法已经定了，资产缺口才暴露，返工成本高。
2. UI 已经成型，图片规格、裁切、明暗、移动端适配可能来不及统一。

也不建议现在立刻大规模生成或替换美术。现在适合做“治理与对齐”，不适合做“大量生产”。

建议时机：

| 阶段 | 美术工作 | 目的 |
|---|---|---|
| v0.16 | 美术资产权威与显示位置治理 | 明确哪些图可做 runtime、hero、review-only、style-sample，避免 UI 减法后资产入口混乱 |
| v0.17 | 战斗/蛊道资产第一批正式化 | 跟战斗和蛊道工作台一起做，不凭空生成 |
| v0.18 | 南疆路线/区域气氛资产 | 跟路线和地域承接一起做 |
| v0.19 | v1.0 release art pack | hero 三件套、宣传截图、OG、短录屏、发布页素材 |
| v1.0 rc | 只做修补和替换，不做大规模新增 | 降低发布前风险 |

当前 `v0.15-art-audit-1` 已经做过资源盘点、重复图、hero 三件套候选和部分正式入库。下一步不应重复全量盘点，而应在 v0.16 做“资产曝光规则”：哪些资产出现在正式 UI，哪些只在开发/审查中出现。
