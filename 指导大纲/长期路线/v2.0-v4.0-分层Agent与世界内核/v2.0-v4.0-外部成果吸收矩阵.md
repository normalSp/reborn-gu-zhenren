# v2.0-v4.0 外部成果吸收矩阵

日期：2026-05-22
状态：长期参考；只吸收思想和工程模式，不引入运行时依赖。

## 使用原则

外部项目不是 RebornG 的替代架构。所有参考必须先转成 RebornG-owned 文档、规则、测试、门禁和版本决策，再进入 runtime。

硬规则：

- 不安装外部多 agent 框架作为当前 runtime。
- 不让外部框架拥有 canon、reward、location、NPC life/death 或 save authority。
- 不把外部论文里的自由 agent 记忆直接映射到 RebornG 存档。
- 不把“可说得像人”误认为“能作为世界事实”。

## 2026 Agent Framework Landscape 覆盖层

2026-05-23 已新增正式专项：

- `指导大纲/长期路线/Agent-Framework-Landscape-2026吸收矩阵.md`

本文件继续保留为论文和早期外部成果的思想映射；凡涉及开源 agent framework、Hermes、self-learning、license/SBOM、PoC、外部 adapter、Dify/Flowise/AutoGPT/Agno/Browser-use/LlamaIndex 等问题，以 `Agent Framework Landscape 2026` 为当前覆盖层。

当前覆盖结论：

- Hermes Agent：P0 架构参考，P1 隔离 PoC 候选；当前只批准 license/SBOM/架构适配评估，不批准 PoC。
- LangGraph / Mastra：继续作为 P0/P1 工程架构参考，但不引入依赖。
- LlamaIndex / Letta：只研究 private canon / knowledge visibility / memory 污染风险，不开放全书 RAG。
- Dify / Flowise / AutoGPT / Agno / Browser-use：按 WorldCore 服务能力评分，不按 star 排名；近期只吸收模式。
- self-learning：只能生成 `candidate_memory_patch` / `candidate_skill_patch` / eval 回流建议，不能自裁决，不能直接改 canon、runtime、save、prompt 或知识库正文。
- 任何外部框架进入 PoC、dependency、vendored subset、项目读写、命令、patch artifact 或 git 权限前，都必须另开用户审批门。

## Generative Agents

参考：

- https://arxiv.org/abs/2304.03442
- https://github.com/joonspk-research/generative_agents

吸收点：

- NPC 需要 `observation -> memory -> reflection -> plan`。
- 反思不是文本装饰，应进入后续行为候选。
- 玩家应该看见 NPC/社会行为痕迹。

RebornG 转化：

- L2/L3 agent 只能写结构化 `memoryEvidence`、`reflectionSummary`、`planCandidate`。
- 写入前必须经过世界内核裁决。
- 反思摘要必须绑定 source refs、public/private visibility、test refs。

## Concordia

参考：

- https://github.com/google-deepmind/concordia

吸收点：

- 社会模拟由可组合组件构成。
- GM / game master 负责给 agent 提供世界状态和裁决边界。

RebornG 转化：

- GM 对应 `WorldCore`，不是 DeepSeek。
- agent 只提交 proposal。
- `WorldCore` 按 canon、state、visibility、rules、tests 裁决。

## AI Town

参考：

- https://github.com/a16z-infra/ai-town

吸收点：

- 多 NPC 行为需要玩家可见，否则“世界活着”只存在后台。
- 区域生活要有短摘要、近期事件、人物动线。

RebornG 转化：

- v2.x 后可以设计 `区域生活摘要`、`NPC 近期公开动作`、`势力注意力`。
- UI 只展示世界内核批准后的公开事实或候选。
- 不展示 hidden/private body。

## Voyager

参考：

- https://arxiv.org/abs/2305.16291
- https://github.com/MineDojo/Voyager

吸收点：

- 技能库。
- 自动课程。
- 自验证。

RebornG 转化：

- Agent Lab 每次通过的场景生成 `scenario skill`。
- 失败样本进入测试矩阵或 future_sample_pool。
- 不能让 agent 自动修改源码、runtime、canon、prompt。
- Hermes/Voyager 类 self-learning 后续统一走 candidate-only pipeline：先生成候选，再由 RebornG-owned 审计、人类/WorldCore/skill sync 决定是否吸收。

## TextWorld

参考：

- https://github.com/microsoft/TextWorld

吸收点：

- 文本环境也需要可验证状态。
- 行动前置、世界状态、目标达成可以做自动评测。

RebornG 转化：

- Agent Lab 场景必须有 `initialState`、`allowedActions`、`forbiddenOutcomes`、`expectedStateDelta`。
- live narrative 不是验收标准，状态一致性才是验收底座。

## SOTOPIA

参考：

- https://github.com/sotopia-lab/sotopia

吸收点：

- 社交智能可用场景、角色、目标和评估器测试。

RebornG 转化：

- NPC 对话评估覆盖：利益一致、身份风险、隐瞒/试探、交易拒绝、势力压力。
- 评价指标不叫“善良/友好”，而是“是否符合原著世界利益和角色处境”。

## CICERO

参考：

- https://ai.meta.com/research/cicero/

吸收点：

- 战略推理和自然语言表达应分离。
- 对话中的承诺、欺骗、联盟需要策略层状态。

RebornG 转化：

- 势力策略由本地/服务端策略层计算。
- DeepSeek 只表达谈判、威胁、试探、虚与委蛇。
- 策略成功仍需世界内核结算。

## DeepSeek Pricing

参考：

- https://api-docs.deepseek.com/quick_start/pricing

吸收点：

- token 成本可支持小规模实验，但价格会变，不能写死为架构前提。
- 真正要工程化的是缓存命中、prompt hash、重试、失败率、eval archive 和预算上限。

RebornG 转化：

- Agent Lab 记录模型、样本、轮次、token、cache、成本、报告路径。
- 标准模型仍按项目现行 `deepseek-v4-flash`，模型切换需用户另批。

## 结论

外部成果证明 RebornG 的分层 agent 方向有现实依据；但没有任何一个项目可以直接替代 RebornG 的世界内核。我们吸收的是模式，不是权威。
