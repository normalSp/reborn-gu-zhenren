# v2.0-v4.0 开放决策与版本路线草案

日期：2026-05-22
状态：长期路线草案；所有版本号都是候选，不等于已批准开发范围。

## 总路线

v2.0-v4.0 的主线不是“马上让所有 NPC 活起来”，而是分三步：

1. v2.0 做稳第一个区域活世界。
2. v2.x 做 Agent Simulation Lab 和薄后端服务边界。
3. v3.x-v4.0 才逐步把受控 agent 接进 runtime，并预备高阶战斗、蛊仙和宿命大势。

## v2.0：第一个区域活世界

目标：

- 做出第一个真正可长测的低阶区域活世界。
- 通过 T3 300+ mixed/live/replay 长测。
- 保持世界内核权威。

不做：

- 完整 agent 社会。
- 任意时代开局。
- 全书 RAG。
- 正式高阶战斗。
- 完整南疆/商家城。

关键门禁：

- v1.9 a2 MiroFish blocking intake。
- v2.0 第一核心区域批准。
- 是否 bump save-format 单独决策。
- 300+ 长测通过。

## v2.1-v2.3：Agent Simulation Lab

目标：

- 离线模拟 20 NPC、3 势力、1 个天道/宿命宏观导演。
- 证明 agent proposal 不会污染世界事实。
- 证明结构化记忆、反思、势力压力、宏观约束可审计。

产物：

- Agent layer schema 草案。
- 离线 scenario runner 草案。
- agent eval 样本。
- 成本/延迟/失败率报告。

不做：

- 不写正式存档。
- 不进玩家 runtime。
- 不开放 writable subagents。

需要用户拍板：

- 是否允许开发离线 runner。
- 是否允许调用 live DeepSeek 进行批量 agent eval。
- 是否允许试点只读/分析型子代理做 coverage 审查。

## v2.4-v2.6：薄后端/BFF 边界

目标：

- 给 agent 时代准备基础设施。
- 优先解决安全和观测，而不是重型服务器。

候选能力：

- DeepSeek API proxy。
- private canon / hidden body 服务。
- agent job queue。
- cost/cache/prompt hash 观测。
- replay/eval archive。
- 云存档预研。

不做：

- 不做多人共享世界。
- 不做服务器权威战斗。
- 不做大型 RAG 平台。

需要用户拍板：

- 是否引入 BFF。
- 部署平台和密钥策略。
- 后端是否只服务开发/eval，还是进入公开 runtime。

## v3.0：有限 runtime agent 接入

目标：

- 只把 L2/L3 的受控能力接入一个小区域。
- agent 只参与候选和表达，不给裁决权。

候选能力：

- 当前场景关键 NPC live 对话。
- 事件后 L2 NPC 反思摘要。
- 小势力压力建议。
- WorldCore post-check。

不做：

- 不让 L4 原著关键角色自由 live。
- 不让 agent 写奖励、地点、NPC 生死。
- 不让 hidden/private 进入 DeepSeek visible context。

验收：

- 50/100/300 轮对照测试。
- live probe 记录成本和失败率。
- Player Advocate 评估玩家是否感到 NPC 有思想。
- 漂移检测确认世界事实稳定。

## v3.x：多区域、多身份、多势力扩展

目标：

- 把 agent 能力从单区域推广到多个低阶区域和身份路线。
- 支持更多 NPC 类型和小队关系。

候选能力：

- 多区域 faction pressure。
- 小队成员长期目标。
- 商队/散修/地方家族多组织反应。
- 同开局差异由世界内核和受控 agent 共同驱动。

硬停：

- 正式身份字段、区域事件 ledger、runFingerprint、经济结算、奖励、NPC 生死仍需单独门禁。

## v4.0：高阶战斗、蛊仙、宿命大势预备

目标：

- 评估蛊仙期和宿命/天道宏观系统。
- 准备新的战斗表现手法。

候选能力：

- theater map。
- killer-move stack。
- domain / formation / Immortal Gu House 状态。
- Fate / HeavenWill pressure engine。
- era anchor graph。

不做：

- 不直接开放所有尊者时代。
- 不直接开放所有仙蛊和杀招。
- 不直接让方源等关键角色自由 agent 化。

## 远期：任意时代与任意身份

这是终极愿景，不建议写进 v4.0 内完成。

需要前置：

- 全书知识库主题索引成熟。
- MiroFish topic-slice 链路稳定。
- private canon service 成熟。
- scenario compiler 成熟。
- era/fate/identity boundary 完成。
- 高阶战斗和经济分层完成。

## 当前结论

v2.0-v4.0 的路线应先追求“一个稳的活世界 + 一个稳的 agent 实验室 + 一个稳的薄后端边界”，而不是一口气追求五域两天全展开。

世界第一不是靠最大 prompt，而是靠最强的世界权威、最稳的测试门、最清楚的 agent 权限边界。
