# v2.0-v4.0 分层 Agent 与世界内核可行性分析

日期：2026-05-22
状态：长期研究线；不授权 runtime、后端、DeepSeek 权限或存档字段。

## 一句话结论

可行，但必须分层。RebornG 不能把“蛊真人世界万物”都做成 live LLM agent；那会带来事实漂移、延迟、成本、隐私、测试和回滚灾难。

正确路线是：

`世界内核负责事实与裁决；agent 负责观察、记忆、意图候选、策略建议和叙事表达。`

也就是说，agent 可以让世界显得有思想，但不能成为世界的法律。

## 可实现程度

| 目标 | 可行性 | 难度 | 建议阶段 | 判断 |
|---|---|---:|---|---|
| 低阶区域内 20 个 NPC 有记忆和目标 | 高 | 中 | v2.1-v2.3 离线 Agent Lab | 可先离线模拟，不写存档 |
| 3 个势力按利益、风险、公开事件反应 | 高 | 中高 | v2.1-v2.3 | 适合规则 + LLM 反思候选 |
| 当前场景关键 NPC live 对话和意图候选 | 高 | 中高 | v3.0 受控 runtime | 必须经过世界内核裁决 |
| 所有路人都有独立思想 | 中 | 很高 | v3.x+ | 用 L1/L2 聚合，不逐个 live |
| 天道/宿命宏观导演 | 高 | 很高 | v2.1 起离线，v4.0 前预备 | 必须是本地宏观系统，不是普通 NPC |
| 任意身份/任意时代开局 | 中 | 极高 | v4.0 之后继续扩展 | 需要时代图谱、身份编译器、私有事实服务 |
| 所有蛊虫/杀招/仙蛊屋可玩 | 中 | 极高 | v3.x-v4.x+ | 内容工厂可做，平衡和测试最难 |
| 蛊仙级战斗完整表现 | 中高 | 极高 | v4.0 预备 | 需要新战斗表达，不只靠 5x3 棋盘 |

## Self-learning 可行性补充

`Agent Framework Landscape 2026` 将 Hermes Agent 纳入 P0 架构参考后，self-learning 可以成为长期路线的一部分，但只能作为“候选生成和流程学习”，不能作为“事实裁决和自动改写”。

可行项：

| 能力 | 可行性 | 适合阶段 | RebornG 用法 |
|---|---|---|---|
| NPC 事件后记忆候选 | 高 | v2.5-v2.7 | 生成 `candidate_memory_patch`，由 WorldCore / visibility / eval 审核 |
| 小队/NPC 技能候选 | 中 | v2.6-v2.8 | 生成 `candidate_skill_patch`，进入 skill sync / test matrix |
| 失败样本回流 | 高 | v2.3+ | eval farm 失败进入 future_sample_pool 或 current_matrix |
| 长测压缩摘要 | 高 | v2.6+ | trajectory / replay archive 摘要，不写事实 |
| 经济/势力策略启发 | 中 | v2.7+ | 生成压力候选和策略候选，正式价格/库存/通缉仍由 WorldCore |

不可行或当前禁止项：

- self-learning 直接修改 canon/runtime/save。
- self-learning 直接修改 DeepSeek prompt、知识库正文或 expert/lore skill。
- self-learning 把玩家谣言、DeepSeek 候选、NPC 误解写成事实。
- self-learning 直接调整经济价格、库存、奖励、NPC 生死、地点、阵营、身份。
- self-learning 自动安装依赖、运行项目命令、生成 patch artifact 或操作 git。

风险判断：

- 最大收益不在“让 NPC 更会聊天”，而在“让长期失败、记忆候选、技能候选、测试样本自动回流”。
- 最大风险是记忆污染和权威错位：agent 越会学习，越需要 WorldCore、visibility gate 和 eval farm 把它锁在候选层。
- 因此 v2.5/v2.6 应研究 self-learning candidate pipeline，而不是 Hermes runtime 接入。

## 为什么现有架构不够

现有 v1.x 架构适合：

- projection-first 的低阶区域样板。
- 路线、低阶生存、社会压力、冲突后果的只读合流。
- 小规模 live probe 和 Player Advocate。
- 本地 deterministic TypeScript core。

它不够支撑终极愿景，原因是：

- hidden/private canon 不能长期完整下发到公开前端。
- 每个 NPC、势力、区域、时代都需要长期记忆和事件历史。
- live DeepSeek 调用需要成本、缓存、失败率、prompt hash 和 eval archive 的集中观测。
- 大规模 agent 需要 job queue、批处理、快照、回放和故障隔离。
- 任意时代/身份开局需要 scenario compiler，而不是靠 DeepSeek 即兴生成开局。

因此 v2.0 后需要薄后端优先评估，但后端不能接管游戏裁决权。

## Agent 分层默认方案

| 层级 | 对象 | 是否 live LLM | 主要技术 | 权限 |
|---|---|---:|---|---|
| L0 | 地点、物品、蛊虫、资源、天气、事件账本 | 否 | 规则、状态机、事件 envelope | 只能被世界内核写入 |
| L1 | 路人、商贩、巡逻、兽群、低阶环境反应 | 否 | 行为树、效用 AI、事件表、权重池 | 可生成候选，不写事实 |
| L2 | 次要 NPC、小队成员、小势力 | 偶尔 | 批处理反思、结构化记忆、目标队列 | 可提交 `AgentProposal` |
| L3 | 当前场景关键 NPC | 是 | DeepSeek 对话、意图候选、语气表达 | 只写候选和叙事，不结算 |
| L4 | 原著关键人物、大势力核心 | 严格受控 | source pointer、私有记忆、强审计、人工门禁 | 默认不进入 live runtime |
| L5 | 天道、宿命、时代大势、兽潮/市场/战争导演 | 不按 NPC 处理 | 宏观导演、压力场、因果债、锚点图 | 世界级约束，不输出玩家事实 |

关键设计点：

- L0/L1 解决“万物有反应”。
- L2/L3 解决“人物像人”。
- L4 解决“原著角色不能乱”。
- L5 解决“宿命和天道不是一个会聊天的 NPC”。

## 游戏性收益

分层 agent 的真正收益不只是文本更像人，而是让玩家行为产生长期压力：

- 玩家骗过一个商队短工，之后商队账房、护卫、城门盘问都可能产生不同反应。
- 玩家在战斗中保住某个队友，该队友未来可以成为证人、债主、牵线人或隐患。
- 玩家反复利用同一漏洞，世界内核可以触发反刷、怀疑、资源枯竭或势力关注。
- 同一开局可以有不同低阶压力和 NPC 侧重，但原著事实和隐藏事实不漂移。

爽点应来自：

- 情报优势。
- 低阶求生后翻盘。
- 准备链兑现。
- 被世界记住。
- 代价明确的 IF 偏离。

不应来自：

- DeepSeek 直接送蛊、送身份、送传承。
- 任意输入立即成功。
- NPC 为了配合玩家而失去原著性格和利益。

## 成本与性能判断

DeepSeek token 成本不是最大瓶颈，尤其 `deepseek-v4-flash` 当前官方价格适合小规模 live/eval 实验。真正瓶颈是：

- live 调用延迟。
- 批量 agent 调度。
- 上下文选择和 hidden/private 隔离。
- 失败重试和 JSON/结构化输出稳定性。
- 长测/eval 的人工复核成本。
- agent 输出进入世界前的本地裁决。

因此首阶段应做离线/后台实验，而不是每回合让所有 NPC 同时思考。

## 是否在造轮子

不是纯造轮子，但也没有现成引擎能直接满足目标。

已有成果覆盖的是局部能力：

- NPC 记忆/反思/计划。
- 多 agent 社会模拟。
- 文本环境评测。
- 战略推理和语言表达分离。
- 技能库和自验证。

RebornG 独有难点是组合问题：

- 原著强约束。
- hidden/private canon。
- 长期存档。
- RPG 数值、战斗、经济、身份、路线、社会同时闭环。
- 玩家自由输入和本地裁决共存。
- 五域两天、任意时代、任意身份的远期广度。

所以我们不是在重造某个已有项目，而是在把多个研究方向工程化成一个二创强约束 RPG 世界内核。

## 最大风险

| 风险 | 后果 | 缓解 |
|---|---|---|
| LLM 获得事实写入权 | 世界观崩坏、奖励失控、NPC 生死乱写 | 所有 agent 只能提交 proposal |
| 每个 NPC 都 live 调用 | 延迟高、不可测、成本和失败率膨胀 | L0-L2 默认非 live，L3 只限当前场景 |
| hidden/private 泄漏 | 方源、宿命、高阶因果提前暴露 | 私有事实服务、source pointer、可见性门禁 |
| agent 记忆自由文本污染 | 长线剧情质量下降 | 结构化记忆、promotion chain、定期摘要和回放 |
| 后端过早重构 | 拖慢 v2.0，打乱现有 core | 先做 BFF 评估，不开重型服务 |
| 高阶战斗仍用低阶棋盘 | 蛊仙期表现力不足 | v4.0 前设计 theater / stack / domain 表达 |

## 推荐第一实验

v2.1-v2.3 的 Agent Simulation Lab：

- 20 个 L1/L2 NPC。
- 3 个势力。
- 1 个 L5 天道/宿命宏观导演。
- 100-300 轮离线事件。
- 不写正式存档。
- 不进入 runtime。
- 每 25-50 轮检查：世界状态、原著边界、玩家目标、NPC 记忆、重复空转、hidden 泄漏、DeepSeek authority drift。

通过标准：

- NPC 反应能解释过去事件。
- 势力压力能随公开行为变化。
- 宏观导演能施加约束但不替玩家结局。
- DeepSeek 不写正式奖励、地点、阵营、NPC 生死。
- 同一 replay 可被复核和复评分。

## 结论

分层 agent + 世界内核是 RebornG 走向世界级深度的正确方向。它不是 v2.0 前要抢跑的功能，而是 v2.0 后的主架构线。

下一步不是实现 agent，而是先把可行性、架构、外部参考、版本路线和开放决策固定为项目文档，让后续开发不靠聊天记忆推进。
