# RebornG 长线叙事防崩坏与青茅 IF 矩阵落位

日期：2026-05-19
状态：长期路线补充；已同步到 v0.18 完成和 v0.19 启动审查草案；进入运行时前必须转化为具体版本范围。

## 问题边界

外部 AI 人生/跑团项目需要处理“从出生到多年人生”的长生命周期叙事。RebornG 当前不把 0 岁到 13 岁式人生模拟作为近期核心压力。

RebornG 当前真正要解决的是：

- 玩家作为原创蛊师，在青茅山连续行动几十到几百轮后，世界事实不乱。
- 玩家能表达自由目标，但行动必须被境界、资源、身份、信息、正史锚点和 IF 代价约束。
- DeepSeek 不靠长上下文硬记全部历史，而是读取本地结构化事实、事件和门禁。
- 原著事实、玩家可见事实、隐藏事实、IF 偏离和 NPC/势力反应能长期共存。

## 已有旧设计

旧 v0.8 设计中已经有可继承思想：

- `src/canon/canon-anchor-registry.json`：正史锚点注册表。
- `src/canon/fate-branch-matrix.json`：宿命/IF 分支轴。
- `src/canon/v080-midgame-anchor-rules.json`：中后期锚点规则、IF 轴、天意/因果代价。
- `src/engine/v080-narrative-engine.ts`：锚点 mutation 校验和 IF branch candidate 校验。
- `指导大纲/v0.8.0/codex/02-剧情分支/出身深线剧情矩阵.md`：出身、资源入口、锚点参与权和 IF 代价矩阵。

这些设计不能原样搬到 v0.12。旧矩阵面向中后期和宿命战级别大锚点，青茅山阶段需要低阶化、区域化、事实卡化。

## 青茅版 IF 矩阵

v0.12 的 IF 矩阵不使用 `protect_fate`、`break_fate`、`venerable_balance` 这类高阶轴作为主轴。

青茅阶段建议轴：

| IF 轴 | 含义 | v0.12 口径 |
|---|---|---|
| `npc_attention` | 被关键人物注意、怀疑、试探 | 只能形成公开反应和风险，不直接改 NPC 生死 |
| `faction_pressure` | 古月/白家/熊家/商队/散修压力 | 写入势力压力、阻拦、机会窗口或谣言 |
| `resource_control` | 月兰草、食料、低阶蛊材、坊市资源偏移 | 必须受资源/经济/反刷门禁约束 |
| `route_escape` | 离开青茅山的路线准备和暴露风险 | v0.12 只做准备、补给、追击压力，不开放成功逃离 |
| `hidden_fact_probe` | 玩家触碰隐藏事实的风险 | 可记录隐藏 ref、公开失败、被注意到，不泄露隐藏正文 |
| `local_survival` | 局部人物受伤、失踪、存活压力 | 只能做候选或局部压力，不改正史核心人物命轨 |
| `canon_anchor_pressure` | 接近正史核心因果时的压力 | 用于阻断、重定向、代价提示和 IF 候选 |

## 防崩坏机制分工

`system prompt + ZodSchema + 双轮裁判 + 语义规则 + 金丝雀断言` 只能拦住格式错误、越权写入、典型爽文崩坏和隐藏事实泄露。

真正防止长线剧情崩坏，还需要本地结构化层：

- 正史锚点表：决定什么不能随口改。
- IF 矩阵：决定哪里可以偏离、偏离代价是什么。
- 事件编年史：决定玩家做过什么，不靠模型记忆。
- NPC 记忆：决定谁记得玩家，记得什么，态度为什么变化。
- 势力压力：决定行为如何扩散成阻拦、机会、追击、交易窗口。
- 检索式上下文：每轮只喂当前相关事实，不塞完整历史。
- 长线回归测试：验证几十轮以后是否泄密、串线、状态污染、选项空转。

## 验收口径

v0.12 完成时，RebornG 不承诺完整活世界，但必须做到：

- 玩家自由目标不再直接交给 DeepSeek 解释成结果。
- 青茅正史锚点和可偏离点有项目真相源。
- 玩家每次公开行动都能留下可测试的事实、压力或后果。
- 重大 IF 只能成为候选、准备链或代价提示，不能一键改写正史。
- 后续 v0.13-v1.0 可以沿同一套矩阵继续扩展，而不是返工重做。
