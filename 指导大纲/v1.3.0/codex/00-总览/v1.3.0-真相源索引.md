## 代码权威

当前 v1.3-b1 已新增 projection-only runtime。a1/a2 代码事实与 b1 新增入口如下：

- `src/store/initialState.ts`：当前 `SAVE_FORMAT_VERSION = 24`。
- `src/store/index.ts`：旧档 migration/default normalization。
- `src/types/index.ts`：`LivingNpcMemoryEntry`、`LivingFactionPressureEntry` 与既有旧社会字段。
- `src/store/livingWorldSlice.ts`：`livingWorldState.npcMemories`、`factionPressure`、`actionConsequences` 的现有写入/清理边界。
- `src/engine/v110-route-location-state.ts`：route/location 本地状态。
- `src/engine/v120-survival-economy-state.ts`：v24 `survivalEconomyState`。
- `src/engine/v130-social-pressure-projection.ts`：v1.3 b1 社会压力 projection-only helper；不写 store/save，不调 DeepSeek。
- `src/components/game/SocialPressurePanel.tsx`：世界面板 `社会` 页签；显示势力压力、记忆痕迹、公开事件和后续候选。
- `tests/e2e/v130-social-pressure-projection.spec.ts`：b1 玩家动作链与 UI 边界回归。
- `src/engine/context-builder.ts`：DeepSeek 上下文与 C27 hidden causality protection。
- `src/api/deepseek.ts`：DeepSeek runtime API。

若 v1.3 新增社会 runtime，必须继续遵守 `canon -> types/schema -> engine -> store -> UI -> tests/docs`，并另行批准 save format、字段、MiroFish blocking 条件和测试强度。

## a1 代码事实结论

- b1 可先读取既有 `livingWorldState` 证据做 projection-only 社会反应层。
- `livingWorldState` 不能被改口为正式关系系统。
- 旧字段 `npcRelations`、`standings`、`npcContacts`、`dynamicNPCs` 暂不作为 v1.3 权威。
- 如未来进入 `SAVE_FORMAT_VERSION = 25`，必须单独新增 migration/defaults/tests，并不得从旧字段自动推导正式关系。

## b1 代码事实结论

- `buildV130SocialPressureProjection()` 只读 `livingWorldState.knownFacts`、`npcMemories`、`factionPressure`、`actionConsequences` 与本地行动账本。
- b1 复用 v0.13 已审查并转写为本地规则的投影器，不读取 MiroFish 原始包。
- b1 输出可重算 view model，不写 `socialRelationState`、不 bump save format、不扩大 DeepSeek 权限。
- b1 UI 排序保证四类社会信号至少各保留一条，避免公开事件被高风险卡挤掉。
