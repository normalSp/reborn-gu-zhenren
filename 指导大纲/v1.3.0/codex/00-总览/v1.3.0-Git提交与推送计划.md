## 总原则

- 不使用 `git add -A`。
- 每个小版本/专项先切语义分支。
- 文档门禁、runtime、测试证据、handoff 分边界提交。
- 不自动部署 EdgeOne。
- 不在未批准前写 public release wording。
- 阶段完成后推送当前分支，除非用户明确要求不推送或远端不可用。
