# v1.3.0-b4 势力前置条件 projection-only

日期：2026-05-21
状态：已完成本地第一刀
主题：势力封锁、招揽、通缉、任务、交易窗口只做前置条件可读化

## 结论

b4 在 b1-b3 的 projection-only 基线上新增 `factionPreconditions` 输出和 UI 区块。它让玩家看懂：

- 哪个势力方向出现压力或机会。
- 这更像招揽前置、通缉风险、封锁压力、任务流程压力、交易窗口，还是普通压力。
- 哪些正式结论没有开放。

本阶段仍然不写通缉、不写招揽或阵营变化、不写正式封锁/追捕结果、不创建任务、不发奖励。

## 测试发现与修正

聚焦 E2E 初跑发现：前置条件文案中出现了 `招揽成功` 这类正式结论词。虽然语义是“阻断”，但玩家可见文本仍不应该出现正式成功措辞。

修正：改为 `正式招揽结果未开放`，并保留 E2E 负向断言。

## 权威边界

| 项 | b4 口径 |
|---|---|
| 通缉 | 只显示风险；不生成、不解除、不定案 |
| 招揽/投靠 | 只显示公开递话、信任缺口和遮掩前置；不写成功 |
| 封锁/追捕 | 只显示压力；不写封锁结果、追捕成败或捕获 |
| 任务 | 只显示来源/流程压力；不创建任务、不发奖励 |
| 交易窗口 | 只显示递话/补给遮掩；不成交、不写价格/库存 |
| DeepSeek | 无新权限 |
