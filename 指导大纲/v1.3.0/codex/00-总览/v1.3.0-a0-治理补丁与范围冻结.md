## 治理补丁 1：Skill 同步审计

新增项目级制度：

- `指导大纲/流程制度/Skill同步审计制度.md`

核心规则：

1. 每次大版本启动、runtime 阶段完成、process 阶段、rc 或跨版本制度变化，都必须审计相关 skill。
2. 每个 skill 必须标为 `updated`、`no_update_needed`、`deferred_with_reason` 或 `blocked`。
3. 审计结果必须写进当前版本文档和 handoff。
4. 若 skill 落后会误导下一步，则不能声称阶段完成。

## 治理补丁 2：MiroFish bridge

决策：

- 不把 `mirofish-reborng-export` 并入 `reborn-expert-council`。
- `reborn-expert-council` 负责治理、决策和门禁。
- `mirofish-reborng-export` 负责生产、导出、coverage、quote-redacted handoff。
- 两者通过文档和 intake review 桥接，不共享 runtime/canon authority。

使用规则：

1. 涉及 MiroFish 新包、基础包 topic slice、coverage、quote-redacted export 时，专家团必须调用或参考 `mirofish-reborng-export`。
2. `mirofish-reborng-export` 不得直接改 RebornG runtime/canon/store/DeepSeek context。
3. 输出进入 RebornG 后，仍必须经过 `指导大纲/vMiroFish/intake-reviews/`。
4. 全书基础包只作为 archive/source-pointer inventory，不能整包吸收。
