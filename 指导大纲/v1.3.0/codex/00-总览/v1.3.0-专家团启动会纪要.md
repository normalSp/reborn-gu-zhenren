# v1.3.0 专家团启动会纪要

状态：启动会已召开；D-130-001 至 D-130-009 已获用户批准；a0 文档先行
日期：2026-05-21

## 会议目标

确定 v1.3 是否可以启动、启动后先做什么、哪些边界必须停下来让用户拍板。

## 输入来源

- `v1.1.0` 已建立 `routeLocationState` 与 D-025/C27 live drift 阻塞门经验。
- `v1.2.0` 已建立 `survivalEconomyState` 最小 pressure ledger 与低阶生存经济第一阶段。
- `v0.13.0` 已做 NPC 与势力反应第一层，留下可复用的 NPC motive、faction pressure、public event chronicle 三类候选包。
- 全书 MiroFish 基础包已在 `指导大纲/vMiroFish/基础包/`，只能作为 archive/source-pointer inventory。
- 长期路线把 v1.3 定位为 `NPC 与势力长期关系第二层`。

## 专家意见

### Project Lead

v1.3 可以启动，但必须先做 a0/a1/a2。这个版本的目标不是把 NPC 做成自由聊天 agent，而是让玩家的行为被社会层稳定记住、解释和限制。

### Lore & World Designer

蛊真人世界里社会关系应当残酷、利益化、证据化。怀疑、招揽、封锁、通缉、接触窗口都可以作为压力，但不能默认给玩家正式阵营身份、NPC 生死结论或隐藏事实暴露。

### Systems Architect

社会状态如果持久化，必须像 v1.1/v1.2 一样使用单一聚合对象，严格迁移/defaults/tests。b1 应优先 projection-only，先证明关系证据和玩家可见反馈能跑通。

### AI Pipeline Architect

DeepSeek 可写语气、传闻、社会压力和候选反应，不得写正式关系数值、通缉结论、招募成功、阵营转移、奖励或 NPC 生死。v1.3-rc live probe 已批准 in principle，但成本、样本和轮次要到 rc 前再确认。

### Player Advocate

玩家需要看懂“谁记住了我、为什么、会影响什么、下一步能做什么”。如果只是多几句 NPC 文案，v1.3 就没有完成社会层地基。

### QA/Test Engineering Guardian

v1.3 必须把 NPC/势力漂移、隐藏事实邻近、关系重复空转、DeepSeek 越权、玩家目标遗失加入测试矩阵。rc 建议按是否新增持久社会字段分 100/120 轮 Player Advocate。

### Git/rollback Steward

v1.3-a0 单独分支、单独提交。runtime 阶段继续按 `canon -> schema -> engine -> store -> UI -> tests/docs`，显式 stage，阶段末推送，不自动部署。

### Canon/Knowledge Curator

MiroFish 基础包只做 source pointer 和 coverage 后勤。v1.3 需要 topic slice，而不是整包导入。正式命名 NPC allowlist、通缉/招揽/隐藏事实邻近规则必须先 blocking intake。

## 会议共识

1. v1.3 主线为 NPC 与势力长期关系第二层。
2. a0/a1/a2 必须先于 runtime。
3. b1 默认 projection-only，先做关系证据、社会压力和玩家可见说明。
4. a1 评估 `SAVE_FORMAT_VERSION = 25`，但不默认批准。
5. v1.3 禁止默认开放正式阵营转移、通缉结论、招募成功、任务奖励或 NPC 生死。
6. MiroFish 初始 preferred；正式命名 NPC/通缉/招揽/隐藏事实邻近升级为 blocking。
7. rc 必跑 live probe，模型固定 `deepseek-v4-flash`；成本、样本和轮次 rc 前再确认。
8. v1.3-a0 必须补 skill sync audit 机械闭环，防止 skill 口径落后。
