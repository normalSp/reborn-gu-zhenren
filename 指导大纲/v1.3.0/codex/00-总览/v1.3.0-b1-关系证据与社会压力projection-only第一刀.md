## Runtime 行为

`buildV130SocialPressureProjection()` 只读取：

- `livingWorldState.knownFacts`
- `livingWorldState.npcMemories`
- `livingWorldState.factionPressure`
- `livingWorldState.actionConsequences`
- `sceneSessionState.localActionLedger`

输出：

- 势力压力
- 记忆痕迹
- 公开事件
- 社会后续候选
- 可见来源
- forbidden writes
- boundary lines

它不会：

- 写 store。
- 写 save。
- 调 DeepSeek。
- 读取 MiroFish 原始包。
- 从 `npcRelations`、`standings`、`npcContacts`、`dynamicNPCs` 或未来 `socialRelationState` 推导关系。

## 测试发现与修正

聚焦 E2E 初跑发现：风险排序会让公开事件被势力/记忆/后续高风险卡挤出前 10，导致玩家看不到完整四类摘要。

修正：`selectSignals()` 先保证 `faction_pressure`、`npc_memory`、`public_event`、`social_followup` 各至少一条，再按风险补齐。这已加入 E2E 覆盖。
