`a0/a1/a2 -> b1/b2/b3/b4 -> process-1/process-2 -> rc`

状态：本启动包已建立。

产出：

- 专家团启动会纪要。
- 启动审查与范围冻结。
- 总体开发大纲。
- 需求决策池。
- 测试矩阵。
- MiroFish 需求判断。
- Git 计划。
- Skill 同步审计制度。

通过标准：

- 不新增 save 字段。
- 不扩大 DeepSeek 权限。
- 记录 D-130-001 至 D-130-009。
- 明确 a1 后需要用户拍板的事项。
- 明确 skill sync audit 结果与补点。

## a1 NPC/势力关系 save-format 设计门禁

目标：

- 冻结 v1.3 authority table。
- 严肃评估是否需要 `SAVE_FORMAT_VERSION = 25`。

- 定义命名 NPC、关系证据、势力压力、公开事件、隐藏事实邻近的权限边界。

- b1 默认 projection-only。

- b1 采用 projection-only 社会反应层。
- b1 不 bump `SAVE_FORMAT_VERSION = 25`，继续使用 v24。
- b1 不新增 `socialRelationState`，仅把它保留为 b2/b3 备选门禁。
- 现有 `livingWorldState.npcMemories`、`factionPressure`、`actionConsequences` 只做投影证据，不做正式关系权威。
- 旧字段 `npcRelations`、`standings`、`npcContacts`、`dynamicNPCs` 暂不作为 v1.3 权威。
- v1.3-a2 执行 MiroFish preferred topic slice；正式命名 NPC allowlist、通缉/招揽/封锁、隐藏事实邻近、NPC 生死或 canon 晋升才升级 blocking。

## b1 关系证据与社会压力 projection-only 第一刀

状态：待用户批准 D-131 且 a2 完成后执行。
