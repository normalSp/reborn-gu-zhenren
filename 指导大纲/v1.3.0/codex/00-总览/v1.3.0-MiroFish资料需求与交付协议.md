## 全书基础包使用规则

全书基础包位于：

`指导大纲/vMiroFish/基础包/`

当前只允许作为：

1. archive/source-pointer inventory。
2. topic slice coverage 查询来源。
3. v1.3 a2 的候选样本来源。
4. 后续知识库索引的 source pointer 后勤。

不得作为：

1. canon truth。
2. runtime authority。
3. DeepSeek visible context。
4. 玩家可见隐藏事实来源。
5. 自动生成正式 NPC/势力规则的依据。

## mirofish-reborng-export 桥接

不合并 skill。分工如下：

| skill | 职责 | 禁止 |
|---|---|---|
| `reborn-expert-council` | 决策、门禁、版本范围、intake 判断 | 不直接生产大包 |
| `mirofish-reborng-export` | MiroFish 导出、coverage、quote-redacted package、handoff | 不直接改 RebornG canon/runtime |

触发以下情况时，专家团必须调用或参考 `mirofish-reborng-export`：

- 新 MiroFish request family。
- 全书基础包 topic slice。
- coverage gap 审查。
- quote-redacted export。
- hidden fact handoff。

## 需要 topic-slice intake 的情况

以下情况必须先做 v1.3 topic-slice intake review：

1. 将基础包条目转写为知识库索引。
2. 将条目晋升为 `V13-*` 测试样本。
3. 将条目晋升为 RebornG-owned rule draft。
4. 将条目用于命名 NPC allowlist。
5. 将条目用于通缉、招揽、封锁、隐藏事实邻近规则。
6. 将条目用于 NPC 生死、命运、方源公开证据长期后果或 `src/canon/*.json` 晋升。

第 4、5、6 项升级为 blocking，并需要用户决策。

## a1 后的 a2 建议执行口径

a2 已按以下口径执行，不做“全书导入”，只做主题切片：

- 先读 v0.13 三包 intake review，确认 NPC 记忆/动机、势力压力、公开事件编年史可复用范围。
- 再从 `指导大纲/vMiroFish/基础包/` 查缺补漏，只记录 source pointer、coverage 缺口和 candidate sample。
- 输出 `v1.3.0-a2-MiroFish-NPC势力关系topic-slice-intake.md`。
- 每个样本必须进入 `current_matrix`、`future_sample_pool` 或 `discarded_for_v1.3`。
- 不把任何 hidden/private 原文或未转写内容放进 DeepSeek visible context。

## forbidden uses

1. 直接进入 DeepSeek visible context。
2. 直接作为 canon truth。
