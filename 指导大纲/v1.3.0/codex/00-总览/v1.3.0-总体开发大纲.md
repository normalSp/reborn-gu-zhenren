# v1.3.0 总体开发大纲

状态：a1 设计门禁已建立；等待 D-131 用户拍板
日期：2026-05-21

## 目标

让 NPC 与势力长期关系从“短期提示/候选反应”推进到更稳定的社会层体验：

- 玩家能看见自己的行动影响了谁。
- NPC/势力反应以证据、公开事件、利益、怀疑、信任、恐惧和接触窗口为基础。
- UI 能说明为什么某人/某势力变得警惕、愿意试探、拒绝交易或提出条件。
- DeepSeek 增强语气和叙事，不拥有正式关系结论。

## 不是目标

v1.3 不负责：

- 完整 NPC agent 仿真。
- LLM 自由维护 NPC 记忆。
- 正式阵营转移。
- 正式通缉结论。
- 招募成功或加入势力。
- 正式任务奖励。
- NPC 生死、捕获、背叛定局。
- 完整南疆社会网络。
- hidden/private canon 可见化。

如果上述任何一项需要推进，必须从需求池单独升级并由用户拍板。

## 权威原则

| 事项 | 权威 |
|---|---|
| 正式 NPC/势力关系、通缉、招揽、阵营、奖励、NPC 生死 | 本地 canon/engine/store，且需用户批准 |
| 社会压力、传闻、候选反应、语气、请求、可见迹象 | DeepSeek 可叙事，不可结算 |
| 原著 NPC/势力事实、隐藏事实、source pointer | MiroFish/知识库 candidate，经 intake 后使用 |
| 玩家可见“谁受影响/为什么/下一步” | UI + Player Advocate gate |
| 长线关系漂移、隐藏事实邻近、DeepSeek 越权 | 测试矩阵 + 长线叙事漂移制度 + rc live probe |

## 建议小版本结构

1. `a0`：启动审查、治理补丁、skill sync audit、范围冻结。
2. `a1`：NPC/势力关系 save-format / authority / scope 设计门禁。
3. `a2`：MiroFish topic slice、知识库、测试矩阵和隐藏事实邻近门禁。
4. `b1`：关系证据与社会压力 projection-only 第一刀。
5. `b2`：最小社会状态 ledger 或继续 projection-only，取决于 D-131 与 b1 验证结果。
6. `b3`：NPC 个人反应与接触窗口第一刀，默认不写正式关系结论。
7. `b4`：势力封锁/招揽/通缉前置条件第一刀，默认不定案。
8. `process-1`：关系反刷、旧档兼容、回滚和测试矩阵复核。
9. `process-2`：长线 NPC/势力漂移、MiroFish/知识库复核。
10. `rc`：质量收束、Player Advocate 100/120 轮、`deepseek-v4-flash` live probe。

## 成功标准

- 玩家看得懂社会反馈和下一步。
- 没有未批准 save field 或 save-format bump。
- 没有 DeepSeek 写正式关系、阵营、通缉、招募、奖励或 NPC 生死。
- MiroFish 仅以 topic-slice、source pointer、rewritten summary、测试样本方式参与。
- 新样本全部进入测试矩阵演进流程。
- rc 前给出 live probe 成本、样本数、轮次和接受标准，由用户最终确认。
