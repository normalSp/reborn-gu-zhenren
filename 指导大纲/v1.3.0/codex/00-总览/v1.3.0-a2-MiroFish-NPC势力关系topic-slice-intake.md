## 1. 本轮 intake 结论

结论：v1.3-b1 可以进入 projection-only 第一刀，不需要新增 MiroFish 包。

允许使用：

- v0.13 三包作为 b1 candidate/rule/test 后勤。
- 全书基础包只作为 source pointer inventory 和 coverage gap 背景。
- 公开事件、NPC 记忆、势力压力的安全摘要，必须由 RebornG 本地重写后再进入 UI/测试。

禁止使用：

- 全书基础包整包导入。
- MiroFish item 直接进入 runtime/canon/DeepSeek visible context。
- hidden/private 内容进入 UI 或 DeepSeek。
- 正式 NPC allowlist、正式通缉/招揽/封锁、NPC 生死、方源公开证据长期后果、canon 晋升在 b1 中落地。

## 3. v1.3 topic slice 分流

### S13-NPC-MEMORY

来源：

- `npcInterestTrigger`
- `npcMemoryCandidate`
- `npcMotiveCandidate`
- `npcSuspicionTrigger`

可进入：

- b1 projection helper 的规则草案。
- UI 文案边界：谁可能记得、为什么、玩家可见理由是什么。
- 测试矩阵：NPC 记忆不等于正式好感/仇恨/效忠。

必须降级：

- NPC 内心动机只能写成“可能关注/可能警惕/可能试探”，不能写成确定心理事实。
- `subjectDisplayName` 不能直接变成 runtime NPC id。

### S13-FACTION-PRESSURE

来源：

- `factionPressure`
- `factionStanceCandidate`
- 部分 `taskSourceCandidate`、`blockadeCandidate`、`recruitmentCandidate`

可进入：

- b1 社会压力卡：怀疑、机会、资源控制、传闻扩散、纪律压力。
- b1 可读提示：为什么某势力可能注意玩家。
- 测试矩阵：通缉/封锁/招揽候选必须降级。

必须降级：

- `blockadeCandidate` 只显示“封锁风险/审查前置条件”。
- `recruitmentCandidate` 只显示“试探窗口/条件不足”。
- `taskSourceCandidate` 只显示“可能的任务来源/不能接取”。

### S13-PUBLIC-EVENT

来源：

- `eventChronicleAnchor`
- `publicEventCandidate`
- `publicActionSummaryCandidate`
- `eventVisibilityRule`

可进入：

- b1 公开事件证据链。
- UI 的“公开原因/谁可能知道/谁会记录”。
- DeepSeek 安全摘要的本地重写模板，但不直接喂 MiroFish item。

必须过滤：

- 当前玩家未经历、不可见、或本地状态未确认的事件，不得显示。
- `promptSafeSummary` 仍是候选，不是 DeepSeek visible context。

### S13-HIDDEN-BOUNDARY

来源：

- 全书基础包 `hiddenFacts`。
- v0.13 包中涉及方源、隐藏因果、后期事件、外部接触或非当前玩家可见轨迹的候选。

处理：

- `quarantined_for_v13_b1`。
- 只进入 hidden-boundary 测试样本。
- 不进入 UI、DeepSeek、runtime、knowledge index。

### S13-BLOCKING-FORMAL-SOCIAL

触发：

- 正式命名 NPC allowlist。
- 正式通缉、招揽、封锁、势力任务、奖励。
- NPC 生死、捕获、背叛定局。
- 方源公开证据长期后果。
- `src/canon/*.json` 晋升。

处理：

- `future_sample_pool` 或 `blocking_if_reopened`。
- 必须另开 MiroFish request/intake 与用户决策。
- 不进入 b1。

## 6. b1 输入契约

若进入 b1，runtime 只能依赖以下安全输入：

1. 本地 save/store 中已有的 `livingWorldState.npcMemories`、`factionPressure`、`actionConsequences`。
2. 本地可见性判断后的公开事实和行动后果。
3. v0.13/a2 形成的 RebornG-owned 分类规则和测试样本。
4. 只读、可重算的 projection view model。

b1 不得依赖：

1. MiroFish JSON 直接读取。
2. 全书基础包直接读取。
3. DeepSeek 生成正式社会结论。
4. 旧字段 `npcRelations`、`standings`、`npcContacts`、`dynamicNPCs` 作为权威。
