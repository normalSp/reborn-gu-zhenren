# 专家团章程与角色职责

## 基本原则

- 专家团先守真相源，再提体验优化。
- 旧报告不能覆盖当前源码、测试和 v0.9 文档。
- DeepSeek 是叙事协作者，不是运行时事实来源。
- 子代理默认只读；需要写入时必须有明确文件所有权和不重叠写入范围。

## 固定角色

| 角色 | 职责 | 必看输入 | 输出 |
|---|---|---|---|
| 项目总策划 | 拆版本目标、排优先级、避免范围失控 | 路线图、阶段跟踪、上下文 | 版本目标、需求取舍 |
| 蛊真人世界观审校 | 守术语、境界、仙凡、宝黄天、宿命/永生边界 | `reverend-insanity-lore`、canon 数据 | 世界观风险清单 |
| 玩法/数值架构师 | 守经济、战斗、奖励、AP、存档协议 | canon、engine、store | 数值闭环与套利风险 |
| 前端创意导演 | 设计现代 UI 节奏、视觉竖切和交互层级 | UI 文档、组件、视觉参考 | 视觉方案与验收截图要求 |
| Motion/Technical Artist | 守 GSAP/Motion 分工、资产 manifest、reduced motion | `reborn-combat-motion`、动效代码 | 动效方案和性能风险 |
| DeepSeek/模型调优专家 | 守模型、缓存、token、JSON 通过率、重试成本 | DeepSeek 管线、usage 日志、官方文档 | 模型矩阵、成本报告、eval 建议 |
| QA/发布稳定专家 | 守测试、长档、移动端、构建、发布说明 | package scripts、e2e、上下文 | 验收命令和剩余风险 |
| Codex 操作安全官 | 守上下文、skill、插件、C 盘、子代理、不可删除路径 | `.codex` 状态、skills、插件缓存 | Codex 使用提醒和恢复建议 |

## 启动闸门

每个小版本开始前必须回答：

- 本轮真相源在哪里？
- DeepSeek 能写什么，不能写什么？
- 哪些文件是权威入口，哪些旧文档已过期？
- 是否需要更新 `game-dev-text`、`reverend-insanity-lore`、`reborn-combat-motion` 或 `reborn-expert-council`？
- 用户需要审批哪些专家团主动提案？

## 完成闸门

每个小版本结束前必须完成：

- 阶段跟踪、路线图、上下文交接状态一致。
- 测试命令和失败/跳过原因记录清楚。
- DeepSeek 成本/缓存相关改动有可观测数据。
- 世界观硬边界没有被 UI、文档或 AI 提示词暗示突破。
