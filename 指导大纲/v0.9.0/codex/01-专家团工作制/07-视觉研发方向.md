# RebornG 视觉研发方向

## 判断

现有 React + Tailwind + GSAP + Framer Motion 足以支撑高表现力 UI。短板不是缺库，而是缺稳定的视觉原型、资产管线、动效 storyboard 和截图验收。

## 方向

- 外层保持可读、密集、可扫描的游戏 UI。
- 关键节点使用强演出：杀招、禁忌蛊、灾劫、传承封印、宿命压力、荒兽入场。
- 不做一次性全局重皮，先做“青茅山凡战创意竖切”。

## 技术分工

- GSAP：章节转场、杀招充能/释放、封印开启、反噬、荒兽入场、重大锚点。
- Motion：面板切换、按钮状态、tooltip、棋盘单位移动、目标高亮、HP/真元/status bar。
- Canvas/WebGL/Three：只在明确需要沉浸式场景或强视觉实验时引入，不默认加重运行时。

## 美术边界

可强化：

- 暗墨底、金色道痕、血色风险、玉色生机。
- 阵纹、窍壁、符箓、蛊虫纹理、战斗轨迹。

不可暗示：

- 玩家已获得仙蛊、永生、十转、宿命蛊。
- 凡人可正式交易宝黄天。
- 未登记蛊材/仙材/蛊方成为正式奖励。

## 第一条竖切

青茅山凡战：

- 标题/进入场景轻重皮。
- 月光蛊、白玉蛊、酒虫资产化。
- 5x3 棋盘强读性。
- 一次杀招或禁忌门槛演出。
- 战斗轨迹解释。
- 桌面、移动端、reduced motion 验收。
