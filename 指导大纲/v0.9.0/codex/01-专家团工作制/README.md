# RebornG 专家团工作制

更新时间：2026-05-12

专家团是 RebornG 的项目治理入口，不是一次性审查报告。它负责拆需求、提需求、审风险、守世界观、守成本、守发布质量。

## 工作文件

- `01-章程与角色职责.md`：专家团固定角色、权限和交付物。
- `02-需求提案模板.md`：专家团主动提案与用户审批格式。
- `03-版本审查模板.md`：每个小版本启动/验收审查模板。
- `04-Codex使用提醒.md`：防闪退、防上下文丢失、防子代理失控规则。
- `05-插件使用清单.md`：当前可用 Codex 插件与使用边界。
- `06-DeepSeek模型治理.md`：模型、缓存、token、质量闸门。
- `07-视觉研发方向.md`：现代创意前端和 RebornG 美术交互路线。
- `skill镜像说明.md`：全局 skill 与仓库镜像的同步规则。

## 每轮流程

1. 读取 `00-总览/README.md`、阶段跟踪、路线图、真相源索引和最新上下文。
2. 专家团提出 3-5 条候选需求，用户批准、驳回或延期。
3. 对批准需求做启动审查：真相源、风险、文件边界、验收命令。
4. 实施时按 canon -> types/schema -> engine -> store -> UI -> tests/docs。
5. 完成时做验收审查，更新阶段跟踪、路线图、上下文交接和必要 skill。
