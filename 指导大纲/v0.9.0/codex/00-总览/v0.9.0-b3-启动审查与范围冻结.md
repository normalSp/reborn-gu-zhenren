# v0.9.0-b3 启动审查与范围冻结

日期：2026-05-12
状态：范围已获用户批准，进入第一刀开发。
阶段主题：青茅山凡战视觉竖切。

## 目标

用一个可运行、可测试的青茅山凡战样板验证：

- 5x3 凡战棋盘在桌面和移动端都能读清。
- 月光蛊、白玉蛊、酒虫三类表现有明确 UI 位置。
- 至少一次强演出来自本地 battlefield 步骤，不由 UI 私自制造战斗事实。
- 战斗轨迹能解释行动、消耗、目标、失败原因和结算来源。
- reduced motion 下仍能完成核心操作。

## 非目标

- 不做全局 UI 重皮。
- 不引入新的外部 GitHub 项目。
- 不批量生成新图片资产。
- 不改写战斗伤害、命中、掉落或奖励公式。
- 不开放仙蛊、永生、十转、宿命蛊归属或宝黄天凡人交易。

处理结果：已批准。
执行口径：美术可以强化暗墨、金色道痕、血色风险、玉色生机、阵纹、窍壁和蛊虫纹理；不得让凡战特效暗示仙蛊、十转、永生、宿命蛊归属或宝黄天正式交易。

## 美术边界冻结

核心原则：视觉是表达，不是事实。

允许：

- 青灰山雾、暗墨地面、旧纸底、旧木与暗铜。
- 月光蛊的冷白月刃、直线切痕和短促余光。
- 白玉蛊的局部玉质护壳和护体轮廓。
- 酒虫的战前真元质变、酒雾、空窍内旋提示。
- 血色作为风险、禁忌、反噬和伤势提示。
- 金色作为规则线、道痕式 UI、目标范围和阵纹提示。

禁止：

- 把酒虫做成普通攻击按钮。
- 把道痕线表现成玩家获得仙级道痕。
- 把护体表现成仙体、永生或十转力量。
- 把宝箱、卷轴、仙蛊光效、交易门户作为凡战奖励暗示。
- 用全屏特效遮挡棋盘、按钮、消耗、失败原因或战斗轨迹。

## 生图提示实验

用户提出：后续用 gptimg2 生成图片时，可以尝试加入“更像完整图片”的 tag，避免厚涂插画出现破碎色块。

暂定实验口径：

```text
complete finished illustration, coherent large color masses, low fragmentation, clean grouped shadows, continuous rim light, clear focal hierarchy, image-like polished final frame
```

中文约束：

```text
完整成片感，连续大色块，低碎片密度，干净暗形，连续边缘光，焦点层级清楚，像完成度高的正式图片，不要破碎厚涂色块
```

本实验不在 b3 第一刀生成新图；等进入资产生产阶段时单张慢产验证。
