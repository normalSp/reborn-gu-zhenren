# v0.9.0-rc-type-debt 验收记录

日期：2026-05-13
状态：已完成。该阶段只处理 rc 发布前类型债，不扩玩法、不扩视觉、不改变 DeepSeek 正式叙事边界。

## 范围

- 清理 `npx tsc --noEmit --pretty false` 暴露的真实类型债。
- 保持 `tsconfig` 不通过关闭大范围严格检查来掩盖问题；只显式确认本项目当前非严格基线的 `noImplicitAny: false`。
- 修复 GSAP、管线阶段、战斗 trace、仙窍/资源节点、成就、传承、灾劫、结局、故事锚点、测试夹具和 store slice 的旧类型接口。
- 不处理新玩法、不引入新模型策略、不重做 UI。

## 主要修复簇

- GSAP 动画模块类型：改用 `gsap` 默认导出的模块类型，并修正异步动画队列返回值。
- UI/store 协议：统一 `ScreenState`、`PipelinePhase`，补 `VALIDATING_L4`、`VALIDATING_L3_RETRY` 等阶段。
- 成就与提示：补 `AchievementStats.byDomain`、`hiddenUnlocked`，修正解锁事件主键。
- 仙窍与蛊材：将旧 `immortal/status/disaster_timer` 字段对齐为 `福地/洞天`、`grade`、`next_disaster_type`、`disaster_countdown`、完整 `ResourceNode`。
- 战斗与动效：补 `BattleTracePhase.settlement`，修正骰子/突破动效 payload，保留战斗事实由 engine 输出。
- v0.8/v0.9 引擎：补灾劫、传承、结局、故事锚点、荒兽狩猎测试夹具的当前类型事实。
- DeepSeek pipeline：仅做类型桥接，不改变模型选择、缓存观测和奖励闸门策略。
