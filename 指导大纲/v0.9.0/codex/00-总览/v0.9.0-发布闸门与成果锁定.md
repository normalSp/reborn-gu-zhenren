# v0.9.0 发布闸门与成果锁定

日期：2026-05-13
状态：用户已选择“先把 v0.9.0 当前成果锁住，再开 DeepSeek 和视觉扩展”。当前 release gate 自动化验证已通过，锁定后 C-049 DeepSeek 默认模型切换与 C-050 发布前走查预检已完成。用户已明确决定不再评估其他模型，并在 2026-05-13 批准将版本标识从 rc 升为正式 `0.9.0`。本文为 `v0.9.0` 当前成果的 release gate 和范围锁定记录。

## 锁定结论

`v0.9.0` 当前成果进入发布闸门。除关键 bug、发布配置、文档一致性和人工验收记录外，不再向 `v0.9.0` 本体新增玩法、视觉资产、模型切换或存档字段。

正式版本标识：

- `package.json = 0.9.0`
- UI 显示：`v0.9.0`
- `SAVE_FORMAT_VERSION = 21`

正式公告标题暂定为 `RebornG v0.9.0《线索入世，青茅开战》`。发布渠道按 EdgeOne Pages / CodeBuddy 集成推进；对外截图/短录屏只使用通过 V-001 边界审查的 v0.9.0 功能素材。

## 纳入 v0.9.0 的成果

- `a1`：产品冲突清账与统一战斗/行动路由表。
- `a2`：道场剧情线索入口重做，存档格式提升到 `21`。
- `a3`：荒兽/荒植/守护者敌库与 7x5 棋盘适配。
- `b1`：道场、传承/福地、灾劫、野外行动首批统一行动协议桥接。
- `b2`：旧 duel/squad 正式入口收束，新正式战斗只登记候选并走本地 route policy。
- `b3`：青茅山凡战 5x3 视觉竖切、行动卡密度、源格/目标格锚点、白玉/禁忌专项验收。
- `b3-5`：青茅视觉资产 manifest 分层、泛用底图样板、入场关键演出、月光/白玉/禁忌衔接抛光、视觉验收台账。
- `rc`：版本标识、公开测试存档、DeepSeek telemetry 目标测试、运行时资产扫描、全量 TypeScript 类型债、发布说明、手测清单和 chunk 体积专项。

## 锁定后允许的改动

- 修复自动化或手测发现的 P0/P1 bug。
- 修正文档、版本说明、手测结论、风险池和上下文交接中的事实冲突。
- 调整发布配置或测试脚本，但不得改变玩法事实、存档协议或世界观边界。
- DeepSeek eval 已关闭；可为视觉扩展建立后续专项文档，但不得把对外素材使用或新视觉功能混入 `v0.9.0` 本体。

## 锁定后禁止的改动

- 新增完整玩法系统、区域剧情、蛊虫/杀招内容池或经济循环。
- 全局 UI 重皮或批量生成美术资产。
- 在 v0.9.0 收束中继续评估或切换 Pro、Reasoner 或其他模型；当前模型策略已冻结为 `deepseek-v4-flash`。
- 降低 schema、Zod、世界观闸门或 retry 规则以换取成本。
- 将青茅凡战特效或素材暗示为仙蛊、永生、十转、宿命蛊归属、凡人宝黄天交易或洞天正式认主。
