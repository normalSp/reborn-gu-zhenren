# v0.9.0 玩法系统概要与产品冲突审计

日期：2026-05-12
来源分支：`codex/v080-c26-worldloop-closure`
目标定位：玩法系统优先，不做全量原著内容优先

## 结论

项目当前不是缺少系统骨架，而是经历 v0.6 至 v0.8 多轮扩展后，出现了老系统、新竖切系统、文档承诺和产品入口之间的路由冲突。

v0.9.0 的首要目标应是统一玩法系统入口与结算路由，让道场、荒兽、剧情战斗、传承、福地、灾劫和野外行动都回到同一个闭环：

`剧情线索 -> 出发/进入场景 -> 本地引擎结算 -> 行动账本 -> 下一轮文本承接`

v0.9.0 不追求一次性复刻全部原著剧情，也不把所有传承、福地、洞天、七至九转万劫链和尊者博弈一次做完。它要先把玩法层做稳，让后续内容可以持续接入。

## 已确认产品决策

- v0.9.0 定位：玩法系统优先。
- 道场定位：剧情线索入口，不再作为随时刷新、随时修炼的独立菜单。
- 正式新战斗内容：统一走 v0.8 battlefield 路由。
- 旧 `duelState` / `squadCombatState`：保留兼容和旧测试，不再作为新剧情内容的默认入口。
- DeepSeek：只写线索、候选、传闻、叙事压力和承接文本；正式数值、战斗胜负、奖励、福地归属、荒兽掉落、道痕变化由本地引擎结算。
- 世界观边界：不开放十转、永生蛊、真正永生、玩家获得宿命蛊、普通战斗击杀尊者。

## 冲突审计

| 冲突点 | 当前源码事实 | 产品风险 | v0.9 处理 |
|---|---|---|---|
| 道场仍像独立刷子系统 | `TrainingGroundPanel` 直接刷新候选、扣费、加道痕、写 cooldown | 与场景 AP 和剧情文本割裂，玩家会觉得是在菜单刷数值 | 改成剧情线索入口，道场行动必须进入场景 AP 和行动账本 |
| 道场刷新无数据 | `training-grounds.json` 的 `chapterRequired` 只覆盖 `qingmaoshan`、`shili_jueqi`、`shanglu_qiusheng`、`nilu_ascent`、`yitian_heaven`；其他章节精确匹配会得到 0 项 | 高转或新章节存档打开道场会空白，看起来像系统坏了 | 无线索时显示条件原因；有剧情/地点/势力/福地资源点线索才显示可出发项 |
| 荒兽不是敌库 | `wild-beasts.json` 主要是规则说明，缺少可直接进棋盘的 stat、行为和杀招 | 白天/黑天 hunt 写了荒兽名，但不能稳定适配棋盘战斗 | 扩展为结构化荒兽/荒植/守护者敌库 |
| 战斗层并存 | 旧 1v1、旧小队战、新 battlefield、剧情候选路由同时存在 | 产品上不清楚何时进入哪个战斗；旧掉落/随机逻辑可能污染新闭环 | 建立统一 `CombatRoutePolicy`，正式新内容只从剧情候选进入 battlefield |
| 传承内容只是样板 | c2.5 只有小传承洞府、三王旁支、待认主福地、洞天传闻 4 类样板 | 玩家想玩全部传承/福地/洞天时会感觉落差明显 | v0.9 建内容池和接入协议，首批精选扩展，全集长期推进 |
| 场景 AP 未全覆盖 | c2.2 建立 AP 账本，但道场等旧入口仍有独立行为 | 同一时段内有的行动影响剧情，有的行动绕过剧情 | 所有正式行动统一消耗场景 AP 或明确标记为 Debug/只读 |
| 版本/文档状态可能漂移 | 底部版本标识可能仍停在 `v0.8.0-c2.5`，文档中 c2.6-c2.8 已完成 | 发布收束时用户看到的版本和实际能力不一致 | v0.9-a1 做版本标识、文档状态和上下文记录核对 |

## 战斗路由协议

v0.9.0 必须把“什么情况触发哪种战斗”写成固定产品协议，不让 UI、AI 或单个组件自行判断。

| 场景 | 路由 | 说明 |
|---|---|---|
| 单名敌人、切磋、复仇、擂台、单体野兽、无复杂地形目标 | `duel`，表现为 1v1 battlefield | 不是旧 `duelState` 默认入口；新内容仍用 v0.8 棋盘事实输出 |
| 少量敌人、蛊虫射程/遮蔽/地形是重点 | `battlefield_5x3` | 适合凡战、追击、小规模遭遇 |
| 小队、援护、士气、保护目标 | `group_5x3` | 适合护送、门派试炼、小队守卫 |
| 伏击、兽群、荒兽狩猎、灾劫战场、传承守护、第三方介入 | `group_7x5` | 作为 v0.9 重点扩展路由，默认承载道场 hunt 和大型试炼 |

旧 `CombatOverlay`、`SquadCombatOverlay` 和 `duelState` / `squadCombatState` 只保留：

- 旧存档兼容。
- 旧测试覆盖。
- Debug/回归入口。
- 后续迁移完成前的兜底显示。

## 道场 v0.9 设计

道场不再是“随时刷新道场候选”的菜单，而是剧情世界中的地点、试炼、猎场、秘境和势力训练资源。

### 进入条件

道场条目必须至少满足一种来源：

- 剧情文本或选择获得线索。
- 当前地点或章节映射到道场区域。
- 出身深线或势力权限提供训练地。
- 传承/福地/仙窍资源点解锁。
- 灾劫、荒兽、护送、商路、战场事件临时开放。

### 类型处理

| 道场类型 | v0.9 处理 |
|---|---|
| `磨练` | 消耗场景 AP，本地结算道痕/状态/风险，结果写行动账本并回流文本 |
| `对决` | 生成剧情战斗候选，按敌人规模进入 1v1 或 5x3 棋盘 |
| `试炼` | 进入剧情选择链，可派生战斗、破障、用蛊、传承线索或失败反噬 |
| `hunt` | 进入荒兽/兽群棋盘战，默认 7x5；掉落遵守荒兽寄生蛊不稳定规则 |

### 空白状态

如果当前没有可进入道场，不显示空列表。必须显示：

- 当前剧情/地点没有道场线索。
- 需要哪些条件：章节、地点、境界、势力权限、福地资源点、传承线索。
- 推荐行动：推进剧情、侦察、询问、查看传承、查看地图。

## 荒兽/守护者敌库设计

`wild-beasts.json` 需要从 lore 规则说明扩展为可运行敌库。

首批结构化条目应覆盖：

- 普通野兽：一至二转压力，用于早期 1v1 或 5x3。
- 百兽王/千兽王/万兽王/兽皇：三至五转压力，用于凡战和群像。
- 荒兽：六转压力，用于 7x5 战场和福地/灾劫/黑白天 hunt。
- 上古荒兽：七转压力，用于高危剧情，不作为普通刷怪。
- 荒植/守护者：传承、福地、灾劫和资源点守护。

每条运行时敌库至少需要：

- `id`、名称、类型、等效境界、生命/攻击/防御/速度档位。
- 流派/道痕倾向，必须来自 `path-registry.json`。
- 天赋杀招或本能行为，不等同蛊师杀招组合。
- 棋盘行为：偏好距离、移动方式、地形亲和、攻击范围、逃跑或守护逻辑。
- 掉落规则：材料、痕迹、传闻、线索；寄生蛊只可低概率、可损毁、可逃脱。

## 测试要求

- 道场：当前章节不匹配时不空白，必须显示“缺少剧情线索/地点权限”的原因；有线索后出现可出发项。
- 战斗路由：同一 `CombatEncounterSpec` 按规模稳定路由到 `duel`、`battlefield_5x3`、`group_5x3` 或 `group_7x5`。
- 荒兽：普通野兽、兽皇、荒兽、上古荒兽四档都能生成本地敌方单位；同 seed 结果一致；禁止 `Math.random`。
- 掉落：荒兽击杀不能稳定产出蛊虫；未知仙蛊、未知材料、洞天归属一律降级为传闻或拦截。
- 场景 AP：道场、传承、灾劫、野外行动都写 `LocalActionLedgerEntry`，下一轮 prompt 必须承接。
- UI：桌面和移动端都能解释为什么能进入、为什么不能进入、风险和代价。

## v0.8 遗留边界

v0.8 已经完成强系统骨架和多个竖切，但仍属于“闭环可沉浸样板”：

- 蛊虫表现化：首批 53 只蛊、15 个杀招，不是全量蛊库。
- 战斗：有 1v1/5x3/7x5 路由能力，但敌库和内容模板仍有限。
- 修行灾劫：一至五转、升仙、六转灾劫竖切完成；七至九转万劫链仍是后续。
- 中后期锚点：义天山、逆流河、梦境影宗、宿命战侧翼有系统框架；完整剧情长线仍需 content 阶段。
- 传承/福地：4 类样板，福地可认主样板存在；全部原著传承、全部无主福地/洞天和长期经营未完成。
- 道场：有数据和引擎底座，但产品入口仍需 v0.9 重做。

## 世界观边界

- 凡人不能稳定交易宝黄天或获取仙材，只能获得传闻、低阶替代资源或高风险线索。
- 蛊仙使用仙元作为能量，仙元石作为货币；文案和 UI 不得混用。
- 传承、蛊方、仙蛊、仙材、福地、洞天必须经过 canon/engine 注册，AI 不能直接发放。
- 洞天在 v0.9 可做高阶线索、禁区、天灵条件和后续入口；是否开放正式认主应单独成小版本决策。
- 荒兽掉落遵守原著边界：寄生蛊不稳定、可能损毁或逃脱，杀荒兽不是可靠刷蛊路径。
