# post-v0.9 视觉扩展与素材审查启动审查

日期：2026-05-13
状态：用户已批准在 `v0.9.0` 当前成果锁定后继续开视觉扩展专项；本专项不改变已锁定的 `v0.9.0` 本体。

## 目标

把 b3/b3-5 已验证的青茅凡战视觉竖切推进为可审查、可扩展、可对外展示的视觉资产流程。

当前优先级不是“全局重皮”，而是：

- 审查现有 b3/b3-5 截图、泛用底图、入场演出和三条短演出能否作为对外素材。
- 建立单张慢产 bitmap 图的下一步入口。
- 明确哪些素材只能做 review-only，哪些可以进入 runtime active。
- 继续保持青茅凡战的凡人尺度、战斗可读性和原著边界。

## 当前资产事实

- `src/canon/qingmao-visual-assets.json` 当前共 7 条：
  - active/runtime：月光蛊、白玉蛊、酒虫、青茅凡战泛用底图。
  - review-only：方源对白凝冰二战、青书对白凝冰林战。
  - blocked：春秋蝉。
- `public/rebrng/scenes/s0-qingmao/qingmao-mortal-battlefield-generic-atmosphere.svg` 是 C-035 泛用棋盘底图样板，不绑定具体人物战斗。
- 现有 bitmap 战斗图包括方源对白凝冰二战、青书对白凝冰林战、千里地狼蛛三族大比武冲杀等，但其中具体人物战斗图不能直接作为 generic 演武背景。
- `doc/art/s0-qingmao-art-roadmap.md` 已建立 `Single Image Cadence`、`Quality Lock Master`、`Composition Contract` 和 `Quality Drift Watch`。

## 非目标

- 不改 `v0.9.0` 已锁定运行时玩法事实。
- 不改战斗结算、伤害、命中、奖励、状态或存档格式。
- 不把具体人物战斗图冒充 generic 背景。
- 不批量生成图片。
- 不把春秋蝉、仙蛊、十转、永生、宿命蛊归属、凡人宝黄天交易或洞天正式认主做成暗示性特效。

## 美术边界

必须保留：

- 暗墨地面、青灰冷雾、金色道痕、血色风险、玉色生机、阵纹、窍壁和符箓感。
- 凡人战斗尺度：月光蛊与白玉蛊可强化辨识，但不能表现成仙蛊或仙体。
- 酒虫仍是战前真元质变/场景门槛支持提示，不作为普通攻击按钮。
- UI 可读性优先：棋盘、行动按钮、失败原因、战斗轨迹和 reduced-motion 信息不能被特效遮挡。

生成图质量提示可继续实验：

- 中文：完整成片感，连续大色块，低碎片密度，干净暗形，连续边缘光，焦点层级清楚，像完成度高的正式图片，不要破碎厚涂色块。
- 英文：`complete finished illustration, coherent large color masses, low fragmentation, clean grouped shadows, continuous rim light, clear focal hierarchy, image-like polished final frame`。

## 首批候选需求

### V-001 对外素材审查包

整理当前 b3/b3-5 Playwright 截图、桌面/移动端/reduced-motion 证据和泛用底图，生成一份“可对外 / 仅内部 / 禁用”的素材清单。

处理结果：已完成第一轮审查，见 `v0.9.0-post-visual-expansion-V001-对外素材审查包.md`；已通过 `npm run capture:qingmao:v001` 捕获 v0.9.0 公告候选截图/短录屏，记录见 `v0.9.0-post-visual-expansion-V001-素材捕获记录.md`。用户已允许将桌面总览、月光执行、白玉护体、禁忌失败和短录屏作为小规模 v0.9.0 公告候选素材。

通过标准：

- 不出现世界观越界暗示。
- 不误导玩家以为当前版本已有全部蛊虫、全部杀招或全局最终美术。
- 截图中文字可读，按钮不遮挡，棋盘事实清楚。

### V-002 青茅凡战 bitmap 泛用底图替换实验

在 C-035 SVG 泛用底图通过的前提下，按单张慢产流程尝试 1 张 bitmap 泛用底图候选。

前置条件：

- 先写 `Composition Contract`。
- 只生成 1 张候选。
- 不绑定方源、白凝冰、青书等具体人物战斗。
- 不通过时只记录拒绝原因，不连续重抽。

### V-003 青茅凡战短录屏脚本

设计 15-30 秒短录屏脚本：标题进入、青茅入场、月光蛊、白玉蛊、禁忌失败、战斗轨迹解释。

说明：V-001 已捕获一段小规模公告候选短录屏，但它不是 15-30 秒正式宣传脚本，不等于 V-003 完成。

前置条件：

- 不把 V-001 短录屏当作完整正式宣传片；它只作为小规模公告功能素材。
- 录屏必须覆盖 normal motion 与 reduced motion 的可读性差异。

### V-004 蛊虫资产补图候选池

从第一卷高价值缺图中挑 1 个低风险蛊虫资产做单张慢产，不在本轮承诺一百多种蛊虫批量补齐。

建议优先级：

- 与当前青茅凡战 UI 直接相关且不会越界的凡蛊。
- 有明确外观 seed、用途边界和 runtime 用法。
- 不先做春秋蝉、仙蛊或概念过强的高阶资产。

## 建议顺序

1. 先做 V-001，对已有素材做清单化审查。
2. 再决定是否做 V-002 或 V-003。
3. 最后才进入 V-004 蛊虫补图候选池。

## 决策边界

以下事项需要用户后续拍板：

- 是否把某张截图、短录屏或生成图用于更正式的宣传渠道。
- 是否生成新的 bitmap 泛用底图。
- 第一张要补的蛊虫或战斗场景资产是哪一个。
- 是否把某张 review-only 具体人物战斗图绑定到具体剧情节点。
