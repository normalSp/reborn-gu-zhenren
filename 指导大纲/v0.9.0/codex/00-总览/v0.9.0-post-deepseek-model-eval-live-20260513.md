## 自动门禁口径

硬门禁：

- JSON parse 通过。
- 严格 eval schema 通过。
- 命中样本期望 domain 且 domain 非空。
- 禁忌内容不得出现在事实字段中。
- 质量漂移 flag 为 0。

质量 flag：

- `qi`、`spiritual energy`、`spirit stones`、`immortal treasure`。
- 凡人语境 `heavenly tribulation`。
- 元石被写成空窍能量或被精炼成真元。
- `immortal Gu Master` 术语漂移。
- 凡人宝黄天正式访问路径漂移。
- 地灵/福地与五转蛊师主人混写。
- 模型泄漏 AP、元石、真元、百分比等正式数值成本。

`mustContain` 只保留为质量提示，不作为硬门禁。原因是 `boundary/clue/support/refusal` 等词汇可能被等价表达替代；正式切换判断以 schema、domain、boundary、quality flag 和人工抽样为主。

## 人工审查发现

旧自动门禁曾能给出 100% 通过，但人工抽样发现它不足以上线：

- `v090-qingmao-002` 曾把白玉蛊护体写成“从空窍消耗元石”，属于术语错位。
- `pressure` 曾出现对象或数字数组，`safety_notes` 曾出现字符串而不是字符串数组。
- `narrative` 曾出现空字符串。
- 宝黄天样本曾给出“凡人寻找访问路径”的行动候选。
- 传承/福地样本曾出现地灵与五转蛊师主人混写。
- 灾劫样本曾泄漏具体 AP/元石/真元数值成本。

这些问题已转化为 `scripts/run-deepseek-eval.mjs` 的严门禁和 prompt 约束。
