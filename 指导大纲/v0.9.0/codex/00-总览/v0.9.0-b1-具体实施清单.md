# v0.9.0-b1 具体实施清单

日期：2026-05-12  
状态：范围冻结已获用户批准；本文用于进入实现前的工作顺序锁定。  
主线：`线索 -> 出发 -> 本地结算 -> 行动账本 -> 回流文本`

## 执行原则

- 顺序遵循 `canon -> types/schema -> engine -> store -> UI -> tests/docs`。
- b1 只做统一协议和首批桥接样板，不补全完整内容池。
- 同一正式行动只扣一次场景 AP。
- DeepSeek 只写叙事、候选、线索、传闻、请求、压力和回流文本；正式奖励、战斗胜负、位置、资源和世界观硬事实由本地引擎决定。
- 不降低 DeepSeek 观测能力：每次正式叙事调用继续保留模型、temperature、tokens、缓存命中、重试成本、耗时和 prompt 前缀 hash。

## 建议文件范围

- 协议与类型：`src/types/index.ts`，必要时新增 `src/engine/v090-world-action-protocol.ts`。
- 场景账本：`src/engine/v080-scene-session-engine.ts`、`src/store/slices/sceneSessionSlice.ts`。
- 道场桥接：`src/engine/v090-training-ground-clue-engine.ts`、`src/store/slices/trainingGroundSlice.ts`、`src/components/game/TrainingGroundPanel.tsx`。
- 传承/福地桥接：`src/engine/v080-inheritance-land-engine.ts`、`src/store/slices/inheritanceLandSlice.ts`、`src/components/game/AperturePanel.tsx`。
- 灾劫桥接：`src/engine/v080-calamity-scene-engine.ts`、`src/engine/v080-cultivation-calamity-engine.ts`。
- 回流文本与模型观测：`src/engine/response-pipeline.ts`、`src/api/deepseek.ts`。
- 验收测试：对应 unit tests、`tests/e2e/v090-*.spec.ts`、长测入口。

## 实施步骤

1. 协议设计（第一刀已完成）  
   定义 `WorldActionCandidate`、`WorldActionDeparture`、`WorldActionResolution`、`WorldActionLedgerEntry`、`NarrativeReturnContext` 的最小字段；字段必须能表达来源、AP、风险、阻断原因、本地结果和回流摘要。

2. 账本投影（第一刀已完成最小投影）  
   先兼容现有 `LocalActionLedgerEntry`，不强行重写旧存档；新增字段必须有 normalize 兜底，除非确有必要，不提升 `SAVE_FORMAT_VERSION`。

3. 道场适配（b1-2 已完成黄金样板）
   把 a2/a3 已有线索、出发、磨练、对决、狩猎结果投影到统一协议；保持荒兽 7x5 战斗走本地 battlefield，不让 DeepSeek 结算掉落或胜负。

4. 传承/福地适配（b1-3 已完成）
   修正出发与试炼 AP 语义；候选可来自 AI 线索，但试炼、认主、奖励、福地归属必须本地校验。

5. 灾劫适配  
   将灾劫预兆、待结算场景和本地结果纳入统一候选与回流文本；不得让 AI 决定灾劫最终后果。

6. 野外行动最小样板  
   只做一条最小可测样板，用于验证协议，不扩展完整地图或资源生态。

7. Prompt 回流  
   下一轮叙事只接收 `NarrativeReturnContext` 的事实摘要，不接收可让 AI 改写数值事实的原始内部状态。

8. UI 验收面  
   首批入口必须能显示来源、AP、风险、阻断原因和本地结果；移动端不出现文字遮挡。

9. DeepSeek 观测验收  
   保留 `prompt_cache_hit_tokens`、总 tokens、重试 tokens、模型、temperature、耗时、失败原因和 prompt 前缀 hash；如果新增成本面板，只读取底层 telemetry，不重复估算。

10. 测试与文档  
   补齐协议、AP 一次扣费、道场/传承/灾劫桥接、回流文本和 DeepSeek telemetry 的测试；完成后更新阶段跟踪、决策记录、上下文交接和需要变更的 skills。

## b1 不处理

- 不全局重皮，不做 b3 视觉竖切。
- 不清理旧 `duelState/squadCombatState` 正式入口，放到 b2。
- 不开放十转、真正永生、永生蛊、稳定获得宿命蛊、凡人稳定交易宝黄天或稳定获取仙材/仙蛊。
- 不把便宜模型直接切入正式叙事；模型切换另走 eval。
