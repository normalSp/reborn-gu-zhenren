# v0.9.0 发布说明候选稿

日期：2026-05-13
状态：正式发布候选稿。`v0.9.0` 当前成果已锁定，用户已批准版本标识升为 `0.9.0`，公告标题由 Codex 拟定为 `RebornG v0.9.0《线索入世，青茅开战》`，对外截图/短录屏可使用通过 V-001 边界审查的 v0.9.0 新功能素材。

## 版本定位

`v0.9.0` 是 RebornG 的正式阶段收束版本，重点是把 v0.9 已完成的玩法协议、战斗入口、青茅凡战视觉竖切、公开测试存档和 DeepSeek 成本观测收束到稳定状态。

## 面向玩家的更新重点

- 道场、传承/福地、灾劫、野外行动现在统一到“线索 -> 出发 -> 本地结算 -> 行动记录 -> 叙事回流”的行动结构，玩家能更清楚看到行动来源、AP 消耗、阻断原因和本地结果。
- 新剧情战斗不再混入旧 duel/squad 正式入口，正式战斗候选由本地战斗路线校验；旧系统继续承担兼容和调试兜底。
- 道场狩猎可进入荒兽/守护者 7x5 battlefield；奖励边界由本地敌库和经济闸门决定，不把荒兽战斗做成稳定刷蛊路径。
- 青茅山凡战 5x3 视觉竖切已完成：月光蛊、白玉蛊、酒虫支持提示、禁忌门槛、棋盘轨迹、青茅战场氛围底图、入场演出和 reduced motion 都有基础表现。
- 公开测试存档已更新为当前格式，可用于快速体验古月/熊家/白家开局、小队战、宝黄天、福地和洞天边界样本。

## 推荐体验入口

- 新手或快速检查：从标题页载入 `public/test-saves/test-01-guyue-start.json`，确认版本、地点、行动入口和叙事可读。
- 战斗体验：载入 `test-05-sanwang-squad-combat.json` 或从 Debug/演武入口进入青茅山凡战，重点看棋盘、行动卡、月光/白玉/禁忌演出。
- 经济边界：载入 `test-08-economy-tyh-pools.json`，确认宝黄天正式交易只面向蛊仙层级。
- 福地/洞天边界：载入 `test-09-inheritance-land.json` 与 `test-10-grotto-heaven-boundary.json`，确认 AI 只承接线索和压力，不直接授予归属或奖励。

## 设计边界

- DeepSeek 负责叙事、线索、候选和承接文本；正式数值、奖励、地点、战斗结果和原著硬事实仍由本地系统决定。
- 凡人阶段不能正式交易宝黄天，也不能稳定获取仙材、仙蛊或洞天归属。
- 当前青茅凡战视觉特效不代表玩家获得仙蛊、永生、十转、宿命蛊归属或未开放高阶事实；春秋蝉仍不作为 b3 凡战 active 资产。
- 酒虫只作为场景门槛/真元支持提示，不作为普通攻击按钮或正式伤害来源。

## 技术与稳定性

- 版本标识已统一为 `v0.9.0` / `0.9.0`。
- DeepSeek 正式叙事默认模型已在 C-049 后切换为 `deepseek-v4-flash`；调用会保留模型、temperature、tokens、缓存命中/未命中、重试成本和 prompt 前缀 hash，便于后续成本分析。
- 公开测试存档、经济回路、青茅资产、运行时资源、长测、构建和 TypeScript 全量检查均已通过 rc-type-debt 后验收。
- C-045 已完成发布包拆分收束，`combat-squad` 和通用 `engine` 不再触发 500KB+ chunk warning。
- 已补 React/React DOM/Node 类型声明，并清理全量 TypeScript 类型债；后续若开启更严格 TypeScript 选项，需要另开专项。
- 当前 `v0.9.0` 本体已进入 release gate；DeepSeek 模型选择已收束为 `deepseek-v4-flash`，视觉扩展不会继续混入本版本功能范围。

## 暂不承诺

- 不承诺 v0.9.0 已完成全部蛊虫、全部杀招、全部区域剧情或完整荒兽生态。
- 不承诺未来会切换到 Pro、Reasoner 或其他模型；用户已决定不再评估其他模型。
- 不承诺当前青茅视觉竖切代表全游戏最终美术风格；它是先验证可读性与关键演出的局部样板。

## 发布执行仍需确认

- EdgeOne Pages 的账号连接、部署项目、域名和发布渠道。
- 对外截图/短录屏素材需要使用 V-001 边界审查后的清单，正式发布文案不得暗示未开放世界观事实。
- DeepSeek 模型策略已冻结为 `deepseek-v4-flash`，不再把其他模型评估列为发布前事项。
