# v0.9.0-rc 启动审查与范围冻结

日期：2026-05-13
状态：用户已批准进入 `v0.9.0-rc`；范围冻结为发布候选收束，不再扩玩法或全局视觉重皮。

## 版本目标

`v0.9.0-rc` 的职责是把 a1/a2/a3、b1、b2、b3/b3-5 已完成的系统收口到可发布候选状态。重点不是“再做一个大功能”，而是确认版本事实一致、长测可读档、经济和 DeepSeek 成本观测不回退、文档不再互相打架。

## 已批准范围

- 长测与关键 e2e 回归。
- 测试存档与公开导入存档格式复核。
- 经济闭环、宝黄天、蛊材/仙材边界复验。
- DeepSeek 模型、tokens、缓存命中、重试成本、prompt 前缀 hash 的可观测性复盘。
- 文档冲突清理、真相源索引和上下文交接同步。
- `package.json`、标题页、游戏页页脚等版本标识收口。
- 关键 bug 修复。

## 非目标

- 不新增完整玩法系统。
- 不全局 UI 重皮。
- 不批量生成美术图。
- 不在 rc 本体中盲切正式叙事模型；锁定后 C-049 已在固定 eval、schema、世界观闸门、成本评估和用户批准后切换到 `deepseek-v4-flash`。
- 不提升 `SAVE_FORMAT_VERSION`，除非出现必须持久化的新字段。

## DeepSeek 复盘结论

当前运行时默认模型以 `src/api/deepseek.ts` 的 `DEEPSEEK_DEFAULT_MODEL` 为准；C-049 后为 `deepseek-v4-flash`。用户已决定不再评估其他模型。

必须继续保留并观测：

- `model`
- `temperature`
- `prompt_tokens`
- `completion_tokens`
- `total_tokens`
- `prompt_cache_hit_tokens` / `cached_tokens`
- 缓存命中率
- 重试次数与重试 token
- 耗时
- 失败原因
- prompt 前缀 hash

`deepseek-v4-flash` 已成为正式运行时默认模型；其他便宜模型、Pro 或 Reasoner 不再作为 v0.9.0 后续评估候选。
