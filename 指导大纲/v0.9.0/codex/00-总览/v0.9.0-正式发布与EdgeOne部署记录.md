## EdgeOne / CodeBuddy 集成

当前 Codex 会话可用插件列表中没有 Tencent Cloud / EdgeOne 专用插件或 skill，因此不能直接在本会话里完成账号授权、创建 EdgeOne 项目或发布到生产域名。仓库侧已补 `edgeone.json`，用于让 CodeBuddy 或 EdgeOne Pages 明确识别 Vite 静态站点构建方式。

仓库配置：

- 配置文件：`edgeone.json`
- 安装命令：`npm ci`
- 构建命令：`npm run build`
- 输出目录：`./dist`
- Node 版本：`22.11.0`

推荐集成路径：

1. CodeBuddy IDE 内使用官方 EdgeOne Pages 部署入口，把当前仓库部署到 EdgeOne Pages。
2. 如果 CodeBuddy/AI 助手支持 EdgeOne Pages Skill，可按官方说明安装 `TencentEdgeOne/edgeone-pages-skills`，安装后重启 CodeBuddy/Codex 让 skill 列表重新加载。
3. 如果需要让 AI 直接查询/创建/管理 EdgeOne 项目，再接 EdgeOne Pages MCP，但 Token、项目 ID、域名等账号信息不要写入仓库。
4. 如果仓库托管在 GitHub，也可以让 EdgeOne Pages 连接 GitHub 仓库，使用 `edgeone.json` 的构建配置自动部署。

官方依据：

- EdgeOne Pages CodeBuddy IDE 部署文档：`https://edgeone.ai/document/205358412301221888`
- EdgeOne Pages AI/MCP/Rules/Skill 入口：`https://pages.edgeone.ai/document/start-with-ai`
- EdgeOne Pages Skill：`https://pages.edgeone.ai/document/pages-skills`
- `edgeone.json` 配置：`https://pages.edgeone.ai/document/edgeone-json`
- Vite 部署入口：`https://pages.edgeone.ai/document/vite`
