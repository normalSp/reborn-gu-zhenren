# v0.9.0-b2 启动审查与范围冻结

日期：2026-05-12  
阶段：`v0.9.0-b2` 启动审查  
状态：范围已获用户批准，进入实现  
关联 skill：`reborn-expert-council`、`game-dev-text`、`reverend-insanity-lore`、`reborn-combat-motion`

## 专家团结论

可以进入 `v0.9.0-b2`。本阶段目标不是删除旧战斗引擎，而是把旧 `duelState/squadCombatState` 收束为旧存档、Debug 和单元测试兼容层；新正式剧情战斗、狩猎、传承守护、灾劫战斗和野外遭遇继续走 battlefield。

## 范围冻结

本阶段只处理旧战斗系统兼容收束：

- 正式剧情链不得再直接 `initDuel()` 或直接进入旧 `duelState`。
- 旧叙事关键词战斗触发必须转成 `combat_event_candidates`，再由本地 route policy 和 battlefield 引擎决定入场。
- AI `combat_result` 不能直写 HP、真元、伤势、元石、材料或掉落，只能降级为线索/日志。
- 行动面板、场景锁和回流文本必须识别 active battlefield，不能在战斗中并行另起野外行动。
- 旧 `duelState/squadCombatState` 仍可被旧存档读取，旧单元测试和 Debug 演示仍可保留。

## 不做什么

- 不删除旧 `combat-engine.ts` 和 `squad-combat-engine.ts`。
- 不重写 battlefield UI。
- 不做 `v0.9.0-b3` 青茅山凡战视觉竖切。
- 不引入外部 GitHub 项目接管流程。
- 不切换 DeepSeek 正式叙事模型。

## 文件边界

- 战斗触发与路由：`src/engine/combat-router.ts`、`src/engine/combat-route-policy.ts`、`src/engine/response-pipeline.ts`
- AI 结算闸门：`src/engine/ai-state-update-validator.ts`、`src/engine/state-update-applier.ts`
- 场景锁：`src/engine/activity-availability.ts`、必要 UI 消费层
- 兼容和验收测试：对应 `*.test.ts`
- 文档同步：当前入口、阶段跟踪、路线图、真相源索引、需求决策池、上下文交接、skill 当前事实

## 验收口径

- 叙事关键词命中的旧 duel 场景只生成战斗候选，不直接生成 `duelState`。
- 接受候选后进入 battlefield；`duel` 语义表示 1v1 battlefield。
- AI 直写 `combat_result` 会被 validator/applier 双重降级。
- active battlefield 会锁定行动面板。
- 旧存档兼容测试继续通过。
- `npm test` 与 `npm run build` 通过；如 `npx tsc --noEmit` 仍有历史类型债，不作为本轮阻断，但要记录。
