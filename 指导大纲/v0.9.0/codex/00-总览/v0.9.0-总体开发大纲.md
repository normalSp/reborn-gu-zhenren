# RebornG v0.9.0 总体开发大纲

更新时间：2026-05-12  
当前入口分支：`codex/v090-a3-beast-guardian-battlefield`  
依据来源：v0.8 阶段跟踪、v0.9 产品冲突审计、a1/a2 上下文、当前源码事实

## 总目标

v0.9.0 的定位是玩法系统优先。v0.8 已经建立战斗、修行、灾劫、锚点、终局、传承、福地和场景 AP 的系统骨架；v0.9 不继续堆独立面板，而是把这些系统统一到同一条世界闭环：

`剧情线索 -> 出发/进入场景 -> 本地引擎结算 -> 行动账本 -> 下一轮文本承接`

玩家仍然是原创角色。DeepSeek 负责叙事、线索、候选、传闻和压力，本地引擎负责血量、真元、仙元、资源、战斗胜负、奖励、掉落、福地归属、道痕变化、灾劫后果和终局结果。

## 核心产品决策

- 正式新战斗只走 v0.8 battlefield 路由；旧 `duelState` / `squadCombatState` 只保留兼容、旧测试和 Debug。
- 道场是剧情线索入口，不再是随时刷新、随时刷收益的独立菜单。
- 荒兽、荒植、守护者必须进入结构化敌库后才能进入棋盘战斗。
- 传承、福地、道场、灾劫、野外行动共用线索、场景 AP、战斗路由和行动账本。
- 不开放十转、永生蛊、真正永生、玩家获得宿命蛊、普通战斗击杀尊者。

## 系统边界

- 凡人不能稳定交易宝黄天或获得仙材；高阶资源只可作为传闻、线索或高压入口，必须由本地规则闸门校验。
- 荒兽寄生蛊不稳定，可能损毁、逃脱或同亡；杀荒兽不是可靠刷蛊路径。
- 洞天在 v0.9 先作为高阶线索、禁区边界、天灵条件和后续入口；是否开放正式认主需要单独小版本决策。
- v0.9 先完成玩法系统统一和精选样板，不承诺一次性复刻全部原著剧情、全部传承和全部福地洞天。

## 关键系统主线

- 战斗路由：`duel`、`battlefield_5x3`、`group_5x3`、`group_7x5` 固化为产品协议。
- 道场线索：剧情、地点、势力、传承、福地资源点解锁正式道场行动。
- 敌库：荒兽、荒植、守护者具备等效境界、流派倾向、棋盘行为、天赋杀招和掉落边界。
- 统一出发协议：传承、福地、道场、灾劫、野外行动都按线索和 AP 进入本地结算。
- 旧入口收束：旧战斗 UI、旧道场刷新、旧即时用蛊不作为正式玩家路径。

## 验收原则

- 每个小版本都要有文档、源码、测试、上下文交接。
- 运行时流派名必须来自 `src/canon/path-registry.json`。
- 涉及奖励、材料、战斗、灾劫、福地、荒兽掉落的逻辑必须有本地引擎测试。
- 涉及 UI 和动效的阶段必须验证桌面、移动端和 reduced motion。
