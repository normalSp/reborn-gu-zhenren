# v0.9.0-b3-5 视觉资产阶段范围冻结

日期：2026-05-13

## 结论

用户希望在 b3 已收束后继续做更炫的视觉/美术扩展。专家团同意单独冻结 `v0.9.0-b3-5`，名称为：

`青茅山凡战视觉资产与关键演出增强阶段`

本阶段仍然不是全局重皮，也不是新玩法阶段。目标是在 b3 已有可读战棋盘基础上，做一层可控的视觉资产和强演出增强，让青茅山凡战更接近现代创意前端/游戏演出质量，同时保持战斗事实、世界观和移动端可读性稳定。

## 阶段目标

1. 建立 b3 视觉资产生产与入库口径，避免“好看的图”直接混进运行时。
2. 选择 1 个青茅山凡战泛用场景底图或氛围资产作为首个视觉资产样板，不使用绑定具体原著人物战斗的场景图充当 generic 背景。
3. 为青茅山凡战增加一段“入场/开战”关键演出设计，仍只消费现有 battlefield state 和 `BattleResolutionStep`。
4. 补齐视觉资产阶段的截图/验收规则：桌面、移动端、reduced motion、性能和世界观边界。
5. 保留 rc 入口，不把 b3-5 扩成无边界美术重做。

## 范围内

- 扩展 `src/canon/qingmao-visual-assets.json` 或新增配套 manifest 字段，用于区分：
  - active runtime 资产。
  - candidate 美术候选。
  - review-only 参考图。
  - blocked/forbidden 资产。
- 基于 `doc/art/s0-qingmao-art-roadmap.md` 的 `Quality Lock Master` 和 `Single Image Cadence`，为 1 张青茅山凡战泛用底图建立 `Composition Contract`。
- 可选择使用现有素材、用户提供素材或单张慢产新图；如果生成新图，每次只生成 1 张候选，必须先验收再入库。
- 青茅山战场 UI 可做局部视觉增强：
  - 入场遮罩/标题节奏。
  - 背景氛围层。
  - 月光蛊/白玉蛊/禁忌门槛的短演出衔接。
  - 棋盘与行动面板的轻微材质统一。
- Playwright 覆盖桌面、移动端、reduced motion，确认文字不遮挡、按钮可点、棋盘非空、战斗事实仍来自 engine。

## 范围外

- 不做全站/全游戏 UI 重皮。
- 不改战斗规则、伤害、命中、AI、AP、奖励、掉落或存档协议。
- 不新增 Three.js/WebGL 运行时，除非后续单独冻结技术实验。
- 不把“方源对白凝冰二战”“青书对白凝冰林战”等具体人物场景图作为 generic 演武背景。
- 不批量生成新图，不一次性做多张候选。
- 不把春秋蝉、仙蛊、十转、永生、宿命蛊归属、宝黄天正式交易放入凡战视觉承诺。

## 美术边界

可强化：

- 暗墨底、青茅山冷雾、旧纸灰绿、金色道痕、血色风险、玉色护体、空窍环、阵纹、符箓、月刃轨迹。

不可暗示：

- 玩家获得仙蛊。
- 白玉蛊变成仙体或绝对免伤。
- 酒虫造成攻击伤害。
- 春秋蝉参与当前凡战。
- 凡人正式交易宝黄天。
- 未登记奖励、蛊材、仙材、蛊方进入运行时事实。

## 技术边界

- React + Tailwind + GSAP + Framer Motion 继续作为技术底座。
- GSAP 只管入场、强演出、关键节点；Motion 只管按钮、面板、棋盘交互状态。
- UI 动效只读：
  - `BattlefieldCombatState`
  - `BattleResolutionStep`
  - `qingmao-visual-assets.json`
  - 已登记 canon 表现
- 不在 UI 中计算或推断伤害、命中、状态、胜负或奖励。

## b3-5 候选需求

### C-033 b3-5 视觉资产阶段范围冻结

目标：把用户提出的“继续做更炫视觉/美术扩展”独立成有边界的小阶段。
交付：本范围冻结文档、路线图/阶段跟踪/需求池同步。
验收：下一阶段入口明确为 `v0.9.0-b3-5`，rc 顺延到 b3-5 完成后。

### C-034 视觉资产 manifest 分层

目标：让青茅山视觉资产从“能显示”升级为“可审查、可入库、可阻断”。
交付：manifest 字段或扫描脚本增强，能区分 active/candidate/review-only/blocked，并解释每张图是否可进入 runtime。
验收：active 只包含安全运行时资产；具体人物战斗图不能误进 generic 背景。

### C-035 青茅凡战泛用场景底图样板

目标：建立 1 张非具体人物绑定的青茅山凡战氛围图，用于 b3 战场视觉增强。
处理结果：已随 C-035 执行。
交付：`Composition Contract`、泛用 SVG 底图样板、manifest 登记、Playwright 加载验收。
验收：不暗示方源/白凝冰/青书等具体战斗身份，不出现仙蛊/十转/宝黄天等越界视觉。

### C-036 青茅战场入场关键演出

目标：增强进入战场时的第一印象，而不遮挡棋盘和行动按钮。
处理结果：已随 C-036 执行。
交付：GSAP 入场/标题节奏或背景氛围层的局部实现。
验收：桌面与移动端可读；reduced motion 下不依赖动画理解战斗。

### C-037 月光/白玉/禁忌演出衔接抛光

目标：让已有月光蛊、白玉蛊、禁忌门槛演出更顺滑、更有层次。
处理结果：已随 C-037 执行。
交付：只基于 `BattleResolutionStep` 的动效细化。
验收：不新增战斗事实，不遮挡失败原因和战斗轨迹。

### C-038 视觉资产阶段验收台账

目标：把 b3-5 形成可复查的视觉验收记录。
交付：截图台账、Playwright 附件、资产扫描结果、剩余风险。
验收：`npm run check:qingmao-assets`、b3 Playwright、必要旧 battlefield 回归通过。

## 开工条件

- 用户确认进入 `v0.9.0-b3-5` 开发。
- 若需要生成新图，必须先选择 1 个资产目标，不能批量摇图。
- 若使用现有图片，必须先判断其是 generic、scene-specific 还是 blocked。

## 完成定义

- 有源码或资产 manifest 证据。
- 有 b3-5 审查文档和上下文交接。
- 有桌面、移动端、reduced motion 验收。
- 有资产扫描结果。
- 不破坏 b3-4 已有月光/白玉/禁忌验收。

## 后续关系

`v0.9.0-b3-5` 完成后再进入 `v0.9.0-rc`。rc 仍负责长测、测试存档、经济复验、DeepSeek 成本观测复盘、文档冲突清理和版本标识收口。
