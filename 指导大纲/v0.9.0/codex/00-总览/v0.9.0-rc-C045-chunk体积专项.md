# v0.9.0-rc C-045 chunk 体积专项

日期：2026-05-13

## 目标

关闭 rc 风险池中的 R-002：历史构建存在 `combat-squad` 500KB+ chunk warning，后续一度转移为通用 `engine` chunk 过大。本专项只做低风险打包边界调整，不改战斗规则、存档协议、世界观事实或 UI 交互语义。

## 改动

- 新增 `src/ui/dice-roll-bus.ts`，把掷骰动画触发函数从 React 组件中抽成非 React 事件总线。
- `DiceRollAnimation.tsx` 只负责绑定和播放动画。
- `combatSlice.ts` 改为从 `dice-roll-bus` 触发掷骰，不再从 React 组件模块反向导入。
- `vite.config.ts` 从旧 `rollupOptions.output.manualChunks` 迁移到 Rolldown 推荐的 `rolldownOptions.output.codeSplitting.groups`。
- 按 vendor、canon、audio、combat、battlefield、action、economy、story、engine 分组，并设置 `maxSize: 450_000`。

## 构建结果

`npm run build` 通过，且不再出现 500KB+ chunk warning。

关键 chunk 结果：

- `combat-squad`：约 65KB。
- `engine-battlefield`：约 172KB。
- `engine-economy`：最大约 248KB。
- `engine-story`：约 178KB。
- `engine-action`：约 87KB。
- 通用 `engine`：约 18KB。

## 风险判断

- 行为风险低：只调整事件总线导入路径和构建拆包规则。
- 世界观风险无：没有改 canon、叙事、奖励、战斗结果或 DeepSeek 边界。
- 运行风险仍需用长测兜底：拆包会改变加载文件边界，最终 rc 门禁必须继续跑长测。

## 后续

- R-002 可关闭。
- 若未来新增大模块导致 500KB+ warning 复发，重新打开 R-002。
- 若要进一步做首屏性能、预加载策略或路由级延迟加载，另开性能专项，不挤入本轮 rc 收束。
