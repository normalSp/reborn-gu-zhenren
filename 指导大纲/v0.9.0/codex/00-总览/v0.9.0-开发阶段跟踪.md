# RebornG v0.9.0 开发阶段跟踪

更新时间：2026-05-12  
当前分支：`codex/v090-a3-beast-guardian-battlefield`  
当前阶段：`v0.9.0-a3` 进入

## 当前状态

v0.9.0 已从 v0.8.0 世界闭环收束后进入玩法系统统一阶段。当前已经完成 a1/a2，正在推进 a3。

## 已完成阶段

### v0.9.0-a1 产品冲突清账与统一战斗/行动路由表

分支：`codex/v090-a1-product-route-closure`  
状态：已完成  
存档协议：保持 `SAVE_FORMAT_VERSION = 20`

落地内容：

- 新增 `CombatRoutePolicy`，统一 `duel / battlefield_5x3 / group_5x3 / group_7x5` 路由。
- `duel` 明确为 1v1 battlefield，不再默认进入旧 `duelState`。
- 道场入口策略输出缺线索、地点不匹配、境界不足、冷却、Debug 等原因。
- 演武入口降级为 Debug/演武，不再与正式剧情战斗混排。
- 版本标识更新为 `v0.9.0-a1`。

### v0.9.0-a2 道场剧情线索入口重做

分支：`codex/v090-a2-training-ground-clues`  
状态：已完成  
存档协议：正式提升到 `SAVE_FORMAT_VERSION = 21`

落地内容：

- 新增持久 `trainingGroundState`。
- `state_update.training_ground_candidates.add` 只能提交线索候选。
- `TrainingGroundPanel` 改为线索账本。
- 磨练消耗场景 AP 并本地结算；对决生成统一战斗候选；试炼写叙事推进意图。
- `hunt` 留给 a3 荒兽/荒植/守护者敌库，不在 a2 结算掉落。

## 当前阶段

### v0.9.0-a3 荒兽/荒植/守护者敌库与棋盘适配

分支：`codex/v090-a3-beast-guardian-battlefield`  
状态：实施中  
存档协议：不提升，继续 `SAVE_FORMAT_VERSION = 21`

目标：

- 扩展 `wild-beasts.json` 为运行时敌库。
- 首批覆盖普通野兽、兽王、兽皇、荒兽、上古荒兽、荒植、守护者。
- 让 `hunt` 道场从线索出发进入 7x5 群像战。
- 敌方阶段使用荒兽本能行动，不伪装成蛊师持蛊。
- 战后只结算合法材料、线索、传闻和风险，不稳定掉蛊或仙蛊。

验收命令：

- `npm test -- src/engine/v090-beast-enemy-registry.test.ts src/engine/v090-training-ground-clue-engine.test.ts src/engine/v080-narrative-combat-orchestration.test.ts`
- `npm test`
- `npm run build`
- `npx playwright test tests/e2e/v090-beast-hunt-battlefield.spec.ts`
- `npm run test:e2e:long`

剩余风险：

- `wild-beasts.json` 旧文本存在编码显示问题，新增运行时字段必须保持 JSON 可解析。
- `tg_white_heaven`、`tg_black_heaven` 旧描述中有“仙蛊掉落率”，a3 必须改为寄生蛊传闻/调查概率，不能直接入背包。
- 旧 `duelState` 仍有随机掉落逻辑，a3 不改旧系统，b2 再做正式旧入口收束。
