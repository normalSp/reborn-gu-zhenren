# v0.9.0-rc 手测走查清单

日期：2026-05-13
状态：发布闸门候选定稿。自动化通过不等于体验通过，本清单用于发布前人工扫一遍关键流程；`v0.9.0` 当前成果已锁定，手测只确认体验和阻断，不新增功能范围。

## 走查环境

- 启动：`npm run dev`
- 构建预览：`npm run build` 后 `npm run preview`
- 视口：桌面 1440x900、移动 390x844
- 浏览器设置：正常动效与 reduced motion 各走一次
- 记录方式：每项用 `通过 / 失败 / 延后` 标记；失败项必须写明复现入口、存档文件和截图或日志。

## 0. 自动化门禁复核

正式手测前先确认最近一次自动化门禁没有回退：

- `npx tsc --noEmit --pretty false`
- `npm test`
- `npm run build`
- `npm run check:runtime-assets`
- `npm run check:qingmao-assets`
- `npm run test:e2e:long`

通过标准：

- 上述命令退出码为 0。
- C-045 后构建不应再出现 500KB+ chunk warning；如果重新出现，打开 R-002。
- 如果出现新的 TypeScript、测试、资产或长测失败，重新打开 rc 风险池，不进入对外素材审查。

## 1. 标题页与 API 连通

- 标题页显示 `v0.9.0-rc`。
- DeepSeek 连通测试成功后，能看到模型、耗时、总 tokens、缓存命中/未命中 tokens、缓存命中率和 prompt prefix hash。
- 默认模型显示为 `DEEPSEEK_DEFAULT_MODEL` 实际值，不写死宣传式“Pro”。
- API 失败时有明确错误，不阻塞本地载入存档。
- 不要求本机必须有可用 API key；无 key 环境下重点确认失败提示和本地存档入口可用。

## 2. 公开测试存档导入

使用 `public/test-saves/`：

- `test-01-guyue-start.json`：一转古月山寨开局，行动入口和叙事可读。
- `test-05-sanwang-squad-combat.json`：三王山小队战，战斗/队伍信息可读。
- `test-08-economy-tyh-pools.json`：宝黄天全品类交易池，确认只对蛊仙开放正式交易。
- `test-09-inheritance-land.json`：待认主福地，确认地灵/资源/归属边界不由 AI 直接决定。
- `test-10-grotto-heaven-boundary.json`：洞天边界传闻，确认只是禁区压力和后续入口，不开放正式认主。

通过标准：

- 导入不报错。
- 屏幕不空白。
- 玩家境界、地点、叙事、行动入口和关键系统面板可读。
- 不出现凡人宝黄天正式交易、仙蛊掉落、洞天认主、十转/永生奖励等越界事实。

## 3. 青茅凡战视觉竖切

- 从 Debug/演武入口进入青茅山凡战。
- 确认 5x3 棋盘非空，行动卡可点，战斗轨迹可读。
- 确认青茅战场氛围底图加载成功；若底图缺失，UI 仍有可读兜底。
- 入场演出不遮挡行动按钮，reduced motion 下以静态信息替代。
- 月光蛊执行后，源格/目标格演出对位清楚。
- 白玉蛊护体后，护体演出不遮挡单位和按钮。
- 禁忌门槛失败时，显示失败/反制原因，不播放成成功攻击。
- 酒虫只作为场景门槛/真元支持提示，不作为普通攻击按钮。
- 移动端与 reduced motion 下仍能理解战斗事实。

## 4. 统一行动协议

重点走：

- 道场线索 -> 出发 -> 本地结算 -> 行动账本 -> 回流文本。
- 传承/福地试炼或认主：确认同一行动不重复扣 AP。
- 灾劫预兆与本地后果：确认 AI 只写压力和承接，不写正式损益。
- 野外采集/侦察：确认材料、地点、战斗触发由本地引擎决定。

## 5. DeepSeek 成本观测

正式叙事调用后检查：

- 游戏日志或返回结果中能看到总 tokens。
- 能看到缓存命中比例或 cached tokens。
- 能区分初次调用与 retry 成本。
- 能看到模型名、temperature 和 prompt prefix hash。
- 同版本同模式下 prompt prefix hash 应保持稳定；若频繁变化，说明缓存前缀被动态内容污染。

## 6. 发布前保留风险

- `npx tsc --noEmit --pretty false` 已在 `rc-type-debt` 后通过；若后续新增类型错误，重新打开 R-001。
- `combat-squad` / `engine` chunk 体积风险已在 C-045 后关闭；若构建重新出现 500KB+ warning，重新打开 R-002。
- 宣传素材使用和更严格 TypeScript 选项都需要单独决策；DeepSeek 其他模型评估已按用户决定关闭。
- DeepSeek eval 与视觉扩展可作为锁定后专项启动，但不得改变当前 `v0.9.0` 的运行时玩法事实、存档协议或世界观边界。
