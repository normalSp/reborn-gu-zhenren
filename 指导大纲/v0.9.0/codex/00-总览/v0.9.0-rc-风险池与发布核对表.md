## P1 工程风险

### R-001 TypeScript 全量类型债

状态：已关闭。
已做：补 `@types/react`、`@types/react-dom`、`@types/node`，并在 `tsconfig.json` 启用 `node` 类型；随后开 `rc-type-debt` 小阶段清理真实类型债。
当前结论：`npx tsc --noEmit --pretty false` 已通过。

已清理类别：

- GSAP 类型导入方式与 `timeline` 类型。
- `ScreenState` / `PipelinePhase` / store slice 类型不一致。
- 福地、洞天、资源节点、传承候选数据结构与 UI 旧字段不一致。
- `BattleTracePhase` 中 `settlement` 等新阶段未统一。
- 旧测试夹具与当前类型定义脱节。
- 若干组件隐式 `any` 与旧字段引用。

验收记录：`v0.9.0-rc-type-debt-验收记录.md`。后续若要开启更严格 TypeScript 选项，应另开专项。

### R-002 构建 chunk 体积

状态：已关闭。
现象：历史构建中 `combat-squad` chunk 超过 500KB，后来转移为通用 `engine` chunk 过大。
已做：C-045 将旧 `manualChunks` 迁移到 Rolldown `output.codeSplitting.groups`，并把掷骰触发函数从 React 组件中抽到非 React 事件总线，避免 store/chunk 反向拖入 UI 组件。
当前结论：`npm run build` 通过，最大业务 chunk 约 248KB，未出现 500KB+ chunk warning。

## P1 质量风险

### R-003 DeepSeek 正式模型切换

状态：已关闭。
当前默认模型以 `src/api/deepseek.ts` 为准，C-049 后为 `deepseek-v4-flash`。pipeline 已记录模型、temperature、tokens、缓存命中/未命中、重试 tokens 和 prompt 前缀 hash。

已关闭依据：

- Flash full live + 严门禁回放 40/40 accepted。
- 用户批准 C-049。
- `npx tsc --noEmit --pretty false`、`npm test`、`npm run build`、`npm run eval:deepseek:dry-run` 和中文 smoke 均通过。

后续重新打开条件：仅当用户未来主动撤销“不再评估其他模型”的决定时，才重新讨论模型评估；不得由专家团主动提出 Pro/Reasoner/其他模型切换。

### R-004 b3 宣传素材/短录屏

状态：部分关闭。
当前 b3/b3-5 已有 Playwright 截图附件口径，V-001 已完成受控捕获。用户已允许制作一组 v0.9.0 新功能对外截图/短录屏素材，允许清单见 `v0.9.0-post-visual-expansion-V001-对外素材审查包.md` 与 `v0.9.0-正式发布与EdgeOne部署记录.md`。

需要用户决策的触发条件：

- 是否把当前 SVG 泛用底图作为对外截图素材。
- 是否生成或采购 bitmap 成片图。
- 是否将某张具体人物战斗图用于宣传或游戏内 generic 背景。

### R-005 锁定后范围回流

状态：已设闸门。
风险：视觉扩展或宣传素材审查很容易把新需求带回 `v0.9.0` 本体，导致 rc 反复膨胀。
当前规则：`v0.9.0` 本体只接受关键 bug、发布配置、文档一致性和验收记录修改；DeepSeek 模型评估已停止，`post-v0.9-visual-expansion` 可以建立文档、样本和候选池，但不得改变已锁定的运行时玩法事实、存档协议或正式发布口径。
