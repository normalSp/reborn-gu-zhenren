# post-v0.9 DeepSeek 模型 eval 启动审查

日期：2026-05-13
状态：用户已批准在 `v0.9.0` 当前成果锁定后开启 DeepSeek 评估专项，并已表示 API token 消耗无所谓；本专项仍不切换正式叙事模型。

## 目标

建立一套可重复的 DeepSeek 模型对照评估，回答三个问题：

- 当前默认模型在正式叙事中是否稳定、便宜、可缓存。
- `deepseek-v4-flash` 或其他便宜模型能否承担正式游戏叙事。
- Pro/Reasoner 类模型是否只应留在离线评审、疑难修复或对照样本中。

## 当前代码事实

## 官方依据

核对时间：2026-05-13。

- DeepSeek Models & Pricing：官方当前列出 `deepseek-v4-flash` 与 `deepseek-v4-pro`，`deepseek-chat` 与 `deepseek-reasoner` 是兼容别名，未来会废弃。价格会变化，正式成本报告必须以评估当天官方页面为准。
- DeepSeek Context Caching：上下文缓存默认启用；命中要求后续请求完整匹配已持久化的缓存前缀单元；API usage 可返回 `prompt_cache_hit_tokens` 与 `prompt_cache_miss_tokens`；缓存是 best-effort，不保证 100% 命中。
- DeepSeek JSON Output：JSON 模式需要设置 `response_format: {'type': 'json_object'}`，并在 system 或 user prompt 中包含 json 约束和期望格式示例；仍需处理空内容和截断风险。

## 非目标

- 不在本专项直接修改 `DEEPSEEK_DEFAULT_MODEL`。
- 不降低 schema、Zod、L3/L4、世界观、奖励和战斗结果闸门。
- 不把 DeepSeek 输出的蛊虫、仙蛊、蛊方、材料、战斗胜负、地点、奖励或洞天/福地归属直接写入正式状态。
- 不用本专项承诺省钱比例；没有真实 usage 与当日价格，不写成本结论。

## 候选模型组

评估组必须至少包含：

- 当前默认：`deepseek-v4-flash`。
- 历史兼容别名：`deepseek-chat`。
- 高质量对照：`deepseek-v4-pro` 或当日官方等价 Pro 模型。

可选组：

- `deepseek-reasoner` 或当日官方等价 thinking mode，用于少量高难世界观/规则对照，不作为运行时默认首选。
- 其他便宜模型只可进入离线草稿或预筛，正式叙事必须先补相同 eval。

## Eval 样本集

首批固定样本建议 8 类，每类至少 5 条，共 40 条起步：

1. 普通叙事承接：玩家状态、地点、境界、短行动后回流。
2. 统一行动协议：道场/传承/灾劫/野外行动的 `线索 -> 出发 -> 本地结算 -> 行动账本 -> 回流文本`。
3. 青茅凡战叙事：月光蛊、白玉蛊、酒虫支持、禁忌门槛，只写叙事和候选，不私算伤害。
4. 经济边界：元石、蛊材、仙材、宝黄天和凡人/蛊仙交易边界。
5. 传承/福地/洞天边界：AI 只写线索、地灵条件和压力，不直接授予归属或奖励。
6. 灾劫压力：AI 只写氛围和选择压力，不写本地损益。
7. 对抗样本：诱导模型发放仙蛊、永生、十转、宿命蛊归属、完整蛊方或宝黄天交易。
8. JSON 稳定性样本：长 prompt、高动态状态、retry 修复和空内容兜底。

## 必须记录的指标

- 首轮 JSON parse 通过率。
- normalize + Zod 通过率。
- L3/L4 critical 违规率。
- 奖励/资源/战斗结果越界率。
- retry 率与 retry tokens。
- prompt tokens、completion tokens、total tokens。
- `prompt_cache_hit_tokens`、`prompt_cache_miss_tokens`、cache hit ratio。
- prompt prefix hash 稳定性。
- elapsed ms。
- 单位有效叙事成本，价格以评估当天官方页面为准。
- 人工可读性评分：叙事自然度、世界观贴合、候选可解释、玩家行动清晰度。

## 通过口径

模型要成为正式叙事候选，至少满足：

- JSON 首轮通过率 >= 95%。
- normalize + Zod 通过率 >= 99%。
- L3/L4 critical 违规率 < 3%。
- 不得出现可被接受的仙蛊、永生、十转、宿命蛊归属、凡人宝黄天正式交易、未登记完整蛊方或未登记正式奖励。
- 同版本同模式 prompt prefix hash 稳定；若 hash 频繁变化，先修 prompt 分层，不急着换模型。
- 成本低于当前默认但叙事质量明显变差时，不允许直接切换；只能作为草稿、预筛或专家团离线提案模型。
