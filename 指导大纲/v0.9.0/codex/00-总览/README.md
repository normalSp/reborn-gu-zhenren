## 专家团入口

专家团工作制位于 `../01-专家团工作制/`。每个小版本开始前必须生成启动审查；每个小版本完成后必须生成验收审查；专家团还要主动提出 3-5 条候选需求，由用户批准、驳回或延期。

## 当前下一步

`v0.9.0-b1` 的目标不是新增一个孤立面板，也不是一次性补全所有传承、福地、灾劫和野外行动内容池，而是先把传承、福地、道场、灾劫、野外行动接到同一个协议：

`线索 -> 出发 -> 本地结算 -> 行动账本 -> 回流文本`

DeepSeek 只能写叙事、候选、线索、传闻和承接文本；正式数值、奖励、位置、战斗结果和原著硬事实必须由本地 canon/engine 决定。

`v0.9.0-b1-启动审查与范围冻结.md` 中的关键决策已获用户批准。进入实现时必须按 `v0.9.0-b1-具体实施清单.md` 推进，并保持 DeepSeek 模型、tokens、缓存命中、重试成本和 prompt 前缀 hash 可观测。
