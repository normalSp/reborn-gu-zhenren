## 专家团入口

专家团工作制位于 `../01-专家团工作制/`。每个小版本开始前必须生成启动审查；每个小版本完成后必须生成验收审查；专家团还要主动提出 3-5 条候选需求，由用户批准、驳回或延期。

## 当前下一步

`v0.9.0-b1` 已把传承、福地、道场、灾劫、野外行动接到同一个协议：

`线索 -> 出发 -> 本地结算 -> 行动账本 -> 回流文本`

DeepSeek 只能写叙事、候选、线索、传闻和承接文本；正式数值、奖励、位置、战斗结果和原著硬事实必须由本地 canon/engine 决定。

`v0.9.0-b2` 已完成旧战斗正式入口收束：叙事关键词命中的旧 duel/narrative 战斗不再直接开 `duelState` 或 transient 结算 prompt，而是登记 `combat_event_candidates`，由本地 route policy 与 battlefield 引擎决定入场；AI `combat_result` 直写会被降级为线索/日志。

`v0.9.0-b3` 青茅山凡战视觉竖切已完成最小样板、storyboard/资产桥接、行动卡信息密度、源格/目标格对位、资产 manifest、白玉护体/禁忌门槛专项验收和 manifest 扫描脚本。`v0.9.0-b3-5` 已完成视觉资产 manifest 分层、C-035 泛用场景底图样板、C-036 入场关键演出、C-037 月光/白玉/禁忌演出衔接抛光和 C-038 验收台账。rc 发布闸门已完成版本标识、公开测试存档 `formatVersion 21` 镜像和目标回归；`rc-type-debt` 已关闭全量 TypeScript 类型债；C-044/C-045 已完成发布说明、手测走查和 chunk 风险收束。`v0.9.0` 当前成果已范围锁定，release gate 自动化已通过，正式版本标识已收口为 `0.9.0` / `v0.9.0`；DeepSeek 模型 eval 已以 C-049 收束到 `deepseek-v4-flash`，用户已决定不再评估其他模型；V-001 已完成现有素材分级审查和对外候选截图/短录屏捕获；C-050 已完成发布前资源/e2e 预检。
