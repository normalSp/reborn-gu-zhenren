# v0.9.0-b3-5 C-037：月光/白玉/禁忌演出衔接抛光

日期：2026-05-13

## 结论

C-037 已完成。本轮只抛光青茅凡战已有月光蛊、白玉蛊和禁忌门槛的短演出衔接，不改战斗规则、伤害、命中、AI、AP、奖励、掉落或存档协议。

新增表现：

- 月光蛊月刃分支增加第二层 `moon echo`，让月刃从源格到目标格的过渡更清楚。
- 白玉蛊护体分支增加 `jade crack`，表现护体承压和消散，不暗示仙体或绝对免伤。
- 禁忌门槛/失败/反制分支增加 `boundary sigil`，并优先于月光、白玉来源判断，避免“月光蛊越界失败”被误播成月刃成功演出。
- e2e 新增 `data-qingmao-polish` 验收标记，分别覆盖 `moon-transition`、`white-jade-transition`、`forbidden-transition`。

## 范围边界

- 演出仍只消费 `BattleResolutionStep`、已登记 storyboard 和 effect anchor。
- UI 不计算或推断伤害、命中、状态、胜负或奖励。
- 失败/反制/禁忌门槛结果优先表达为边界阻断，而不是继续按来源蛊虫播成功特效。
- 不引入 Three.js/WebGL，不新增图片，不扩玩法。
- 不暗示仙蛊、十转、永生、宿命蛊归属、凡人宝黄天交易或未开放禁术奖励。
