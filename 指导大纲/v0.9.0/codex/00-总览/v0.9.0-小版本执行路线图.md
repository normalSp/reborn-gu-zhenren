# RebornG v0.9.0 小版本执行路线图

更新时间：2026-05-13
当前入口分支：`codex/v090-b1-world-action-protocol`
当前阶段：`v0.9.0` 正式发布配置；用户已批准锁住 `v0.9.0` 当前成果，第一刀版本标识、公开测试存档、目标回归、类型债、发布说明/手测走查、chunk 体积专项、release gate 自动化复核、锁定后 C-049 DeepSeek 默认模型切换、C-050 发布前走查预检、正式版本标识收口和 EdgeOne Pages 仓库配置均已完成。用户已决定不再评估其他模型，后续不再向 `v0.9.0` 本体新增玩法、视觉资产、模型切换或存档字段。

## 总原则

v0.9.0 采用玩法系统优先路线。所有新系统按 `canon -> types/schema -> engine -> store -> UI -> tests/docs` 推进。DeepSeek 只写候选、线索、传闻、叙事压力和承接文本，不写正式数值结果。

## 固定边界

- 不开放十转、永生蛊、真正永生、玩家获得宿命蛊、普通战斗击杀尊者。
- 凡人不能稳定获取仙材、交易宝黄天或认主洞天。
- 荒兽掉落不作为稳定刷蛊路径。
- 洞天正式认主、太古荒兽常规战、完整七至九转万劫链需单独后续版本决策。

## 每阶段完成定义

- 有源码、测试、文档、上下文交接。
- 涉及 UI 的阶段通过桌面、移动端、reduced motion 验证。
- 涉及本地结算的阶段必须有 seeded RNG 测试，并禁止直接调用 `Math.random`。
- 阶段完成后提交并推送当前 `codex/v090-*` 分支。
