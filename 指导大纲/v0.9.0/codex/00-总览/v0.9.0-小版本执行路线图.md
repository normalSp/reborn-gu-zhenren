# RebornG v0.9.0 小版本执行路线图

更新时间：2026-05-12  
当前入口分支：`codex/v090-a3-beast-guardian-battlefield`

## 总原则

v0.9.0 采用玩法系统优先路线。所有新系统按 `canon -> types/schema -> engine -> store -> UI -> tests/docs` 推进。DeepSeek 只写候选、线索、传闻、叙事压力和承接文本，不写正式数值结果。

## 小版本路线

| 小版本 | 目标 | 交付 | 验收重点 |
|---|---|---|---|
| `v0.9.0-a1` | 产品冲突清账与统一路由表 | 战斗路由策略、道场入口策略、旧入口 Debug 降级、版本核对 | 新剧情战斗不触发旧 `duelState/squadCombatState` |
| `v0.9.0-a2` | 道场剧情线索入口重做 | `trainingGroundState`、线索账本、AP 消耗、对决候选、试炼意图 | 无线索不显示空白；有线索后可出发 |
| `v0.9.0-a3` | 荒兽/荒植/守护者敌库与棋盘适配 | 结构化敌库、hunt 7x5 战斗、荒兽本能行动、合法掉落 | 白天/黑天狩猎可进入棋盘，敌方会行动，不稳定掉蛊 |
| `v0.9.0-b1` | 统一出发协议 | 传承、福地、道场、灾劫、野外行动共用线索、出发、账本、回流文本 | 所有正式行动都能解释来源、AP、风险、结果 |
| `v0.9.0-b2` | 旧战斗系统兼容收束 | 旧 duel/squad 降级兼容、正式入口清理、迁移/测试兜底 | 新内容只走 battlefield，旧存档仍可读 |
| `v0.9.0-rc` | 发布候选 | 长测、测试存档、经济复验、文档冲突清理、版本标识收口 | `npm test`、`npm run build`、长 e2e 全部通过 |

## 固定边界

- 不开放十转、永生蛊、真正永生、玩家获得宿命蛊、普通战斗击杀尊者。
- 凡人不能稳定获取仙材、交易宝黄天或认主洞天。
- 荒兽掉落不作为稳定刷蛊路径。
- 洞天正式认主、太古荒兽常规战、完整七至九转万劫链需单独后续版本决策。

## 每阶段完成定义

- 有源码、测试、文档、上下文交接。
- 涉及 UI 的阶段通过桌面、移动端、reduced motion 验证。
- 涉及本地结算的阶段必须有 seeded RNG 测试，并禁止直接调用 `Math.random`。
- 阶段完成后提交并推送当前 `codex/v090-*` 分支。
