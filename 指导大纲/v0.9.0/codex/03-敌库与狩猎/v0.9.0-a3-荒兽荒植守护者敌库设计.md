# v0.9.0-a3 荒兽/荒植/守护者敌库设计

更新时间：2026-05-12  
执行分支：`codex/v090-a3-beast-guardian-battlefield`

## 目标

a3 把 `hunt` 道场从“待系统”推进为正式剧情线索入口：玩家通过剧情获得狩猎线索，消耗场景 AP 出发，进入 7x5 battlefield，与荒兽、荒植、守护者进行本地结算战斗。战后结果回到行动账本和下一轮叙事。

## 敌库结构

`wild-beasts.json` 保留现有 lore 规则，并新增运行时字段：

- `enemyTiers`：普通野兽、百兽王、千兽王、万兽王、兽皇、荒兽、上古荒兽、荒植、守护者。
- `enemyRegistry`：首批结构化敌人，记录名称、来源、等效境界、流派倾向、生命/攻击/防御/速度、地形偏好、棋盘行为、天赋杀招和掉落策略。
- `huntGroundBindings`：道场和敌人池绑定，首批覆盖白天、黑天。
- `dropTables`：材料、线索、传闻、寄生蛊调查结果；不直接发仙蛊或稳定蛊虫。

## 战斗规则

- 荒兽不是蛊师，不使用普通 `guNames`。
- 敌方单位通过 `BeastInstinctMove` 行动，例如扑击、毒雾、缠绕、风翼突袭、魂波压迫、守护封印。
- `group_7x5` 是 hunt 默认路由，使用 `ambush_7x5` 预设。
- 敌方阶段必须播放行动轨迹，不能空转。
- UI 和动效只消费 `BattleResolutionStep[]`，不私算伤害、掉落或胜负。

## 掉落边界

- 胜利可获得已登记材料、通用流派蛊材/仙材、线索、传闻或后续调查入口。
- 寄生蛊可能损毁、逃脱、同亡或形成线索；不得直接稳定进入背包。
- 未知材料、未知仙蛊、十转、永生蛊、宿命蛊归属、真正永生全部拦截或降级为传闻。
- 失败和撤退只产生压力、伤势、线索损失或风险记录，不发正收益。

## UI 表现

- `TrainingGroundPanel` 显示敌情预览、AP 成本、路线、风险、掉落边界和寄生蛊不稳定提示。
- `ChoicePanel` 显示荒兽狩猎、兽群、荒植守护、传承守护标签。
- `BattlefieldCombatOverlay` 显示当前阶段、敌方荒兽行动、天赋杀招和战后结算轨迹。
- Motion 负责卡片、tooltip、列表和棋盘状态；GSAP 只用于荒兽入场、伏击、荒植缠绕、守护封印等独立效果层。

## 验收

- `tg_white_heaven` 和 `tg_black_heaven` 有线索后可进入 7x5 战斗。
- 敌方阶段有本能行动并输出轨迹。
- 同 seed 下敌人池、行动和掉落一致。
- 不稳定掉蛊、仙蛊、十转、永生相关内容不会进入正式库存。

## 实施落点（2026-05-12）

已按本设计完成 a3 竖切：
- `wild-beasts.json` 使用 `enemyRegistry`、`huntBindings`、`dropPolicies` 承载运行时敌库。
- `v090-beast-enemy-registry` 是敌库唯一运行时入口，UI 与 store 不私算敌人池或掉落。
- `tg_white_heaven`、`tg_black_heaven` hunt 线索可生成 7x5 battlefield encounter。
- 荒兽单位使用 `instinctMoves`，敌方阶段输出 `BattleResolutionStep[]`。
- 掉落只允许已登记材料、通用材料、线索、传闻、拦截记录；寄生蛊、仙蛊、十转、永生蛊不进入正式背包。
