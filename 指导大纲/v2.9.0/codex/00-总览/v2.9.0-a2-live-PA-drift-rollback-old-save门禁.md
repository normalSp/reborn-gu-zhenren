## 一句话结论

v3.0 如果进入有限 runtime agent 试点，必须先有 live probe、Player Advocate、长线漂移、rollback 和 old-save 方案。v2.9-a2 只冻结方案，不调用 live DeepSeek。

## Live Probe 方案

默认模型：`deepseek-v4-flash`。

建议 v3.0 设计门禁前确认两档：

| 档位 | 用途 | 建议样本 |
|---|---|---|
| smoke | b1/b2 小改后快速确认 agent 文本边界 | 3 samples x 4 rounds |
| rc | 进入 v3.0 runtime agent 前硬门 | 5 samples x 12 rounds，必要时 replay/rescore |

必须记录：

- 是否调用 live DeepSeek。
- 模型。
- 样本数。
- 轮次。
- 成本。
- report path。
- P0/P1/P2。
- retry / repair / rejected counts。

## Player Advocate 模板升级

每份 v3.0 Player Advocate 记录必须新增：

- 是否调用 live DeepSeek：否 / 是。
- 若是：模型、样本、轮次、成本、报告路径。
- agent layer：L2 / L3。
- agent output type：dialogue / intent / memory / social follow-up candidate。
- 玩家是否能分清候选表达和正式事实。
- 玩家是否知道下一步怎么利用 NPC/势力反馈。
- 玩家是否能看见选择造成了可解释后果。

## 长线漂移门禁

v3.0 触碰 runtime agent 后，必须检查：

- 世界状态不漂。
- agent 不越权写正式事实。
- 不重复空转。
- 不遗忘玩家目标。
- 不把 private/hidden/source-pointer body 写进 visible text。
- 不把 L4/L5 事件误判成普通 NPC 意图。

## Rollback / Old-save 方案

v3.0 如果改 runtime：

- 必须有 old-save compatibility record。
- 必须说明是否 bump `SAVE_FORMAT_VERSION`。
- 若新增 save field，必须同改 migration/defaults/tests。
- 若不新增 save field，必须说明 agent candidate 如何不进入持久状态。
- 必须提供 rollback path：关闭 agent candidate surface 后，WorldCore / local deterministic path 仍可运行。

## 退出结论

a2 完成。冻结 `v290_a2_live_pa_drift_rollback_oldsave_gate_v1`。v2.9 不实跑 live，不改变 prompt/context/model/authority。
