# v2.9.0-a1 runtime agent contract 与 WorldCore post-check 门禁

日期：2026-05-24
状态：completed
冻结模型：`v290_a1_runtime_agent_contract_worldcore_postcheck_v1`

## 一句话结论

v3.0 若进入设计门禁，只能讨论“一个小区域内有限 L2/L3 agent 输出候选和表达”。agent 不写事实，WorldCore 保持最终裁决权。

## Runtime Agent Contract 草案

未来 v3.0 agent output 必须满足：

| 字段 | 要求 |
|---|---|
| `agentLayer` | 只允许 `L2` / `L3` |
| `scope` | 一个小区域；不能扩展完整南疆、完整商家城或完整 agent society |
| `authority` | `proposal_only` |
| `worldCoreFinalAuthority` | 必须为 true |
| `requiresWorldCorePostCheck` | 必须为 true |
| `ttlTurns` | 必须有限；L3 expression 推荐 1-2，L2 memory/intent 推荐 4-12 |
| `sourceRefs` | 只能引用已允许的 public/synthetic/source-pointer evidence |
| `privateRefs` | 默认禁止；若需要 private canon，必须未来 gate |
| `allowedOutputs` | dialogue candidate、intent candidate、memory candidate、manual review note、safe next step |
| `prohibitedOutputs` | save/runtime/canon/prompt/knowledge/body writes、formal reward/location/faction/identity、NPC life-death |

## WorldCore Post-Check 草案

post-check 必须输出四类结果：

| 结果 | 含义 |
|---|---|
| `accepted_candidate` | 只作为候选进入表达层或人工复核 |
| `lab_only` | 能保留在实验室，不进入玩家 runtime |
| `rejected_violation` | 越权或危险，必须拒绝 |
| `needs_user_decision` | 触发 F-290/F-300，停止自动推进 |

## 机械拒绝项

以下任一出现，必须 `rejected_violation` 或 `needs_user_decision`：

- agent 写 save/runtime/store/canon/prompt/knowledge-index。
- agent 直接创建正式地点、阵营、身份、奖励、NPC 生死。
- DeepSeek visible lore/RAG/hidden/private body。
- L4/L5 runtime。
- 天道/宿命 runtime 裁决。
- backend/BFF/service/job queue/eval archive service/cloud save。
- external framework PoC/dependency/vendored/read-only scan/patch artifact/subagents。
- MiroFish export/intake 或真实原著事实 without blocking review。
- self-learning 直接改 canon/runtime/save/prompt。

## v3.0 最小可讨论能力

可讨论：

- L2：短期记忆候选、目标候选、社会后续候选。
- L3：当前场景语言表达候选、意图候选、压力提示候选。

不可讨论：

- L1 live LLM routine。
- L4 原著关键人物 live agent。
- L5 天道/宿命 runtime agent。
- 完整 agent society。
- external framework runtime adapter。

a1 完成。冻结 `v290_a1_runtime_agent_contract_worldcore_postcheck_v1`。可以进入 a2 live/PA/drift/rollback/old-save 门禁。
