`v3.0 前安全收束：权限、prompt、eval、Player Advocate、MiroFish 边界统一复核`

5. 冻结 v3.0 live probe / Player Advocate / drift gate 档位草案。
6. 冻结 v3.0 MiroFish need classification 与升级条件。
7. 输出 v3.0 go/no-go checklist。

## 核心设计原则

### WorldCore 最终裁决

v3.0 即使接入有限 L2/L3 runtime agent，agent 也只能输出候选和表达。正式事实、数值、地点、奖励、身份、NPC 生死、战斗后果、canon promotion 仍由 WorldCore/local core 裁决。

### AgentProposal 单一入口

agent 输出必须进入结构化 `AgentProposal`，再由 WorldCore post-check 判定：

- `accepted_candidate`
- `lab_only`
- `rejected_violation`
- `needs_user_decision`

### L2/L3 限定

v3.0 只能讨论少量 L2/L3。L4/L5 不进入 runtime，除非未来单独审批。

### private canon 不进 DeepSeek visible

hidden/private body 不进入 prompt，不进入 archive body，不进入玩家可见文本。MiroFish/source-pointer 只能作为候选、规则、测试输入，必须经过 intake review。

## 成功标准

v2.9 完成时必须能回答：

- v3.0 是否可以进入设计门禁。
- 若可以，试点区域、agent 层级、能力、禁止项、回滚、测试和用户审批边界是什么。
- 若不可以，阻塞项是哪一类：eval、prompt、MiroFish、private canon、old-save、rollback、live probe、Player Advocate、外部框架、self-learning 或代码一致性。

## 不成功标准

以下任一出现，v2.9 不能进入 v3.0：

- P0/P1 false negative。
- 发现文档/代码阻塞差异但未修复或未降级。
- 需要 runtime agent / save / prompt / backend / MiroFish blocking / external dependency，但没有用户审批。
- live probe / Player Advocate / rollback / old-save 方案不完整。
- v3.0 被描述为完整 agent society、所有 NPC live、完整南疆或高阶 L4/L5 runtime。
