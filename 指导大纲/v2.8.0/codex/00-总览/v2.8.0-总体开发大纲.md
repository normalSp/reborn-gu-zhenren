## 设计原则

- WorldCore 拥有最终事实和裁决权。
- Agent 只能产出 `AgentProposal`。
- L2/L3 准入必须先证明可撤销、可审计、可拒绝。
- 准入不等于上线，v2.8 不批准 runtime agent。
- L4 原著关键人物 live 和 L5 天道/宿命 runtime 裁决默认拒绝。
- self-learning 只能生成候选，不自裁决。
- 外部 framework 只能作为架构参考，不做 PoC、依赖或扫描。

## 工程边界

允许：

- 文档、schema、admission criteria、capability matrix、test matrix、truth-source index。
- 自有 zero-dependency / dry-run / report-only admission checker，前提是 D-280-007 获批。
- synthetic / redacted fixtures。
- report-only artifact。
- dashboard、PROJECT-STATE、AGENTS、skill sync audit。

不允许：

- runtime/source/UI/store/prompt/save 变更。
- backend/BFF/service。
- live DeepSeek。
- MiroFish export/intake。
- 外部依赖、PoC、subagents、patch artifact。
- 正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。

## 与 v3.0 的关系

v2.8 只能回答：

- 哪些 L2/L3 能力可能进入 v3.0 设计讨论。
- 进入 v3.0 前需要哪些 Player Advocate / live probe / drift / rollback / old-save 证据。
- 哪些能力必须继续 `lab_only`。
- 哪些能力应直接 `rejected`。

v2.8 不能回答：

- runtime agent 已经批准。
- live DeepSeek NPC 已经可上线。
- 后端或 job queue 应立即实现。
- 原著关键人物或天道/宿命可以 live。
