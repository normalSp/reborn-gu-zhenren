## 专家团意见

### Project Lead

v2.8 不能直接变成 v3.0 的 runtime agent 实现。它应该做“准入门禁”：把 v2.1-v2.7 的证据翻译成明确规则，判断哪些能力可进入 v3.0 设计，哪些必须继续 report-only，哪些直接拒绝。

### Systems Architect

准入门禁应分三类：

- `admissible_candidate`：未来可进入 v3.0 小区域试点设计。
- `lab_only`：继续留在 Agent Lab，不进 runtime。
- `rejected`：违反 WorldCore / lore / save / hidden / authority 边界，直接拒绝。

任何可准入能力都只能输出 `AgentProposal`，WorldCore 仍拥有最终事实、数值、奖励、位置、NPC 生死、正史锚点和存档裁决权。

### AI Pipeline Architect

v2.8 不应调用 live DeepSeek。它可以设计 v3.0 的 live probe 档位建议，例如 L2/L3 表达候选需要多少样本、多少轮、什么失败率才可进 runtime，但实际调用应留到未来 D 项。

### Canon / Knowledge Curator

v2.8 默认 MiroFish `not_needed`。原因：准入门禁只处理 synthetic / redacted / report-only 证据。若准入规则开始引用命名 NPC、真实势力、hidden-adjacent、方源公开证据、正式 lore 结论或 L4/L5 原著锚点，MiroFish 必须升级 `blocking`。

### Lore Guard

L2/L3 可以成为未来低阶区域的“表达和候选意图”来源，但不能成为原著事实解释器。L4 原著关键人物 live、L5 天道/宿命 runtime 裁决、正式地点/阵营/奖励/NPC 生死仍应保持 future gate。

### QA / Eval Guardian

v2.8 的核心是定义准入证据链：

- offline runner 通过。
- eval farm 通过。
- hidden/private visibility gate 通过。
- WorldCore post-check 通过。
- Player Advocate 可理解性方案明确。
- 长线漂移方案明确。
- rollback / old-save / no-authority-expansion 方案明确。
