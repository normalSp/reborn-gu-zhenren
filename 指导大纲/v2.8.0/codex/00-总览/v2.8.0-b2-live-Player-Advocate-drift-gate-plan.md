# v2.8.0-b2 live / Player Advocate / drift gate plan

日期：2026-05-24
状态：completed

## v2.8 当前豁免

v2.8 不运行 Player Advocate 与 live DeepSeek。

理由：

- 不改玩家可见 runtime/UI。
- 不改 DeepSeek prompt/context/model/authority。
- 不进入正式剧情链路。
- 不写 save、runtime、canon 或知识库正文。

## v3.0 建议档位

若 v3.0 批准有限 L2/L3 runtime agent 试点，建议：

- a1/a2：先设计门禁，不跑 live。
- b1：deterministic proposal replay，至少 40 条 synthetic/redacted cases。
- b2：Player Advocate 20-30 轮，检查玩家是否能理解 agent 候选与 WorldCore 裁决差异。
- rc：live DeepSeek probe 40-60 轮，模型沿用 `deepseek-v4-flash`，只测表达/意图候选，不测试正式 outcome。

## 必须停机的 live 条件

- live probe 需要改变 prompt/context/model/authority。
- 出现 P0/P1 hidden leak、formal authority drift、WorldCore bypass。
- 需要把 DeepSeek visible lore/RAG 接入上下文。
- live 样本触及 L4/L5、真实原著事实、命名 NPC、hidden-adjacent。

## 长线漂移建议

v3.0 若进入 runtime agent，应至少增加：

- L2 memory contamination drift。
- L3 intent repetition / empty narration drift。
- WorldCore rejection consistency drift。
- same-start replay difference audit。
- old-save rollback audit。

## MiroFish blocking 条件

只要 v3.0 讨论进入真实地点、真实势力、命名 NPC、方源公开证据、hidden-adjacent 或正式 lore 结论，MiroFish 从 `not_needed` 升级为 `blocking`。
