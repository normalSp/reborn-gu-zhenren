# v4.2.0 例外停机清单

状态：completed；执行过程中未触发例外停机。

若发生以下任一事项，立即停止 `/goal` 自动推进并回到用户决策。

## 运行时与存档

- 需要修改 runtime/source/UI/store/prompt/save 行为。
- 需要迁移凡阶战斗 runtime 或改变玩家低阶战斗操作方式。
- 需要新增 save field、migration、`SAVE_FORMAT_VERSION` bump 或 `runFingerprint`。
- 需要把 report-only mapping 写入正式存档、正式 ledger 或 runtime canon。

## 战斗与表现

- 需要实现 theater UI、Auto-Theater runtime、纯自走棋 runtime 或高阶战斗 runtime。
- 需要生成、接入或晋升 Auto-Theater 视觉素材、动画资产、GSAP timeline 或 Motion UI。
- 需要新增战斗数值、伤害公式、命中判定、奖励、掉落、NPC 生死、地点破坏或正式战后结论。

## DeepSeek / MiroFish / lore

- 需要调用 live DeepSeek，或扩大 prompt/context/model/authority。
- 需要让 DeepSeek 裁决战斗结果、资源消耗、奖励、地点变化或 NPC 生死。
- 触及真实原著战役、命名 NPC、真实势力、hidden-adjacent、方源证据、HeavenWill/Fate 真实锚点、L4/L5 原著锚点或正式 lore conclusion。
- MiroFish need level 从 `not_needed` 升级为 `preferred` 或 `blocking`。
- 需要写 knowledge-index body、runtime canon、hidden/private body 或 DeepSeek visible lore/RAG。

## 基础设施与外部依赖

- 需要 backend/BFF/service/job queue/eval archive/cloud save。
- 需要外部 framework PoC、dependency、vendored subset、read-only scan、patch artifact 或 subagents。
- 需要外部 agent 项目写入、命令执行或 Git 操作权限。

## 发布与 Git

- 需要公开口径、法律/版权边界修改、EdgeOne 部署、branch protection 修改、force-push、history rewrite 或 auto-main merge。
- GitHub Actions / 本地验证连续失败，且需要改变方案或降低门禁。

## 设计取舍

- 需要从“凡阶 Lite 映射”转向“凡阶重做”“纯自走棋”“高阶 runtime first cut”。
- 需要在玩家可读性与战斗深度之间做重大取舍。
- 需要把候选、准备、pressure 或失败原因表达成正式事实。
