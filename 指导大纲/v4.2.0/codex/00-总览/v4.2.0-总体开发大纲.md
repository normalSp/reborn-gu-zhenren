# v4.2.0 总体开发大纲

状态：startup_prepared；等待 D-420 审批。

## 主线

`Auto-Theater Lite / 凡阶战斗映射设计门禁`

## 版本目标

1. 把现有凡阶战斗拆成可映射的 Lite 结构。
2. 定义 `AutoTheaterLiteMortalMappingV1`：source action、preparation、transcript、Combat Ledger Lite、expression authority、evidence。
3. 建立凡阶样本族：月光蛊攻击、白玉防御、酒虫支援、低阶兽战、失败原因、移动端/reduced-motion 可读性。
4. 明确未来 runtime 迁移前必须通过的 checker、Player Advocate 和 old-save/no-save/rollback 证据。
5. 防止 v4.2 被误用为高阶战斗、纯自走棋、theater UI 或凡阶 runtime 迁移授权。

## 设计原则

- 凡阶先 Lite，不直接 Full Theater。
- 现有棋盘 / 行动卡先保留，Auto-Theater Lite 先做解释层和证据层。
- Combat Core / WorldCore 拥有最终裁决权。
- DeepSeek 只可表达或提出候选，不能裁决战斗。
- 表现层不得暗示未开放的奖励、掉落、NPC 生死、地点破坏或正式身份/阵营变化。
- 移动端与 reduced-motion 必须是第一等验收项。

## 完成后能说什么

- 可以说：项目已经有 Auto-Theater Lite 映射门禁，可以用同一合同解释现有凡阶战斗。
- 可以说：未来凡阶 combat UI 或 runtime 迁移前，有可测试的 mapping matrix 与 negative fixtures。
- 不能说：凡阶战斗已迁移到 Auto-Theater runtime。
- 不能说：高阶战斗、theater UI、纯自走棋、DeepSeek 战斗表达或素材生成已开放。
