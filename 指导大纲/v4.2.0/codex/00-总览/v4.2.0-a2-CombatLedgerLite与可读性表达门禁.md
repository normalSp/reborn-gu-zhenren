# v4.2.0-a2 Combat Ledger Lite 与可读性表达门禁

状态：completed。
冻结名：`v420_a2_lite_ledger_readability_expression_gate_v1`。

## Combat Ledger Lite lane

| Lane | 允许表达 | 禁止暗示 |
|---|---|---|
| `facts` | Combat Core / WorldCore 已确定事实 | 不新增命中、伤害、奖励或死亡 |
| `preparation` | 蓄势、站位、目标选择、资源准备 | 不把准备写成已发生事实 |
| `support` | 真元、辅助、牵制、队友配合 | 不把支援写成攻击或伤害来源 |
| `defense` | 抵挡、规避、承受、代价提示 | 不写成无敌、反杀或新防御公式 |
| `failure_reason` | 距离、真元、状态、目标、风险门禁 | 不把失败写成奖励、惩罚或正式结论 |
| `candidate` | 未裁决候选行动/意图 | 不把候选画成事实 |
| `rejected` | 被门禁拒绝的行动或表达 | 不让拒绝结果影响 runtime |
| `needs_user_decision` | 需要玩家/用户选择的分叉 | 不自动替玩家选择 |

## 表达权限

- UI：可以展示短句、lane 标签、证据摘要和拒绝原因；不得计算或暗示战斗结果。
- 动效：可以表达已确定事实和候选态势；不得让特效承担命中、击杀、奖励或地点破坏裁决。
- DeepSeek：未来若另批，只能做战斗表达和候选意图；不得裁决命中、伤害、胜负、奖励或 NPC 生死。
- checker/report：可以保存 redacted evidence 与 source refs；不得保存 hidden/private body。

## 可读性门禁

- 移动端必须优先短句、摘要和可折叠 lane，不遮挡行动按钮。
- reduced-motion 下必须通过文本、图标、顺序和标签读懂战斗，不依赖动画节奏。
- 玩家必须能区分“事实、准备、支援、防御、失败原因、候选、拒绝、需决策”。

## 结论

a2 通过。`CombatLedgerLiteV1` 是表达/证据合同，不是战斗结算器；它为 v420 checker 和未来 UI/素材设计提供边界。
