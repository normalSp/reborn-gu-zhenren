# v4.2.0-a1 AutoTheaterLite 凡阶映射设计门禁草案

状态：completed；D-420 已获用户批准，本门禁已冻结。
冻结名：`v420_a1_auto_theater_lite_mortal_mapping_gate_v1`。

## 目标

把现有凡阶战斗映射到 Auto-Theater Lite 的解释和证据结构，而不是迁移 runtime。

## 核心合同草案

`AutoTheaterLiteMortalMappingV1` 应包含：

| 字段 | 说明 | 权威 |
|---|---|---|
| `sourceActionRef` | 现有战斗 action / card / trace 的引用 | existing runtime |
| `preparationCue` | 行动前准备、站位、意图或资源提示 | mapping/report-only |
| `resolutionFactRef` | 已由 Combat Core / WorldCore 产生的事实结果 | Combat Core / WorldCore |
| `liteTranscript` | 玩家可读的短文本战斗过程 | expression only |
| `combatLedgerLite` | lane 化证据：fact/support/defense/failure/candidate/rejected | report-only |
| `expressionAuthority` | UI/DeepSeek/动效可表达范围 | v4.1 contract |
| `forbiddenImplications` | 不得暗示的奖励、死亡、地点破坏、隐藏事实等 | lore/governance |

## 映射样本族

| 样本族 | Lite 映射重点 | 禁止暗示 |
|---|---|---|
| 月光蛊攻击 | 攻击意图、距离/目标、已确定结果、失败原因 | 不新增伤害、命中或杀伤结论 |
| 白玉防御 | 防御 lane、抵挡/减伤原因、代价提示 | 不写成无敌或额外奖励 |
| 酒虫支援 | support lane、真元/准备辅助、局限性 | 不写成普通攻击或伤害来源 |
| 低阶兽战 | 威胁、位置、失败代价、撤退/准备原因 | 不新增掉落、兽尸奖励、正式地点 |
| 行动失败 | 缺真元、距离、目标、状态、风险、门禁原因 | 不把失败变成剧情惩罚或奖励 |

## Combat Ledger Lite lane

建议 lane：

- `fact`：Combat Core / WorldCore 已确定事实。
- `preparation`：准备、姿态、蓄势、目标选择。
- `support`：支援、辅助、资源调度。
- `defense`：防御、抵挡、规避、承受。
- `failure_reason`：失败或未生效原因。
- `candidate`：可考虑但未裁决候选。
- `rejected`：被门禁拒绝的表达或行动。
- `needs_user_decision`：需要玩家/用户决策，不自动裁决。

## 表达权限

- UI 可以显示 ledger lane、短原因、候选/拒绝/事实标签。
- 动效可以表达已确定事实和候选态势，但不能把候选画成事实。
- DeepSeek 未来只能基于 WorldCore/Combat Core 输出表达文本或候选意图，不得裁决。
- checker/report 可以存 redacted evidence 和 source refs，不得存 hidden/private body。

## 移动端与 reduced-motion 验收

- 单条 Lite transcript 应短句优先，不堆长段叙事。
- ledger lane 必须能折叠或摘要，不遮挡行动按钮。
- reduced-motion 下必须通过文本、图标、色阶和顺序理解战斗。
- 不能依赖动画节奏才能知道结果。

## 明确不做

- 不迁移凡阶 runtime。
- 不改变战斗数值。
- 不写 save。
- 不调用 live DeepSeek。
- 不做 theater UI。
- 不生成素材。
- 不使用真实原著战役或 hidden-adjacent。

## a1 通过标准

- 能用一套字段解释现有凡阶 action 如何映射到 Lite ledger。
- 能说明每个样本族的 forbidden implications。
- 能给 b1 checker 提供结构化样本方向。
- 玩家不会误以为新战斗模式已经上线。

## 结论

a1 通过。`AutoTheaterLiteMortalMappingV1` 仅作为 report-only 映射合同和 checker 输入，不写 runtime、UI、store、save、prompt，不改变现有凡阶战斗操作方式，也不开放 Auto-Theater Lite runtime。
