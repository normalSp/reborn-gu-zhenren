# v1.4.0 总体开发大纲

状态：D-140 已批准；a1 设计门禁已建立，等待 D-141 审批
日期：2026-05-21

## 目标

把 v1.1、v1.2、v1.3 的三条地基揉成一个南疆早期低阶区域样板：

- 玩家知道自己处于什么类型的低阶区域。
- 区域有可见风险、补给压力、身份检查和社会反应。
- 玩家能做安全的下一步，而不是被一句“去南疆”悬空。
- DeepSeek 只增强叙事和候选，不写正式区域/经济/社会结论。

## 不是目标

v1.4 不负责：

- 完整南疆地图。
- 完整商家城或商家城核心玩法。
- 正式城市进入/定居/身份认证。
- 正式阵营或势力身份。
- 正式交易系统。
- 正式 NPC 生死与剧情大定局。
- 蛊仙、宝黄天、尊者、仙蛊或高阶传承。

## 权威原则

| 事项 | 权威 |
|---|---|
| 正式地点/区域进入、区域状态、奖励、NPC 生死 | 本地 engine/store，且需用户批准 |
| 区域氛围、风险、候选事件、传闻、路人反应 | DeepSeek 可叙事，不可结算 |
| 低阶区域事实与原著边界 | MiroFish/知识库 candidate，经 intake 后转写 |
| 玩家下一步理解 | UI + Player Advocate gate |
| 长线漂移 | deterministic soak 优先；live probe 到 rc 前再确认 |

## 建议小版本结构

1. `a0`：启动审查、范围冻结、Skill sync audit、Git 分支和文档入口。
2. `a1`：南疆低阶区域样板 save-format / authority / scope 设计门禁。专家团建议 b1 保持 v24、无新区域持久字段、projection-first。
3. `a2`：MiroFish topic slice、知识库、测试矩阵和 hidden boundary 门禁。
4. `b1`：低阶区域 projection-first 第一刀。
5. `b2`：route/location + survival pressure 组合可读性。
6. `b3`：商队/散修/城外缘接触窗口。
7. `b4`：区域事件模板与风险前置条件。
8. `process-1`：反刷、旧档兼容、回滚和测试矩阵复核。
9. `process-2`：长线区域漂移与知识库复核。
10. `rc`：质量收束、Player Advocate、必要时 live probe。

## 成功标准

- 玩家能看懂区域类型、限制、风险和下一步。
- 没有未批准 save field 或 save-format bump。
- 没有正式南疆/商家城/地点/阵营/奖励/NPC 生死结论。
- MiroFish 只作为 topic-slice candidate/rule/test 后勤。
- 新增高风险样本全部进入测试矩阵演进流程。
