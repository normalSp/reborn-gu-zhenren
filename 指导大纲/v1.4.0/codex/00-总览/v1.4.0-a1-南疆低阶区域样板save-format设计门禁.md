## 专家团结论

本阶段建议：v1.4-b1 不 bump `SAVE_FORMAT_VERSION = 25`，不新增持久区域聚合对象，先做 projection-first 第一刀。

理由：

- v1.1 已有 `routeLocationState`，足够表达路线/地点/区域的最小正式状态。
- v1.2 已有 `survivalEconomyState`，足够表达补给/食料/交易窗口的最小压力 ledger。
- v1.3 已有社会压力 projection 结构，足够表达 NPC/势力反应的只读证据。
- v1.4-b1 的真实目标是把三条地基揉成“玩家看得懂的区域样板”，不是立刻写一个新区域存档事实。
- 现在直接新增 `regionSampleState` 容易把“样板投影”误升格成正式南疆区域 authority。

## 两条路线评估

| 路线 | 做法 | 收益 | 风险 | 专家团建议 |
|---|---|---|---|---|
| A：保持 v24 projection-first | 不 bump save；新增纯 engine projection helper；UI 只读展示区域类型、风险、补给、社会压力、下一步候选 | 最稳；旧档零迁移；不误开完整区域；能快速验证玩家理解 | 不保存“区域样板历史”；后续若要长期区域状态需再开 v25 | 推荐 b1 使用 |
| B：v25 + 单一 `regionalSampleState` | 新增持久聚合，记录区域样板 id、阶段、压力、窗口、最近事件 | 后续长线区域状态更直接；可为 v2.0 区域活世界铺路 | 需要 schema/migration/defaults/tests；容易让样板变正式地点/阵营事实；会扩大旧档兼容面 | 暂缓，等 b1/b2 证明字段必要 |
| C：把区域事实塞进既有字段 | 复用 `routeLocationState` 或 `livingWorldState` 写更多区域信息 | 省一次 save bump | 语义污染；后续迁移困难；容易让旧字段承担超额 authority | 不推荐 |

## b1 建议 authority 模型

新增只读 projection 类型或内部返回对象，不进入 save：

```ts
type V140RegionSampleProjection = {
  scopeId: 'southern_border_low_rank_outer_sample';
  savePolicy: 'no_new_persistence_v24';
  regionPosture: 'route_outer_edge' | 'caravan_contact_window' | 'rogue_settlement_hint' | 'city_outer_threshold';
  routeSignals: string[];
  survivalSignals: string[];
  socialSignals: string[];
  marketSignals: string[];
  nextStepCandidates: string[];
  forbiddenWrites: string[];
};
```

该结构只允许从既有本地事实推导：

- `routeLocationState`
- `survivalEconomyState`
- `livingWorldState.knownFacts`
- `livingWorldState.npcMemories`
- `livingWorldState.factionPressure`
- `livingWorldState.actionConsequences`
- scene-local action ledger

## 禁止写入

b1 不得写入：

- `SAVE_FORMAT_VERSION = 25`
- `regionSampleState`
- `currentRegion`
- `route_entered`
- 正式地点 unlock
- 正式商队身份
- 正式散修身份
- 正式城市进入
- 正式势力通行/封锁/招揽结论
- 交易成功、价格表、库存、奖励
- NPC 生死、捕获、背叛
- DeepSeek 结算出的地点/奖励/身份/生死/canon 事实
