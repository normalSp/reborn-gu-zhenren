# v1.4.0-b2 route/location + survival pressure 组合可读性

状态：已完成
日期：2026-05-21

## 目标

让区域样板同时读到路线范围与生存压力，让玩家知道“我为什么需要准备补给、身份、遮掩和交易窗口”，而不是只看到抽象区域名。

## 落地方式

`buildV140RegionSampleProjection()` 读取：

- `buildV110RouteLocationOverview()`：路线、地点范围、区域范围、drift checkpoint。
- `buildV120LowRankSurvivalEconomyProjection()`：路线补给、蛊虫喂养、炼养用准备、交易窗口、灰色渠道边界、反刷。

UI 中 `证据组` 展示：

- 路线证据。
- 生存压力。
- 区域边界。

## 边界

- 不写新的生存经济字段。
- 不把 survival pressure 说成真实库存/价格/消耗。
- 不把商队窗口说成正式交易。
- 不发材料、元石、蛊虫、蛊方或路线奖励。

## 验证

- focused unit 覆盖 materialBag 可见但不结算。
- focused e2e 覆盖市场窗口 -> 区域样板的 UI 路径。
- `v140-region-boundaries` 明示不写交易、价格、库存、奖励。

## 结论

b2 作为 b1 helper 的组合可读性部分完成；无需新增单独 store/action。
