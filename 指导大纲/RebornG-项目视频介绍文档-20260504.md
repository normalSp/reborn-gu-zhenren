## 一、项目概况

### 这个项目是什么

蛊真人世界·人生重来模拟器（RebornG）是一款基于《蛊真人》原著世界观、由 AI 大模型驱动的深度文字 RPG 游戏。玩家可以从一个普通凡人开始，在五域两天的广阔世界中炼蛊修行、建立势力、改变命运——甚至挑战宿命本身。

### 技术定位

纯前端浏览器运行，零后端，AI 引擎为 DeepSeek V4 Pro（128K 上下文窗口），技术栈为 React 18 + TypeScript 5 + Vite 8 + Zustand 5 + Tailwind CSS + Framer Motion + GSAP。采用 SIM（模拟）核心 + VN（视觉小说）叙事层 + RPG 成长系统的混合架构。

游戏的每一次叙事、每一个 NPC 的对话、每一场战斗的描写，都由 AI 实时生成——但受到四层世界观一致性防线的约束，确保不偏离原著设定。

### 开发历程

单人开发（PlumSnow 11411），采用 Vibe Coding 模式，约 5 天高强度开发（约 18 小时编码时间），完成 P1（基础设施）+ P2（核心系统）两个大阶段，进入 P3 开发规划阶段。全项目约 182 个源文件，包含 26 个 Zustand 状态切片、38 个 UI 组件、13 个世界观数据文件、9 个测试文件（158 用例全通过）。

## 二、世界观与叙事系统

### 五域22章完整叙事架构

游戏覆盖原著五大域：南疆、北原、东海、西漠、中洲。每个域拥有独立的章节弧光系统：

- **南疆**：13 章，从青茅山开窍之日到逆流河不可能突破，覆盖原著核心主线
- **北原**：5 章，从草原少年到巨阳前夜，黄金血脉与部落争霸线
- **东海**：4 章，从海岛散修到散修联盟，深海机缘线
- **西漠**：4 章，从沙漠行者到蓝海传说，绿洲秘境与人祖遗迹线
- **中洲**：4 章，从宗门弟子到宿命线索，正道修行与天庭初窥线

每个章节包含：触发条件（expression 表达式引擎）、关键 NPC 列表、关键势力、场景约束（mustHappen/mustNotHappen）、目标体系（里程碑/经济/境界门槛）、跨域暗示（五域感知文本+干预注释）、以及 chapterPriceMultiplier（价格缩放系数）。

### 8大核心事件 · L0-L3 四层涟漪系统

这是游戏的叙事创新核心。原著中 8 个关键事件——三王山事件、王庭福地事件、义天山事件、春秋蝉波动、巨阳传承、逆流河上溯、宿命大战、天庭终战——被设计为四层涟漪传播：

- **L0**：事件发生域的直接冲击（叙事文本变化）
- **L1**：邻域的传闻/流言（NPC 对话话题注入）
- **L2**：远方域的推测/预警（经济影响、势力态度变化）
- **L3**：全局世界观 flag 改变（如 `fate_destroyed`、`tianting_fallen`、`era_without_heaven`）

这意味着一个域发生的大事，会像石子入水一样在其他四个域激起涟漪，玩家在不同域会感受到同一事件的不同层次影响。系统定义了 12 个全局标志（flag），控制世界状态的永久性改变。

玩家可以选择 9 个叙事弧线进入游戏，并非只能从第一章开始：

### 32+ 域内名场面 · 跨域感知

每个域的事件都包含向其他四个域的跨域提及（crossDomainMentions），让玩家在东海也能听到南疆三王山之战的传闻，在北海也能感知天庭坠落的影响。事件各有场景约束（sceneConstraint），确保"不应该发生的事不会发生"。

### 202 个 NPC · 跨域管理系统

游戏中收录了 202 个具名 NPC，包括原著角色（方源、白凝冰、古月方正、铁若男等）和原创角色。每个 NPC 拥有：

- 域标签（domainTags）与动态称号
- 完整人格模型
- 跨域亲密度衰减规则（离开域后关系逐渐冷却）
- NPC 跨域迁移系统（部分 NPC 会跨越域边界移动）
- 六话题对话树系统（情报/交易/任务/关系/闲聊/离去）

### 因果追踪 · 蝴蝶效应

游戏内置因果追踪系统，记录玩家的每一个重大选择产生的蝴蝶效应。时间线偏离度从 0（完全贴合原著）到 100（面目全非），玩家可以看到自己的命运如何一步步偏离或回归原著轨迹。

## 三、核心游戏系统

### 角色养成系统

**四大基础属性**：资质、根骨、心智、气运——共同决定角色的上限和发展方向。

**九转境界体系**：从凡人到一转蛊师，最终攀升至九转蛊尊。每转包含初阶、中阶、高阶、巅峰四个小境界。突破需要满足境界门槛，并有完整的突破动画效果。

**道心四维**：杀伐、仁善、算计、野心——四个维度影响角色的行为倾向和 NPC 对你的态度。

**天赋系统**：角色创建时可选择天赋，每项天赋提供增益和代价的双向修正。

**26 条成就**：覆盖成长、战斗、经济、探索等多个维度，跨存档独立持久化，解锁时有 Toast 弹窗动画。

### 蛊虫系统

**85 条蛊虫**：全面覆盖原著中出现的核心蛊虫——春秋蝉、酒虫、月光蛊、玉皮蛊、冰刃蛊、血颅蛊、天元宝莲、希望蛊等。每条蛊虫拥有：

- 品阶（1-5转凡蛊 + 6转仙蛊）
- 所属流派（金木水火土 + 特殊流派）
- 喂养需求三元组（食物类型 + 频率 + 消耗）
- 饥饿状态机（鼎盛→饱足→饥饿→濒饿→垂死→死亡）
- 空窍容量限制（1转最多3只，5转最多15只；6转仙蛊无容量限制）

**本命蛊系统**：玩家可绑定一只蛊虫为本命蛊，获得额外加成，但本命蛊死亡会对宿主造成严重反噬（30回合冷却）。

**炼蛊系统**：包含完整的炼制引擎，从配方到炼制到精炼，含失败率（20%-60%，炼道天赋可修正）和炼制耗时。

**蛊虫进化**：进阶叠加层动画，蛊虫可通过特殊事件或炼制进化到更高品阶。

### 杀招系统

18 个预设杀招模板，支持玩家自创杀招。杀招拥有：

- 类型分类（攻击/防御/辅助/特殊）
- 战斗属性（伤害/消耗/冷却）
- 熟练度体系
- 进化路线
- 与蛊虫的联动（组合→强化→加速→反噬）

### 战斗系统

**回合制决斗引擎**（约 200 行核心代码）：完整的战斗状态机（init → player_turn → enemy_turn → ended → resolution），包含 7 级境界修正和 15 流派克制矩阵。

**双轨战斗体验**：

- 决斗轨道（CombatOverlay）：全屏回合制，HP/ATK/DEF 数值驱动
- 叙事轨道（NarrativeCombatPanel）：AI 生成战斗叙事，重大战役使用

**战斗 UI**：全屏 CombatOverlay（Framer Motion 动画），轻型 BattleOverlay 信息弹出，杀招动画，暴击/受击/爆炸音效。

### 经济系统

**双层货币体系**：

- 元石（凡间货币）：日常交易、商店买卖、债务系统（日利率 5%）
- 仙元石（蛊仙货币）：六转以上使用，一颗仙元等价于无限元石

**经济闭环**：

- 商家城：南疆商业中心，47 条 shop-items，含演武场功能
- 宝黄天：蛊仙专属交易平台（拍卖引擎已实现）
- 元石流水账：完整收支记录与统计
- Gu 市场价格公式：品阶 × 稀有度乘数（common 1x 到 epic 5x，legendary 不可购买）

**炼蛊经济**：完整的炼制成本/收益闭环，含失败率修正后的预期净收益计算。

### 势力与社交系统

**五域势力网络**：每个域拥有 keyFactions 定义，包含势力声望体系（-100 死敌 → +100 崇拜）。

**NPC 关系矩阵**：亲密度/信任度独立追踪，六话题对话树（情报/交易/任务/关系/闲聊/离去），NPC 跨域迁移与亲密度衰减。

### 空窍与仙窍系统

**空窍（凡窍）**：1-5 转使用，SVG 五层同心圆可视化渲染，元海颜色随资质变化（青铜→赤铁→白银→黄金→紫晶），窍壁状态机（坚实→潮汐初现→潮汐涌动→壁薄如纸）。

**仙窍（福地/洞天）**：六转以上使用，拥有面积、光阴流速、资源节点数、道痕密度等属性。月度仙元石产出由公式计算：面积 × 时间流速 × 资源节点数 × 0.05。

**道痕系统**（DaoMarkPanel）：按流派追踪道痕积累，六转以上核心战力指标，与战斗引擎深度联动。

### 遭遇与奇遇

随机遭遇系统：22 条遭遇模板，7 步过滤引擎（基于域/境界/已触发 Set/冷却计时），含近期遭遇去重机制。

### 起源解锁系统

5 条可选出身起源，通过成就系统解锁。不同起源对应不同的起始资源、境界和难度。

## 四、AI 引擎架构

### PIPE 管线模型

游戏的核心创新之一。AI 交互不是简单的"输入→输出"，而是五阶段管线：

1. **BUILDING_CONTEXT**：组装三层混合记忆（System Prompt 静态世界观 + 结构化玩家状态 JSON + 关键事件索引/滚动摘要）
2. **FETCHING**：向 DeepSeek API 发送请求
3. **VALIDATING_L4**：第四层验证——双轮校验（格式+内容）
4. **VALIDATING_L3**：第三层验证——语义规则引擎
5. **RESOLVED**：更新游戏状态，返回叙事文本

### 四层世界观一致性防线

这是确保 AI 不偏离《蛊真人》设定的核心机制：

- **L1 · Few-shot Prompt**：在 System Prompt 中植入大量原著场景示例，覆盖率约 88%
- **L2 · 双轮校验**：第一轮校验 JSON 格式，第二轮校验叙事内容是否符合世界观规则
- **L3 · 语义规则引擎**：硬性约束——绝不允许的词汇（如"能量"→应为"真元"、"魔法"→应为"杀招"）和必遵循的叙事规则
- **L4 · 12 条金丝雀断言**：12 条关键规则（如"九转灭世蛊永不可炼制"、"春秋蝉不可复制"），AI 输出必须通过这些断言才能通过

实测压力测试：20 个场景，防崩溃率约 92%。

### 三层混合记忆

- **System Prompt**（静态）：完整世界观设定，永久注入每一轮对话
- **结构化玩家状态 JSON**：角色属性、当前域、章节进度、关键选择的压缩摘要
- **关键事件索引/滚动摘要**：过去 20 轮对话的精简索引

实测 128K Token 窗口下，AI 能 100% 记住 250+ 轮对话中的所有重要事件。

### DeepSeek API 优化

移除了 4 处缓存破坏点，实现强缓存：System Prompt 后端缓存（24h TTL），温度→0.1 纯确定性输出，结构化玩家状态附带 120s Cache-Control。Cache 命中率约 65-80%，显著降低了 API 调用成本。

### 双通道输出协议

AI 的每次回复都是结构化的：

- **narrative** 通道：玩家看到的叙事文本
- **state_update** 通道：JSON 格式的状态变更指令（属性变化/事件触发/成就解锁等），直接更新 Zustand store

## 七、创新亮点

1. **AI 驱动的 TRPG 体验**：不是预设文本，而是每一轮都由 AI 根据玩家状态实时生成——但受四层防线约束，不偏离原著设定

2. **四层涟漪事件系统**：一个域发生的大事会影响所有其他域，12 个全局 flag 控制世界状态永久改变——这是传统文字游戏难以做到的

3. **时间线起点系统**：15 个不同起点，从凡人到蛊仙，不同时代不同难度——类似《骑马与砍杀》的出身选择，但集成在 AI 叙事框架中

4. **三层混合记忆架构**：在 128K Token 窗口下实现 100% 长程记忆持久性，AI 不会"遗忘"玩家的关键选择

5. **双通道输出协议**：AI 一次回复同时输出叙事文本和结构化状态更新，实现了叙事自由度和系统严谨性的统一

6. **PIPE 管线模型**：五阶段 AI 交互管线，每个阶段有独立的验证和回退机制，保证稳定性

7. **纯前端零后端**：完全在浏览器中运行，无需服务器，一键访问

8. **深度尊重原著**：85 条蛊虫、202 个 NPC、五域世界观、48 流派体系、九转境界——全部严格遵循《蛊真人》原著设定

9. **系统接线完整性野心**：26 个 Zustand 切片管理游戏的所有维度——从因果追踪到元石流水账，从流派晋升到蝴蝶效应

10. **单人 Vibe Coding 极速开发**：5 天约 18 小时编码，完成可玩的 AI 驱动深度 RPG——展示了 AI 辅助开发的潜力
