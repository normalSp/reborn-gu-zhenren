# v4.0.0 Auto-Theater Combat 美术与交互总纲

状态：design guide draft；需 D-400 批准后作为 v4.0-a1 的上游设计基线。
日期：2026-05-26。

## 定位

Auto-Theater Combat 应成为 RebornG 后续战斗表现的核心招牌。它不是纯自走棋、不是普通棋盘扩容、也不是大粒子演出堆叠，而是：

`水墨战争沙盘 -> 道痕规则可视化 -> 杀招栈舞台 -> Combat Ledger 复盘 -> DeepSeek 表达`

本文件只定义美术、交互、动效和验收语法，不授权 theater UI implementation、runtime 接入、素材生成、存档字段、DeepSeek 权限、MiroFish intake 或高阶战斗结算。

## 美术原则

### 第一原则：状态解释优先

每个视觉元素都必须回答至少一个问题：

- 现在谁占优。
- 哪个区域危险。
- 哪个杀招在蓄势。
- 哪个事实已由 WorldCore 裁决。
- 哪个结果只是候选。
- 为什么某个候选被拒绝。
- 玩家下一步能影响什么。

不能只为了“帅”添加效果。任何命中、伤害、NPC 生死、奖励、地点毁灭、环境破坏、仙蛊屋控制权都必须来自 WorldCore / combat core 事实，不能由 UI、动效或 DeepSeek 私自暗示。

### 视觉语言

| 语义 | 主视觉 | 用法 |
|---|---|---|
| 世界底色 | 暗墨、灰黑、宣纸颗粒 | 战场底、面板底、远景压力 |
| 规则/资源 | 金色道痕、细线、节点 | 结算路径、资源消耗、阵眼、证据链 |
| 危险/禁忌 | 血红、裂纹、短促闪断 | 反噬、禁忌、失败、不可逆风险 |
| 生机/护持 | 玉青、浅绿光晕 | 治疗、防护、撤退保护、队友支援 |
| 阵法/领域 | 环形阵纹、符节、网格投影 | 覆盖范围、领域压制、阵法边界 |
| 天道/宿命 | 冷白、淡金、远景压线 | pressure / constraint / risk，不做结局裁决 |

避免通用科幻 HUD、霓虹赛博、营销卡片、装饰渐变球、过度耀斑、无法读清文字的粒子层。

## 核心界面结构

### Battle Theater

中央主舞台，不再是单纯 5x3 棋盘。高阶形态应像一张可读的战争剖面图：

- 空域：飞行、上升、俯冲、云层、空中封锁。
- 地面：站位、遮蔽、地形、建筑、阵位。
- 地下：潜行、地脉、陷阱、遁地、暗流。
- 水域：河流、湖泊、湿地、水道压力。
- 洞天/福地：独立空间、入口、边界、压制规则。
- 阵法/领域：覆盖、阵眼、破绽、压制、排斥。
- 仙蛊屋：位置、完整度、核心阵、能量、控制权候选。

凡阶/中阶只显示局部镜头，保留棋盘和行动卡；蛊仙以上才进入 full theater。v4.0 只设计 envelope 和视觉语法，不实现 UI。

### Preparation Rail

战前/战中准备区，表达玩家编排能力：

- 蛊虫与杀招组合。
- 队友职责。
- 资源预算。
- 底牌暴露策略。
- 撤退阈值。
- 风险承受度。
- 战术倾向：试探、压制、牵制、拖延、斩首、撤退。

这一区域吸收自走棋的“准备和构筑爽点”，但不把游戏变成纯摆棋子自动打。

### Killer-Move Stack

杀招栈是高阶战斗的第二主舞台，显示：

- 准备。
- 蓄势。
- 锁定。
- 释放。
- 反制。
- 打断。
- 反噬。
- 失败降级。
- 连锁。

每个栈节点必须有来源：GuExpressionSpec、KillerMoveExpressionSpec、WorldCore resolution 或 report-only envelope。DeepSeek 可写旁白，不能写 cost/result。

### Combat Ledger

底部或侧边复盘账本，必须清晰分四类：

- fact：WorldCore 已裁决事实。
- candidate：可考虑但未裁决候选。
- rejected：被规则拒绝的候选。
- needs_user_decision：必须停下来让用户审批的内容。

Ledger 是防止 AI 乱写和玩家误解的核心界面。它应记录资源消耗、反噬、失败降级、底牌暴露、撤退阈值、环境候选、势力关注候选、因果债候选和拒绝原因。

### Macro Pressure Strip

顶部或远景压力条，只显示 HeavenWill / Fate / L5 的：

- pressure。
- constraint。
- bias。
- exclusion。
- risk。

不得显示“天道判你失败”“宿命杀死某 NPC”“正史强制改写”等结局裁决。

## 动效分工

### GSAP

只用于时间轴强演出：

- 开战 / 破阵 / 入阵。
- 杀招蓄势、释放、被打断、反噬。
- 仙蛊屋核心阵亮起或崩裂候选。
- 领域压制瞬间。
- 重伤、失败降级、撤退窗口。
- 章节级 HeavenWill / Fate 压力转场。

GSAP 只控制独立 effect layer，必须 `pointer-events: none`，不能控制 Motion 管辖的布局、按钮、文字或滚动容器。

### Motion / Framer Motion

用于 UI 状态和交互：

- 面板切换。
- theater layer 展开/折叠。
- 栈节点进入/重排。
- ledger 条目进入。
- hover/press/selection。
- Tooltip。
- 移动端布局切换。
- reduced-motion 的静态替代。

### Reduced Motion

低动效模式必须保留：

- 战场层级标签。
- 杀招栈顺序。
- Ledger fact/candidate/rejected/needs_user_decision。
- 资源、风险、拒绝原因。
- 可点击按钮和下一步提示。

低动效可以去掉大幅位移、闪烁、震屏、连续镜头推进，但不能减少信息。

## 素材分层

| 层级 | 内容 | 状态 |
|---|---|---|
| L0 语法资产 | 线、点、道痕、阵纹、标签、风险色 | 可作为设计 token |
| L1 泛用战场 | 山地、林地、水域、洞天、阵法底纹 | 只能 generic/candidate |
| L2 蛊虫/杀招 motif | 月光、白玉、血、雷、水、金等路径效果 | 必须绑定登记规格 |
| L3 场景关键图 | 青茅、商家城、义天山等构图候选 | 需来源和边界 |
| L4 原著关键角色/事件 | 方源、尊者、正史战役、隐藏因果 | MiroFish blocking |
| L5 宿命/天道 | 宏观压力符号、时代锚点 | 只做 pressure/risk |

v4.0 不新增素材生成任务，不把现有候选图晋升为 theater runtime 资产。未来若要生成高阶 battle theater 资产，必须先有 art request、source boundary、intake review、asset manifest 和 Playwright visual acceptance。

## 后续路线

### v4.0

- 冻结 Auto-Theater Combat 的视觉语法。
- 冻结 Auto-Theater Lite 与 Full 的阶段边界。
- 建立 report-only checker 样本，覆盖 visual overclaim、candidate/fact 混淆、DeepSeek 裁决越权。
- 写 Player Advocate 记录时验证玩家是否能读懂 fact/candidate/rejected/pressure/ledger。

### v4.1-v4.3 候选

- 先做静态 wireframe / text-first prototype。
- 只用 synthetic/generic 数据。
- 不接 save，不接 live DeepSeek，不接真实原著锚点。
- 用 Playwright 截图检查桌面/移动/reduced-motion。

### v4.4+ 候选

- 若前期验证稳定，再申请 theater UI first cut。
- 若触及真实地点、原著角色、正史战役、隐藏事实，升级 MiroFish blocking。
- 若需要 PixiJS / React Flow / Canvas / Three.js，先做 license/SBOM/架构适配报告，再单独申请 PoC。

## 防偏机制

必须阻断以下偏移：

- 把 Auto-Theater 做成普通自走棋。
- 把 Auto-Theater 做成不可交互的播片。
- 把高阶战斗继续压进低阶棋盘。
- UI 私算伤害、奖励、生死、环境破坏。
- DeepSeek 旁白被当成结算事实。
- 美术暗示已获得仙蛊、杀招传承、正式地点、NPC 生死或正史改变。
- HeavenWill/Fate 压力条变成万能失败按钮。
- 为了视觉效果绕过 reduced-motion、移动端可读性或 ledger 证据链。

## Playwright 验收计划

未来进入 UI 原型后，至少验证：

- 桌面 1440x900：theater、preparation、stack、ledger、pressure strip 均可读。
- 移动端 390x844：核心信息不重叠，ledger 可滚动，按钮不被遮挡。
- reduced-motion：无大幅位移/闪烁，仍能读清事实和候选。
- candidate/fact/rejected/needs_user_decision 视觉状态差异明确。
- DeepSeek 文案不被 UI 显示成 WorldCore fact。
- 任何环境破坏、NPC 生死、奖励、地点变化都没有在无证据时被视觉暗示。

v4.0 当前只写此验收计划，不执行 browser/runtime UI 测试。
