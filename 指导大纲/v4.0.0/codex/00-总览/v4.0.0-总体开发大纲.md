# v4.0.0 总体开发大纲

状态：startup pending user decision。

## 主题

`Auto-Theater Combat 与 HeavenWill/Fate Pressure 高阶世界表达模型`

## 总目标

v4.0 把高阶世界拆成两个 report-only / design-gate-first 系统：

1. `AutoTheaterCombatEnvelope`：描述自走棋式准备、WorldCore 自动结算、battle theater、killer-move stack、Combat Ledger、领域/阵法、仙蛊屋状态、环境影响和 WorldCore 证据链。
2. `AutoTheaterArtInteractionGuide`：描述水墨战争沙盘、道痕规则可视化、杀招栈舞台、Combat Ledger 信息架构、GSAP/Motion 分工、素材分层和 Playwright 截图验收。
3. `MacroPressureEnvelope`：描述 Fate state、HeavenWill pressure、era anchor、causality debt、L5 macro director 和 lore visibility 边界。

二者都必须保持“候选/压力/约束不是事实”。正式事实仍由 WorldCore、combat engine、canon gate、save gate 和用户决策裁决。

长期承接基线是 `指导大纲/长期路线/v4.0-v5.0-AutoTheater到高阶世界Runtime总体大纲.md` 与 `指导大纲/长期路线/v4.0-v12.0-RebornG终局形态总体大纲.md`。v4.0 完成后，后续应按 v4.1-v4.9 逐步硬化 Auto-Theater contract、凡阶 Lite 映射、高阶 theater、杀招栈、Combat Ledger、HeavenWill/Fate/L5 宏观压力、MiroFish/lore 准入和 v5.0 go/no-go，而不是从 v4.0 直接跳到完整高阶 runtime。更远期以 v12 final-candidate planning horizon 约束方向，不把 v5 第一刀误写成项目终局。

## 分层职责

| 层 | 职责 | v4.0 默认状态 |
|---|---|---|
| WorldCore | 最终事实、奖励、地点、阵营、生死、save 写入、环境破坏裁决 | 保持最终权威 |
| Combat Core | 自走棋式准备、自动结算、theater、stack、ledger 的未来结算模型 | 只做 envelope / schema / checker |
| AgentProposal | NPC 意图、敌方战术意图、势力压力、表达候选 | 不扩 persistent state |
| DeepSeek | 未来叙事表达与意图候选 | 启动阶段不调用；未来不能裁决命中/伤害/生死/奖励/宿命 |
| MiroFish | 原著 source-pointer 候选 | 启动阶段 not_needed |
| UI/Motion | 表达已裁决事实、候选边界、Combat Ledger 复盘和 Auto-Theater 视觉语法 | 不新增高阶 theater runtime、自走棋 runtime 或素材生成 |

## 完成口径

v4.0 完成时可以说：

- 项目拥有高阶战斗与宏观压力的第一版工程门禁。
- 项目明确纯自走棋不作为高阶主路线，`Auto-Theater Combat` 作为后续高阶战斗设计基线。
- 项目明确 Auto-Theater Combat 是 RebornG 后续战斗表现的核心招牌，并拥有第一版美术与交互语法。
- 项目拥有 `v4.0-v5.0` 长期路线基线，知道 v4.1-v4.9 各自负责 contract、Lite、高阶 theater、杀招栈、宏观压力、MiroFish/lore 准入、runtime 准入与 v5.0 安全收束。
- 项目拥有 `v4.0-v12.0` 终局路线基线，知道 v5 只是高阶第一刀，v6-v9 继续扩多区域、agent society、经济和高阶战斗，v10-v12 才进入关键原著时代、任意身份和终局质量收束。
- 项目明确凡阶战斗未来只做 `Auto-Theater Lite` 底层预备，不在 v4.0 直接迁移 runtime。
- 项目知道哪些高阶能力可进入后续设计，哪些必须继续 future gate。
- 玩家理解、lore 边界、WorldCore 权威、DeepSeek/MiroFish/external framework/save 边界有可追溯证据。

v4.0 完成时不能说：

- 已开放完整蛊仙。
- 已开放高阶战斗 runtime。
- 已开放凡阶战斗 runtime 迁移。
- 已开放纯自走棋 runtime。
- 已实现 theater UI 或新增 Auto-Theater 素材。
- 已开放天道/宿命裁决。
- 已开放原著关键人物 agent。
- 已吸收全书知识库或 MiroFish 原著事实。
