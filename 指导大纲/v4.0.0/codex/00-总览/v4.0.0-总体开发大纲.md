# v4.0.0 总体开发大纲

状态：startup pending user decision。

## 主题

`高阶战斗 theater 与 HeavenWill/Fate 宏观压力双预备`

## 总目标

v4.0 把高阶世界拆成两个 report-only / design-gate-first 系统：

1. `HighRankCombatTheaterEnvelope`：描述高阶战斗的空间、杀招栈、领域/阵法、仙蛊屋状态、环境影响和 WorldCore 证据链。
2. `MacroPressureEnvelope`：描述 Fate state、HeavenWill pressure、era anchor、causality debt、L5 macro director 和 lore visibility 边界。

二者都必须保持“候选/压力/约束不是事实”。正式事实仍由 WorldCore、combat engine、canon gate、save gate 和用户决策裁决。

## 分层职责

| 层 | 职责 | v4.0 默认状态 |
|---|---|---|
| WorldCore | 最终事实、奖励、地点、阵营、生死、save 写入、环境破坏裁决 | 保持最终权威 |
| Combat Core | 未来高阶战斗结算模型 | 只做 envelope / schema / checker |
| AgentProposal | NPC 意图、势力压力、表达候选 | 不扩 persistent state |
| DeepSeek | 未来叙事表达候选 | 启动阶段不调用 |
| MiroFish | 原著 source-pointer 候选 | 启动阶段 not_needed |
| UI/Motion | 表达已裁决事实或候选边界 | 不新增高阶 theater runtime |

## 完成口径

v4.0 完成时可以说：

- 项目拥有高阶战斗与宏观压力的第一版工程门禁。
- 项目知道哪些高阶能力可进入后续设计，哪些必须继续 future gate。
- 玩家理解、lore 边界、WorldCore 权威、DeepSeek/MiroFish/external framework/save 边界有可追溯证据。

v4.0 完成时不能说：

- 已开放完整蛊仙。
- 已开放高阶战斗 runtime。
- 已开放天道/宿命裁决。
- 已开放原著关键人物 agent。
- 已吸收全书知识库或 MiroFish 原著事实。
