## 升级为 blocking 的条件

以下任一条件出现时，必须停止并走 MiroFish 双仓 topic-slice：

- 命名 NPC。
- 真实原著势力或地点。
- 方源公开证据。
- hidden-adjacent material。
- L4/L5 fact anchor。
- HeavenWill/Fate 真实原著锚点。
- 尊者布局、时代大势、正史锚点的 formal lore conclusion。
- knowledge-index body。
- runtime canon。
- DeepSeek visible lore/RAG。

## 交付链

仍使用：

`RebornG request -> MiroFish export -> RebornG intake review -> rule/test/candidate material`

MiroFish 输出不得直接进入：

- runtime。
- UI。
- DeepSeek visible context。
- save。
- `src/canon`。
- knowledge-index body。
- prompt body archive。
