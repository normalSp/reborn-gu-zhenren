# v4.0.0-a1 Auto-Theater Combat 与杀招栈设计门禁草案

状态：draft；需 D-400 批准后进入正式 a1。

## 目标

为未来高阶战斗建立第一版工程语义，不实现 runtime。

本门禁采用 `Auto-Theater Combat`：吸收自走棋式准备、自动结算和复盘优点，但不采用纯自走棋作为高阶战斗主容器。棋盘保留为凡阶/中阶/局部战术镜头；凡阶只做 `Auto-Theater Lite` 设计预备，高阶主表达使用 battle theater、killer-move stack 和 Combat Ledger。

本门禁的表现层上游是 `v4.0.0-Auto-Theater-Combat美术与交互总纲.md`。a1 设计必须同时守住工程语义和视觉语法：水墨战争沙盘、道痕规则可视化、杀招栈舞台、Combat Ledger 复盘、DeepSeek 表达分层。

## 建议概念

### Auto-Theater Combat

Auto-Theater Combat 负责描述：

- 准备阶段：蛊虫、杀招、队友、资源、态势、底牌暴露策略、撤退阈值。
- 自动结算阶段：WorldCore / combat core 按本地规则产出事实、候选和拒绝原因。
- 态势表达阶段：battle theater 展示空间、领域、阵法、仙蛊屋、环境压力。
- 复盘阶段：Combat Ledger 记录事实、消耗、失败降级、候选、压力和后果。
- DeepSeek 表达阶段：只可生成旁白和意图候选，不能写结算。

### Auto-Theater Lite

Auto-Theater Lite 是凡阶/中阶战斗的未来底层预备：

- 保留现有棋盘/行动卡可读性。
- 逐步引入准备项、战术倾向、敌方意图候选、撤退阈值和战后复盘。
- 不在 v4.0 迁移凡阶 runtime。
- 不把早期教学战斗改成完全自动观看。

### Battle Theater

高阶战斗不再只依赖低阶棋盘或传统自走棋棋盘。battle theater 负责表达：

- 空域 / 地面 / 地下 / 水域 / 洞天福地等层级。
- 距离、遮蔽、地形压力、环境破坏候选。
- 领域、阵法、杀招覆盖、仙蛊屋位置。
- WorldCore 已裁决事实与未裁决候选的分离。

### Art / Interaction Grammar

表现层负责让玩家看懂战斗，不负责创造战斗事实：

- 暗墨底色承载战场压力。
- 金色道痕表达规则、资源和证据链。
- 血色表达禁忌、反噬和不可逆风险。
- 玉青表达护持、恢复和撤退保护。
- GSAP 只做独立 effect layer 的时间轴强演出。
- Motion 只做 UI 状态、节点重排、hover/press 和 reduced-motion 替代。
- Playwright 验收必须覆盖桌面、移动端和 reduced-motion。

### Killer-Move Stack

杀招栈负责表达：

- 准备、蓄势、锁定、释放、反制、打断、反噬。
- 资源成本、冷却、克制关系的证据来源。
- 多杀招连锁与失败降级。
- DeepSeek 只可表达旁白，不能写入栈结算。

### Immortal Gu House State

仙蛊屋不应作为普通装备处理。建议作为高阶系统状态候选：

- 结构完整度。
- 核心蛊阵状态。
- 能量/仙元消耗。
- 防护/移动/领域/压制能力。
- 所属与控制权必须由 WorldCore 裁决。

### Combat Ledger

Combat Ledger 负责表达：

- 每个阶段的 WorldCore 证据来源。
- 资源消耗、反噬、失败降级、暴露底牌和撤退阈值。
- 环境破坏候选、势力关注候选、因果债候选。
- DeepSeek 文本与本地裁决事实的对应关系。
- 候选被拒绝或升级用户决策的原因。

## 禁止项

- 不把 v4.0 做成纯自走棋。
- 不实现自走棋 runtime。
- 不迁移凡阶战斗 runtime。
- 不实现 theater UI。
- 不生成 Auto-Theater 素材。
- 不新增战斗结算数值。
- 不开放任何仙蛊、杀招、传承、仙蛊屋。
- 不让 UI/Motion 生成命中、伤害、环境破坏、NPC 生死或奖励。
- 不把 DeepSeek 战斗旁白当成事实结算。

## 验收样本方向

- 高阶战斗候选被正确标为 candidate。
- 自走棋式准备只进入 envelope，不写 runtime/save。
- 凡阶 Auto-Theater Lite 只进入设计 envelope，不迁移现有棋盘 runtime。
- 环境破坏候选缺 WorldCore 证据时被拒绝。
- NPC 生死、地点毁灭、奖励掉落被 blocking。
- 杀招栈不能让 DeepSeek 直接写 cost/result。
- 仙蛊屋状态只允许 report-only envelope。
- Combat Ledger 能区分事实、候选、拒绝和需要用户决策。
- 美术语法不能把候选画成事实、奖励、NPC 生死、地点毁灭或正史改变。
- reduced-motion 下 theater / stack / ledger 的核心信息仍可读。
