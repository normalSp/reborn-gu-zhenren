# v4.0.0-b2 negative fixtures 与 deterministic hardening 记录

状态：completed。
日期：2026-05-26。
冻结标识：`v400_b2_negative_fixture_deterministic_hardening_v1`。

## 目标

把 v4.0 最危险的误读全部变成机器样本：Auto-Theater Combat 不是已实现 runtime，凡阶战斗没有迁移，自走棋不是高阶主容器，天道/宿命不是玩家结局裁决器，美术/动效/DeepSeek/MiroFish/外部框架不能写正式事实。

本阶段仍是 report-only，不改 runtime、store、save、prompt、UI、DeepSeek、MiroFish、后端、外部依赖或素材。

## hardening 样本族

| 样本族 | 当前结论 |
|---|---|
| `V400-COMBAT-CANDIDATE` | Auto-Theater 只能是 envelope / candidate / ledger，不写 runtime |
| `V400-MORTAL-LITE` | 凡阶 Auto-Theater Lite 只做未来设计预备，不迁移现有棋盘 |
| `V400-AUTO-BATTLER` | 纯自走棋 runtime 被阻断；高阶主路线是 theater / stack / ledger |
| `V400-COMBAT-BLOCK` | 命中、伤害、环境破坏、NPC 生死、奖励越权被阻断 |
| `V400-STACK` | 杀招栈 runtime / cost / result 不能由 DeepSeek 或 UI 自写 |
| `V400-LEDGER` | Combat Ledger 必须区分 fact / candidate / rejected / needs_user_decision / pressure |
| `V400-ART-LANGUAGE` | 美术语法不能把候选画成事实、奖励、生死、地点毁灭或正史改变 |
| `V400-MOTION-SPLIT` | GSAP/Motion 只能表达，不控制业务事实或按钮可用性 |
| `V400-REDUCED-MOTION` | reduced-motion 下仍保留 theater / stack / ledger / pressure 可读性 |
| `V400-FATE` | Fate state 只在 envelope 中作为时代约束，不写 save |
| `V400-HEAVEN` | HeavenWill pressure 不能成为万能失败按钮 |
| `V400-L5` | L5 macro director 不是 NPC，不写 canon/runtime/save |
| `V400-HIDDEN` | hidden-adjacent / 方源证据 / 原著真实锚点升级 MiroFish blocking |
| `V400-DEEPSEEK` | DeepSeek 不裁决命中、伤害、死亡、奖励、宿命 |
| `V400-SYSTEM` | 系统连续性、PA、old-save/no-save/rollback 必须在 rc 复核 |
