## 本版本目标

v4.0 建议冻结为：

`Auto-Theater Combat 与 HeavenWill/Fate Pressure 高阶世界表达模型`

v4.0 的价值是把未来高阶世界的两个核心问题变成工程门禁：

1. 高阶战斗如何从低阶棋盘进化到 Auto-Theater Combat：自走棋式准备、WorldCore 自动结算、battle theater、killer-move stack、Combat Ledger、domain / formation / Immortal Gu House state。
2. 天道、宿命、时代大势和 L5 宏观压力如何作为约束系统，而不是任意裁决或万能失败按钮。
3. Auto-Theater Combat 如何成为 RebornG 的核心战斗表现招牌：水墨战争沙盘、道痕规则可视化、杀招栈舞台、Combat Ledger 复盘和截图验收。

总体设计讨论已确认：纯自走棋不是高阶主路线；凡阶/中阶保留棋盘和行动卡，底层按 `Auto-Theater Lite` 预备；v4.0 不实施凡阶 runtime 迁移。美术与交互总纲已确认：v4.0 只冻结视觉语法和验收计划，不实现 theater UI 或素材生成。

长期路线已补齐：`指导大纲/长期路线/v4.0-v5.0-AutoTheater到高阶世界Runtime总体大纲.md` 作为 v4.1-v4.9 与 v5.0 第一刀的规划基线。它不扩大 v4.0 范围，也不授权 v4.x/v5.0 runtime。

## 本版本非目标

- 不实现完整蛊仙战斗。
- 不实现纯自走棋 runtime。
- 不迁移凡阶战斗 runtime。
- 不实现 theater UI，不生成或晋升 Auto-Theater 素材。
- 不开放五域两天完整游玩。
- 不开放方源、尊者、原著关键人物 live agent。
- 不开放天道/宿命 runtime 裁决。
- 不新增存档字段、迁移或 `SAVE_FORMAT_VERSION`。
- 不新增 DeepSeek 权限或 MiroFish intake。
- 不引入后端、外部框架 PoC、依赖或子代理。
