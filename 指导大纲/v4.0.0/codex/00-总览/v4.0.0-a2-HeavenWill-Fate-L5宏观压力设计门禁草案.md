# v4.0.0-a2 HeavenWill / Fate / L5 宏观压力设计门禁冻结

状态：completed；原文件名保留“草案”用于入口兼容，本文内容已冻结为 a2 设计门禁。
日期：2026-05-26。
冻结标识：`v400_a2_heavenwill_fate_l5_macro_pressure_gate_v1`。

## 目标

为天道、宿命、时代大势和 L5 宏观压力建立第一版架构边界，不实现 runtime 裁决。

## 总原则

天道/宿命在 RebornG 中只能先作为 `pressure / constraint / risk` 系统设计，不能成为玩家结局裁决器。L5 macro director 不是 NPC，不是 DeepSeek 可聊天对象，也不是正史改写器；它只可在未来生成宏观压力候选，且必须经过 WorldCore post-check 和 lore visibility gate。

## 建议概念

### Fate State

沿用强边界：

- `intact`
- `fractured`
- `destroyed`

Fate state 是时代背景与约束条件，不是玩家随手可切换的状态，也不是 UI 私有字段。

v4.0 只允许在 report-only envelope 中描述 Fate state；不写 save，不做 UI 控件，不生成玩家可改变宿命的 runtime 入口。未来若要进入 runtime，必须先明确时代锚点、source pointer、visibility、WorldCore 裁决链和 MiroFish blocking intake。

### HeavenWill Pressure

天道压力只输出：

- pressure：压力。
- constraint：约束。
- bias：机会偏置。
- exclusion：排斥。
- risk：风险。

不得输出：

- 结局裁决。
- NPC 生死。
- 正史改变。
- 奖励/惩罚。
- 隐藏因果正文。

HeavenWill pressure 必须带有可解释边界：

- `whyVisible`：玩家可见层面能理解的压力原因。
- `hiddenRefOnly`：隐藏因果只能做 ref class，不写正文。
- `worldcoreRequired`：任何结果性后果都必须等待 WorldCore。
- `playerAgencyPreserved`：压力不能变成万能失败按钮。

### Era Anchor Graph

时代锚点图只允许 source-pointer class 与 visibility 标记：

- era。
- region。
- public anchor。
- hidden anchor ref。
- macro pressure ref。
- candidate impact。

真实原著锚点、hidden-adjacent material、L4/L5 fact anchor 必须 MiroFish blocking。

v4.0 不创建真实 era anchor graph 数据。a2 只冻结未来 envelope 字段：

| 字段 | 含义 | v4.0 状态 |
|---|---|---|
| `eraRef` | 时代级 source-pointer class | report-only |
| `regionRef` | 区域级 source-pointer class | report-only |
| `publicAnchorRef` | 可见公共锚点 class | report-only |
| `hiddenAnchorRef` | hidden ref class，不含正文 | report-only，未来 MiroFish blocking |
| `macroPressureRef` | 宏观压力候选 ref | report-only |
| `candidateImpact` | 对玩家局势的候选影响 | 不写 save |

### Causality Debt

因果债描述偏离正史后世界压力如何上升。它是风险与压力，不是强制失败按钮。

因果债不能直接产生惩罚、奖励、死亡、捕获、路线封锁、身份变化或正史改写。它只能解释“为什么风险上升、哪些势力可能注意、哪些路线更难”，并在 future gate 获批前保持 report-only。

### L5 Macro Director

L5 不是可聊天 NPC agent。它只能输出 pressure / constraint / risk 候选，并且必须经过 WorldCore post-check。

L5 输出建议未来统一成：

- `macroPressure`: 宏观压力候选。
- `constraint`: 世界规则约束候选。
- `risk`: 风险候选。
- `requiresMiroFish`: 是否触发原著/hidden/L4/L5 blocking。
- `requiresUserDecision`: 是否超出当前授权包。
- `worldcorePostCheck`: WorldCore 必须复核的项目。

## 禁止项

- 不实现 HeavenWill/Fate runtime。
- 不让天道/宿命裁决玩家结局。
- 不开放方源、尊者、原著关键人物 live agent。
- 不暴露隐藏因果链给 UI 或 DeepSeek visible context。
- 不未经 MiroFish intake 写真实原著锚点。
- 不让 L5 以 NPC 方式对话、奖励、惩罚或操控玩家。
- 不把天道/宿命压力当成剧情偷懒按钮。

## 验收样本方向

- `FateState` 只可出现在设计 envelope，不写 save。
- HeavenWill 万能失败按钮样本必须 blocking。
- L5 直接改 canon/runtime/save 样本必须 blocking。
- hidden-adjacent / Fang Yuan evidence 样本必须 MiroFish blocking。
- macro pressure 文案必须清楚标注“压力/候选，不是事实”。

## 结论

a2 设计门禁通过。v4.0 可以进入 b1 report-only checker；但 a2 不授权 HeavenWill/Fate runtime、L5 runtime、方源或尊者等原著关键人物 agent、真实时代锚点写入、MiroFish export/intake、hidden body、DeepSeek visible lore/RAG 或任何正式剧情结局裁决。
