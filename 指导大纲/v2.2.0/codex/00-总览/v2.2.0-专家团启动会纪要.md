# v2.2.0 专家团启动会纪要

状态：completed；D-220 / D-221 已记录，a2 已完成。
日期：2026-05-23
主题：Agent Lab 扩展离线模拟第一阶段。

## 开会背景

v2.1 已把多 agent 世界的第一层工程约束跑通：

- Agent 输出只能是 `AgentProposal`。
- WorldCore post-check 保持最终裁决。
- 自有 zero-dependency runner 能做 dry-run/report-only 审计。
- 10 个合成样本通过，P0/P1/P2=0/0/0。
- 外部框架、live DeepSeek、子代理、BFF/backend、MiroFish export 均未开放。

但 v2.1 还只是骨架验证。它没有证明多 NPC、势力、记忆、宏观压力在一个离线场景里能长期协作，也没有证明失败分类能覆盖多对象互动。

## 专家团总建议

推荐 v2.2 主线：

`Agent Lab 扩展离线模拟第一阶段`

一句话目标：

> 不急着接 runtime，先把 20 NPC / 3 势力 / 1 个 L5 宏观导演的离线样板设计成可重复、可审计、可失败归类的实验室场景。

## v2.2 好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| 20 NPC 分层而不是平铺 | 保持 L0/L1/L2/L3/L5 权限清楚 | 不把所有 NPC 都当 live agent |
| 3 势力 pressure ledger | 测试势力之间的信息传播与冲突 | 仍是 report-only，不写正式势力状态 |
| L5 宏观导演 pressure-only | 让天道/宿命作为约束场进入实验 | 不作为可聊天 NPC，不宣判结局 |
| memory contamination 样本 | 提前抓住谣言/候选/hidden 晋升事实的问题 | memory 只 report-only |
| same-start replay lane for agents | 验证同起点不同 agent 决策差异 | 不新增 `runFingerprint` 或正式存档 |
| 失败回流 future_sample_pool | 让每次失败都变成后续门禁资产 | 需要测试矩阵演进制度兜底 |

## 不推荐现在做

- 不做 runtime agent 接入。
- 不做 live DeepSeek。
- 不做 Claude Code / LangGraph / Mastra / ADK / AutoGen / CrewAI / PydanticAI / Letta / OpenHands PoC。
- 不启用子代理。
- 不建 BFF/backend。
- 不新增 save 字段。
- 不写知识库正文。
- 不新增 MiroFish export。
- 不引入命名原著 NPC、hidden-adjacent、方源公开证据或正式 lore 结论。
