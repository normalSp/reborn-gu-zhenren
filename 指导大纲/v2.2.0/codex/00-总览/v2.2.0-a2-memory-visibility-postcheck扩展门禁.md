## memory 分类

v2.2 Agent Lab 中的 memory 不是 runtime memory、不是 save、不是 canon、不是 DeepSeek visible context。

| 类型 | 允许内容 | 禁止内容 |
|---|---|---|
| `public_observation_candidate` | agent 对公开事件的观察摘要 | hidden/private body、正式事实写入 |
| `subjective_memory_candidate` | L2/L3 对玩家或其他 agent 的主观印象 | 关系数值、忠诚/背叛结论、正式阵营结果 |
| `rumor_memory_candidate` | 未验证传闻、怀疑、误会 | 晋升为事实、触发正式惩罚或奖励 |
| `reflection_candidate` | agent 自我反思、下一步意图理由 | 改写 canon、决定 NPC 生死或任务结果 |
| `intent_memory_candidate` | agent 下一轮意图候选 | 强制玩家路线、正式通行/雇佣/身份 |
| `faction_pressure_memory_candidate` | 势力压力的短期印象 | 正式 faction state、通缉、封锁、招揽结论 |
| `l5_pressure_trace` | L5 宏观压力 trace | 结局宣判、正史锚点改写、奇遇/奖励授予 |

## visibility 分层

| visibility | 用途 | 可进入 report | 可进入 runtime / save / DeepSeek visible |
|---|---|---:|---:|
| `public_pressure` | 公开压力、公开观察、公开下一步 | 是 | 否 |
| `agent_private_candidate` | agent 主观私有候选，仅供离线审计 | 是 | 否 |
| `ref_only` | 只记录 source ref 或 synthetic ref | 是 | 否 |
| `gated` | 需要 WorldCore 或用户门禁再判断 | 是 | 否 |
| `rejected_hidden_body` | 命中 hidden/private body 或禁词 | 只记录拒绝原因 | 否 |
| `rejected_formal_authority` | 命中正式事实/奖励/地点/身份/NPC 生死 | 只记录拒绝原因 | 否 |

注意：`agent_private_candidate` 不是 RebornG hidden/private lore，也不是可以喂给 DeepSeek 的“私有事实”。它只是离线 Agent Lab 报告中的主观候选。

## AgentProposal a2 扩展字段

```ts
type V22MemoryStatus =
  | "none"
  | "candidate"
  | "rumor"
  | "reflection"
  | "intent"
  | "rejected";

type V22Visibility =
  | "public_pressure"
  | "agent_private_candidate"
  | "ref_only"
  | "gated"
  | "rejected_hidden_body"
  | "rejected_formal_authority";
```

建议字段：

| 字段 | 要求 |
|---|---|
| `memoryStatus` | 必须在允许枚举内；默认 `none` |
| `memoryScope` | `agent` / `faction_pressure` / `scene` / `l5_pressure` |
| `visibility` | 必须在 a2 visibility 枚举内 |
| `claimStrength` | `observation` / `rumor` / `suspicion` / `intention` / `pressure` |
| `sourceClass` | `synthetic` / `approved_source_pointer`；a2 默认只允许 `synthetic` |
| `evidenceRefs` | 只能是 synthetic refs 或已批准 source-pointer refs |
| `blockedEffects` | 明列不得写 save/canon/store/runtime/DeepSeek visible |
| `requiresWorldCoreAdjudication` | 必须为 true |

## WorldCore post-check 流程

| 步骤 | 检查 | 失败严重度 |
|---|---|---|
| 1 | schema 可解析、枚举合法、必填字段齐全 | P0/P1 |
| 2 | `sourceClass` 和 `evidenceRefs` 合法 | P0/P1 |
| 3 | visibility 不得绕过 gate | P0/P1 |
| 4 | hidden/private body 不得进入 payload | P0 |
| 5 | proposal 不得写 save/canon/store/runtime/DeepSeek visible | P0 |
| 6 | reward/location/identity/faction/NPC life-death 不得正式化 | P0 |
| 7 | memory 不得把 rumor/candidate/pressure 晋升事实 | P1 |
| 8 | L3 dialogue 不得给正式通行/雇佣/任务结论 | P1 |
| 9 | L5 不得宣判结局、奇遇、宿命结论或正史锚点 | P0 |
| 10 | 多 agent 不得 repeated idle loop 或互相复读 | P2，严重时 P1 |

## MiroFish 判定

a2 继续 `not_needed`。

原因：

- 只使用抽象低阶 NPC / 势力 archetype。
- `sourceClass` 默认只允许 `synthetic`。
- 不引入命名原著 NPC、hidden-adjacent、方源公开证据、正式 lore 结论或 L4 原著关键人物。

升级 blocking 的触发仍为：

- 命名 NPC / 原著关键人物进入样本。
- L4/L5 具体原著事实锚点进入样本。
- hidden-adjacent、方源公开证据、正式 lore 结论进入样本。
- 正式地点、阵营、身份、奖励、NPC 生死或 canon promotion 进入样本。
- 需要知识库正文或 DeepSeek visible lore/RAG。
