# v2.2.0-a1 20NPC 三势力一 L5 场景模型设计门禁

状态：completed；等待 D-221 决策。
日期：2026-05-23

## 目标

把 v2.2 的扩展对象定义成可验证的离线场景模型，而不是直接写 runner。

本门禁回答：

1. 20 个 NPC 如何分层。
2. 3 个势力如何表达压力而不变成正式阵营状态。
3. L5 宏观导演如何只输出 `heaven_will_pressure`，不宣判结局。
4. 哪些字段可以进入 `AgentProposal`。
5. 哪些字段必须被 WorldCore post-check 拒绝。

## 总体场景模型

场景名暂定：

`outer_edge_agent_lab_synthetic_v1`

定位：

- 合成低阶外缘生活场景。
- 不引用命名原著 NPC。
- 不引用 hidden/private body。
- 不引用具体原著地点新增事实。
- 不写正式存档。
- 不进入玩家 runtime。

规模：

- 20 NPC。
- 3 势力。
- 1 个 L5 宏观导演。
- 6 至 10 个离线回合为 a1/a2/b1 默认设计范围；是否实现由 D-221 决定。

## NPC 分层

| 层级 | 数量 | 类型 | 允许输出 | 禁止输出 |
|---|---:|---|---|---|
| L1 | 8 | 路人、巡逻、苦力、临时摊贩、低阶兽群迹象 | public_pressure、routine_hint、risk_hint | 记忆、正式身份、奖励、地点、NPC 生死 |
| L2 | 8 | 小队成员、短工同伴、低阶商队边缘人、散修聚点熟脸 | memory_candidate、reflection_candidate、intent_candidate、relationship_pressure | 正式关系数值、忠诚结论、背叛结论、战斗结果 |
| L3 | 4 | 当前场景关键 NPC、临时雇主、盘问者、冲突调停者 | dialogue_intent_candidate、negotiation_pressure、scene_goal_candidate | 正式通行、正式雇佣、正式阵营加入、正式任务奖励 |

L4 默认不进入 v2.2 a1/a2。

如后续引入 L4 原著关键人物或大势力核心，必须升级 MiroFish `blocking` 并另列用户决策。

## 三势力模型

三势力只作为抽象 pressure source：

| 势力代号 | 抽象角色 | 允许输出 | 禁止输出 |
|---|---|---|---|
| `local_order_pressure` | 地方秩序/盘查/低阶规矩 | suspicion_delta_candidate、access_pressure、watch_hint | 正式通缉、正式封锁、正式身份审定 |
| `market_labor_pressure` | 商队/劳力/临时交易窗口 | wage_pressure、trust_window_candidate、market_access_hint | 正式商队身份、价格表、库存、稳定工资 |
| `rogue_shelter_pressure` | 散修/临时遮蔽/人情债 | shelter_risk、favor_debt_candidate、rumor_pressure | 正式加入散修势力、庇护结论、NPC 生死 |

三势力不得写入正式 faction state。

## L5 宏观导演模型

L5 代号：

`heaven_will_pressure`

允许输出：

- macro_pressure。
- coincidence_pressure。
- constraint_hint。
- escalation_risk。
- fate_echo_ref。

禁止输出：

- 直接宣判玩家结局。
- 直接改写正史锚点。
- 直接决定 NPC 生死。
- 直接授予奖励、地点、身份或奇遇。
- 作为可聊天 NPC。
- 暴露 hidden/private body。

L5 的所有输出必须带：

- `requiresWorldCoreAdjudication: true`
- `visibility: "public_pressure" | "ref_only"`
- `authority: "proposal_only"`

## AgentProposal 最小字段

v2.2 a1 建议继续沿用 v2.1 的 proposal-only 思路，并为 v2.2 扩展以下设计字段：

```ts
type V22AgentProposalKind =
  | "public_pressure"
  | "routine_hint"
  | "memory_candidate"
  | "reflection_candidate"
  | "intent_candidate"
  | "dialogue_intent_candidate"
  | "faction_pressure_candidate"
  | "heaven_will_pressure"
  | "worldcore_review_required";
```

建议字段：

| 字段 | 要求 |
|---|---|
| `proposalId` | 稳定 id，便于 replay/rescore |
| `roundId` | 离线回合 id |
| `agentId` | 抽象 agent id，不使用原著命名 NPC |
| `agentLayer` | L1/L2/L3/L5 |
| `kind` | 必须在允许枚举内 |
| `visibility` | public / ref_only / gated |
| `sourceRefs` | 只能是 synthetic 或已批准 source-pointer ref |
| `payload` | 只能是候选、压力、记忆摘要、意图摘要 |
| `forbiddenEffects` | 明列不得写入的效果 |
| `requiresWorldCoreAdjudication` | 必须为 true |

## WorldCore post-check 必拒项

出现以下任一项，a1 规定为 P0/P1：

| 严重度 | 条件 |
|---|---|
| P0 | hidden/private body 出现在 payload |
| P0 | proposal 写入 save/canon/store/runtime/DeepSeek visible context |
| P0 | agent 宣判正式奖励、正式地点、正式身份、正式阵营、NPC 生死 |
| P0 | L5 宣判结局或改写正史锚点 |
| P1 | L2 memory 把谣言、候选、压力晋升为事实 |
| P1 | L3 dialogue 产生正式通行/雇佣/任务结论 |
| P1 | 势力 pressure 被写成正式 faction state |
| P2 | 文字表达偏正式凭信词，但未产生事实写入 |

## MiroFish 判定

a1 当前保持 `not_needed`。

原因：

- 全部采用抽象低阶 NPC / 势力 archetype。
- 不引入命名 NPC。
- 不引入 hidden-adjacent。
- 不产出正式 lore 结论。

升级 blocking 的条件继续沿用 v2.2 MiroFish 协议。

## a1 结论

a1 可以通过为设计门禁：

- 20 NPC / 3 势力 / 1 L5 只是离线设计目标。
- L1/L2/L3/L5 权限边界明确。
- 三势力只输出 pressure，不写正式 faction state。
- L5 只输出 `heaven_will_pressure`，不宣判结局。
- b1 runner 扩展仍未自动批准。

下一步需要用户审批 D-221，决定是否进入 a2，以及是否在 a2 后允许 b1 runner 扩展。
