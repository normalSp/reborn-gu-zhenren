# v2.0.0 总体开发大纲

状态：b4 completed locally；D-200/D-201 已全批准。
日期：2026-05-22
主题候选：第一个区域活世界

## 总目标

v2.0 的目标不是打开完整南疆，而是在一个可控小区域内证明 RebornG 可以长期运行：

- 玩家每一步选择都能留下可解释后果。
- 路线、生存经济、NPC/势力、冲突、身份候选和叙事表达能够合流。
- 同开局可以出现不同体验。
- 区域事件可回放、可审计、可长测。
- DeepSeek 保持叙事能力，但不拥有世界事实裁决权。

## 推荐范围

第一核心区域：

`南疆早期低阶外缘小区域`

包含候选：

- 山路歇脚点。
- 临时商队接触窗口。
- 散修落脚/短工圈。
- 城门外缘门槛。
- 路途补给、盘问、冲突和人情压力。

不包含：

- 完整南疆。
- 完整商家城。
- 正式商队系统。
- 正式散修势力。
- 正式城市身份。
- 正式任务、工资、奖励、库存、价格。
- NPC 生死或 hidden/private 揭示。

## WorldCore 原则

v2.0 runtime 若获批，应采用：

`EventEnvelope -> WorldCore adjudication -> regionalEventLedger -> DeepSeek narrative`

含义：

- `EventEnvelope` 是候选，不是事实。
- WorldCore 决定可发生、不可发生、需要前置、后果等级。
- `regionalEventLedger` 只记录已裁决的公开事件、来源、压力和待处理后续。
- DeepSeek 只根据已批准事实写叙事表达。

## save-format 默认建议

用户已批准专家团建议：

- v2.0 做真正区域活世界，允许 b1 bump `SAVE_FORMAT_VERSION = 25`。
- v25 只新增一个最小聚合：`regionalEventLedger`。
- 暂不新增 `runFingerprint`。
- 暂不新增 `regionalLifeState` / `areaLivingState`。
- 暂不新增 `identityRouteState` / `professionState`。

理由：

- 区域事件需要跨回合承接和回放。
- 多个状态字段会制造多权威。
- 同开局差异可以先由公开 pressure deck、事件候选顺序、玩家选择和 replay evidence 支撑。
