# v2.0.0-b4 UI可读性旧档回滚与入口一致性

日期：2026-05-22
状态：completed locally
分支：`codex/v200-b4-ui-save-rollback`

## 一句话

b4 不新增字段、不改 DeepSeek 权限，而是在现有 v25 `regionalEventLedger` 和 `账本` tab 内补一个旧档/回滚/入口一致性 review，让玩家和测试都能看懂：旧档空账本安全、手改账本要保守复核、b1-b3 共用同一入口、回滚不牵连其他系统。

## 范围

本阶段只做：

- 新增纯本地 helper `buildV200LedgerReadinessReview()`。
- 在现有 `账本` tab 内显示“旧档与回滚”审计卡。
- 用 unit/e2e/live smoke/Player Advocate 验证旧档默认、入口一致、回滚边界、T3 前置状态。

## 不做

- 不新增 `SAVE_FORMAT_VERSION = 26`。
- 不新增 `runFingerprint`。
- 不新增 `regionalLifeState` / `areaLivingState`。
- 不新增 `identityRouteState` / `professionState`。
- 不新增 store action、migration/defaults 或新持久字段。
- 不新增 DeepSeek visible knowledge / RAG / ledger write authority。
- 不新增 BFF/backend、子代理、公测口径或 EdgeOne 部署。
- 不开放正式地点、阵营、奖励、NPC 生死、正式商队身份、正式通行、正式价格/库存。

## review 口径

| 状态 | 触发 | 玩家/测试看到什么 |
|---|---|---|
| `old_save_default` | 旧档或空账本 | v25 空账本安全，不制造 replay 差异、地点、身份或奖励 |
| `ready_for_replay` | 公开事件与 follow-up 已登记 | 入口与回滚可读，可进入 T3 样本池 |
| `blocked_for_review` | authority/region 被手改 | 不信任编辑档，要求回到本地 WorldCore 重新登记 |

## 回滚边界

- b4 回滚优先移除 `v200-ledger-readiness-review` helper 和 UI 卡。
- 不删除 v25 `regionalEventLedger` 字段；该字段由 b1 save-format 门禁管理。
- 不删除 b2 去重或 b3 replay helper，除非后续专项明确要求。
- 不影响路线、生存、社会、冲突、身份候选等既有证据。
