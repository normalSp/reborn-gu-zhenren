## 新增存档字段

字段：`regionalEventLedger`

允许保存：

- `publicEvents`：公开区域事件，不含 hidden body。
- `pendingFollowUps`：待本地系统复核的后续。
- `pressureSummary`：公开压力汇总。
- `evidenceRefs` / `sourceRefs`：只保留安全 ref。
- `audit`：WorldCore / DeepSeek / hidden policy 说明。

禁止保存：

- DeepSeek raw narrative。
- 原著正文 / raw quote / source body。
- hidden/private body。
- 正式地点、正式身份、正式阵营、奖励、价格表、库存、工资。
- 战斗胜负、掉落、NPC 生死、通缉、封锁、招揽。
- `runFingerprint`、`regionalLifeState`、`areaLivingState`、`identityRouteState`、`professionState`。

## WorldCore 裁决口径

`buildV200RegionalEventEnvelopes()` 读取既有 v1.7 区域生活 projection 的 visible pressure cards，将它们转成区域事件 envelope。

`resolveV200WorldCoreRegionalEventLedgerSync()` 只在有 visible regional-life context 时写账本：

- 成功时：写入 `status = events_tracked`、`authority = worldcore_region_engine`、`activeRegionKey = southern_border_outer_edge_low_rank`。
- 失败时：保留旧账本，返回 `needs_regional_life_visible_context`。
- 编辑档异常时：`normalizeRegionalEventLedger()` 保守归一化，非法 authority / region 进入 `blocked`。

## 旧档与回滚

- v24 及更早旧档归一化时得到空账本：`status = not_started`、`authority = migration_default`。
- 新字段不影响既有 `livingWorldState`、`routeLocationState`、`survivalEconomyState`。
- persist partialize 明确排除 `syncRegionalEventLedgerAction()`。
- 回滚路径：移除 v200 slice/UI/engine/test 后，旧 v24 字段仍不依赖 b1 新字段；但已升级 v25 的档案会丢失区域事件账本。
