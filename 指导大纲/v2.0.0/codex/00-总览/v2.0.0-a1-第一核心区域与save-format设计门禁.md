## `SAVE_FORMAT_VERSION = 25` 评估

### 正面收益

- 区域事件可跨回合保留，不依赖临时 projection。
- 300+ 长测可检查事件连续性和回放一致性。
- 玩家行为能形成 pending follow-up，而不是每次都靠叙事记忆。
- DeepSeek 输出可被本地 ledger 校验，降低长期漂移。

### 负面风险

- 存档迁移和旧档回滚成本上升。
- ledger 字段若太大，会变成第二个世界状态。
- 事件记录若不严格，会把临时凭证、传闻、盘问误写成正式地点/身份/奖励。
- 测试矩阵、Player Advocate、T3 长测必须同步加重。

### 决策结论

用户已批准 v25 条件：

- 只新增一个最小聚合 `regionalEventLedger`。
- 不同时新增 `runFingerprint`、`regionalLifeState`、`identityRouteState`。
- migration/defaults/tests 必须同一 commit。
- 所有写入必须来自 WorldCore / local engine，不来自 DeepSeek。

## `runFingerprint` 评估

已批准暂缓。

理由：

- v2.0 的关键是事件承接，不是随机种子。
- 过早持久化 fingerprint 会增加复现、回滚和测试复杂度。
- 同开局差异可先由公开 pressure deck、事件候选排序、玩家选择、ledger pending follow-up 和叙事表达实现。

晋升条件：

- T3 证明没有持久差异字段会导致 100+ 轮后体验收敛。
- replay 证明 fingerprint 不会造成事实漂移。
- 用户单独批准字段表和 migration。

## T3 长测硬门

建议采用：

- total rounds：320。
- live rounds：不少于 160。
- replay/deterministic rounds：不少于 160。
- live model：`deepseek-v4-flash`。
- checkpoint：每 20 轮记录一次。

检查维度：

- 世界状态一致性。
- DeepSeek authority。
- 原著/IF 边界。
- hidden/private 保护。
- 区域事件连续性。
- 玩家下一步理解。
- 重复空转和叙事疲劳。
- 同开局差异是否真实存在。
