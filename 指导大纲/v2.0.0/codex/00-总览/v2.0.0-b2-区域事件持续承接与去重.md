# v2.0.0-b2 区域事件持续承接与去重

日期：2026-05-22
状态：completed locally
分支：`codex/v200-b2-regional-event-continuity`

## 一句话

b2 不新增字段、不扩 DeepSeek 权限，而是在现有 v25 `regionalEventLedger` 内把区域事件从“一次登记”推进到“跨回合可持续承接、可去重、可兼容 b1 旧 id”。

## 范围

本阶段只改现有账本链路：

- `EventEnvelope -> WorldCore -> regionalEventLedger` 的事件 id 稳定化。
- 同一 `eventKind + publicSummaryKey` 的公开事件跨回合同步时合并，不重复膨胀。
- pending follow-up 跟随稳定 event id 重挂。
- b1 已生成的 `v200b1_*_t<turn>` 旧 id 在下一次 sync 时升级为稳定 id。
- UI 显示“去重后预览”和承接键，方便玩家/测试理解为什么不是重复刷事件。

## 不做

- 不新增 `runFingerprint`。
- 不新增 `regionalLifeState` / `areaLivingState`。
- 不新增 `identityRouteState` / `professionState`。
- 不新增 `SAVE_FORMAT_VERSION = 26`。
- 不新增 DeepSeek visible knowledge / RAG / ledger write authority。
- 不新增 BFF/backend、子代理、公测口径或 EdgeOne 部署。
- 不开放正式地点、阵营、奖励、NPC 生死、正式商队身份、正式通行、正式价格/库存。

## 去重策略

WorldCore 使用稳定 logical key：

```text
eventKind + publicSummaryKey
```

这意味着：

- 同一盘问事件跨回合再次出现时，更新 `turn`、合并 source refs，不复制新事件。
- 新的公开压力类型仍可新增事件。
- b1 的旧 id 不作为事实权威，只作为兼容输入；下一次 sync 后会被稳定 id 替换。
- pending follow-up 不再跟随 turn 生成新条目，而是跟随稳定 event id。

## 旧档与回滚

- v24 旧档仍由 b1 migration/defaults 补空账本。
- b1 已登记账本不会被丢弃；下一次 b2 sync 会把旧 turn-suffixed id 升级为稳定 id。
- 非法 authority/region 仍进入 `blocked`，不信任编辑档。
- b2 没有新 migration/defaults，也没有新持久字段。

## 结论

b2 可以作为 v2.0 区域事件账本的第二个 runtime 小刀收束：它把 b1 的最小账本变成可长期 replay 的稳定账本，但没有扩大世界事实权威。下一步建议进入 `v2.0.0-b3-同开局差异与replay对照`，继续暂缓 `runFingerprint`，先用现有账本和 replay 证据判断是否真的需要新字段。
