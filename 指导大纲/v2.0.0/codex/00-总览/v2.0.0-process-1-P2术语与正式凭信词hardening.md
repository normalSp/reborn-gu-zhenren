## 触发原因

b3 / b4 live smoke 都没有出现 P0/P1 越权，但持续出现 P2 词面风险：

- `青毛` / `Qingmao`：地名术语错字或英文漂移。
- `通行证`、`登记`、`正式凭证`、`凭证`：容易被玩家理解成正式凭信已发放。
- `商队成员`、`成员资格`、`护卫身份`：容易被玩家理解成正式身份已成立。

这些不是 v2.0-b4 的阻断项，但会污染 rc T3 长测，因此 process-1 必须把它们收束成硬门。

## 本阶段范围

本阶段只做术语、prompt、evaluator、runtime 动态上下文护栏和记录收束。

本阶段不做：

- 不新增存档字段，不 bump save format。
- 不新增 `runFingerprint`。
- 不新增 `regionalLifeState` / `areaLivingState`。
- 不新增 `identityRouteState` / `professionState`。
- 不新增 DeepSeek 知识库 / RAG / full-book visible summary。
- 不新增 BFF/backend、子代理、public wording 或 EdgeOne 部署。
- 不开放正式地点、阵营、奖励、NPC 生死、正式身份、正式凭信或 canon promotion。

## 代码变更

### live probe hardening

文件：`scripts/run-v200-regional-event-ledger-live-probe.mjs`

已完成：

- `sanitizeCarryText()` 会把错误地名、正式凭信词、正式身份词压成安全泛称，避免跨轮 carry 污染。
- `--max-p2=0` 现在合法，用于 process-1 严门禁。
- 新增 `eval:deepseek:v200-regional-ledger-live:process1`，固定 5 samples x 4 rounds，accepted >= 0.9，P2<=0。
- system prompt 和 per-round user prompt 强化输出禁令。
- evaluator 扩展 P2 扫描词。
- 新增 retry-on-P2 repair：若最终 JSON 出现正式凭信 / 术语漂移 P2，自动追加修复指令并重试；最终仍按 P0/P1/P2 计分。
- 已恢复的 JSON / protocol retry 不再伪装成内容 P2，但保留 `retryAttemptCount` 与 `recoveredRoundCount` 统计。

### runtime context guard

文件：`src/engine/context-builder.ts`

新增动态上下文段：

`【v2.0 正式凭信词与区域账本叙事护栏】`

作用：

- 告诉 DeepSeek 区域账本只提供公开压力、证据留痕和后续前置条件。
- 玩家要求通行、准入、跟队、名单、身份、凭信、批准或长期庇护时，只能写成压力、候选、前置条件或后续门禁。
- 若必须提及这类概念，只使用 `正式凭信词`、`正式名单词`、`正式身份词`、`后续门禁`。
- 地名统一 `青茅`，避免英文地名或误写地名。
- 再次明确本地引擎负责事实、路线、账本、奖励、身份与后果裁决。

这个段落放在 dynamic context，不污染 cacheable system prompt。
