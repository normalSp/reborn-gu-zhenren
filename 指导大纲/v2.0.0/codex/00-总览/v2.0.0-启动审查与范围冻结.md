# v2.0.0 启动审查与范围冻结

状态：approved；D-200/D-201 已全批准。
日期：2026-05-22

## 推荐主题

`第一个区域活世界：南疆早期低阶外缘小区域`

## 为什么 v2.0 可以启动

v1.9 已完成 v2.0 前置 readiness：

- 第一核心区域候选清楚。
- MiroFish v2 prelude topic-slice 已完成 intake。
- 区域事件 envelope 已有 report-only 草案。
- `runFingerprint` 风险和替代方案已评估。
- T3 300+ mixed/live/replay 计划已建立。
- P2 正式凭证词和英文术语风险已进入 hardening。

## 为什么 v2.0 仍不能直接写 runtime

v2.0 一旦进入 runtime，至少会触碰区域状态、事件后果、玩家长期理解、DeepSeek 长线叙事和旧档兼容。它比 v1.7/v1.8 projection-first 风险更高。

因此 a1 前不得直接：

- bump `SAVE_FORMAT_VERSION`。
- 新增 `regionalEventLedger`。
- 新增 UI tab / store action / migration。
- 改 DeepSeek prompt/context。
- 开 live probe。
- 启用子代理。

## 允许范围

- 建立 v2.0 启动包。
- 建立 D-200 / D-201 决策池。
- 细化第一核心区域边界。
- 设计最小 `regionalEventLedger` 字段表和迁移/回滚要求。
- 明确 `runFingerprint` 暂缓或晋升条件。
- 设计 T3 320 轮 mixed/live/replay 长测硬门。
- 复用 v1.9 a2 MiroFish intake 作为 source-pointer 基线。
- 更新仪表盘、PROJECT-STATE、AGENTS 和 skill audit。

## 暂缓范围

- runtime code。
- save-format bump。
- migration/defaults/tests。
- MiroFish 新 export。
- live DeepSeek probe。
- Player Advocate 实测。
- BFF/backend。
- public wording。
- EdgeOne。

## 禁止范围

- 完整南疆。
- 完整商家城。
- 正式地点、正式阵营、正式商队/散修/城市身份。
- 正式价格、库存、工资、交易结算。
- 正式奖励、战斗掉落、NPC 生死。
- hidden/private 或方源私密因果可见化。
- DeepSeek visible full-book knowledge/RAG。
- 子代理写 runtime/canon/docs/git。

## Player Advocate / live probe

本启动包是 docs/governance，live DeepSeek 豁免。

若 a1 后进入 runtime，建议标准：

- b1：30-50 轮 Player Advocate；20 轮 live smoke。
- b2/b3：按风险增加 40-80 轮 mixed/live。
- rc：100-150 轮 Player Advocate；T3 320 total rounds，其中 live 建议不低于 160 轮。

所有 live 使用 `deepseek-v4-flash`，必须记录模型、样本、轮次、成本和报告路径。

## 当前结论

v2.0-a0 已开会并建立启动包，D-200/D-201 已全批准。

下一步可以进入 b1 runtime 第一刀，但范围只限：

- `SAVE_FORMAT_VERSION = 25`。
- 单一最小 `regionalEventLedger`。
- `EventEnvelope -> WorldCore -> Ledger -> DeepSeek` 链路。
- migration/defaults/tests/rollback。

仍不得新增 `runFingerprint`、正式区域状态、正式身份状态、DeepSeek RAG、BFF/backend、子代理、formal outcome 或 EdgeOne。
