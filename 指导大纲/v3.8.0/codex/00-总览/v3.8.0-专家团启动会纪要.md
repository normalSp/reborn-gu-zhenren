# v3.8.0 专家团启动会纪要

状态：completed。
日期：2026-05-25。
主线建议：`transient proposal graph 长期稳定性与多小势力压力复核`。

## 背景

`v3.4` 已打开最小 transient proposal-only L2/L3 first cut；`v3.5` 做 lifecycle hardening；`v3.6` 做 synthetic L2/L3 微扩；`v3.7` 做 multi-NPC / small-faction proposal graph。到这里为止，agent 仍然没有持久状态、没有存档写入、没有 live DeepSeek 权限、没有正式世界事实裁决权，WorldCore 仍然是最终权威。

长期路线要求 `v3.7-v3.8` 继续复核多 NPC、小势力、同开局差异、长期记忆污染和漂移。专家团一致认为 v3.8 不应跳到完整 runtime agent 或真实势力社会，而应先做 proposal graph 的长线稳定性验证。

## 专家意见

Project Lead：
v3.8 应该是 v3.7 的稳定性延展，不是权限扩张。目标是把“多个候选意图和小势力压力同时存在”从短场景提升到更长轮次，并判断是否足以进入 v3.9 安全收束。

Lore & World Designer：
继续使用 synthetic/generic NPC 和 pressure source。只要触及命名 NPC、真实势力、hidden-adjacent、方源公开证据、原著关键人物或正式 lore 结论，立即升级 MiroFish blocking 并停机。

Systems Architect：
v3.8 的价值不在于新增系统字段，而在于证明多压力 proposal graph 不会偷偷变成 standing、warrant、recruitment、blockade、reward 或 NPC life/death。所有候选必须仍可被 WorldCore post-check 拒绝、降级或标记 `needs_user_decision`。

AI Pipeline Architect：
不需要 live DeepSeek。v3.8 仍以 deterministic synthetic probe 为核心，验证 graph stability、rejection stability、manual-review lane、candidate expiration、same-start variation。live probe 可留到 v3.9 或未来单独审批。

Player Advocate：
玩家需要看懂“多个 NPC / 小势力压力”只是意图候选和传闻压力，不是游戏已经承认的事实。v3.8 UI/copy 若改动，必须继续用明确标签区分候选、拒绝、过期、需玩家决策、事实。

QA/Test Engineering Guardian：
测试应从 v3.7 的 120 轮 deterministic 和 40 轮 PA 升级为更长稳定性门禁：建议 150 轮 deterministic proposal graph stability，50 轮 PA。测试重点是长期漂移、重复空转、误把候选当事实、多压力互相污染、old-save/no-save/rollback。

Git/rollback Steward：
继续使用版本分支 `codex/v380-startup-proposal-graph-stability`。启动会提交只放文档；后续开发如果获批，提交边界按 a0/a1/a2/b1/b2/b3/rc 分段或按明确实现包收束。版本完成后记录是否建议合并 main，但不自动合并。

Skill Sync Steward：
本轮触发 `reborn-expert-council`、`game-dev-text`、`reverend-insanity-lore`。`reborn-combat-motion` 默认 `no_update_needed`，除非后续 UI/动效显著变化。`mirofish-reborng-export` 默认 `no_update_needed`，因为 v3.8 当前 MiroFish 为 `not_needed`。

## 路线比较

| 路线 | 内容 | 收益 | 风险 | 专家建议 |
|---|---|---|---|---|
| A | proposal graph 长期稳定性与多小势力压力复核 | 稳住 v3.7 成果，补 v3.9 前关键证据 | 产出不如新功能显眼 | 推荐 |
| B | 直接扩大 runtime agent 权限 | 体验变化更明显 | 易越过持久状态、事实裁决、hidden/lore 边界 | 不推荐 |
| C | 先跑 live DeepSeek / MiroFish 真实事实切片 | 可提前看真实文本表现 | 会把 v3.8 变成权限和 lore 专项 | 暂不推荐 |

## 结论

v3.8 采用路线 A。用户已批准 `D-380-001` 至 `D-380-012`；`F-380` 全部继续 `future_gate_required`。若后续触发 `F-380` 或例外停机清单，立即停止自动推进并请用户决策。
