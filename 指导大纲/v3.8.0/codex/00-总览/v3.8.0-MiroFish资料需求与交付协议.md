# v3.8.0 MiroFish 资料需求与交付协议

状态：startup proposed。
当前 need level：`not_needed`。

## 为什么当前不需要 MiroFish

v3.8 使用 synthetic/generic NPC 与 generic 小势力 pressure source，只验证 proposal graph 稳定性、候选表达、WorldCore post-check 和测试门禁。它不需要真实原著事实、命名 NPC、真实势力、隐藏事实、方源公开证据或正式 lore 结论。

## 立即升级 blocking 的条件

出现以下任意情况，必须停止并进入 MiroFish blocking/intake review：

- 命名 NPC。
- 真实势力或真实地点。
- hidden-adjacent 材料。
- 方源公开证据或相关误判风险。
- L4/L5 原著关键人物、天道/宿命、时代锚点。
- 正式 lore 结论。
- 知识库正文或 runtime canon promotion。

## 允许使用的材料

- v3.7 已有 synthetic/generic 样本。
- 当前项目规则、测试制度、WorldCore/post-check 约束。
- 长期路线中对 v3.7-v3.8 的抽象目标。

## 禁止

- 不复制 MiroFish 原始包正文。
- 不把全书基础包直接喂给 DeepSeek。
- 不把 MiroFish 输出直接提升为 canon/runtime。
- 不让 MiroFish 决定 NPC 生死、奖励、正式阵营、正式地点或 hidden/private body。
