# v3.8.0 例外停机清单

状态：startup proposed。
用途：若 v3.8 进入 `/goal` 自动推进，触发以下任意项必须立即停止并告知用户。

## 必停条件

1. 需要新增 save field、migration、`SAVE_FORMAT_VERSION` bump、`runFingerprint`、persistent agent state、store slice。
2. 需要修改 DeepSeek prompt/context/model/authority，或需要 live DeepSeek probe 超出已批档位。
3. deterministic 或 Player Advocate 出现 P0/P1，或失败需要改变方案而不只是修复实现缺陷。
4. 触及真实原著事实、命名 NPC、真实势力、hidden-adjacent、方源公开证据、L4/L5、天道/宿命、正式 lore 结论。
5. 需要 MiroFish export/intake、知识库正文、runtime canon、hidden/private body、prompt body archival。
6. 候选 pressure 需要升级为正式 standing、warrant、recruitment、blockade、reward、identity、location、NPC life/death。
7. 需要 backend/BFF/service/job queue/eval archive/cloud save。
8. 需要外部 agent framework PoC/dependency/vendored subset/read-only scan/patch artifact/subagents。
9. 需要改变公开发布、法律边界、EdgeOne、branch protection、main merge。
10. 发现 v3.7/v3.8 文档与代码事实冲突，且无法用当前授权包内的文档/测试修正解决。

## 不必停但需记录

- 文档措辞收紧。
- synthetic/generic 测试样本增加，但不引入真实 lore。
- focused unit/e2e 补测。
- Skill sync audit 记录为 `updated` 或 `no_update_needed`。
- Git 提交边界细化，但不自动合并 main。
