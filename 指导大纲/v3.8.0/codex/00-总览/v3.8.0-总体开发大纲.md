# v3.8.0 总体开发大纲

状态：completed。
主线：`transient proposal graph 长期稳定性与多小势力压力复核`。

## 总目标

v3.8 的目标不是扩大 agent 权限，而是证明 v3.7 的 proposal graph 能承受更长轮次和更多 generic 小势力压力源，且仍不会污染事实层、存档层、DeepSeek 权限层或原著 lore 边界。

## 产物目标

- a0：复盘 v3.7，冻结 v3.8 只做 proposal graph stability。
- a1：设计 `v380` long-horizon proposal graph stability model。
- a2：设计 multi-small-faction pressure / same-start variation copy gate。
- b1：若获批，新增或扩展自有 transient/report-only `v380` stability checker/engine，不写 save/store/prompt。
- b2：150 轮 deterministic same-start graph stability。
- b3：50 轮 Player Advocate + old-save/no-save/rollback 证据。
- rc：质量收束、Skill sync、dashboard/AGENTS/PROJECT-STATE 同步、commit/push/CI。

## 实现默认

若后续进入实现，建议新增自有文件和脚本：

- `src/engine/v380-transient-agent-proposal-graph-stability.ts`
- `buildV380TransientAgentProposalGraphStability(input)`
- `V380_TRANSIENT_AGENT_PROPOSAL_GRAPH_STABILITY_VERSION`
- `npm run check:v380-transient-agent-proposal-graph-stability`

上述设计默认已在用户批准 D-380 后落地；实现仍保持 no-save/no-runFingerprint/no-live-DeepSeek/no-MiroFish/no-backend/no-external-framework。

## 成功标准

- proposal graph 候选不会升级成事实。
- 多 pressure source 不会生成正式 standing/warrant/recruitment/blockade。
- long-horizon deterministic gate 中 P0/P1 false negative 为 0。
- Player Advocate 能看懂候选和事实区别。
- old-save/no-save/rollback 证据明确。
- 所有 `F-380` 保持 future gate。
