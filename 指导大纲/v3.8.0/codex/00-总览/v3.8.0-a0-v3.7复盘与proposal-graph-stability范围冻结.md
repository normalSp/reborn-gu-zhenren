# v3.8.0-a0 v3.7复盘与proposal graph stability范围冻结

日期：2026-05-25
状态：completed
冻结版本：`v380_a0_v37_replay_scope_freeze_v1`

## v3.7 复盘

`v3.7.0` 已完成 transient multi-NPC / small-faction AgentProposal proposal graph：

- 引入 `v370` proposal graph 层，连接多个 synthetic/generic L2/L3 NPC 候选与小势力公开压力源。
- 通过 120 轮 deterministic gate、40 轮 Player Advocate、old-save/no-save/rollback 证据。
- 保持 no-save、no-runFingerprint、no-live-DeepSeek、no-MiroFish、no-backend、no-external-framework。
- 仍只在 `意图` tab 表达候选，不写正式事实、势力 standing、奖励、身份、NPC 生死或 canon。

## v3.8 主要问题

v3.7 已证明 graph 可用，但还需要验证：

- 更长轮次下 graph 是否会把候选误累积成事实。
- 多个 generic 小势力压力源叠加时，玩家是否还能理解“pressure only”。
- 同开局差异是否能增加重玩感，同时不依赖 `runFingerprint` 或 save 写入。
- rejected/expired/needs_user_decision 是否会污染后续候选表达。
- WorldCore final authority 是否在 UI、测试、文档中持续可见。

## 范围冻结

v3.8 只做：

- `v380` transient proposal graph stability layer。
- synthetic/generic multi-pressure lanes。
- same-start variation copy gate。
- memory contamination audit。
- 150 轮 deterministic gate。
- 50 轮 Player Advocate。
- old-save/no-save/rollback evidence。

v3.8 不做：

- broader runtime agent society。
- persistent agent memory。
- save/store/prompt/canon 写入。
- live DeepSeek。
- MiroFish topic-slice。
- backend/BFF。
- external framework PoC 或 dependency。
- L4/L5、天道/宿命、原著关键人物 agent。

## 结论

v3.8 可进入 `a1`，但必须坚持“proposal graph stability 是候选表达稳定性复核，不是世界事实系统扩权”。
