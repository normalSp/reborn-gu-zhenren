# v3.8.0 分层 Agent 与 WorldCore 测试体系进化审计

状态：startup proposed。

## 当前测试体系是否制度化

是。v2.1 至 v3.7 已形成连续测试链：

- Agent Lab offline runner：验证 AgentProposal 基础 schema 和越权分类。
- Eval farm：验证 failure family、hidden leak、formal credential、memory contamination。
- Runtime admission / go-no-go：验证哪些能力只能 report-only，哪些可进入设计门禁。
- Runtime rehearsal：验证 proposal -> post-check -> expression chain。
- v3.4 transient first cut：验证最小 `意图` tab 候选表达。
- v3.5 lifecycle hardening：验证 candidate/rejected/expired/needs_user_decision。
- v3.6 micro-expansion：验证多个 synthetic L2/L3 candidate lane。
- v3.7 proposal graph：验证 multi-NPC / small-faction proposal graph。

## v3.8 需要进化的点

v3.7 已证明 proposal graph 可以在短到中等长度 deterministic/PA 内保持边界。v3.8 应把测试焦点升级为：

- 长期稳定性：更长轮次里 graph signature 不漂。
- 多 pressure interference：多个 generic 小势力压力不会互相污染。
- 同开局差异：差异来自候选表达和排序，不来自隐形事实随机化。
- memory contamination：候选、传闻、拒绝、过期不会进入事实层。
- player comprehension：玩家能稳定理解“这不是世界事实”。

## v3.8 建议门槛

- deterministic：150 轮。
- Player Advocate：50 轮。
- live DeepSeek：否，除非用户另批。
- MiroFish：not_needed，触发真实 lore 即 blocking。
- WorldCore：最终裁决权不变。
- Save：无写入、无迁移、无版本 bump、无 `runFingerprint`。

## 结论

测试制度跟着开发在进化。v3.8 的关键不是增加炫技能力，而是让 proposal graph 在更像“长期游戏”的压力下仍然表现得像候选系统，而不是越界成为事实系统。
