日期：2026-05-24
状态：startup completed；等待用户审批 D-310

## 主题

runtime agent 门禁二次加固。

## 核心目标

v3.1 要把 v3.0 的设计门禁升级为更可执行、更难误开的准入制度：

- AgentProposal 必须有完整 provenance、authority、visibility、persistence、rollback、manual review 字段。
- WorldCore post-check 必须显式给出 approved_expression、rejected_violation、needs_user_decision、manual_review、future_gate_required。
- Player Advocate / live metadata 必须清楚记录是否调用 live DeepSeek；若调用，记录模型、样本、轮次、成本、报告路径。
- old-save / rollback / same-start difference 仍只能作为 report-only 证据，不写玩家存档。
- v3.2 rehearsal 只能在 v3.1 质量收束后进入。

## 开发原则

- Schema-first：先冻结 contract，再写 checker。
- Report-only：所有输出都是报告和合成样本，不进入 runtime。
- Boundary-first：每个 positive sample 至少有一个对应 negative sample。
- No hidden body：报告里不得出现真实 hidden/private body 或原文正文。
- WorldCore final authority：agent 只给候选，不裁决。
