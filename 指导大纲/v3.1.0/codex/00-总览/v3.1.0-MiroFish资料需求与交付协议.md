## 当前结论

v3.1 是 runtime agent 门禁二次加固，默认使用 synthetic/report-only boundary evidence，不需要 MiroFish export 或 intake。

## 保持 not_needed 的条件

以下条件全部满足时，MiroFish 保持 `not_needed`：

- 只讨论 AgentProposal contract、WorldCore post-check、rollback、old-save、PA/live metadata。
- 只使用合成 NPC、合成势力压力、合成 hidden marker。
- 不写知识库正文。
- 不做 runtime canon promotion。
- 不使用真实原著事实、命名 NPC、真实势力或 Fang Yuan 相关证据。

## 立即升级 blocking 的条件

触发以下任一项，必须停止并找用户决策：

- 命名 NPC 或原著关键人物进入样本。
- 真实势力、真实地点、正式 lore 结论进入样本。
- hidden-adjacent、方源公开证据、L4/L5 原著锚点进入 scope。
- 需要写知识库正文、runtime canon、DeepSeek visible lore/RAG。
- 需要从 `指导大纲/vMiroFish/基础包/` 提取 topic slice。

## 若未来升级 blocking

必须走：

1. RebornG request。
2. MiroFish quote-redacted export。
3. RebornG intake review。
4. rule draft / test matrix。
5. 用户审批。

MiroFish 输出只能进入 candidate/rule/test/source-pointer，不得直接进入 runtime、prompt、DeepSeek visible context、UI 或 canon。
