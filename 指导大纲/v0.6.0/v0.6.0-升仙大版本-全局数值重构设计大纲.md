### 1.1 原著六步 → 游戏三步流程

```

```

### 1.2 升仙难度公式（原著考证版）

**原著依据**：
- "积累越浑厚，升仙的难度就越大。但若是成功，日后的成就也就越高。"（太白云生升仙观战）
- "十绝体升仙，天劫地灾果然恐怖，是寻常蛊仙的数倍！" → "不是数倍，而是至少十倍。"（黑楼兰升仙）
- "十绝体升仙冒着巨大危险，一旦成功福地皆是特等福地"（>1000万亩，>50颗仙元，至少2只仙蛊）
- "僵尸升仙灾劫更恐怖，威力暴涨数十倍"（方源评估怜香仙子）
- 力道/变化道在冰原升仙→狂蛮魔尊真意共鸣→额外难度+额外奖励

**最终难度系数**：

```
难度系数 = 基础(1.0)
  × 资质系数（资质≤7→1.0, 8→1.3, 9→1.8, 10→3.0）
  × 积累系数（人气总量/100, 钳制1.0~2.0）
  × 十绝体乘数（资质=10时×3.0，累计×9.0）
  × 放蛊加成（1.0 + 放入蛊数量×0.5 + tier5蛊×1.0每只）
  × 状态修正（僵尸×5.0, 仙僵×8.0, 正常×1.0）
  × 地域修正（共鸣地域×1.5, 普通×1.0）
```

**难度系数影响**：
- 三气差值增长速率：`基础3%/回合 × 难度系数`
- 死亡阈值：`50% - 难度系数×5%`（十绝体约35%即刻致死）
- 灾劫类型：`<1.5→普通天劫, 1.5-2.5→大型灾劫, >2.5→十大凶灾级`

**资质-风险-收益对照表**：

| 资质 | 难度系数 | 灾劫类型 | 福地等级 | 额外仙蛊 |
|------|---------|---------|---------|---------|
| 3-5(丙丁) | 1.0~1.2 | 普通天劫 | 小福地(100-300万亩) | 0 |
| 6-7(乙等) | 1.0~1.5 | 普通~大型 | 小-中福地(200-600万) | 0-1 |
| 8-9(甲等) | 1.8~3.2 | 大型灾劫 | 中-上等福地(400-900万) | 1-2 |
| 10(十绝) | 9.0 | 十大凶灾级 | 特等福地(>1000万) | ≥2 |

#### 1.4.2 数值模型

| 参数 | 公式 | 说明 |
|------|------|------|
| 人气上限 | `资质×10 + 体魄×5 + 心智×5 + 道痕总量/10` | 原著：人气=蛊师自身积累总和 |
| 天气上限 | `人气上限 × (1.0 + 地脉加成)` | 地脉0→×1.0, 地脉丰沛→×1.3 |
| 地气上限 | 同上 | |
| 吸纳速率 | 天气=18, 地气=15, 人气=12 | |
| 最大操作次数 | `8 + floor(资质/2)` | 十绝体=13次, 普通=10次 |
| 三气差值 | `max(三比例) - min(三比例)` | |
| 死亡概率 | 差值≥致死线: `(差值-致死线)×2%/回合` | 最高70% |

**玩家操作**：

| 操作 | 效果 | 风险 | 策略含义 |
|------|------|------|---------|
| 多吸天气 | 天气-18, 人气-8, 地气+5 | 人气消耗 | 追求高等级福地 |
| 多吸地气 | 地气-15, 人气-8, 天气+5 | 同上 | 追求大面积 |
| 维持平衡 | 天气+地气各-11, 人气+10 | 安全慢 | 保底策略 |

**胜利/失败**：
- 胜利：操作次数用完 且 差值<25%
- 轻伤：差值30-35% → HP-30%, 福地降一等
- 重伤：差值35-50% → HP-60%, 回退五转巅峰
- 死亡：差值>致死线 → 游戏结束

**产出**：
- 福地面积：`100 + floor(总吸纳/人气上限 × 800)` 亩
- 洞天概率：上等福地+差值<10% → 50%
- 流速比：`5 + floor(总吸纳/人气上限 × 28)`
- 资源节点：`1 + floor(总吸纳/人气上限 × 5)`
- 十绝体特等福地：>1000万亩, >50仙元, ≥2仙蛊

### 1.5 第3步：放蛊+灾劫+仙窍开辟（AI叙事收束）

**放蛊子步骤**：
- 玩家从 `inventory` 选一只凡蛊放入气茧（可跳过）
- 升仙成功率：`tier/10 × (三气平衡度/100)`
- **仙蛊唯一规则检测**：放蛊前检查 `timeline-immortal-gu-state.json`（见第四节）
- 若目标蛊已被占用 → 阻止并提示"此仙蛊已为XXX所持，天地间不容第二只"
- 成功 → 凡蛊蜕变为仙蛊 → 写入占用表

**灾劫子步骤**：
- AI据三气量+放蛊选择生成灾劫叙事
- 十绝体/高难度系数 → 十大凶灾级（八重魔音劫、万兽劫等）
- 地域共鸣（如冰原力道升仙）→ 额外BOSS战（狂蛮魔尊真意外显）
- 玩家2-3个关键决策

**仙窍开辟子步骤**：
- AI生成开辟叙事
- 调用 `immortalSlice.initializeAperture()`
- 空窍→仙窍存储迁移
- 元石→仙元石兑换（1000:1）

## 五、宝黄天扩展（仙材市场 + 仙蛊残方拍卖 + 玩家上架）

> 设计原则：原著中宝黄天是蛊仙专属的完整交易所，蛊仙可买卖仙蛊、仙材、杀招、仙蛊方。当前仅实现四分之一（仙蛊拍卖）。

### 5.1 仙材市场（新增标签页）

在 `AuctionPanel.tsx` 中新增"仙材"标签页。从 `MATERIAL_GRADE_MAP` 筛选 `grade === '仙材'` 的条目，按流派分类展示。

| 属性 | 设定 |
|------|------|
| 刷新周期 | 每5回合自动刷新 |
| 每次上架 | 6-12种仙材（随机） |
| 流派偏向 | 每种主修流派对应仙材出现概率×2 |
| 价格 | `economy.json` 定价 × (0.8~1.5) 随机浮动 |
| 购买 | 消耗仙元石，仙材进入 `materialBag` |
| 库存 | 每项限量1-5份，售罄标记 |

### 5.2 仙蛊残方拍卖

从 `fragment-recipes.json` 筛选 `targetTier >= 6` 或 `sourceType === 'ruins'` 的古方，与仙蛊拍卖同池但独立展示。

| 属性 | 设定 |
|------|------|
| 上架数 | 每次拍卖会1-2个残方 |
| 起始价 | `targetTier × 500` 仙元（6转=3000, 9转=4500） |
| 竞价机制 | 复用现有NPC竞价逻辑 |
| 补全后 | 解锁对应仙蛊炼制配方 |

### 5.3 玩家上架仙蛊拍卖（替代固定回收）

**设计依据**：原著宝黄天是双向市场，蛊仙可在拍卖行寄售仙蛊。珍贵的仙蛊常引发哄抢热潮（如红玉散人拍卖会）。固定回购价做死了市场活力，对玩家不公平。

**上架流程**：

1. 玩家在 `AuctionPanel` →"上架"标签页→从仙窍库存中选择一只仙蛊
2. 设置起始价（不能低于底价）：`底价 = tier² × 100`（6转=3600, 7转=4900, 8转=6400, 9转=8100）

3. 拍卖开始：AI竞争者（3-6位虚拟蛊仙）参与竞价
   - **仙蛊稀有度修正**：`rank=legendary` → 竞争者+2，竞价积极度+30%；`rank=rare` → 竞争者-1
   - **流派热门度修正**：炎道/力道/剑道（热门战斗流派） → 竞争者+1，最终成交价×1.2
   - **仙蛊功能修正**：治疗/移动/侦察类 → 竞争者+1（泛用性高）
   - **竞价轮次**：5-8轮
   - **每轮涨幅**：当前价×5-20%（随机）

4. **成交**：最后一轮竞价×(`rarity系数：legendary 1.3, epic 1.1, rare 1.0`)
5. **手续费**：宝黄天抽取成交价10%作为佣金
6. **玩家到手** = 成交价 × 90%

**AI竞争者模拟**（复用 `auction-engine.ts` 的 `simulateNPCBids`，扩展参数兼容出售方向）：

```
竞争者数量 = 3 + tier-6 + rarityBonus + pathPopularityBonus
每轮：30%概率有竞争者退出 → 剩余竞争者继续出价
最后出价 ≈ 底价 × rarityMultiplier × pathMultiplier × competitionFactor
```

**极端情况**：竞争者全部退出→流拍。玩家可取回仙蛊（无损失）。

| **流血 Bleed** | 血道 | 受击时额外HP-8%×potency | 3 | 3层 |
| **混乱 Confuse** | 智道 | 50%概率随机选择招式 | 2 | 1层 |
| **失明 Blind** | 暗道 | 命中率-40% | 3 | 1层 |
| **虚弱 Weaken** | 力道 | DEF-20% | 3 | 2层 |

**状态触发**：杀招/蛊技附带状态概率（`move.statusChance: 0.0~0.5`），在伤害计算后独立判定。

**流互克对状态的影响**：克制流派的状态效果额外+1回合。

### 6.4 敌方AI系统

四种行为模式，按 `enemy.aggression` 字段切换：

| 模式 | 行为规则 | 适用敌人 |
|------|---------|---------|
| **aggressive** | 始终使用最高伤害招式，不防御 | 野兽/魔道狂徒/试炼傀儡 |
| **balanced** | HP>50%进攻→HP<50%攻防各半 | 正道蛊师/散修/佣兵 |
| **defensive** | HP>70%进攻→HP<70%优先防御+恢复 | 治疗型/辅助型敌人 |
| **coward** | HP<40%时50%概率逃跑 | 散修/弱敌/情报型NPC |

### 6.5 非致命收场

**训练KO**：道场/切磋中HP归零→判定为"被击倒"，HP恢复至1，战斗结束。不触发死亡。

**投降机制**：玩家/敌方可在行动中选择投降（消耗该回合），投降者退出战斗序列。

### 6.6 剧情触发的战斗类型

## 十、蛊虫/杀招影响力升级（剧情双向反馈修复）

> 审计结论：当前蛊虫→剧情为单向断裂（AI看到蛊虫但不会据此生成差异化选项），杀招→剧情为双向断裂（AI不授予杀招也看不到coreGu约束），蛊虫→战斗缺乏类型特异性。v0.6.0用最低成本修复三个断裂。

### 10.1 蛊虫→剧情：AI上下文引导（非硬编码门控）

在 `context-builder.ts` 的 `buildDynamicContext` 末尾追加一段**蛊虫能力解锁建议**：

```
【蛊虫能力 — 选项生成规则】
玩家当前拥有的核心蛊虫及其能力，每个选项至少应有一条关联到某种蛊虫能力：
- 月光蛊(光道/1转): 可用于远程侦察、夜间照明、发送光信号
- 毒液蛊(毒道/4转): 可用于暗中下毒、削弱敌人、恐吓
- 火鸦蛊(炎道/3转): 可用于远程火攻、点燃障碍、取暖
当玩家面临社交/战斗/探索场景时，应优先生成使用蛊虫能力的选项。
```

**不是硬编码门控**（太重且难维护），而是AI引导（轻量、灵活、与AI叙事能力自然融合）。AI据此生成"使用月光蛊侦察"等选项时自然带`risk`和`risk_note`。

### 10.2 蛊虫→战斗：类型特异性状态效果

v0.6.0战斗系统升级（第六节）中新增的8种状态效果已按流派分配。本轮追加：

- 每个蛊虫条目在 `gu-database.json` 中新增可选字段 `combatStatus?: { type: CombatStatusType; chance: number; potency: number }`
- 战斗中使用对应流派蛊虫的杀招时，按 `chance` 概率触发状态效果
- 默认值：凡蛊 `chance=0.15~0.25`, `potency=1`；仙蛊 `chance=0.3~0.5`, `potency=2`

### 10.3 杀招获取渠道（不包含AI叙事授予）

**原著设定**：杀招难自创（需宗师级以上）、可从传承/购买/交易获得、仙道杀招极为珍贵。

| 渠道 | v0.6.0实现 | 细节 |
|------|-----------|------|
| 自创 | ✅ 已有 `KillMoveCreationPanel` | 需流派境界≥大师 |
| NPC传授 | ✅ 已有（`teachKillMove`） | 需好感度+流派境界 |
| 战斗掉落 | ✅ 已有（残卷） | 强化掉落率 |
| 宝黄天残方 | ✅ v0.6.0新增 | 仙蛊残方拍卖（第五节） |
| 势力/传承 | 🔵 延至v0.7.0 | 与成就系统重做捆绑 |
| **AI叙事授予** | ❌ **明确不做** | 过激进，违背原著设定 |

### 13.2 最大风险

**战斗系统双引擎维护**：1v1引擎（`combat-engine.ts`）和小队引擎（`squad-combat-engine.ts`）并行可能导致伤害公式不一致。

**缓解**：提取共享模块 `src/engine/combat-formulas.ts`（`calculateDamage`, `applyDaoMarkMult`, `applyPathMatrix`）供两个引擎共用。1v1引擎的函数签名不变，内部调用共享模块。
