## 用户已批准的减法方向

这些是方向批准，不等于 runtime 已实施：

1. `行动` 与 `自由目标` 合并为一个一级入口。
2. `青茅/凡战/群像/大阵` 从正式底栏移入开发/演示抽屉。
3. `蛊虫/杀招/炼蛊/蛊材` 合并为 `蛊道`。
4. `传承/宿命/终局/商会` 做境界、剧情、场景化显示，不作为常驻一级按钮。
5. 正式 NPC/势力系统前，做旧系统与 `livingWorldState` 权威归并表。
6. 正式经济系统前，做 material/price/shop/refine 权威归并表。

## 当前阻塞与必须决策

当前 `v1.0.0-rc` 本地质量门已通过，待提交/远端 CI。若要形成对外发布承诺、自动部署、EdgeOne 正式发布、hero 正式绑定或生成大规模新图，必须停下来让用户决策。

v1.0 当前建议必须决策：

1. 是否批准 v1.0 定位为“活世界早期正式版”，不是完整蛊界。
2. 是否批准主线为“青茅山到南疆早期路线 + 低阶蛊师闭环 + 自由意图收束”。
3. 是否批准 a2 做 API key/hidden fact/public-private canon 威胁模型，但暂不实现 BFF/backend。
4. 是否允许 b1 做 route/location 持久字段设计门禁；若真正新增字段，再单独批准 `SAVE_FORMAT_VERSION = 23`。
5. 是否批准一次性请求 v1.0 三个 MiroFish 包。
6. 是否批准 v1.0 rc Player Advocate 提升到 150 轮以上。

进入 runtime 实施或扩大范围前，仍必须停下来让用户决策：

1. 新增或修改存档字段。
2. 扩大 DeepSeek 权限。
3. 开放正式奖励、地点、阵营、NPC 生死。
4. 引入 MiroFish 阻塞包或原著/IF 边界变化。
5. 做对外发布承诺。
6. 新增后端/BFF、外部运行时依赖或自动部署。

## 硬停规则

以下事项必须停下来让用户决策：

- 原著边界或 IF 边界变化。
- 新增/修改存档字段。
- DeepSeek 权限扩大。
- 正式奖励、地点、阵营、NPC 生死。
- MiroFish 阻塞包。
- 对外发布承诺。
