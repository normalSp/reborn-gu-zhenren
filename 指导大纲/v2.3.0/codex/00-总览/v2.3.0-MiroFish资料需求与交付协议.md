# v2.3.0 MiroFish 资料需求与交付协议

状态：v2.3 本地完成；当前分类 `not_needed`。
日期：2026-05-23

## 当前结论

v2.3 全程不需要新 MiroFish 包。a1 的 failure taxonomy 使用 synthetic refs 和 synthetic hidden marker 边界完成；a2 的 report archive 合同明确禁止保存真实 hidden/private body、MiroFish 原始输出或知识库正文；b1-b3 runner / samples / report 也只使用 synthetic fixtures。

理由：

- 当前主线是 Agent eval farm 与 failure taxonomy。
- 样本应使用 synthetic refs 和 v2.2 抽象 Agent Lab 场景。
- 不引入命名 NPC、真实隐藏事实、方源公开证据、正式 lore 结论或知识库正文。
- 负例可以模拟 hidden leak，但必须使用 synthetic marker，不使用真实 hidden/private body。
- 当前 artifact `artifacts/v2.3.0/agent-eval-farm/2026-05-23T09-10-30-682Z/report.json` 只包含 synthetic refs、failure family、severity、disposition 和 report-only evaluator verdict。

## 升级 blocking 的条件

一旦出现以下任一项，MiroFish 需求升级为 `blocking`：

- 命名原著 NPC 或具体原著势力进入样本。
- hidden-adjacent、方源公开证据、L4/L5 事实锚点进入样本。
- 需要判断某个原著人物意图、命运、关系或大势走向。
- 需要把样本晋升为知识库正文、runtime canon、DeepSeek visible summary/RAG 或玩家可见文本。
- 需要正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。

## 允许材料

- v2.1/v2.2 synthetic samples。
- v2.2 report-only runner 报告。
- 项目已有流程制度。
- 长期路线中的抽象 L0-L5 / WorldCore 架构。

## 禁止材料

- 全书 MiroFish 基础包直接导入。
- 原著正文或长摘录。
- hidden/private body。
- MiroFish 原始输出直接复制进 runtime、prompt、UI 或知识库正文。

## 当前协议

若后续需要 MiroFish：

`RebornG request -> MiroFish export -> RebornG intake review -> test matrix / rule draft`

在 intake review 通过前，不得把任何材料放入 runtime、DeepSeek visible context、UI 或正式 canon。
