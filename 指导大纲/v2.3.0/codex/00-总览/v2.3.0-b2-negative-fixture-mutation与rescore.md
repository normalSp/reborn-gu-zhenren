## false positive 处置

`V23-A1-EVAL-FP-001` 被保留为 deliberate false-positive calibration case：好样本因为 wording-only matcher 被过度阻断时，不直接算失败，而是进入 `falsePositiveReview`，建议 `calibrate_evaluator`。

这条样本的意义是防止 eval farm 走向“越严越好”的假安全。真正目标是 P0/P1 不漏判，同时控制误杀噪声。

## b2 通过意义

v2.3-b2 通过说明当前 eval farm 能抓住主要 synthetic 越权、隐藏、记忆、L5、WorldCore 绕过、目标丢失和势力压力停滞风险。它仍不证明真实 live DeepSeek、多 agent runtime 或完整 NPC 社会可开放。
