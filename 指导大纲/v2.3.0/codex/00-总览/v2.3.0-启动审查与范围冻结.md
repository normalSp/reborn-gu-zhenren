# v2.3.0 启动审查与范围冻结

状态：completed；D-230 已获用户批准，a1 已完成。
日期：2026-05-23

## 版本主题

`Agent eval farm 与失败分类硬化`

## 为什么现在做

v2.1 和 v2.2 已经把 Agent Lab 的 positive path 跑通。下一步最危险的不是 runner 不会跑，而是 runner 只会证明“好样本看起来很好”，却抓不住真实接入前最致命的失败：

- agent 越权写事实。
- hidden/private 内容泄露或 echo。
- `agent_private_candidate` 被误升为 canon/save/runtime truth。
- L5 宏观压力替玩家或世界宣判结局。
- NPC 长线目标丢失、空转、重复。
- 势力压力不响应公开行为。
- evaluator 对 P0/P1 false negative。

因此 v2.3 的启动边界是评估制度，不是 runtime 功能。

## 本阶段允许

- 新建 v2.3 文档入口。
- 复盘 v2.2 expanded offline runner。
- 起草 D-230 决策池。
- 起草 failure taxonomy、severity rubric、eval farm、trend archive 的路线。
- 更新项目仪表盘、PROJECT-STATE、AGENTS、handoff。
- 做 Skill sync audit。

## 本阶段不允许

- 不改 runtime/source/UI/store/prompt/save。
- 不新增 save 字段，不 bump `SAVE_FORMAT_VERSION`。
- 不新增 DeepSeek prompt/context/model/authority。
- 不调用 live DeepSeek。
- 不启用 subagents。
- 不引入第三方 agent SDK/framework 依赖或 PoC。
- 不接 BFF/backend。
- 不新增 MiroFish export/intake。
- 不写知识库正文。
- 不开放正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。
- 不新增 `runFingerprint`。
- 不自动部署 EdgeOne。

## MiroFish 分类

当前分类：`not_needed`。

理由：

- v2.3 startup/a0/a1/a2 默认只评估 synthetic AgentProposal、failure taxonomy、evaluator 边界和 report-only 归档。
- 不触碰命名 NPC、hidden-adjacent、方源公开证据、正式 lore 结论或 L4 原著关键人物。

升级为 `blocking` 的触发条件：

- 引入命名原著 NPC 或真实势力结论。
- 引入 hidden/private body、方源公开证据或 hidden-adjacent 样本。
- 让 L4/L5 绑定具体原著事实锚点。
- 任何样本需要进入知识库正文、runtime canon、DeepSeek visible lore/RAG 或玩家可见文本。

## Player Advocate / live probe

当前豁免：

- v2.3 startup/a0 是纯文档、启动审查和决策池。
- 无玩家可见变化。
- 无 runtime、prompt、DeepSeek context 或 UI 变化。

若后续 b1/b2 仍是 report-only/offline runner，可继续豁免 Player Advocate 和 live DeepSeek，但必须在对应阶段记录原因。
