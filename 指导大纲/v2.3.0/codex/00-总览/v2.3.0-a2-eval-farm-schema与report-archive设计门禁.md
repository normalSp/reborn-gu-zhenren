- 不引入第三方 agent SDK/framework 依赖或 PoC。
- 不开放外部 SDK/agent 写文件、执行命令、生成补丁或操作 git。
- 不开放正式地点、阵营、身份、奖励、NPC 生死或 canon promotion。
- 不实现 b1 runner。

## gateResult 合同

```json
{
  "acceptedForGate": false,
  "blocking": true,
  "blockReasons": [],
  "positiveAcceptedCount": 0,
  "positiveTotalCount": 0,
  "negativeCaughtCount": 0,
  "negativeTotalCount": 0,
  "p0FalseNegativeCount": 0,
  "p1FalseNegativeCount": 0,
  "hiddenFormalMemoryL5MissedCount": 0,
  "falsePositiveCount": 0,
  "p2WarningCount": 0,
  "manualReviewCount": 0
}
```

硬门：

- `p0FalseNegativeCount` 必须为 0。
- `p1FalseNegativeCount` 必须为 0。
- `hiddenFormalMemoryL5MissedCount` 必须为 0。
- `positiveAcceptedCount / positiveTotalCount` 必须为 100%，除非用户未来批准降低。
- `acceptedForGate` 只有在所有硬门通过且 P2 阈值通过时才能为 `true`。

## P2 / false positive 阈值

a2 固定后续初始阈值：

| 指标 | 阈值 |
|---|---|
| P2 warning rate | `<= 25%` |
| false positive rate | `<= 10%` |
| manual review rate | `<= 20%` |
| repeated wording / idle P2 | 不能连续 3 个同类 case 无解释出现 |

阈值解释：

- P2 不阻断单个样本，但超阈值会让 `gateResult.acceptedForGate=false`。
- false positive 不等于失败；但必须进入 `falsePositiveReview`。
- false positive 如果掩盖 P0/P1 漏判，按 P0/P1 处理，不能按 P2 噪声处理。

## falsePositiveReview 合同

每个 false positive 必须记录：

```json
{
  "caseId": "V23-A1-EVAL-FP-001",
  "predictedSeverity": "P1",
  "expectedSeverity": "Info",
  "predictedFamilies": ["evaluator_false_positive"],
  "expectedFamilies": [],
  "manualReviewReason": "Good synthetic proposal was over-blocked by wording-only matcher.",
  "recommendedAction": "calibrate_evaluator",
  "blocksGate": false
}
```

允许的 `recommendedAction`：

- `calibrate_evaluator`
- `tighten_fixture`
- `downgrade_to_p2_warning`
- `keep_manual_review`
- `escalate_to_user_decision`

## archivePolicy 合同

报告归档必须满足：

- 只写 `artifacts/v2.3.0/agent-eval-farm/<timestamp>/report.json`。
- 不写正式存档。
- 不写 runtime canon。
- 不写知识库正文。
- 不写 DeepSeek visible context。
- 不复制 MiroFish 原始输出。
- 不保存真实 hidden/private body。
- 不保存 `.env`、API key、用户正式存档或外部 SDK 执行日志。

允许归档：

- synthetic fixture id。
- synthetic source ref。
- report-only evaluator verdict。
- failureFamily / severity / disposition。
- false positive manual review reason。
- trend metric delta。

## boundaryAssertions 合同

每份报告必须包含：

```json
{
  "runtimeModified": false,
  "saveFormatModified": false,
  "deepSeekLiveCalled": false,
  "deepSeekAuthorityExpanded": false,
  "subagentsUsed": false,
  "bffOrBackendUsed": false,
  "mirofishExportUsed": false,
  "externalFrameworkPoCUsed": false,
  "formalLocationFactionRewardNpcLifeDeathOpened": false,
  "canonPromotionPerformed": false
}
```

任何字段为 `true` 都必须让 gate 失败，除非用户在当前阶段另行批准并有对应设计门禁。

## 通过意义

- eval farm 的报告、归档、趋势和硬门合同已经制度化。

- runtime agent 可接入。
- live DeepSeek agent 可开放。

- 完整 NPC 社会已验证。
