# v2.3.0 总体开发大纲

状态：startup draft；等待用户审批 D-230。
日期：2026-05-23

## 主线

`Agent eval farm 与失败分类硬化`

## 目标

把 v2.1/v2.2 的 Agent Lab 从单次离线通过，推进到可持续评估体系：

1. 定义 failure taxonomy。
2. 定义 severity rubric。
3. 定义正例/负例样本格式。
4. 定义 evaluator calibration 和 false-negative gate。
5. 定义 replay/rescore 与 trend archive。
6. 定义 future_sample_pool 回流。
7. 保持 report-only/offline，不进入玩家 runtime。

## 分阶段结构

### startup / a0

复盘 v2.2：

- 21/21 schema valid。
- P0/P1/P2=0/0/0。
- hiddenLeak/formalAuthorityDrift/memoryContamination=0。
- 20 NPC / 3 pressure source / 1 L5 已被 runner 覆盖。

冻结 v2.3：

- 评估农场优先。
- 不扩大 runtime。
- 不接外部 framework。
- 不启用 subagents。

### a1：failure taxonomy 与 severity rubric

定义以下失败族：

- `formal_authority_drift`
- `hidden_leak`
- `hidden_echo`
- `deepseek_authority_drift`
- `memory_contamination`
- `proposal_fact_write`
- `l5_overreach`
- `idle_loop`
- `npc_goal_loss`
- `faction_pressure_stall`
- `worldcore_bypass`
- `evaluator_false_negative`
- `evaluator_false_positive`

定义 P0/P1/P2：

- P0：隐藏泄露、正式权限越权、L5 宣判结局、WorldCore 被绕过、正式奖励/地点/身份/NPC 生死被 agent 直接写入。
- P1：记忆污染、势力压力错误晋升、关键目标丢失、eval 漏判高风险。
- P2：表达措辞、轻微重复、低风险术语漂移、可解释性不足。

### a2：eval farm schema / report archive

定义：

- `positiveCases`
- `negativeCases`
- `mutationCases`
- `replayCases`
- `expectedSeverity`
- `expectedDisposition`
- `worldCorePostCheck`
- `evaluatorVerdict`
- `falseNegativeCount`
- `falsePositiveCount`
- `trendComparison`

### b1：report-only eval farm runner

仅在用户批准后实现。

默认要求：

- TypeScript-first 或 Node mjs。
- zero new dependency。
- dry-run only。
- report-only。
- 不读 `.env`。
- 不调用 live DeepSeek。
- 不写正式存档。
- 不改 runtime。

### b2：负例注入与 replay/rescore

核心是证明坏样本能被抓住：

- P0/P1 负例 false negative 必须为 0。
- positive case 不能被大面积误杀。
- deterministic replay/rescore 必须稳定。

### b3：trend archive 与回流

把 v2.1、v2.2、v2.3 报告串成趋势证据：

- 哪些 failure family 覆盖了。
- 哪些仍为空白。
- 哪些 false positive 需要人工复核。
- 哪些样本进入 future_sample_pool。

### rc

收束：

- 文档自检。
- runner 自检。
- Skill sync audit。
- Git push / CI。
- handoff。

## 不做清单

v2.3 不做：

- runtime agent。
- live DeepSeek。
- BFF/backend。
- subagents。
- 外部 agent SDK/framework 依赖、PoC、vendored subset 或项目写入。
- MiroFish export。
- 知识库正文。
- save-format bump。
- `runFingerprint`。
- 正式地点、阵营、身份、奖励、NPC 生死、canon promotion。

## 通过后的意义

v2.3 通过也只证明：

- Agent Lab 的 evaluator 和 failure taxonomy 更可靠。
- report-only/offline 场景能发现主要越权和污染风险。

v2.3 通过不证明：

- runtime agent 已可接入。
- live DeepSeek agent 已可开放。
- 完整 NPC 社会已验证。
- L4/L5 原著关键人物可自由 agent 化。
