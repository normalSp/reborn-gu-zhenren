# v2.3.0 专家团启动会纪要

状态：completed；D-230 已获用户批准，a1 已完成。
日期：2026-05-23
主题：Agent eval farm 与失败分类硬化。

## 开会背景

v2.1 完成了 Agent Lab 启动版：

- Agent 输出统一为 `AgentProposal`。
- WorldCore post-check 保持最终裁决。
- 自有 zero-dependency runner 能做 dry-run/report-only 审计。
- 10 个合成样本通过，P0/P1/P2=0/0/0。

v2.2 完成了扩展离线模拟第一阶段：

- 场景 `outer_edge_agent_lab_synthetic_v1`。
- 20 NPC / 3 抽象 pressure source / 1 个 L5 `heaven_will_pressure`。
- expanded offline runner 通过，21/21 schema valid，P0/P1/P2=0/0/0。
- hiddenLeak / formalAuthorityDrift / memoryContamination 均为 0。

专家团判断：这证明 Agent Lab 可以跑，但还没有证明 Agent Lab 能长期发现失败。v2.3 的价值不是继续扩大样本数量，而是建立评估农场，让失败能被稳定复现、分类、归档和回流。

## 专家团总建议

推荐 v2.3 主线：

`Agent eval farm 与失败分类硬化`

一句话目标：

> 让 Agent Lab 从“能跑通”升级为“能抓错、能分级、能复测、能比较趋势”，为 v2.4+ 基础设施评估和 v3.0 runtime agent 准入打地基。

## v2.3 好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| 正例/负例双轨 eval | 不只证明好样本能过，也证明坏样本会被拒 | 负例必须 synthetic，不放真实 hidden body |
| false-negative gate | 防止 evaluator 漏掉 P0/P1 | P0/P1 false negative 必须 blocking |
| failure taxonomy 版本化 | 让后续 v2.4-v3.0 比较趋势 | 不把 taxonomy 写成 runtime 权限 |
| deterministic mutation | 复用同一场景生成边界变体 | 不引入随机种子存档或 `runFingerprint` |
| trend archive | 比较 v2.1/v2.2/v2.3 的质量演化 | report-only artifact，不是正式存档 |
| evaluator calibration notes | 明确 false positive / false negative 的处理 | 不让 evaluator 自动改 canon 或 runtime |

## 不推荐现在做

- 不接 runtime agent。
- 不运行 live DeepSeek。
- 不启用 subagents。
- 不做外部 framework PoC。
- 不接 BFF/backend。
- 不新增 save 字段或 `SAVE_FORMAT_VERSION`。
- 不写知识库正文。
- 不新增 MiroFish export。
- 不触碰命名原著 NPC、hidden-adjacent、方源公开证据或正式 lore 结论。
