# v0.16.0 小版本执行路线图

日期：2026-05-18
状态：已完成

## 总路线

v0.16 先设计门禁，再做玩家可见入口合并，最后做权威归并验证和 rc 收束。

## 小版本路线

| 小版本 | 名称 | 核心目标 | 主要文件范围 | 验收 |
|---|---|---|---|---|
| `process-0` | 启动前置与范围冻结 | 建立 v0.16 文档包，记录用户批准，更新仪表盘和路线 | docs/governance only | 文档完整、diff check、Git 状态记录 |
| `a1` | 信息架构与权威表设计门禁 | 冻结主导航、入口去向、旧系统权威表、测试矩阵 | docs first | 第一版完成；不改 runtime，用户可审 |
| `a2` | 仪表盘与流程入口收束 | 确认根目录仪表盘、AGENTS、PROJECT-STATE、skill/handoff 引用 | docs/governance | 后续只更新根目录仪表盘 |
| `b1` | `行动 + 自由目标` 合并第一刀 | 玩家主入口合并，行动语义统一，底层权威不扩大 | GameScreen/ActionPanel/FreeGoalPanel/tests/docs | focused tests、e2e、20 轮 Player Advocate |
| `b2` | `蛊道` 工作台第一刀 | 蛊虫、杀招、炼蛊、蛊材入口归并，候选/事实边界清楚 | Gu/Refine/Material UI/tests/docs | focused tests、e2e、20 轮 Player Advocate |
| `b3` | 高阶/调试入口场景化 | 演示入口移出正式底栏，高阶入口按条件显示 | GameScreen/navigation/tests/docs | e2e、20 轮 Player Advocate |
| `b4` | 权威归并验证 | NPC/势力、经济/material/shop/refine authority map 和防重复测试 | docs + possible helper/tests | 不新增 save/DeepSeek 权限；10-20 轮视改动而定 |
| `rc` | 质量收束 | 全量质量门、长测、Player Advocate、Git/CI 收束 | docs/tests | full gate + 60 轮 Player Advocate |

## 阶段顺序原则

1. 不先动 runtime，先完成 a1 设计门禁。
2. 不先做视觉重皮，先让入口和权威清楚。
3. 不先做战斗深化，先把战斗入口从调试/演示逻辑迁入正式行动/蛊道结构。
4. 不先做正式经济/市场，先确认 v0.15 经济边界和旧系统权威。
5. 不先新增存档字段，除非用户单独批准。

## 每阶段必须更新

- 对应阶段记录文档。
- `指导大纲/项目仪表盘.md`。
- 测试矩阵或测试豁免说明。
- Git 状态和 handoff。
- 如触及玩家可见 runtime，Player Advocate 记录和 `check:player-advocate-gate`。
