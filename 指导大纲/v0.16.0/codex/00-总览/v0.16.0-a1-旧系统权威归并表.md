# v0.16.0-a1 旧系统权威归并表

日期：2026-05-18
状态：第一版

## 目的

v0.16 做 UI 减法时，不能只移动按钮。旧系统和 v0.11-v0.15 的活世界系统已经同时存在，如果不先定权威，后续会出现：

- 同一 NPC 态度由两个系统同时解释。
- 同一材料来源由商店、蛊材袋、低阶经济候选同时私算。
- 同一行动既像正式任务，又像自由目标候选。
- UI 把只读预览包装成已经发生的世界事实。

本表只定权威，不迁移数据，不新增存档字段。

## 总权威原则

| 层 | 可以做什么 | 不能做什么 |
|---|---|---|
| UI | 分组、折叠、隐藏、说明候选/只读/未开放状态 | 私自发奖励、改地点、改阵营、决定 NPC 生死 |
| Store actions | 通过已批准 action protocol 写现有字段 | 绕过 controlled patch 或写未批准持久字段 |
| Local engine/canon | 决定规则、候选、前置、边界、正式本地结算 | 未经 MiroFish/用户边界新增原著硬事实 |
| DeepSeek | 写叙事、候选、线索、传闻、压力文本 | 写正式奖励、地点、阵营、NPC 生死、正史事实、数值结算 |
| MiroFish | 提供 quote-redacted 候选材料 | 直接成为 runtime truth 或 DeepSeek authority |

## 行动与自由目标

| 系统/文件域 | 当前责任 | v0.16 权威归属 | b 阶段处理 |
|---|---|---|---|
| `ActionPanel` | 当前行动、青茅区域行动、资源 loop、战斗候选 | `行动 / 当前行动` | b1 合并入口，不扩大写权限 |
| `FreeGoalPanel` | 自由目标裁决、路线/交易/社会候选、行动后续 | `行动 / 自由目标/线索后续` | b1 合并入口，候选/正式前置要更清楚 |
| World Intent Engine | 玩家自由输入的本地裁决和 DeepSeek contract | 入口裁决器/路由器 | 不能替代战斗、经济、奖励、地点权威 |
| `livingWorldState.playerGoals` | 确认后的目标草案 | 目标账本 | 不等于正式任务或成功事实 |
| local action ledger | 本地行动回流依据 | 行动账本 | 记录已执行本地动作，不由 DeepSeek 补写 |

## NPC 与势力

| 系统/字段 | 当前风险 | v0.16 权威归属 | b4 验证方向 |
|---|---|---|---|
| `npcRelations` | 旧关系矩阵可能与新记忆投影重复 | 旧系统历史/数值关系来源，未升级前不决定新活世界公开反应 | 写测试或 helper 防止同一反应双重解释 |
| `livingWorldState.npcMemories` | 新公开行动记忆和候选印象 | v0.13 后的公开社会影响来源 | 只读/候选/公开痕迹，不开放命运结论 |
| `playerFaction` | 旧阵营身份可能被误当作新招揽结果 | 当前身份字段，正式转移需用户决策和后续版本 | v0.16 不新增阵营转移 |
| `factionEvents` | 旧势力事件可能和 factionPressure 重叠 | 历史事件记录 | 不直接产生正式声望、通缉、招揽 |
| `livingWorldState.factionPressure` | 新压力、接触窗口、公开反应 | 候选/压力/前置来源 | 不写正式 standing/warrant/recruitment |
| `actionConsequences` | 行动后果摘要 | 公开行动回流依据 | 不私自决定 NPC 生死、捕获、路线成功 |

## 经济、材料、商店、炼蛊

| 系统/字段 | 当前风险 | v0.16 权威归属 | b4 验证方向 |
|---|---|---|---|
| `materialBag` | 蛊材袋可能被 UI/商店/炼蛊重复消耗或发放 | 已拥有材料的展示和既有状态来源 | v0.16 不新增正式消耗 |
| v0.15 supply/feeding engines | 低阶经济候选、缺口、边界 | 候选/前置/提示来源 | 不变成正式库存或价格表 |
| `shopState` | 旧商店库存容易暗示正式市场已开放 | 历史/旧系统状态 | b3 不把商会常驻一级化 |
| `materialShelf` | 旧货架/材料展示可能与 materialBag 重复 | 展示/历史来源 | b4 防重复来源 |
| `auctionItems` | 高阶交易感过强 | 高阶/场景化/未开放 | 一转青茅开局不常驻 |
| `RefinePanel` | 容易被理解为正式炼蛊成功 | 炼蛊 UI/边界展示 | 不新增蛊方成功、材料消耗、奖励蛊 |
| `MerchantPanel` | 容易被理解为正式商会/市场系统 | 场景化入口或旧系统入口 | 不开放正式价格表、库存、交易结算 |

## b4 验证问题清单

1. 是否存在一个玩家行为同时被旧关系矩阵和新 `npcMemories` 判成两个不同社会事实？
2. 是否存在一个阵营目标被 `playerFaction`、`factionPressure`、prerequisite preview 同时解释成不同状态？
3. 是否存在一个材料由 `materialBag`、`materialShelf`、shop、v0.15 candidate 同时作为正式来源？
4. 是否存在一个炼蛊入口让玩家误以为已经能正式消耗材料并获得蛊？
5. 是否存在一个商会入口让玩家误以为正式市场已开放？
6. 是否存在一个演武入口让玩家误以为剧情战斗已正式触发？

## a1 冻结结论

v0.16 的 UI 合并必须是“入口归并”，不是“权威迁移”。任何权威迁移都必须在对应小版本单独记录、测试，并在触及存档/DeepSeek/正式世界事实时停下来让用户决策。
