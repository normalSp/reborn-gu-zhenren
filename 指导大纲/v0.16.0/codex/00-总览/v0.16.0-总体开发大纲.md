# v0.16.0 总体开发大纲

日期：2026-05-18
状态：已完成
主题：`系统收束、UI 减法与权威归并`

## 目标

让 RebornG 从“功能很多的开发工具台”收束成“玩家能理解、能行动、能感到世界反馈”的游戏界面和系统结构。

v0.16 的核心不是新增大玩法，而是把 v0.9 到 v0.15 累积出来的系统重新排好：

- 玩家想做事时，优先进入统一 `行动` 工作台。
- 蛊虫、杀招、炼蛊、蛊材归并到 `蛊道` 工作台。
- 调试/演示入口迁出正式底栏。
- 高阶入口按境界、剧情或场景出现，不常驻压迫低阶体验。
- 旧 NPC/势力、旧经济/材料/商店/炼蛊系统与 `livingWorldState`、v0.15 低阶经济明确权威分工。

## 版本价值

没有 v0.16，后续 v0.17 战斗、v0.18 南疆路线、v0.19 发布工具会继续往现有底栏和面板里堆入口。短期看开发快，长期会出现：

- 玩家找不到行动入口。
- 同一事实由多个系统解释。
- 候选/只读/未开放内容被玩家误认为已发生事实。
- 高阶系统过早暴露，破坏青茅低阶蛊师体验。
- 每新增一个系统都要补一次 UI 整理。

v0.16 是给 v1.0 前的玩家体验和工程秩序打地基。

## 核心交付

1. 主导航减法方案与冻结。
2. `行动 + 自由目标` 合并设计与第一刀。
3. `青茅 / 凡战 / 群像 / 大阵` 迁出正式底栏。
4. `蛊虫 / 杀招 / 炼蛊 / 蛊材` 归并为 `蛊道`。
5. `传承 / 宿命 / 终局 / 商会` 的境界、剧情、场景化显示规则。
6. 旧 NPC/势力系统与 `livingWorldState` 权威归并表。
7. 旧 material/shop/refine/economy 系统与 v0.15 低阶经济权威归并表。
8. UI 减法测试矩阵、Player Advocate 走查、rc 质量收束。

## 设计边界

### 可做

- 改主导航信息架构。
- 移动/合并玩家入口。
- 增加入口条件、开发抽屉、只读说明和 UI 可读性测试。
- 写旧系统权威表，必要时补只读 helper 或测试。
- 更新文档、仪表盘、Git 计划、技能提醒。

### 不做

- 新增正式地点、阵营、奖励、NPC 生死。
- 新增或修改存档字段。
- 把 DeepSeek 权限扩大到事实、奖励、地点、阵营、数值或结局。
- 新增原著事实或隐藏事实 runtime 权威。
- 以“减法”为名大规模重构 engine/store 目录。
- v0.16 内打开完整南疆路线、完整市场、正式黑市、正式委托或蛊仙期。

## 完成标志

- 正式玩家底栏不再暴露大量主按钮。
- `行动` 工作台能承接当前行动、自由目标、线索后续、路线/交易/社会机会。
- `蛊道` 工作台承接蛊虫、杀招、炼蛊、蛊材。
- 调试/演示入口与正式玩家入口隔离。
- 高阶入口不会在一转青茅山开局常驻误导玩家。
- 旧 NPC/势力、旧经济系统与新活世界/低阶经济权威分工明确。
- Player Advocate 能证明玩家知道下一步去哪、能做什么、哪些只是候选/预览。

## 风险

- 如果只藏按钮，不定权威，后续仍会重复膨胀。
- 如果一次性重写 UI，会拖慢 v1.0 主线且引入大量布局回归。
- 如果把候选内容做得像正式事实，会破坏隐藏事实和玩家信任。
- 如果高阶入口过早显眼，会冲淡低阶蛊师阶段的压迫感。

## 完成结果

`v0.16.0` 已按本大纲完成：

- 主导航实际落地为 `地图 / 行动 / 角色 / 蛊道 / 世界 / 记录`。
- `行动 + 自由目标`、`蛊虫/杀招/炼蛊/蛊材`、高阶/演武入口归并均已实现。
- 旧入口 e2e 已迁移到新工作台路径。
- `src/engine/v016-authority-map.ts` 和测试完成权威归并验证。
- b1/b2/b3 20 轮 Player Advocate、rc 60 轮 Player Advocate 均通过。
- 全量单测、类型、构建、扫描、完整 e2e、长测、production preview 均通过。

下一步停下来让用户决策 `v0.17.0` 启动审查与范围冻结。
