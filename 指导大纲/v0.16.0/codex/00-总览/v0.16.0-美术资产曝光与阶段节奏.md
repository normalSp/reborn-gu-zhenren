# v0.16.0 美术资产曝光与阶段节奏

日期：2026-05-18
状态：第一版

## 结论

美术资产不建议等到 v1.0 发布前才集中处理，也不建议 v0.16 立刻大规模生成或替换。

v0.16 的美术任务是治理曝光位置和权威状态：

- 哪些资产能进入正式 runtime UI。
- 哪些资产只能做 review-only。
- 哪些资产是 style-sample。
- 哪些资产是 v1.0 hero/release 候选。
- UI 减法后哪些资产不应通过旧入口继续曝光。

## 阶段节奏

| 阶段 | 美术工作 | 目的 |
|---|---|---|
| v0.16 | 美术资产权威与显示位置治理 | 避免 UI 减法后资产入口混乱 |
| v0.17 | 战斗/蛊道资产第一批正式化 | 跟战斗和蛊道工作台一起做，不凭空生成 |
| v0.18 | 南疆路线/区域气氛资产 | 跟路线和地域承接一起做 |
| v0.19 | v1.0 release art pack | hero 三件套、宣传截图、OG、短录屏、发布页素材 |
| v1.0 rc | 修补和替换 | 不做大规模新增，降低发布前风险 |

## v0.16 只做治理

可做：

- 检查 `public/rebrng` 与 `doc/art` 中资产状态。
- 确认 `runtime_active`、`candidate`、`review-only`、`style-sample`、`selected_candidate` 的显示规则。
- 确认 v1.0 hero 三件套仍为 release 候选，不自动绑定 UI。
- 确认战斗图、蛊虫图、角色图是否应该在新 `行动` / `蛊道` / `世界` 结构中出现。

不做：

- 不批量生成新图。
- 不全局替换 UI 背景。
- 不把 review-only 图误接入 runtime。
- 不把 specific_scene_only 图当泛用图。
- 不把高阶/未开放事实通过图片暗示给玩家。

## 美术边界

v0.16 仍遵守现有美术边界：

- 可以强化暗墨、金色道痕、血色风险、玉色生机、阵纹、窍壁、符箓。
- 不能暗示玩家已经获得仙蛊、永生、十转、宿命蛊、宝黄天正式交易等未开放事实。
- 方源、方正、白凝冰等角色图如未进入 runtime 事实链，只能作为候选或 review-only。
- 战斗图必须匹配具体场景和事实，不用作泛用背景。

## 验收

v0.16 中如果触及美术资产曝光，必须：

- 更新 asset manifest 或对应美术文档。
- 通过 `npm run check:qingmao-assets` 和 `npm run check:runtime-assets`。
- 如进入玩家 UI，补 e2e 或截图证据。
- 在 `指导大纲/项目仪表盘.md` 记录状态。
