# v0.16.0 MiroFish 资料需求与交付协议

日期：2026-05-18
状态：第一版

## 当前判定

v0.16 当前主线是 `系统收束、UI 减法与权威归并`，主线 blocking MiroFish need 为：

`not_needed`

原因：

- 不新增原著事实。
- 不新增隐藏事实。
- 不新增 NPC/势力规则事实。
- 不新增路线事实。
- 不新增 IF 规则。
- 不把 MiroFish 输出并入 runtime。

但为了降低 b2/b3 的术语和高阶边界返工，已建立两个 preferred request：

| request | 输出名 | 用途 | 是否阻塞 |
|---|---|---|---|
| `指导大纲/vMiroFish/requests/v0.16.0/2026-05-18-v016-gu-dao-low-rank-ui-boundary-pack.md` | `v016_gu_dao_low_rank_ui_boundary_pack_export_ready.json` | `蛊道` 工作台术语、低阶可见性、候选/正式边界和禁止暗示样本 | 不阻塞 a1/b1；建议 b2 前交付 |
| `指导大纲/vMiroFish/requests/v0.16.0/2026-05-18-v016-high-tier-scene-gating-boundary-pack.md` | `v016_high_tier_scene_gating_boundary_pack_export_ready.json` | `传承 / 宿命 / 终局 / 商会` 与演示入口的场景化显示边界 | 不阻塞 a1/b1/b2；建议 b3 前交付 |

现有 `2026-05-18-v016-battle-scene-composition-anchor-pack.md` 属于美术构图 optional/art 包，不阻塞 v0.16 主线。

## 需要重新判定的情况

如果任何小版本出现以下变更，必须停下重判：

| 触发 | 默认 need |
|---|---|
| 新增 canon fact / fact card | `blocking` 或 `preferred` |
| 新增 hidden fact / hidden boundary | `blocking` |
| 新增 NPC motive / named NPC rule | `preferred` 或 `blocking` |
| 新增 faction pressure / route pressure | `preferred` |
| 新增 Fang Yuan public evidence | `blocking` |
| 新增 route/supply/pursuit 原著事实 | `preferred` 或 `blocking` |
| 新增 IF branch rule | `preferred` |
| 仅移动 UI 入口、合并面板、写 authority map | `not_needed` |

## 交付处理规则

如后续确实需要 MiroFish：

1. 先在 `指导大纲/vMiroFish/requests/v0.16.0/` 写 request。
2. 用户转交 MiroFish 线程。
3. 交付后在 `指导大纲/vMiroFish/intake-reviews/v0.16.0/` 写 intake review。
4. 只能进入 `candidate_pool`、`fact_card_draft`、`rule_draft`、`test_sample` 或 `deferred`。
5. 不可直接成为 runtime truth、DeepSeek authority、player-visible hidden body 或正式奖励/地点/阵营/NPC 生死依据。

## 当前结论

v0.16-a1/a2/b1 默认不需要 MiroFish。

v0.16-b2/b3 可以在 preferred 包未交付时继续保守推进，但不得新增原著事实、隐藏事实或高阶入口文案；若需要改写术语、开放边界或加入原著依据，必须先等待对应 MiroFish 包并完成 intake review。

v0.16-b4 默认复用 v0.13/v0.15 已通过 intake 的 NPC/势力与经济包，不新增专门 MiroFish 包，除非 authority map 发现新的原著事实缺口。
