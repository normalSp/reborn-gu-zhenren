# v0.16.0-b4 权威归并验证

日期：2026-05-18
状态：已完成
类型：engine helper + tests/docs

## 目标

用可测试的权威表验证 v0.16 的 UI 减法没有把旧系统权力混在一起，尤其是行动、自由目标、NPC/势力、经济、炼蛊、战斗演示、高阶系统和 DeepSeek 权限。

## 实现

- 新增 `src/engine/v016-authority-map.ts`。
- 新增 `src/engine/v016-authority-map.test.ts`。
- 权威表包含：
  - v0.16 一级导航列表。
  - 旧入口到新工作台的归并关系。
  - 每个系统的 owner、allowed、forbiddenInV016。

## 关键验证点

- 一级导航固定为 `map / actions / role / gu_dao / world / records`。
- `free_goal` 归入 `actions`，模式为 `hub_tab`。
- `refine/material_bag/gu_inventory/kill_moves` 归入 `gu_dao`。
- `qingmao_demo/mortal_battle_demo/group_battle_demo/large_group_battle_demo` 归入 `world` 的 `dev_drawer`。
- DeepSeek forbidden 明确包含奖励、地点、阵营、NPC 生死、正史事实。
- 蛊道/经济 forbidden 明确包含 UI 私算成功、材料消耗、发放蛊虫或正式价格表。

## 测试证据

- `npm test -- src/engine/v016-authority-map.test.ts --reporter=dot`：通过。
- `npx tsc --noEmit --pretty false`：通过。

## Player Advocate

本阶段以权威表和测试为主，未新增玩家行为。玩家侧体验已由 b1/b2/b3 与 rc 走查覆盖。

## 后续

v0.17 开始做战斗、杀招、小队和阵法深化时，必须先查本权威表，避免把演武/调试入口误升级为正式战斗触发。
