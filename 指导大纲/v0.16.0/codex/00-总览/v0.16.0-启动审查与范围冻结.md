# v0.16.0 启动审查与范围冻结

日期：2026-05-18
状态：已完成；v0.16 范围已按冻结内容执行
主题：`系统收束、UI 减法与权威归并`

## 启动背景

用户审阅 `RebornG-v0.6到v0.15系统回顾与减法审查.md` 后，批准了 UI 减法和权威归并方向，并继续批准 `RebornG-v0.16至v1.0减法专项与路线调整草案.md` 第 9 节的 6 项路线决策。

v0.16 的定位已经从“战斗深化”调整为“系统收束、UI 减法与权威归并”。战斗深化顺延到 v0.17，南疆路线承接顺延到 v0.18，v1.0 发布工具放到 v0.19，蛊仙期推迟到 v1.0 之后。

## 已批准路线决策

1. 批准 `v0.16` 主线改为 `系统收束、UI 减法与权威归并`。
2. 批准 `v0.17` 顺延为 `战斗、杀招、小队与阵法深化`。
3. 批准 `v0.18` 定为 `南疆路线与多区域承接`。
4. 批准 `v0.19` 定为 `v1.0 发布工具、长测和内容生产准备`。
5. 批准蛊仙期推迟到 v1.0 之后。
6. 批准美术资产按 `v0.16 治理 -> v0.17 战斗 -> v0.18 区域 -> v0.19 release pack` 节奏推进。

## 已批准 v0.16 方向

- `行动` 与 `自由目标` 合并为一个一级入口。
- `青茅 / 凡战 / 群像 / 大阵` 从正式底栏移入开发/演示抽屉。
- `蛊虫 / 杀招 / 炼蛊 / 蛊材` 合并为 `蛊道`。
- `传承 / 宿命 / 终局 / 商会` 做境界、剧情或场景化显示，不作为常驻一级按钮。
- 正式 NPC/势力系统前，做旧系统与 `livingWorldState` 的权威归并表。
- 正式经济系统前，做 material/price/shop/refine 权威归并表。

## 范围冻结

### 本版本做

1. 信息架构：正式主导航目标、入口归并、旧入口去向。
2. 行动工作台：把 `行动` 与 `自由目标` 的玩家语义合并。
3. 蛊道工作台：把蛊虫、杀招、炼蛊、蛊材的玩家语义合并。
4. 场景化入口：调试入口与高阶入口从正式底栏中移出或按条件显示。
5. 权威归并：旧 NPC/势力系统、旧经济/材料/商店/炼蛊系统的 authority map。
6. 测试与走查：UI 可读性、入口不重复、候选/事实不混淆、Player Advocate。
7. 文档治理：根目录 `项目仪表盘.md`、Git 计划、handoff、skill 提醒保持同步。

### 本版本不做

1. 不正式开放南疆路线、完整商家城、完整市场或完整黑市。
2. 不新增正式奖励、材料、元石、蛊虫、蛊方、地点、阵营或 NPC 生死结论。
3. 不新增持久字段，不 bump `SAVE_FORMAT_VERSION`。
4. 不扩大 DeepSeek 权限。
5. 不新增原著事实或隐藏事实 runtime 权威。
6. 不做大规模 engine/store 目录重组。
7. 不做完整新美术生产或全局重皮。

## MiroFish need

当前启动前置和 a1/a2 文档门禁为 `not_needed`。

如果 b 阶段只做 UI 入口归并和既有系统权威表，仍为 `not_needed`。

如果任何阶段新增 canon 事实、NPC/势力规则、路线事实、隐藏事实或 IF 规则，必须停下来重判 MiroFish need。

## 存档影响

启动前置阶段：无存档影响。

v0.16 默认不新增持久字段。若任何小版本必须新增字段，必须停下来让用户批准，并在同一变更中提供 migration/defaults/tests。

## DeepSeek 权限影响

启动前置阶段：无 DeepSeek 权限变更。

v0.16 默认只改变 UI/文档/权威边界，不让 DeepSeek 新增奖励、地点、阵营、NPC 生死、正史或数值写入权。
