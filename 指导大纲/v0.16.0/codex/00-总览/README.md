# RebornG v0.16.0 Codex 当前入口

日期：2026-05-18
状态：`v0.16.0` 已完成为本地开发里程碑，下一步停下来讨论 `v0.17.0` 启动审查
主题：`系统收束、UI 减法与权威归并`

## 定位

`v0.16.0` 承接 `v0.15.0 低阶蛊师经济、补给、炼养用深循环`。v0.11 到 v0.15 已经让 RebornG 有自由目标、正史/IF、NPC/势力投影、路线承接和低阶经济边界；现在的问题不是缺功能，而是入口太多、旧系统和新活世界系统权威重叠、玩家主界面更像工具台。

v0.16 的目标不是重皮 UI，也不是删除系统，而是把已有功能收束成更少、更清楚、更符合玩家心智的入口：

`玩家意图 -> 行动工作台 -> 本地裁决/候选/前置 -> 统一回流 -> 记录/世界/蛊道分区`

## 用户已批准的路线决策

1. `v0.16` 主线改为 `系统收束、UI 减法与权威归并`。
2. `v0.17` 顺延为 `战斗、杀招、小队与阵法深化`。
3. `v0.18` 定为 `南疆路线与多区域承接`。
4. `v0.19` 定为 `v1.0 发布工具、长测和内容生产准备`。
5. 蛊仙期推迟到 v1.0 之后。
6. 美术资产按 `v0.16 治理 -> v0.17 战斗 -> v0.18 区域 -> v0.19 release pack` 节奏推进。

## v0.16 已完成的减法方向

- `行动` 与 `自由目标` 合并为一个一级入口。
- `青茅 / 凡战 / 群像 / 大阵` 从正式底栏移入开发/演示抽屉。
- `蛊虫 / 杀招 / 炼蛊 / 蛊材` 合并为 `蛊道`。
- `传承 / 宿命 / 终局 / 商会` 做境界、剧情或场景化显示，不作为常驻一级按钮。
- 正式 NPC/势力系统前，做旧系统与 `livingWorldState` 的权威归并表。
- 正式经济系统前，做 material/price/shop/refine 权威归并表。

## 非目标

除非用户另行批准，v0.16 不做：

- 正式地点切换、路线成功、南疆完整地域开放。
- 正式阵营转移、正式招揽、正式通缉、正式任务奖励。
- NPC 生死、捕获、重伤等命运结论。
- 正式材料、元石、蛊虫、蛊方、市场库存、价格表、交易结算。
- 新增持久字段或提升 `SAVE_FORMAT_VERSION`。
- DeepSeek 获得奖励、地点、阵营、NPC 生死、正史或数值写入权。
- 大规模 engine 目录重组、全局重皮、完整新 UI 框架替换。
- EdgeOne 自动部署、branch protection、auto merge。

- 纯文档/规划/Git 阶段可豁免，但必须写明。
- `a1/a2` 设计和仪表盘阶段默认豁免 runtime 走查。
- `b1/b2/b3` 玩家可见 UI 小版本建议 20 轮。
- `b4` 权威归并验证若只写文档/测试可豁免；若触及 UI 或行为，至少 10-20 轮。
- `rc` 建议 60 轮，重点验证“玩家下一步是否清楚”“是否误以为候选就是事实”“是否找得到行动入口”。
