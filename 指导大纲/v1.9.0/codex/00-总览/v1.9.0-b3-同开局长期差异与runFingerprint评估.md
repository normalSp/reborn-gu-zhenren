# v1.9.0-b3 同开局长期差异与 runFingerprint 评估

状态：complete locally；评估-only
日期：2026-05-22
性质：长期差异度方案评估；不新增 `runFingerprint`。

## 结论

专家团建议：

`v1.9 和 v2.0 第一刀仍不要新增 runFingerprint。`

理由：

- 当前同开局差异可由公开 pressure deck、事件 envelope 顺序、局部表达和玩家选择后果实现。
- 过早写 per-save 随机字段会增加复现、回滚、长测、跨版本迁移和 bug 定位成本。
- v2.0 前最重要的是先证明稳定事实不漂移，再证明差异度足够。

## 方案比较

| 方案 | 收益 | 风险 | 当前建议 |
|---|---|---|---|
| 无持久字段，使用公开 pressure deck + deterministic variant index | 旧档安全、可复现、测试简单 | 长线重复体验可能不足 | v1.9/v2.0 第一刀推荐 |
| session-local seed，不进存档 | 同一会话更有变化，不影响旧档 | 跨会话不可复现 | 可作为 T3 dry-run 候选 |
| per-save `runFingerprint` | 长期可复现差异、利于 replay archive | 需要 v25、迁移、回滚、隐性事实漂移防护 | 仅列 future gate |
| 从 `regionalEventLedger` 派生差异 | 差异与玩家历史强绑定 | 必须先有事件账本，复杂度更高 | v2.0 后再评估 |

## 当前可用差异来源

允许：

- 同一公开外缘开局分别偏向盘问、商队短工、市场失败、临时落脚、路障/补给压力。
- 同一事实下，候选下一步排序不同。
- DeepSeek 未来获批时只改表达风格，不改本地事实。
- 玩家选择、小队状态、补给压力、社会压力、冲突风险不同，导致后续压力不同。

禁止：

- 随机改正式地点、身份、阵营、奖励、NPC 生死。
- 随机揭示 hidden/private。
- 用 MiroFish item 直接决定 runtime 事实。
- 同一存档内前后矛盾。

## T3 评估指标

v2.0 前若要重新讨论 `runFingerprint`，至少需要 T3 给出：

| 指标 | 通过线 |
|---|---|
| 事实稳定率 | 关键事实 100% 不漂移 |
| hidden 安全 | P0 hidden leak = 0 |
| 差异可见度 | 同开局 5 条路线至少 3 条 pressure family 不同 |
| 玩家理解 | Player Advocate 下一步理解率达到制度要求 |
| 复现能力 | 失败样本能通过 replay id / sample id 定位 |
| 回滚成本 | 字段可安全缺省、压缩、迁移和删除 |

## future decision trigger

只有以下情况同时成立，才建议让用户重新决策 `runFingerprint`：

1. T3 证明无持久字段差异度不足。
2. 差异不足不是文案或 event deck 可解决。
3. v25 字段表、migration/defaults/tests 已准备。
4. rollback 和旧档策略已准备。
5. DeepSeek 不会因为 fingerprint 获得事实裁决权。
