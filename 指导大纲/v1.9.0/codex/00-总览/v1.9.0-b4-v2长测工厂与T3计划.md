# v1.9.0-b4 v2 长测工厂与 T3 计划

状态：complete locally；计划-only
日期：2026-05-22
性质：v2.0 前 T3 300+ mixed/live/replay 长测计划；本阶段不实际跑 live。

## T3 定义

T3 是 v2.0 区域活世界入场硬门，不是普通 smoke。

目标：验证第一个核心区域在长线运行中不会出现：

- 世界状态漂移。
- DeepSeek 越权。
- 原著/hidden 边界漂移。
- 同开局差异靠事实乱变实现。
- 玩家目标遗失。
- 区域事件重复空转。
- 正式地点/身份/奖励/NPC 生死误授予。

## 推荐总轮次

v2.0 前建议至少 300+ total rounds，分为四类：

| 轨道 | 轮次 | 用途 | 是否 live |
|---|---:|---|---|
| T3-A deterministic replay | 120 | 无模型、固定样本、检查本地状态和 forbidden outcomes | 否 |
| T3-B mixed scripted probe | 80 | 用固定 prompt / sample 驱动候选，检查上下文拼接和权威边界 | 可 dry-run |
| T3-C live narrative probe | 80 | `deepseek-v4-flash`，检查真实叙事漂移、术语、hidden、下一步 | 是 |
| T3-D regression rescore | 20+ | 对失败/边缘样本复评，确认修复后不过拟合 | 可 live/replay |

最低线：300 轮；高风险线：400-500 轮。

## 样本族

| 样本族 | 覆盖 |
|---|---|
| route/location | 青茅后外缘、小路、歇脚点、远城门槛 |
| identity/replay | 短工、护送、散修落脚、公开观察 |
| survival/economy | 补给缺口、压价、拒绝、低阶损耗 |
| social/faction | 中介、管事、盘问、临时人情 |
| conflict | 路障、护送、冲突余波、避战 |
| hidden/private | 方源私密因果、hidden 只作 ref，不可见 |
| formal conclusion traps | 正式入城、正式商队成员、工资、奖励、NPC 生死 |
| same-start replayability | 5 次同开局不同 pressure family |

## Checkpoint

每 25-50 轮产出一次 checkpoint：

| 检查项 | 失败例 |
|---|---|
| 世界状态 | 无批准字段却出现正式 ledger / fingerprint |
| DeepSeek authority | DeepSeek 写地点、奖励、身份、NPC 命运 |
| 原著边界 | 完整南疆/商家城/hidden 被提前开放 |
| 连续性 | 玩家前一步被拒，后一步突然正式入城 |
| 差异度 | 同开局 5 次只有措辞不同 |
| 玩家下一步 | 叙事没有可执行下一步 |
| 空转重复 | 连续多轮盘问/等待无新压力 |
| P2 术语 | 木牌/令牌/member 被写成正式身份 |

## 报告格式

建议未来报告目录：

`artifacts/v2.0.0/t3-regional-life/<run-id>/`

建议内容：

- `manifest.json`：模型、样本、轮次、成本、prompt hash、commit。
- `rounds.jsonl`：每轮输入、候选、输出、标签。
- `checkpoints.md`：每 25-50 轮人工/规则审查。
- `findings.md`：P0/P1/P2 列表。
- `rescore.md`：修复后 replay/rescore。
- `go-no-go.md`：是否允许进入 v2.0 runtime。
