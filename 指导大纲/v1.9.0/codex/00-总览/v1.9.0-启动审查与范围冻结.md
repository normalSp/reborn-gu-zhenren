# v1.9.0 启动审查与范围冻结

状态：draft；等待用户批准 D-190。
日期：2026-05-22

## 推荐主题

`v2.0 区域活世界预备与门禁收束`

## 为什么不是直接进入 v2.0

v1.7/v1.8 的 live gate 证明的是小规模 projection-first 叙事稳定，不证明一个区域可以支撑 300+ 回合长期运行。

v2.0 要打开的是“一个真正能长期运行的区域活世界”。这需要在 v1.9 先回答：

- 第一个核心区域到底是哪一个。
- 区域事件是否需要持久 ledger。
- 同开局差异是否需要 `runFingerprint`。
- 身份路线、路线状态、生存经济、社会压力、冲突后果如何合流。
- MiroFish source pointer 如何支撑，不让 DeepSeek 或全书基础包越权。
- 长线 drift / Player Advocate / live probe / replay archive 如何成为硬门。

## 推荐范围

允许：

- 建立 v1.9 启动包、D-190 决策池和 v2.0 readiness 草案。
- 把 `regionalEventLedger`、`runFingerprint`、`regionalLifeState`、`identityRouteState` 放入 a1 严肃评估。
- 将 v1.8 P2 正式道具词/英文术语风险纳入 v1.9 current_matrix。
- 设计 MiroFish v2 prelude topic-slice request 方向。
- 设计 v2.0 前 300+ 区域活世界长测硬门。

暂缓：

- 任何 save-format bump。
- 任何 runtime code。
- 任何 MiroFish export 或 intake review。
- live DeepSeek probe。
- 子代理。
- BFF/backend/public wording/EdgeOne。

禁止：

- 直接新增 `regionalEventLedger`、`runFingerprint`、`regionalLifeState`、`identityRouteState`。
- 正式开放完整南疆、完整商家城、正式地点/阵营/身份/奖励/NPC 生死。
- 将 MiroFish 基础包整包导入知识库、runtime 或 DeepSeek visible context。
- 让 DeepSeek 生成区域事实、奖励、地点、阵营、战斗结论或 hidden/private 可见事实。
