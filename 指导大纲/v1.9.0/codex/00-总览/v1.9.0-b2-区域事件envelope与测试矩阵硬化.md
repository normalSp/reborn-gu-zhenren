# v1.9.0-b2 区域事件 envelope 与测试矩阵硬化

状态：complete locally；report-only
日期：2026-05-22
性质：事件候选结构与测试样本硬化；不新增 `regionalEventLedger`。

## 目标

b2 将 a1/a2 的区域事件草案变成 v2.0 可审计的 envelope 规则。它只用于文档、测试矩阵和未来 pure helper 设计，不是存档账本，也不是 runtime canon。

## Envelope 最小结构

| 字段 | 用途 | 禁止事项 |
|---|---|---|
| `envelopeId` | 稳定测试/报告 id | 不等于存档 ledger id |
| `sourceFamily` | route / identity / survival / social / conflict / public-observation / hidden-risk | 不创建新权威来源 |
| `sourcePointers` | a2 reviewed source pointer ids | 不复制原文、quote、hidden body |
| `visiblePressure` | 玩家可见压力 | 不写正式结果 |
| `entryPreconditions` | 公开前置条件 | 不自动授予地点、身份、阵营 |
| `candidateNextSteps` | 可被拒绝、可失败的下一步 | 不结算奖励、工资、库存、NPC 生死 |
| `forbiddenOutcomes` | 必须阻断的越界结果 | 必须覆盖 DeepSeek 越权 |
| `matrixRefs` | 测试矩阵引用 | 不等于 runtime 测试已执行 |
| `v2PromotionGate` | 晋升到 runtime 前需要的门禁 | 必须写用户决策/字段/测试条件 |

## 晋升条件

任一 envelope 想进入 v2.0 runtime 前，必须至少满足：

1. 用户批准是否新增持久账本。
2. 若新增字段，同提交补 migration/defaults/tests。
3. 通过 MiroFish intake，不含原文/hidden body。
4. 通过 DeepSeek authority 样本：DeepSeek 只能表达本地候选，不写事实。
5. 通过 Player Advocate：玩家能看懂压力、代价和下一步。
6. 进入 T3 300+ mixed/live/replay 覆盖。

## 反刷和旧档边界

b2 不新增 store，因此没有新的旧档迁移和反刷状态。但 envelope 必须保留未来反刷要求：

- 同一 envelope 不得稳定产出奖励。
- 失败推进不得变成可重复刷钱/刷身份。
- 临时凭据不得变成永久通行。
- public observation 不得累计成 hidden reveal。

## Player Advocate / live DeepSeek

- 是否调用 live DeepSeek：否。
- 原因：b2 是 report-only envelope 设计，不触碰玩家可见 runtime。
