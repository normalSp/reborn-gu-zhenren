## 字段评估

| 字段/状态 | 当前建议 | 正面收益 | 负面风险 | 后续条件 |
|---|---|---|---|---|
| `SAVE_FORMAT_VERSION = 25` | v1.9 b1 暂不 bump | 避免旧档迁移、回滚和测试面膨胀；保留 projection-first 速度 | v2.0 前仍需再次设计持久状态 | 只有新增持久字段时才 bump，且同提交补 migration/defaults/tests |

| `identityRouteState` / `professionState` | 暂不新增 | v1.8 身份候选不被误读为正式职业/阵营；旧档稳定 | 无法持久记录正式身份路线 | 只有正式身份系统获批、MiroFish blocking intake 通过、经济/势力/奖励边界明确后再评估 |

## 区域选择

推荐 v2.0 第一核心区域候选继续锁为：

`南疆早期低阶外缘小区域`

允许进入 a2 request / rule draft / test matrix 的范围：

- 山路歇脚点。
- 临时商队接触窗口。
- 散修落脚/短工圈。
- 城门外缘门槛。
- 路途补给、盘问、冲突、人情债和市场压价。

禁止升格：

- 完整南疆。
- 完整商家城。
- 正式进入城市。
- 正式商队成员、正式散修势力、正式护卫组织。
- 稳定工资、价格、库存、任务奖励。
- NPC 生死、招揽、通缉、封锁、阵营归属。

## 区域事件 envelope 草案

v1.9 建议先设计事件 envelope，而不是 ledger。

最小 envelope 字段仅用于文档、测试和后续 pure projection：

| 字段 | 含义 | 禁止事项 |
|---|---|---|
| `eventId` | 稳定测试样本 id | 不等于存档 ledger id |
| `sourceFamily` | route / identity / survival / social / conflict / hidden-risk | 不创建新权威来源 |
| `visiblePressure` | 玩家可见压力摘要 | 不写 hidden/private body |
| `preconditions` | 进入事件候选所需公开前置 | 不写正式地点/身份 |
| `candidateOutcomes` | 可能的下一步候选 | 不写奖励、NPC 生死、阵营结论 |
| `forbiddenOutcomes` | 必须阻断的结果 | 必须覆盖 DeepSeek 越权 |
| `testRefs` | 测试矩阵引用 | 不等于 runtime canon |
