# v1.9.0 专家团启动会纪要

状态：已召开；等待用户批准 D-190。
日期：2026-05-22
主题候选：v2.0 区域活世界预备与门禁收束

## 开会背景

v1.7 完成了低阶区域活世界 projection-first 第一阶段，证明路线、生存、社会、区域、冲突和 MiroFish topic-slice 可以合流成一个小范围可测试态势。

v1.8 完成了低阶身份路线与同开局差异度 projection-first 地基，证明同一开局可以通过本地 pressure deck 和表达槽位形成不同体验，同时保持事实稳定。

但这两版都刻意没有打开以下内容：

- `regionalEventLedger` 或等价区域事件账本。
- `runFingerprint` 或 per-save 持久差异度。
- `regionalLifeState` / `areaLivingState`。
- `identityRouteState` / `professionState`。
- 正式地点、正式阵营、正式身份、奖励、NPC 生死。
- DeepSeek full-book RAG 或可见知识摘要。
- v2.0 级 300+ 轮区域活世界长测硬门。

因此 v1.9 的价值不是继续堆一个新 tab，而是把 v2.0 真正打开区域活世界前的“手术台”搭好。

## 专家团总建议

推荐 v1.9 主线为：

`v2.0 区域活世界预备与门禁收束`

一句话目标：

> 选定第一个 v2.0 核心区域，整理区域事件、NPC/势力、资源、路线、战斗、经济、身份和隐藏事实边界，设计 save-format / event-ledger / run-difference / long-soak 门禁；但 v1.9 默认仍不直接开放正式区域活世界。

## 集思广益：好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| v2.0 区域 readiness checklist | 让 v2.0 不靠感觉开工 | v1.9 只做门禁，不声称 v2.0 已达成 |
| 区域事件 envelope 草案 | 统一事件来源、触发、可见性、禁止结论 | 不等于 `regionalEventLedger` 已批准 |
| 同开局长期差异轴 | 区分短期表达差异和长期事件差异 | `runFingerprint` 只评估，不默认持久化 |
| 区域 pressure deck 合流 | 把路线、生存、社会、冲突、身份组成一个区域态势 | 必须保持 projection-first，避免自动写正式状态 |
| P2 术语 hardening | 收束木牌/令牌/登记/member 等风险 | 作为 prompt/eval hardening，不创造正式凭证 |
| MiroFish v2 prelude slice | 从全书基础包取南疆低阶区域生活 source pointers | 只做 quote-redacted topic slice 和 intake review |
| v2.0 300+ mixed soak 计划 | 提前定义长线剧情质量硬门 | v1.9 不必立即跑 300+ live，先设计和小规模验证 |
| 只读子代理审查试点 | 加速 coverage 和测试矩阵比对 | 需用户单独批准，且不能写文件/提交 |

## 不推荐现在做

- 直接新增 `regionalEventLedger`、`runFingerprint`、`regionalLifeState`、`identityRouteState` 或等价持久字段。
- 正式开启完整南疆、完整商家城、正式商队/散修/身份系统。
- 正式区域事件结算、奖励、工资、价格、库存、战斗掉落或 NPC 生死。
- DeepSeek full-book RAG、可见知识摘要、hidden/private 可见化。
- BFF/backend 重构、公开测试口径、EdgeOne 自动部署。
- 可写子代理或多代理并行改 RebornG 文件。

## 专家团结论

v1.9 最稳路线是：D-190 批准后进入 a1 设计门禁。a1 必须回答：

1. v1.9 是否以 v2.0 区域活世界预备为主线。
2. v2.0 第一核心区域是否默认选南疆早期低阶外缘小区域。
3. `SAVE_FORMAT_VERSION = 25` 是否需要，以及哪些字段必须继续暂缓。
4. `regionalEventLedger` 和 `runFingerprint` 的收益、风险、替代方案与最小字段边界。
5. MiroFish v2 prelude topic-slice 是否在 a2 升级为 blocking。
6. v1.9 的 Player Advocate、live probe、T2/T3 长测设计如何定。
7. 是否允许只读/分析型子代理在后续 coverage/test 审查中试点。
