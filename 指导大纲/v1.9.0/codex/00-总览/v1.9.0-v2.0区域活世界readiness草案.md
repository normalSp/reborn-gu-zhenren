# v1.9.0 v2.0 区域活世界 readiness 草案

状态：v1.9 已完成本地 report-only readiness；v2.0 入场仍需用户决策。
日期：2026-05-22

## 目标

本草案定义 v2.0 进入第一个区域活世界前必须回答的问题。a1 已将 save-format、事件账本、长期差异和区域/身份状态拆成 D-191 决策项，且用户已全批准；a2 已完成 MiroFish quote-redacted source-pointer intake，并只晋升为候选、规则草案和测试样本；b1-b4 / process / rc 已按 report-only 完成 readiness、envelope、差异度、T3、P2 和子代理评估。

它不是 v2.0 批准书，也不是 runtime 设计完成证明。

## 推荐 v2.0 第一核心区域

候选：

`南疆早期低阶外缘小区域`

最小可运行边界：

- 山路歇脚点。
- 临时商队接触。
- 散修落脚/短工。
- 城门外缘门槛。
- 路途补给、盘问、冲突和人情压力。

不得扩大为：

- 完整南疆。
- 完整商家城。
- 正式商队系统。
- 正式散修势力。
- 正式城市身份、户籍、任务、奖励。
- NPC 生死或隐藏事实揭示。

## v2.0 go/no-go 草案

进入 v2.0 runtime 前，至少需要：

1. 用户明确批准第一核心区域。
2. 用户明确批准是否 bump save-format。
3. MiroFish v2 prelude topic-slice intake 通过。
4. 测试矩阵覆盖 route / identity / economy / social / conflict / hidden / DeepSeek authority。
5. 长线叙事漂移制度给出 T3 300+ 计划或已执行证据。
6. Player Advocate 证明玩家能理解下一步。
7. 所有 formal location/faction/reward/NPC-life/canon promotion 仍有独立审批边界。
