# v1.9.0-process-1 P2 术语与正式凭证 hardening

状态：complete locally；规则/样本 hardening
日期：2026-05-22
性质：收束 v1.8/v1.9 暴露的正式道具词、英文身份词和凭证误授予风险；不改 prompt/runtime。

## 背景

v1.8 rc live/replay 留下 P2 风险：正式道具词、英文术语和身份词容易让玩家误读为已经获得正式身份或地点权限。v1.9-a2 也再次把木牌、令牌、登记、报到、负责人点头等列为风险词。

process-1 将它们纳入 v1.9 当前矩阵和后续 prompt/eval hardening 输入。

## 风险词分级

| 类别 | 风险词 | 允许表达 | 禁止表达 |
|---|---|---|---|
| 凭证物 | 木牌、令牌、腰牌、牌子 | 临时凭据、待复核凭证、降低盘问的公开理由 | 正式通行证、身份牌、入城凭证 |
| 流程词 | 登记、报到、名册、负责人点头 | 报到前置、临时记名、等待复核 | 正式登记入册、长期身份 |
| 组织词 | 商队成员、护卫、散修落脚点、情报人 | 短工、杂务、护送候选、公开观察 | 正式成员、职位、组织身份 |
| 英文词 | member、license、pass、official、guild、city entry | 转写为中文临时压力和前置条件 | 作为正式系统术语出现 |
| 奖励词 | 工资、赏钱、任务奖励、价格表 | 补给压力、被压价、临时让步 | 稳定收益、库存价格、奖励结算 |

## hardening 规则

1. 有凭证词时，必须同时出现“临时/待复核/可被拒绝/不能正式通行”的限制。
2. 有商队/护卫/散修相关词时，必须优先写“短工/杂务/候选/外缘接触”。
3. 英文身份词不得作为玩家事实；应转写为中文公开压力。
4. 奖励、工资、价格、库存必须降级为压力，不进入结算。
5. DeepSeek 未来 live 输出若出现正式词，eval 必须判断是否越权。
