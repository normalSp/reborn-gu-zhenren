## v2 readiness envelope hint

v1.9-a2 只给未来 envelope 设计样本，不批准 `regionalEventLedger`。

| envelope slot | 推荐表达 | 禁止事项 |
|---|---|---|
| `sourceFamily` | route / identity / survival / social / conflict / public-observation | 不创建新权威来源 |
| `visiblePressure` | 盘问、商队接触、压价、临时收留、路障、补给让步 | 不写 hidden/private body |
| `preconditions` | 货物、人情、低调身份、短工意愿、公开观察 | 不写正式地点/身份/阵营 |
| `candidateOutcomes` | 被拒后找短工、绕行、传话、等待、付出力气、降低注意 | 不写奖励、NPC 生死、稳定职位 |
| `forbiddenOutcomes` | 正式通行证、商队成员、商家城进入、价格结算、战斗奖励、hidden reveal | 必须进入测试矩阵 |
| `testRefs` | `V19-A2-SLICE-*`、`V19-P2-*`、`V19-DS-AUTH-*` | 不等于 runtime canon |

## Same-start replayability rule

同一开局重复运行时，v1.9 允许差异来自：

- 公开压力牌组不同：盘问、商队接触、市场失败、临时落脚、路障、补给让步。
- 同一公开事实下，下一步候选顺序和叙事表达不同。
- local projection/report 对既有 v1.1-v1.8 状态的合流解释不同。
- DeepSeek 只在未来获批 live probe 时表达本地给出的公开压力。

禁止差异来自：

- `runFingerprint` 或持久随机字段未经批准写入存档。
- 正式身份、地点、阵营、奖励、NPC 生死变化。
- hidden/private 泄露。
- MiroFish item 直接进入 runtime 叙事。
- 同一存档内前后事实矛盾。

## Formal prop-word hardening rule

以下词必须按 P2/P1 风险处理，除非后续门禁单独批准正式系统：

- 木牌、令牌、腰牌、名册、登记、报到。
- 负责人点头、管事安排、跟队、临时帐篷。
- 商队成员、护卫身份、散修落脚点、情报人。
- city entry、member、license、pass、official、guild 等英文/混写表达。

允许写法：

- 临时凭据。
- 需要复核的前置条件。
- 能降低盘问强度的公开理由。
- 可被拒绝或被安排杂务的候选窗口。
- 只在 report/test 里作为风险词，不作为玩家事实。

禁止写法：

- 正式加入商队/势力。
- 获得稳定职位、工资、身份牌或任务奖励。
- 解锁商家城、正式地点、正式路线通行权。
- 绑定命名 NPC 权威或 NPC 命运。

## b1 projection/report hint

若 b1 获批第一刀，建议优先做 report/projection-first：

`routeLocationState + survivalEconomyState + livingWorldState + v1.3/v1.4/v1.5/v1.7/v1.8 projection + localActionLedger + v1.9-a2 reviewed source pointers -> v2 readiness pressure/envelope report`

默认输出只显示：

- 已有地基能支撑的区域生活压力。
- 区域事件 envelope 候选和 forbidden outcomes。
- 同开局差异的公开来源。
- 正式凭证词/英文术语风险。
- 缺口：哪些必须等 v2.0 / v25 / T3 长测。

不写：

- 新 save 字段。
- 新 store action。
- 正式区域、地点、身份、职业、组织。
- 工资、价格、库存、任务奖励。
- NPC 生死或 hidden/private。
- DeepSeek visible MiroFish summary。
