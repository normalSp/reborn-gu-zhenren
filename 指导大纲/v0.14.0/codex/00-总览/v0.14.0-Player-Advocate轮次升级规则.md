# v0.14.0 Player Advocate 轮次升级规则

日期：2026-05-17
状态：第一版

## 背景

v0.13 追补 50 轮走查证明默认制度有效，但 v0.14 首次涉及“路线承接”和可能的地点入口，玩家体验风险更高。

因此 v0.14 不把所有小版本都机械设为 10 轮，而是按风险升级。

## 默认规则

| 阶段类型 | 轮次 | 说明 |
|---|---:|---|
| 纯文档 / CI / Git / 内部脚本 | 0 | 可豁免，必须写明 |
| 普通玩家可见小版本 | 10 | 不触碰地点、阵营、隐藏事实、旧档 |
| 高风险玩家可见小版本 | 20 | 触碰路线承接、地点入口、阵营前置、隐藏事实、旧档迁移 |
| 普通 rc | 50 | 项目默认 |
| v0.14 rc | 60 | 首次路线承接，升级 |

## 高风险判定

出现任一项即升级到 20 轮：

- 正式路线承接。
- 地点入口或地域状态。
- 阵营目标前置条件。
- hidden fact probe。
- 旧档迁移或 save format 影响。
- DeepSeek prompt 可见事实变化。
- 移动端承载关键按钮或路线状态。

## v0.14 rc 60 轮覆盖

| 覆盖项 | 目标 |
|---|---:|
| 主线路线推进 | >= 20 |
| 自由目标与路线意图 | >= 10 |
| 阵营目标前置 | >= 8 |
| 失败/阻断/风险/代价 | >= 10 |
| 移动端或 reduced-motion | >= 6 |
| 旧档/测试存档/生产预览 | >= 6 |

## 通过阈值

继承项目制度：

- 小版本下一步可理解率 >= 80%。
- rc 下一步可理解率 >= 85%。
- 小版本严重困惑 <= 2/10 或 <= 4/20。
- v0.14 rc 严重困惑 <= 6/60。
- P0/P1 玩家体验阻断为 0。
- 隐藏事实泄露为 0。
- UI 私算奖励/地点/阵营/NPC 生死为 0。

## 脚本

当前 `scripts/check-player-advocate-gate.mjs` 支持 min rounds 和基础阈值。

v0.14 如果需要强制 20/60 专属困惑阈值，应在 a1 或 b4 前扩展脚本；否则先用现有脚本验证轮次和 85% rc 理解率，再由走查记录人工确认 v0.14 专属覆盖项。
