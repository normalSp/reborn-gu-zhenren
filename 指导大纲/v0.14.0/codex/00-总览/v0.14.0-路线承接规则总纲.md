# v0.14.0 路线承接规则总纲

日期：2026-05-17
状态：第一版

## 定义

路线承接不是“点击后传送到新地图”。

路线承接是：

玩家的长期目标、公开行动、路线准备、补给、追击压力、NPC 记忆和势力态度共同决定后续路线是否可进入、需要什么前置、有什么风险、哪些路线暂不可行。

## 基本闭环

`自由目标 -> 路线意图 -> 条件预览 -> 前置行动 -> 路线承接状态 -> 后续入口`

## 路线类型

v0.14 只允许第一层 route archetype：

| routeKey | 名称 | 口径 |
|---|---|---|
| `merchant_caravan_contact` | 商队接触路线 | 先做递话、遮掩、付代价、随行机会，不直接到商家城 |
| `mountain_pass_escape` | 山路逃离路线 | 强调补给、追击、遮蔽、野外风险 |
| `baijia_contact_window` | 白家接触窗口 | 只做接触/递话/试探，不投靠成功 |
| `rogue_cultivator_path` | 散修过渡路线 | 只做生存压力和低阶落脚，不开放完整散修系统 |
| `three_clan_remnant_pressure` | 三寨残余压力 | 作为追击/阻拦/谣言，不结算大战结果 |

## 条件来源

路线条件必须从以下本地来源读取：

- `playerGoals`
- `knownFacts`
- `hiddenFactRefs`
- `npcMemories`
- `factionPressure`
- `actionConsequences`
- `worldClock.lastActionId`
- `sceneSessionState.localActionLedger`
- v0.12 route/supply/pursuit helpers
- v0.13 social projection helpers

## 结果等级

| 等级 | 含义 | 是否可写 runtime |
|---|---|---|
| `blocked` | 当前不可做 | 可写公开阻断解释 |
| `needs_preparation` | 需要前置行动 | 可生成前置行动 |
| `candidate` | 路线候选可见 | 只读展示 |
| `committed_preparation` | 玩家已执行前置 | 可写现有 v22 字段 |
| `route_ready` | 条件已足够进入路线承接 | 需要阶段决策 |
| `route_entered` | 已正式进入路线状态 | 需要用户决策和测试门禁 |

默认 v0.14-b2 前不得写 `route_entered`。

## 权限边界

路线承接可写：

- 公开 known fact。
- action consequence。
- faction pressure。
- npc memory。
- player goal status。
- local action ledger。
- return context。

路线承接不可写，除非用户另行批准：

- location unlock。
- faction transfer。
- reward/material/Gu。
- NPC death/capture。
- canon anchor mutation。
- hidden fact body。
- DeepSeek authority。

## 阵营目标处理

玩家输入“投靠白家”“加入商队”“离开古月”时：

1. 系统必须承认目标。
2. 系统必须显示当前不可一步完成的原因。
3. 系统必须显示可做的前置行动。
4. 系统必须显示风险和代价。
5. 系统不得直接改变阵营。

## 隐藏事实处理

路线承接不能因为玩家要走某条路就泄露：

- 方源隐藏因果。
- 春秋蝉、重生、回溯。
- 灵泉秘密正文。
- 花酒行者隐藏遗藏正文。
- 原著未到时机的关键因果。

只能写 hidden ref 和公开失败/被注意到的后果。

## Player Advocate 口径

玩家必须能看懂：

- 我现在想去哪。
- 为什么能去或不能去。
- 缺什么条件。
- 谁会阻拦或帮助。
- 这一步是否真的改变地点。
- 这一步是否只是准备。
