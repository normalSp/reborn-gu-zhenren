# v0.14.0-b1 候选后续到正式前置行动桥

日期：2026-05-17
状态：已实现，待验证收束

## 目标

把 v0.13 的一个 `社会影响 / 局势后续` 候选升级成可点击、可记录、可回归测试的正式前置行动样板。

本阶段选择：

`followup_prepare_public_reason -> qingmao_cover_escape_tracks_probe`

玩家感受目标：

- 不是只看到“以后可以做”，而是真的能做一个低风险小动作。
- 做完后世界账本会回应：公开理由、低可见度窗口、残余痕迹、NPC 公开记忆。
- 仍然明确：这不是离开青茅山成功，不是阵营转移，不是任务奖励。

## 行动协议

| 字段 | 值 |
|---|---|
| actionId | `qingmao_cover_escape_tracks_probe` |
| 来源候选 | `followup_prepare_public_reason` |
| 触发前置 | 已确认逃离青茅山目标，或已完成路线准备上下文 |
| 写入源 | `action_protocol` |

| 是否消耗 AP | 否 |

## 关键写入

| 类型 | id | 说明 |
|---|---|---|
| knownFact | `qingmao_escape_tracks_cover_baseline` | 玩家完成遮掩痕迹前置，但仍未离开青茅山 |
| factionPressure | `faction_pressure_qingmao_cover_tracks_low_visibility_window` | 形成低可见度行动窗口 |
| factionPressure | `faction_pressure_qingmao_cover_tracks_<faction>_residual_trace` | 遮掩本身仍有残余痕迹 |

| actionConsequence | `consequence_qingmao_cover_escape_tracks_probe` | 记录这次前置行动的公开后果 |

## Player Advocate

b1 按 v0.14 高风险玩家可见阶段执行 20 轮走查。

本阶段走查采用：

- deterministic walkthrough：必需。
- e2e player path：已覆盖核心点击路径。
- live DeepSeek probe：本阶段不需要。

不调用 live DeepSeek 的理由：

- 未改 DeepSeek prompt、schema、model、上下文构建或叙事回流协议。
- 本阶段是本地 action protocol 写入验证。
- live probe 应用于叙事回流、公开候选文本或 rc/public candidate，而不是替代本地确定性门禁。

## 阶段结论

`v0.14.0-b1` 是第一个“候选后续 -> 正式前置行动”样板。它证明 v0.13 的社会反应候选可以被安全升级为玩家可执行的小步行动，并且不会越权成地点、阵营、奖励或隐藏事实结算。

下一步建议：

- `v0.14.0-b2 青茅离开路线第一刀`。
- 进入 b2 前必须再次确认：是否触及正式路线状态、地点变化或新持久字段；若触及，停下来让用户决策。
