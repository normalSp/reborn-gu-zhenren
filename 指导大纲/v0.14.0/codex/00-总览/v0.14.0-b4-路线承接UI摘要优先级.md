# v0.14.0-b4 路线承接 UI 与摘要优先级

日期：2026-05-17
状态：已完成
MiroFish：not_needed，本阶段只调整 RebornG-owned UI 摘要和可读性
Player Advocate：20 轮

## 目标

当自由目标面板同时出现目标账本、社会影响、阵营前置和路线承接时，玩家先看到“下一步该看什么”，再进入详细面板。

本阶段解决 b3 走查留下的 P2 问题：移动端信息量偏重，玩家需要一个第一眼可读的摘要。

## 非目标

本阶段不做：

- 新路线规则。
- 新 MiroFish 吸收。
- 新存档字段。
- `route_entered`。
- `faction_transfer`。
- `standing_delta`。
- `formal_task`。
- `task_reward`。
- `location_unlock`。
- NPC 生死、抓捕、追杀成败。
- 隐藏事实正文展示。
- DeepSeek prompt、schema、model 或权限变化。

## UI 设计

`FreeGoalPanel` 新增 `优先摘要` 区块：

- `data-testid="free-goal-next-step-summary"`
- `data-testid="free-goal-summary-goal"`
- `data-testid="free-goal-summary-route"`
- `data-testid="free-goal-summary-faction"`
- `data-testid="free-goal-summary-social"`

摘要区只排序和压缩信息，不替代详细面板：

- 当前目标：显示最新目标或提示先输入自由目标。
- 路线：优先显示非阻断路线，如 `山路逃离路线`。
- 阵营/身份：命中投靠、商队、商家城、散修时显示第一张前置卡。
- 社会影响：统计公开 NPC、势力、编年史、后续候选信号，并给出第一条高优先级提示。

## 边界文案

摘要固定展示：

`摘要只排序信息，不写状态、不转阵营、不发奖励、不进地点。`

这句话是 b4 的玩家侧边界锚点，防止摘要被误读为正式结算。

## MiroFish 判定

本阶段 MiroFish 需求为 `not_needed`。

原因：

- 不新增 canon、IF、NPC 动机、势力压力或路线事实。
- 不吸收新包，不提升候选材料等级。
- 只读取现有 `routeContinuation`、`factionGoalPrerequisites` 和 v0.13 社会影响投影结果。

后续如果要新增商队递话正式行动、南疆入口、命名 NPC 关系或更细路线规则，再按 MiroFish gate 重新判断。

## 完成口径

玩家在移动端和桌面打开自由目标面板后，应能先从摘要读到：

- 当前目标是否已记录。
- 路线当前是阻断、需前置、候选还是预览满足。
- 阵营目标是否只是前置展示。
- 当前是否已有公开社会影响。
- 摘要不会写状态、转阵营、发奖励或进入地点。
