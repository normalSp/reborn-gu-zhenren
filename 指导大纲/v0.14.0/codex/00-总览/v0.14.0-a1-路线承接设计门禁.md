# v0.14.0-a1 路线承接设计门禁

日期：2026-05-17
状态：第一版，待执行

## a1 目标

先把路线承接的规则、字段、权限、测试和资料需求定义清楚，不先写 runtime。

## 必须回答

1. 哪些路线可作为 v0.14 第一层 route archetype。
2. 路线条件从哪些本地账本读取。
3. 哪些结果只能候选，哪些可以写现有 v22 字段。
4. 什么时候必须停下来让用户决策。
5. MiroFish 包缺失时如何缩小范围。
6. Player Advocate 轮次如何升级。

## a1 通过条件

- 路线承接规则总纲存在。
- 路线条件字段表存在。
- 测试矩阵存在。
- MiroFish 请求文件存在。
- Git 计划存在。
- 真相源索引指向 v0.12/v0.13 地基。

## Stop Gates

以下任何一项出现，必须停止并让用户决策：

- 新增持久 route/location 字段。
- bump `SAVE_FORMAT_VERSION`。
- 正式进入新地域。
- 阵营身份变化。
- 正式任务/奖励/声望/通缉/招揽。
- NPC 生死、抓捕、追杀成功。
- 正史锚点变化。
- DeepSeek 新写入权。
- 薄后端/BFF 开工。
- EdgeOne 自动部署或 branch protection。

## 默认批准边界

可以在 a1 直接做：

- 文档规则。
- 字段草案。
- 测试矩阵。
- MiroFish 请求。
- 后续阶段启动条件。

不可在 a1 做：

- runtime 写入。
- store 字段新增。
- UI 行为改动。
- DeepSeek prompt 权限变化。

## 专家团关注

- Project Lead：确认 v0.14 不膨胀成完整南疆。
- Lore Guard：确保路线不越过原著事实和隐藏事实。
- Systems Architect：确认 route state 是否能复用 v22。
- Player Advocate：确认玩家看得懂路线条件。
- QA Guardian：确认 20/60 轮走查和自动测试矩阵。
- Git Steward：确认提交边界和回滚锚点。
