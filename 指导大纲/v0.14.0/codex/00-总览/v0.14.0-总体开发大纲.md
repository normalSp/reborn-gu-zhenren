# v0.14.0 总体开发大纲

日期：2026-05-17
状态：b4 路线承接 UI 摘要优先级已完成，下一步进入 rc 质量收束
主题：`青茅后续路线承接`

## 目标

玩家能带着青茅山阶段的事实、路线准备、补给压力、追击风险、NPC 记忆和势力压力，进入南疆后续人生的第一层路线承接。

这不是全量南疆开放，而是把“准备离开青茅山”推进为“可解释的路线状态和后续入口”。

## 版本价值

v0.11 让玩家能表达目标。
v0.12 让青茅有正史和 IF 边界。
v0.13 让 NPC/势力能记住和解释玩家。
v0.14 要让这些结果开始改变玩家未来路线。

## 核心交付

1. 路线承接规则总纲。
2. 路线条件表与账本读取。
3. 候选后续到正式前置行动桥。
4. 青茅离开路线第一刀。
5. 阵营目标前置条件展示。
6. 路线承接 UI 可读性与 Player Advocate 修正。
7. v0.14 rc 60 轮玩家视角走查。

## 设计边界

### 可做

- 路线条件、前置行动、路线状态、公开后果解释。
- 读取并引用 v0.12 route/supply/pursuit 和 v0.13 social impact。
- 展示“为什么能走 / 为什么不能走 / 还缺什么”。
- 写入受控、低风险的 existing v22 字段，或在明确批准后再评估 save format。

### 不做

- 地点瞬移。
- 阵营转移。
- 商家城完整系统。
- 正式任务奖励。
- NPC 生死结算。
- 隐藏事实正文展示。
- DeepSeek route/location/faction authority。

## 完成标志

- 玩家可以从“我要逃离青茅山”看到至少一条可解释的路线承接路径。
- 路线选择受身份、资源、关系和前期行为影响。
- v0.12 路线准备、补给、追击压力能进入正式路线条件。
- v0.13 NPC/势力记忆能影响路线机会或阻力。
- 玩家能理解“当前只是路线承接，不是完整南疆开放”。
- 全部 Player Advocate 升级轮次通过。

## 风险

- 如果直接开放路线成功，容易变成文本跳转。
- 如果开放阵营转移，会牵涉声望、任务、身份、追杀、奖励和正史锚点。
- 如果路线状态新增持久字段，必须同步 save migration 和旧档测试。
- 如果 MiroFish 资料不足，路线可能变成泛泛的“往南走”。
