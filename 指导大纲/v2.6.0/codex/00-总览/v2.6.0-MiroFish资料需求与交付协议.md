# v2.6.0 MiroFish 资料需求与交付协议

日期：2026-05-24
状态：startup draft；等待 D-260 审批

## 当前 need level

`not_needed`

## 理由

v2.6 startup 与建议 checker 只使用：

- synthetic fixtures。
- source-pointer classes。
- redacted envelope。
- fake ids。
- abstract boundary cases。

不使用：

- 真实原著事实。
- 命名 NPC。
- hidden-adjacent。
- 方源公开证据。
- 正式 lore 结论。
- 原著正文。
- MiroFish raw output。
- hidden/private body。

## 升级为 blocking 的触发条件

任一阶段出现以下需求时，必须停机并请求用户决策：

- 需要真实原著事实或命名 NPC。
- 需要 L4/L5 fact anchor。
- 需要 hidden-adjacent、方源公开证据或正式 lore 结论。
- 需要知识库正文、runtime canon、DeepSeek visible lore/RAG。
- 需要把 full-book base、MiroFish 基础包或原著正文放进 checker fixture。

## 如果未来升级

仍使用项目级双仓流水线：

`RebornG request -> MiroFish export -> RebornG intake review -> test matrix / rule draft`

v2.6 默认不创建 request，不调用 MiroFish，不写 intake review。

## 禁止项

- 不复制 MiroFish raw output 到 RebornG docs/runtime/prompt。
- 不用 MiroFish 输出直接做 checker fixture。
- 不把 source pointer 当 runtime canon。
- 不把 hidden/private body 写入 report artifact。
