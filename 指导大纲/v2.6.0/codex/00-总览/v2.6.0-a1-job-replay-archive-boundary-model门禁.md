## 一句话结论

v2.6-a1 冻结本地 report-only 的 job / replay / archive boundary model。它是未来 private canon、eval archive、job queue、replay archive 的工程预备协议，不是 service、queue、backend 或真实数据归档。

## 输入

- `v250_b3_v26_private_canon_eval_archive_admission_checklist_v1`
- D-260-001 至 D-260-012：已批准。
- F-260-001 至 F-260-012：全部保持 `future_gate_required`。

## Envelope

最小 envelope 字段：

| 字段 | 要求 |
|---|---|
| `envelopeId` | synthetic id |
| `executionMode` | 固定 `dry_run_report_only` |
| `dataMode` | 固定 `synthetic_redacted_only` |
| `sourcePointerRefs` | 只允许 synthetic/source-pointer ref，不允许正文 |
| `metadata.promptHash` | 必填，记录 prompt/config 指纹；本版本不存 prompt body |
| `metadata.evaluatorVersion` | 必填 |
| `metadata.cost` | 可记录 model=`none` / estimatedUsd=0 |
| `metadata.cache` | 可记录 cache hit/miss |
| `metadata.retry` | 可记录 attempt/recovered 统计 |
| `redaction.*` | 必须证明 hidden/private/prompt/original/MiroFish/DeepSeek visible body 均未进入 envelope |

## Denied Data Classes

本版本 envelope 不允许保存：

- 真实 hidden/private body。
- prompt body。
- 原著正文、MiroFish raw output、full-book base body。
- knowledge-index body、runtime canon body。
- DeepSeek visible lore/RAG/context。
- 玩家正式存档或 `.env` / key。
- backend service endpoint、job queue service、cloud save 配置。

## WorldCore Contract

每个 checker case 必须带：

| 字段 | 要求 |
|---|---|
| `worldCorePostCheck.present` | `true`，负例可故意置 false 以验证拦截 |
| `worldCorePostCheck.finalAuthority` | `WorldCore` |
| `worldCorePostCheck.disposition` | `accepted_for_gate` / `needs_manual_review` / `rejected_violation` |

WorldCore 仍拥有最终事实、可见性、奖励、位置、NPC 生死、canon、save 和 runtime 裁决权。

## No-Go

a1 不授权：

- private canon service。
- eval archive service。
- job queue service。
- backend/BFF/API proxy。
- 真实 archive store。
- runtime/source/UI/store/save/prompt 改动。
- live DeepSeek。
