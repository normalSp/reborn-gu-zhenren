# v2.6.0-a2 private canon classifier interface 门禁

日期：2026-05-24
状态：completed
冻结模型：`v260_a2_private_canon_classifier_interface_v1`

## 一句话结论

v2.6-a2 冻结 private canon classifier 的本地接口：它只负责把候选材料分到 visibility / allowed uses / forbidden uses / manual review，不负责写知识库正文、runtime canon、DeepSeek visible context 或正式存档。

## Classifier Input

| 字段 | 要求 |
|---|---|
| `candidateId` | synthetic 或 source-pointer id |
| `sourceClass` | `synthetic` / `source_pointer_only` / `redacted_summary_candidate` |
| `visibilityHint` | `public_pressure` / `ref_only` / `agent_private_candidate` / `gated` |
| `bodyClass` | 本版本只允许 `none` / `redacted_summary_stub` |
| `triggerFlags` | hidden/private/original/prompt/MiroFish/DeepSeek visible/formal authority/self-learning |

## Classifier Output

| 字段 | 要求 |
|---|---|
| `visibility` | `ref_only` / `agent_private_candidate` / `manual_review_required` / `rejected` |
| `allowedUses` | `eval_boundary_report` / `test_matrix` / `manual_review_queue` |
| `forbiddenUses` | 必须显式列出 runtime canon、DeepSeek visible、save、UI、player-visible copy |
| `requiresManualReview` | source-pointer ambiguity、safe summary 候选、self-learning 候选默认 true |
| `worldCorePostCheckRequired` | 固定 true |

## Severity

| 触发 | severity | disposition |
|---|---|---|
| hidden/private body archival | P0 | rejected_violation |
| prompt body archival | P0 | rejected_violation |
| original/MiroFish raw body archival | P0 | rejected_violation |
| DeepSeek visible lore/RAG | P0 | rejected_violation |
| runtime/save/canon/knowledge body write | P0 | rejected_violation |
| backend/service/external framework/subagent | P0 | rejected_violation |
| formal location/faction/reward/NPC life-death | P0 | rejected_violation |
| self-learning direct write | P0 | rejected_violation |
| memory candidate promoted to canon/confirmed fact | P1 | rejected_violation |
| ambiguous source-pointer summary | P2 | needs_manual_review |

## Manual Review Chain

1. Producer classification。
2. Redaction check。
3. Classifier decision。
4. WorldCore post-check。
5. Test matrix triage。
6. Skill sync audit。
7. User gate if any F-260 item appears。
8. Git/CI evidence。

## No-Go

a2 不授权：

- 知识库正文。
- safe summary body。
- runtime canon。
- full-book RAG。
- MiroFish export/intake。
- private canon service。
- external framework PoC。

a2 完成。可以进入 `v2.6.0-b1-report-only-schema-checker第一刀.md`。
