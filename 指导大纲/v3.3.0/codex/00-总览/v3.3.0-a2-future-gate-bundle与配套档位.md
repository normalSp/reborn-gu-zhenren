# v3.3.0-a2 future gate bundle 与配套档位

日期：2026-05-24
状态：completed
冻结模型：`v330_a2_future_gate_bundle_v1`

## 目标

a2 将 F-300 reopen 可能牵动的 future gate 拆成可审批 bundle，防止未来一句“做 runtime agent”连带打开过多权限。

## Bundle 0：继续 report-only

授权内容：

- 文档、矩阵、离线合成样本、报告型 checker。

不打开：

- F-330 全部继续 future gate。

适用：

- v3.4 继续 hardening 或 defer。

## Bundle 1：最小 transient proposal-only runtime

授权内容候选：

- L2/L3 小区域。
- transient AgentProposal。
- WorldCore post-check。
- expression-only 输出。
- 不写 save，不写 canon，不写地点/奖励/NPC 生死。

必须单独审批：

- F-330-001。

仍保持 future gate：

- F-330-002 save field / `SAVE_FORMAT_VERSION` / `runFingerprint`。
- F-330-004 backend/BFF/service。
- F-330-007 external framework PoC/dependency。
- F-330-009 L4/L5 runtime。
- F-330-010 self-learning direct write。

可能需要另批：

- 若调用 live DeepSeek，打开 F-330-003。
- 若触及真实 lore / 命名 NPC / hidden-adjacent，打开 F-330-005。

## Bundle 2：live DeepSeek 表达与意图候选

授权内容候选：

- `deepseek-v4-flash` 只生成语言、意图候选、局部反应候选。
- DeepSeek 不裁决事实、数值、奖励、地点、NPC 生死、隐藏事实、save 写入。

必须单独审批：

- F-330-003。

配套要求：

- 每份 Player Advocate 明确写：是否调用 live DeepSeek、模型、样本、轮次、成本、报告路径。
- P0/P1 直接停机。
- prompt/context diff 必须可审计。

## Bundle 3：真实 lore / MiroFish blocking

授权内容候选：

- RebornG request -> MiroFish export -> RebornG intake review -> rule/test/candidate。

必须单独审批：

- F-330-005。

继续禁止：

- MiroFish 基础包直接进 runtime、prompt、DeepSeek visible context、知识库正文或 canon。

## Bundle 4：持久化差异与 save-format

授权内容候选：

- run identity / persistent variation / agent memory summary / ledger。

必须单独审批：

- F-330-002。

配套要求：

- migration/defaults/old-save/rollback。
- long replay。
- pollution cleanup。

## Bundle 5：后端、外部框架、自学习

当前建议：

- 不随最小 runtime first cut 一起打开。
- 继续保持 future gate。

必须单独审批：

- F-330-004 backend/BFF/service。
- F-330-007 external framework PoC/dependency/read-only scan/subagent。
- F-330-010 self-learning direct write 永久禁止，候选自学习也需另做风险报告。

## a2 结论

未来若要开第一刀，最小可讨论包是 Bundle 1，且默认不包含 live DeepSeek、MiroFish、save-format、backend、external framework 或 self-learning。任何扩大都必须逐项审批。
