# v3.3.0-b2 PA/live/drift/rollback/old-save 测试档位

日期：2026-05-24
状态：completed
冻结模型：`v330_b2_runtime_agent_test_tiers_v1`

## 目标

b2 给出未来若打开 F-300-001 时的测试档位。v3.3 自身不运行这些测试，因为没有 runtime 改动。

## 档位 T0：report-only / docs-only

适用：

- v3.3 当前版本。

要求：

- 文档自检。
- 敏感词边界扫描。
- skill sync audit。
- Git/CI。

不需要：

- Player Advocate。
- live DeepSeek。
- runtime 测试。

## 档位 T1：transient proposal-only 第一刀

适用：

- 未来单独批准 F-330-001，且不打开 save-format / live DeepSeek。

要求：

- focused unit：AgentProposal schema、WorldCore post-check、rejection families。
- old-save smoke：旧档打开不变。
- rollback smoke：agent proposal rejected 后状态不变。
- Player Advocate：至少 10 轮。
- deterministic replay：至少 30 轮。
- boundary scan：无事实写入、无 save 写入、无 formal credential。

live DeepSeek：

- 默认不跑。

## 档位 T2：player-facing transient + live DeepSeek

适用：

- 未来同时批准 F-330-001 与 F-330-003。

要求：

- T1 全部要求。
- Player Advocate：至少 50 轮。
- live probe：`deepseek-v4-flash`，建议 3-5 样本，每样本 6-10 轮。
- drift checkpoint：每 10 轮检查事实、目标、地点、NPC 权限、隐藏事实。
- 成本记录：模型、样本、轮次、输入输出 token、费用、报告路径。

停机：

- P0/P1 立即停机。
- DeepSeek 输出越权、hidden leak、正式事实裁决、NPC 生死裁决，立即停机。

## 档位 T3：persistent memory / save-format

适用：

- 未来批准 F-330-002。

要求：

- T2 或同等强度。
- migration/defaults/unit。
- old-save matrix。
- rollback matrix。
- long replay 150-300 轮。
- memory pollution / stale summary / same-start replay 差异检查。

建议：

- 不作为 v3.4 第一刀。

## v3.4 成本预估

若只做 T1，主要成本是工程与 QA 时间。若进入 T2，DeepSeek token 成本预计仍低，但评审成本会上升；关键成本不是钱，而是样本设计、P0/P1 复核、prompt/context diff 审计。

## b2 结论

未来最小 implementation 的测试底线是 T1。只要调用 live DeepSeek，就必须升级 T2。只要写存档或持久记忆，就必须升级 T3。
