## 专家团共识

v3.3 可以开，但不能把“开会讨论 F-300”误读为“已经打开 F-300”。v3.3 的价值在于把下一步分岔讲透：

- 继续 report-only hardening。
- 打开最小 runtime first cut。
- 打开更重的 save/live/backend/MiroFish/external framework 配套。

专家团当前倾向：v3.3 先做决策包，不实现。若未来用户决定打开最小 runtime 第一刀，建议只考虑 L2/L3 小区域、proposal-only、WorldCore post-check、expression-only、可回滚、旧档无害、可被 Player Advocate 和 live probe 观察的最小路径。

## 角色意见

### Project Lead

v3.3 应把用户要拍板的东西一次性列清楚，避免后续每个小阶段再碎片化审批。决策包必须明确“批准不等于什么”，尤其不等于 F-300-001 已开。

### Systems Architect

最小 runtime first cut 如果未来获批，应优先考虑 transient proposal path，不先加持久字段。只要引入 agent memory、same-start persistent variation、agent event ledger 或 replay fingerprint，就会触发 save-format / migration / old-save 风险，必须单独打开 F-300-002。

### AI Pipeline Architect

runtime agent 可以先不调用 live DeepSeek，用 synthetic / deterministic proposal rehearsal 验证链路。如果要体现同开局差异和 NPC 意图鲜活度，live DeepSeek 最终大概率要进场，但它必须被约束为意图候选和表达候选，不能获得事实、奖励、地点、NPC 生死、隐藏事实或存档写入权。

### Lore & World Designer

v3.3 当前 MiroFish 为 `not_needed`。一旦用户希望 runtime first cut 涉及命名 NPC、真实势力、方源公开证据、hidden-adjacent、L4/L5 锚点、正式 lore 结论，必须升级为 blocking topic-slice intake。

### QA & Release Guardian

如果未来 F-300-001 被打开，验收不能只靠单测。至少需要 old-save / rollback / deterministic replay / Player Advocate / live probe / drift checkpoint 的组合。P0/P1 必须停机，不能靠措辞降级。

### Player Advocate

玩家能感知的收益应是“NPC 意图与机会窗口更鲜活，同开局有可解释差异”，不是“世界事实随机变化”。每个玩家选择和 NPC 反应的后果仍必须让玩家看得懂。

### Skill Sync Steward

本次 startup 与 rc 均触发 expert / game-dev / lore current override 更新。combat-motion 和 mirofish-export 暂不需要更新，因为 v3.3 不进入战斗表现或 MiroFish export。
