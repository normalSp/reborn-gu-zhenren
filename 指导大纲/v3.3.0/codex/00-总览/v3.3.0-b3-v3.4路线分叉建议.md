# v3.3.0-b3 v3.4 路线分叉建议

日期：2026-05-24
状态：completed
冻结模型：`v330_b3_v34_route_split_v1`

## 目标

b3 为 v3.4 给出三条路线，供用户下一步决策。

## 路线 1：继续 hardening

内容：

- 不开 runtime。
- 继续扩展 report-only rehearsal、manual review、negative fixtures、same-start variance。

适合：

- 用户希望再压低风险。
- 仍担心 player-facing agent 表达误导玩家。

收益：

- 安全。
- 不触发 F-330。

代价：

- 游戏体验继续停留在投影/报告型证据。

## 路线 2：最小 transient proposal-only 第一刀

内容：

- 未来用户单独批准 F-330-001。
- L2/L3 小区域。
- AgentProposal 只生成候选。
- WorldCore post-check 裁决。
- expression-only，不写事实、不写存档。
- 不调用 live DeepSeek，不做 MiroFish，不引入后端。

收益：

- 第一次让玩家感到 NPC 意图和同开局差异更鲜活。
- 保持 save 风险最低。

风险：

- 需要高强度测试和 Player Advocate。
- 需要非常清楚的 UI/文案表达，避免候选被误认为事实。

## 路线 3：继续 defer 到 v3.5+

内容：

- 暂不做 v3.4 runtime agent。
- 转向高阶战斗、天道/宿命、agent memory 污染、external framework PoC 的更长线评估。

收益：

- 避免过早进入 player-facing agent。

代价：

- 与“agent 进入游戏”的目标距离拉长。

## 专家团建议

推荐 v3.4 启动会先让用户在路线 1 和路线 2 之间选择。

专家团倾向：

- 如果用户愿意承担受控 player-facing 风险，选择路线 2。
- 如果用户希望继续保持零 runtime 风险，选择路线 1。
- 不建议路线 3，除非 v3.4 前发现新的架构或边界问题。

## 必须再次审批

路线 2 必须单独审批 F-330-001，并重新确认 F-330-002 至 F-330-012 是否继续 future gate。v3.3 不替 v3.4 授权。
