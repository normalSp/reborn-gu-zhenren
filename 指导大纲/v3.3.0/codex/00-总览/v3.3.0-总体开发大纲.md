# v3.3.0 总体开发大纲

日期：2026-05-24
状态：startup completed；waiting D-330 approval

## 目标

v3.3 的目标是完成 F-300 reopen 决策包，让用户能判断下一步是否进入最小 runtime agent 第一刀。

本版本不默认实现 runtime agent。它只把实现前的关键选择拆清楚：

- 打开还是不打开 F-300-001。
- 是否需要 save-format / `runFingerprint`。
- 是否需要 live DeepSeek。
- 是否需要 MiroFish blocking。
- 是否需要 backend/BFF。
- 是否允许 external framework PoC。
- 测试、Player Advocate、live probe、rollback、old-save、long drift 的最小档位。

## 候选路线

| 路线 | 内容 | 收益 | 风险 |
|---|---|---|---|
| A | 继续 report-only hardening | 最稳，延续当前安全曲线 | 玩家体验不会立刻变强 |
| B | 未来打开最小 transient L2/L3 runtime proposal path | 可首次让 agent 候选影响表达链 | 需要强 post-check、PA、rollback、live/drift 档位 |
| C | 未来打开 persistent memory / runFingerprint / save-format | 长线差异更强 | 存档迁移、旧档、回滚、污染风险显著上升 |

专家团 startup 建议：v3.3 只做决策包；若未来打开 implementation，优先评估路线 B，不先做路线 C。

## 默认阶段

- startup：启动会与授权包。
- a0：v3.0-v3.2 证据复盘与 F-300 reopen 范围冻结。
- a1：最小 runtime first cut 选项分析。
- a2：配套 future gate bundle 风险收益。
- b1：report-only F-300 reopen 决策矩阵。
- b2：测试档位、成本、失败处理。
- b3：v3.4 路线分叉建议。
- process/rc：治理、知识边界、skill sync、质量收束。

## 不做

本版本不写 runtime，不跑 live probe，不做 MiroFish，不引入后端或外部依赖，不做 save-format。
