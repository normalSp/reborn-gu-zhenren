# v3.3.0-a1 最小 runtime first cut 选项分析

日期：2026-05-24
状态：completed
冻结模型：`v330_a1_minimal_runtime_first_cut_options_v1`

## 目标

a1 比较 F-300 reopen 的三条路线，给出收益、风险、需要打开的 gate 和专家团推荐。

## 路线 A：继续 report-only hardening

内容：

- v3.4 继续做离线/报告型 rehearsal。
- 不接 runtime，不改 UI，不改 prompt，不改存档。
- 扩大样本、负样本、manual review、same-start variance 证据。

收益：

- 风险最低。
- 不触发 save、live DeepSeek、MiroFish、backend、external framework。
- 适合继续压低 false negative 和 boundary drift。

风险：

- 玩家体验没有立刻增强。
- 长期可能陷入“只评估不落地”。

需要 gate：无新增 F gate。

## 路线 B：最小 transient proposal-only runtime 第一刀

内容：

- 未来若用户单独批准，只允许 L2/L3 小区域生成 transient AgentProposal。
- AgentProposal 只进入候选和表达链，不写事实、不写存档、不写 runtime canon。
- WorldCore post-check 保持最终裁决权。
- 表达层可让 NPC 意图、机会窗口、同开局差异更鲜活。

收益：

- 是第一个能被玩家感知的 agent runtime 试点。
- 不必先引入持久记忆或 save-format。
- 能验证“agent 候选 + WorldCore 裁决 + 表达链”的真实体验。

风险：

- 即使 transient，也会触发 player-facing 风险。
- 需要更强 Player Advocate、rollback、old-save、drift、live probe。
- 如果表达链让玩家误以为事实已成立，会产生权威漂移。

需要 gate：

- 必须单独打开 F-300-001 / F-330-001。
- 如果调用 live DeepSeek，还需打开 F-330-003。
- 如果触及真实 lore / 命名 NPC，还需打开 F-330-005。

## 路线 C：persistent memory / runFingerprint / save-format 路线

内容：

- 引入持久同开局差异、agent memory、agent event ledger、run fingerprint 或 save field。

收益：

- 长线差异最强。
- 能更接近“每次同开局也不同”的终局目标。

风险：

- 触发 save-format、migration、old-save、rollback、污染清理、长期记忆漂移。
- 一旦设计不稳，后续版本会背负沉重兼容债。
- 可能过早把 agent 错误固化进存档。

需要 gate：

- F-330-001。
- F-330-002。
- 大概率 F-330-003、F-330-004、F-330-010。

## 专家团推荐

当前不建议在 v3.3 打开任何 implementation。

若用户未来希望 v3.4 开第一刀，专家团推荐只考虑路线 B 的最小 transient proposal-only 试点，并把它拆成单独审批包。路线 C 不应在 B 跑通前打开。
