# v3.4.0 MiroFish 资料需求与交付协议

日期：2026-05-24
状态：startup
当前 need level：`not_needed`

## 当前判断

v3.4 startup 只做路线选择和授权包，不使用真实原著事实、命名 NPC、真实势力、hidden-adjacent、方源公开证据、L4/L5 fact anchor 或正式 lore 结论，因此 MiroFish 当前为 `not_needed`。

## Route 2 下仍保持 not_needed 的条件

如果后续用户选择 Route 2，只有在以下范围内才继续不需要 MiroFish：

- synthetic / RebornG-owned 泛化小区域样本。
- 非命名 NPC。
- 抽象小势力压力对象。
- 无真实原著地点/势力/人物结论。
- 无 hidden-adjacent 内容。
- 无 formal lore conclusion。

## 升级 blocking 的触发器

触发任一项时，MiroFish 必须升级为 `blocking`：

- 命名 NPC。
- 真实原著势力或真实原著地点。
- 方源公开证据。
- hidden-adjacent material。
- L4/L5 fact anchor。
- 天道/宿命真实原著锚点。
- 正式 lore 结论。
- knowledge-index body。
- DeepSeek visible lore/RAG。
- formal location/faction/identity/reward/NPC life-death。

## 交付链

若升级 blocking，必须走：

`RebornG request -> MiroFish export -> RebornG intake review -> rule/test/candidate material`

不得直接把 MiroFish 基础包、hidden body 或原著正文导入 runtime、UI、prompt、DeepSeek visible context 或 knowledge-index body。
