# v3.4.0 总体开发大纲

日期：2026-05-24
状态：startup；route pending

## 目标

v3.4 的第一目标不是直接写代码，而是让用户在 report-only hardening 与最小 runtime first cut 之间做一次清晰决策。

若用户选择路线 2，后续 v3.4 才进入最小 transient proposal-only runtime first cut。

## 路线 1：report-only hardening

候选阶段：

- a0：复盘 v3.3 决策包和 v3.0-v3.2 checker 证据。
- a1：扩展 same-start variance / manual review / negative fixtures。
- a2：更新 v3.5 runtime first cut 准入包。
- b1：report-only checker 增量。
- b2：Player Advocate / drift / rollback 档位再校准。
- rc：决定是否把 route 2 推到 v3.5。

## 路线 2：最小 transient proposal-only runtime first cut

候选阶段：

- a0：F-330-001 / F-340-001 窄口授权确认。
- a1：runtime first cut contract 门禁：AgentProposal transient-only、WorldCore post-check、expression-only。
- a2：UI/文案理解门禁：候选不等于事实，rejected proposal 不显示为世界事实。
- b1：第一刀 runtime 接入，只允许本地 deterministic proposal path，不调用 live DeepSeek。
- b2：post-check rejection / rollback / old-save / no-save-write 验证。
- b3：Player Advocate 10 轮和 deterministic 30 轮。
- rc：质量收束，决定 v3.5 是否扩展或回退。

## 路线 3：defer

候选阶段：

- a0：记录 defer 原因。
- a1：转向高阶战斗或 HeavenWill/Fate 预备讨论。
- a2：确认 runtime agent 回到 v3.5+。
- rc：更新长期路线。

## 专家团推荐

推荐路线 2。理由：

- v3.0-v3.3 已经完成足够多 report-only 证据。
- 长期路线原本就把 v3.4-v3.6 设为最小 L2/L3 小区域试点窗口。
- transient-only 可以避免 save-format、migration、live DeepSeek、MiroFish、backend 等高风险项。
- 真实 runtime 摩擦必须尽早验证，否则继续 report-only 会开始失真。

## 不建议打开的内容

即使选择路线 2，也不建议 v3.4 打开：

- agent memory save。
- `runFingerprint`。
- live DeepSeek。
- named NPC / real faction / hidden-adjacent。
- formal location / faction / identity / reward / NPC life-death。
- backend/BFF。
- external framework PoC。
- L4/L5 runtime。
- high-rank combat runtime。
