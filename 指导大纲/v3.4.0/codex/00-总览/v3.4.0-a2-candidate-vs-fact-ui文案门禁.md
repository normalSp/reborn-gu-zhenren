## UI 原则

v3.4 是第一次让 runtime agent 相关对象进入玩家可见界面，因此文案必须把“候选”和“事实”分开。

硬规则：

- 面板标题可以写 `Runtime AgentProposal`，但必须说明它是候选表达。
- 所有卡片必须写明：候选表达，不是事实。
- 不使用“已加入、已获得、已死亡、已通缉、已封锁”等完成态文案。
- 不暗示 L2/L3 拥有裁决权。
- 不展示命名 NPC、原著关键人物、hidden/private body。
- 不把 rejected proposal 的原始敏感内容显示给玩家，只显示被阻断的类别和边界原因。
