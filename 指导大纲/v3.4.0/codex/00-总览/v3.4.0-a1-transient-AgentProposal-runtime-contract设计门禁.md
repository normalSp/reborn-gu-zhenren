## Contract

`AgentProposal` 在 v3.4 中不是 agent 自治体，也不是存档状态。它只是本地 runtime 中的一个临时候选表达对象。

最小字段：

| 字段 | 含义 |
|---|---|
| `id` | deterministic candidate id |
| `layer` | 只允许 `L2` / `L3` |
| `agentRole` | 泛化角色，不是命名 NPC |
| `areaKey` | 只允许 `southern_border_outer_edge_low_rank` |
| `persistenceMode` | 固定 `transient_proposal_only` |
| `intent` | 候选意图 |
| `publicExpression` | 玩家可见表达候选 |
| `safeNextStep` | 不结算的下一步建议 |
| `candidateEffects` | 必须为空或只含非权威候选；任何 forbidden effect 阻断 |
| `evidenceRefs/sourceRefs` | 只能指向现有公开/合成/项目文档证据 |
| `audit` | proposal-only、expression-only、no-save、no-live metadata |

## WorldCore post-check

post-check 决策：

| decision | 含义 |
|---|---|
| `approved_expression_candidate` | 可显示为候选表达 |
| `rejected_violation` | 越权，不能显示为候选 |
| `needs_user_decision` | 触发 future gate，需要用户单独审批 |

P0 阻断：

- non-L2/L3。
- 非单一区域。
- 非 transient persistence。
- save write / save field / migration / `runFingerprint`。
- live DeepSeek / prompt / context / model / authority change。
- MiroFish / named NPC / hidden-adjacent / formal lore。
- backend / external framework / dependency / subagent。
- L4/L5 / Heaven-Fate verdict。
- formal location/faction/identity/reward/NPC life-death/warrant/recruitment/blockade。
