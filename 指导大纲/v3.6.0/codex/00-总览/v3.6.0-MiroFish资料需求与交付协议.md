# v3.6.0 MiroFish 资料需求与交付协议

日期：2026-05-25
状态：startup

## Need Level

`not_needed`

## 理由

v3.6 只做 synthetic / RebornG-owned generic L2/L3 micro-expansion hardening：

- 不使用真实原著人物。
- 不使用真实势力或正式地点。
- 不使用 hidden-adjacent、方源公开证据或 L4/L5 锚点。
- 不写知识库正文或 runtime canon。
- 不把 MiroFish 内容放入 DeepSeek visible context。

## 升级触发器

以下任一项出现，MiroFish 必须升级为 `blocking`：

- 命名 NPC。
- 真实原著势力、地点、事件或派系结论。
- hidden-adjacent、方源公开证据、L4/L5 source pointer。
- 正式 lore 结论、canon promotion、知识库正文。
- DeepSeek visible lore/RAG。

## 交付协议

若未来触发 blocking，必须走：

`RebornG request -> MiroFish export -> RebornG intake review -> test matrix / rule draft`

MiroFish 输出仍只能进入 candidate/rule/test/deferred，不得直接进入 runtime truth、UI、prompt 或 save。
