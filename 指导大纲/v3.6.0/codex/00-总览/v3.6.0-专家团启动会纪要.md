# v3.6.0 专家团启动会纪要

日期：2026-05-25
状态：completed
主题：v3.5 lifecycle v2 后，是否微扩 synthetic L2/L3 候选 lane

## 启动前证据

- v3.4 已完成最小 `transient proposal-only` runtime first cut。
- v3.5 已完成 lifecycle v2、copy guard、20 轮 Player Advocate、60 轮 deterministic drift。
- v3.5 completion commit `e3b004dd` 已推送，GitHub Actions run `26368124209` passed。
- 当前 `意图` tab 仍不写 store、save、prompt、canon，不调用 live DeepSeek，不使用 MiroFish。

## 路线 A：微扩硬化（推荐）

继续使用 v3.5 窄口，只增加 synthetic / RebornG-owned generic L2/L3 lane：

- 多个不同低阶场景压力下的 NPC 意图候选。
- 多个 rejection / expired / needs_user_decision 负样本。
- 同开局差异只表现为候选 lane 差异，不改变 WorldCore 事实。
- 90 轮 deterministic 证明不需要 `runFingerprint` 也能提供稳定可审计差异。

收益：

- 比原地硬化更接近未来 v3.7/v3.8 多 NPC 小势力扩展。
- 仍保持低风险：没有真实 lore、没有持久 agent、没有 DeepSeek live。
- 玩家能更早感受到“同样开局但候选表达不同”的雏形。

风险：

- UI 信息量会增加，需要继续压 copy guard。
- synthetic 样本可能让体验看起来更“实验室”，因此必须明确这不是正式 NPC 社会。

## 路线 B：继续原地硬化

不增加候选广度，只继续补 rejection、rollback、old-save、copy guard。

收益是最稳；风险是 v3.x 进展会偏慢，对“同开局差异度”的信息增量不足。

## 路线 C：准备 live probe 决策包

不实装，先设计小规模 live DeepSeek probe 的成本、样本和接受标准。

收益是能更快触碰真实叙事质量；风险是 v3.6 会从 runtime 窄口硬化转向 DeepSeek 专项，且当前没有必要。

## 专家团建议

选择路线 A。v3.6 应以“微扩但不越权”为目标：扩大 synthetic/generic lane 的覆盖面，暂不打开 live DeepSeek、MiroFish、persistent agent、backend 或外部框架。
