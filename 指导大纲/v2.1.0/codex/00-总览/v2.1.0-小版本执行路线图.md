## 当前建议

当前 a2 已完成。下一步把 D-211 go/no-go 决策项交给用户，不进入 b1。

原因：

- b1 如果写 runner，会接触脚本、样本、eval archive，虽仍可保持 report-only，但已不是单纯文档。
- 是否调用 DeepSeek、是否引入 SDK、是否使用只读子代理，都需要用户在 a2 后统一拍板。
- 是否允许 LangGraph、Mastra、OpenAI Agents SDK、Google ADK 等任一框架做隔离 PoC，也必须等 a2 后统一拍板。

专家团默认推荐：

- 批准 D-211-001、002、003、005、010、012：做 TypeScript-first、zero-dependency、dry-run only、report-only offline runner，并允许写 `artifacts/v2.1.0/` 报告。
- 暂缓或拒绝 D-211-004、006、007、008、009、011：不在 b1 调 live DeepSeek，不做框架 PoC，不给外部 SDK 文件/命令/git 权限，不启用子代理，不做 memory persistence，不启动 BFF/private canon/job queue。

## 阻断条件

如果任何阶段试图做以下事项，必须停止并新增设计门禁：

- runtime agent。
- 引入任何 agent SDK / framework 依赖。
- BFF/backend。
- DeepSeek prompt/context/model/authority 变化。
- save-format bump。
- `runFingerprint` 或其他新持久字段。
- live DeepSeek 批量 eval。
- 子代理参与 RebornG 文件、runtime、canon 或 git。
- MiroFish topic-slice。
- 正式地点、阵营、奖励、NPC 生死或 canon promotion。
