# v2.1.0 专家团启动会纪要

状态：D-210-001 至 D-210-010 已批准；a0/a1 completed，当前进入 a2。
日期：2026-05-22
主题：Agent Simulation Lab 启动版与 Claude Code / 开源 Agent 框架架构尽调。

## 开会背景

v2.0 已完成第一个区域活世界第一阶段：`regionalEventLedger`、WorldCore 链路、同开局 replay lane、旧档/回滚 review、P2/P1/P0 retry hardening 和 T3 320 轮硬门均通过。

但 v2.0 T3 也暴露出下一阶段的真实问题：

- DeepSeek 对正式凭信词、准入、身份、名单、通行一类表达有自然越界倾向。
- 单区域样本已经能守住，但样本覆盖还不代表完整南疆、完整商家城或高阶区域。
- retry 和 eval 已经证明有用，需要升格成 Agent Lab 的基础设施。
- 未来多 agent 世界不能靠一个大 prompt，也不能靠每个 NPC 都 live DeepSeek。

## 专家团总建议

推荐 v2.1 主线：

`Agent Simulation Lab 启动版`

一句话目标：

> 先把多 agent 时代的权限、上下文、工具、记忆、proposal、WorldCore post-check、eval farm 和成本观测设计清楚，再决定是否写离线 runner。

## Claude Code 与开源 Agent 框架判断

专家团建议：做专项尽调，但只吸收架构模式。

可参考：

- Agent SDK / agent loop。
- tool permission / approval。
- hooks。
- sessions。
- MCP 工具接口。
- subagents 的隔离和职责描述。
- plugin / skill 的分层知识注入。
- LangGraph / Mastra 的 workflow、state graph、eval、MCP 思路。
- OpenAI Agents SDK / Google ADK / PydanticAI 的 guardrails、tracing、typed proposal、human-in-loop 思路。
- AutoGen / CrewAI 的多 agent 消息与流程分离思路。
- Letta 的长期记忆风险样本。
- OpenHands 的 agent server、sandbox、RBAC 产品形态。

不可采用：

- 不复制反编译、泄露或未授权源码。
- 不把 Claude Code 的工具执行模型直接当 RebornG runtime agent。
- 不让工具代理写 RebornG canon、save、runtime、DeepSeek context 或 git。
- 不把 subagent 当作 NPC agent 的实现捷径。
- 不引入 LangGraph、Mastra、OpenAI Agents SDK、Google ADK、AutoGen、CrewAI、PydanticAI、Letta、OpenHands 依赖。
- 不让任何外部框架接管 WorldCore、hidden/private facts、save、canon 或 NPC 生死。

## v2.1 好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| Agent Lab 先离线 | 可以测试 20 NPC / 3 势力 / 1 宏观导演，不污染存档 | b1 是否写 runner 需 a2 后单独批准 |
| `AgentProposal` 标准 envelope | 把 agent 输出限定为候选、记忆、压力、表达 | 必须强制 `requiresWorldCoreAdjudication=true` |
| Claude Code permission 模式 | 可转成 RebornG 的 tool/canon/save/DeepSeek 权限矩阵 | 不复制源码，只吸收思想 |
| 开源 Agent 框架吸收矩阵 | 防止只盯 Claude Code，把 LangGraph/Mastra/ADK/AutoGen 等纳入可比口径 | 只做文档尽调，不引入依赖或 PoC |
| T3 retry hardening 迁移到 Agent Lab | 让正式凭信词、hidden echo、schema gap 进入 agent eval | 先做文档和样本，不跑 live |
| Session / memory 分层 | 区分当前场景记忆、NPC 记忆、势力记忆、长期世界账本 | 不能把 hidden/private body 暴露给 DeepSeek |
| L5 宏观导演只输出 pressure | 天道/宿命可以作为压力场存在 | 不作为可聊天 NPC，不宣判结局 |

## 不推荐现在做

- 不做 runtime agent 接入。
- 不做 BFF/backend。
- 不做 Claude Code SDK 原型。
- 不做任何第三方 Agent framework PoC。
- 不启用 Codex/Claude 子代理写文件。
- 不新增 save 字段。
- 不新增 DeepSeek visible knowledge/RAG。
- 不写知识库正文。
- 不新增 MiroFish export。

## 专家团结论

建议把 D-210 作为 v2.1 启动决策池，待用户批准后再进入 a0/a1/a2：

1. v2.1 主线为 Agent Simulation Lab 启动版。
2. 第一批只做启动会、T3 复盘、Claude Code 与开源 Agent 框架架构尽调、Agent Lab 设计门禁。

4. b1 是否写 report-only/offline runner，由 a2 后另行决策。
