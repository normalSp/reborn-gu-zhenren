# v2.1.0 总体开发大纲

状态：draft；等待 D-210 批准。
日期：2026-05-22
主题：Agent Simulation Lab 启动版。

## 核心目标

把未来多 agent 世界从长期愿景推进到工程可执行的实验室方案：

- agent 如何分层。
- agent 能看见什么。
- agent 能输出什么。
- WorldCore 如何 post-check。
- eval farm 如何发现越权、漂移、重复、空转和 hidden leak。
- Claude Code 与开源 Agent 框架的工具治理、workflow、memory、schema 和 eval 架构哪些可借鉴，哪些不能碰。

## 版本结构

| 阶段 | 目标 | 是否改 runtime |
|---|---|---|
| a0 | v2.0 T3 复盘与 Agent Lab 范围冻结 | 否 |
| a1 | Claude Code 与开源 Agent 框架架构尽调 | 否 |
| a2 | Agent Lab 设计门禁草案 | 否 |
| b1 候选 | report-only/offline runner 第一刀 | 待用户决策 |
| b2 候选 | agent proposal eval matrix | 待用户决策 |
| process 候选 | agent 权限/成本/审计流程固化 | 待用户决策 |
| rc 候选 | Agent Lab 文档/runner 质量收束 | 待用户决策 |

## a0 输出

- v2.0 T3 复盘。
- retry / formal credential / hidden protection / schema gap 经验提炼。
- Agent Lab 只做离线设计的范围冻结。
- D-210 决策池。

## a1 输出

- Claude Code 与开源 Agent 框架架构尽调。
- 官方资料来源索引。
- 可吸收模式：agent loop、tool permission、hooks、sessions、MCP、subagents、plugin/skill、workflow graph、schema/guardrails、tracing/eval、long-term memory 风险样本。
- 不可吸收内容：反编译/泄露/未授权源码、直接源码改造、直接 runtime 接入。
- Agent 框架吸收矩阵：Claude Code、LangGraph、Mastra、OpenAI Agents SDK、Google ADK、AutoGen、CrewAI、PydanticAI、Letta、OpenHands。
- RebornG 转译：Agent Orchestrator、WorldCore post-check、权限矩阵、eval archive。

## a2 输出

- L0-L5 agent 层级。
- `AgentProposal` envelope 草案。
- 场景样本类型。
- eval 指标。
- failure taxonomy。
- WorldCore post-check 清单。
- b1 是否写离线 runner 的 go/no-go 决策项。

## 成功标准

- 用户能明确判断 v2.1 b1 要不要写离线 runner。
- Claude Code 与开源 Agent 框架参考边界被写成项目制度，不停留在对话里。
- Agent Lab 不会被误解成已批准 runtime agent。
- 所有未来 agent 输出必须走 WorldCore post-check 的原则写入文档。
