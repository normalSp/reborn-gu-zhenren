# v2.1.0-b1 Agent Lab report-only offline runner 第一刀

状态：completed
日期：2026-05-23

## 入场依据

b1 基于 D-211 与 D-213 执行：

- D-211 批准自有 TypeScript-first / zero-dependency / dry-run only / report-only offline runner。
- D-213 批准 v2.1 在一个 goal 内连续完成 b1/b2/rc，但不扩大权限。

本阶段不做：

- runtime / UI / store / prompt 改动。
- save-format bump 或持久字段。
- live DeepSeek。
- 外部 agent SDK/framework 依赖或 PoC。
- 子代理。
- BFF/backend。
- MiroFish export/intake。
- 正式地点、阵营、奖励、NPC 生死或 canon promotion。

## 实现内容

新增自有离线 runner：

- `scripts/run-v210-agent-lab-offline-runner.mjs`
- npm script：`npm run check:v210-agent-lab-offline`

新增合成样本：

- `tests/evals/v210-agent-lab/samples.json`

runner 只读取合成样本，验证：

- `AgentProposal` envelope schema。
- `agentLayer` / `proposalKind` / `confidence` 枚举。
- source refs / visible fact refs 是否在样本 allowlist 内。
- hidden refs 是否 ref-only 且需要 user decision。
- candidate effect 是否只用允许的候选类型。
- forbidden keys / forbidden effect / hard forbidden text 是否出现。
- WorldCore post-check 是否输出预期结果：`accepted_candidate` 或 `needs_user_decision`。

## b1 样本覆盖

当前合成样本覆盖：

- L0 public event candidate。
- L1 patrol / low-level pressure。
- L2 memory reflection。
- L2 squad combat tactic candidate。
- L3 scene NPC intent。
- L3 narrative expression。
- L5 macro pressure。
- L4 key figure gate。
- hidden ref-only gate。
- external SDK permission ladder。

这些样本是抽象/合成样本，不引入新原著对象，不需要 MiroFish。

## 报告口径

runner 输出 `artifacts/v2.1.0/agent-lab-offline-runner/<timestamp>/report.json` 与 `summary.md`。

报告是 evidence/archive，不是 runtime authority：

- 不写 save。
- 不写 canon。
- 不写 DeepSeek context。
- 不调用 live DeepSeek。
- 不使用外部 agent framework。
- 不启用子代理。
