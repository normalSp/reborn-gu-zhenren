# v2.1.0-a2 Agent Lab 设计门禁草案

状态：draft；等待 D-210 批准后执行。
日期：2026-05-22

## 设计目标

Agent Lab 的第一版不是 NPC runtime，而是离线实验室设计：

- 证明 agent 可以产出有用 proposal。
- 证明 proposal 不污染事实。
- 证明 WorldCore 能拒绝越权。
- 证明 eval 能捕获漂移、泄露、重复和正式结论暗示。

## Agent 分层

| 层级 | 对象 | v2.1 处理 |
|---|---|---|
| L0 | 地点、资源、天气、事件账本 | 纯规则，不调用 LLM |
| L1 | 路人、商贩、巡逻、兽群 | 行为树/效用 AI 草案 |
| L2 | 次要 NPC、小队成员、小势力 | 结构化记忆与批处理反思草案 |
| L3 | 当前场景关键 NPC | live 候选设计，不执行 |
| L4 | 原著关键人物、大势力核心 | 默认不进入 v2.1 Lab；需 blocking intake |
| L5 | 天道、宿命、时代大势 | 只设计 pressure，不作为聊天 NPC |

## AgentProposal 草案

长期建议 envelope：

```ts
type AgentProposal = {
  proposalId: string
  agentId: string
  agentLayer: "L0" | "L1" | "L2" | "L3" | "L4" | "L5"
  sourceEventRefs: string[]
  visibleFactRefs: string[]
  hiddenFactRefs?: string[]
  proposalKind:
    | "memory_reflection"
    | "npc_intent"
    | "faction_pressure"
    | "region_event_candidate"
    | "combat_tactic_candidate"
    | "heaven_will_pressure"
    | "narrative_expression"
  publicSummary: string
  privateSummaryRef?: string
  candidateEffects: unknown[]
  forbiddenEffects: string[]
  confidence: "low" | "medium" | "high"
  requiresWorldCoreAdjudication: true
}
```

注意：这是 a2 设计门禁草案，不是当前 runtime 类型。

## WorldCore post-check

每个 proposal 必须检查：

- source refs 是否存在。
- visible refs 是否允许当前 agent 看到。
- hidden refs 是否只作为 ref，不含 body。
- candidateEffects 是否越过正式地点/阵营/奖励/NPC 生死。
- publicSummary 是否暗示正式凭信、正式身份、正式通行或正式奖励。
- 是否试图写 DeepSeek visible knowledge / RAG。
- 是否需要用户决策或 MiroFish blocking intake。

## 场景样本族

| 样本族 | 用途 |
|---|---|
| formal credential | 测凭信、名单、身份、准入、通行暗示 |
| hidden raw input | 测玩家输入中隐藏/受保护词回显 |
| repeated social pressure | 测 NPC/势力反应是否空转复读 |
| agent memory reflection | 测反思是否只写候选和摘要 |
| faction pressure | 测势力是否生成正式通缉/招揽结论 |
| L5 pressure | 测天道/宿命是否只给宏观压力 |

## b1 go/no-go 问题

a2 完成后需要用户决策：

1. 是否写 report-only/offline runner。
2. 是否允许 dry-run only。
3. 是否允许 live DeepSeek eval。
4. 是否允许只读/分析型子代理做 coverage 审查。
5. 是否把 Agent Lab 输出写入 `artifacts/v2.1.0/`。

## a2 结论

在这些问题获批前，Agent Lab 只停留在设计门禁，不进入实现。
