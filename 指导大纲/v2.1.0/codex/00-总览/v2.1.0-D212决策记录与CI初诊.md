## 隔离 PoC 是什么

“隔离 PoC”是把某个开源 SDK/agent 当作实验对象，在与 RebornG 主仓严格隔离的环境里跑一个最小验证。

它允许验证：

- 这个框架的 workflow、state graph、memory、tool permission、eval、tracing 是否真的适合 RebornG。
- 它的输出能否被转写成 RebornG 自有 `AgentProposal`。
- 它是否容易产生越权、幻觉事实、hidden/private 泄露、长期记忆污染或不稳定重试。
- 它的工程成本、维护成本、依赖风险和 license 风险是否值得继续。

它必须禁止：

- 读取或写入 RebornG 主仓真实文件。
- 读取 `.env`、DeepSeek key、私有日志、SQLite、plugin cache、hibernation 或未授权私有文件。
- 操作 RebornG git、生成 commit、执行项目命令或修改 lockfile。
- 接入 runtime、store、save、canon、MiroFish intake、知识库正文或 DeepSeek visible context。
- 使用真实原著 hidden/private body 或玩家正式存档。

推荐形式：

- 在临时目录或单独实验仓运行。
- 只使用合成 scenario、假 NPC、假 faction、假 WorldCore 输入。
- 输出一份 report：能力、失败、越权样本、依赖、成本、是否值得继续。
- 若需要和 RebornG 对照，只复制最小 schema 形状或 mock，不复制真实项目文件。

一句话：隔离 PoC 是“看它能不能帮我们设计 Agent Lab”，不是“让它碰 RebornG 主项目”。

## patch artifact 是什么

`patch artifact` 是一种最低权限的“补丁建议文本”，不是可直接应用的补丁权限。

允许的未来形态：

- 外部 SDK/agent 只在独立输出目录生成说明、伪 diff、建议清单或设计片段。
- 输出必须进入 `artifacts/` 或独立报告文件。
- Codex 再人工审查、重写、拆分，并按 RebornG 正常门禁决定是否手工实现。

禁止的形态：

- 外部 SDK/agent 直接调用 `apply_patch`。
- 外部 SDK/agent 直接改 RebornG 文件。
- 外部 SDK/agent 直接运行测试、脚本、shell、package manager。
- 外部 SDK/agent 直接 `git add`、`git commit`、`git push`。

一句话：patch artifact 是“建议稿”，不是“执行权”。
