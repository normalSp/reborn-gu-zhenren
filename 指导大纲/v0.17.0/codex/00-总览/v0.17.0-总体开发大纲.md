# v0.17.0 总体开发大纲

日期：2026-05-18
状态：已完成，本地开发里程碑收束
主题：战斗、杀招、小队与阵法深化

## 背景

v0.9 到 v0.16 已经建立了统一行动协议、青茅凡战视觉竖切、青茅区域行动、活世界状态、自由目标、正史/IF 边界、NPC/势力投影、路线承接、低阶经济和 UI 收束。

当前战斗系统已有基础：

- `v080-battlefield-combat-engine.ts`：棋盘战斗、地形、遮蔽、阵点、士气、小队、撤退等本地规则基础。
- `squad-combat-engine.ts`：旧小队战斗、阵型、士气、协同、疲劳。
- `killer-moves.json` / `killer-move-expression-specs.json` / `gu-expression-specs.json`：杀招、蛊虫表现与战斗展示资料。
- `v010-qingmao-combat-pack.ts`：青茅低阶战斗候选登记和本地引擎入口。
- `ActionHubPanel` / `GuDaoPanel` / `WorldHubPanel`：v0.16 后的新入口结构。

v0.17 的任务不是重写全部战斗，而是把旧战斗能力和新活世界/行动工作台接起来，先做低阶、可解释、可测试的第一层。

## 版本目标

1. 玩家能从 `行动` 看懂为什么进入战斗、准备什么、风险是什么。
2. 战斗过程能解释位置、目标、消耗、反制、失败原因和后果。
3. 杀招/蛊虫不只是卡片说明，而能参与战斗准备、反制说明和轨迹解释。
4. 小队、阵法、撤退、追击先做第一层，不抢完整大战场。
5. 战斗结果能回流到 `记录`、`蛊道`、NPC/势力/路线风险，但不越权写奖励、地点、阵营、NPC 生死。

## 范围切分与完成情况

### a1：战斗权威与旧系统接缝设计门禁

- 已盘点旧 combat/squad/battlefield/killmove 系统权威。
- 已明确 engine、UI、DeepSeek 权限边界。
- 已定义战斗入口必须通过 `行动`，演武入口只保留 demo/dev。

### a2：字段表、trace schema、测试矩阵

- 已定义战斗准备、战斗 trace、行动后果回流字段表。
- 已明确不新增存档字段时只使用现有 ledger / return context / UI view。
- 已建立低阶战斗测试矩阵与 Player Advocate 轮次。

### b1：行动工作台到战斗准备/入场第一刀

- 已在 `行动` 工作台展示 v0.17 战斗候选准备态。
- 战斗候选已说明来源、敌我信息、反制/失败边界和无奖励边界。
- 未开放新掉落，未新增奖励。

### b2：低阶战斗轨迹与后果回流第一刀

- 已提供战斗 trace/backflow view，并在战后面板显示复核说明。
- 回流只读展示，不写正式战利品、永久伤残或 NPC 生死。
- 旧兽材掉落链通过不设置 `enemySpecIds` 规避。

### b3：杀招/蛊虫反制与风险显示第一刀

- 已把蛊虫/杀招反制、风险、阻断结果显示接到 `蛊道` 与战斗准备。
- UI 只展示 RebornG-owned rules 和 engine/canon 边界，不计算私有伤害。
- 高阶杀招、仙蛊、宝黄天等继续阻断。

### b4：小队/阵法/撤退/追击第一刀

- 已做小队、阵法、撤退、追击的只读提示层。
- 已展示士气、守卫、阵点、撤退/追击压力与阻断结果。
- 未开放完整大军团战，未开放正式 NPC 死亡/捕获。

### rc：质量收束

- 全量本地测试、e2e、long e2e、production preview smoke 已通过。
- Player Advocate rc 60 轮已通过。
- 仪表盘、handoff、Git 状态进入收束更新。

## DeepSeek 边界

DeepSeek 允许：

- 描写战斗气氛、候选线索、传闻、战后叙事回流。
- 根据本地 engine 输出写玩家可见叙事。

DeepSeek 不允许：

- 计算伤害、命中、消耗、战利品、掉落、NPC 生死。
- 发放蛊虫、材料、元石、蛊方、杀招。
- 解锁地点、阵营、正式任务或正史锚点。
- 暴露隐藏事实、方源私密因果、春秋蝉、重生等高阶信息。

## 完成标志

v0.17 完成时，玩家应能感到：

- 战斗不是孤立演示，而是自己行动后的自然结果。
- 每一步战斗都有位置、目标、消耗、反制和后果解释。
- 蛊虫/杀招/小队/阵法开始形成低阶策略感。
- 系统仍保持本地裁决、测试可回归、DeepSeek 不越权。

## 收束结论

上述完成标志已达到第一层。`v0.17.0` 不新增存档字段、不扩大 DeepSeek 权限、不发放正式奖励、不解锁地点/阵营、不结算 NPC 生死；战斗深化以候选入口、只读边界、可解释 trace/backflow 为主，为后续正式伤势、补给消耗、战斗奖励和多区域战斗保留用户决策门槛。
