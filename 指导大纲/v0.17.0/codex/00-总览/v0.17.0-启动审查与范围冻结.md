## 专家团审查

### 项目总策划

v0.17 是合理下一步。v0.16 已经把入口收干净，现在必须用战斗系统证明“减法后能更像游戏”，否则底栏只是变少，体验深度没有增加。

### 世界观审校

v0.17 必须低阶化。青茅/南疆凡级战斗可以深化，但不能借杀招、阵法、小队名义打开仙蛊、宝黄天、福地、尊者、高阶命运线。

### 系统架构

不建议重写 combat engine。应先做权威表和桥接：旧 squad/duel/battlefield 各有能力，先定谁是第一阶段权威，避免 UI 与多个 engine 私算。

### AI Pipeline

DeepSeek 不需要新权限。战斗叙事可以读本地 trace，但 trace、伤害、结果、奖励、地点和 NPC 命运仍由本地 engine 决定。

### Player Advocate

玩家最关心三件事：为什么打、能做什么、打完影响什么。v0.17 的 UI 不能只堆更多战斗参数，要把风险、反制、结果回流讲清楚。

### Frontend/Motion

战斗深化必须守住 v0.10 之后的可读性经验：棋盘、行动卡、战斗轨迹和底栏不能互相遮挡。动效只能展示 engine facts。

### QA/Test Engineering

v0.17 是高风险 runtime 阶段。每个 player-facing 小版本建议 20 轮 Player Advocate，rc 60 轮；新增极端战斗意图必须进测试矩阵。

## 冻结范围

本版本允许：

- 入口、准备态、战斗 trace、低阶后果回流。
- 低阶蛊虫/杀招反制显示。
- 小队/阵法/撤退/追击第一层。
- 战斗 UI 可读性和 reduced motion 验证。

本版本不允许：

- 正式奖励、掉落、材料消耗、蛊虫获得、蛊方获得。
- 正式地点/路线解锁。
- 正式阵营转移、招揽、通缉、NPC 生死/捕获/永久重伤。
- 高阶蛊仙战斗系统。
- DeepSeek 新写入权。
- 存档字段变更，除非用户单独批准。

## MiroFish need 判定

| 包 | need | 建议阶段 | 说明 |

| low-rank combat encounter pack | preferred | b2 前 | 用原著候选材料约束低阶战斗场景、反制、后果，不作 runtime truth |
