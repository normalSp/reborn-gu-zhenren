# v0.17.0-a1 战斗权威与旧系统接缝设计门禁

日期：2026-05-18
状态：已完成
类型：设计门禁 + runtime 接缝约束

## 目标

把 v0.17 的战斗深化放回新信息架构中，明确哪些系统有权决定战斗事实，哪些系统只能显示候选或提示。

## 权威表

| 领域 | 权威来源 | v0.17 使用方式 | 禁止事项 |
|---|---|---|---|
| 战斗候选入口 | `CombatEventCandidate` + 本地 entry validation | 行动工作台登记候选 | DeepSeek 直接开战、给奖励、改地点 |
| 低阶棋盘战斗 | `v080` battlefield engine | 5x3 凡战入场、行动、trace | UI 私算胜负/伤害 |
| 战后回流 | `buildBattleOutcomeSummary` + local action ledger | 文本只承接本地战果 | 文本追加奖励、NPC 生死 |
| 蛊虫/杀招可用性 | `gu-expression-registry` + v0.17 boundary hints | 展示可用、不可用、反制、风险 | 候选文本替玩家获得蛊虫或杀招 |
| 小队/阵法提示 | v0.17 squad tactic rules | 只读士气、守位、撤退/追击压力 | 正式阵营、正式路线成功、关键 NPC 结局 |
| 旧决斗/旧小队 | legacy combat/squad paths | 保留历史能力，不作为 v0.17 新入口 | 通过旧链路自动发材料、残卷或永久伤亡 |

## 接缝原则

1. v0.17 只新增候选规则和 UI/engine 桥，不新增存档字段。
2. 本轮战斗候选不得设置 `enemySpecIds`，避免触发旧兽材掉落链。
3. `dropPolicyId` 只能是 `local_engine_only` 或 `candidate_clue_only`。
4. 所有 MiroFish 来源只能进入 RebornG-owned candidate rules，不直接进入 canon、DeepSeek prompt 或奖励系统。
5. 战斗结果进入行动账本后，文本层只能承接，不得改写胜负、资源、地点、阵营或 NPC 生死。
