# v1.7.0 专家团启动会纪要

状态：已召开；等待用户批准 D-170
日期：2026-05-21
主题候选：第一个可控区域活世界 topic-slice 与公开测试/BFF 门禁复核

## 开会背景

v1.1-v1.5 已经分别完成路线、低阶生存、社会关系、区域样板、冲突后果五个 projection-first 地基。v1.6 又把 MiroFish 基础包 inventory、知识索引边界、intake 晋升链、长测 replay 和过期入口检查做成 report-only 工具链。

现在 RebornG 需要回答一个更关键的问题：

> 这些地基要在什么“第一个小世界”里合流？

如果继续分散堆功能，玩家会看到很多 panel 和提示，但世界仍然不像一个能长期生活的地方。v1.7 应该选择一个小范围活世界纵切，让路线、生存、社会、区域、冲突和知识治理第一次被同一个目标牵住。

## 专家团总建议

推荐 v1.7 主线为：

`低阶区域活世界纵切选型与第一阶段门禁`

推荐目标：

`南疆早期外缘：山路歇脚点 / 临时商队接触 / 散修落脚 / 城门外缘门槛`

这不是完整南疆，也不是完整商家城。它是一个小而硬的活世界实验台。

## 角色意见

| Lore & World Designer | 选“南疆早期外缘”合理，因为它离青茅后路线最近，低阶可见事实多，隐藏事实和高阶内容可以继续隔离。不能直接写正式商家城、正式势力名册、NPC 生死或隐藏因果。 |

| QA/Test Engineering Guardian | 建议 v1.7 的核心不是一个新 UI，而是一组纵切测试：7 日行程 deterministic replay、30 轮 Player Advocate、标准 live narrative probe、隐藏事实/正式地点/奖励/NPC 生死越权样本、反刷样本。 |

## 集思广益：好点子池

| 点子 | 价值 | 风险/处理 |
|---|---|---|
| “一日一地一事”活世界样本 | 每个 checkpoint 问：今天在哪、缺什么、谁注意、发生什么、下一步是什么 | 先做测试样本，不急着进 runtime |
| 7 日低阶行程 replay | 比单轮 UI 更能测漂移和目标遗失 | deterministic 为骨架，抽关键 checkpoint 做 `deepseek-v4-flash` live |
| 区域 micro-biome 四格 | 山路、商队、散修、城门外缘形成小循环 | 不命名正式地点，不开完整商家城 |
| 世界态势卡 | 把路线/生存/社会/冲突合成玩家可读态势 | 先设计，不默认新增存档字段 |
| topic-slice 先产测试样本 | 让知识库先服务 QA，而不是先服务 runtime | 防止内容一上来就变 canon |
| BFF 安全雷达 | 复核 API key、hidden/private canon、云存档是否已成硬需求 | 作为 process sidecar，不默认重构 |
| DeepSeek 安全摘要候选池 | 为未来 prompt/RAG 做 schema 准备 | v1.7 不接入 visible context |
| 区域生活“失败也推进”原则 | 缺补给、被拒绝、被盘问也能生成后果 | 不能写正式通缉/NPC 生死 |

## 不推荐现在做

- 完整南疆地图。
- 完整商家城。
- 正式区域持久字段直接落地。
- 全书知识库导入 runtime。
- DeepSeek 全书 RAG。
- 公测/BFF 大重构。
- 正式地点、阵营、奖励、NPC 生死。
- 任意时代开局或尊者时代内容。

## 专家团结论

v1.7 最稳的路线是：先做 `a1 区域活世界 topic-slice / save-format / authority 设计门禁`，决定是否保持 projection-first、是否需要 v25、MiroFish 需要到什么级别、Player Advocate 和 drift 强度如何定，再进入任何 runtime。

当前需要用户批准 D-170-001 至 D-170-010。
