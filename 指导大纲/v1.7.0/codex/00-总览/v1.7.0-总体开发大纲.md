# v1.7.0 总体开发大纲

状态：completed
日期：2026-05-21
主题：低阶区域活世界纵切选型与第一阶段门禁

## 目标

让 v1.1-v1.6 的地基第一次合流到一个小而可测的活世界纵切中，而不是继续堆孤立功能。

新增 a1 评估项：同开局可重玩差异度。目标是在不扩大 DeepSeek 权限、不新增未批准持久字段、不破坏原著边界的前提下，设计“同一开局也能出现不同低阶生活压力与候选体验”的长期路线。

新增 a1 治理项与 a2 样板：MiroFish 双仓 topic-slice 流水线已把 `RebornG request -> MiroFish export -> RebornG intake review -> 测试矩阵/规则草案` 固化成项目级制度、只读检查和首个可复跑样板。

## 推荐主线

`低阶区域活世界纵切选型与第一阶段门禁`

推荐样板：

`南疆早期外缘：山路歇脚点 / 临时商队接触 / 散修落脚 / 城门外缘门槛`

## 第一刀建议

a1 第一刀仍是设计门禁，不改 runtime：

- 评估是否保持 `SAVE_FORMAT_VERSION = 24`。
- 评估是否禁止新增 `regionalLifeState`。
- 定义 `V170RegionalLifeProjection` 或等价纯投影类型。
- 定义 MiroFish topic-slice 是否 blocking。
- 定义双仓流水线 request/export/intake/test 引用链和自动检查。
- 定义测试矩阵和 Player Advocate 强度。
- 定义 live narrative probe 档位、样本来源、轮次、成本记录和报告路径。
- 定义同开局可重玩差异度：本地事件牌组 / pressure deck / run fingerprint 的近期和远期边界。

## 当前结论

v1.7 已完成本地开发里程碑。runtime 落地为 `buildV170RegionalLifeProjection()` 与世界面板 `活世` 页签；仍不新增持久字段、不 bump save、不开完整南疆/商家城、不扩 DeepSeek 权限。
