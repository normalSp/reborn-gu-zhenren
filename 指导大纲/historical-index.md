# RebornG 历史索引

日期：2026-05-20
状态：当前历史入口；不是 runtime authority

## 定位

本文件是旧版本与历史制度的导航索引，用来减少当前工作入口里的长清单。

它只回答三件事：

1. 旧版本完成了什么。
2. 仍然有效的历史硬规则在哪里。
3. 详细证据应该去哪个原文件查看。

本文件不替代原始阶段记录、rc 质量记录、Player Advocate 记录、MiroFish intake review、GitHub Actions run id 或源码测试。

## 当前入口优先级

做当前开发时，先读：

1. `AGENTS.md`
2. `指导大纲/项目仪表盘.md`
3. `.codex/skills/reborn-expert-council/references/PROJECT-STATE.md`
4. 当前版本入口，例如 `指导大纲/v1.2.0/codex/00-总览/README.md`
5. `指导大纲/流程制度/README.md`
6. 本历史索引

只有当当前版本文档、流程制度或本索引明确指向旧版本文件时，旧版本文件才作为当前工作的补充依据。

## 展示压缩规则

当前入口只保留：

1. 当前版本状态。
2. 最近完整里程碑。
3. 项目级流程制度。
4. 必须停手的硬边界。
5. 指向本索引和原始证据目录的链接。

旧版本的阶段清单、commit/run 明细、逐项验证记录不再铺在当前仪表盘或 AGENTS 中。需要复核证据时，从本索引跳转到原版本目录。

## v1.6 自动检查脚本占位

用户已批准在 v1.6 建立过期入口自动检查脚本。建议到 v1.6 再实现：

- `scripts/check-doc-entrypoints.mjs`
- 可选 npm script：`check:doc-entrypoints`

建议检查项：

1. 当前仪表盘、AGENTS、PROJECT-STATE 是否引用本历史索引。
2. 当前版本 README 是否指向流程制度入口。
3. 已标 historical 的旧入口是否仍被当作当前门禁。
4. 当前版本决策池、测试矩阵、Git 计划、仪表盘是否同步最新 D 项。
5. 新项目级制度是否已进入 skill 和 `流程制度/README.md`。

v1.6 前不要把该脚本加入 CI 硬门。脚本规则稳定后，再由用户决定是否进入本地门禁或 GitHub Actions。

如发现本文件与原始版本文档冲突，以原始版本文档和当前版本用户决策为准，并在当前版本文档中修正索引摘要。
