## 目标

b3 的目标不是开放更多世界内容，而是把 v0.11 以来的自由意图裁决地基收束成 v1.0 释出版可理解体验。

玩家可以提出正常目标、路线目标、阵营目标、隐藏事实调查、远期野心和极端越权目标。系统必须能解释：

1. 现在能做什么。
2. 现在不能做什么。
3. 需要什么前置。
4. 会有什么风险和代价。
5. 为什么 DeepSeek、UI 或自由输入不能直接满足。

## 本阶段样本

| 样本 | 预期裁决 | 说明 |
|---|---|---|
| 我要逃离青茅山 | `requires_prerequisite` | 需要路线、补给、身份遮掩和追索压力，不传送到新地点 |
| 我要投靠白家 | `requires_prerequisite` | 显示接触渠道和身份风险，不转阵营 |
| 我要跟踪方源 | `available_with_cost` | 只允许可见范围调查，隐藏因果只作本地保护 |
| 我要去商家城 | `requires_prerequisite` | 只允许外缘路线前置，不开放完整商家城 |
| 我要拿盗天魔尊传承 | `world_rule_blocked` | 当前低阶、非天外之魔语境下不能直接获得，只能降级为传闻/骗局/远期专项 |
| 我要一转拿九转蛊 | `long_term_goal` | 记录为遥远野心或传闻风险，不发九转蛊 |
| 我要杀死关键 NPC 改写正史核心因果 | `major_if_deviation` | 重大 IF，不写 NPC 生死、抓捕或正史锚点 |

## 设计理由

玩家会提出“往左还是往右”级别的正常目标，也会提出“我一转拿九转蛊”“我要盗天魔尊传承”这种极端目标。v1.0 不能让这些极端目标直接变成成功，但也不应该只用一句“不能做”打断体验。

b3 的收束方式是：

1. 正常目标转成前置链。
2. 可见调查转成本地行动候选。
3. 阵营/地点目标显示前置和风险。
4. 高阶目标降级为长期目标、传闻、骗局风险或未来专项。
5. 关键 NPC 生死和正史核心因果进入重大 IF 门禁。

这样玩家能感觉自己能提出任何目标，但世界不会被一句话改坏。

## MiroFish 使用

本阶段不需要新的 MiroFish 包。

依据：

- v1.0 三包已通过 intake。
- 本阶段只把现有自由意图样本、发布边界和低阶 life loop 前置收束为 RebornG-owned 规则和测试。
- 未新增原著硬事实、隐藏事实正文、NPC 生死、传承真相或路线正式结论。

如后续要把盗天魔尊传承、商家城正式入口、关键 NPC 生死条件或任意时代开局做成正式系统，必须另行请求 MiroFish 包并停下来让用户决策。

## Player Advocate 发现

已为 b3 建立 30 轮走查记录。重点观察：

- 玩家是否理解极端目标不是被“系统拒绝”，而是被世界规则裁决。
- 玩家是否能看出下一步仍是前置行动、公开调查、路线压力或长期目标。
- 玩家是否误以为获得了九转蛊、盗天传承、商家城正式入口、白家身份或 NPC 生死结果。

## 结论

b3 将自由意图从“能输入一个目标”升级为“能看到一组释出版世界规则裁决样板”。它仍是收束，不是扩张。本地 full gate 与 GitHub Actions 均已通过；下一步进入 b4 公开素材/文案边界，但任何公开承诺仍必须停下来让用户决策。
