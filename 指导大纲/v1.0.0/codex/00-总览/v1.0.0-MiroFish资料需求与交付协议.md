## 判定

v1.0 触及青茅到南疆早期连续体验、低阶蛊师生活闭环、公开素材与发布文案，因此需要 MiroFish 提供候选材料。

但 MiroFish 输出仍然只能是：

- 候选材料。
- rule draft 输入。
- test sample 输入。
- copy/art boundary 输入。
- source pointer 与审校状态。

它不是 canon 真相源，不是 runtime 权威，不是 DeepSeek 权威，不能直接进入 UI 或公开文案。

## 交付格式要求

MiroFish 输出必须：

- quote-redacted。
- 不含 `quote`、`originalText`、`excerpt`、`verbatim`、`rawText`、`sourceText`。
- 使用 summaries、source pointers、classification、review status。
- hidden facts 只能 `hidden_ref_only` 或 `deferred`。
- 不直接生成 RebornG runtime code。
- 不把候选结论写成“正史事实已确认”。

## intake review 规则

Codex 拿到包后必须写 intake review，结论只能是：

- `accepted_for_candidate_pool`
- `accepted_for_rule_draft`
- `accepted_for_test_sample`
- `accepted_for_copy_boundary`
- `accepted_for_art_caption_boundary`
- `deferred`
- `quarantined`
- `rejected`

任何含原文摘录、隐藏事实泄露、未审候选当真相、直接要求 runtime 执行的包，必须 quarantine 或 reject。
