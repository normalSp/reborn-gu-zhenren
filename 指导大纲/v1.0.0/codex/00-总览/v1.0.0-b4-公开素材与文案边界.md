# v1.0.0-b4 公开素材与文案边界

日期：2026-05-19
状态：b4 候选包已完成；2026-05-20 用户批准正式发布标题、正文、hero 三件套和截图/短录屏方向
阶段：`v1.0.0-b4`
阶段性质：公开素材 / FAQ / release note 边界；正式发布动作仍由发布准备补丁复验后执行

## 目标

b4 的目标是把 v1.0 能对外展示的内容先收成候选包，避免到了发布前才发现文案过度承诺、截图暗示未开放系统、hero 图被误解成最终美术或正式 runtime 绑定。

本阶段只做：

1. hero 三件套候选与 caption 边界。
2. v1.0 release note 候选稿。
3. public FAQ 候选稿。
4. 截图 / 短录屏素材说明边界。
5. hidden fact / 高阶事实 / 未开放系统的红线检查。

本阶段不做：

- 不自动发布。
- 不自动部署 EdgeOne。
- 不让 hero 成为玩法、地点、奖励、原著事实或 EdgeOne 自动部署依据。
- 不生成或替换大规模新图。
- 不新增原著硬事实、隐藏事实正文、正式奖励、正式地点、正式阵营或 NPC 生死结果。

## MiroFish 使用

本阶段使用已通过 intake 的 `v100_public_release_copy_art_boundary_pack`，只作为：

- `copy_boundary`
- `faq_boundary`
- `art_caption_boundary`

不得作为：

- DeepSeek visible context。

当前不需要新的 MiroFish 包。若后续要把公告终稿绑定具体原著事实、关键 NPC 命运、商家城开放、盗天传承、任意时代开局或其他高阶承诺，必须重新停下来让用户决策，并视情况请求新包。

## Release Note 候选

正式标题：

`RebornG v1.0《青茅之后，活世界初成》`

批准正文：

RebornG v1.0 把前期多个系统收束成一段可连续体验的低阶蛊师人生。玩家可以从青茅山后续压力出发，看到南疆早期路线候选、补给与修行压力、NPC/势力反应、公开事件摘要和自由目标裁决。DeepSeek 仍只负责叙事、候选、线索、传闻和压力；正式奖励、地点、阵营、NPC 生死和世界事实由本地系统决定。

本版本不是完整蛊界，也不是“AI 想写什么就是什么”的跑团壳。它验证的是一个更长期的方向：玩家可以提出目标，世界会给出前置、风险、代价、拒绝或降级，而不是被一句话改坏。

## 本版可说

- 青茅山到南疆早期路线已经有候选连续体验。
- 低阶蛊师的修行、补给、炼养用、战斗和交易窗口已有释出版闭环。
- 自由目标会进入本地裁决，极端目标会被降级、阻断或转成长期目标。
- NPC、势力、路线压力和公开事件会形成可读回流。
- 隐藏事实不会直接暴露给玩家或 DeepSeek。

## 本版不能说

- 不能说完整蛊界已经开放。
- 不能说完整南疆、完整商家城、五域大地图已经开放。
- 不能说玩家可任意时代开局或附身任意人物。
- 不能说玩家能获得九转蛊、仙蛊、盗天魔尊传承或尊者级内容。
- 不能说路线、地点、阵营、NPC 生死已经成为正式持久化结果。
- 不能说 hero 图是玩法权威、最终全量美术或 EdgeOne 已自动部署。
- 不能说 EdgeOne 已自动部署或已公开发布。

## Public FAQ 候选

| 问题 | 候选回答 |
|---|---|
| v1.0 是完整蛊界吗？ | 不是。v1.0 是活世界早期正式版候选，重点是低阶蛊师人生、青茅到南疆早期承接、自由意图裁决和可读回流。 |
| 玩家可以想做什么就做什么吗？ | 玩家可以提出目标；系统会按世界规则判断能否尝试、需要什么前置、有什么风险，以及哪些目标当前只能成为长期野心或被阻断。 |
| 我能一转拿九转蛊或盗天魔尊传承吗？ | 当前不能直接获得。这类目标会被裁决为长期目标、传闻、骗局风险或未来专项，不会直接写奖励或传承。 |
| 能去完整商家城吗？ | v1.0 只处理南疆早期路线和外缘入口边界，不开放完整商家城。 |
| DeepSeek 会决定奖励或 NPC 生死吗？ | 不会。DeepSeek 只写叙事、候选、线索、传闻和压力，正式结果由本地系统决定。 |
| 这些图是最终美术吗？ | 不是。hero 三件套已批准用于 v1.0 正式发布首屏、公告/手动发布交接和社交预览，但不代表最终全量美术，也不是玩法事实来源。 |
| 会公开发布到 EdgeOne 吗？ | 本次可称为 v1.0 正式发布；EdgeOne 实际部署由用户 / CodeBuddy 手动执行，Codex 只提供验证与回滚清单。 |
