## public/private canon 门禁

### public canon

允许进入玩家可见 UI、公开文案或 DeepSeek visible context 的内容：

- 玩家已知事实。
- public/player_visible fact summaries。
- 公开压力。
- 公开路线锚点。
- 可解释的前置条件。
- 不含隐藏因果的公开失败结果。

### private/hidden canon

不得进入玩家可见 UI、公开文案或 DeepSeek visible context：

- hidden fact body。
- 方源私人因果、回溯、春秋蝉相关隐藏因果。
- 关键角色真实动机和未公开未来知识。
- 高阶世界机制和未开放事实。
- MiroFish hidden_ref_only 的完整说明。

### deferred canon

可进入测试/红线/未来池，不进入 v1.0 正式体验：

- 高代价 IF 偏离。
- 直接进入商家城核心。
- 直接获得高阶资源。
- 决定关键 NPC 生死。
- 任意时代开局。

## DeepSeek 门禁

DeepSeek 允许：

- 叙事。
- 候选解释。
- 线索。
- 传闻。
- 压力描述。
- 风险提示。

DeepSeek 禁止：

- 写正式奖励、正式材料、正式蛊虫、完整蛊方。
- 写正式地点、正式路线状态、正式阵营结果。
- 写 NPC 生死/抓捕/关键命运。
- 写隐藏事实 body。
- 改写正史硬事实。
- 输出公开发布承诺。

## API key/BFF 结论

D-003 已批准：v1.0 a2 只做威胁模型，不实现 BFF/backend。

如果 v1.0 公开版需要使用项目方共享 DeepSeek key，必须停下来另开决策，因为纯前端不能安全隐藏共享密钥。

如果 v1.0 只是本地/私有/用户自带 key 流程，可继续沿用前端-first 架构，但公开文档必须避免暗示项目方已提供安全托管 API 服务。

## a2 结论

a2 不改 runtime。后续 b1-b4 若触及公开发布、hidden fact、DeepSeek prompt、API key、BFF/backend、正式 route/location 字段，必须回到本门禁。
