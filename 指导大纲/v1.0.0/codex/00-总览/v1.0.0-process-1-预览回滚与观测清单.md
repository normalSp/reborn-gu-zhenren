## EdgeOne 手动预览清单

EdgeOne 由用户 / CodeBuddy 集成操作；Codex 当前不自动部署。

手动预览时检查：

| 项目 | 检查方式 | 失败处理 |
|---|---|---|
| 首屏是否黑屏 | 打开 EdgeOne 预览 URL | 回退到上一成功 commit，记录 console error |
| JS chunk 是否 404 | 浏览器 Network 查 `assets/*.js` | 重新构建/检查 deploy 输出目录 |
| 生产环境循环依赖 | Console 查 `undefined` / chunk init error | 对照 v0.10 黑屏经验，优先检查模块顶层初始化 |
| 底栏入口 | 进入 `地图 / 行动 / 角色 / 蛊道 / 世界 / 记录` | 若入口缺失或遮挡，阻断发布 |
| v1.0 b1/b2/b3 | 行动工作台查看连续体验、life loop、自由意图收束 | 若按钮误写正式结果，阻断发布 |
| b4 公开文案 | FAQ / release note / hero caption 候选 | 若过度承诺，回到用户审批 |
| 移动端 | 窄屏查看入口与文案 | 若核心流程不可读，阻断发布 |

## 回滚清单

若预览出现 P0/P1：

1. 停止继续发布，不扩散预览链接。
2. 记录当前 branch、commit、Actions run id、EdgeOne 预览 URL、报错截图、console error、network 失败资源。
3. 优先回退到最近通过 GitHub Actions 的上一 commit。
4. 不使用 `git reset --hard` 或强推；需要回滚代码时用新 commit 或让用户明确批准历史改写。
5. 如果是 EdgeOne 平台配置问题，先回退部署版本，不改 runtime。
6. 如果是 runtime 黑屏，优先查：
   - 模块顶层初始化。
   - chunk 间循环依赖。
   - store 初始状态同步调用 engine。
   - JSON import / build split 顺序。
   - public asset path。

## 错误观测最小口径

| 来源 | 必收信息 |
|---|---|
| 浏览器 Console | 第一条 red error、堆栈、chunk 文件名、行列号 |
| Network | 404/500 资源、chunk hash、MIME 错误 |
| GitHub Actions | run id、失败 step、失败命令 |
| EdgeOne 预览 | URL、部署 commit、部署时间 |
| 玩家反馈 | 操作路径、截图、设备、浏览器、是否移动端 |

若后续接入 Sentry 或其他线上监控，必须另设专项，不在 v1.0 process-1 自动引入外部 runtime 依赖。

## 发布前硬停门禁

以下情况必须停下来让用户决策：

- 是否公开发布。
- 是否使用某个公告标题或正文。
- 是否绑定 hero 到 title/landing/OG。
- 是否执行 EdgeOne 发布或回滚。
- 是否新增或替换任何公开图。
- 是否执行 live DeepSeek probe。
- 是否改 EdgeOne、GitHub branch protection、CI required checks。
- 是否新增 BFF/backend、Sentry runtime、云存档、账号系统。
