## 结论先行

`v1.0.0` 已具备进入公开发布准备的工程基础，但还不能直接宣称已经公开发布。

当前可以做：

1. 复核 v1.0 实际可玩内容、非目标和公开文案边界。
2. 复跑 production preview、玩家可见文案、内容治理和资产扫描。
3. 整理 EdgeOne 手动发布交接清单。

5. 标出必须由用户批准的公开发布决策。

当前不能直接做：

1. 不自动部署 EdgeOne。
2. 不直接发布公告。
3. 不把 hero 三件套绑定为正式 title / landing / OG。
4. 不擅自把 `package.json`、标题页和游戏页显示改成 `v1.0.0`。
5. 不新增公开图、不生成新图、不替换运行时美术。
6. 不引入 BFF/backend、Sentry runtime、自动部署、分支保护或新外部运行时依赖。

## 发布准备范围

本轮发布准备只处理“公开前能不能看、能不能说、出问题能不能退”。

| 类别 | 本轮处理 | 边界 |
|---|---|---|
| 工程质量 | 复跑发布前 smoke 和静态检查 | 不新增玩法 |
| 公开文案 | 整理公告/FAQ/可说不可说 | 终稿需用户批准 |
| 素材 | 复核 hero 三件套和截图/短录屏候选边界 | 不绑定 runtime，不生成新图 |

## 可公开表达的 v1.0 能力

可以说：

1. 青茅山到南疆早期路线已经形成候选连续体验。
2. 低阶蛊师修行、补给、炼养用、战斗、交易窗口和路线压力已经有释出版闭环。
3. 自由目标会进入本地裁决，极端目标会被拒绝、降级、转长期目标或要求前置。
4. NPC、势力、公开事件、路线压力和行动账本会形成可读回流。
5. DeepSeek 只负责叙事、候选、线索、传闻和压力，正式结果由本地系统决定。

不能说：

1. 完整蛊界、完整南疆、完整商家城已经开放。
2. 任意时代开局或任意人物附身已经可玩。
3. 玩家能获得仙蛊、九转蛊、盗天魔尊传承、尊者线或宝黄天正式交易。
4. route/location/currentRegion 已经成为正式持久化结果。
5. hero 图是最终美术或已经绑定正式发布页。
6. EdgeOne 已经自动部署或已经公开发布。

## EdgeOne 手动发布交接

Codex 当前不直接操作 EdgeOne。用户 / CodeBuddy 执行发布前，应确认：

1. 发布 commit 明确。
2. GitHub Actions 对应 commit 通过。
3. `npm run check:production-preview` 在本地通过。
4. EdgeOne build command 仍为 `npm run build`。
5. EdgeOne output directory 仍为 `./dist`。
6. 预览 URL 打开后首屏不黑屏。
7. Console 无 P0/P1 red error。
8. Network 无 JS/CSS/PNG 404。
9. 移动端入口和底栏可用。
10. 出现 P0/P1 时按 `v1.0.0-process-1-预览回滚与观测清单.md` 回滚。
