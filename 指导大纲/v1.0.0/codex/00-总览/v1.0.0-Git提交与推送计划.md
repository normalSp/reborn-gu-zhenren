## 原则

- 每个小版本至少一个 commit。
- runtime 小版本必须 push 并等待 GitHub Actions 通过。
- 文档-only 小版本可在同一启动包 commit 中提交，但需要记录清楚。
- 不使用 `git add -A`。
- 只 stage 当前阶段明确文件。
- 不 stage 历史 dirty 文件、美术候选、bgm、外部参考目录，除非本阶段明确需要。
- 不自动部署 EdgeOne。

## 每次 handoff 必填

- 当前 branch。
- commit hash。
- push 状态。
- GitHub Actions run id。
- 未提交文件是否与本阶段有关。
- 是否存在不能回滚或不应回滚的用户文件。
