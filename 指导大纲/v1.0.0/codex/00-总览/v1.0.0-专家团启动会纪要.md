# v1.0.0 专家团启动会纪要

日期：2026-05-19
状态：草案

## 会议结论

专家团一致建议进入 v1.0 启动审查，但 v1.0 必须是收束版，不是大扩张版。

推荐定位：

`活世界早期正式版：青茅山到南疆早期路线，低阶蛊师 life loop，自由意图裁决，NPC/势力回流，公开安全与长测闭环。`

## 分角色意见

### 项目总策划

v1.0 要回答玩家最关心的问题：我在这个世界里到底能做什么，我的选择有没有后果，系统是不是在认真维护世界规则。

本版本不应继续追逐“更多入口”，而应让既有系统形成一个可连续体验的版本。

### 蛊真人世界观审校

必须继续保护：

- 隐藏事实。
- 正史核心因果。
- 方源相关不可见动机与回溯信息。
- 高阶世界规则。
- 未开放蛊仙/宝黄天/尊者线。

MiroFish 只提供候选材料。任何 MiroFish 项进入 runtime 前都必须由 RebornG 重写、分类、测试。

### 玩法/数值架构师

v1.0 不应追求“一百多种蛊虫完整开放”。低阶闭环优先级更高：

- 玩家为什么要补给。
- 为什么要养蛊。
- 为什么某些杀招只能在特定条件下用。
- 为什么路线选择有身份、追索、资源、时间代价。

### 前端创意导演

v1.0 的 UI 应继续做减法。正式玩家第一眼不能看到大量未开放入口。

重点是：

- 下一步明确。
- 风险解释清楚。
- 玩家影响可读。
- 高阶/调试/演示内容场景化或隐藏。

### Motion/Technical Artist

v1.0 视觉应使用已有候选资产和 hero 三件套，不在发布前重开大规模美术生产。

公开图文不能暗示玩家已经能获得仙蛊、永生、尊者机缘、宝黄天正式交易。

### DeepSeek/模型调优专家

继续使用 `deepseek-v4-flash`。不评估其他模型，不扩大 DeepSeek 权限。

可选做固定样本 live probe，但需用户批准样本、范围和执行时机。live probe 只能用于发现叙事越权，不作为事实来源。

### QA/发布稳定专家

v1.0 rc 建议 150 轮 Player Advocate。公开候选版本必须增加：

- 旧档回流。
- 移动端。
- 极端意图。
- public copy。
- hidden fact leakage。
- EdgeOne preview smoke。

### Codex 操作安全官

每小版本必须更新：

- 版本文档。
- 测试矩阵。
- 项目仪表盘。
- Git 状态。
- handoff。

不要自动部署，不要用可写子代理，不要绕过用户决策。
