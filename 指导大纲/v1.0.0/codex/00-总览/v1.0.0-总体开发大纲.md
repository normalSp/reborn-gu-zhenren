# v1.0.0 总体开发大纲

日期：2026-05-19
状态：已批准；v1.0.0 已完成为本地开发里程碑，rc commit `53bdf98`，Actions `26098733549` 通过

## 一句话目标

`v1.0.0` 要让玩家相信：自己不是在和 AI 跑团，而是在一个有规则、有代价、有记忆、有正史压力的蛊真人风格活世界里，开始一段低阶蛊师人生。

## 玩家最终能玩到什么

v1.0 的玩家可体验目标：

1. 从青茅山局势出发，看到青茅后续路线压力，并能进入南疆早期承接的候选路径。
2. 以低阶蛊师视角处理修行、补给、蛊虫、杀招、战斗、势力压力、路线选择。
3. 输入自由目标时，系统能判断“现在能做什么、不能做什么、需要什么前置条件、会有什么风险”。
4. 对 NPC、势力、路线、公开事件产生可读影响，而不是只得到一次性文本回应。
5. 看不见隐藏事实和高阶秘密，但能感到世界背后有压力和暗流。
6. UI 不再像开发面板堆入口，正式体验以 `地图 / 行动 / 角色 / 蛊道 / 世界 / 记录` 为主。
7. 公开素材、截图、说明文案不承诺未开放能力。

## v1.0 最小完成定义

| 维度 | 最小完成 |
|---|---|
| 世界连续性 | 青茅山到南疆早期路线有连续行动链、压力回流和可读下一步 |
| 自由意图 | 正常、短期、长期、极端意图都有裁决说明和测试样本 |
| NPC/势力 | 行为能进入可读记忆/压力/公开事件摘要，并影响后续候选 |
| 低阶蛊师闭环 | 修行、补给、炼养用、战斗、交易/委托边界形成释出版闭环 |
| 正史/IF | 正史硬事实、玩家可见事实、隐藏事实、IF 偏离点分离 |

## 版本主线

### a0/a1：先定发布口径

先冻结 v1.0 不是完整蛊界，而是低阶蛊师活世界早期正式版。

输出：

- 接受标准。
- 公开承诺边界。
- 权威冻结表。
- 必须停手清单。

### a2：发布威胁模型

v1.0 接近公开，需要明确：

- public canon 与 private/hidden fact 的分界。
- DeepSeek prompt 与玩家可见文案的泄露风险。
- API key 在纯前端部署中的风险。
- 旧档、调试入口、未开放高阶入口的风险。

这一步默认不实现 BFF/backend，只写威胁模型、检查清单和后续门禁。若要实现 BFF/backend，必须另行让用户决策。

### b1：青茅到南疆连续体验

把 v0.14/v0.18 的路线候选、前置条件、压力回流收束成 v1.0 主线第一刀。

重点不是“一键开放大地图”，而是让玩家看懂：

- 为什么现在能离开或还不能离开。
- 哪条路线需要补给、身份、掩护、时机。
- 进入南疆早期后还只是早期边界，不是完整南疆。

若需要新增正式 route/location 持久字段，必须先停下来让用户批准 `SAVE_FORMAT_VERSION = 23` 与迁移策略。

### b2：低阶蛊师 life loop 释出版闭环

把 v0.15/v0.17/v0.18 的经济、补给、炼养用、战斗、路线压力整理成玩家能理解的一套闭环。

目标是“知道自己为什么缺资源、为什么要准备、为什么这只蛊或这招现在不能乱用”，而不是扩充所有蛊虫和所有杀招。

### b3：自由意图与极端意图收束

v1.0 必须处理玩家真实会输入的目标：

- 我要逃离青茅山。
- 我要投靠白家。
- 我要跟踪方源。
- 我要去商家城。
- 我要拿盗天魔尊传承。
- 我要一转拿九转蛊。
- 我要杀死关键 NPC 或改变正史核心因果。

处理方式必须是本地裁决：可行、不可行、延期、降级、前置条件、风险、代价、隐藏事实保护。

### b4：公开素材与 release pack

整理 v1.0 hero 三件套、截图、短录屏候选、OG、release note 草稿、公开 FAQ。

所有公开素材必须过：

- 原著边界检查。
- 未开放内容检查。
- 隐藏事实检查。
- DeepSeek 权限检查。
- 用户最终发布承诺审批。

### process-1：预览、回滚、观测

只做手动 EdgeOne preview smoke、错误上报/回滚清单、黑屏排查 SOP。

不自动部署，不替用户发布，不改 CodeBuddy/EdgeOne 集成方式。

### rc：release candidate

rc 必须证明“能公开试玩”而不是“能开发者自测”。

最小门禁：

- `npm test`
- `npx tsc --noEmit --pretty false`
- `npm run build`
- runtime/Qingmao/player-visible-copy/content-governance scans
- full e2e + long e2e
- production-preview smoke
- v1.0 specific preflight
- Player Advocate 150 轮以上
- GitHub Actions 通过

## 硬停边界

以下事项必须停下来让用户决策：

1. 新增或修改存档字段，尤其是 route/location/currentRegion。
2. `SAVE_FORMAT_VERSION = 23`。
3. 正式地点解锁、正式阵营转移、正式奖励、NPC 生死/抓捕/死亡。
4. DeepSeek 权限扩大。
5. MiroFish 阻塞包缺失或 intake 不合格。
6. BFF/backend/API key 保护实现。
7. 自动部署或公开发布承诺。
8. 大规模新图生成或替换既有美术风格。

## 专家团判断

v1.0 应该收束，不应该扩张。它的价值不是“承诺可以玩遍蛊界”，而是让玩家第一次看到 RebornG 的核心差异：AI 文本在外层，本地引擎在内核，玩家自由在规则里生长。
