# v0.15.0 低阶经济炼养用规则总纲

日期：2026-05-18
状态：设计启动版

## 核心原则

v0.15 的经济和炼养用规则必须先服务低阶蛊师生存，而不是做通用 RPG 商店。

玩家可以通过行动争取资源，但每个资源结果都必须能回答：

- 来源是什么。
- 成本是什么。
- 上限是什么。
- 失败代价是什么。
- 谁有权结算。
- 如何防止反复刷。
- 是否会影响路线、NPC、势力或公开事件。

## 权限归属

| 权限 | owner | 说明 |
|---|---|---|
| 蛊材/食料/残方候选 | canon + RebornG rule draft | MiroFish 只能提供候选证据 |
| 材料/元石/蛊虫/蛊方发放 | local engine | DeepSeek/UI 不得发放 |
| 炼蛊成功/失败/代价 | refine engine | 不由叙事文本决定 |
| 市场/商队机会 | route/social/economy engine | 初期为候选窗口，不是完整交易 |
| 反刷规则 | local engine + tests | 必须可回归 |
| 玩家可见解释 | UI | 显示原因，不私算结果 |
| 叙事润色 | DeepSeek | 候选/传闻/压力，不写正式奖励 |

## 设计边界

### 材料

材料必须来自注册表或候选规则池，不得由 DeepSeek 即兴创造。

第一刀优先处理：

- 已存在青茅资源循环。
- 已存在材料注册表。
- 小额、低阶、上限明确的补给或喂养缺口。

### 食料

食料是蛊材/材料的一种使用角色，不另造独立 fantasy resource 类。

v0.15 初期允许显示食料压力和缺口；真实消耗库存必须经过 b1/b2 阶段测试。

### 残方与蛊方

残方不等于完整蛊方。

DeepSeek 不得给完整蛊方。

UI 必须明确：

- 当前是残方、线索、传闻还是完整注册蛊方。
- 缺什么材料或条件。
- 失败代价由本地规则决定。

### 市场与商队

商队/市场窗口是机会，不是无限商店。

v0.15 初期只允许：

- 递话。
- 询价。
- 补给准备。
- 公开理由准备。
- 风险提示。

不允许：

- 完整价格表。
- 完整库存。
- 自动加入商队。
- 直接进入商家城。
- 直接购买未注册材料/蛊虫/蛊方。

### 黑市与委托

黑市/委托默认 deferred。

可先进入：

- 反刷样本。
- 诈骗/陷阱样本。
- 势力关注样本。
- 禁止直接奖励边界。

不得直接开放：

- 正式黑市交易。
- 委托代售收益。
- 灰色交易库存。

## 反刷底线

每个可执行资源/补给/市场动作必须至少考虑：

- 同场景重复刷限制。
- 单次奖励上限。
- 每日/阶段/路线窗口限制。
- 失败或风险反馈。
- 与 NPC/势力关注的关系。
- DeepSeek 叙事不得绕过本地 rewardPolicy。

## 测试底线

每个进入 runtime 的经济行为必须有：

- schema/canon 测试。
- engine 结算测试。
- DeepSeek 越权样本。
- 反刷样本。
- Player Advocate 走查样本。
- 如写存档，migration/default/default-old-save 测试。
