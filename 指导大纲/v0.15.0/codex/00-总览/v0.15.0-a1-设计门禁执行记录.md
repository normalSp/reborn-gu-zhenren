## 对六个问题的回答

### 哪些数据只属于候选

- MiroFish 原始交付 item。
- 商队/市场窗口。
- 黑市/委托/灰色交易边界。
- 未注册的材料、食料、残方、秘方、蛊方、价格和库存。

### 哪些数据可成为 RebornG-owned rule draft

- 低阶材料候选。
- 喂养缺口。
- 炼蛊/炼化失败代价。
- 残方与完整蛊方边界。
- 补给准备。
- 商队接触窗口。
- 反刷规则。
- 灰色交易延期边界。

### 哪些 outcome 必须由本地 engine 结算

- 材料、元石、蛊虫、蛊方、残方发放。
- 炼蛊成功/失败、消耗和失败代价。
- 市场/交易/委托结果。
- 反刷上限、冷却、递减收益。
- NPC/势力压力写入。

### 哪些内容只能 UI 展示

- 缺什么。
- 为什么缺。
- 当前不能做什么。
- 可以准备的小步。
- 商队/市场/黑市/委托风险。
- DeepSeek 无权决定的边界说明。

### 哪些行为触发 save format 决策

- 新增 `economyState`、`marketState`、`feedingSchedule`、`refinementHistory`、`tradeLedger`。
- 持久记录市场库存、价格、委托收益、炼蛊历史或喂养倒计时。
- 改变旧档默认经济状态。

### 哪些内容必须延后

- 完整坊市。
- 正式黑市。
- 委托代售。
- 完整商家城交易。
- 动态价格/库存系统。
- 二转/三转完整成长曲线。
- 高阶资源、仙材、宝黄天、福地或蛊仙经济。
