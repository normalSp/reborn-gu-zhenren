## 提交原则

- 每个小版本至少一个清晰 commit。
- 文档/规则和 runtime 实现尽量分开。
- 不使用 `git add -A`。
- 只 stage 本阶段相关文件。
- 不混入 `RebornG_codebuddy.zip`、`artifacts/`、`bgm/`、`.cursor/`、`指导大纲/大方向/`、美术候选资产等历史未跟踪项。
- 不提交 secrets、`.env`、`.codex` runtime state、plugin cache、SQLite/log 文件。
- 不自动部署 EdgeOne。

## 推送原则

需要推送：

- v0.15 文档规则建立后。
- 每个 runtime 小版本验证通过后。
- rc 收束后。
- 跨工具/跨会话 continuation 前。

不自动推送：

- EdgeOne 自动部署。
- branch protection。
- auto merge。
- full Playwright required PR gate。

这些都需要用户另行决策。
