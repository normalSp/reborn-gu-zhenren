## 交付质量

三包均满足：

- JSON 可解析。
- 无 `quote`、`originalText`、`excerpt`、`verbatim` 禁入字段。
- `forbiddenTextKeyCount = 0`。
- `highRankLeakTermCount = 0`。
- 所有 item 均为 `export_ready`。
- 所有 item 均为 `runtimeAuthority = candidate_only`。
- 所有 item 均为 `runtimeVisible = false`、`deepSeekVisible = false`。
- 所有 item 均为 `requiresHumanCanonReview = true`。

## 吸收规则

允许吸收为：

- `candidate_pool`
- `rule_draft`
- `fact_card_draft`
- `test_sample`
- `deferred`

禁止直接吸收为：

- runtime truth。
- canon authority。
- DeepSeek authority。
- 玩家可见隐藏事实正文。
- 正式奖励、材料、元石、蛊虫、蛊方、完整任务、地点、阵营或 NPC 生死结论。

## 使用顺序

1. a1/a2 使用低阶经济/炼蛊/喂养包。
2. b1/b2 优先处理补给准备、喂养缺口、残方/失败代价。
3. b3 使用商队/市场包做递话、询价、公开理由和风险提示。
4. b4 或后续版本再使用黑市/委托包做边界测试。

## caveat

- `食料` 精确词面在 0001-0600 中缺失，当前以喂养、酒虫、青竹酒等相邻信号覆盖。
- `坊市/路引` 精确词面缺失，当前以商队、客栈、店铺、盘问、担保等公开信号覆盖。
- `黑市` 精确词面缺失，当前是灰色交易和委托边界候选。

这些 caveat 不阻塞 v0.15 启动，但阻止我们把对应系统做成完整正式开放。

## 追加 request 触发条件

只有出现以下需求时，才需要用户转交新 request 给 MiroFish：

- 正式开放坊市或路引系统。
- 正式开放黑市、委托代售或灰色交易收益。
- 正式进入完整商家城交易系统。
- 建立明确价格表、库存表或动态市场。
- 要求 0001-0600 之外的证据覆盖。
