# v1.1.0 专家团启动会纪要

日期：2026-05-20
状态：D-001 至 D-024 已批准；D-025 后续已单独批准并执行；v1.1 本地开发已完成

## 会议结论

专家团一致建议：v1.1 不做完整南疆或大时代扩展，而是把 v1.0 的候选连续体验推进到路线/地点/区域状态地基。

用户已批准主线选择 A，并批准 b1 采用单一聚合 `routeLocationState` 与 `SAVE_FORMAT_VERSION = 23` 的实装方向。用户也已批准新增 `v1.1.0-process-2`：长线叙事漂移、全书知识库治理、历史索引、入口压缩与 v1.6 过期入口检查规划。D-025 小规模 live DeepSeek probe 后续已单独批准并执行；v1.1 的 live 结论是发现 hidden-name echo 并加 runtime C27 防护，不得宣称 pure live narrative quality 通过。

## 各专家意见

| 专家 | 意见 |
|---|---|
| 项目总策划 | v1.1 是 v1.x 地基，路线/地点不稳，后续经济、NPC、区域都会悬空 |
| 世界观审校 | 青茅 -> 南疆早期外缘可以推进，但完整商家城、尊者、大时代开局必须远期化 |
| 玩法/数值架构师 | route/location 字段要和 `livingWorldState`、行动账本、低阶 life loop 协同，不能 UI 私算 |
| 前端创意导演 | 位置与路线压力必须玩家可读，避免继续堆复杂面板 |
| Motion/Technical Artist | 只做轻量路线状态反馈，不做大规模重皮或高阶视觉暗示 |
| DeepSeek/模型调优专家 | 模型保持 `deepseek-v4-flash`，不扩权；必要时增加固定 eval 样本 |
| QA/发布稳定专家 | 若 bump save format，旧档迁移和 Player Advocate 必须升规格 |
| 长线叙事漂移审查员 | v1.1 runtime 前建立 checkpoint；后续检查世界状态、玩家目标、DeepSeek 越权、重复空转和原著漂移 |
| Canon/Knowledge Curator | 全书 MiroFish 包进入 `vMiroFish -> 知识库 -> runtime canon` 三层晋升链，不让知识库直接变 canon 或 DeepSeek context |
| Codex 操作安全官 | 先文档/门禁，再 runtime；每阶段同步 dashboard/Git/PROJECT-STATE；旧版本证据先经 historical-index 导航 |

## v1.1 的一句话

让世界第一次正式承认玩家的位置与路线。

## process-2 的一句话

让长线剧情、知识来源和历史入口第一次拥有跨版本硬门禁，避免“文档写完没人管”。

## 长期愿景口径

RebornG 的终极方向是“活着的蛊世界”，不是通用 AI 跑团。大时代开局、任意身份、蛊仙期和尊者时代是远期目标，必须等低阶区域活世界跑通后再进入专项。
