# v1.1.0-preflight bug 清理记录

日期：2026-05-19

定位：进入 `v1.1.0` runtime 前的质量清理，不改变版本主线、不新增存档字段、不扩大 DeepSeek 权限。

## 修复范围

1. 新建角色进入游戏后停在“等待天命显现...”，需要存档读档才正常。
2. 五转初阶修行 UI 误显示“尝试升仙”。
3. 手动编辑导出存档进入升仙/仙窍状态后，蛊虫在凡窍 inventory 与仙窍 apertureInventory 中重复显示。
4. DeepSeek 缓存命中率低，已针对 system prompt 动态内容做结构性调优。
5. 全量 E2E 捕获的移动端 DebugOverlay 遮挡旧结局面板按钮问题。

## 根因与处理

### 新建角色首帧卡住

- 根因：`GameScreen` 共用一个 `startedRef` 控制 BGM 和 DeepSeek 叙事管线。创建角色时 `currentDomain` 先于 `screenState=game_play` 写入，BGM effect 提前置位，叙事管线进入游戏后被跳过。
- 处理：拆分为 `pipelineStartedRef` 与 `bgmStartedRef`，离开游戏/读档只重置叙事管线标记，BGM 独立控制。
- 回归：`tests/e2e/pre-v110-bugfixes.spec.ts` 覆盖新建角色无需存档读档即可进入首段叙事。

### 五转初阶误显示升仙

- 根因：engine 已要求五转巅峰，但 `AperturePanel` 只检查 `realmGrand === 5`。
- 处理：UI 只有在 `realmGrand === 5 && subRank === '巅峰'` 时显示升仙；五转非巅峰显示继续突破和门槛说明。
- 回归：新增 E2E 覆盖五转初阶不显示升仙；新增 cultivation engine 测试覆盖五转巅峰前不可升仙。

### 手改升仙存档蛊虫重复

- 根因：升仙迁移只复制 `inventory` 到 `apertureInventory.gu`，未清空来源，也未合并去重；仙人蛊虫面板又拼接两处数据。
- 处理：`migrateToApertureStorage()` 合并/去重蛊虫与材料后清空凡窍来源；存档归一化修复手改 immortal saves；GuInventoryPanel 对仙人视角做防御性去重。
- 回归：`src/store/slices/immortalSlice.test.ts` 与 `src/store/save-normalization.test.ts` 覆盖迁移和手改存档修复。

### DeepSeek 缓存前缀不稳

- 根因：`ContextBuilder.buildSystemPrompt(mode, store)` 注入了道痕、当前蛊虫、NPC、对话、旗标等易变 store 状态，导致可缓存前缀频繁变化。
- 处理：system prompt 保留稳定规则、权限边界和格式约束；store 相关内容迁入 `buildDynamicContext()`，作为 user/dynamic context 传入。
- 回归：`src/engine/context-builder-cache.test.ts` 断言普通玩家状态变化不改变 system prompt，同时 dynamic context 仍包含道痕、余额、蛊虫等必要状态。
- live 证据：`node scripts/run-deepseek-eval.mjs --live --confirm-cost --models deepseek-v4-flash --sample-limit 3 --max-retries 0 --temperature 0.2 --timeout-ms 30000` 通过，3/3 accepted，cacheHitRatio 0.8157，报告位于 `artifacts/deepseek-eval/2026-05-19T16-14-14-407Z/report.json`。

### 移动端 DebugOverlay 遮挡

- 根因：调试浮层固定在右下角，窄屏时内部按钮拦截旧结局面板按钮点击。
- 处理：DebugOverlay 在窄屏挪到右上角，避免遮挡底部和中下部操作区。
- 回归：`npm run test:e2e -- tests/e2e/v080-ending-framework.spec.ts` 通过。

## 后续建议

1. 进入 `v1.1.0-a1` 前，以本记录作为 preflight 质量基线。
2. 后续如果玩家再次反馈缓存命中低，优先检查 system prompt hash 是否稳定、dynamic context 是否被错误拼到 system message、以及 API usage 中 `prompt_cache_hit_tokens/prompt_cache_miss_tokens` 是否正常透传。
3. 不建议为了缓存命中牺牲世界状态准确性；应保持稳定 prefix + 动态上下文分层，而不是删掉必要玩家状态。
