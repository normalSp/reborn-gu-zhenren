## 设计原则

1. 先存最小可验证状态，不存完整地图。
2. 先记录路线/地点范围，不开放地点功能。
3. 先 coarse scope，不存细粒度坐标。
4. 先 local engine 权威，不让 DeepSeek 或 UI 私写。
5. 先旧档保守迁移，不误传送、不误解锁。
6. 先能支撑 b1-b4，不为 v2.0 预支复杂字段。
7. 先支持长线漂移测试，让 checkpoint 能比对世界状态。

## 方案比较

| 方案 | 内容 | 是否 bump save | 优点 | 风险 | 专家团建议 |
|---|---|---:|---|---|---|
| A | 不新增字段，继续用 `livingWorldState`、行动账本和派生 UI 推断路线/地点 | 否 | 最安全，不碰迁移 | 世界仍不能稳定回答“玩家在哪里”；长线漂移难审计；b2/b3 会继续补丁式查询旧账本 | 不推荐作为 v1.1 主线，只适合用户决定继续保守 |
| B | 新增单一聚合对象 `routeLocationState`，只存粗粒度路线/地点范围 | 是，建议 v23 | 可审计、可迁移、可被 UI/测试/漂移门稳定引用；避免多个顶层字段扩散 | 需要迁移/default/normalization/tests；字段命名必须克制 | 推荐 |
| C | 新增分散顶层字段，如 `currentRouteId`、`currentLocationId`、`currentRegionId`、`routeFlags` | 是 | 查询直接 | 顶层字段扩张快，后续经济/NPC/区域系统会继续加字段；旧档迁移复杂 | 不推荐 |
| D | 把路线/地点做成后端/BFF/session 状态 | 是或否 | 未来可云同步 | v1.1 明确不做 BFF/backend；当前复杂度过高 | 排除 |

## 专家团推荐

推荐进入用户决策的是方案 B：

> 在 b1 中新增一个聚合型 `routeLocationState`，并将 `SAVE_FORMAT_VERSION` 从 `22` 提升到 `23`。

但字段必须严格限制在“路线与地点范围地基”，不得夹带：

- unlocked location list
- faction standing
- reward state
- NPC life/death
- market/shop access
- complete Southern Border map
- Shang clan city access
- hidden fact body
- DeepSeek authority

## 推荐字段草案

字段名建议：

```ts
routeLocationState
```

草案结构：

```ts
type RouteLocationState = {
  status: 'not_started' | 'preparing_departure' | 'route_in_progress' | 'outer_edge_projection' | 'blocked';
  routeId: string | null;
  locationScopeId: 'qingmao_mountain' | 'qingmao_exit_path' | 'southern_border_outer_edge' | 'unknown_conservative';
  regionScopeId: 'qingmao' | 'southern_border_outer_edge' | 'unknown_conservative';
  authority: 'start_profile' | 'living_world_engine' | 'route_location_engine';
  evidenceLedgerEntryIds: string[];
  sourceRefs: string[];
  lastUpdatedAtTurn: number | null;
  migrationNote?: string;
};
```

字段含义：

| 字段 | 用途 | 禁止含义 |
|---|---|---|
| `status` | 玩家路线/地点状态的粗粒度阶段 | 不代表任务成功、奖励、阵营或地点功能开放 |
| `routeId` | 当前路线候选或正式路线 id | 不代表路线必然成功 |
| `locationScopeId` | 玩家所处地点范围 | 不代表完整地点解锁 |
| `regionScopeId` | 玩家所处区域范围 | 不代表完整区域系统开放 |
| `authority` | 写入来源 | 不允许 DeepSeek/UI/narrative-text |
| `evidenceLedgerEntryIds` | 迁移或更新依据 | 不存原文、不存隐藏事实正文 |
| `sourceRefs` | RebornG/MiroFish/知识库引用 | 不是 runtime 权威本体 |
| `lastUpdatedAtTurn` | checkpoint 和漂移测试用 | 不作为时间线 canon |
| `migrationNote` | 旧档保守迁移说明 | 不给玩家展示隐藏或内部结论 |

## 可选命名替代

如担心 `location` 一词过于正式，可改名为：

- `routeScopeState`
- `travelState`
- `positionScopeState`

专家团仍推荐 `routeLocationState`，因为它最直接，但所有字段文案必须强调 `scope`，避免把外缘路线写成完整地点开放。

## v22 旧档迁移矩阵

用户已批准方案 B 和 `SAVE_FORMAT_VERSION = 23`，迁移必须保守：

| 旧档状态 | 识别依据 | v23 默认 | 说明 |
|---|---|---|---|
| 新档或无离开目标 | 无相关 `playerGoals`、无路线账本 | `not_started` + `qingmao_mountain` | 不误判离开 |
| 有离开青茅目标但未执行准备 | `livingWorldState.playerGoals` 中有离开/逃离目标 | `preparing_departure` + `qingmao_mountain` | 只承认目标，不承认路线 |
| 已执行路线准备 | 行动账本有 route/supply/cover/pursuit preparation | `preparing_departure` + 对应 route candidate | 不承认到达 |
| 已有路线 continuation candidate | v0.14/v0.18 相关 candidate 账本 | `route_in_progress` 或 `outer_edge_projection` | 仍是路线/外缘投影，不是完整南疆 |
| 账本冲突或无法识别 | 多个互斥路线、缺关键字段、旧手改档 | `unknown_conservative` + `migrationNote` | 保守回落，不误传送、不误解锁 |
| 编辑/异常存档 | 字段类型异常、id 不在 allowlist | normalize 到安全默认 | 不抛黑屏，不开放地点 |

迁移硬规则：

- 默认不能把玩家迁到商家城。
- 默认不能授予阵营身份。
- 默认不能写奖励或 NPC 生死。
- 默认不能把 `outer_edge_projection` 写成完整南疆。
- 迁移不得把 hidden/private source ref 展示给 UI 或 DeepSeek。

## 写入权威边界

| 层 | 权限 |
|---|---|
| canon/rules | 定义允许的 route/location/region scope id、阶段和禁止组合 |
| migration/default | 从 v22 旧档生成保守默认值 |
| route_location_engine | 唯一可正式更新 `routeLocationState` 的本地引擎 |
| store action | 只接收 engine 结果，不在 UI 私算状态 |
| UI | 只展示当前位置范围、路线压力、下一步和缺口 |
| DeepSeek | 只写叙事、压力、传闻、候选，不写字段、不声明正式结果 |
| MiroFish | 只提供 candidate/rule/test/source pointer，不是 runtime authority |
| 知识库 | 只做项目索引，不直接进入 runtime 或 DeepSeek |

## b1 最小实现边界

在 a2 前置完成并进入 b1 后，第一刀只允许：

- 新增字段、类型、默认值、迁移和 normalization。
- 新增 route/location scope allowlist。
- 新增一个本地 engine，用于从既有行动账本和明确动作更新 `routeLocationState`。
- 新增最小 UI 投影，告诉玩家当前位置范围、路线状态和下一步缺口。
- 新增 focused tests、e2e、copy scan、Player Advocate 和 T0 deterministic soak 记录。

第一刀禁止：

- 完整地图。
- 完整南疆。
- 商家城正式开放。
- 阵营 standing / recruitment / task。
- 奖励、掉落、市场、正式库存经济。
- NPC 生死、捕获、命运结论。
- DeepSeek 写字段或给出正式地点结果。
- 全书基础包整包导入。
