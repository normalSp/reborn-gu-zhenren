# v1.1.0 总体开发大纲

日期：2026-05-19
状态：草案；等待用户审阅

## 主题

`路线、地点与区域状态地基`

## 版本目标

v1.1 的目标不是扩张地图规模，而是建立“正式路线/地点/区域状态”的最小可信地基。

具体来说，v1.1 要解决：

- 玩家从青茅山离开后，系统如何记录路线与地点。
- 旧档如何迁移，不把旧玩家错误传送到新区域。
- UI 如何让玩家理解当前位置、路线压力和下一步。
- DeepSeek 如何叙事，但不拥有地点、路线、阵营、奖励或 NPC 生死权威。
- MiroFish 候选材料如何进入规则草稿和测试样本，而不是直接变成 canon/runtime 权威。

## 设计原则

1. 先设计门禁，再写 runtime。
2. 先 route/location 权威表，再 store 字段。
3. 先旧档迁移矩阵，再 `SAVE_FORMAT_VERSION` bump。
4. 先小范围青茅 -> 南疆早期，不开完整南疆。
5. 先玩家可读位置面板，不做全地图。
6. DeepSeek 只写叙事、候选、传闻、压力说明。
7. 正式地点、阵营、奖励、NPC 生死、路线成功/失败必须由本地 engine/store 决定。

## 范围内

- route/location/currentRegion 或等价字段方案评估。
- `SAVE_FORMAT_VERSION = 23` 设计评估。
- 旧档迁移/default/normalization/test 方案。
- 青茅 -> 南疆早期路线状态第一刀。
- 区域状态面板和玩家可读位置说明。
- 前期账本对路线/补给/追踪/身份压力的回流。
- 南疆早期外缘行动承接，不开放完整商家城。
- MiroFish v1.1 包 intake 与候选吸收。
- Player Advocate 升级轮次和旧档迁移走查。

## 范围外

- 完整南疆、完整商家城、五域大地图。
- 蛊仙期、宝黄天正式交易、尊者线、任意时代开局。
- 正式阵营转移、通缉结论、招揽成功、NPC 生死。
- 正式奖励、掉落池、完整市场/黑市/委托经济。
- BFF/backend、云存档、账号系统。
- 自动 EdgeOne 部署或公开发布承诺。
- DeepSeek 权限扩大。

## 版本成功标准

- 玩家能明确看懂“我在哪里、我为什么在这里、我下一步能做什么、有什么风险”。
- 旧档迁移和新档默认状态稳定。
- route/location 不与 `livingWorldState`、行动账本、NPC/势力反应、低阶 life loop 冲突。
- 极端意图仍会被正确降级或阻断。
- MiroFish 内容只进入候选/规则/测试，不进入 runtime 权威。
- 全量质量门、关键 e2e、生产预览 smoke 和 Player Advocate gate 通过。

## 主要风险

| 风险 | 说明 | 对策 |
|---|---|---|
| 存档字段过早膨胀 | route/location 字段会影响后续所有系统 | a1 先做字段表和迁移矩阵 |
| 路线候选误变正式地点 | 玩家可能以为已进入完整南疆/商家城 | UI 文案和 engine 输出必须区分路线状态、区域外缘、正式地点 |
| DeepSeek 越权 | 叙事可能写成“你已进入商家城并获得身份” | DeepSeek contract、copy scan、e2e 和 Player Advocate 拦截 |
| 原著事实不足 | 南疆早期外缘和商队/散修压力需要事实支撑 | v1.1 MiroFish 包必须 intake review |
| 旧档误迁移 | 旧档可能缺少路线信息 | migration default 必须保守，旧档默认仍在原状态 |

## 与长期路线的关系

v1.1 是 v1.x 的第一块地基。它不负责让世界变大，而是让世界有稳定的位置和路线概念。后续 v1.2 经济、v1.3 社会关系、v1.4 区域扩展、v1.5 冲突深化，都需要依赖 v1.1 的路线/地点状态。

外部活世界参考见 `指导大纲/长期路线/RebornG-外部活世界参考映射.md`。v1.1 主要吸收 ReAct、Reflexion、Voyager 的思想：

- ReAct：自由意图必须转为本地裁决和行动协议，不直接变事实。
- Reflexion：MiroFish intake、Player Advocate 和 bug 经验必须回流测试矩阵。
- Voyager：专家团需求池和测试样本矩阵要像能力库一样持续成长。
