# v1.1.0 Git 提交与推送计划

日期：2026-05-19
状态：草案

## 原则

- 每个小版本独立 commit。
- runtime 与证据/文档可分 commit，但不得混入无关历史脏文件。
- 不使用 `git add -A`。
- 只 stage 当前阶段明确文件。
- 每个 runtime commit push 后等待 GitHub Actions 通过并补录 run id。

## 建议提交边界

| 阶段 | Commit 边界 |
|---|---|
| a0 | 长期路线重整与 v1.1 启动包 |
| a1 | route/location/save-format 设计门禁 |
| a2 | MiroFish intake 与字段表 |
| b1 | 正式路线状态第一刀 |
| b2 | 区域状态与位置面板 |
| b3 | 前期账本压力回流 |
| b4 | 南疆早期外缘行动承接 |
| process-1 | 旧档迁移/预览/回滚专项 |
| rc | v1.1 质量收束 |

## Handoff 必填

每个阶段完成后更新：

- `指导大纲/项目仪表盘.md`
- 当前阶段文档
- `PROJECT-STATE.md`
- 最新 handoff/context 文件
- Git commit hash、push 状态、Actions run id

## 禁止

- 不在 v1.1 commit 中夹带历史 art、MiroFish 原包、zip、外部分析目录或无关 dirty files。
- 不自动部署 EdgeOne。
- 不设置 branch protection 或 auto-merge，除非用户另行批准。
