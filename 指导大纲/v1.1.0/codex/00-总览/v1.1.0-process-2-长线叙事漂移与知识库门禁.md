## 目标

把“长线剧情不漂、世界观不漂、DeepSeek 不越权、MiroFish 不越权、知识库不混层”从口头提醒变成阶段门禁。

本专项不改 runtime、不新增存档字段、不 bump `SAVE_FORMAT_VERSION`、不扩大 DeepSeek 权限、不开放正式地点/阵营/奖励/NPC 生死。

## 减法审查

本专项同时建立“流程减法”原则：

1. 不删除历史证据。
2. 先收入口，再标历史，再加替代指针。
3. 只有当前版本、项目级制度、仪表盘、PROJECT-STATE 和 skill 都指向新入口后，旧入口才可标为 historical。
4. 若某制度仍被阶段完成门禁、测试、交接或 skill 引用，不得废弃。
5. 任何会降低安全性、测试覆盖、原著边界或用户决策可见性的减法，必须停下来让用户决策。

### 当前可做的低风险减法

- 将“长线叙事漂移”“知识库治理”统一收入口到 `指导大纲/流程制度/`，避免散落在 v1.1 单个阶段。
- 将全书 MiroFish 包只作为 `vMiroFish -> 知识库 -> runtime canon` 三层链路处理，避免每个版本重复解释权威边界。
- 将旧版本流程文件保留为历史证据，不再作为当前入口；当前入口以 `指导大纲/流程制度/README.md`、`指导大纲/项目仪表盘.md`、`PROJECT-STATE.md` 为准。

### 本轮已批准的减法

- 建立 `指导大纲/historical-index.md`。
- 压缩 `指导大纲/项目仪表盘.md` 和 `AGENTS.md` 中旧版本阶段清单展示。
- 将 v1.6 过期入口自动检查脚本写入长期路线规划。

### 仍需用户决策的减法

- 删除任何旧版本原始文件或历史证据。

- 废弃仍被当前阶段门禁、测试、交接或 skill 引用的流程入口。
