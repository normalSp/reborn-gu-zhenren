# v1.1.0 测试矩阵

日期：2026-05-19
状态：草案

## 测试目标

v1.1 的测试重点是路线/地点/存档字段风险，而不是战斗数值或完整区域内容。

## Current Matrix

| ID | 类型 | 样本 | 期望 |
|---|---|---|---|
| V11-SAVE-001 | 旧档迁移 | v22 旧档无 route/location 字段 | 默认仍保守，不误传送、不误解锁 |
| V11-SAVE-002 | 新档默认 | 新建角色未离开青茅 | route/location 默认与 StartProfile/当前状态一致 |

## Future Sample Pool

- 玩家要求改名换身份混入白家或商队。
- 玩家主动暴露青茅秘密换取庇护。
- 玩家长期逃避追踪导致区域势力误判。
- 玩家旧档已有多个逃离目标，迁移后如何聚合。
- 玩家在路线中发生战斗失败，是否影响地点状态。
- 外部参考映射样本：ReAct 链路断裂时，玩家自由意图是否被 DeepSeek 直接写成事实。
- 外部参考映射样本：Reflexion 回流失败时，Player Advocate 困惑是否没有进入测试矩阵。
- MiroFish intake 样本：`human_review` 与 `hidden_ref_only` 项是否被错误展示给 UI 或 DeepSeek。

## 自动化建议

进入 b1 后至少需要：

- focused unit tests：route/location default、migration、normalization。
- store tests：受控 patch 来源和禁写边界。
- e2e：玩家能看懂当前位置、路线压力和下一步。
- scans：player-visible copy 不承诺完整南疆/商家城/奖励/阵营。
- production-preview smoke：防止存档迁移或 chunk 循环导致黑屏。
