## 1. 设计Token体系

### 1.1 设计哲学

蛊真人世界的视觉灵魂存在于三个对立统一中：**墨与金的对抗**（虚空未知 vs 真元道痕）、**古与今的并置**（古籍手抄本肌理 × 现代磨砂玻璃质感）、**静与动的张力**（修真沉静节奏 × 战斗瞬间爆发）。所有Token选择围绕这三组对立展开，不做单维度的"黑暗修真"或"华丽仙侠"。

核心约束：不使用纯黑（#000000），不使用霓虹辉光，不使用过饱和色彩，不使用emoji作为图标。这些被列为Forbidden Patterns，确保最终视觉不落入廉价网页游戏的陷阱。

### 1.2 色板

色板采用三层架构：**底墨层**（背景/面板基底）、**道痕层**（交互/描边/文本）、**命火层**（关键状态/突出信息/战斗提示）。每一层内部按透明度或明度变化形成子色阶。

```css
/* ═══════ 蛊真人色板 — 全局CSS变量 ═══════ */

:root {
  /* ── 底墨层（虚空/未知/纸面基底）── */
  --gu-bg-deep:        #0D0D12;      /* 最深底（主背景） */
  --gu-bg-standard:    #1A1A24;      /* 标准底（面板/卡片基底） */
  --gu-bg-elevated:    #252236;      /* 抬高底（弹窗/Modal） */
  --gu-bg-glass:       rgba(26, 26, 36, 0.72); /* 磨砂玻璃底 */

  /* ── 道痕层（真元/道痕/修行气息）── */
  --gu-trace-gold:        #C9A96E;   /* 暗金描边色（道痕金） */
  --gu-trace-gold-dim:    #8B7648;   /* 暗金弱光 */
  --gu-trace-gold-bright: #E0C78A;   /* 暗金高光（hover/active） */
  --gu-trace-gold-text:   #D4B87A;   /* 暗金文字色 */
  --gu-trace-slate:       #4A5068;   /* 青灰描边（山水/过渡） */
  --gu-trace-slate-light: #6B7294;   /* 青灰亮调 */

  /* ── 命火层（生命/危险/杀戮/提示）── */
  --gu-life-crimson:      #C44B4B;   /* 血赤（HP变化/死亡/危险） */
  --gu-life-crimson-dim:  #8B3A3A;   /* 血赤弱光（禁选状态） */
  --gu-life-verdant:      #4B8B6E;   /* 暗绿（恢复/成长/仁心） */
  --gu-life-azure:        #4B6E8B;   /* 暗蓝（智道/谋心/冷却中） */

  /* ── 文本层级 ── */
  --gu-text-primary:      #E8E4DC;   /* 主文本（浅米白，模拟古籍墨色褪色后的暖白） */
  --gu-text-secondary:    #9B9585;   /* 次文本 */
  --gu-text-disabled:     #5C5860;   /* 禁用文本 */
  --gu-text-inverse:      #1A1A24;   /* 反色文本（用于暗金底上） */

  /* ── 状态色 ── */
  --gu-danger:            #C44B4B;
  --gu-success:           #4B8B6E;
  --gu-warning:           #C99B4B;
  --gu-info:              #4B6E8B;
}
```

**色板使用规则**：
- 底墨层永远占画面面积最大（≥60%），避免道痕金喧宾夺主
- 命火层的blood-red仅在数值变化、死亡威胁、战斗致命选择时使用——不用于静态UI装饰
- 道痕金描边厚度默认为1px，hover时变为1.5px（用box-shadow而非实际border-width变化以避免布局抖动）
- 所有状态色必须同时提供文字色和背景色配对，确保4.5:1对比度

### 1.5 动效参数

蛊真人不是快节奏游戏。动画速度整体比标准Web应用慢约30%，体现修真的沉静步调。

```css
:root {
  /* ── 时长Token（毫秒）── */
  --gu-duration-instant: 100ms;   /* 按钮按下/link hover */
  --gu-duration-fast:    200ms;   /* 状态切换/数值闪烁 */
  --gu-duration-normal:  350ms;   /* 面板展开/淡入 */
  --gu-duration-slow:    600ms;   /* 场景过渡/转换 */
  --gu-duration-epic:    1200ms;  /* 战斗高潮/章节过渡 */

  /* ── Easing Token ── */
  /* 所有ease-in-out使用自定义cubic-bezier而非CSS默认ease-in-out */
  --gu-ease-enter:   cubic-bezier(0.16, 1, 0.3, 1);   /* 入场：easeOutExpo风格 */
  --gu-ease-exit:    cubic-bezier(0.7, 0, 0.84, 0);   /* 退场：easeInExpo风格 */
  --gu-ease-both:    cubic-bezier(0.65, 0, 0.35, 1);  /* 对称过渡 */
  --gu-ease-bounce:  cubic-bezier(0.34, 1.56, 0.64, 1);/* 反弹感 */
}
```

**Spring配置（Motion React专用）**：

```typescript

```

## 2. 组件动效规范（Motion React）

Motion React（framer-motion）负责**所有组件级微交互**——面板展开、数值波动、选项入场、对话气泡、Toast、Hover响应。这些是频繁触发、生命期短、与状态变化紧密耦合的动画。GSAP不参与此层。

**核心原则**：
- 同一组件内部不使用GSAP和Motion混用
- 永恒动画（如持续微弱的暗金呼吸光）必须包裹在React.memo叶子组件中
- 所有动画属性仅限GPU合成属性：`transform`、`opacity`、`filter`、`clip-path`
- 所有动画必须尊重 `prefers-reduced-motion` 媒体查询

### 2.1 StatusBar 状态栏

**组件职责**：实时显示HP、真元、暴露、债务、道心四维等核心数值，每条用进度条+数值标签呈现。

**动效方案**：

```
数值变化时：
  数字：scale(1.0 → 1.3 → 1.0) 弹跳，颜色在命火色与默认色之间闪变
  进度条宽度：使用 Motion layoutId 自动追踪变化
  伤害时：血赤脉冲从0→1→0 opacity叠加在HP条上

降低时：HP条从暗金渐变为血赤，1200ms后恢复暗金（给玩家时间注意到变化）
增加时：HP条短暂泛暗绿微光

道心四维：每次变化使用staggerChildren，四维各延迟60ms依次更新（制造"涟漪扩散"感）
```

**Motion React代码骨架**：

```tsx
// StatusBar动效片段
<motion.div
  className="hp-bar-fill"
  layout
  style={{
    backgroundColor: hpPercent < 25
      ? 'var(--gu-life-crimson)'
      : 'var(--gu-trace-gold)'
  }}
>
  <motion.span
    key={hpValue}
    initial={{ scale: 1.3, color: 'var(--gu-life-crimson)' }}
    animate={{ scale: 1, color: 'var(--gu-text-primary)' }}
    transition={{ type: "spring", damping: 12, stiffness: 200 }}
  >
    {hpValue}
  </motion.span>
</motion.div>
```

### 2.2 NarrativePanel 叙事面板

**组件职责**：展示AI生成的叙事文本，是游戏的核心信息输出区。

**动效方案**：

```
新段落进入：
  整个段落 opacity: 0→1, y: +12px→0
  使用 AnimatePresence mode="wait" 做段落切换
  旧段落: opacity 1→0, y: 0→-8px（轻微上移消失）
  动效时长: 400ms ease-out

长篇文本：
  不做逐字打字（文字RPG逐字太慢，影响阅读节奏）
  改为整段淡入，给玩家"我正在读"的节奏控制权

特殊文本样式（系统提示/内心独白/传音入密）：
  不同颜色的左边框脉冲动画
  使用 box-shadow 的暗金微光呼吸效果（2s循环，opacity 0→0.12→0）
```

**动效约束**：叙事文本的动画必须是"辅助阅读"而非"干扰阅读"。永远不在文本出现时做位移超过12px或scale变化——文本可读性优先于一切动效。

### 2.3 ChoicePanel 选择面板

**组件职责**：展示2-5条玩家可选的行动选项，是游戏的核心输入区。

**动效方案**：

```
选项入场：
  staggerChildren: 每个选项delay 80ms
  每个选项: x: -24px→0, opacity: 0→1
  spring: GU_SPRING_DEFAULT
  4个选项总入场时长: ~670ms

选项hover：
  x: 0→4px (微右移)
  border-left的暗金描边: 1px→3px
  box-shadow暗金微光出现
  时长: 200ms

选项press：
  scale: 1→0.98
  时长: 100ms
  
选项选中/确认：
  选中选项: 暗金底色加深 + 其余选项opacity→0.4
  确认后所有选项: opacity→0, y→+8px（退场）
  
"返回上一级选择"按钮：
  独立入场，delay在所有当前层选项之后
  
危险选项（致命/高风险）：
  使用血赤左边框而非暗金
  hover时暗红微光而非暗金微光
```

### 2.4 BattleOverlay 战斗覆盖层（P2将重命名为CombatOverlay）

**组件职责**：战斗场景的全屏覆盖界面，包含HP条、杀招选择、战斗日志、逃跑按钮。

**动效方案**：

```
战斗覆盖层入场：
  从底部滑入，clip-path: inset(100% 0 0 0) → inset(0 0 0 0)
  时长: 500ms, ease: var(--gu-ease-enter)
  背景暗化: overlay bg opacity 0→0.85

回合切换：
  当前回合提示: scale(0.9→1), opacity 0→1
  使用 AnimatePresence 切换 "你的回合" / "敌方回合" 标签

杀招选择面板：
  从右滑入: x: +100%→0
  杀招卡片 stagger 入场

伤害数值弹出：
  从敌人位置飘出: y: -40px, opacity 1→0, scale 1→1.5
  伤害类型颜色: 物理=白、火道=赤、冰道=蓝白、毒道=绿
  
HP条变化：
  使用 layout 属性自动追踪宽度变化
  临界态（HP<25%）：周期间歇性红色脉冲（2s循环）

战斗日志：
  新日志条目从下方滑入: y: +20→0, opacity 0→1
  旧条目自动向上推出视口
  最大显示5条，超出使用 Motion layout 自动调整
```

### 2.5 NPCInteractionPanel NPC交互面板

**组件职责**：NPC对话界面，包含对话气泡、态度指示器、交互选项（闲聊/交易/委托/深交/挑衅）。

**动效方案**：

```
对话气泡入场：
  从对应侧滑入（NPC左侧/玩家右侧）
  NPC气泡: x: -16px→0, opacity 0→1
  玩家气泡: x: +16px→0, opacity 0→1
  时长: 350ms, spring: GU_SPRING_DEFAULT

态度指示器（affinity变化）：
  数值变化时用 color tween: 绿↔黄↔红
  正变化: 暗绿脉冲
  负变化: 血赤脉冲

交互选项面板：
  底部上滑入场: y: +40px→0, opacity 0→1
  staggerChildren: 每项delay 50ms
  
深交谈心选项（高affinity解锁）：
  独立入场动画: 从选项面板中"解锁展开"
  scale: 0.8→1 + 暗金边框发光
```

### 2.6 AchievementToast 成就弹窗

**组件职责**：成就解锁时的短暂通知弹窗。

**动效方案**：

```
弹窗入场:
  scale: 0.7→1, opacity 0→1
  spring: GU_SPRING_BOUNCE (damping:12, stiffness:200)
  暗金边框: box-shadow从0扩展到12px暗金微光再收回

停留: 2500ms
  暗金微光呼吸动画（2s循环，opacity渐变）

弹窗退场:
  scale: 1→0.9, opacity 1→0
  y: 0→-16px (上飘消失)
  时长: 400ms

多个成就排队:
  使用队列而非堆叠
  当前弹窗退场后200ms下一个弹窗入场
```

## 3. GSAP场景动画规范

GSAP负责**场景级大动画**——不可中断、时间线编排复杂、跨多个DOM元素的动画序列。动画时序严格用GSAP Timeline编排，不在React组件生命周期内管理。Motion React不参与此层的任何动画。

**核心原则**：
- 每个场景动画封装为独立的自执行函数，返回GSAP Timeline实例
- Timeline创建时使用 `gsap.context()` 或 `useGSAP()` 确保React卸载时自动清理
- 动画期间可同时发起AI API请求，利用动画延迟掩盖网络等待
- 移动端（<768px）禁用所有GSAP场景动画（替代方案：简单的opacity fade）
- 低性能设备（见第七章）自动降级为简化版本

### 3.1 转章过渡动画

**触发时机**：章节切换点（如"青茅山→商队期"、"王庭之争→逆流河"）

**技术方案**：

```
动效时长: 1800ms
工具: GSAP Timeline + CSS clip-path + SVG 水墨遮罩

Timeline编排:
  0ms:     当前场景 overlay暗化 (opacity 0→0.85, 600ms)
  200ms:   水墨纹理从中心向外扩散 (clip-path: circle(0%) → circle(100%), 1000ms)
  600ms:   章节标题文字从暗金渐显 (opacity 0→1 + y: +32→0, 400ms)
  800ms:   章节副标题逐字渐显 (每个字delay 40ms, 600ms)
  1200ms:  水墨纹理从中心收缩消失 (clip-path: circle(100%) → circle(0%), 600ms)
  1400ms:  新场景背景开始显现 (opacity 0→1, 400ms)
  1600ms:  overlay暗化消退 (opacity 0.85→0, 200ms)

关键要素:
  - 章节标题: "第X章 · 章节名"
  - 章节摘引: 原著经典对应段落（8-16字）
  - 水墨遮罩: SVG feTurbulence生成的水墨扩散纹理
  - 音效同步: 预留Audio节点，可绑定章节切
```

**降级方案（低端设备）**：简化为600ms暗化+fade切换，去掉SVG水墨遮罩。

**与AI配合时机**：

```typescript
const handleChapterTransition = async (nextChapter: ChapterId) => {
  // 1. 开始播放转章动画
  const tl = playChapterTransition(nextChapter);
  
  // 2. 动画开始同时发起首段叙事AI请求
  const narrativePromise = fetchInitialNarrative(nextChapter);
  
  // 3. 动画播放完毕(1800ms)时AI大概率已返回
  await tl.finished;
  const narrative = await narrativePromise;
  
  // 4. 展示叙事内容
  showNarrative(narrative);
};
```

### 3.2 战斗高潮 — 杀招释放动画

**触发时机**：玩家或敌人在战斗中使用高消耗杀招（消耗类型为"重"或"极重"的杀招）

**技术方案**：

```
动效时长: 1500-2500ms（取决于杀招类型）
工具: GSAP Timeline + Canvas粒子 + 屏幕震动

Phase 1 — 蓄力 (30%时长):
  杀招名称大字从画面中心渐显 (scale: 0.5→1.2→1, elastic)
  蓄力粒子从四周向中心汇聚（Canvas 2D）
  屏幕轻微暗化

Phase 2 — 释放 (40%时长):
  粒子爆散 (从中心向四周)
  暗金/血赤/冰蓝（根据杀招流派）光效扩散
  屏幕震动: x: ±4px, y: ±2px, duration: 200ms, repeat: 2
  伤害数值大字体弹出: scale 0→1.5→1, 颜色对应流派

Phase 3 — 命中 (30%时长):
  敌方HP条剧烈缩短 + 红色脉冲
  余光粒子逐渐消散
  战斗日志出现杀招描述

杀招颜色映射:
  纯粹物理攻击: 苍白→金
  火道杀招: 橙红→暗红渐变粒子
  冰道杀招: 蓝白→淡蓝渐变粒子
  毒道杀招: 暗绿→紫绿渐变粒子
  暗器杀招: 暗金→黑渐变粒子
```

**GSAP Timeline骨架**：

```typescript
const playKillerMove = (killerMove: KillerMove, isPlayer: boolean) => {
  const tl = gsap.timeline({ defaults: { ease: 'power2.inOut' } });
  
  // Phase 1: 蓄力
  tl.to('.killer-move-name', {
    opacity: 1, scale: 1, duration: 0.3, ease: 'back.out(1.7)'
  })
  .to('.battle-screen', { filter: 'brightness(0.7)', duration: 0.2 }, '-=0.1')
  .call(() => particleBurst('gather', killerMove.color))
  
  // Phase 2: 释放
  .to('.battle-screen', {
    x: '+=4', duration: 0.05, repeat: 3, yoyo: true,
    ease: 'power2.inOut'
  })
  .call(() => particleBurst('explode', killerMove.color))
  .to('.killer-move-name', { scale: 1.3, duration: 0.2 })
  .to('.killer-move-name', { scale: 1, duration: 0.2 })
  
  // Phase 3: 命中
  .to('.enemy-hp-fill', { width: `${newHpPercent}%`, duration: 0.5,
    ease: 'power2.in' })
  .to('.damage-number', { opacity: 1, y: -40, scale: 1.5, duration: 0.3 })
  .to('.damage-number', { opacity: 0, duration: 0.5 }, '+=0.5')
  
  return tl;
};
```

**降级方案（低端设备）**：取消Canvas粒子，仅保留名称渐显+屏幕震动+HP条变化。

### 3.3 名场面涟漪动画

**触发时机**：玩家触发全局名场面事件（如"三王福地开启"、"天庭入侵"、"逆流河显现"）

## 5. 技术栈选型与依赖一览

### 5.1 新增npm依赖

```json
{
  "dependencies": {
    "framer-motion": "^11.0.0",
    "gsap": "^3.12.0",
    "@gsap/react": "^2.0.0"
  }
}
```

**版本选择理由**：
- `framer-motion@11`: 最新的稳定主版本，支持layout动画、AnimatePresence、staggerChildren。11.x是长期支持版本。
- `gsap@3.12`: GSAP 3.x的最新稳定版。不推荐4.x测试版。
- `@gsap/react@2`: GSAP官方React集成hook（useGSAP），自动处理Timeline清理和SSR兼容。

### 5.2 懒加载策略

```typescript
// GSAP在需要时才import（场景动画不频繁触发）
const loadGSAP = () => import('gsap').then(mod => mod.default);
const loadGSAPReact = () => import('@gsap/react').then(mod => mod.useGSAP);

// 转章过渡调用
const handleChapterTransition = async (chapter: ChapterId) => {
  const gsap = await loadGSAP();
  // ... Timeline编排
};

// Motion React在组件文件顶部静态导入（所有组件都可能用到）
import { motion, AnimatePresence } from 'framer-motion';
```

**包大小核算**：
- GSAP Core: ~60KB gzipped
- Motion React: ~35KB gzipped
- 合计: ~95KB（可接受，约增加首屏加载200ms）
- 后加载的GSAP插件（ScrollTrigger/Flip等）不计入首屏，按需加载

### 5.3 TailwindCSS集成

Motion React和GSAP与TailwindCSS无冲突：
- Motion React通过 `className` 透传Tailwind类名
- GSAP操作内联样式，不涉及CSS类名
- 磨砂玻璃类定义为Tailwind的 `@layer components` 

```css
/* tailwind.css */
@layer components {
  .gu-glass-panel { /* 见4.2节完整定义 */ }
  .gu-glass-bubble { /* 见1.6节 */ }
  .gu-glass-overlay { /* 见1.6节 */ }
  .gu-divider-gold { /* 见4.3节 */ }
  .gu-corner-accent { /* 见4.3节 */ }
  .gu-bg-paper { /* 见4.1节 */ }
  .gu-panel-texture { /* 见4.1节 */ }
}
```

### 5.4 不做的事

以下技术在蛊真人项目中不推荐：
- **Three.js / React Three Fiber**: 3D渲染对文字RPG无增益，大幅增加包大小和技术复杂度
- **PixiJS**: 2D Canvas引擎，GSAP + Motion已覆盖所有动效需求，引入新渲染器增加维护成本
- **Lottie**: 需要额外依赖和JSON动画文件，CSS动画+GSAP已足够
- **CSS Modules / styled-components**: 项目已使用TailwindCSS，保持单一CSS方案

## 7. 性能降级策略

### 7.1 设备性能检测

```typescript
// src/utils/deviceCapability.ts

interface DeviceCapability {
  tier: 'high' | 'medium' | 'low';
  supportsWebGL: boolean;
  supportsBackdropFilter: boolean;
  cpuCores: number;
  isMobile: boolean;
}

const detectDeviceCapability = (): DeviceCapability => {
  const cpuCores = navigator.hardwareConcurrency || 2;
  const isMobile = /Mobi|Android/i.test(navigator.userAgent) ||
    (window.innerWidth < 768);

  const canvas = document.createElement('canvas');
  const supportsWebGL = !!(
    canvas.getContext('webgl') || canvas.getContext('webgl2')
  );

  const supportsBackdropFilter =
    CSS.supports('backdrop-filter', 'blur(1px)') ||
    CSS.supports('-webkit-backdrop-filter', 'blur(1px)');

  if (cpuCores <= 2 || (isMobile && cpuCores <= 4)) {
    return { tier: 'low', supportsWebGL, supportsBackdropFilter, cpuCores, isMobile };
  }
  if (cpuCores <= 4 || isMobile) {
    return { tier: 'medium', supportsWebGL, supportsBackdropFilter, cpuCores, isMobile };
  }
  return { tier: 'high', supportsWebGL, supportsBackdropFilter, cpuCores, isMobile };
};

// 存储在Zustand中供全局组件使用
export const deviceCapability = detectDeviceCapability();
```

### 7.2 降级对照表

| 特性 | High Tier | Medium Tier | Low Tier |
|------|-----------|-------------|----------|
| 磨砂玻璃 | backdrop-filter: blur(16px) | backdrop-filter: blur(8px) | 实色 background: rgba(26,26,36,0.9) |
| Canvas粒子（战斗杀招） | 完整粒子系统 | 简化粒子（1/3粒子数） | 禁用，仅文字动画 |
| SVG水墨遮罩（转章） | 完整SVG feTurbulence渲染 | SVG简化（静态模糊PNG替代） | 禁用，仅opacity fade |
| 屏幕震动（战斗） | x: ±4px, 3次 | x: ±2px, 2次 | 禁用 |
| 背景噪点纹理 | opacity: 0.04 | opacity: 0.02 | 禁用 |
| 暗金微光呼吸 | opacity: 0→0.12 2s循环 | opacity: 0→0.08 2s循环 | 禁用 |
| GSAP场景动画 | 完整Timeline | 简化Timeline（去掉粒子层） | 全部降级为CSS transition |
| Motion组件动画 | 完整spring | spring参数调快(stiffness+50) | duration: 200ms instant |
| backdrop-filter | blur(16px) | blur(8px) | none |
| 字体加载 | font-display: swap | font-display: swap | font-display: optional（不阻塞渲染） |

### 7.3 移动端专属限制

```typescript
// 移动端永远禁用的特性（与tier无关）
const MOBILE_DISABLED = {
  parallax: true,         // 无视差滚动
  gsapPin: true,          // 无ScrollTrigger pin
  3dTransforms: true,     // 无rotateX/Y/Z
  backdropFilter: true,   // 无backdrop-filter（即使支持也禁用，性能成本高）
```

### 7.4 内存预算

| 资源 | 预算 | 说明 |
|------|------|------|
| GSAP Timeline实例 | ≤ 3个同时活跃 | 多余的排队等待 |
| Canvas元素 | ≤ 1个（战斗粒子共享Canvas） | 战斗粒子与蛊虫升转粒子共用 |
| SVG内联纹理 | ≤ 2个（噪点+水墨） | 预生成，不运行时生成 |
| Motion animate节点 | ≤ 20个同时动画 | 超出自动合并为stagger批次 |
| 字体文件总大小 | ≤ 1.5MB（.otf + .ttf） | 超出则使用font-display: optional |

### 7.5 动画队列

防止多个场景动画同时播放导致的性能崩溃：

```typescript
class AnimationQueue {
  private queue: Array<() => gsap.core.Timeline> = [];
  private isPlaying = false;

  enqueue(animFactory: () => gsap.core.Timeline): void {
    this.queue.push(animFactory);
    if (!this.isPlaying) this.playNext();
  }

  private async playNext(): Promise<void> {
    const factory = this.queue.shift();
    if (!factory) { this.isPlaying = false; return; }
    this.isPlaying = true;
    const tl = factory();
    await tl.play();
    this.playNext();
  }

  clear(): void {
    this.queue = [];
    this.isPlaying = false;
  }
}

export const animQueue = new AnimationQueue();
```
