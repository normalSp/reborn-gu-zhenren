## 1. 审计方法

审计对象：

- 源码：`src/engine`、`src/store`、`src/components/game`、`src/types`、`src/canon`
- 设计大纲：`指导大纲/v0.7.0/v0.7.0-小队势力-成就重做-宝黄天完整版-设计大纲.md`
- 本地原著资料：`doc/original work/reverend-insanity.txt`
- 开发资料索引：`doc/lore/`

审计标准：

- “已完成”：存在源码实现，并有测试、canon 数据或 UI/Store 接入证据。
- “部分完成”：接口或局部实现存在，但玩法闭环、UI、测试或数据缺一。
- “未完成”：仅设计文档存在，源码为空实现/占位/未接入。
- “不可信”：跟踪结论与源码证据相反。

## 4. 真实阻塞项

1. 版本标识不统一：`package.json` 是 `0.5.0`，文档和存档已进入 v0.6/v0.7/v0.8 混合状态。
2. 成就设计还没重做完成：当前只是补齐 26 条现有成就的奖励和条件字段，尚未扩展到设计大纲要求的 50+ 成就与里程碑体系。
3. 小队势力闭环未完全证明：已有成员字段、编成面板和势力 tick，但“势力成员池 -> 邀请入队 -> 小队战 -> 岗位产出暂停/恢复”需要完整实现和手测。
4. 宝黄天三类交易已从空池变成可购买，但残方合成、完整传承去重、购买后 UI 反馈和经济平衡还要继续扩测。
5. 十绝体只有基础约束和亲和矩阵，灾劫差异、空窍可视化差异、叙事事件链仍属后续阶段。
6. 音频目前只有 Store 层 voice/ui/ducking，尚未完成配音 manifest、角色曲、触发脚本和诊断日志。

### v0.7.0-b 核心玩法

目标：把“小队势力 + 成就 + 宝黄天 + 十绝体”做成可玩的闭环。

里程碑：

1. 势力闭环：创建、维护、贸易、招募、岗位、成员状态。
2. 小队闭环：势力成员池、邀请入队、信任/利益判断、队伍编成、小队战斗、战后结算。
3. 成就重做：50+ 成就、里程碑替代单纯 `realmNum >= N`、奖励领取 UI、重复奖励防护、杀招解锁。
4. 宝黄天闭环：仙材购买入库、蛊方/残方收藏、杀招残方合成、完整传承学习、价格曲线与稀有度。
5. 十绝体玩法：角色创建显式选择/觉醒叙事、高压槽位 UI、道痕禁制提示、低 HP 操作拦截测试。

验收标准：

- 自动测试：auction 池非空与定价、achievement 条件/奖励、十绝体 HP<5% 阻止操作、state_update passthrough、势力维护/贸易 tick。
- 手测存档：创建势力、招募 NPC、四人小队战、成就领取、宝黄天三类购买、十绝体高压槽位。
- UI 中不再出现“v0.7.0-b 即将开放”的占位文案，除非该 Tab 确实无本轮货物。

### v0.7.0-c 扩展系统

目标：把 v0.7 从“能玩”推进到“有世界差异和可调试能力”。

里程碑：

1. 十绝体特有灾劫：不同体质触发不同灾劫、反噬、叙事约束。
2. 空窍/仙窍可视化差异：十绝体元海、窍壁压力、仙窍道痕密度差异可视化。
3. 地形/阵法战斗影响：小队战与叙事战都能读地形、阵法、克制关系。
4. 音频/配音：`master/bgm/sfx/voice/ui` 五通道、voice ducking、配音 manifest、角色曲触发。
5. 诊断日志：AI state_update、成就、宝黄天、势力 tick、音频触发有统一日志。
6. 测试存档：扩展 `_generate_test_saves.cjs`，生成 v0.7 专项存档。

验收标准：

- Full `npm test` 与 `npm run build` 通过。
- 每个扩展系统至少有一个可复现测试存档。
- 诊断日志能解释“为什么触发/为什么没触发”。
