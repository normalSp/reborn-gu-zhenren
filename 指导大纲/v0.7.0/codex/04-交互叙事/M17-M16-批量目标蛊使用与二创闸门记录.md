# M17/M16 批量目标蛊使用与二创闸门记录

更新时间：2026-05-08

## 工作依据

- 用户补充要求：能以自己或友方单位为目标的蛊虫不应只有 `鸿运齐天蛊`；原著蛊按原著描述登记，二创蛊必须契合世界观、数值平衡和机制平衡。
- `reverend-insanity-lore`：AI 可以提出剧情机会，但蛊虫、仙蛊、流派、材料等有数值效果的内容必须由真相源和引擎校验。
- `v0.7.0-pre 最终收束执行计划 v2`：M17 扩展 `GuUseRegistry` 与统一目标选择；M16 建立 provenance 与世界观发布闸门。

## 已完成

### 1. 批量目标蛊注册

扩展 `src/canon/gu-use-registry.json`，首批登记：

- 低中转友方/自身目标：`月霓裳`、`九叶生机草`、`治愈蛊`、`金风送爽蛊`。
- 仙蛊/高价值支援：`鸿运齐天蛊`、`人如故`、`金甲神盾蛊`、`圣光普照蛊`。
- 战略或剧情门槛：`妇人心蛊`、`血颅蛊`、`酒虫`、`净魂蛊`、`气运蛊`、`功倍蛊`、`洗髓蛊`、`换骨蛊`、`招灾蛊`、`排难蛊`、`态度蛊`。
- 仅作设计锚点/后续接入：`爱情蛊`、`胆识蛊`、`逆流护身印（设计锚点）`。

每个条目登记 `useMode`、`targetRule`、代价、冷却、是否消耗、效果、反噬/副作用、`loreRef`、`provenance`、`balanceTier`。

### 2. 统一目标类型

`GuUseTarget` 扩展为：

- `self`
- `known_npc`
- `dynamic_npc`
- `squad_member`
- `scene_target`
- `aperture_or_location`

自身目标会立即结算属性变化；非自身目标写入 `targetedGuEffects`，避免把 NPC/队友增益误加到玩家身上。

### 3. 场景蛊校验

新增 `resolveSceneGatedGuUseSuggestion`：

- `妇人心蛊`、`血颅蛊` 等普通点击拒绝。
- DeepSeek 只能返回 `state_update.gu_use_suggestions.add` 候选。
- 引擎校验持有、目标类型、场景通过、代价和反噬后才执行。
- 未通过时只记诊断日志，不写数值效果。

### 4. UI 接入

`GuInventoryPanel` 的“使用”按钮不再默认只对自己生效：

- 若蛊支持多个目标，按钮显示 `目标` 并打开目标选择弹窗。
- 目标来源包含玩家自身、人物图鉴 contacts、动态 NPC、后续小队成员、当前场景目标、当前仙窍/地点。
- `scene_gated` 与 `lore_only` 条目仍显示为剧情门槛，不开放普通按钮。

### 5. provenance 与流派闸门

新增 `src/engine/provenance-gate.ts`：

- `unknown` 可交付，但必须有审核白名单、世界观说明、设计职责、数值档位。
- `gu-use-registry` 运行时条目必须有 `provenance`、`loreRef`、`balanceTier`。
- 效果中声明的 `path` 必须来自运行时流派注册表，blocked/category/prototype 不得进入运行时。

### 6. 全库运行时 provenance 策略与旧 path 清理

新增 `src/canon/provenance-policy.json`：

- 覆盖 `gu-database`、`immortal-gu`、`killer-moves`、`npcs`、`training-grounds`、`encounters`、`shop-items`、`auction-runtime`。
- 每个运行时数据集声明来源类型、运行时池、世界观说明、设计职责、数值档位。
- `provenance-gate` 会检查这些数据集的运行时池声明、来源说明和 review 白名单。

同步清理旧 path 污染：

- `蛇信蛊`：`侦察` 是功能标签，不是流派；改为 `信道` 并加复核说明。
- `第二空窍蛊`：根据原著后期“第二空窍蛊为天道蛊虫”的证据，从 `特殊` 改为 `天道`。
- `态度蛊`：`太古` 是时代/来源标签，不是流派；暂对齐项目蛊虫库的 `智道` 口径。
- `起死回生蛊`：`生死道` 未确认，保留条目但 `runtimeAllowed=false`，不进入宝黄天、奖励、战斗或蛊使用池。
- `暗箭仙蛊`：`箭道` 未进入运行时流派注册表，保留条目但 `runtimeAllowed=false`，待复核后再决定是否归入暗道、兵道或仅作为杀招效果。
- `散修联盟训练场`、`传功堂`：不再用 `通用` 作为 `pathType`，分别改为 `力道`、`人道` 并加说明。
- `生死门`：不再写作“生死道天地秘境”，改为天地秘境/规则类奇观，`path=null` 且 `runtimePathAllowed=false`。

## 后续建议

- v0.7.0-a/b/c 做小队系统时，将 `targetedGuEffects` 的目标从 “记录” 升级为真正影响队友/NPC 战斗与剧情。
- 后续每新增一批蛊虫/仙蛊，都必须先补 `GuUseRegistry` 或显式标为 `passive/lore_only`，避免 UI 出现“每只蛊都能随手乱用”的问题。
- `逆流护身印` 等杀招锚点后续应迁入杀招使用注册表，当前只保留设计索引，不进入蛊虫按钮。
