## 已落地

### M1 宝黄天运行时价格收束

源码证据：

- `src/canon/economy-balance.json`
- `src/engine/auction-engine.ts`
- `src/engine/economy-simulation.ts`
- `src/engine/auction-economy-loop.test.ts`

结论：

- 六转/七转/八转/九转仙蛊基础价改为 `3600 / 12000 / 40000 / 120000`。
- 仙蛊方改为仙阶公式：`targetTier * targetTier * 300 + fragmentsRequired * 120`。
- 保留市场热度浮动，运行时不再使用六转仙蛊 `60` 仙元石的低价常量。

### M2 商会蛊材刷新与食料保底

源码证据：

- `src/engine/shop-engine.ts`
- `src/store/slices/merchantSlice.ts`
- `src/components/game/MerchantPanel.tsx`
- `src/engine/merchant-material-shelf.test.ts`

结论：

- 商会蛊材页新增 8 格货架刷新池。
- 每回合 1 次免费刷新；若已有蛊虫安全回合小于等于 3 且低中转食料库存为 0，可触发食料保底上架。
- 三王山前夜不再受固定 7 件材料列表卡死。
- 1-3 转食料优先兜底；4-5 转仍可紧急替代但会逐步收紧；仙蛊食料不进入凡人商会。

### M3 喂养可达性与非物质食料信用

源码证据：

- `src/engine/feeding-rules.ts`
- `src/engine/material-source-audit.ts`
- `src/store/slices/guSlice.ts`
- `src/components/game/GuInventoryPanel.tsx`
- `src/store/slices/gu-feeding-economy.test.ts`
- `src/engine/recipe-source-closure.test.ts`

结论：

- `GuFeedingClosureRow` 增加 `sourceReliability`、`turnsToAcquireEstimate`、`recommendedAction`。
- `regional_generation` UI 标签改为“野外/地域掉落”，它是权重来源，不再等同稳定来源。
- `捧杀蛊=虚情假意` 改为消耗 `feedingCredits.虚情假意`；无信用时不能假喂。
- 事件白名单和地域生成不再默认算稳定闭环，只能算剧情门槛或权重来源。

### M4 势力 runtime 收益五账本

源码证据：

- `src/engine/faction-economy.ts`
- `src/store/slices/factionSlice.ts`
- `src/store/slices/playerSlice.ts`
- `src/components/game/FactionPanel.tsx`
- `src/engine/faction-economy.test.ts`

结论：

- 新增五账本：`grossIncome / maintenanceCost / riskLoss / feedingReserved / netIncome / currencyKind`。
- `advanceTurn` 改为优先调用 `tickFactionEconomy()`，每回合只结算一次势力经济。
- 六转稳定期势力净收益从旧模型 `80-300` 仙元石/回合压回约 `1-2` 仙元石/回合。
- UI 显示毛收入、维护、风险、食料预留与净收益。
