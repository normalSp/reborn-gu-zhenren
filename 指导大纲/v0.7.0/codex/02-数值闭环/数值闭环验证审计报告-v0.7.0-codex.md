## 3. 货币闭环

### 当前公式与来源

| 来源 | 公式/口径 | 问题 |
| --- | --- | --- |
| `src/canon/economy.json` | 仙元石是蛊仙通用货币，可兑换海量元石 | 叙事正确，但没有运行时换算比例。 |
| `src/components/game/MerchantPanel.tsx:119` | 蛊仙购买凡蛊时 `xianPrice=max(1, round(price/10000))` | 隐含 1 仙元石 = 10000 元石，且 UI 文案写成“仙元”。 |
| `src/store/slices/factionSlice.ts:68`、`:81` | 5转用元石，6转用仙元石，不再强制换算 | 方向正确，但和商店 UI 仍不统一。 |

### 判定

当前最危险的不是某个单价，而是“多个子系统各自发明换算”。
v0.7.0 前必须确定唯一货币真相源：

- `元石`：1-5 转交易货币。
- `仙元石`：6-9 转交易货币。
- `仙元`：能量，不是货币，不得在购买 UI 中替代 `仙元石`。
- 凡蛊商店是否允许蛊仙用仙元石付款，需要一个明确换算表，而不是 UI 局部 `price/10000`。

## 4. 势力经济闭环

### 当前公式

证据：`src/store/slices/factionSlice.ts:64-81`。

- 维护费：`level*12 + overSlotMembers*12`。
- 5转返回元石：上述值乘 100。
- 6转返回仙元石：上述值不乘 100。
- 贸易收入：`min(300, 80 + level*10 + members*5 + random(0..49))`。
- 5转返回元石：上述值乘 100。
- 6转返回仙元石：上述值不乘 100。

### 场景粗算

| 场景 | 维护 | 收入 | 净值 | 判定 |
| --- | ---: | ---: | ---: | --- |
| 5转 Lv1 0成员 | 1200 元石 | 9000-13900 元石 | +7800 到 +12700 | 创建成本 8000 元石可能 1 turn 回本，通胀。 |
| 5转 Lv5 6成员 | 6000 元石 | 13500-18400 元石 | +7500 到 +12400 | 满级仍强正收益，维护压力不足。 |
| 6转 Lv1 0成员 | 12 仙元石 | 90-139 仙元石 | +78 到 +127 | 创建成本 3000 仙元石约 24-39 turn 回本，尚可但需定义 turn=日/月。 |
| 6转 Lv10 12成员 | 144 仙元石 | 240-289 仙元石 | +96 到 +145 | 高级势力仍强正收益，缺少岗位、风险、事件损耗。 |

### 判定

势力系统已经从“断裂”变为“过度稳定的正收益”。
v0.7.0 前要补：

1. turn 时间单位：天、旬、月必须统一。
2. 维护费随成员质量、岗位、忠诚、敌对势力压力变化。
3. 贸易收入加入上限、风险、冷却或区域依赖。
4. 成员入队后岗位产出是否暂停，需要规则和测试。

| 杀招传承 | `generateKillerMovePool` 基于 6级以上杀招 | `auction-engine.ts:252-270` | 残方/完整传承的学习、重复处理、战力增幅缺闭环。 |

### 判定

宝黄天已可玩，但还不能算数值可信。最大风险是：势力或仙窍每 turn 产出一旦偏高，玩家可以太快购买 6转仙蛊、仙材和传承。

进入要求：

1. 用 20 turn、100 turn、300 turn 三个窗口模拟蛊仙购买力。
2. 仙材价格要与资源点建造成本、训练场掉落率、炼蛊消耗关联。
3. 仙蛊价格要与福地月产、势力贸易、灾劫消耗关联。
4. 杀招传承必须计算战力增幅，避免低价买到决定性战力。

## 6. 仙窍与资源点闭环

### 当前实现

证据：`src/store/slices/immortalSlice.ts:143-183`。

- 节点产出：`floor(output_rate * externalDays * time_flow_ratio * quality/100 * (1 + dao_mark_density[type]*0.01))`。
- 仙元石产出：当总道痕 `>100` 时，`floor(timeMultiplier * totalDao * 0.001)`。
- 节点建造成本：`100 * gradeMultiplier`，仙材 grade 为 8 倍，见 `immortalSlice.ts:220-226`。

### Critical 漏洞

`immortalSlice.ts:229-239`：

1. `currentCurrency = fullStore.immortalCurrency || fullStore.currency || 0`。
2. 如果 `immortalCurrency=0` 且 `currency=10000`，检查会认为有 10000 可用。
3. 但扣费分支因为 `immortalCurrency !== undefined`，会写回 `immortalCurrency = currentCurrency - cost`。
4. 结果是用元石余额通过仙元石字段扣费，可能造成跨币种污染或凭空增加仙元石。

这是 v0.7.0 前必须修的阻塞项。

### 战斗

- 蛊仙倍率由 `combat-stats.ts:88-93` 给出。
- 标准敌人数值由 `combat-stats.ts:148` 给出。
- 小队战已有阵型、士气、协同，但缺少以“玩家四人队 vs 荒兽/上古荒兽”为对象的 TTK 和失败率测试。

进入要求：

2. 小队战至少覆盖 1、2、4 人队；普通荒兽、荒兽、上古荒兽三档。
3. 十绝体高压/HP<5% 禁止操作要进入战斗与训练入口，而不只在角色创建或 UI 上体现。

当前成就字段完整度已经从“缺失”修到“可测”：

- `achievements.json:5` 标注 26 条已补 `reward/progressMax`。
- `achievement-v070.test.ts:57-64` 验证每条都有 `reward` 和 `progressMax`。

仍需补的不是字段，而是奖励强度：

1. 前期成就奖励不能跳过商店/炼蛊早期循环。
2. 道痕奖励不能让单流派过早突破十绝体或灾劫门槛。
3. 杀招奖励必须处理重复学习和已有杀招升级。
