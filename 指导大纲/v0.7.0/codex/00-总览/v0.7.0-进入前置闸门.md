## 4. 进入 v0.7.0 前必须完成的事项

1. 建立二创 provenance 审计字段和数据迁移计划。
2. 统一货币换算与术语，特别是 `仙元` 与 `仙元石`，以及 `元石 -> 仙元石` 的 UI 换算。
3. 修正资源点建造扣费路径，避免 `immortalCurrency || currency` 导致跨币种扣费或凭空铸币。
4. 给宝黄天、势力、仙窍、训练场、小队战建立可重复的数值闭环脚本或单测。
5. 对旧报告加 superseded 标记或在新版跟踪中明确引用 codex 报告，避免后续开发继续引用过期结论。

v0.7.0 当前不建议直接开始“功能扩张式开发”。推荐先执行 `v0.7.0-pre-gate`，目标不是新增大功能，而是把世界观来源、数值闭环、版本标识和测试闸门固定下来。
完成该 gate 后，再进入 v0.7.0-a/b/c 三段式开发会更稳。
