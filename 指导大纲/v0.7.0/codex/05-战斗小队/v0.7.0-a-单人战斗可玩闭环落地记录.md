## 已完成

### A1 数据地基

- `src/canon/extreme-physique-daomark-affinity.json`：修正非运行时流派，`火道` 统一到 `炎道`，`阳道` 不再作为战斗 path。
- `src/canon/gu-database.json`：核心 42 只蛊补 `combatStatus`，记录战斗角色、使用模式、目标规则、消耗、冷却与风险档位。
- `src/canon/npcs.json`：核心 32 名 NPC 补 `adventureTrust`、`interestDrive`，为 v0.7.0-b 小队招募与信任入口预埋。
- `src/canon/achievements.json`：清理凡人阶段成就绕过仙元石经济的奖励。

### A2 数值真相源

- `src/canon/combat-config.json` 升级为 `2.0-v0.7.0-a`。
- 境界压制表采用“原著压制 + 逆转窗口”：低二转挑战不鼓励硬刚，仙凡跨界仍强压制。
- 流派克制倍率收紧到 `0.70-1.35`，普通优势 `1.15/0.90`，强克制 `1.25/0.80`。
- 道痕公式改成对数夹取：极高道痕不再线性爆炸。

### A3 战斗行动与闸门

- `src/engine/combat-formulas.ts`：新增 seedable RNG、`calcDaoResonance`、可复现伤害测试入口。
- `src/store/slices/combatSlice.ts`：战斗初始化时将已学杀招转换为 `DuelMove`；杀招按钮走真实 `gu_skill` 结算。
- 战斗胜利掉落改走 `MaterialRegistry`，不再生成 `${enemyPath}蛊材` 这类临时 key。
- `src/schemas/narrative.schema.ts` 与 `src/engine/state-update-applier.ts`：新增 `combat_event_candidates.add`，DeepSeek 只可提出战斗候选事件，不能直接写伤害、奖励、任务或正式地点。
- `src/engine/context-builder.ts`：战斗中注入 phase、回合、敌我资源、可用杀招、最近 trace，并明确 AI 只能给候选事件。

### A4 UI 纵切

- `src/components/game/CombatOverlay.tsx`：杀招面板按钮改为寻找对应 `DuelMove` 后调用 `executePlayerAction('gu_skill', moveIndex)`。
- `DuelState.trace` 已开始记录战斗轨迹，供 UI 和 LLM 上下文读取。

## 验证结果

- `npm test`：32 个测试文件 / 258 个用例通过。
- `npm run build`：通过；仍保留 Vite 500KB+ chunk 警告，为非阻塞优化项。
- 浏览器烟测：`http://127.0.0.1:5174/` 首页加载成功，无 console error。

## 尚未完成的 v0.7.0-a 细化项

- 三条完整浏览器路径仍需做深测：一转商队开局、三王山前夜、蛊仙期。
- TTK 需要继续扩充自动模拟：同转普通战、精英战、低一转、低二转、仙凡跨界。
- 战斗 UI 还需要更明确显示：境界压制、道痕共鸣、流派克制、资源扣除、逃跑概率。
- 仙蛊唯一性与短借/场景授权已有规则口径，但仍需在更多具体仙蛊上补运行时校验。

## v0.7.0-b 入口

- 小队战可以复用本轮的 `DuelMove`、`BattleTrace`、`combat_event_candidates` 和数值真相源。
- 小队信任/背叛、外派、多线剧情不应绕过经济与战斗闸门。
- 杀招/仙蛊特效资源可继续接入 `battle-asset-manifest`，但不得阻塞确定性结算。
