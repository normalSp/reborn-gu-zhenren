## 已完成主体纵切

- `src/canon/extreme-physique-daomark-affinity.json`：清理非法运行时 path，`火道` 统一到 `炎道`，`阳道` 不再作为战斗 path。
- `src/canon/gu-database.json`：核心 42 只蛊补 `combatStatus`，记录战斗角色、可用模式、目标规则、资源消耗、冷却和风险档位。
- `src/canon/npcs.json`：核心 32 名 NPC 补 `adventureTrust`、`interestDrive`，为 v0.7.0-b 小队招募和信任系统留入口。
- `src/canon/achievements.json`：复核凡人阶段奖励，不再通过成就绕过仙元石经济和高阶资产门槛。
- `src/canon/combat-config.json`：升级为 `2.0-v0.7.0-a`，境界压制、流派克制、道痕共鸣、资源消耗、逃跑、负伤和掉落策略配置化。
- `src/engine/combat-formulas.ts`：新增 seedable RNG、对数道痕共鸣、流派克制夹取和可复现公式入口。
- `src/store/slices/combatSlice.ts`：已学杀招进入 `DuelMove`，战斗中杀招按钮真实走伤害/资源/冷却/熟练度结算。
- `src/engine/state-update-applier.ts`、`src/schemas/narrative.schema.ts`：新增 `combat_event_candidates.add`，DeepSeek 战斗中只能提出候选事件，不能直写伤害、奖励、任务完成或正式地点。
- `src/engine/context-builder.ts`：战斗上下文注入 phase、回合、敌我资源、可用杀招、trace，并明确玩家是原创角色。

## 细化项收束

### 阶段 1：战斗 UI 信息可信化

- 新增 `src/engine/combat-preview.ts`，输出运行时派生的 `BattlePressureSummary`、`BattleActionPreview`、`EscapePreview`，不进入存档。
- `src/types/index.ts` 新增上述预览类型，避免 UI 各自猜公式。
- `src/components/game/CombatOverlay.tsx` 接入 `buildBattlePreview`，显示境界压制、流派克制、道痕共鸣、攻击预览、逃跑概率、推荐杀招、资源不足原因和可滚动 trace。
- 验证：`npm test -- src/engine/v070a-combat-formulas.test.ts` 通过；`npm run build` 通过。

### 阶段 2：TTK 与战斗收益套利闸门

- 新增 `src/engine/combat-simulation.ts`，提供 seedable 战斗模拟和战斗收益套利审计。
- 新增 `src/engine/v070a-combat-simulation.test.ts`，覆盖同转普通、精英、Boss、低一转、低二转、凡人对蛊仙，以及 100/300 回合刷战斗收益阈值。
- 修复 `src/engine/combat-engine.ts` 中敌方回合和逃跑逻辑引用 `config/C` 未定义的问题，使运行时和测试读取同一份 `combat-config.json`。
- 验证：`npm test -- src/engine/v070a-combat-simulation.test.ts` 通过；战斗相关 3 个测试文件 13 个用例通过；`npm run build` 通过。

### 阶段 3：仙蛊唯一性与场景授权

- `src/engine/gu-use-registry.ts` 新增 `validateImmortalGuBattleUse`，统一校验仙蛊战斗使用的唯一性、持有/短借/剧情授权、境界门槛、仙元成本、长冷却、反噬和 `lore_only` 限制。
- `src/store/slices/guSlice.ts` 的 `useGu` 接入仙蛊闸门，失败时给明确原因，成功时统一结算仙元消耗，避免多次 `setEssence` 造成 stale state。
- `src/canon/gu-use-registry.json` 补 `春秋蝉` 为 `lore_only` 时间重生锚点，不允许普通玩家按钮直接使用。
- `src/engine/gu-use-registry.test.ts` 覆盖凡人直用仙蛊被拒、短借/剧情授权允许、六转仙元不足被拒、六转有仙元可用、`春秋蝉` 普通战斗使用被拒。
- 验证：`npm test -- src/engine/gu-use-registry.test.ts` 通过；战斗相关回归通过；`npm run build` 通过。

### 阶段 4：三路径浏览器深测

浏览器验证使用 Playwright + 本地 Vite dev server `http://127.0.0.1:5174/`，通过受控测试存档注入三条路径，并 mock DeepSeek 续档回复，未读取或记录真实密钥。

- 一转商队开局：可见攻击、逃跑、战斗预览；点击攻击后无错误边界、无 Zod 死循环、无未知材料入库。
- 三王山前夜：可见杀招/推荐杀招信息；点击攻击后 trace 仍可读，未出现 console error。
- 蛊仙期：可见六转、仙元、蛊仙语境；仙蛊门槛和战斗预览不与经济模型冲突。
- 验证结果：三条路径 `failed=0`，`consoleErrorCount=0`，未出现 `蛊虫失控`、`Maximum update depth`、`ZOD_FAIL`、`FETCH_FAIL`。

## 剩余风险

- 浏览器深测是受控存档和 mock DeepSeek 回复，不等于全量真实长线跑档；真实 DeepSeek 战斗候选事件仍需在 v0.7.0-b/c 前继续抽样。
- 本轮只收束单人战斗，不实现完整小队战、外派、多线战争和杀招闪图成品。
- 仙蛊条目已建立统一闸门，但后续新增二创仙蛊仍必须逐条登记 provenance、授权模式、仙元成本、反噬和数值档位。
- 战斗收益套利测试已有阈值，但完整经济平衡仍需和宝黄天、势力五账本、仙窍资源点在 v0.7.0-b 继续联测。

## v0.7.0-b 入口

- 小队战复用 `DuelMove`、`BattleTrace`、`combat_event_candidates`、`combat-config.json` 和战斗预览口径。
- 小队信任/背叛、外派、多线剧情不得绕过本轮的经济、战斗和 AI 候选事件闸门。
- 杀招/仙蛊特效资源可接入 `battle-asset-manifest`，但表现层不能阻塞确定性结算。
