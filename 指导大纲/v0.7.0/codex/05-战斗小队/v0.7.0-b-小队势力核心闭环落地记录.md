# v0.7.0-b 小队/势力核心闭环落地记录

## 执行依据

- `codex上下文信息/2026-05-08-v0.7.0-a-detail-closure-context.md`：确认 v0.7.0-a 单人战斗纵切已完成，v0.7.0-b 可以进入小队编成、小队战、信任/背叛和势力回流。
- `指导大纲/v0.7.0/codex/05-战斗小队/战斗与小队深度设计包.md`：小队战必须采用确定性引擎 + 受控 LLM 事件，LLM 只给候选，不直接写伤害、奖励或正式任务。

- `指导大纲/v0.7.0/v0.7.0-小队势力-成就重做-宝黄天完整版-设计大纲.md`：v0.7.0-b 的核心闭环是“势力成员池 -> 小队编成 -> 小队战 -> 奖励/关系回流”。

## 总目标

v0.7.0-b 交付一个可信、可测、可扩展的小队/势力核心闭环：

1. 从势力成员池或动态 NPC 评估招募意愿。
2. 将符合条件的成员编入最多 4 人小队，入队成员暂停势力岗位产出。
3. 小队战复用 v0.7.0-a 的 `combat-config.json`、`DuelMove`、`BattleTrace`、`combat_event_candidates` 和经济奖励闸门。
4. 战斗结果回流成员信任、忠诚、士气、声望、道心、奖励和传闻。
5. 成就与十绝体只接 b 阶段基础入口，不抢 v0.7.0-c 的灾劫、外派、多线战争和特效大包。

## 必守约束

- 小队战不能另起一套战斗数值源。
- AI 不能直接写伤害、奖励、任务完成、材料入库或正式地点。
- 敌方蛊虫默认死亡、自毁或剧情白名单掉落，不能常规刷蛊。
- 二创 NPC、蛊虫、仙蛊和杀招可以参与，但必须有 provenance、世界观说明、流派白名单和数值档位。
- 原著压制感保留，但必须给战术姿态、队友协作、地形和剧情候选事件留下逆转窗口。
