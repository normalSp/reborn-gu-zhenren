# v0.7.0 前置修复路线图

更新时间：2026-05-07
目标：在正式进入 v0.7.0-a/b/c 功能开发前，先完成可信文档、二创治理、数值闭环和工程闸门。

## 1. 固定开发顺序

所有修复按以下顺序执行，不跳步：

1. canon 数据。
2. TypeScript 类型。
3. engine 计算逻辑。
4. Zustand store。
5. UI。
6. 测试存档与文档。

## 2. Milestone P0：可信基线

交付目标：

- codex 目录下存在本轮四类文档：总览、二创治理、数值闭环、执行计划。
- `codex上下文信息` 有可交接上下文记录。
- 旧数值报告被明确标记为 superseded，不再作为新开发依据。

验收标准：

- 每条关键结论能回指到源码、canon 数据、测试或构建结果。
- 文档不复制原著正文，只记录开发摘要和来源等级。

## 3. Milestone P1：二创 provenance 数据闸门

交付目标：

- 给角色、蛊虫、仙蛊、杀招、训练场、遭遇增加来源治理字段。
- `unknown` 数据不进入宝黄天、训练场掉落、成就奖励、小队招募候选。

推荐字段：

```ts
provenance: 'canon' | 'derived' | 'original' | 'unknown';
sourceNote?: string;
designRole?: string;
balanceTier?: 'low' | 'mid' | 'high' | 'critical';
```

验收标准：

- `immortal-gu.json`、`killer-moves.json`、`training-grounds.json`、`encounters.json` 先达到 100% 标注。
- `gu-database.json`、`npcs.json` 至少达到 95% 标注，剩余条目进入阻塞清单。
- 新增 provenance 完整性单测。

## 4. Milestone P2：货币与数值真相源

交付目标：

- 新增统一货币策略，消除 `仙元` 与 `仙元石` 混用。
- 修复 `MerchantPanel` 的 UI 局部换算。
- 修复 `immortalSlice` 资源点扣费漏洞。
- 把宝黄天、资源点、势力参数从内联常量逐步迁移到 canon 配置。

验收标准：

- 单测覆盖 1-5 转元石支付、6-9 转仙元石支付、禁止能量 `仙元` 当货币。
- 单测覆盖 `immortalCurrency=0` 且 `currency>0` 时不会给 `immortalCurrency` 写入错误余额。
- 文档中所有价格单位一致。

## 5. Milestone P3：核心闭环模拟

交付目标：

- 势力 5/20/100 turn 经济模拟。
- 宝黄天 20/100/300 turn 购买力模拟。
- 仙窍月产与逐日 tick 对齐模拟。
- 白天/黑天训练场风险收益模拟。
- 小队战 TTK 模拟。

验收标准：

- 5转势力不能 1 turn 回本。
- 6转玩家不能靠 Lv1 势力短期稳定购买 6转仙蛊。
- 黑天狩猎在七转门槛下是高风险高收益，而不是秒杀或白送。
- 四人小队战至少覆盖普通荒兽、荒兽、上古荒兽三档。

## 6. Milestone P4：v0.7.0-a 可开发状态

交付目标：

- provenance 字段可被运行时过滤。
- 成就、宝黄天、势力、小队、十绝体的 P0/P1 风险解除。
- `SAVE_FORMAT_VERSION` 迁移补齐 v0.7.0 新字段。

验收标准：

- `npm test` 通过。
- `npm run build` 通过。
- v0.7.0 专项测试存档能加载。
- 开发阶段跟踪文档引用新版 codex 报告。
