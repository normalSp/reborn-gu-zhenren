# v0.7.1-polish 战斗炼蛊物资商会移动端落地记录

更新日期：2026-05-09  
分支：`codex/v070-release-closure`  
版本事实：`package.json = 0.7.1-rc.0`，`SAVE_FORMAT_VERSION = 15`

## 目标

本轮处理浏览器走查中暴露的发布级体验问题：

- 战斗遭遇弹窗在同一段叙事中反复出现，干扰正常剧情选择。
- 升炼没有展示所需蛊材，拆炼没有展示预计回收。
- 本命蛊已有底层字段，但玩家不知道绑定条件、收益和代价。
- 物资袋超过容量后缺少 UI 反馈和实际软惩罚。
- 商会可以买蛊材但不能卖蛊材，玩家整理库存不顺手。
- 炼蛊、商会、蛊虫、战斗弹窗等主要面板在窄屏下容易变形或遮挡。

本轮不扩大到 v0.8 深系统：完整本命蛊成长、完整移动端布局系统、SceneTimeContext 下的战斗/行动/剧情统一调度，仍归入 v0.8。

## 源码落地

### 1. 战斗遭遇候选一次性消费

证据：

- `src/engine/combat-offer-lock.ts`
- `src/engine/response-pipeline.ts`
- `src/components/game/NarrativeCombatPanel.tsx`
- `src/store/slices/encounterSlice.ts`
- `src/engine/combat-offer-lock.test.ts`

处理：

- 新增 `CombatOfferLock`，按 narrative hash、turn、scene/chapter 记录战斗候选冷却。
- `response-pipeline` 对普通叙事关键词只生成一次战斗候选，近期相同候选不再连续覆盖剧情节奏。
- `NarrativeCombatPanel` 在玩家选择策略后立即清空 `transientCombatConstraint`，避免同一候选反复弹出。
- `encounterSlice` 移除“章节过滤永真”的 fallback，改为章节/地域/地点匹配，减少全模板误触发。

v0.8 承接：

- 战斗候选、野外行动、伏击、剧情选择后续应统一纳入 `SceneTimeContext` 与 `CombatIntent`，普通狩猎不应总是进入战斗系统，蛊兽、敌对蛊师、伏击、挑战和剧情白名单才进入完整战斗。

### 2. 升炼/拆炼预览补齐

证据：

- `src/engine/refine-engine.ts`
- `src/components/game/RefinePanel.tsx`

处理：

- 新增 `previewAscendGu`，显示目标转数、成功率、耗时、元石、所需蛊材、缺口和失败后果。
- 新增 `previewDisassembleGu`，显示预计回收蛊材、数量、回收率和不可拆原因。
- 预览复用 `refine-engine`、`recipe-registry`、`MaterialRegistry` 的真实公式，不在 UI 层重写假逻辑。
- 本命蛊、死亡蛊、超出支持转数、缺配方等情况会显示阻断原因，而不是空白。

### 3. 本命蛊解释入口

证据：

- `src/components/game/GuInventoryPanel.tsx`

处理：

- 蛊虫页新增“本命蛊”解释块。
- 未绑定时说明 v0.7.1 不强制一转开局选择本命蛊，完整成长留给 v0.8。
- 已绑定时显示身份绑定、不可普通升炼/拆炼/出售、死亡反噬等收益与代价。
- 本轮不把本命蛊做成普通装备槽，避免背离原著“性命交修”的味道。

v0.8 承接：

- 本命蛊深系统应进入突破、升仙、道途、空窍压力、灾劫、死亡反噬和重生/因果线，不作为普通数值装备处理。

### 4. 物资袋超载与野外软惩罚

证据：

- `src/engine/material-overload.ts`
- `src/engine/field-action.ts`
- `src/components/game/MaterialBagPanel.tsx`
- `src/engine/material-overload.test.ts`

处理：

- 新增 `MaterialOverloadStatus`，按容量计算正常、轻度超载、严重超载。
- 物资袋 UI 显示超载数量和影响说明。
- 野外行动会读取超载状态，降低成功率、提高风险，并在结算标签中展示“物资袋超载”。
- 超载不直接硬惩罚死亡，而是作为发布级软惩罚，促使玩家整理、出售或回城。

v0.8 承接：

- 高级采集生态、负重、行军、伏击、撤离和小队携行应统一进 `SceneTimeContext` 和地域生态。

### 5. 商会出售蛊材

证据：

- `src/engine/shop-engine.ts`
- `src/components/game/MerchantPanel.tsx`
- `src/engine/shop-engine-sell.test.ts`

处理：

- 新增 `quoteMaterialSellBasePrice`。
- 商会出售页新增“出售蛊材”区域，凡人商会只收凡材和低中转材料，未知材料或仙材在凡人商会拒收。

v0.8 承接：

- 黑市、宝黄天、仙窍资源交易和势力商路可承接仙材、高阶蛊材和特殊材料处理。

### 6. 移动端发布可用修复

证据：

- `src/components/game/NarrativeCombatPanel.tsx`
- `src/components/game/RefinePanel.tsx`
- `src/components/game/MerchantPanel.tsx`
- `src/components/game/GuInventoryPanel.tsx`

处理：

- 战斗弹窗在窄屏下限制高度并内部滚动，策略按钮改单列。
- 炼蛊面板从固定左右栏改为窄屏上下布局，列表和详情都能滚动。
- 商会头部、分类页和出售蛊材区域支持窄屏换行与横向标签滚动。
- 蛊虫卡片窄屏改单列，避免卡片挤压。

v0.8 承接：

- 当前是“发布可用修复”，不是完整移动端交互重构。v0.8 应建立统一移动端布局系统：全屏抽屉、底部导航、触控尺寸、战斗/炼蛊/商会/舆图专用移动模式。

## 剩余风险

- 真实浏览器仍需用户走查：战斗候选在长剧情链中是否还会误触发；商会卖蛊材价格是否过低或过高；手机端实际触摸体验是否足够顺手。
- 完整 SceneTimeContext 未在 v0.7.1 重写，因此行动、剧情、战斗之间的深层时间协议仍是 v0.8 任务。
- 本命蛊在 v0.7.1 只解释和阻断高风险操作，完整玩法价值仍需要 v0.8 设计落地。
