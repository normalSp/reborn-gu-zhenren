## 源码变更摘要

- `src/canon/audio-source-manifest.json` 新增为运行时音频真相源。
- `src/engine/audio-resource-policy.ts` 不再从 `public/audio` 导入 JSON，避免 Vite public import 警告。
- `src/canon/character-bgm-manifest.json` 扩展为“私用曲禁用 + 免费 fallback 可运行”的双轨触发。
- `src/engine/character-bgm-trigger.ts` 修正同一角色存在禁用私用曲与启用 fallback 时的 actor 匹配。
- `src/store/slices/soundSlice.ts`、`CombatOverlay.tsx`、`SquadCombatOverlay.tsx` 更新真实 BGM 路径。
- `src/utils/audio.ts` 将 UI 确认/保存音效映射到真实免费素材。

## 验证结果

- `npm test -- src/engine/audio-resource-policy.test.ts`：通过 1 文件 / 5 用例。
- `npm test`：通过 41 文件 / 304 用例。
- `npm run build`：通过；Vite 500KB+ chunk 警告仍为非阻塞优化项。
- `npm run test:e2e:long`：通过 12 条 Playwright 长测路径。

## 剩余风险

- 用户确认的角色曲文件尚未本地提供，因此免费公布包使用 fallback，不是最终社区曲体验。
- 五域 BGM 已可运行，但风格仍是“免费替代”，后续可以继续按蛊真人氛围精挑更贴脸的素材。
- `public/audio/bgm/scene/menu.mp3` 仍未启用，也没有运行时代码加载；若后续主菜单需要音乐，应先登记 manifest 再放文件。
- 项目仍存在构建体积 500KB+ 警告，发布前可进一步做代码分包。
