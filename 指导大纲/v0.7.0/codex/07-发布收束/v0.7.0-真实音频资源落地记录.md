# v0.7.1-rc.0 真实音频资源落地记录

更新时间：2026-05-10

## 执行结论

- 已把用户提供的 `bgm/v0.7` 曲目整理进 runtime 目录，不让运行时代码直接引用根目录 `bgm/`。
- 已启用私用二创包 manifest：`runtimePackage=nonprofit_fan_pack`，仅用于非盈利二创版本。
- 免费五域 BGM、战斗 BGM、UI/SFX 继续保留现有可用素材，并补充 manifest 与 attribution；本轮没有盲目替换成新的外部下载资源。
- 所有 `runtimeEnabled=true` 的 BGM/SFX 均要求文件存在且非 0 字节，音频策略测试已覆盖用户曲和多曲轮换。
- `SAVE_FORMAT_VERSION` 未变化，音频资源、测试存档、宣发物料不新增存档字段。

## 免费 BGM/SFX 状态

- 五域 BGM、普通战、Boss 战、死亡/逆流河 fallback 已有可运行文件和 attribution。
- UI 点击、确认、悬停、战斗命中、暴击、爆炸、成就、警示、突破、死亡、发现、环境风雨洞窟等 SFX 已纳入 manifest。
- 本轮没有额外从 Kenney、OpenGameArt、Pixabay 或 Freesound 下载新文件，原因是现有文件已覆盖发布前所需品类；盲目替换会增加来源审计成本。
- 后续若要提升音色，可以按 manifest 替换具体文件，不需要改运行时逻辑。

## 源码变更摘要

- `src/canon/audio-source-manifest.json`：启用私用二创包用户曲，补充 SFX 条目。
- `public/audio/audio-source-manifest.json`：同步发布镜像 manifest。
- `src/canon/character-bgm-manifest.json`：启用角色曲和场景曲，补充红莲/马鸿运多曲条目。
- `src/engine/audio-resource-policy.ts`：允许 `user_supplied` 且属于私用二创包的本地曲目通过 runtime 校验。
- `src/engine/character-bgm-trigger.ts`：支持同角色多曲可复现轮换。
- `src/engine/audio-resource-policy.test.ts`：补充用户曲启用、缺失曲禁用、多曲轮换测试。
