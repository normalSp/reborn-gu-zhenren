## 问题结论

用户指出的 v0.6.0 遗留问题成立：部分出身、天赋、道具在 UI 中承诺了“炼蛊成功率 +5%”“商会交易优惠 5%”等效果，但旧实现没有统一 modifier 子系统，导致部分效果只停留在描述层。

## 本轮工程决策

- `modifier` 是工程内部数值修正概念，不是蛊真人世界观资源分类，不写入 MaterialRegistry。
- 出身、天赋、道具不再让各系统私写判断；统一从 `modifier-registry.json` 解析。
- UI 只显示“已接入”的效果，避免继续把未落地效果包装成已生效。
- 本轮未新增持久化字段，不提升 `SAVE_FORMAT_VERSION`。
