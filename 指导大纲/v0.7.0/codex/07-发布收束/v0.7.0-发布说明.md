## 本轮 P2 发布收束内容

- AP 协议收束：AP 是当前时段行动余裕，`advanceTurn` 进入下一时段并回满 `3/3`；调息、修行、突破、采集、侦察、小队入队和外派统一消耗 AP。
- 自然恢复：每日/时段真元恢复走自然恢复，不扣 AP、不扣元石；元石/仙元石恢复改为显式调息动作。
- 调息：凡人消耗元石补真元，蛊仙消耗仙元石补仙元，均消耗 1 AP 并推进回合；野外/商路保留被干扰风险。
- 修行与突破：新增轻量修行进度和突破失败惩罚池；失败可能造成 HP 损伤、真元/仙元震荡、蛊虫饥饿/受伤、空窍压力和进度损失。
- 野外行动：采集、侦察、陷阱检查、撤离准备进入统一 `field-action`；稳定收益行动消耗 AP，采集收益被限制在低转日常经济范围。
- 承诺效果消费者：声望/NPC、杀招熟练、仙窍资源产出、仙元自然恢复、灾劫压力均已接入 modifier。
- UI 诚实化：创建页可按效果状态筛选；蛊虫页显示食料节流进度；遭遇日志显示危险遭遇权重修正来源。

## 已知边界

- P2 没有把完整大境界突破、升仙失败、复杂灾劫、夜修/昼修差异、高级采集生态全部做成成品系统；这些继续以“待系统”或叙事效果展示。
- 上述深系统已正式分流到 v0.8.0：夜修/昼修与高级采集进入前中期地图生态；伏击链路进入遭遇与战斗候选；完整大境界突破/升仙失败、复杂灾劫后果进入宿命前夜、天意/因果与结局矩阵。
- 发布前规则：`已登记未消费` 不作为可发布状态残留；每条数值承诺必须是 `运行时生效`、`创建时生效`、`待系统`、`叙事效果` 或 `需降级` 中的一类。
- 真正角色曲/社区曲文件仍以用户本地提供为准；本说明不把音乐资源作为本轮阻塞项。
- 调息、修行、突破、野外行动已有 engine/store 能力，但发布前仍建议补更明确的 UI 入口和手测记录。

## 进入 v0.8.0 前建议

- 必须做：让用户浏览器走查创建页、调息、蛊虫喂养、采集/侦察、小队外派、宝黄天、十绝体压力，确认体验和文案足够直观。
- 最好做：给修行/突破/野外行动补更明显的 UI 按钮和反馈，不让这些能力只停留在 store action。
- 可优化：继续清理创建页/文档中因历史编码造成的显示乱码，提升公布版第一印象。
- 不建议做：不要在 v0.7.0 继续扩完整昼夜修行、升仙失败、复杂灾劫、伏击全链路、高级采集生态；这些已经进入 v0.8.0 深化矩阵。
