# v0.7.0 P1 承诺效果覆盖审计

更新时间：2026-05-09

## 结论

本轮审计确认：v0.6.0 遗留的“UI/文案承诺了数值加成，但运行时未生效”问题确实存在，且不是单点 bug，而是缺少统一 modifier 真相源、运行时 consumer 和 UI 覆盖状态标注造成的系统性问题。

P0 已解决炼蛊、升炼、残方补全、仙蛊升炼、商会买卖价格等低风险入口；P1 本轮继续补齐两个已经登记但未消费的低风险入口：蛊虫喂养消耗与遭遇风险。采集、陷阱、夜修、仙窍产出、灾劫抗性、杀招悟性等仍缺对应稳定子系统，本轮不伪实现，统一进入 `planned-needs-system`。

## 审计方法

- 扫描范围：`src/data/talents.ts`、`src/data/talents-p4.ts`、`src/canon/faction-data.json`、`src/canon/modifier-registry.json`、遭遇、训练、商会、宝黄天、战斗、喂养、炼蛊和创建 UI。
- 关键词：`成功率`、`消耗`、`折扣`、`优惠`、`风险`、`产出`、`采集`、`陷阱`、`伏击`、`逃跑`、`恢复`、`修行`、`突破`、`声望`、`悟性`、`灾劫`、`反噬`、`先手`、`夜间`、`白天`、`收益`、`加成`、`提升`、`降低`、`+N%`、`-N%`、`×N`。
- 判定口径：每条显示承诺必须能回指到数据来源、modifier key、runtime consumer 和测试证据；没有 consumer 的数值承诺不得标为“已生效”。

## 覆盖状态定义

| 状态 | 含义 | 发布处理 |
| --- | --- | --- |
| `运行时生效` | 已登记 modifier，且有实际 consumer 和测试 | 可展示数值 |
| `创建时生效` | 只在角色创建/初始资源/初始属性时一次性写入 | 可展示，但不得暗示持续运行时加成 |
| `已登记未消费` | 已有 registry key，但还没有稳定 consumer | 不应标为已生效 |
| `待系统` | 需要采集、陷阱、夜修、灾劫、仙窍等新子系统 | 降级展示或标注“待系统” |
| `叙事效果` | 只影响身份、风格、关系倾向、文案语气 | 不展示成数值承诺 |
| `需降级` | 文案有数值，但无法映射到现有系统 | 发布前改文案或进入后续计划 |

## 本轮工程改动

- `gu_feed_cost_mult` 不再直接把 1 个食料打折成小数食料，而是采用“食料节流进度”：每次喂养累计节省值，累计满 1 时抵扣一次合法食料消耗。
- 新增持久化字段 `feedingDiscountProgress`，默认 `{}`，存档格式从 `14` 升至 `15`，旧存档迁移补默认值。
- `encounter_risk_mult` 接入遭遇触发概率与危险遭遇权重；该效果只影响引擎候选，不放宽 DeepSeek 写奖励、写任务、写材料的权限。
- 创建界面不再只写“已接入”，改为按条展示 `运行时生效`、`创建时生效`、`待系统`、`叙事效果`、`需降级` 等覆盖状态。

## 高风险/缺子系统清单

| 承诺类型 | 当前来源示例 | 风险 | P1 处理 |
| --- | --- | --- | --- |
| 每日真元恢复、修行速度、突破稳定 | `勤能补拙`、`稳扎稳打`、高阶天赋 | 需要统一修行/突破/调息日结算 | 标记待系统，不伪接入 |
| 采集、草药鉴定、材料鉴定 | `蛊材行家` 等 | 需要采集行动、地域资源、鉴定难度和掉落表 | 标记待系统 |

| 夜间/白天修行、环境真元消耗 | 多个流派亲和天赋 | 需要时间段、地形、修行行为和消耗模型 | 标记待系统 |
| 杀招威力、杀招悟性、流派战斗加成 | 大量流派天赋 | 需要战斗动作、杀招熟练、伤害公式逐条接入 | 标记待系统 |
| 仙窍面积、资源节点、灾劫抗性/强度 | 蛊仙天赋与十绝体相关描述 | 需要仙窍经济、灾劫和道痕禁制共同结算 | 标记待系统 |
| 势力声望、NPC 初始亲和、正魔阵营冷淡 | 社交/道心类天赋 | 需要声望与 NPC 关系政策统一消费 | 部分叙事效果，数值项待系统 |

## UI 诚实化标准

创建页、天赋页、势力归属页后续新增文案时必须遵守：

- 如果已经有 modifier consumer，显示 `运行时生效`。
- 如果只是初始资源、初始属性、本命蛊保底，显示 `创建时生效`。
- 如果需要尚未完成的系统，显示 `待系统`，或把数值描述降级成非数值叙事。
- 如果只影响口吻、身份、关系倾向，不写成数值。
- 如果文案写了百分比但没有系统承接，发布前必须修改或进入本清单。

## 后续 P2 建议

P2 不应继续零散补 if 分支，而应按系统建消费者：

1. 修行/突破/调息日结算：承接每日恢复、修行速度、突破稳定、夜修/昼修。
2. 侦察/采集/陷阱/逃跑：承接采集成功、材料鉴定、伏击概率、逃脱成功。
3. 声望/NPC 政策：承接正魔阵营、NPC 初始亲和、势力影响力。
4. 战斗与杀招熟练：承接杀招威力、杀招悟性、先手、首击、持久战。
5. 仙窍/灾劫：承接仙窍面积、资源节点产出、灾劫强度、十绝体抗性。

### 仍不伪实现的承诺

- “夜间修行/白天修行差异”“伏击全链路”“完整大境界突破/升仙失败”“复杂灾劫后果”“高级采集生态”仍属于后续系统深化，不在 v0.7.0 发布前强行补成数值捷径。
