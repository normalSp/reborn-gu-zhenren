# v0.7.0 行动面板与战斗表现增强记录

更新时间：2026-05-09  
分支：`codex/v070-release-closure`  
版本：`package.json=0.7.0-pre.0`，`SAVE_FORMAT_VERSION=15`

## 目标

本轮处理发布前体验问题：P2 已经把调息、修行、突破、野外行动接入 engine/store，但缺少统一前台入口；战斗系统已有确定性结算和闪图队列，但视觉表达仍偏工具化。本轮只做 UI 与视觉表达增强，不改战斗公式、不新增顶层存档字段。

## 已完成

1. 独立“行动”面板
   - `GameScreen` 新增侧栏入口 `actions`，懒加载 `ActionPanel`。
   - `ActionPanel` 汇总显示 AP、当前时段、真元/仙元、修行进度、调息、修行、突破和野外行动。
   - 面板调用既有 store action：`meditateWithPrimevalStone`、`meditateWithImmortalStone`、`practiceCultivation`、`attemptBreakthrough`、`performFieldAction`。
   - 新增 `buildActivityPanelState` 作为派生预览层，显示成功率、风险、AP 消耗、资源消耗和 modifier 来源。

2. 流派视觉 profile
   - 新增 `path-visual-profiles.json` 与 `path-visual-profiles.ts`。
   - 首批覆盖核心流派：力道、血道、宙道、宇道、魂道、梦道、炎道、水道、冰道、金道、木道、智道、运道。
   - `validatePathVisualProfiles` 会校验 profile 中的运行时流派必须存在于 `path-registry.json` 且 `runtimeAllowed=true`。

3. 杀招/仙蛊闪图 fallback 增强
   - `DuelMove` 增加可选 `path`，`convertKillMoveToDuelMove` 写入杀招流派。
   - `BattleVisualEffectEvent` 增加 `pathId`、`motif`、`secondaryTint`、`intensity` 等派生视觉字段。
   - `buildBattleVisualEffectFromDuelMove` 会根据杀招流派选择 fallback 颜色与动效；无图片时不再全部走单一金色或蓝白底。
   - `BattleFlashOverlay` 增加流派动效层，支持力道冲击、血道斩痕、宙道涟漪、宇道裂纹、梦道雾化、运道金线等 CSS fallback。

4. 战斗 UI 初步显性化
   - `CombatOverlay` 从窄卡片扩展为更宽的战斗面板。
   - 顶部状态区改为我方 / VS / 敌方三段式，保留压制、流派、道痕、行动预览和 trace。
   - 本轮不改小队战核心逻辑，不新增战斗数值源。

## 验证

- `npm test -- --run src/engine/activity-panel.test.ts src/engine/path-visual-profiles.test.ts`：通过，2 个测试文件 / 5 个用例。
- `npm test`：通过，48 个测试文件 / 337 个用例。
- `npm run build`：通过，ActionPanel 独立分包，未出现 500KB+ 主包警告。

## 剩余风险

- 真实浏览器中的视觉观感仍建议用户手动走查：行动面板是否足够顺手，战斗闪图是否过强或过弱。
- 当前特效是 CSS fallback，不依赖真实杀招图片。后续用户放入美术资源后，只需扩展 `battle-asset-manifest.json`。
- 仍有历史编码显示问题需要单独整理，不属于本轮行为增强范围。

## 后续建议

- 若用户认可“行动”面板交互，可在 v0.8.0 剧情模式中把特定剧情行动、调查、潜伏、闭关等也挂到同一行动协议下。
- 战斗 UI 下一步可以补“战斗前确认页”，提前展示敌我差距、推荐准备、可用杀招和逃跑窗口。
