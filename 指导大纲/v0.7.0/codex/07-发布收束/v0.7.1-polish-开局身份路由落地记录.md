# v0.7.1-polish 开局身份路由落地记录

日期：2026-05-09  
分支：`codex/v070-release-closure`  
目标：修复“选择商家/武家/熊家等出身后，首段剧情仍把玩家写成古月山寨族人”的发布级身份串线问题。

## 一、问题复核

旧逻辑以时间线章节作为最高约束。南疆一转默认落到 `qingmaoshan` 章节，而章节位置写死为古月山寨，context-builder 又持续向 DeepSeek 注入“当前章节位置”，导致非古月出身也容易被写成古月族人或古月族学弟子。

这不是单纯文案问题，而是开局身份路由缺失：出身、出生地、初始声望、保底资产、章节约束和 AI 禁令没有统一真相源。

## 二、本轮实现

- 新增 `src/canon/start-profiles.json`：按时间线、势力、境界定义 `StartProfile`。
- 新增 `src/engine/start-profile.ts`：提供开局路由解析、保底蛊降级、flag payload、覆盖校验。
- 拆分熊家与武家：
  - 熊家寨新增为青茅山本地三寨分支，保留力道/熊力蛊口径。
  - 武家改为南疆超级势力边支，不再混用熊家力道设定，一转阶段不直接发放越阶风道蛊。
  - 白家寨新增为青茅山本地三寨分支，深线保底蛊留到 v0.8。
- 角色创建/时间线配置 UI 增加开局身份预览：出生地、身份、二创/原著边界、开局前提和保底资产说明。
- `context-builder` 将 `StartProfile` 作为高优先级身份锚点。若章节默认位置和开局身份冲突，以开局路由为准。
- `narrative-consistency` 新增开局身份一致性修复：非古月 start profile 被写成“古月族人/古月族学弟子”时，自动替换为路由身份并记录 issue。

| 出身 | v0.7.1 开局路由 | 处理方式 |
|---|---|---|
