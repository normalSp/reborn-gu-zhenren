## A. 势力经济闭环（5种场景建模）

**代码关键参数**：
- 5转山寨创建: 8000元石 (`指导大纲/v0.7.0/设计大纲.md` §1.2.2)
- 6转仙门创建: 3000仙元石
- 维护公式: `等级×12 + (成员数-基础槽位)×12`（标注"仙元/回合"）
- 贸易收入: 80-300/回合（标注"仙元"）
- 仙窍产出: 上等福地 25-40仙元/月（economy.json参考值）

### 场景1: 5转初始（无仙窍，Lv1势力，3成员）
| 项目 | 数值 | 说明 |
|------|------|------|
| 创建消耗 | 8000元石 | 5转玩家基准20000元石，消耗40% |
| 维护/回合 | 24 | 公式写"仙元/回合"，5转无仙元来源 |
| 收入/回合 | 0 | 无仙窍产出，贸易是否适用于5转未定义 |
| 判定 | **❌断裂** | 5转势力无明确定义的收入来源 |

### 场景2: 5转满级（无仙窍，Lv5势力，6成员）
| 项目 | 数值 |
|------|------|
| 维护/回合 | 108 |
| 收入 | 0（同场景1问题） |
| 判定 | **❌断裂** | 108/回合不可持续，20000元石仅撑185回合 |

### 场景3: 6转初始上等福地（Lv1→Lv6，6成员）
| 指标 | Lv1 | Lv6 |
|------|-----|-----|
| 维护 | 12 | 108 |
| tickAperture产出 | ~1/turn | ~2-3/turn |
| 贸易(中值) | 190 | 190 |
| 净收益 | **+179** | **+85** |
| 判定 | **✅可行** | 持续正向收益 |

### 场景4: 6转满级（Lv10势力，12成员，上等福地）
- 维护: 228，净收益(中值): **-33**
- 判定: **⚠️需调参** — 依赖贸易高区间(>150)才盈利

### 场景5: 7转满级（Lv10势力，12成员，上等福地）
- 维护: 228（与6转相同！），净收益(中值): **-29**
- 判定: **⚠️需调参** — 维护不随转数递减，成长收益感弱

## B. 宝黄天交易闭环

| 场景 | 判定 | 详情 |
|------|------|------|
| 1. 6转(40000)买第二空窍(8-12k) | ✅可行 | 占20-30%资产 |
| 2. 6转买3个6转杀招(32.4k) | ⚠️偏贵 | 消费81%资产 |
| 3. 5转(20000元石)宝黄天 | ⚠️限制 | 需realm≥6，5转不可用 |
| 4. 上架7转仙蛊回收 | ✅合理 | 底价4900，成交5500-6500 |

### 🔴 严重断裂：TIER_BASE_PRICE
```typescript
// auction-engine.ts:27
const TIER_BASE_PRICE = { 6:4, 7:12, 8:35, 9:100 };
```
6转仙蛊起拍 **3-5仙元石**，设计应为 **3600+仙元石**。缺少×1000倍率。

## C. 仙窍产出成长曲线

### C1: 设计公式 vs 实现对比
| 层面 | economy.json设计 | 代码实现 |

| 公式 | `面积×流速×节点×0.05` | `timeMultiplier×totalDao×0.001` |

**判定: ❌断裂** — 两条公式不匹配，economy.json从未被代码调用。

## 断裂点汇总

| # | 断裂点 | 严重度 | 修复方向 |
|---|--------|--------|---------|
| 1 | 5转势力无收入源 | 🔴严重 | 改用元石维护；增加山寨税收20-50元石/turn |
| 2 | TIER_BASE_PRICE差2000倍 | 🔴严重 | 改为 `{6:4000, 7:12000, 8:35000, 9:100000}` |
| 3 | 黑狱龙ATK=400秒杀6转 | 🔴严重 | 分三阶段: ATK 60→120→250 |
| 4 | 仙窍产出双轨不一致 | 🟡中等 | 统一产出路径；修正economy.json公式 |
| 5 | 势力维护不随转数递减 | 🟡中等 | 加入转数因子: `-(转数-6)×20` |
| 6 | 荒兽无独立数值表 | 🟡中等 | 建立: 普通HP=150+r×50, 上古HP=300+r×150 |
| 7 | 特等福地未实现 | 🟡中等 | 评定≥95时返回特等福地参数 |
| 8 | 黑天8000仙元风险过高 | 🔵低 | 降成本至5000 或 增加失败保底产出 |
