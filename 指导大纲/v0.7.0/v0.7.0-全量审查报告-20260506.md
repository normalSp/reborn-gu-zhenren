## 〇、执行摘要

**总体结论**：v0.6.0代码库基础设施扎实，但v0.7.0设计大纲中约60%的新系统在代码中处于"占位/空白"状态。三份设计文档间存在5处关键不一致（含1处已确认待修复）。数值系统整体设计合理，但5转势力创建原方案存在经济闭环断裂（已在本次审查中修正）。

**关键发现（Top 10）**：

| # | 严重度 | 发现 |
|---|--------|------|
| 1 | 🔴 高 | `SquadCombatOverlay.tsx` 为零字节空桩（`return null`），小队战UI需从零构建 |
| 2 | 🔴 高 | `factionSlice.ts` 仅为v0.6.0版，完全不包含PlayerFaction/SquadMember/招募/忠诚度/信任-利益模型等任何v0.7.0势力系统状态 |
| 3 | 🔴 高 | `combat-formulas.ts` 缺少`checkAffinityBlock`函数，十绝体道痕禁制逻辑未实现 |
| 4 | 🟡 中 | `AchievementCheckState`缺少7个v0.7.0新条件字段，`AchievementPanel`无奖励领取机制 |
| 5 | 🟡 中 | `context-builder.ts` 无十绝体/势力/NPC组队/名场面/域外交5个AI注入管道的任何代码 |
| 6 | 🟡 中 | 音频系统为2通道（BGM+SFX共用），与v0.7.0四通道架构差距巨大 |
| 7 | 🟡 中 | `combat-constraints.json` 仅4个南疆场景，无地形系统、无荒兽战斗数据 |
| 8 | 🟡 中 | 5转势力创建原方案（1000仙元石+5000元石）存在经济闭环断裂→已修正为8000元石纯元石 |
| 9 | 🟢 低 | `PlayerState`/`MortalAperture` 缺少 `extremePhysiqueType` 字段 |
| 10 | 🟢 低 | 13个SFX MP3文件已存在但从未被代码调用，全量使用OscillatorNode合成音 |

**工作量评估**：v0.7.0-std需要28-32天，与实际代码差距匹配。但SquadCombatOverlay和factionSlice的从头构建是最大的工程瓶颈。

## 三、数值系统统一性验证

### 3.1 货币获取-消耗闭环

**5转山寨创建（修正后）**：
- 创建成本：**8000元石**（纯元石，本次审查修正）
- 5转玩家标准元石：20000（审计报告§1.3）
- 创建后剩余：12000元石
- 评估：**✅ 可行**，玩家保留60%资产用于初期运营

**6转仙门创建**：
- 创建成本：3000仙元石
- 6转玩家标准仙元石：40000（审计报告§1.3）
- 创建后剩余：37000仙元石
- 评估：**✅ 可行**，消耗7.5%

**势力维护经济（修正公式）**：
```
维护成本 = 等级 × 12 + (成员数 - 基础槽位) × 12 (仙元/回合)
基础槽位 = 等级对应的免费成员数

示例计算：
- 等级1, 初始3人: 1×12 + (3-3)×12 = 12仙元/回合
- 等级5, 满编6人: 5×12 + (6-5)×12 = 72仙元/回合
- 等级10, 满编12人: 10×12 + (12-10)×12 = 144仙元/回合
```

**收入来源**：
| 收入渠道 | 金额范围 |
|---------|---------|
| 仙窍月产（上等福地）| 25-40仙元/回合 |
| 势力贸易收入 | **80-300仙元/回合**（本次审查上调） |
| 势力任务讨伐 | 200-800仙元/次（不定期） |

**闭环验证**：
- 等级10 × 144维护 - 上等福地40 = 104仙元赤字
- 贸易收入80-300覆盖 → **✅ 闭环完整，等级10时显著正向收益**
- 低等级时维护极低(12-24)，鼓励玩家在低转阶段就体验势力系统
