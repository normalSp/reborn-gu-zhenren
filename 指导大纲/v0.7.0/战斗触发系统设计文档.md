# RebornG 战斗触发系统设计文档

**版本**: v0.7.0  
**状态**: 设计阶段  
**关联文件**: `combat-router.ts`, `encounter-injector.ts`, `combat-constraints.json`

## 一、设计目标

在蛊真人世界观中，战斗不应是"走到第N回合必然触发"的机械事件。本设计文档构建一个**三入口统一战斗触发体系**，使战斗成为AI叙事、随机遭遇、小队遭遇三种入口协调产出的有机结果。

### 核心设计原则

1. **叙事驱动优先**：AI叙事中的战斗关键词触发权重最高，确保战斗与剧情紧密耦合
2. **频率可控**：全局冷却 + 章节密度差异化，防止玩家被连续战斗疲劳轰炸
3. **分级体验**：杂兵→精英→Boss→天灾→奇遇五级体系，难度梯度自然过渡
4. **三入口互斥**：叙事触发 > 随机遭遇 > 小队遭遇，同一回合仅触发一种战斗

## 三、疲劳与冷却系统

### 3.1 全局战斗冷却

```typescript
interface GlobalCombatCooldown {
  // 上次战斗回合
  lastCombatTurn: number;
  // 最小战斗间隔（回合数）
  minIntervalTurns: number;
  // 章节内已触发战斗数
  combatCountInChapter: number;
  // 最大章节内战斗数
  maxCombatsPerChapter: number;
}
```

**冷却规则**：

| 冷却类型 | 默认值 | 说明 |
|---------|--------|------|
| `minIntervalTurns` | 3 | 任意两场战斗间至少间隔3回合 |
| `maxCombatsPerChapter` | 见§4 | 章节内战斗数量上限 |
| `sameTypeCooldown` | 5 | 同类型战斗（如连续两次决斗）间隔 |
| `sameNpcCooldown` | 10 | 同一NPC战斗间隔 |

### 3.2 战斗疲劳累积

```typescript
interface CombatFatigue {
  // 疲劳值 0~100
  value: number;
  // 每次战斗累加
  perCombatGain: number;
  // 每回合自然衰减
  decayPerTurn: number;
  // 疲劳阈值（超过后触发debuff）
  thresholds: { warning: number; critical: number };
}
```

**疲劳效果**：

| 疲劳值 | 状态 | 效果 |
|--------|------|------|
| 0~30 | 正常 | 无影响 |
| 31~60 | 疲惫 | 战斗触发率×0.7，HP恢复-20% |
| 61~80 | 力竭 | 战斗触发率×0.4，属性临时-15% |
| 81~100 | 油尽灯枯 | 强制禁止战斗触发，需休息恢复 |

```typescript
// 疲劳计算公式
fatigueGainPerCombat = 15 + combatDifficulty × 5
  // difficulty: 杂兵=0, 精英=1, Boss=2, 天灾=3

fatigueDecayPerTurn = 3 + (physique / 10) × 2
  // 体魄越高，恢复越快
```

## 四、章节密度平衡

### 4.1 基于章节类型的差异化配置

每个章节在定义时携带战斗密度元数据，存储在 `chapters.json` 或 `combat-constraints.json` 中：

```json
{
  "chapterId": "qingmaoshan",
  "combatDensity": {
    "type": "balanced",
    "maxCombats": 4,
    "minInterval": 4,
    "baseTriggerModifier": 1.0,
    "encounterWeights": {
      "杂兵": 0.45,
      "精英": 0.30,
      "Boss": 0.15,
      "奇遇": 0.10
    }
  }
}
```

### 4.2 章节密度类型

| 密度类型 | 最大战斗数 | 最小间隔 | 触率修正 | 适用章节 |
|---------|-----------|---------|---------|---------|
| `peaceful` | 2 | 6 | ×0.5 | 新手引导、城镇休整 |
| `balanced` | 4 | 4 | ×1.0 | 主线冒险（青茅山、商路） |
| `dangerous` | 6 | 3 | ×1.3 | 野外探索（南疆深处） |
| `warzone` | 8 | 2 | ×1.6 | 势力战、大规模冲突 |
| `boss_rush` | 3 | 5 | ×1.0 | Boss连战章节 |

### 4.3 章节进度动态调整

```typescript
function getChapterDensityModifier(
  chapterId: string,
  chapterProgress: number, // 0~1
): number {
  const config = getChapterCombatConfig(chapterId);
  const baseMod = config.baseTriggerModifier;
  
  // 章节初期：低触发（铺垫期）
  if (chapterProgress < 0.2) return baseMod × 0.4;
  // 章节中期：正常触发
  if (chapterProgress < 0.6) return baseMod × 1.0;
  // 章节后期：高潮期，触发增强
  if (chapterProgress < 0.9) return baseMod × 1.4;
  // 章节结尾：降低，为转场留白
  return baseMod × 0.5;
}
```

### 5.3 概率分布动态调整

```typescript
function getAdjustedEncounterWeights(
  chapterConfig: CombatDensityConfig,
  playerRealm: number,
  playerFatigue: number,
): Record<EncounterTier, number> {
  const base = chapterConfig.encounterWeights;
  
  // 境界压制：高转玩家杂兵概率降低
  const realmModifier = {
    '杂兵': Math.max(0.1, 0.55 - (playerRealm - 1) * 0.1),
    '精英': 0.20 + (playerRealm - 1) * 0.02,
    'Boss': 0.08 + (playerRealm - 1) * 0.01,
    '天灾': 0.02 + (playerRealm - 1) * 0.005,
    '奇遇': 0.15,
  };
  
  // 疲劳影响：高疲劳时天灾/奇遇概率降低
  if (playerFatigue > 60) {
    realmModifier['天灾'] *= 0.3;
    realmModifier['奇遇'] *= 0.7;
  }
  
  return realmModifier;
}
```

## 六、三入口协调机制

### 6.1 优先级规则

```
叙事触发 (NarrativeTrigger)
  │
  ├─ 优先级: P0 (最高)
  │  条件: AI叙事匹配 combat-constraints.json 场景
  │  行为: 立即触发，跳过其他入口
  │
  ├─ 失败 ↓ (无匹配或冷却中)
  │
随机遭遇 (EncounterTrigger)
  │
  ├─ 优先级: P1
  │  条件: encounter-injector 概率判定通过
  │  行为: 注入遭遇模板到AI提示词
  │
  ├─ 失败 ↓ (概率未通过或冷却中)
  │
小队遭遇 (SquadTrigger)
  │
  ├─ 优先级: P2
  │  条件: 玩家主动派遣小队 (非自动触发)
  │  行为: 独立小队战斗流程
  │
  └─ 失败 → 本回合无战斗
```

### 6.2 互斥规则

```typescript
interface CombatTriggerResult {
  triggered: boolean;
  source: 'narrative' | 'encounter' | 'squad' | 'none';
  combatType: 'duel' | 'narrative_combat' | 'squad_combat' | null;
  data: NarrativeCombatTrigger | EncounterInjectionContext | SquadEncounterTrigger | null;
  skipReasons: string[];  // 各入口未触发的原因
}

function coordinateTriggers(
  narrativeResult: NarrativeCombatTrigger | null,
  encounterResult: EncounterTriggerResult,
  squadRequest: SquadEncounterTrigger | null,
  cooldown: GlobalCombatCooldown,
  fatigue: CombatFatigue,
): CombatTriggerResult {
  // 疲劳封顶：禁止任何触发
  if (fatigue.value >= 81) {
    return { triggered: false, source: 'none', combatType: null, data: null,
      skipReasons: ['玩家力竭，需休息恢复'] };
  }

  // P0: 叙事触发优先
  if (narrativeResult) {
    return { triggered: true, source: 'narrative',
      combatType: narrativeResult.combatType, data: narrativeResult,
      skipReasons: [] };
  }

  // P1: 随机遭遇（受冷却约束）
  if (encounterResult.triggered && isGlobalCooldownReady(cooldown)) {
    return { triggered: true, source: 'encounter',
      combatType: 'duel', data: encounterResult.injectionContext,
      skipReasons: [] };
  }

  // P2: 小队主动遭遇（不受冷却约束，但受疲劳影响）
  if (squadRequest && fatigue.value < 61) {
    return { triggered: true, source: 'squad',
      combatType: 'squad_combat', data: squadRequest,
      skipReasons: [] };
  }

  return { triggered: false, source: 'none', combatType: null, data: null,
    skipReasons: collectSkipReasons(narrativeResult, encounterResult, squadRequest, cooldown) };
}
```

```typescript
function normalizeTriggerKeywords(raw: string[] | KeywordTiers): KeywordTiers {
  if (Array.isArray(raw)) {
    // 旧格式兼容：全部归为 explicit 级别
    return { explicit: raw, implicit: [], environmental: [] };

  return raw;
}
```
