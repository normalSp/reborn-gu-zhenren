# v3.7.0-a1 proposal graph scenario model 设计门禁

日期：2026-05-25
状态：completed
冻结版本：`v370_a1_multi_npc_small_faction_proposal_graph_v1`

## 模型目标

把 v3.6 的 synthetic/generic 多 lane 升级为 proposal graph：

- NPC 节点：3-5 个泛化 L2/L3 候选，分别表达短工旁观、山路问话、补给听风、临场小队谨慎等低阶场面压力。
- 小势力压力节点：1 个泛化公开压力源，只聚合公开场面压力，不写正式势力规则。
- 安全节点：继承 rejected / expired / needs_user_decision，继续 reason-only。

## 节点字段

每个 graph node 必须记录：

- `graphKind`、`lifecycleStatus`、`layer`、`genericArchetype`。
- `publicEventRef`、`motive`、`pressureVector`。
- `rumorFactBoundary`、`pressureHandoffBoundary`、`safeNextStep`。
- `factsCommitted=false`、`saveWritten=false`、`promptWritten=false`、`canonPromoted=false`。
- `formalRelationWritten=false`、`factionStandingWritten=false`、`warrantWritten=false`、`recruitmentWritten=false`、`blockadeWritten=false`。

## 通过标准

- `buildV370TransientAgentProposalGraph()` 能从 v360 report 构建 graph。
- `smallFactionPressureNode.connectedNodeIds` 覆盖所有 NPC candidate nodes。
- graph 只读显示，不进入 store/save/prompt/canon。
- source refs 必须包含 D-370，保留 v3.4-v3.6 继承证据。

## 结论

a1 通过。模型可进入 a2 文案门禁与 b1 runtime 第一刀。
