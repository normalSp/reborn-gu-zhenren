# v3.7.0 例外停机清单

日期：2026-05-25
状态：startup proposed；pending user approval

批准 D-370 后，Codex 可在授权包内连续完成 v3.7。出现以下任一情况必须停机并询问用户。

## 技术与存档

- 需要新增 save field、migration 或 bump `SAVE_FORMAT_VERSION`。
- 需要新增 `runFingerprint`。
- 需要 store slice、persistent agent state、persistent agent memory。
- 需要改 DeepSeek prompt/context/model/authority。
- 需要 live DeepSeek probe 或 live narrative eval。

## Lore / MiroFish

- 触及真实命名 NPC、真实势力、真实地点、hidden-adjacent、方源公开证据、L4/L5 原著锚点。
- 需要正式 lore 结论、knowledge-index body、runtime canon promotion。
- 需要 MiroFish export/intake。
- 需要把 hidden/private body 放进 UI、DeepSeek、report body 或 archive。

## Gameplay authority

- 需要 formal relation、faction standing、warrant、recruitment、blockade。
- 需要正式地点、阵营、身份、奖励、NPC 生死、战斗结果。
- 需要将小势力 pressure 从 candidate 变成正式事实。

## Infra / external

- 需要 backend/BFF/service/job queue/eval archive/cloud save。
- 需要外部 framework PoC/dependency/read-only scan/subagent/patch artifact。
- 需要项目外写入、外部 agent 运行、自动 git 或自动修补。

## Release / legal

- 需要 public wording、release、legal/copyright 口径变化。
- 需要 EdgeOne 部署或部署自动化。

## Testing

- CI 或本地测试连续失败，且修复需要改变 v3.7 授权包。
- Player Advocate 出现方向性困惑、玩家误以为候选已成事实，且需要改 scope。
- deterministic / drift gate 出现 P0/P1，且无法在授权包内修复。
