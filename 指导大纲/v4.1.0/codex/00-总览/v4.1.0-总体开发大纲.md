# v4.1.0 总体开发大纲

状态：startup drafted。

## 主线

`Auto-Theater Contract / Schema / Checker 加固`。

## 目标

1. 把 v4.0 的 Auto-Theater 概念收束为统一工程合同。
2. 建立后续 v4.x 共用的 preparation / theater / stack / ledger / expression / evidence 语言。
3. 用 report-only checker 和 synthetic/generic negative fixtures 验证边界。
4. 确认玩家能理解候选、事实、拒绝、压力、需要用户决策的区别。

## 预期阶段

- a0：v4.0 证据复盘与 v4.1 scope freeze。
- a1：Auto-Theater contract v1 设计门禁。
- a2：ExpressionAuthority / WorldCore evidence / negative fixture 门禁。
- b1：若 D-410 获批，新增 `v410` report-only checker 与样本。
- b2：deterministic contract stability / negative fixtures hardening。
- b3：Player Advocate 与 old-save/no-save/rollback 证据。
- rc：质量收束、Skill sync、dashboard/AGENTS/PROJECT-STATE、提交推送和 CI。

## 完成口径

可以说：

- v4.1 建立了 Auto-Theater 的第一版工程合同。
- 后续 v4.2-v4.9 可用同一份 contract 继续推进。
- 项目已有 checker 覆盖候选伪装事实、表现层越权、DeepSeek 越权、HeavenWill/Fate 裁决误用等风险。

不能说：

- 已实现 Auto-Theater UI。
- 已迁移凡阶战斗。
- 已开放高阶战斗 runtime。
- 已生成素材。
- 已开放杀招栈数值或仙蛊屋状态。
- 已接入 HeavenWill/Fate runtime。
