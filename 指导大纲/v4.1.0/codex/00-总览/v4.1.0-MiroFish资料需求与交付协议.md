# v4.1.0 MiroFish 资料需求与交付协议

状态：startup drafted。

## 当前 need level

`not_needed`。

理由：

- v4.1 只做 Auto-Theater contract/schema/checker。
- 样本必须使用 RebornG-owned synthetic/generic，不使用真实原著战役、命名 NPC、方源、尊者、hidden-adjacent 或 formal lore conclusion。
- v4.1 不新增知识库正文、runtime canon、DeepSeek visible lore/RAG。

## 升级 blocking 的条件

以下任一条件出现，必须停止并向用户申请：

- 需要真实原著战役、真实地点、真实势力或命名 NPC。
- 需要方源、尊者、天道/宿命真实锚点、隐藏因果或原著关键人物证据。
- 需要 hidden-adjacent material 或 private/canon body。
- 需要 formal lore conclusion、knowledge-index body 或 runtime canon promotion。
- 需要把 MiroFish 基础包切片导入 v4.1 样本或 checker。

## 若未来需要 MiroFish

仍按双仓流水线：

`RebornG request -> MiroFish export -> RebornG intake review -> test matrix / rule draft`

RebornG 只吸收 rewritten summary、source pointer、candidate/rule/test material，不复制原文、不暴露 hidden body、不直接进入 runtime 或 DeepSeek visible context。
