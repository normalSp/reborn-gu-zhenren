# v4.1.0-process-2 长线漂移与知识边界复核

状态：completed。
日期：2026-05-26。

## 复核目标

v4.1 的漂移风险不来自运行时剧情，而来自战斗表达合同的语义漂移：

- Auto-Theater 被误读成 runtime 已上线。
- 候选、pressure、rejected、needs_user_decision 被误读成事实结论。
- UI/DeepSeek/动效/素材从表达层滑向裁决层。
- HeavenWill/Fate pressure 被误读成玩家结局裁决。
- 原著命名 NPC、hidden-adjacent、正式地点/奖励/NPC 生死被误写进样本。

## 检查方式

- `v410` checker：35 个 synthetic/generic 样本。
- deterministic：240 轮 synthetic report-only contract stability。
- Player Advocate：50 轮玩家可理解性复核。
- MiroFish need-level：`not_needed`。
- live DeepSeek：否。

## 知识边界

本版本没有使用 MiroFish export/intake，没有引用原著真实隐藏事实，没有新增命名 NPC，没有写 knowledge-index body，没有晋升 runtime canon。

允许：

- 使用通用层级名：air / ground / underground / water / aperture / formation / domain / house / environment。
- 使用通用战斗合同词：preparation envelope、killer-move stack frame、combat ledger、pressure、candidate、rejected、needs_user_decision。
- 使用 generic/synthetic 样本模拟越权与 future gate。

禁止：

- 正式原著地点、阵营、身份、奖励、NPC 生死。
- 方源或其他原著关键人物 runtime agent。
- hidden/private body。
- 真实原著事实结论。
- DeepSeek visible lore/RAG。

## 结论

process-2 通过。v4.1 的长线漂移和知识边界风险被限定在 report-only contract gate 内；没有出现 runtime 叙事漂移、原著知识泄漏或正式 lore 晋升。
