# v4.1.0-a2 表达权限与证据链门禁

状态：completed。
冻结标识：`v410_a2_expression_authority_worldcore_evidence_gate_v1`。
日期：2026-05-26。

## 目标

明确 Auto-Theater 中谁能表达，谁能裁决，谁只能提出候选。v4.1 的核心安全价值在于：UI、DeepSeek、动效、美术、report 都不能把候选伪装成事实。

## ExpressionAuthorityV1

| 表达主体 | 可做 | 不可做 |
|---|---|---|
| UI | 展示 WorldCore fact、candidate、rejected、needs_user_decision、pressure lane | 计算命中、伤害、奖励、NPC 生死、地点破坏 |
| DeepSeek | 对已给定材料写旁白、解释候选意图、改写 player-readable summary | 裁决战斗、宿命、奖励、正式地点、正式阵营、隐藏事实 |
| 动效 / 美术 | 表达已批准事实或明确候选/压力状态 | 用视觉暗示候选已发生、仙蛊已获得、地点已毁、NPC 已死 |
| report checker | 分类样本、指出越权、生成证据报告 | 成为 runtime truth 或 canon |
| Player Advocate | 验证玩家理解 | 修改规则或裁决事实 |

## WorldCoreEvidenceChainV1

所有 `fact` lane 必须能追溯：

```text
WorldCore / Combat Core / canon gate
-> evidenceRef
-> ledger entry
-> UI/DeepSeek/动效表达
```

v4.1 不产生 runtime fact；只验证缺 evidence 的样本应被拒绝或 future-gated。

## 负样本门禁

v410 checker 必须拒绝或 future-gate：

- UI 私有结算。
- DeepSeek 裁决命中/伤害/生死/奖励/宿命。
- 动效把候选画成事实。
- Combat Ledger 泄露 hidden/private body。
- theater layer 缺 evidence 却标为 fact。
- HeavenWill/Fate pressure 被写成结局裁决。
- MiroFish/原著真实锚点绕过 intake。
- save/runtime/store/prompt 写入。

## Player Advocate 重点

玩家需要能说清：

- 准备不是胜负。
- 候选不是事实。
- pressure 不是宿命按钮。
- ledger 是证据链，不是万能旁白。
- DeepSeek 是表达层，不是裁决层。

## 边界

本门禁不实现 UI，不调用 DeepSeek，不生成美术，不写 runtime。
