# v0.12.0 需求决策池

日期：2026-05-16
状态：推荐组合已获用户批准；v0.12 本地开发里程碑已完成，rc 质量收束通过。

本文件只收 v0.12 候选需求。用户批准后才进入实现；驳回或延期的需求不得被夹带进代码。

## 候选需求

### D-012-001 v0.12 主线：青茅山正史锚点与 IF 框架

状态：已批准。

内容：

- 原著事实卡扩展。
- 正史锚点表。
- 玩家可见事实/隐藏事实分离。
- 青茅低阶 IF 矩阵、偏离规则和代价。
- 青茅关键压力链第一轮。

价值：

- 让青茅山从区域样板变成有正史压力和 IF 空间的活世界局势。
- 防止后续 route、reaction、方源旁证凭印象开发。

### D-012-002 第一刀先做事实卡/锚点，不先做路线行动

状态：已批准。

内容：

- `v0.12.0-a1` 先扩事实卡和锚点表。
- route/supply/pursuit 放到 a1/a2 之后。

价值：

- 降低返工。
- 保证路线和 IF 判断有事实来源。

### D-012-003 青茅低阶 IF 矩阵、偏离规则和代价

状态：已批准。

内容：

- 继承 v0.8 剧情锚点/IF 矩阵思想，但只做青茅低阶转译。
- 定义 `npc_attention`、`faction_pressure`、`resource_control`、`route_escape`、`hidden_fact_probe`、`local_survival`、`canon_anchor_pressure` 七类 IF 轴。
- 定义每类 IF 轴的偏离等级、代价、可见后果、阻断说明和测试样本。
- 绑定 `livingWorldState.ifDeviations` 或等价受控账本。

风险：

- 若新增持久化字段，需要升存档版本。
- 初版建议优先静态 rules + 现有字段。
- 不得直接复用 `protect_fate`、`break_fate`、`venerable_balance` 等宿命战/尊者级高阶轴。

### D-012-004 逃离青茅山 route / supply / pursuit 第一刀

- 仍不开放逃离成功、新地点、奖励或阵营变化。

### D-012-005 NPC / faction reaction bridge 第一刀

状态：已批准，第一刀已完成。

第一刀结果：

- 已新增局势反应桥，能从公开行动痕迹写入 `factionPressure`、`npcMemories`、`actionConsequences`。
- 仍只写现有 v22 字段，不提升存档版本。

### D-012-006 方源公开旁证事实卡与询问行动

状态：第一刀已完成。

内容：

- 仍只写现有 v22 字段，不提升存档版本。
