## v0.12 已关闭范围

### a1 原著事实卡与正史锚点

已建立青茅事实卡/锚点基础。事实卡只存摘要、source pointer、分类和设计边界，不复制原著正文。

### a2 青茅低阶 IF 矩阵

已将 IF 规则低阶化为青茅阶段可用的轴和代价，不直接复用宿命战/尊者级矩阵。

### b1 route / supply / pursuit

已把逃离青茅山从长期目标推进为准备链：路线候选、补给缺口、追击触发和阻断原因。仍不开放逃离成功或新地域。

### b2 NPC / faction reaction bridge

已让青茅轻量势力/NPC 能根据公开行动痕迹产生可解释反应。仍不开放正式声望、阵营变化或 NPC 生死。

### b3 方源公开旁证询问

已建立方源公开旁证询问入口。只输出公开事实/公开反应/隐藏边界引用，不泄露隐藏因果，不改变方源轨迹。

### process-1 GitHub/CI 工程门禁

已建立 `.github` 最小工程门。PR/push 默认跑确定性质量门，Playwright smoke 暂时保留手动触发。

## 剩余风险

- v0.12 仍是青茅活世界骨架，不是完整青茅主线结局。
- route / reaction / Fang Yuan inquiry 都是保守第一刀，未开放最终结算。
- GitHub Actions 需要在实际 push/PR 后获得第一次远端运行证据。
- 后续若开放正式逃离、阵营身份、声望、NPC 生死、追踪/抓捕、隐藏事实展示或新地域，必须单独走用户决策和测试矩阵。
