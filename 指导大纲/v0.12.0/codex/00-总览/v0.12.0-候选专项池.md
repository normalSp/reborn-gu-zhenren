# v0.12.0 候选专项池

日期：2026-05-16
状态：候选池；已被 v0.12 正式大纲草案吸收，但每项仍需用户在启动审查中最终批准。
说明：前 3 项来自 `v0.11.0-b2` 青茅活世界闭环复盘，后续项来自流程工程化与 v0.11-after backlog。排序不是最终优先级。

## C12-010 青茅逃离路线：route / supply / pursuit 系统

来源：`v0.11.0-b2-6` 已把“逃离青茅山”长期目标拆成路线准备第一步，但没有真正开放路线、补给和追击结算。

内容：

- 路线候选：山道、密林、商队、白家/熊家边缘、临时藏身点等。
- 补给需求：食物、元石、低阶蛊材、伪装/遮掩条件。
- 追击压力：当前身份、公开行为、势力怀疑、NPC 记忆。
- 只允许先做准备、评估和压力回流，不直接 teleport 或解锁新大区。

价值：

- 把“我要逃离青茅山”从长期目标变成连续、可解释、可测试的行动链。
- 为后续地域选择打地基，但不急着开放完整南疆大地图。

## C12-011 NPC / faction reaction engine 第一刀

来源：`v0.11.0-b2` 已写入 `factionPressure`、`npcMemories` 和 `actionConsequences`，但还没有统一的反应引擎。

内容：

- 从公开事实、压力、NPC 记忆和玩家身份推导轻量反应。
- 输出警惕、观望、试探、阻拦、交易窗口、谣言扩散等低风险反应。
- 反应只能写入受控字段，不能直接改阵营身份、奖励、NPC 生死或正史锚点。

价值：

- 让世界开始“回应玩家”，而不是只记账。
- 为白家接触、逃离路线、后续商队/散修/三寨支线统一反应规则。

## C12-012 方源公开旁证事实卡与询问行动

来源：`v0.11.0-b2` 已验证方源调查的隐藏保护失败账本，但正式旁证询问被延期。

内容：

- 先补更多公开可见事实卡和来源指针。
- 只允许玩家询问公开可观察行为、传闻和矛盾，不得直接看到春秋蝉、重生、回溯、私密因果。
- 输出结果以“无证据、公开怀疑、被注意到、关系压力”形式回流。

价值：

- 让玩家能围绕方源行动，但不破坏第三者视角和隐藏事实保护。
- 这是一条高风险线，必须先扩事实卡和测试矩阵。

## C12-013 青茅低阶 IF 矩阵专项

来源：旧 v0.8 有 `canon-anchor-registry.json`、`fate-branch-matrix.json`、`v080-midgame-anchor-rules.json` 和出身深线剧情矩阵，但那些主要面向中后期大锚点。v0.12 需要青茅山低阶版本。

内容：

- 建立青茅 IF 轴：`npc_attention`、`faction_pressure`、`resource_control`、`route_escape`、`hidden_fact_probe`、`local_survival`、`canon_anchor_pressure`。
- 定义每个轴的允许输入、阻断条件、代价、可写字段、不可写字段和 DeepSeek 权限。
- 将 `major_deviation_candidate` 固定为候选/压力/准备链，不直接写正史结果。
- 增加测试矩阵：隐藏事实探查、方源旁证、白家接触、逃离路线、资源控制、NPC 注意、正史锚点压力。

价值：

- 把旧 IF 矩阵思想继承下来，但不让青茅山阶段被宿命战/尊者级复杂度压垮。
- 为 v0.12-b1 route/supply/pursuit 和 v0.12-b2 reaction bridge 提供统一规则源。
- 防止玩家自由目标变成“想什么就成功”，也防止系统过度保守导致没有 IF 自由感。

## C12-001 GitHub 工程门禁专项

来源：v0.11 process-1、用户关于推代码与流程工程化的追问。

内容：

- `.github/workflows`。
- PR template。
- issue template。
- 最小 CI：`npm test`、`tsc`、`build`、关键 e2e 或资产扫描。
- 分支保护建议。

为什么不是 v0.11 现在做：

- 当前工作区历史脏项较多。
- 当前分支名落后于实际阶段。
- v0.11 主线更需要先完成活世界最小闭环。

## C12-002 ADR-lite 补档专项

内容：

- 为以下重大决策补 ADR：
  - `livingWorldState` 和 `SAVE_FORMAT_VERSION = 22`。
  - `WorldIntentEngine` 只做裁决/路由，不做结算。
  - DeepSeek 权限边界。
  - 原著事实卡摘要/来源指针/隐藏 ref 策略。
  - Git 提交与推送制度。

价值：

- 防止后续重复争论“为什么这样做”。
- 给 v1.0 前的大版本收束提供决策索引。

## C12-003 `.learnings` 强化专项

内容：

- 把 `.learnings/ERRORS.md` 从历史记录升级为防复发索引。
- 每个严重 bug 绑定至少一个测试、门禁或检查命令。
- 建立“复发 bug”标记。

价值：

- 真正实现 Reflexion：错误经验进入工程防线，而不是写完就忘。

## C12-004 只读子代理试点

内容：

- 只读 QA/Test Engineering Guardian。
- 只读 Lore & World Designer。
- 不写文件。
- 不处理当前关键路径。
- 只输出问题清单和风险评级。

为什么不是现在做：

- 用户担心稳定性。
- Codex/CodeBuddy 长任务已经有闪退风险。
- 可写多代理会增加冲突和上下文漂移。

## C12-005 Skill 渐进式披露与事实去重

内容：

- 将长版本历史从 skill 主体迁入 references。
- 主体保留硬边界、操作顺序、触发规则。
- 每次只重构一个 skill。

价值：

- 降低上下文占用。
- 减少重复更新和过期事实。

## C12-006 Prompt / Context Budget 可观测

内容：

- system/static/dynamic prompt 估算 token。
- prompt hash。
- cache hit/miss。
- retry cost。
- 高成本路径报告。

边界：

- 不重新评估其他模型。
- 不改变默认 `deepseek-v4-flash`，除非用户重新批准。

## C12-007 Player Advocate 前 10 轮走查制度

内容：

- 固定青茅山前 10 轮走查表。
- 检查目标是否清楚、选择是否有意义、反馈是否像蛊真人世界。
- 走查结论进入需求池，不直接改代码。

价值：

- 防止系统很强但玩家不知道自己在做什么。

## C12-008 Canon Schema 试点

内容：

- 选一个小而关键的 canon JSON 做 test-only schema 校验。
- 候选：事实卡、青茅区域板、资源循环。
- 先记录不一致，不做全量大修。

价值：

- 降低大型 canon 数据字段漂移风险。

## C12-009 技术文章与开发案例整理

内容：

- 内部整理 RebornG 的 AI 叙事管道、专家团治理、测试工程化、活世界裁决。
- 不暴露密钥。
- 不复制原著正文。
- 不承诺开源或发布时间。

价值：

- 后续对外传播或复盘时有材料。

## 明确延期到更后

- 大时代开局。
- 蛊仙期大规模扩权。
- 完整尊者时代因果系统。
- 可写多代理公司化流程。

这些不属于 v0.12 默认候选范围，仍放长期路线。
