# v0.12.0 总体开发大纲

日期：2026-05-16
状态：本地开发里程碑已完成，rc 质量收束通过。
主线：`青茅山正史锚点与 IF 框架`

## 版本目标

v0.12.0 的目标不是继续扩很多玩法，而是把青茅山从“区域样板”升级成有原著事实来源、正史压力和 IF 偏离空间的活世界局势。

本版本要回答：

- 哪些青茅事实是正史硬事实。
- 哪些事实玩家当前可见。
- 哪些事实只能以 hidden ref 形式存在，不能泄露给玩家和 DeepSeek。
- 哪些地方可以低阶 IF 偏离，偏离需要什么条件、代价和账本。
- 玩家要逃离、接触白家、调查方源、介入三寨局势时，系统如何用原著事实和本地规则裁决。

## 专家团结论

用户提出的长期路线定义适合写入 v0.12 正式大纲。
专家团只做一处收束：v0.12 不应该一次性完成青茅山全剧情，也不应该直接进入完整 NPC/势力系统。v0.12 应该先把“正史/可见/隐藏/IF”的事实骨架铺稳，再把 v0.11 已经出现的逃离路线、白家接触、方源旁证纳入受控行动链。

## 核心交付

1. 原著青茅山事实卡扩展。
2. 正史锚点表。
3. 玩家可见事实、隐藏事实、IF 偏离点分类。
4. 青茅低阶 IF 矩阵、偏离规则和代价。
5. 青茅关键压力链第一轮：族学、三寨、资源、路线、狼潮前置压力、战后路线前置，以原著抽取为准。
6. 逃离青茅山 route / supply / pursuit 系统第一刀。
7. NPC / faction reaction bridge 第一刀：只做青茅局势轻量反应。
8. 方源公开旁证询问：放在事实卡扩展和隐藏门禁之后。
9. GitHub/CI 工程门禁：作为 process 小刀穿插。

## 完成标志

- 系统能区分 `canon_hard_fact`、`player_visible_fact`、`hidden_fact_ref`、`if_deviation_point`。
- 系统能把旧 v0.8 的剧情锚点/IF 思想低阶化为青茅 IF 轴，而不是直接复用宿命战/尊者级矩阵。
- 玩家可以侧翼参与青茅局势，但不能随口改写正史核心因果。
- 逃离青茅山不再只是长期目标文字，而能被拆成路线、补给、掩护、追击风险等准备链。
- NPC/势力能对玩家公开行为产生轻量、可解释、可回归测试的反应。
- 方源相关调查只允许公开旁证，不泄露隐藏因果。
- DeepSeek 仍然只写叙事、候选、线索、传闻、压力，不拥有正史、奖励、地点、NPC 生死、IF 结论的最终权。

## 非目标

- 不做完整狼潮主线。
- 不直接推进到青茅山结局。
- 不开放完整南疆大地图。
- 不让玩家直接改变古月一代、白凝冰、方源隐藏因果等核心锚点。
- 不开放正式白家投靠、阵营身份变化或完整势力声望体系。
- 不新增蛊、蛊方、材料、奖励池。
- 不重新评估模型；默认仍为 `deepseek-v4-flash`。
- 不上大规模可写子代理。

## 风险池

- 原著事实抽取不充分会导致 IF 判断凭印象。
- 直接复用 v0.8 宿命战/尊者级 IF 轴会让青茅阶段过重；必须先做低阶矩阵。
- 将 NPC reaction 提前过多会抢 v0.13 的完整 NPC 系统。
- 方源旁证最容易泄露隐藏事实，必须最后做。
- 路线系统容易变成直接开放地图，必须先做准备链和压力。
- 若新增持久化字段，必须提升 `SAVE_FORMAT_VERSION` 并补迁移/默认值/测试。

## 开工建议

建议用户批准 v0.12 主线为：

`青茅山正史锚点与 IF 框架`

第一刀做：

`v0.12.0-a1 原著事实卡扩展与正史锚点表`

不要先做路线行动或 NPC reaction；否则会缺事实地基。
