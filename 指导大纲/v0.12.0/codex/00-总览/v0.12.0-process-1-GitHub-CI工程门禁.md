## CI 策略

默认 PR / push 门禁只跑确定性检查：

- `npm ci`
- `npx tsc --noEmit --pretty false`
- `npm test -- --reporter=dot`
- `npm run build`
- `npm run check:runtime-assets`
- `npm run check:qingmao-assets`
- `npm run check:player-visible-copy`

Playwright smoke 暂时设为 `workflow_dispatch` 手动输入 `run_e2e=true` 后执行：

- `npm run test:e2e -- tests/e2e/v011-free-goal-panel.spec.ts`

原因：v0.12 当前需要先把 GitHub 门禁跑起来，PR 默认门不应立刻被浏览器安装、长测或平台差异拖慢。涉及 UI、自由目标、活世界行动入口、布局遮挡、EdgeOne 预览风险时，PR 模板要求补相关 e2e 证据；rc 再跑完整本地验收。

## PR 模板门禁

PR 必须写清：

- 版本阶段。
- 用户批准的决策编号。
- MiroFish 需求等级。
- 存档版本影响。
- DeepSeek 权限是否扩张。
- hidden fact 是否仍被保护。
- dashboard / handoff 是否更新。
- 哪些测试已经跑过。

## Issue 模板用途

`phase_task.yml` 用于专家团小版本/候选需求落地：

- 标记用户输入类型：direction、boundary、priority、feel、tradeoff、release、not_needed。
- 标记 MiroFish 需求：not_needed、optional、preferred、blocking。
- 写清 in/out scope 和工程门禁。

`bug_report.yml` 用于 bug 到学习闭环：

- 症状。
- 复现路径。
- 影响域。
- 反复防线：测试、扫描或 `.learnings/ERRORS.md`。

## 非目标

- 不创建大型项目管理系统。
- 不引入外部 runtime 依赖。
- 不改变 EdgeOne 部署配置。
- 不自动推送、自动发布或自动合并。
- 不要求所有 PR 默认跑长 e2e。
- 不处理历史脏工作区。
