# v0.12.0 启动审查与范围冻结

日期：2026-05-16
状态：主线与推荐组合已获用户批准；等待 a1 开工
输入类型：方向 + 边界 + 优先级

## 专家团启动结论

建议将 `v0.12.0` 定为：

`青茅山正史锚点与 IF 框架`

原因：

- v0.11 已完成活世界状态、自由意图裁决、原著事实卡试点、可见调查、正式行动样板和长期目标拆行动链。
- 下一步如果直接开放逃离、投靠、狼潮、方源调查，会缺少正史锚点和 IF 代价。
- 长期路线已经把 v0.12 定义为青茅正史/IF 框架，符合当前地基节奏。
- 用户希望“活着的蛊真人世界”，而不是 DeepSeek 即兴改剧情；这必须先由本地事实卡、锚点表和 IF 规则托住。

## 范围冻结草案

### 进入范围

- 原著青茅山事实卡扩展。
- 正史锚点表。
- 玩家可见事实/隐藏事实/IF 偏离点分类。
- 青茅低阶 IF 矩阵、偏离规则和代价。
- 青茅关键压力链第一轮：族学、三寨、资源、路线、狼潮前置压力、战后路线前置。
- 逃离青茅山 route / supply / pursuit 第一刀。
- NPC / faction reaction bridge 第一刀，仅限青茅轻量反应。
- 方源公开旁证询问，必须在更多公开事实卡之后。
- GitHub/CI 工程门禁小刀，可穿插。

### 不进入范围

- 完整狼潮主线。
- 青茅山结局全量重现。
- 逃离青茅山成功或新大区正式开放。
- 白家正式投靠、阵营身份变更、完整声望体系。
- 方源隐藏因果、春秋蝉、重生/回溯信息泄露。
- 古月一代、白凝冰重大 IF、NPC 生死、正史核心因果改写。
- 新奖励、蛊虫、蛊方、材料池。
- DeepSeek 获得正式结算或事实写入权。
- 大规模可写多代理。

## 关键冲突

### 冲突 1：正史压力 vs IF 自由

玩家应该能侧翼参与青茅局势，但不能随口改写正史核心因果。
旧 v0.8 的剧情锚点/IF 矩阵可以作为方法参考，但不能把宿命战、尊者、天意层级的高阶分支轴直接搬到青茅凡人阶段。

裁决：

- 正史硬事实必须先入锚点表。
- v0.12-a2 只做青茅低阶 IF 轴：`npc_attention`、`faction_pressure`、`resource_control`、`route_escape`、`hidden_fact_probe`、`local_survival`、`canon_anchor_pressure`。
- 不得直接复用 `protect_fate`、`break_fate`、`venerable_balance` 等高阶轴。
- IF 偏离必须有条件、代价、可见后果和账本。
- 重大 IF 只做候选和阻断说明，不直接开放。

### 冲突 2：逃离路线 vs 地图开放

玩家想逃离青茅山，但 v0.12 不能直接开放南疆大地图。

裁决：

- v0.12 先做路线准备、补给、掩护、追击压力。
- 不做逃离成功。
- 不新增正式地域。

### 冲突 3：NPC 反应提前 vs v0.13 规划

长期路线中完整 NPC/势力反应在 v0.13。

裁决：

- v0.12 只做 reaction bridge：从公开事实、压力和记忆推导轻量反应。
- 不做完整关系、声望、招揽、通缉、任务网络。
- 完整 NPC 系统仍留到 v0.13。

### 冲突 4：方源旁证玩法 vs 隐藏事实保护

玩家可能想跟踪方源、调查方源，但第三者玩家不该知道回溯、春秋蝉和私密因果。

裁决：

- 先补公开事实卡。
- 询问只允许公开行为、传闻、矛盾和被注意风险。
- 输出不得泄露 hidden fact body。
