## 零、版本定位与硬边界

### 0.1 当前基础

- v0.7.0 已完成经济、炼养用、宝黄天、小队、势力、单人战、小队战、十绝体基础、外派、地形、闪图、音频资源管理、发布闸门等核心地基。
- `gameMode = canon | if` 已存在，时间线选择已有正史线/IF线入口。
- 玩家身份协议已建立：玩家永远是原创角色，方源只能作为原著 NPC、远景事件或剧情交会对象，不能把玩家称作方源。
- DeepSeek 已被约束为候选生成者：不能直接创造材料、蛊方、仙蛊、正式任务、正式地点、数值奖励或关键剧情改写。
- v0.8.0 的主要风险不是缺功能，而是剧情量、IF 分支量、原著边界、天意/宿命结算复杂度过高，必须拆阶段做。

### 0.2 v0.8.0 核心目标

v0.8.0 交付“剧情完整版骨架”：

1. **正史模式**：十个原著锚点按原著大方向发生，玩家以原创参与者身份影响侧线、局部关系、资源与战斗，但不抢方源主角命轨。
2. **IF 模式**：允许关键锚点偏移，采用“护宿命 / 毁宿命”双路线对立，所有偏移进入分支矩阵，有原因、代价、影响范围和后续反噬。
3. **天意与因果系统**：天意不再是万能惩罚按钮，而是压力、修正、排斥、注视、机缘偏移和叙事约束系统；因果债记录玩家行为的长期回响。
4. **结局解析器**：根据宿命状态、锚点偏移、玩家势力、方源关系、天意压力、因果债、尊者平衡、道心倾向生成多结局。
5. **永生边界**：十转、永生、永生蛊只作为未证悬念、求道余韵和远景线索，不作为 v0.8 可达成胜利结局。

### 0.3 必须遵守的原著边界

| 主题 | v0.8.0 处理方式 | 禁止事项 |
|---|---|---|
| 方源 | 原著 NPC / 远景核心变量 / 可能的盟友、敌人、旁观者 | 玩家不能成为方源；AI 不能把玩家称作方源 |
| 宿命蛊 | IF 可护宿命或毁宿命；正史按锚点约束 | AI 不能直接改宿命状态；玩家不能随手获得宿命蛊 |
| 天意 | 压力、修正、注视、机缘偏移、事件概率 | 不作为“因为天意所以失败”的万能解释 |
| 无极魔尊/疯魔窟 | 求证失败、事实浮冰、混沌线索、十转未证 | 不得宣称无极已成功永生 |
| 永生/十转 | 未证悬念、终极追求、结局余韵 | 不开放“炼成永生蛊”“真正永生”结局 |
| 尊者 | 大局棋手，影响结局矩阵 | 不随意降格成普通可招募 NPC 或普通 boss |
| 二创 IF | 可存在，但必须标注 `if-derived | original-if` | 不把二创补完混称原著设定 |

### 0.4 模式定义

| 模式 | 核心体验 | 锚点权限 | 结局权限 |
|---|---|---|---|
| 原著模式 `canon` | 参与原著洪流，见证并影响局部 | 锚点不可改写，只可局部变体 | 原著近线、玩家侧线成败、势力存续 |
| IF 模式 `if` | 玩家成为变量，改变宿命与大势 | 锚点可偏移，必须记录分支代价 | 宿命永存、宿命裂痕、宿命破碎、玩家势力、疯魔求证未证等 |

## 二、正史模式

### 2.1 正史模式体验目标

正史模式不是“玩家什么都不能改变”，而是“玩家不能抢走原著核心因果”。玩家可影响：

- 自己的生死、修为、流派、蛊虫、杀招、势力、小队、道心。
- 与原著 NPC 的关系状态：已闻、已见、短暂合作、敌对、交易、误会、救援、利用。
- 原著事件周边的局部结果：旁支人物存活、资源归属、局部战斗胜负、声望、线索、传闻。
- 后续支线的可见性：某些地点、黑市、传承、势力任务是否提前出现。

玩家不可影响：

- 方源作为原著主角的核心成长节点。
- 宿命战正史大方向。
- 尊者级别大局结果。
- 原著未完结后的真正终局。

### 2.2 正史锚点保护规则

```typescript
interface CanonAnchorPressure {
  anchorId: string;
  pressure: number;          // 0-100
  reason: string;
  attemptedMutation: string;
  engineDecision: 'allow_local_variation' | 'redirect' | 'block';
  fallbackNarrativeHint: string;
}
```

规则：

- 玩家行为靠近锚点核心时，系统增加 `canonAnchorPressure`。
- 低压力：允许局部变体。
- 中压力：允许旁观、协助、间接影响，但不改写核心。
- 高压力：引擎重定向剧情，给出合理阻力，如境界压制、情报缺失、势力封锁、方源提前行动。
- 不允许用“天意强行失败”偷懒，必须优先用世界内逻辑解释。

### 2.3 正史模式 DeepSeek 提示要求

`context-builder` 必须明确注入：

- 当前为 `canon` 模式。
- 玩家是原创参与者。
- 方源是原著 NPC，不是玩家。
- 当前锚点可改写范围。
- 禁止直接改写项。
- 可提供的支线方向。

DeepSeek 可生成：

- 旁支战斗、探索、交易、交涉、情报、传闻、人物接触。

DeepSeek 不可生成：

- 方源死亡、方源无故投靠玩家、宿命蛊被玩家随手得到、尊者无代价被玩家击败、永生蛊可炼成。

## 三、IF 模式：护宿命 / 毁宿命双路线

### 3.1 IF 模式体验目标

IF 模式是“玩家成为变量”，但不是“玩家成为万能主角”。它允许大分支：

1. **护宿命线**：玩家站到天庭、秩序、稳定、宿命约束的一侧。结局可能是宿命永存、天庭秩序延续、众生命轨继续锁定。
2. **毁宿命线**：玩家参与或利用宿命破碎后的自由乱世。结局可能是宿命破碎争渡、尊者棋局乱世、玩家势力立足。

两条线都必须有代价：

- 护宿命：自由度降低、NPC 命运被锁、天庭阵营约束、魔道/散修敌对、玩家自我选择受限。
- 毁宿命：乱世风险上升、天意反馈模糊、旧秩序崩坏、势力冲突增多、因果债加重。

### 3.2 宿命状态

```typescript
type FateState = 'intact' | 'fractured' | 'destroyed';

interface FateSystemState {
  fateState: FateState;
  fateIntegrity: number;       // intact时高，fractured时波动，destroyed时为0
  heavenlyCourtControl: number;
  redLotusDamage: number;
  playerFateAlignment: 'protect_fate' | 'break_fate' | 'exploit_fate' | 'undecided';
  fangYuanFatePressure: number;
  lastMajorChangeTurn: number;
}
```

解释：

- `intact`：宿命完整，重大命轨强约束。
- `fractured`：宿命有裂痕，局部命轨可偏移，红莲影响、春秋蝉、梦境、尊者布置可形成异常。
- `destroyed`：宿命破碎，自由意志释放，但天意仍作为天道压力与混沌风险存在。

### 3.3 IF 分支向量

```typescript
interface IfBranchVector {
  id: string;
  anchorId: string;
  axis: 'protect_fate' | 'break_fate' | 'faction_shift' | 'npc_survival' | 'resource_control' | 'venerable_balance' | 'heaven_will_debt';
  delta: number;
  cause: string;
  cost: string;
  downstreamImpact: string[];
  provenance: 'if-derived' | 'original-if';
  createdTurn: number;
}
```

规则：

- 任何 IF 锚点偏移都必须写入 `IfBranchVector`。
- `delta` 不是好坏，而是偏移强度。
- `provenance=if-derived` 表示基于原著逻辑推演。
- `provenance=original-if` 表示纯二创扩展，必须在 UI 和文档中标注。

### 3.4 护宿命线阶段

| 阶段 | 玩家行为 | 系统反馈 |
|---|---|---|
| 早期 | 亲近正道、天庭、秩序势力，压制魔道/异数 | 天庭信任上升，黑市/魔道线关闭或涨风险 |
| 中期 | 阻止宿命裂痕扩散，举报或破坏逆天传承 | 宿命完整度上升，自由分支减少 |
| 宿命战 | 协助天庭护宿命，阻止方源或反宿命势力 | 进入宿命永存/天庭秩序类结局池 |
| 后期 | 接受命轨束缚或成为秩序维护者 | 玩家势力稳定但上限受限 |

### 3.5 毁宿命线阶段

| 阶段 | 玩家行为 | 系统反馈 |
|---|---|---|
| 早期 | 接触魔道、散修、逆天传承、红莲线索 | 天意注视上升，机缘和危机同时增加 |
| 中期 | 参与宿命裂痕、梦境、逆流河、影宗侧线 | IF 分支增多，锚点压力升高 |
| 宿命战 | 破坏天庭布局，协助或利用方源行动 | 进入宿命破碎/裂痕乱世结局池 |
| 后期 | 在无命时代建立势力或被尊者棋局吞没 | 结局取决于势力、道心、因果债和尊者平衡 |

## 四、天意与因果系统

### 4.1 天意不是万能失败按钮

天意在 v0.8.0 中只承担以下职责：

- 注视：记录玩家是否频繁逆天、破坏命轨、夺取不属于当前阶段的机缘。
- 修正：提高事件风险、增加敌对势力关注、改变资源/情报出现概率。
- 排斥：对极端偏移给予世界内阻力，例如天庭封锁、梦境反噬、灾劫提前。
- 诱导：在某些阶段投放机缘或陷阱，引导玩家进入更高风险的选择。
- 反馈：作为 UI 中可读的压力计，而不是不可见的强制惩罚。

### 4.2 天意账本

```typescript
interface HeavenWillLedger {
  attention: number;      // 被天意注意程度
  correction: number;     // 修正压力
  rejection: number;      // 排斥压力
  ambiguity: number;      // 宿命破碎后天意模糊度
  lastTriggers: HeavenWillTrigger[];
}

interface HeavenWillTrigger {
  kind: 'fate_mutation' | 'venerable_contact' | 'forbidden_gu' | 'massacre' | 'rescue' | 'oath_break' | 'inheritance_seized' | 'chaos_contact';
  delta: number;
  reason: string;
  anchorId?: string;
  turn: number;
}
```

### 4.3 因果债账本

```typescript
interface KarmicDebtLedger {
  totalDebt: number;
  byKind: Record<string, number>;
  pendingReturns: KarmicReturn[];
}

interface KarmicReturn {
  id: string;
  sourceEventId: string;
  expectedWindow: [number, number];
  severity: 'low' | 'medium' | 'high' | 'catastrophic';
  narrativeHint: string;
  resolved: boolean;
}
```

因果债来源：

- 杀戮、背叛、守诺、救援、夺宝、献祭、利用 NPC、违背誓言、篡改锚点。

因果债用途：

- 后续 NPC 反应、势力追杀、商会拒绝、黑市接纳、天意注视、结局矩阵。

## 五、疯魔求证与永生边界

### 5.1 原著边界

v0.8.0 必须明确：

- 原著没有完结，没有给出真正永生已达成的结论。
- 无极魔尊疯魔窟求证失败，不能作为“十转成功”证据。
- 事实浮冰、混沌、十转只是求证与悬念方向。
- 永生蛊可作为传说、执念、未证目标，不可作为运行时可炼蛊虫。

### 5.2 可做内容

| 内容 | 可做程度 | 说明 |
|---|---|---|
| 疯魔窟线索 | 可做 | 作为后期高危探索与尊者棋局线索 |
| 事实浮冰 | 可做 | 作为真相碎片、推演资源、剧情证据 |
| 混沌外灾 | 可做 | 作为世界边界压力，不做普通刷怪 |
| 十转猜想 | 可做 | 只做理论、传承残片、结局余韵 |
| 永生蛊 | 仅传说 | 不进入蛊虫库、炼蛊、宝黄天、奖励池 |

### 5.3 不开放结局

以下结局禁止进入 v0.8.0：

- 玩家炼成永生蛊。
- 方源或玩家确认真正永生。
- 玩家成为十转并稳定统治蛊界。
- 无极魔尊成功复现永生。

可开放：

- 疯魔求证未证线。
- 混沌边界窥见线。
- 十转理论残片线。
- 为 v0.9 或独立原创补完包留下悬念。

## 六、剧情候选协议与 LLM 闸门

### 6.1 新增候选协议

```typescript
interface StoryEventCandidate {
  id?: string;
  anchorId?: string;
  type: 'side_event' | 'npc_contact' | 'rumor' | 'faction_move' | 'inheritance_hint' | 'danger' | 'other';
  title: string;
  summary: string;
  risk: 'low' | 'medium' | 'high';
  source: 'ai-rumor' | 'engine';
  engineValidation?: 'pending' | 'accepted' | 'blocked';
}

interface IfBranchCandidate {
  id?: string;
  anchorId: string;
  axis: IfBranchVector['axis'];
  proposedDelta: number;
  summary: string;
  costHint: string;
  downstreamHint: string[];
  source: 'ai-rumor' | 'engine';
  engineValidation?: 'pending' | 'accepted' | 'blocked';
}
```

DeepSeek 只能返回：

- `story_event_candidates.add`
- `if_branch_candidates.add`
- `canon_anchor_pressure.add`
- `discoveries.add`
- `dialogue_requests.add`
- `combat_event_candidates.add`

DeepSeek 不能直接写：

- `fateState`
- `CanonAnchor.result`
- 关键 NPC 生死
- 正式任务
- 正式地图地点
- 正式奖励
- 材料/蛊方/仙蛊/杀招传承入库
- 结局结果

### 6.2 引擎校验顺序

1. Zod 只校验 JSON 形状。
2. AI 语义验证层过滤非法奖励、非法流派、未知资产。
3. 剧情候选验证层判断锚点权限、模式、宿命状态、天意压力、因果代价。
4. 引擎接受后才写入锚点、IF 分支、传闻或任务候选。
5. UI 展示“候选/已接受/被阻止”状态，避免玩家误以为 AI 文本已经生效。

### 6.3 context-builder 注入

必须注入：

- `playerRole=original_participant`
- 当前 `gameMode`
- 当前 `fateState`
- 当前锚点与锚点压力
- 可偏移/不可偏移字段
- 近期 IF 分支向量
- 天意账本摘要
- 因果债摘要
- 方源关系状态
- 关键 NPC 生死/立场
- 经济与奖励白名单

### 7.2 结局族群

| 结局族 | provenance | 触发方向 | 说明 |
|---|---|---|---|
| 原著近线 | `canon-near` | 正史模式或低偏移 IF | 原著大方向未改，玩家支线收束 |
| 宿命永存线 | `if-derived` | 护宿命成功 | 天庭秩序延续，众生命轨被锁 |
| 宿命裂痕乱世线 | `if-derived` | 宿命未完全毁也未稳固 | 世界规则半碎，天意与宿命互相拉扯 |
| 宿命破碎争渡线 | `if-derived` | 毁宿命成功 | 自由时代开启，代价是乱世与尊者棋局 |
| 玩家势力立足线 | `if-derived/original-if` | 势力、小队、经济、声望高 | 玩家未必改变世界根基，但拥有立足之地 |
| 疯魔求证未证线 | `if-derived` | 后期接触疯魔/混沌/事实浮冰 | 只给真相残片和悬念，不给永生成功 |
| 身死道消线 | `canon-near/if-derived` | 死亡或因果债爆发 | 按死亡总结和定场诗系统收束 |

### 7.3 结局禁止词

结局文本不得宣称：

- “玩家炼成永生蛊”
- “玩家真正永生”
- “无极魔尊已成功永生”
- “十转已被玩家稳定掌握”

允许表达：

- “你窥见了更高层的一线痕迹”
- “混沌之外仍无答案”
- “事实浮冰证明了某条路曾被求证，但不证明它通向永生”
- “长生与永生之间仍隔着不可跨越的雾”

## 九、剧情内容制作规范

### 9.1 每个锚点至少包含

- 入口事件 3-5 个。
- 旁支行动 6-10 个。
- 原著 NPC 接触 2-5 个。
- 关键战斗或冲突 1-3 个。
- 交易/情报/地图线索 3-6 个。
- 正史保护重定向 2-4 个。
- IF 偏移候选 3-6 个。
- 失败/撤退/错过结果 2-4 个。

### 9.2 奖励规范

奖励必须走现有系统：

- 元石/仙元石：经济服务。
- 蛊材/仙材：MaterialRegistry。
- 蛊方/残方：RecipeRegistry / fragment-recipes。
- 蛊虫/仙蛊：蛊虫库、唯一性、境界门槛。
- 杀招：killer-moves provenance。
- 地点：rumorLocations 先传闻，验证后已知地点。
- 任务：dialogue_requests / story_event_candidates 先候选，白名单后正式。

### 9.3 NPC 规范

- 原著 NPC 必须按原著性格、利益、境界、阵营、时间线处理。
- 原著 NPC 可与玩家合作，但不无代价贴靠玩家。
- 二创 NPC 可补量，但必须标注来源、职责、数值档位和世界观说明。
- 方源对玩家的关系不能默认友好，应根据利益、风险、利用价值、时间线位置判断。

## 十二、风险与沟通点

### 12.1 需要持续和用户沟通的点

1. 哪些原著 NPC 在 IF 模式中允许死亡、转阵营或长期离线。
2. 玩家能否在护宿命线中成为天庭外围/正式成员/秩序代理人。
3. 毁宿命线中玩家与方源的关系是否允许从互相利用发展到短期同盟。
4. 疯魔求证线的开放深度：只做线索，还是允许玩家进入后期高危探索。
5. 结局文本的风格：更冷峻、史书式、定场诗式，还是带更强个人回望。

### 12.2 我建议的默认方案

- 原著 NPC 生死：正史模式不改；IF 模式只允许局部和中低风险 NPC 大改，尊者和方源相关必须走高代价白名单。
- 天庭线：允许玩家成为秩序代理人，但不让玩家取代龙公、星宿等核心位置。
- 方源关系：默认互相利用，不默认朋友。
- 疯魔线：v0.8 只做求证未证和悬念，不做成功超脱。
- 结局风格：定场诗 + 冷峻史书摘要 + 玩家一生回望。

### 14.4 v0.8.0-a：前中期生态落地

范围：青茅山、商家城、三王山、王庭。

- 昼夜修行：洞府/客栈/商队营地/野外/敌对区域的修行收益与风险不同。夜间可提高隐蔽修行或梦道/魂道/暗处行动机会，但增加伏击、走火、被巡查或遭遇风险。
- 区域采集：低中转蛊材可从地域生态中获得，但受 AP、危险、再生周期、商会价格和剧情白名单约束，不能取代经济闭环。
- 伏击前半链：侦察、陷阱、警戒、先手和撤离窗口先接入遭遇与单人/小队战入口，形成“能预判、能规避、能反制”的体验。

### 14.5 v0.8.0-b：宿命前夜与高风险成长

范围：义天山、逆流河、梦境/影宗、宿命战。

- 完整大境界突破：突破不再只是一次按钮结算，而是准备、风险评估、尝试、失败后果、叙事回流。普通失败不随意降境界，但会伤身、损真元/仙元、损蛊、加空窍压力或损进度。
- 升仙失败：必须符合蛊真人口径，重点是三气、天劫地灾、空窍/仙窍、蛊虫、道痕和天意压力，不写成普通 RPG “扣经验/掉一级”。
- 伏击全链路：伏击可进入战斗 phase、第三方发现、追击、谈判或撤退候选，由引擎校验，不由 AI 文本直接造成伤害或奖励。
- 灾劫后果：灾劫开始影响资源点、道痕、蛊虫状态、身体伤势、势力损耗和天意/因果账本。

### 14.6 v0.8.0-c：尊者博弈与长线后果

- 高级采集生态扩展到仙材、福地/洞天资源点、宝黄天价格和势力外派，确保高级资源来自高风险、高门槛或高成本路径。
- 灾劫伤痕、突破失败记录、因果债、天意压力进入结局解析器，影响“玩家势力立足线”“疯魔求证未证线”“身死道消线”等结果。
- 不开放“真正永生”“永生蛊可炼可得”“玩家稳定十转”等违背当前原著边界的结果。

### 14.7 发布验收

- `已登记未消费` 不允许出现在 v0.8.0 发布 UI 中。
- `待系统` 若仍存在，必须标注阶段归属和不可用原因。
- 每个深化系统都要有：数据真相源、engine consumer、UI 解释、测试、长测存档和原著/二创边界说明。

## 十五、各出身深线剧情与身份长线

v0.7.1-polish 只解决“开局身份不串线”：玩家选择商家、武家、熊家、铁家、散修等出身时，出生地、身份、初始声望、保底蛊和 AI 禁令必须与选择一致。v0.8.0 承接的是“深线剧情”：每个出身不只是一句背景，而是长期身份压力、专属关系网、资源入口、敌对关系、IF 偏移和结局权重。

### 15.1 设计原则

- 原著有明确基础的出身优先靠近原著：青茅山三寨、商家、铁家、北原黄金血脉、天庭/中洲宗门等。
- 原著描述少但适合玩法的出身走 `derived | original`，必须标注二创边界，不混称原著正史。
- 玩家仍是原创角色。出身决定身份压力和起点资源，不让玩家替代方源、商心慈、白凝冰、黑楼兰等原著人物。
- 出身深线必须接入道心、声望、势力、人物图鉴、舆图、经济、剧情锚点和 IF 分支矩阵。
- 所有深线奖励继续走 v0.7.0 的经济与资产闸门：不能因为“家族出身”绕过蛊虫库、蛊方、材料、宝黄天和 provenance。

### 15.2 v0.7.1 开局路由承接

v0.8.0 的出身深线必须以 `StartProfile` 为入口。`StartProfile` 已在 v0.7.1 记录：

- `timelineNodeId`：时间线锚点，例如 `qingmaoshan`。
- `factionId/factionName`：出身势力。
- `startLocation`：出生地与可活动地点。
- `playerFactionRole`：玩家初始身份。
- `chapterConstraintMode`：本地锚点、锚点变体、外部分支、域默认。
- `openingPremise`：开局叙事前提。
- `initialStandingPolicy`：初始声望与原因。
- `starterAssetPolicy`：保底蛊是否继承、禁用或降级。
- `promptLocks`：AI 禁令，例如“商家开局不得写成古月族人”。

v0.8.0 不再从章节默认位置反推玩家身份，而是由 `StartProfile -> SceneTimeContext -> CanonAnchor` 串联。

### 15.5 验收标准

- 所有可选出身都有 `StartProfile`、深线矩阵条目、阶段归属和 prompt 禁令。
- DeepSeek 若把非古月出身写成古月族人，必须被修复、重试或降级提示。
- 出身声望不再套用同一数值，必须解释“族内登记、外来商队、边支外务、巡查执法、散修无庇护”等差异。
- 出身不能只给数值 buff，必须影响 NPC 态度、商会/黑市入口、舆图传闻可信度、锚点旁支和 IF 结局权重。

## 十六、最终发布标准

v0.8.0 完成时必须达到：

- 正史模式能从青茅山骨架一路跑到天庭末章骨架。
- IF 模式能明确进入护宿命或毁宿命路线。
- 每个锚点有源码/canon/测试证据。
- 天意与因果不是纯文案，而能影响风险、选项、事件、结局。
- EndingResolver 不产生违反原著硬边界的结果。
- 玩家永远是原创角色。
- 方源作为 NPC 的身份稳定。
- 不把永生、十转、永生蛊当作已证实可达成系统。
- 所有剧情奖励、地点、任务、资产都通过 v0.7.0 已建立的闸门。
- 从 v0.7.0 继承的承诺效果延期项完成分流或深化，不再出现“文案承诺但系统没有解释”的发布状态。
