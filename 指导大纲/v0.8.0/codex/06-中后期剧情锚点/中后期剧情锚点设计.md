# v0.8.0-b3 中后期剧情锚点设计

## 目标

本阶段把中后期正史锚点从临时 `flags` 升级为正式 `storyAnchorState`，让义天山、逆流河、梦境与影宗、宿命战侧翼成为长期可追踪系统，而不是一次性叙事提示。

DeepSeek 只能提交候选、压力和传闻。本地引擎决定锚点入场、正史边界、IF 向量、天意账本、因果债、拦截记录和结局输入。

## 覆盖范围

| 锚点 | b3 做什么 | b3 禁止什么 |
| --- | --- | --- |
| 义天山 | 侧翼战场、梦境试炼、局部阵营选择、线索发现 | 玩家夺取至尊仙胎主线位置，提前公开影宗全局真相 |
| 逆流河 | 个人试炼、平行追击、情报获得、撤退或失败 | 替代方源完成逆流河核心名场面，直接接触宿命蛊本体 |
| 梦境与影宗 | 梦境探索、魂道交易、线索传闻、侧支战斗 | 无代价破解影宗全局，随意改变幽魂级大局 |
| 宿命战侧翼 | 侧翼战斗、救援或撤退、战后声望、阵营见证 | 玩家替代方源摧毁宿命蛊，玩家获得宿命蛊，龙公无代价投降玩家 |

`尊者博弈` 与 `天庭末章` 只保留后续入口，不在 b3 做终局解析。

## 状态模型

新增正式持久字段：

- `storyAnchorState.fateState`: 只允许 `intact / fractured / destroyed`。
- `storyAnchorState.anchorResults`: 每个 canon anchor 的长期状态。
- `storyAnchorState.anchorRecords`: 入场、活跃、拦截、偏移记录。
- `storyAnchorState.ifBranchVectors`: IF 模式通过本地校验后生成。
- `storyAnchorState.heavenWillLedger`: 天意关注、修正、排斥、模糊余地。
- `storyAnchorState.karmicDebtLedger`: IF 或越界压力产生的长期因果债。
- `storyAnchorState.storyEventCandidates`: AI 或引擎提交的剧情候选。
- `storyAnchorState.canonAnchorPressureLog`: 正史压力与拦截记录。
- `storyAnchorState.lastResolutionSteps`: UI 可播放/可读的本地结算轨迹。

旧 `flags.fateState`、`flags.ifBranchVectors`、`flags.storyEventCandidates`、`flags.canonAnchorPressureLog`、`flags.heavenWillLedger` 会迁移并保留只读兼容镜像。

## 引擎边界

`v080-midgame-anchor-engine.ts` 是唯一结算入口：

- `evaluateStoryAnchorEntry`
- `validateStoryEventCandidate`
- `resolveStoryEventCandidate`
- `resolveIfBranchCandidate`
- `recordCanonAnchorPressure`
- `applyHeavenWillTrigger`
- `buildEndingResolverInput`

正史模式下，固定锚点只能旁观、侧翼、救援、撤退、关系变化或局部战果。直接改写核心因果会被 `block` 或 `redirect`。

IF 模式下，合法轴会生成 `IfBranchVector`，同时写入天意、因果债或势力代价。IF 不是免费改写，也不能绕过禁区。

## UI 与动效

新增底部面板 `宿命`：

- 展示四个 b3 锚点进度。
- 展示正史/IF 模式、宿命状态、天意账本、因果债。
- 展示候选事件、IF 向量、压力拦截和本地结算轨迹。
- `ChoicePanel` 显示 `正史侧翼 / IF偏移 / 天意压力 / 禁区拦截` 标签。

Motion 负责卡片、标签、tooltip、列表重排和 reduced-motion fallback。GSAP 只保留给重大锚点压力或章节级转场，不控制按钮、文字或列表。
