# v0.8.0 小版本执行路线图

更新时间：2026-05-10  
当前入口分支：`codex/v080-roadmap`  
前置发布分支：`codex/v070-release-closure`  
阶段定位：v0.8.0-pre 工程入场与后续执行拆分

## 总原则

v0.8.0 的目标不是继续堆功能数量，而是把 RebornG 从“系统能跑”推进到“战斗有深度、剧情能长期承载、前端有游戏感”。所有小版本都按 `canon -> types/schema -> engine -> store -> UI -> tests/docs` 的顺序推进。

硬边界：

- DeepSeek 只提供叙事、候选、压力和文本表现，不直接结算血量、真元、资源、掉落、胜负、境界、灾劫和正式战斗结果。
- 运行时流派名必须来自 `src/canon/path-registry.json`。
- v0.8.0 先做凡战深度；仙蛊可以做剧情授权、临时借用、见闻和高压演出，仙蛊屋不做常规运行时系统。
- 不开放“永生蛊”“十转蛊”“真正永生”作为可获得运行时奖励、配方、数值胜利或稳定结局。
- 只有需要持久化新战斗状态时才提升 `SAVE_FORMAT_VERSION`；纯 UI 播放状态不入存档。

## v0.8.0-pre：工程入场

目标：把 v0.7.1 收束和 v0.8 战斗/动效地基推上 GitHub，并让后续开发能从文档与 skill 直接续接。

交付：

- 当前 `codex/v070-release-closure` 通过 `npm test`、`npm run build`、`npm run test:e2e:long` 后推送到 GitHub。
- 新建并推送 `codex/v080-roadmap`。
- 新增本文件，作为 v0.8.0 小版本总路线图。
- 新增 `指导大纲/v0.8.0/codex/04-前端动效/reborn-combat-motion-SKILL.md`，镜像本机 skill，便于仓库追踪和后续恢复。

验收：

- GitHub 上可见 v0.7.1 收束和 v0.8 地基提交。
- 后续开发者只读本文件、战斗深化文档、动效规范和上下文交接即可开始 v0.8.0-a1。

## v0.8.0-a1：凡战引擎竖切

目标：让首批凡蛊和杀招从“规格数据”进入本地可结算战斗引擎。

交付：

- 建立 5x3 `BattlefieldCombatState` 运行时结算入口。
- 实现单位、格子、地形、遮蔽、阵位、危险格、道痕场、持续效果的纯函数模型。
- 实现蛊虫行动校验：持有、冷却、消耗、射程、目标类型、遮蔽、地形限制、禁忌门槛。
- 实现杀招行动校验：核心蛊、辅助蛊、蓄势、打断、环境条件、失败模式、反噬。
- 所有正式战斗输出统一落为 `BattleResolutionStep[]`。

验收：

- 射程、遮蔽、地形、阵位、状态叠加、冷却、资源不足、缺核心蛊、打断、撤退都有单元测试。
- UI 不计算命中、伤害、状态变化和胜负，只消费引擎输出。

## v0.8.0-a2：战斗 UI 与动效竖切

目标：把凡战引擎接入可玩的棋盘战斗界面。

交付：

- 中央 5x3 棋盘，显示地形、遮蔽、阵位、危险格、道痕场、单位和持续效果。
- 左侧我方单位/小队，右侧敌方与战场情报。
- 底部行动栏分为：蛊虫、杀招、阵法、身法、观察、撤退。
- 悬停/选中蛊虫时显示射程、影响格、消耗、反制、剧情用途和不可用原因。
- 战斗日志升级为“战斗轨迹”，逐步播放 `BattleResolutionStep[]`。
- GSAP 只负责开战、杀招蓄势/释放、阵法展开/破阵、重伤、反噬、终结等时间轴演出。
- Motion 负责棋盘单位移动、选中反馈、Tooltip、行动面板、状态条和列表重排。

验收：

- 桌面 1440x900 和移动端 390x844 可操作、无文本重叠。
- reduced motion 模式可用。
- 动效不遮挡按钮、原因文本、血量、真元和战斗轨迹。

## v0.8.0-a3：蛊虫剧情表现化

目标：让蛊虫描述在剧情选择中也成为可调用能力，而不是只在背包里有说明。

交付：

- 剧情上下文注入玩家持有蛊虫的 `sceneUtilities`。
- 场景选择识别侦察、追踪、治疗、解毒、破障、遮掩、威慑、禁忌仪式等用途。
- DeepSeek 可以提出候选行动与叙事描述，但候选必须映射到已注册 `GuExpressionSpec` / `KillerMoveExpressionSpec`，否则降级为传闻或线索。
- 前中期锚点先覆盖青茅山、商队、商家城、三王山相关场景。

验收：

- 同一剧情场景中，有无对应蛊虫会产生不同可选解法。
- AI 文本不得直接写入正式奖励、掉落、战斗胜负或资源变化。
- 剧情选择能清楚解释“为什么能用、为什么不能用、风险是什么”。

## v0.8.0-b1：群像战斗

目标：把单人凡战扩展为小队、伏击、阵位和第三方介入。

交付：

- 小队单位上场、援护、牵制、受伤、撤退和士气状态。
- 伏击链路接入侦察、遮蔽、追踪、地形和先手。
- 阵位/破阵进入 `BattleResolutionStep`，不在 UI 私算。
- 第三方介入、友军误伤、保护目标和护送目标进入战斗目标模型。

验收：

- 至少覆盖一场商队遭遇、一场三王山混战、一场阵法遗迹战和一场高压撤退战。
- 战斗结束后能写回关系、伤势、消耗、战利品候选和后续剧情压力。

## v0.8.0-b2：修行与灾劫深化

目标：把大境界突破、升仙失败、灾劫和资源压力做成可玩的长期系统。

交付：

- 大境界突破尝试记录门槛、资源、风险、失败代价和后续伤势。
- 升仙失败按身体、空窍/仙窍、蛊虫、道痕、天意压力和势力后果结算。
- 地灾天劫接入战斗、资源、伤势、声望和剧情压力。
- 高级采集生态、商会、黑市、宝黄天传闻和势力商路统一进入经济/场景时间协议。

验收：

- 失败不是简单扣血或掉级，而是生成可追踪后果。
- 低成本 AP 行动不能稳定刷出高级蛊材、仙材或仙窍收益。

## v0.8.0-b3：中后期剧情锚点

目标：把义天山、逆流河、梦境、影宗、宿命战侧翼等中后期压力接入系统。

交付：

- `fateState`、`HeavenWillLedger`、`KarmicDebtLedger` 进入中后期剧情候选。
- 原著模式只允许旁观、侧翼战场、救援、撤退、关系变化和局部战果。
- IF 模式允许站队和偏移，但必须写明资源、关系、天意、因果和势力代价。
- 出身深线影响阵营站队、突破资源、NPC 态度和锚点入口。

验收：

- 正史锚点不被随意改写。
- IF 分支可偏移，但每次偏移都有可追踪原因、代价和后续反噬。

## v0.8.0-c1：终局框架

目标：建立尊者博弈、宿命状态、玩家势力收束和结局解析器。

交付：

- 结局解析输入包含：战斗记录、杀招反噬、禁忌蛊代价、群战伤亡、灾劫伤痕、因果债、天意压力、出身债务和势力关系。
- 结局标注 `canon-near | if-derived | original-if`。
- 尊者不降智；玩家可成为变量，但不能无代价替代尊者。

验收：

- 不出现“炼成永生蛊”“真正永生”“稳定十转玩家”等越界结局。
- 结局说明能追溯到玩家长期选择，而不是临门随机文本。

## v0.8.0-c2：全局重皮与体验收束

目标：把 v0.8.0 的前端体验统一成有蛊真人味、信息清楚、交互有游戏感的整体界面。

交付：

- 状态栏、底部导航、存档、图鉴、蛊虫背包、选择面板、人物图鉴、商会、事件日志统一暗墨底、金色道痕、血色风险、玉青生机、阵纹交互。
- 长日志可滚动，紧凑面板文字不溢出。
- 所有 UI 都能解释“能用/不能用/风险/代价/后续影响”。
- 移动端布局形成稳定规范，不再只做局部修补。

验收：

- Playwright 截图覆盖桌面和移动端主路径。
- reduced motion 下仍可完整游玩。
- 新增动效不得引入不可读文本、遮挡按钮或布局抖动。

## v0.8.0-rc：发布候选

目标：冻结功能，做全量回归、长测和文档冲突清理。

交付：

- 全量测试存档覆盖凡战、群战、阵法、剧情蛊虫用途、突破、灾劫、IF 偏移、终局解析。
- 清理 v0.7.x 与 v0.8.0 文档中已经 superseded 的旧结论。
- 更新上下文交接，写清分支、提交、验证结果、剩余风险和下版本入口。

验收：

- `npm test`
- `npm run build`
- `npm run test:e2e:long`
- 手动浏览器走查关键路径：开局、读档、战斗、剧情选择、存档、移动端。

## 每个小版本的固定完成定义

- 代码、测试、文档和上下文交接一起完成。
- 每个阶段提交信息使用中文、具体、可追踪描述。
- 涉及战斗结算必须有单元测试；涉及 UI/动效必须有浏览器或 e2e 验证。
- 推送到 GitHub 后，下一阶段从新的 `codex/v080-*` 分支继续，不在旧收束分支上长期开发。
