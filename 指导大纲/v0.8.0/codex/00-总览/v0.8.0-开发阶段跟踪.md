# RebornG v0.8.0 开发阶段跟踪

更新时间：2026-05-09  
当前分支：`codex/v070-release-closure`  
依据大纲：`指导大纲/v0.8.0/v0.8.0-正史锚点-IF宿命双路线-剧情完整版-设计大纲.md`

## 总目标

v0.8.0 不继续堆系统数量，而是把 RebornG 从“系统可玩”推进到“长线剧情可信 + 战斗有深度 + 前端有游戏感”。核心交付是：正史锚点骨架、IF 宿命双路线、天意与因果压力、剧情候选闸门、结局矩阵解析、蛊虫表现化战斗、GSAP/Motion 全局动效重皮。

玩家仍然是原创角色。方源、龙公、白凝冰、商心慈等原著角色只能作为 canon NPC、远景核心变量或剧情交会对象存在，不能替代玩家，也不能被玩家无代价替代。

v0.8.0 同时接手 v0.7.0 P1/P2 承诺效果分流出来的深系统：夜间/白天修行差异、伏击全链路、完整大境界突破/升仙失败、复杂灾劫后果、高级采集生态。它们不再作为 v0.7.0 发布阻塞，但必须在 v0.8.0 大纲和阶段验收中明确归属。

v0.8.0 还接手 v0.7.1-polish 新建的开局身份路由：`StartProfile` 已保证商家、武家、熊家、铁家、散修等开局不再被写成古月族人；v0.8.0 要把这些身份推进为各出身深线剧情，详见 `codex/02-剧情分支/出身深线剧情矩阵.md`。

v0.8.0 新增一级重点：战斗深化。当前 `combat-engine`、小队战、地形/阵法和战斗闪图已有底座，但玩家操作仍过窄。v0.8 先做凡战深度：让蛊虫和杀招的描述进入可结算、可显示、可被剧情选择调用的表现协议。仙蛊在 v0.8 仅作剧情授权、临时借用、见闻和高压演出；仙蛊屋暂不做运行时战斗系统。

v0.8.0 新增一级重点：全局动效重皮。GSAP 负责战斗/章节级时间轴演出，Motion 负责 UI 状态、布局、选择和棋盘交互。目标不是装饰性动画，而是让状态、代价、反制、风险和剧情用途被玩家看懂。

## 硬边界

- 原著模式不允许改写关键锚点大结果，只允许侧线、关系、资源、局部战果变化。
- IF 模式允许偏移，但必须登记 `IfBranchVector`，写清原因、代价、影响范围和后续反噬。
- 宿命轴只允许 `intact / fractured / destroyed` 三态，由引擎推导，DeepSeek 不得直接写入。
- 天意不是万能失败按钮，只是压力、修正、排斥和叙事约束系统。
- 无极魔尊疯魔窟求证失败；十转、混沌、事实浮冰和永生只能作为未证悬念，不开放“炼成永生蛊”或“真正永生”结局。
- 仙蛊屋 v0.8.0 暂不做玩家可操作战斗系统；可作为剧情见闻、势力威压、拆解线索或后续版本接口。
- 蛊虫/杀招表现不得只停留在文案：可执行效果必须来自本地表现规格或被降级为传闻/线索。

## 阶段拆解

### v0.8.0-pre：剧情地基

状态：进行中，已完成第一轮落地。

交付：
- `canon-anchor-registry.json`：十个正史锚点真相源。
- `fate-branch-matrix.json`：护宿命/毁宿命 IF 轴与偏移规则。
- `heaven-will-rules.json`：天意压力账本规则。
- `ending-outcomes.json`：结局族群与永生禁区。
- `v080-narrative-engine.ts`：锚点校验、IF 候选校验、天意触发、结局解析。
- `context-builder` 与 `state-update-applier` 接入 v0.8 剧情候选协议。
- 承诺效果深化协议：`CultivationTimeProfile`、`AmbushChainState`、`MajorBreakthroughAttempt`、`AscensionFailureOutcome`、`CalamityConsequenceProfile`、`GatheringEcologyNode` 的真相源设计与 DeepSeek 禁写边界。
- v0.7.1-polish 继承接口：`SceneTimeContext`、`ActionTimePolicy`、`SceneLockState`、`PathTendencyProfile`、`CombatIntent`。v0.7.1 已做派生 UI 诚实化，v0.8 需要把它们升级为统一时间/场景协议。
- 出身深线协议：所有 `StartProfile` 映射到 `OriginDeepLineProfile`，明确原著/二创边界、身份压力、专属 NPC、资源入口、敌对关系、锚点参与方式和 IF 偏移。
- 蛊虫表现化战斗协议：
  - `src/canon/gu-expression-specs.json`：首批 53 只蛊虫表现规格。
  - `src/canon/killer-move-expression-specs.json`：首批 15 个杀招表现规格。
  - `src/engine/gu-expression-registry.ts`：读取和剧情用途查询入口。
  - `GuExpressionSpec`、`KillerMoveExpressionSpec`、`BattlefieldCombatState`、`BattleResolutionStep` 类型地基。
- 前端动效规范：
  - 独立 skill：`reborn-combat-motion`。
  - 设计文档：`codex/04-前端动效/GSAP-Motion-全局重皮规范.md`。
  - GSAP/Motion 分工明确，UI 不得私算战斗结果。

验收：
- 正史模式不能直接改宿命战大结果。
- IF 模式可以提出护宿命/毁宿命候选，但必须经引擎校验。
- EndingResolver 不得返回“炼成永生蛊/真正永生”。
- AI 不得直接写境界、升仙结果、灾劫结果、高级资源入库或正式资源点。
- 首批可战斗/可剧情使用蛊必须有 `GuExpressionSpec`，流派名来自 `path-registry.json`。
- 首批杀招必须有核心蛊、辅助蛊、蓄势、失败模式、反噬和视觉节拍。
- 禁忌蛊必须是强场景门槛，不进入普通战斗按钮。

### v0.8.0-a：前中期正史锚点

范围：
- 青茅山、商家城、三王山、王庭四段剧情骨架。
- 原著模式侧线参与，IF 模式局部偏移。
- 人物图鉴、舆图传闻、NPC 对话行动卡与剧情候选联动。
- 凡战深化第一轮：5×3 轻地图棋盘、蛊虫行动栏、杀招行动栏、观察/撤退、地形/阵位/危险格展示。

重点：
- 不把玩家写成方源。
- 不让 DeepSeek 直接创建正式任务、奖励或地图资源点。
- 侧线选择要回流到道心、声望、关系、经济和后续锚点压力。
- 昼夜修行、区域采集、侦察/陷阱/伏击前半链必须开始服务前中期地图生态。
- 行动-时间-场景协议必须解决 v0.7.1 暂时只解释未重写的问题：同一时段 AP、多动作、场景锁、剧情未结算时禁止推进无关行动。
- 高级蛊材和仙材仍受经济闸门限制，不能通过低成本 AP 稳定刷出。
- 出身深线开始落地：青茅山本地三寨、商队/商家外围、散修前中期体验必须能跑出不同出生地、身份压力和资源入口。
- 蛊虫描述要进入剧情选择：侦察、追踪、治疗、解毒、破障、遮掩、威慑等 `sceneUtilities` 必须可被上下文识别。
- 战斗 UI 初版必须展示“能用/不能用/风险是什么”，不能只给漂亮按钮。

### v0.8.0-b：中后期大锚点

范围：
- 义天山、逆流河、梦境/影宗、宿命战。
- 正式接入 `fateState`、`HeavenWillLedger`、`KarmicDebtLedger`。
- IF 双路线开始显性分叉：护宿命与毁宿命。

重点：
- 宿命战是 v0.8 的最大剧情闸门。
- 原著模式只允许旁观、侧翼战场、救援、撤退、关系变化。
- IF 模式允许站队，但必须付出资源、关系、天意压力和势力代价。
- 完整大境界突破、升仙失败、伏击进入战斗/剧情候选和复杂灾劫后果在本阶段形成可玩闭环。
- 升仙失败按原著口径处理身体、空窍/仙窍、蛊虫、道痕和天意压力，不做普通 RPG 掉级。
- 商家城、武家、铁家、北原/东海/中洲跨域出身深线进入宿命前夜，影响阵营站队、正魔关系、突破资源和剧情候选。
- 小队群战、伏击链路、阵法/破阵接入表现化战斗协议。

### v0.8.0-c：尊者博弈与末章解析

范围：
- 尊者博弈、天庭末章、玩家势力收束、疯魔求证未证线。
- 接入结局解析器 UI 与死亡/通关总结。

重点：
- 尊者不能降智，玩家势力可以成为变量，但不能无代价替代尊者。
- 疯魔窟、混沌、事实浮冰只给“真相残片/未证悬念/代价极重”的体验。
- 结局必须标注 `canon-near | if-derived | original-if`。
- 高级采集生态、灾劫伤痕、突破失败记录、因果债与天意压力进入结局解析器。
- 战斗记录、杀招反噬、禁忌蛊代价和群战伤亡进入结局解析输入。
- 仍不得开放“真正永生”“炼成永生蛊”“玩家稳定十转”。
- 出身债务进入结局解析器：家族残债、商路利益、正道执法、天庭秩序、散修无庇护等长期压力影响结局权重。

## 当前阻塞与风险

- v0.8.0 还没有真正实现各锚点的剧情 UI 与章节浏览器长测。
- `story_event_candidates` 和 `if_branch_candidates` 已有协议地基，但正式任务系统、地图系统、NPC 关系系统仍需逐段接入。
- 原著模式与 IF 模式的体验差异要谨慎控制：原著模式不能显得玩家无事可做，IF 模式不能变成随意爽文改写。
- DeepSeek 长上下文容易忘记玩家原创身份和宿命边界，必须持续注入锚点/宿命/永生禁区。
- DeepSeek 长上下文也容易把外来出身拉回章节默认古月身份；v0.8 必须继续以 `StartProfile` 和出身深线矩阵作为身份最高优先级。
- 从 v0.7.0 继承的承诺效果深系统容易被误当成“几个 modifier 没接完”。后续实现必须以 `codex/01-系统深化/v0.7承诺效果继承深化矩阵.md` 为准，先做真相源和引擎校验，再做 UI。

## v0.7.1-polish 新增继承项

更新时间：2026-05-09

v0.7.1 已完成战斗遭遇循环、升炼/拆炼预览、本命蛊解释、物资袋超载、商会卖蛊材和移动端窄屏发布修复。下列内容不再回塞 v0.7.1，进入 v0.8 正式阶段：

- `SceneTimeContext`：统一剧情、行动、战斗、对话、商会、AP、时段和场景锁。普通狩猎、采集、蛊兽、敌对蛊师、伏击、挑战必须有清晰触发协议。
- 本命蛊深系统：选择、成长、收益、反噬、突破、升仙、灾劫、道途和结局权重。不得把本命蛊做成普通装备槽。

- `v0.8.0-a`：青茅山、商队、商家城前中期行动/狩猎/采集/本命蛊基础体验。
- `v0.8.0-b`：伏击全链路、突破/升仙失败、本命蛊受损、高级资源流通。
