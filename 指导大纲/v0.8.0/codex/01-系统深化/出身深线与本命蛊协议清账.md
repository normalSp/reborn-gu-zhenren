# v0.8.0-c1.2 出身深线与本命蛊协议清账

更新时间：2026-05-11
执行分支：`codex/v080-c12-origin-lifebound-closure`

## 目标

c1.2 的目标是把 c2 前仍容易显示成“待系统”的三类承诺清账为可读、可测、可降级的系统事实：

- 所有 `StartProfile` 映射到 `OriginDeepLineProfile`，出身不再只是开局文案或数值标签。
- 本命蛊映射到 `LifeboundGuGrowthProfile`，普通出售、移除、拆炼、升炼不能绕过本命协议。
- 青茅山、商家城、三王山、王庭等前中期路线显式映射到 canon anchor，为 c2 的解释型 UI 提供事实来源。

## 真相源

- `src/canon/v080-origin-deep-line-profiles.json`
  - 覆盖 24 个 `StartProfile`。
  - 记录身份边界、初始压力、长期压力、关键 NPC、友好/敌对势力、资源入口、正史锚点参与权、IF 偏移代价和终局债务标签。
- `src/canon/v080-front-mid-anchor-map.json`
  - 显式登记 `qingmao_mountain`、`shang_clan_city`、`san_wang_mountain`、`northern_plains_wangting`。
  - 每个映射写清正史边界、IF 边界、允许角色、禁止改写和交接系统。
- `src/canon/v080-lifebound-gu-growth-profiles.json`
  - 登记本命蛊基线、力道、炼道、月道、血道禁忌、智/信道、身法、疗护等首批成长协议。
  - 普通 `remove/sell/refine/disassemble/ascend` 均为 block，`feed/use` 允许但必须保留成长与反噬语义。

## 运行时协议

- `v080-origin-lifebound-closure.ts` 是 c1.2 的唯一读取与校验入口。
- `context-builder` 注入当前出身深线、前中期锚点边界、本命蛊成长协议和禁止越权规则。
- `narrative-consistency` 先使用 `OriginDeepLineProfile` 降级跨出身身份文本，再保留旧的 start profile 兜底拦截。
- `guSlice.removeGu` 通过 `validateLifeboundGuOperation` 阻止普通背包移除本命蛊；出售按钮最终也无法绕过该保护。
- `v080-ending-framework-engine` 把出身债务、本命蛊成长协议、风险标签和终局权重写入 `EndingEvidenceSummary`。

## 边界

- c1.2 不提升 `SAVE_FORMAT_VERSION`，因为新增的是 truth source、校验、上下文和可选证据字段，不引入新的长期玩家状态。
- 本命蛊成长协议不在 c1.2 实现完整七至九转长线伤损，只保证不会被普通物品逻辑误处理。
- 前中期锚点映射不等于完整 A1-A4 剧情内容，只提供系统识别边界、禁止改写和后续入口。
- DeepSeek 仍只能写候选、压力、传闻和叙事表现；出身、锚点、终局证据和本命蛊保护由本地协议决定。

## 验收

- 所有 `StartProfile` 恰好被一个 `OriginDeepLineProfile` 覆盖。
- 所有前中期 anchor id 来自 `canon-anchor-registry.json`。
- 跨出身身份越权会被降级，例如商家线被写成古月族人时改为“商路蛊师”。
- 本命蛊不能被普通 `removeGu` 移除，也不能通过出售间接移除。
- 终局输入可以读取出身深线、本命蛊成长协议、风险和权重证据。
