# v0.8.0-c2.5 传承与待认主福地/洞天竖切设计

日期：2026-05-11
分支：`codex/v080-c25-inheritance-land-claim`
存档协议：`SAVE_FORMAT_VERSION = 20`

## 目标

c2.5 的目标是把“传承、地灵条件、待认主福地、洞天边界”接入 v0.8 已经形成的剧情闭环，而不是做一个独立刷奖励按钮。

核心闭环：

1. 剧情或本地样板登记传承/福地候选。
2. 本地引擎校验境界、地点、锚点、IF 边界、出身、场景 AP、战斗/灾劫风险和持有线索。
3. 合法候选进入试炼、守护战、资源压力或地灵执念条款。
4. 试炼/认主结果由本地引擎结算，写入 `inheritanceLandState`、场景 AP 账本和必要的福地状态。
5. 下一轮剧情只承接本地结算事实，DeepSeek 不改写奖励或归属。

## 首批样板

- 小传承洞府：入门级传承，允许产出已登记凡蛊、材料、配方碎片、杀招碎片。
- 三王传承旁支：只做旁支争夺、试炼和线索，不改写三王传承正史核心结果。
- 待认主福地：要求六转以上、场景 AP、地灵执念条件、试炼/守护/资源压力通过后才能认主。
- 洞天边界传闻：v0.8 只作为高压传闻、禁区边界和后续入口，不开放正式认主、吞并或洞天资源节点。

## 数据与引擎

真相源：`src/canon/v080-inheritance-land-rules.json`

本地引擎：`src/engine/v080-inheritance-land-engine.ts`

公开 API：
- `createDefaultInheritanceLandState`
- `normalizeInheritanceLandState`
- `evaluateInheritanceEntry`
- `stageInheritanceCandidate`
- `resolveInheritanceTrialAction`
- `evaluateLandClaimEntry`
- `resolveLandClaimAttempt`
- `formatInheritanceContextForPrompt`

规则：
- 所有随机结算使用 seeded RNG，禁止 `Math.random`。
- 任何未知奖励、未知仙蛊、未知材料、洞天认主、十转、永生蛊、真正永生、玩家获得宿命蛊都必须降级为传闻或拦截记录。
- 引擎只允许登记规则源中的 siteId；运行时不能发明正式传承地点。
- 福地认主只生成 `福地` 级归属；洞天只能进入传闻/禁区状态。

## Store 与 Pipeline

新增持久状态：`inheritanceLandState`

新增 slice：`src/store/slices/inheritanceLandSlice.ts`

动作：
- `recordInheritanceCandidateAction`
- `startInheritanceTrialAction`
- `resolveInheritanceTrialAction`
- `attemptLandClaimAction`
- `dismissInheritanceCandidateAction`
- `clearExpiredInheritanceCandidatesAction`

Pipeline：
- `state_update.inheritance_land_candidates.add` 只允许 AI 提交候选。
- `state-update-applier` 调用本地 action 登记候选；不合法候选写入降级日志或拦截记录。
- `context-builder` 注入可进入传承、待认主条件、洞天禁区、风险、AP 成本和禁止越权规则。
- 传承试炼和福地认主消耗 c2.2 场景 AP，写入 `LocalActionLedgerEntry`，供下一轮叙事承接。

## UI

新增底部面板：`传承`

展示内容：
- 四类样板入口。
- 当前候选与状态。
- 进入条件和不可进入原因。
- 奖励预览和奖励来源。
- 地灵执念条款。
- 待认主/已认主福地摘要。
- 洞天传闻和禁区拦截。
- 本地结算轨迹。

扩展：
- `ChoicePanel` 增加 `传承线索 / 待认主福地 / 洞天传闻 / 禁区拦截` 标签。
- `AperturePanel` 补充待认主/已认主福地摘要，六转以上优先展示仙窍、福地、道痕密度、资源节点和灾劫压力。
- `ActionPanel` 六转及以上不得显示“五转升仙 / 升仙三气 / 尝试升仙”。
- 移动端抽屉为底部导航预留点击区，长面板不遮挡工具带。

## 边界

- c2.5 是有限竖切，不做完整原著所有传承。
- c2.5 不做所有福地/洞天认主、吞并洞天、仙蛊屋或大型战争。
- 福地可以做首个可认主样板；洞天只作为传闻、禁区和高阶入口。
- 不开放十转、永生蛊、真正永生、玩家获得宿命蛊。
- 传承和福地必须依赖剧情场景、AP、战斗/灾劫/锚点系统闭环，不能成为独立刷奖励按钮。

## 验收

- 所有传承 siteId、anchorId、流派、奖励类型来自已登记 canon/registry。
- 小传承可进入、可失败、可结算已登记奖励。
- 三王传承旁支允许争夺旁支，不允许改写核心正史。
- 六转以上可尝试待认主福地；条件不足时输出明确失败原因。
- 洞天候选只能生成传闻/禁区压力，不生成归属或资源节点。
- AI 直接写奖励、福地归属、洞天归属、仙蛊、关键 NPC 生死会被阻断或降级。
- v19 存档迁移到 v20 后有完整 `inheritanceLandState`。
- 桌面和移动端 reduced motion 下 `传承` 面板、Choice 标签、tooltip、空窍/仙窍摘要可读可点。
