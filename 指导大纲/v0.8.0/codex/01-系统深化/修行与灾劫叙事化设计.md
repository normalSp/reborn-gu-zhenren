# v0.8.0-c2.4 修行与灾劫叙事化设计

更新时间：2026-05-11
分支：`codex/v080-c24-cultivation-calamity-narrative`

## 目标

c2.4 解决一个体验断层：修行、升仙、灾劫不能只在行动面板里独立扣 AP 或扣资源，而要进入当前剧情场景，成为下一轮叙事可承接的本地事实。

本阶段不新增存档协议，不替代 b2 的正式灾劫后果结算；它在 b2 引擎和 c2.2 场景 AP 预算之间补一层“灾劫场景规格”：

- 凡人阶段：空窍、真元、窍壁压力、突破。
- 五转阶段：升仙三气、升仙准备、是否可升仙。
- 六转以上：仙窍/福地、仙元、道痕密度、灾劫预兆。
- 七转/八转档不得再出现“五转升仙”按钮或五转说明。

## 本地协议

新增 `CalamitySceneSpec`，由本地引擎从 `CalamityPreview` 与当前 store 事实生成：

- `kind`：灾劫场景类型。
- `sceneId`：当前剧情场景。
- `realmGrand`：当前大境界。
- `affectedResourceNodeIds`：受影响资源节点。
- `allowedResponses`：观察、阵位、资源防护、接战、牺牲蛊虫、修补仙窍、退守仙窍。
- `possibleConsequences`：后续可能由本地引擎结算的损伤或收益。
- `combatScale`：必要时桥接 c2.3 剧情战斗候选。

灾劫场景类型首批覆盖：

- 自然灾害类。
- 荒兽/荒植入侵类。
- 人劫/敌袭类。
- 道痕化形类。
- 类仙道杀招压迫类。
- 资源节点失衡类。

## 接入边界

`stageCalamityScene()` 只做四件事：

1. 校验当前是否有可进入剧情的灾劫预兆。
2. 消耗 c2.2 场景 AP。
3. 写入 `pendingCalamitySceneSpec` 与 `calamity_warning` 轨迹。
4. 如果需要接战，生成 c2.3 `combatEventCandidates`。

它不会直接扣面积、资源、道痕、HP、蛊虫状态或终局证据。正式后果仍走：

- b2 `resolveCalamityConsequence`。
- c2.3 剧情战斗编排。
- 后续资源防护、阵位、牺牲蛊虫、修补仙窍等本地 action。

DeepSeek 只能承接预兆、压迫、选择描述和场景气氛，不能改写数值后果。

## UI 要求

`ActionPanel` 保留快捷入口，但调息/修行细节长期建议迁移到空窍/仙窍面板：

- 凡人：展示修行进度、窍壁压力、突破校验。
- 五转：展示升仙三气和升仙准备。
- 蛊仙：展示福地/仙窍、道痕密度、灾劫倒计时和灾劫入场。

按钮规则：

- `尝试升仙` 只在五转阶段显示。
- `灾劫入场` 只在六转以上且存在灾劫预览时显示。
- 六转以上不得出现“五转阶段”“升仙三气”作为主行动说明。

## 验收

- 七转演示档打开行动面板时无五转升仙按钮。
- 点击灾劫入场后，场景 AP -1，`cultivationState.calamityLedger` 不增加，`pendingCalamitySceneSpec` 写入，轨迹首条为 `calamity_warning`。
- 下一轮 prompt 注入灾劫场景规格，明确 DeepSeek 不得结算正式后果。
- 桌面与移动端 reduced motion 下，灾劫类型、倒计时、资源点、可处理方向和轨迹可读。
