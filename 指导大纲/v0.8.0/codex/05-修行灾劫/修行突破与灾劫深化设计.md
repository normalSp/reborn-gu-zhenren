# 修行突破与灾劫深化设计

版本：v0.8.0-b2  
状态：已落地核心竖切  
规则源：`src/canon/v080-cultivation-calamity-rules.json`

## 目标

b2 把修行从“按钮加进度”推进为可持久、可回放、可被剧情压力引用的本地系统。它覆盖一至五转修行、大境界突破、五转巅峰升仙和六转福地灾劫；七至九转只保留后续钩子。

核心原则：

- 境界、突破、升仙、灾劫、资源损益必须由本地引擎结算。
- DeepSeek 只能写预兆、传闻、候选和叙事压力，不能直接写正式结果。
- 六转升仙只生成 `福地`，不在 b2 生成 `洞天`。
- 所有可复现结算统一使用 seeded RNG，禁止 `Math.random`。

## 持久状态

`SAVE_FORMAT_VERSION` 从 `15` 升为 `16`。新增 `state.cultivationState`，由 `createDefaultCultivationState` 和 `normalizeCultivationState` 兜底。

核心字段：

- `progress`：当前大阶段修行进度。
- `apertureWallPressure`：空窍/窍壁压力。
- `ascensionPreparation`：升仙准备、三气平衡、天意压力、业债。
- `breakthroughHistory`：大境界突破记录。
- `ascensionHistory`：升仙记录。
- `calamityLedger`：灾劫账本。
- `lastResolutionSteps`：最近一次本地结算轨迹，供 UI 播放。

迁移要求：

- v15 旧档迁移到 v16 时必须生成默认 `cultivationState`。
- `测试存档/v0.7.0` 全量更新为 `formatVersion = 16`。
- 新增 b2 专项存档：五转巅峰升仙准备、升仙失败压力、六转福地灾劫。

## 本地引擎 API

运行时入口：`src/engine/v080-cultivation-calamity-engine.ts`

公开 API 固定为：

- `createDefaultCultivationState`
- `normalizeCultivationState`
- `buildCultivationEnvironmentProfile`
- `resolveCultivationSession`
- `validateMajorBreakthroughAttempt`
- `resolveMajorBreakthroughAttempt`
- `validateAscensionAttempt`
- `resolveAscensionAttempt`
- `buildCalamityPreview`
- `resolveCalamityConsequence`

引擎只输出事实和变更建议，不直接操作 zustand store。正式写入由 `cultivationSlice` 完成，避免 UI 私算。

## 结算模型

修行 session 读取：

- 时段：白日、黄昏、夜间等。
- 地点：宗门静室、洞府、商路、野外、福地。
- 安全度：影响收益、遭遇风险和反噬概率。
- 体质：十绝体提高进度上限和压力，也提高失控风险。
- 真元/仙元与 AP：不足时本地阻断。

大境界突破读取：

- 当前境界与下一境界门槛。
- 修行进度、窍壁压力、资质、极端体质、环境安全。
- 失败时写入 HP/真元压力、业债或窍壁损伤，不做简单掉级。

升仙读取：

- 五转巅峰门槛。
- 天地人三气平衡。
- 天意压力、业债、极端体质风险。
- 成功生成六转福地，写入 `heavenlyLand` 与 `aperture`；失败写入压力和伤痕。

灾劫读取：

- 福地等级、道痕、资源节点、天意压力和上次灾劫记录。
- 结算资源节点损伤、道痕变化、蛊虫损伤、业债变化和灾劫账本。

## AI 安全边界

`state-update-applier` 是最后一道门：

- AI 直接写 `state_update.player.realm` 跨大境界时，降级为叙事线索或 warning。
- AI 给低阶角色写 `essenceType: "immortal"` 时阻断。
- AI 不得初始化仙窍、福地、灾劫后果或资源损益。
- 合法改变必须由 `practiceCultivationDeep`、`attemptMajorBreakthrough`、`attemptAscension`、`resolveApertureCalamity` 等本地 action 执行。

## UI

UI 入口：`src/components/game/ActionPanel.tsx`

展示内容：

- 当前时段、地点、安全度。
- 修行进度、窍壁/福地压力。
- 突破成功率与失败后果。
- 升仙三气与天意压力。
- 灾劫预览、资源/道痕/蛊虫风险。
- `CultivationResolutionStep[]` 本地结算轨迹。

动效分工：

- Motion：卡片、tooltip、进度条、轨迹列表、低动效 fallback。
- GSAP：突破、升仙、灾劫结果的独立演出层；不控制 Motion 管辖的文字、按钮和进度条。

## 测试

必须覆盖：

- 规则数据的流派、灾劫类型、后果标签合法性。
- 修行收益、压力、突破失败、升仙成功/失败、福地生成、灾劫损伤。
- `Math.random` spy 抛错时仍可通过 seeded RNG 复现。
- v15 到 v16 迁移、测试存档 v16 完整性。
- AI 直接写境界、升仙、灾劫、资源损益会被阻断或降级。
- 桌面、移动端和 reduced motion 的修行面板验收。
