## 源码落地

### canon 数据

- `src/canon/canon-anchor-registry.json`
  - 登记十个剧情锚点：青茅山、商家城、三王山、王庭、义天山、逆流河、梦境/影宗、宿命战、尊者博弈、天庭末章。
  - 每个锚点包含正史状态、玩家角色、关键 NPC、关键势力、正史禁改项、正史可变项、IF 偏移轴、后续影响。
- `src/canon/fate-branch-matrix.json`
  - 登记 `fateState = intact | fractured | destroyed`。
  - 登记护宿命/毁宿命两条路线与 IF 偏移轴。
- `src/canon/heaven-will-rules.json`
  - 登记天意注意、修正、排斥、模糊度账本。
  - 明确天意不是万能失败按钮。
- `src/canon/ending-outcomes.json`
  - 登记结局族群：原著近线、宿命永存、宿命裂痕乱世、宿命破碎争渡、玩家势力立足、疯魔求证未证、身死道消。
  - 明确禁止“炼成永生蛊/真正永生/稳定十转”等断言。

### 类型与 schema

- `src/types/index.ts`
  - 新增 `TimelineMode`、`FateState`、`CanonAnchor`、`IfBranchVector`、`HeavenWillLedger`、`KarmicDebtLedger`、`StoryEventCandidate`、`IfBranchCandidate`、`CanonAnchorPressure`、`EndingResolverInput`、`EndingOutcome`。
  - `StateUpdate` 新增 `story_event_candidates`、`if_branch_candidates`、`canon_anchor_pressure`。
- `src/schemas/narrative.schema.ts`
  - 显式接受 v0.8 三类候选字段。
  - DeepSeek 可以提交候选，但不会因此获得改写宿命、锚点或结局的权限。

### engine 与 pipeline

- `src/engine/v080-narrative-engine.ts`
  - `validateAnchorMutation`：校验正史锚点是否可改。
  - `validateIfBranchCandidate`：校验 IF 候选只能在 IF 模式生效。
  - `applyHeavenWillTrigger`：结算天意压力账本。
  - `resolveEnding`：根据宿命状态、IF 向量、天意、因果债、势力和道心解析结局。
  - `validateEndingText`：扫描禁止永生/十转断言。
- `src/engine/state-update-applier.ts`
  - `story_event_candidates.add` 只登记到 `flags.storyEventCandidates`。
  - `if_branch_candidates.add` 在正史模式拦截，在 IF 模式校验后写入 `flags.ifBranchVectors`。
  - `canon_anchor_pressure.add` 只登记压力日志，不直接改剧情结果。
- `src/engine/context-builder.ts`
  - 注入 v0.8 剧情锚点与宿命闸门。
  - 明确 DeepSeek 禁止直接改 `fateState`、关键 NPC 生死、锚点结果、正式结局、永生/十转定论。

## 下一步建议

1. v0.8.0-a：实现青茅山、商家城、三王山、王庭四个前中期锚点的剧情数据与浏览器长测。
2. v0.8.0-a 需要先做章节路由与锚点映射：当前 `currentChapterId` 未必等于 v0.8 的 `CanonAnchor.id`，建议增加显式 `currentCanonAnchorId` 计算或映射表。
3. v0.8.0-b：宿命战前必须先完成 `HeavenWillLedger` 与 `KarmicDebtLedger` 的运行时 UI 展示，否则玩家看不懂 IF 代价。
4. v0.8.0-c：结局解析器必须先出预览/调试面板，再做正式通关表现。

## 风险提示

- 原著模式要避免“玩家只能旁观”的无力感，需要用侧线、关系、资源和局部战斗给足可玩性。
- IF 模式要避免爽文式改写，需要每次偏移都有代价、天意压力、因果债和后续反噬。
- DeepSeek 仍可能口头宣告“玩家成尊/永生/替代方源”，必须继续依赖 context-builder 注入和 state-update 语义闸门。
