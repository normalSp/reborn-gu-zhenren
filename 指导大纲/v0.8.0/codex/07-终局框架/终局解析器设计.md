# v0.8.0-c1 终局解析器设计

更新时间：2026-05-11  
执行分支：`codex/v080-c1-ending-framework`

## 目标

c1 建立“终局框架”，不是一次性做完所有终局剧情。它把 b2 修行/灾劫、b3 中后期锚点、IF 向量、天意账本、因果债、战斗与群像战代价、出身债务和玩家势力收束为可追踪的本地终局候选。

终局候选必须明确标注来源：`canon-near`、`if-derived` 或 `original-if`。v0.8.0-c1 允许玩家势力成为变量，但不允许玩家无代价替代尊者，不允许普通战斗击杀尊者，不开放永生蛊、十转或真正永生结局。

## 持久化

c1 正式提升 `SAVE_FORMAT_VERSION` 到 `18`，新增 `state.endingState`。

`endingState` 持有：
- `status`：`idle / ready / committed / blocked`
- `lastInput`：最近一次由本地引擎汇总的终局输入
- `candidates`：本地生成的终局候选
- `pressureLog`：DeepSeek 或剧情文本尝试越过终局边界时的压力记录
- `lastResolutionSteps`：UI 可播放的终局轨迹
- `commitRecord`：正式提交后的终局记录

v17 旧档迁移时会补默认 `endingState`。测试存档目录 `测试存档/v0.7.0` 已全量升级为 `formatVersion = 18`，并新增 c1 专项档：尊者博弈压力、天庭末章入口、宿命破碎候选、势力立足候选、身死道消总结。

## 本地引擎

真相源：
- `src/canon/v080-ending-framework-rules.json`
- `src/canon/ending-outcomes.json`
- `src/canon/canon-anchor-registry.json`
- `src/canon/fate-branch-matrix.json`

引擎入口：
- `createDefaultEndingFrameworkState`
- `normalizeEndingFrameworkState`
- `buildEndingResolutionInput`
- `evaluateEndingReadiness`
- `generateEndingRouteCandidates`
- `validateEndingRouteCandidate`
- `resolveEndingRoute`
- `commitEndingOutcome`
- `recordEndingPressure`

所有候选、门槛、禁区和提交结果都由本地引擎决定。DeepSeek 只能提供候选、压力和叙事表达，不能直接写 `endingState`、`endingOutcome`、`finalOutcome`、尊者击杀、十转或永生结论。

## 输入证据

c1 终局输入包括：
- b3 `storyAnchorState`：宿命状态、锚点结果、IF 向量、天意账本、因果债
- b2 `cultivationState`：突破成败、升仙结果、灾劫伤痕、天意压力、业债
- 战斗/群像战统计：战斗次数、胜场、小队伤亡、援护与逃脱等
- 势力和关系：玩家势力分、关系数量、势力事件数量
- 出身债务：商路利益、正道门第、执法身份、散修无庇护等长期压力
- 本命蛊代价：本命蛊受损或反噬会降低终局稳定度

证据只用于加权候选和解释，不允许 UI 私算结局。

## 候选家族

首批终局家族来自 `ending-outcomes.json`：
- `canon_near`
- `fate_intact_order`
- `fate_fractured_chaos`
- `fate_destroyed_struggle`
- `player_faction_foothold`
- `crazed_demon_unproven`
- `death_and_dust`

`crazed_demon_unproven` 必须保持“未证”边界，只能给真相残片、混沌压力和未证悬念，不能写成永生成功。

## Store 与 UI

`endingSlice` 提供：
- `previewEndingFramework`
- `refreshEndingCandidatesAction`
- `commitEndingCandidateAction`
- `recordEndingPressureAction`

底部面板新增 `终局` 入口，`EndingResolverPanel` 展示：
- 当前状态
- 候选路线
- 证据摘要
- 禁区/压力日志
- 本地解析轨迹
- 正式提交按钮

提交成功后进入 `GameOverScreen` 的终局总结页，复用死亡记录结构但标注 `endingFamilyId`、`endingProvenance`、终局理由和未证警示。终局总结不播放死亡音效，导出文件使用 `ending-record-*`。

## 动效边界

Motion 负责终局面板卡片、候选列表、证据条、tooltip、按钮和 reduced-motion fallback。  
GSAP 只保留给重大终局压力、宿命状态变动、章节级终局转场等独立 effect layer，不控制候选卡、按钮、文本或列表。

## 安全边界

必须阻断或降级：
- 玩家获得宿命蛊
- 炼成永生蛊
- 真正永生
- 稳定十转
- 普通战斗击杀尊者
- 尊者集体降智
- DeepSeek 直接写 `endingState`、`endingOutcome`、`finalOutcome`

允许：
- 侧翼战场
- 局部筹码
- 尊者注意或压力
- 势力站队
- IF 向量偏移
- 混沌/疯魔求证的未证悬念
