## 一、设计结论

v0.7.1 解决的是发布级错误：商家、武家、熊家、铁家、散修等开局不得被 DeepSeek 或章节默认位置写成古月族人。  
v0.8.0 要解决的是深线体验：每个出身都应有身份压力、专属 NPC/势力关系、资源入口、敌对关系、正史锚点参与方式和 IF 偏移代价。

硬边界：

- 玩家始终是原创角色，不能成为方源、商心慈、白凝冰、黑楼兰、武庸等原著人物。
- 原著有明确描述的地方按原著靠近；原著少写的出身按 `derived | original` 标注二创边界。
- 出身不能绕过资产闸门。蛊虫、蛊方、仙蛊、材料、杀招传承必须仍来自登记真相源。
- DeepSeek 只能提出候选剧情，不能直接改出身、声望、关键 NPC 生死、正式任务或资源点。

## 二、数据协议

v0.8.0 的出身深线以 `StartProfile` 为入口，并扩展为 `OriginDeepLineProfile`。

```ts
interface OriginDeepLineProfile {
  originId: string;
  startProfileIds: string[];
  provenance: 'canon-near' | 'derived' | 'original' | 'unknown';
  loreBoundary: string;
  initialIdentityPressure: string[];
  longTermIdentityPressure: string[];
  keyNpcs: string[];
  friendlyFactions: string[];
  hostileFactions: string[];
  resourceEntrances: string[];
  canonAnchorAccess: string[];
  ifDeviationAxes: string[];
  heavenWillAndKarmaHooks: string[];
  releaseAcceptance: string[];
}
```

## 四、阶段拆解

### v0.8.0-pre

- 建立 `OriginDeepLineProfile` 真相源。
- 每个 `StartProfile` 必须映射到一个深线 profile。
- context-builder 注入“当前出身深线边界”：身份、可见 NPC、禁写项、可提出候选。
- DeepSeek 若提出跨出身越权，如“商家学徒突然成为古月族学弟子”，进入修复或降级。

### v0.8.0-a

- 青茅山三寨、商家商队、散修前中期可玩。
- 商家/散修不再强行落入古月族学；可在古月山寨附近、商路、山外营地形成合理交会。
- 初始声望、图鉴 contact、舆图传闻、商会折扣、黑市入口与出身关联。

### v0.8.0-b

- 商家城、武家、铁家、北原/东海/中洲跨域深线开始成型。
- 出身影响突破资源、升仙准备、伏击链路、正魔阵营站队、宿命战前的立场。
- 武家不再与熊家混用力道设定；熊力蛊保留给熊家寨或熊家相关线。

### v0.8.0-c

- 出身进入尊者博弈与结局矩阵。
- 家族、商队、宗门、散修、天庭等身份债务影响 `KarmicDebtLedger`、`HeavenWillLedger` 和 `EndingResolver`。
- IF 模式下，护宿命/毁宿命的代价会按出身不同表现：天庭线自由受限更重，散修线资源断供更重，商家线利益背叛更敏感。

### v0.8.0-release

- 全出身浏览器长测：至少覆盖南疆一转的古月、熊家、白家、商家、武家、铁家、散修。
- 验收身份稳定：首段剧情、行动地点、声望、人物图鉴、上下文 prompt 均不得串线。
- 验收奖励稳定：保底蛊不越阶，未知蛊只进线索，深线奖励不绕过经济闸门。

## 五、v0.7.1 到 v0.8 的交接清单

- `src/canon/start-profiles.json` 是 v0.8 出身深线入口，不另建一套冲突入口。
- `src/engine/start-profile.ts` 的覆盖测试必须继续保持通过。
- `context-builder` 已将 start profile 作为高优先级身份锚点；v0.8 只扩展，不绕开。
- `narrative-consistency` 已拦截非古月出身被写成古月族人的文本；v0.8 应把拦截从“文案替换”升级为“候选重试 + 线索降级 + 诊断日志”。
