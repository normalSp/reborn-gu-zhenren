# RebornG v0.7.1 项目介绍

## 一句话定位

RebornG 是一款基于《蛊真人》世界观的非盈利二创互动文字游戏。玩家不是方源，而是一个原创参与者，在原著正史锚点、二创支线与 IF 分支之间求生、经营、修行、战斗并承担选择后果。

## 视频开场建议

大家好，这里是 RebornG。它不是单纯复述原著剧情，也不是让玩家替代方源，而是把玩家放进五域蛊界中，作为一个原创角色参与青茅山、商家城、三王山、王庭、逆流河、宿命战等关键节点。我们希望做的是一款有原著味、能长期演化、又能被规则托住的 AI 互动文字游戏。

## 项目边界

- 非盈利二创项目，面向个人游玩、开发展示和社区交流。
- 原著 NPC 保持独立身份，方源只能作为原著 NPC、远景核心变量或剧情交会对象存在。
- 正史模式中，原著锚点不会被玩家随意改写；IF 模式中，锚点可以偏移，但必须记录原因、代价、影响范围和后续反噬。
- DeepSeek 负责叙事、氛围和候选机会；真正的材料、蛊虫、任务、地图、战斗伤害、奖励和关键剧情状态都由引擎与真相源校验。

## v0.7.1 主要完成度

### 可信前置

- 重建 v0.7.0-pre 文档体系，建立数值、世界观、经济、二创 provenance 和 AI 幻觉闸门。
- 整理本地原著资料索引与 `reverend-insanity-lore` 口径，避免运行时自创流派、乱写宝黄天或把玩家写成方源。
- 建立 `MaterialRegistry`、`RecipeRegistry`、`PathRegistry`、`GuUseRegistry`、音频 manifest、角色曲 manifest 等真相源。

### 经济与炼养用

- 蛊材、仙材、蛊虫、仙蛊、蛊方、残方、杀招传承、食料、商会、宝黄天进入统一经济闭环审计。
- 炼蛊、升炼、拆炼不再只看 UI 文案，材料缺口、来源、成功率、失败后果和回收预览都回指真实公式。
- 喂养不再只扣元石，食料来源、缺口、节流折扣、非物质食料信用等进入可解释路径。
- 宝黄天改用候选价表，凡人无法交易，蛊仙期购买力进入 20/100/300 回合模拟。

### 战斗、小队与势力

- 单人战斗完成确定性纵切：境界压制、流派克制、道痕共鸣、杀招结算、仙蛊门槛、奖励闸门和 `BattleTrace`。
- 小队战复用同一套 `combat-config.json`、`DuelMove`、`BattleTrace` 和奖励闸门，不另开一套膨胀公式。
- 小队编成、成员信任、低信任高利益招募、外派任务、战斗回流和势力五账本收入已经有发布级入口。
- 战斗事件仍允许 AI 参与，但 AI 只能提交候选，例如第三方发现、地形变化、追击或谈判，不能直接写伤害或奖励。

### 叙事与人物

- 开局身份路由修复：古月、白家、熊家、商家、武家、铁家、散修等出身不会再串成古月族人。
- 人物图鉴支持关系 NPC、动态 NPC 和已闻/已见 contact；NPC 对话支持行动卡，但任务、交易、奖励仍需引擎确认。
- 道心、声望、人物关系、传闻地点和事件日志进入更清晰的回流体系。
- 死亡记录支持一生总结与原创定场诗兜底。

### 音频与演出

- 用户提供的角色曲已整理进私用二创包 manifest：方源、龙公、白凝冰、商心慈、黑楼兰、盗天、红莲、幽魂、薄青、古月药乐、凤九歌、吴帅、太白云生、墨瑶、马鸿运等。
- 同一角色多曲时使用回合和 seed 做可复现轮换，避免永远固定第一首。
- 战斗命中、暴击、UI、成就、警示、事件等 SFX 已纳入 manifest；真实免费素材后续可继续替换强化。
- 杀招与重要仙蛊支持闪图 manifest，没有美术时使用流派色和文字 fallback。

### 测试与发布可信

- 新增中文测试存档，覆盖七出身开局、单人战斗、小队战、闪图、角色曲、宝黄天、外派、地形、十绝体、移动端 smoke 等。
- 旧英文测试存档继续保留，保证 e2e 兼容。
- 核心回归目标为 `npm test`、`npm run build`、`npm run test:e2e:long`。

## 核心玩法循环

1. 玩家以某个出身、境界和身份进入蛊界。
2. DeepSeek 根据当前身份、地点、世界状态和硬事实生成叙事候选。
3. 玩家选择行动，行动经过引擎校验后影响资源、关系、声望、道心、战斗、地图或线索。
4. 经济、炼养用、战斗、小队、势力与剧情结果回流到上下文，推动下一段叙事。
5. 长线目标从生存、修行、炼蛊、建立势力，逐步推进到原著锚点和 IF 分支。

## v0.8.0 展望

v0.8.0 的重点不再是补基础系统，而是做剧情骨架和长线因果。

- 正史锚点：青茅山、商家城、三王山、王庭、义天山、逆流河、影宗、宿命战、尊者博弈、天庭末章。
- IF 宿命双路线：护宿命与毁宿命两条大轴，宿命状态只允许 `intact / fractured / destroyed`。
- 出身深线：古月、白家、熊家、商家、武家、铁家、散修、北原、东海、中洲等长期身份压力与专属剧情。
- SceneTimeContext：统一地点、时段、昼夜、安全度、场景锁、AP、修行、交易、外派、伏击和采集。
- 深系统承接：昼夜修行、伏击全链路、完整大境界突破、升仙失败、复杂灾劫、高级采集生态。
- 永生与十转边界：只做未证悬念和疯魔求证，不开放真正永生或“永生蛊”作为可获得资产。

## 视频结构建议

### 1. 30 秒开场

展示项目定位和一句话卖点：原创蛊师进入五域蛊界，在规则约束下与 AI 叙事共同推进。

### 2. 1 分钟讲世界观可信

强调玩家不是方源，原著锚点、流派、宝黄天、炼养用、宿命和永生边界都有独立真相源。

### 3. 2 分钟讲系统

依次展示创建页、行动/空窍、炼蛊、蛊虫、商会、宝黄天、战斗、小队、人物图鉴、事件日志。

### 4. 1 分钟讲 AI 闸门

展示 AI 可以写剧情、给候选，但不能直接创造未知材料、蛊虫、正式任务、地图地点或战斗奖励。

### 5. 1 分钟讲 v0.8 展望

展示正史锚点、IF 宿命双路线、出身深线、SceneTimeContext 和高阶修行灾劫。

### 6. 结尾

邀请玩家或开发者反馈：哪些原著锚点最想看到，哪些出身深线最值得优先做，哪些 UI 和数值体验还需要打磨。
