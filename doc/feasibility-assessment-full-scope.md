## 三、架构瓶颈分析

### 3.1 Token 预算（最大瓶颈）

当前 System Prompt 结构（估算）：
- Layer 1（核心指令）: ~200 tokens
- Layer 2（格式速查）: ~200 tokens
- 风格指南: ~300 tokens
- Canon/IF 模式: ~150 tokens
- 世界规则: ~400 tokens
- 经济规则: ~300 tokens
- NPC 上下文（12人）: ~600 tokens
- 术语速查: ~200 tokens
- 蛊虫知识: ~200 tokens
- **当前总计**: ~2,500 tokens

全书覆盖后需新增：
- 章节上下文（目标+名场面约束）: ~500 tokens
- 全局事件影响（当前篇章受影响的名场面）: ~300 tokens
- 五域势力动态: ~200 tokens
- **新增总计**: ~1,000 tokens

DeepSeek V4 上下文窗口: 128K tokens（足够）

**结论**: Token 预算不是瓶颈，但需严格控制 prompt 膨胀。建议 NPC 注入上限保持 10人，全局事件约束每条不超过 50 tokens。

### 3.2 AI 叙事一致性（中等瓶颈）

全书覆盖时，AI 需记住：
- 40+ 名场面的因果链
- 200+ NPC 的身份和关系网络
- 五域独立且互联的世界状态
- 玩家行为导致的蝴蝶效应

缓解策略：
1. **滚动摘要**（已有 `rollingSummary`）：每10轮压缩历史
2. **章节隔离**：不同章节独立约束集，减少 AI 需同时考虑的规则
3. **名场面模板**：每个名场面预定义 scene_constraint，注入时缩小 AI 自由度
4. **金丝雀扩展**：C14-C20 覆盖跨域约束

### 3.3 开发工作量（最大瓶颈）

按单人开发估算：

| 阶段 | 内容 | 工时估算 |
|------|------|---------|
| P1 | 3章（已有设计） | 3-4天 |
| P2 | 五域开局篇章（5×2章 = 10章） | 15-20天 |
| P2 | 全局事件系统（名场面触发/波及） | 10-12天 |
| P2 | 章节目录重构（线性→条件路径） | 5-7天 |
| P3 | 后4篇章（三王山→天庭终战） | 20-30天 |
| P3 | 跨域NPC交互 + 金丝雀扩展 | 5-7天 |
| **总计** | | **58-80天** |

以单人每周20小时开发量计算：**3-4个月全职**。

## 六、可行性综合判断

### 技术上可行吗？

**可行。** 现有 AI 驱动架构是本书覆盖的最佳技术选择：
- Context injection 模式天然支持动态约束
- Flag 驱动系统天然支持分支叙事
- PIPE 管线架构与章节数无关
- 128K token 窗口足够全书上下文

**但需注意**：不是让 AI 记住整本书——而是每轮注入「当前相关的」约束。这是关键设计原则。

### 有必要吗？

**部分必要，部分不必要。** 分析如下：

- **五域开局**：强烈推荐。这是增加重玩价值最有效的手段。每域开局体验完全不同（部落 vs 山寨 vs 宗门），4条甚至更多叙事线并行发展。
- **全书 40+ 名场面**：推荐但须分阶段。P1 验证基础架构，P2 扩展到前4卷（覆盖大部分玩家能达到的内容），P3 补齐后4卷。
- **全局事件波及系统**：必要但精简。8个全局事件（三王山/义天山/逆流河/宿命大战/天庭终战/春秋蝉/巨阳传承/天意之战），每域有不同"波及叙事"。

### 建议的最小可行方案（MVP）

```
P1 (3-4天):    南疆3章（青茅山→商队→南疆初探）
                + 章节基础架构
                + 验证 AI 章节注入可行性
                  ↓ 验证通过后继续
P2 (30-40天):  五域开局（各4章 = 共20章）
                + 8个全局事件系统
                + 章节路由重构
                  ↓ 覆盖全书前4卷
P3 (20-30天):  补齐后4篇章
                + 金丝雀扩展
                + 原 P3 内容（宝黄天/蛊仙/联网）
```

## 七、需要用户确认的决策

### 决策 19：P2 是否增加五域开局？

**选项 A**: 按原计划，P2 仅做体验打磨（音频/成就/引导），五域开局推迟到 P3。
- 优点：P2 工作量极小，快速验证章节系统
- 缺点：长期只有一条故事线，重玩价值低

**选项 B**: P2 启动五域开局（推荐）。
- 优点：大幅增加重玩价值，每条线体验不同
- 缺点：P2 工时增加 25-30天

### 决策 20：全局事件的数量

**选项 A**: 8个关键全局事件（三王山/义天山/逆流河/宿命/天庭/春秋蝉/巨阳/天意）。
**选项 B**: 精简约5个（三王山/义天山/宿命/天庭/春秋蝉）。
**选项 C**: 全部 40+ 名场面都做波及系统。

推荐 **A（8个）**——覆盖所有跨越域界的顶级事件，数量可控。

### 决策 21：章节路由架构

**选项 A**: 保持线性目录（每域独立章节列表，章节间顺序推进）。
**选项 B**: 条件路由图（章节间存在多重路径选择）。

推荐 **A**——实现简单，与当前 chapterSlice 设计兼容。条件路由需要额外 5-7 天重构。

## 八、总体结论

**全书覆盖在技术上是完全可行的**，核心得益于 AI 驱动架构——我们不是在写 250 轮手写脚本，而是在每轮注入"当前应该发生什么"的约束文本。40+ 名场面意味着 40 条 scene_constraint，而非 40 个手写剧情分支。

**但是**，P2 和 P3 的工作量会显著增加（P2 从 4-8天 增至 30-40天，P3 从 22-35天 增至 42-65天）。这不是"能不能做"的问题，而是"要用多少时间做"的问题。

**建议最务实的路线**：
1. 当前仅启动 P1 三章（按原计划），不扩大 scope
2. P1 验证章节系统可行后，再决定是否启动 P2 五域扩展
3. 五域开局可在 P2 分批实现（先做南疆+北原，再做东海+西漠+中洲）
