## 一、P1 — 核心玩法（7模块，预估总工时 12-17天）

### 模块 1：章节叙事弧光

**优先级**: 最高（最小改动，最大效果）
**预估工时**: 3-4天

**数据结构** — 新增 `src/canon/chapter.json`：

```
chapters:
  - id: "qingmao"          名称: "青茅山篇"
    境界范围: 一转 - 二转
    触发条件:
      - flag: "opened_aperture"  = true
      - 手动确认弹框
    目标:
      - "获得第一只本命蛊"
      - "在狼潮中存活"         (达到门槛可跳过，但丢失奖励)
      - "达到二转"            (完美达成条件)
    关键NPC: 古月博, 古月方正, 古月青书, 白凝冰
    关键事件: 开窍大典 → 蛊师初成 → 狼潮来袭 → 山寨覆灭
    开场叙事: "你在古月山寨的破旧木屋中醒来，今天是你参加开窍大典的日子..."

  - id: "caravan"           名称: "商队篇"
    境界范围: 二转 - 三转
    触发条件:
      - flag: "guyue_fell" = true
      - realm.min >= 2
      - 手动确认弹框
    目标:
      - "加入商队"
      - "完成第一次商队任务"
      - "与商心慈建立信任（态度≥50）"
    关键NPC: 商心慈, 商家商队NPC
    关键事件: 投奔商队 → 商队初练 → 商路遇险 → 南疆商会
    开场叙事: "山寨的废墟还冒着青烟，你跟着逃难的人流向南而行..."

  - id: "nansha"            名称: "南疆初探"
    境界范围: 三转 - 四转
    触发条件:
      - flag: "caravan_trust_50" = true
      - realm.min >= 3
      - 手动确认弹框
    目标:
      - "在南疆商会建立声望"
      - "参与一场势力争斗"
      - "掌握3种以上杀招"
    关键NPC: 南疆各大势力代表
    关键事件: 商会立足 → 势力选择 → 初次斗法 → 南疆风云
```

**context-builder 改造**：在 `buildSystemPrompt` 中新增章节上下文注入——
- 当前章节：当前激活的 chapter entry
- 章节目标：当前目标的进度状态
- 可用NPC：当前章节的关键NPC列表
- 叙事引导：AI 应引导剧情朝章节目标前进，但不强制线性

**转章流程**：
1. 条件检测（每轮 RESOLVED 后）：检查 flags + 境界是否满足下一章触发条件
2. 弹框确认：`「前方有新的机缘...」是 / 等等`
3. 转章叙事：AI 生成过渡叙事（约300字）→ 新章节 flag 激活
4. 失败目标处理：未达成目标标记为 `failed`，对应奖励永久丢失

### 模块 2：NPC 交互系统

**优先级**: 中（需章节框架稳定后接入）
**预估工时**: 4-6天

**子模块 2.1 — 对话系统**

玩家点击 NPC → 弹出话题选项面板（复用 ChoicePanel 模式）：

```
NPC 对话界面:
┌──────────────────────────────────────┐
│ 姓名: 古月方正                         │
│ 身份: 古月山寨 · 蛊师                  │
│ 态度: ████████░░ 72 (友好)            │
│                                      │
│ "你有什么事吗，方源？"                  │
├──────────────────────────────────────┤
│ 你能聊什么：                           │
│ [打探情报 —— 关于狼潮的传闻...]         │
│ [交易蛊虫 —— 你最近炼了什么蛊？]         │
│ [闲聊 —— 今天山寨里有什么新鲜事？]       │
│ [专属 —— 还记得小时候...]              │
└──────────────────────────────────────┘
```

**话题生成逻辑**：
- 通用话题（3条）：打探情报 / 交易蛊虫 / 闲聊 —— 始终可用
- 章节话题（1-2条）：基于 `chapter.json` 中 `key_events` 动态生成
- 好感度话题（1条）：态度 ≥ 70 解锁（如回忆往事、私人委托）
- 特殊关系话题（1条）：特定任务线完成后解锁

**上下文注入**：每次对话请求走 PIPE 管线，System Prompt 追加：
```
【当前对话对象】
- 姓名：古月方正
- 身份：古月山寨 · 蛊师（canon限制：不可担任山寨高层）
- 性格：正直但软弱，对方源有兄弟之情但心存警惕
- 当前心情：忧虑（狼潮将至）
- 与你的关系态度：72（友好）
- 话题：["关于狼潮的传闻..."]
```

**情感记忆**：每次对话结束后更新 `npc_xxx_attitude` flag（态度 ±1-5，取决于对话质量）。

**子模块 2.2 — 交易系统**

复用 `MerchantPanel` 模式，新增：
- 好感度折扣：态度 ≥ 80 → 8折，态度 ≥ 50 → 9折，态度 ≤ 20 → 拒绝交易
- NPC 专属商品池：每个 NPC 有 3-8 件专属商品（蛊虫/蛊材/情报）
- 限量商品：部分稀有商品售出后永久消失

**子模块 2.3 — 任务系统**

新增 `src/canon/quests.json`：

```
主线任务（绑定章节）：
  - 青茅山·开窍大典：参加开窍仪式，获得空窍
    奖励：空窍开启 | 元石×50
  - 青茅山·第一只蛊：从族中领取或自行炼制蛊虫
    奖励：随机1级蛊虫×1 | 元石×100

支线任务（模板刷新）：
  - 收集蛊材：收集 X 种蛊材 (元石×30-80)
  - 击败敌人：击败指定敌人 (元石×100 + 蛊虫)
  - 递送物品：送达物品到指定NPC (态度+10)
  - 炼制蛊虫：成功炼制指定蛊虫 (经验+炼制配方)
```

新增 `questSlice` Zustand slice：
- `activeQuests`: 当前活跃任务列表
- `completedQuests`: 已完成任务
- `acceptQuest(questId)`: 接受任务
- `completeQuest(questId)`: 完成任务（发放奖励）
- `refreshSideQuests()`: 刷新支线任务（每5轮调用）

**子模块 2.4 — 轻度感情线**

态度达到 80 + 完成该 NPC 专属任务线 → 触发 `special_relationship_npc_xxx` flag。

特殊关系效果：
- 对话文案略微调整（语气更亲近）
- 交易享受永久 8 折
- 解锁专属对话话题
- 该 NPC 可能在关键剧情中主动帮助玩家

约束：不开放婚姻/恋爱对话系统，C13 守卫检测 NPC 关系不超过 canon 允许范围。

### 模块 3：回合制战斗系统

**优先级**: 末（最复杂，复用前两个模块的基础设施）
**预估工时**: 5-7天

**核心架构 — BattleEngine 状态机**：

```
INIT → PLAYER_SELECT → AI_JUDGE → ENEMY_TURN → RESULT_CHECK
  ↑                                                    │
  └──────────────── (loop if both alive) ───────────────┘
                                           │
                                     ┌─────┴─────┐
                                 WIN_RESULT  LOSE_RESULT
```

**子模块 3.1 — 战斗触发**

触发条件（由 AI 叙事 state_update 驱动）：
- `state_update.combat.start = true`
- `state_update.combat.enemy`: { name, realm, hp, gu[] }
- `state_update.combat.context`: 战斗开场叙事

**子模块 3.2 — 蛊虫选择面板**

复用 `GuInventoryPanel` 卡片 UI，新增战斗选择模式：
- 每回合选择：1-3 只蛊虫激活（上限 = 空窍容量）
- 蛊虫分类：攻击型（红框）、防御型（蓝框）、治疗型（绿框）、控制型（紫框）
- 流派互斥提示：如选择炎道蛊虫后，水道蛊虫显示为灰色（被克）

**子模块 3.3 — 杀招系统（混合模式）**

新增 `src/canon/killer-moves.json`：

```
杀招定义：
  - id: "moon_fang"
    名称: "月牙斩"
    需求蛊虫: ["月光蛊", "月光蛊"]
    流派: 月光道
    基础威力: 15
    消耗真元: 20
    描述: "两道月光交叉斩出，锐不可当"

  - id: "iron_wall"
    名称: "铁壁"
    需求蛊虫: ["铁骨蛊", "石甲蛊"]
    流派: 土道
    基础防御: 20
    消耗真元: 15
    描述: "大地之力凝聚成壁，抵御攻击"
```

**杀招槽系统**：玩家可在非战斗时配置 3 个杀招槽（从已学会的杀招中选择）。战斗中可在杀招栏直接释放。

**自由蛊虫槽**：每回合额外可选 1 只蛊虫做自由操作——使用治疗蛊、逃跑蛊、控制蛊等。

**子模块 3.4 — 伤害计算**

```
基础攻击力 = 杀招基础威力 × (1 + 相关道痕 / 10000)

实际伤害 = 基础攻击力
         × (攻击方境界系数 / 防御方境界系数)
         × 流派相克系数（克制 ×1.5 | 被克 ×0.7）
         × 暴击随机因子（1.0 ~ 1.5, 暴击率 = 5%）
```

境界系数表：
| 境界 | 攻击系数 | 防御系数 |
|------|---------|---------|
| 凡人 | 0.5 | 0.5 |
| 一转 | 1.0 | 1.0 |
| 二转 | 1.5 | 1.3 |
| 三转 | 2.2 | 1.8 |
| 四转 | 3.5 | 2.5 |
| 五转 | 5.5 | 3.5 |

流派相克来源：`world-rules.json` 互斥表。

**子模块 3.5 — AI 判定（PIPE 复用）**

每回合将战斗状态作为 state_update 发送给 AI：

```json
{
  "combat": {
    "turn": 3,
    "player_action": "月牙斩",
    "player_hp": 72,
    "player_essence": 65,
    "enemy_hp": 45,
    "damage_dealt": 12,
    "narrative_hint": "描述这一击的效果和敌人的反应"
  }
}
```

AI 返回：敌人行动 + 伤害计算 + 叙事描述。

**子模块 3.6 — 逃跑机制（条件逃跑 + 代价递增）**

- 逃跑基础成功率 = 60%
- 速度蛊/烟雾蛊：+20%
- 境界差：每高于敌人 1 转 +10%
- 连续逃跑：每多逃跑 1 次 -15%
- 怯战 flag：逃跑次数累计 → NPC 态度变化（"那个胆小鬼"）

成功：安全脱离，丢失少量真元（-10%）
失败：受到额外伤害 + 下回合无法逃跑 + 丢失随机 1-2 只蛊虫

**子模块 3.7 — 战斗 UI 布局**

```
┌─────────────────────────────────────────────────┐
│  BattleScene (敌人描述 + HP条)                    │
│  "古月蛮石狞笑着举起石拳蛊，肌肉暴涨..."            │

## 二、P2 — 体验打磨（3模块，预估总工时 4-8天）

### 模块 4：音频系统

**优先级**: P1 完成后启动
**预估工时**: 2-3天

**三通道架构**：

```
AudioManager:
├── BGM通道（循环播放，按场景切换）
│   ├── 山寨BGM — 古月山寨/安全定居区
│   ├── 野外BGM — 探索/旅行/荒野
│   ├── 战斗BGM — 遭遇战/Boss战
│   └── 商会BGM — 宝黄天/交易场景
├── SFX通道（一次性触发）
│   ├── UI交互音效（点击/确认/取消）
│   ├── 战斗音效（蛊虫激活/命中/闪避）
│   └── 事件音效（境界突破/成就解锁）
└── Ambient通道（环境音循环）
    ├── 森林（鸟鸣/风吹树叶）
    ├── 山洞（滴水/回声）
    ├── 市集（人声喧闹/讨价还价）
    └── 战场（刀剑碰撞/哀嚎）
```

**资源来源**：
- 音效：Freesound.org（CC0 / CC BY 许可证）
- BGM：ccMixter（CC BY，已有 bgm-library skill 支持）
- 用户自定义：`public/audio/custom/` 文件夹供高级用户替换

**实现方案**：
- Web Audio API：`AudioContext` + 增益节点（已有基础 `audio.ts` 框架）
- `<audio>` 标签预加载 + AudioContext 控制播放/音量/淡入淡出
- BGM 切换：crossfade 2秒（上一首淡出 + 下一首淡入）
- 音量：BGM / SFX / Ambient 三通道独立滑块，持久化到 localStorage

**场景 → BGM 映射**：
```
场景检测（基于 flags + 当前章节）:
  current_location = "山寨"    → M_VILLAGE
  current_location = "野外"    → M_WILD
  combat.active = true         → M_BATTLE
  trade_panel.open = true      → M_MARKET
```

### 模块 6：新手引导

**优先级**: P1+P2 全部完成后
**预估工时**: 1-2天

**触发策略**：
- 每次新游戏自动触发（检测 `game_round === 0`）
- 右上角「跳过引导」按钮（立即可见）
- 设置面板中「重新查看引导」入口

**引导步骤**：

```
Step 1: 角色面板
  高亮区域：左上角角色名 + 境界显示
  文案："这是你在蛊界的身份。境界决定你的实力上限，但道痕和心计才是真正的武器。"
  交互：等待玩家点击或3秒后自动下一步

Step 2: 选项面板
  高亮区域：底部 ChoicePanel
  文案："每轮你会面对3个选择。高风险意味着高回报，也意味着高后果。善用你的道心判断。"

## 四、里程碑与时间线

```
Week 1-2:  P1-章节弧光（3-4天）→ M1 达成
Week 2-3:  P1-NPC交互（4-6天）→ M2 达成
Week 3-4:  P1-战斗系统（5-7天）→ M3 达成
Week 5-6:  P2-音频 + 成就（3-5天）→ 内部测试
Week 6:    P2-新手引导 + 技术债务（2-3天）→ M4 达成
Week 7:    内部测试 + Bug修复 + 反馈迭代
Week 8+:   进入 P3 远期规划
```

里程碑定义：
- **M1**（章节弧光完成）：玩家有明确的三章推进感，转章叙事流畅
- **M2**（NPC交互完成）：可以对话/交易/接任务，NPC 态度有反馈
- **M3**（战斗系统完成）：完整的回合制战斗体验，杀招+蛊虫选择+AI判定
- **M4**（P2全部完成）：音效+成就+引导，游戏完整可玩
