## 本轮目标

继续处理 v0.6.0 遗留的“UI/文案承诺了出身、天赋、恢复、采集、突破、声望等加成，但运行时没有实际作用”的问题。P2 目标不是把所有复杂系统做满，而是建立可信、可复用的消费者，并把没有完整子系统的效果继续标为“待系统”。

## 当前未解决/建议用户走查

- 除真实音乐资源外，进入 v0.8.0 前没有新的 P0 工程阻塞；但建议用户浏览器手走以下路径确认体验：
  - 创建页效果状态筛选是否直观。
  - 蛊虫页食料节流进度是否好懂。
  - 元石/仙元石调息按钮是否符合预期。
  - 采集/侦察/陷阱/撤离准备目前已有 store/engine 能力，但还缺更显眼的正式 UI 入口。
