# 2026-05-08 v0.7.0-pre 经济可信闸门上下文

本文只记录可交接上下文，不包含隐藏系统/开发者指令或内部推理。

## 用户目标

用户要求按 `game-dev-text` 与 `reverend-insanity-lore` 口径继续 v0.7.0-pre 工作，先完成经济可信闸门，再进入 v0.7.0-a/b/c。重点包括：

- 宝黄天 runtime 价格采用候选价表。
- 商会蛊材种类不能太少，需要刷新与食料保底。
- “地域”来源要改成清晰的“野外/地域掉落”，不能误导为稳定渠道。
- `捧杀蛊=虚情假意` 等剧情食料要有实玩机制，不得只靠文档白名单。
- 势力 runtime 收益迁移到五账本模型，列入 v0.7.0-pre 必修。

## 本轮落地

- `src/engine/auction-engine.ts`：仙蛊价格改读 `economy-balance.json` 候选价；仙蛊方定价改为仙阶公式。

- `src/store/slices/merchantSlice.ts`：新增 `materialShelf` 状态、每回合免费刷新、应急刷新记录。
- `src/components/game/MerchantPanel.tsx`：蛊材页接入刷新按钮和食料保底货架。
- `src/engine/feeding-rules.ts`：新增非物质食料信用需求。
- `src/store/slices/guSlice.ts`：`捧杀蛊` 这类非物质食料需要消耗对应 `feedingCredits`。
- `src/engine/material-source-audit.ts`：喂养矩阵增加可达性字段：稳定、权重、剧情门槛、特殊、阻塞。
- `src/components/game/GuInventoryPanel.tsx`：来源标签改为“野外/地域掉落”，并显示推荐获取行动。
- `src/engine/faction-economy.ts`：新增势力五账本经济结算。
- `src/store/slices/factionSlice.ts` / `src/store/slices/playerSlice.ts`：`advanceTurn` 改为每回合只结算一次势力五账本。
- `src/components/game/FactionPanel.tsx`：显示势力毛收入、维护、风险、食料预留、净收益。
