## 2026-05-08 追加：NPC 对话行动卡与线索闸门

- 用户反馈：人物图鉴对话中 DeepSeek 返回了 `narrative.choices`，但对话窗内没有可点击回应，导致 NPC 提出的猎杀、交易、追问选项无法继续。
- 设计决策：保留 NPC 功能，但把 `交易 / 委托 / 请教杀招 / 挑衅` 从“万能立即执行按钮”降级为受场景、好感、白名单和引擎校验约束的对话入口。
- 实现方向：
  - 新增 `DialogueActionCard`，将 NPC 回包里的 choices 显示为对话窗内行动卡。
  - `selectDialogueActionCard(cardId)` 会写入玩家回应气泡，并通过 `dialogue_choice:<cardId>` 继续走 ResponsePipeline。
  - `context-builder` 注入 NPC 对话内行动上下文，明确这是对话追问/回应，不是主线推进。
  - `dialogue_requests.add` 只落入 `flags.npcRequestCandidates`，不创建正式任务、不扣钱、不发奖励。
  - `discoveries.add` 继续承接未知地名、未知交易物、未知目标，默认无数值效果。
- 已补单测：
  - DeepSeek choices 转行动卡。
  - 点击行动卡进入等待回复状态。
  - `dialogue_requests.add` schema 可接受并保持候选语义。
- 本轮验证：
  - `npm test`：29 个测试文件 / 248 个用例通过。
  - `npm run build`：通过；仍有既有 500KB+ bundle 警告。
  - 浏览器 smoke：人物图鉴 NPC 对话可显示行动卡；点击行动卡后继续回包；`dialogue_requests.add` 落入 `flags.npcRequestCandidates`；无 runtime exception。
