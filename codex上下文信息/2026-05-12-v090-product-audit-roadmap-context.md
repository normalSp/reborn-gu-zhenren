# 2026-05-12 v0.9.0 玩法系统概要与产品冲突审计上下文

日期：2026-05-12
分支：`codex/v090-product-audit-roadmap`
基线：`codex/v080-c26-worldloop-closure`

## 本轮目标

根据用户要求，对 v0.8 末期系统做产品冲突复审，并新增 v0.9.0 玩法系统概要文档。用户已明确：

- v0.9.0 定位为玩法系统优先。
- 道场定位为剧情线索入口，不再作为随时刷新的独立菜单。

## 落地内容

- 新增 `指导大纲/v0.9.0/codex/00-总览/v0.9.0-玩法系统概要与产品冲突审计.md`。
- 文档记录了老系统、新 v0.8 竖切系统和文档承诺之间的主要冲突：
  - 道场仍是独立本地结算面板，没有接入场景 AP 与剧情战斗路由。
  - 道场刷新无数据的核心原因是 `chapterRequired` 精确匹配过硬，当前章节不在 5 个白名单时候选为 0。
  - `wild-beasts.json` 目前是 lore 规则说明，不是可直接进棋盘的敌库。
  - 旧 `duelState` / `squadCombatState` 与新 `battlefieldCombatState` 并存，需要 v0.9 统一路由。
  - c2.5 传承/福地仍是 4 类样板，全集内容需进入 v0.9 长线内容池。
- 文档明确 v0.9 玩法闭环：
  - `剧情线索 -> 出发/进入场景 -> 本地引擎结算 -> 行动账本 -> 下一轮文本承接`。
- 文档提出 v0.9 小版本路线：
  - `v0.9.0-a1` 产品冲突清账与统一路由表。
  - `v0.9.0-a2` 道场剧情线索入口重做。
  - `v0.9.0-a3` 荒兽/荒植/守护者敌库与棋盘适配。
  - `v0.9.0-b1` 统一出发协议。
  - `v0.9.0-b2` 旧战斗系统兼容收束。
  - `v0.9.0-rc` 发布候选。

## 验证

- 本轮仅新增文档与上下文记录，不改运行时代码、不改测试、不提升 `SAVE_FORMAT_VERSION`。
- 已通过只读审查确认：
  - `training-grounds.json` 有 29 个道场，类型含 `磨练 / 对决 / 试炼 / hunt`。
  - 道场章节白名单为 `qingmaoshan / shili_jueqi / shanglu_qiusheng / nilu_ascent / yitian_heaven`。
  - `wild-beasts.json` 仍是规则说明型数据。
  - `CombatEncounterSpec` 已可路由到 `duel / battlefield_5x3 / group_5x3 / group_7x5`。
