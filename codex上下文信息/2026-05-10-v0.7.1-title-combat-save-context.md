## 本轮目标

用户反馈两点：

1. DeepSeek 密钥页在测试连通前不应露出“继续冒险”；测试成功后才显示 `直接进入 / 继续冒险 / 载入存档`。
2. 使用测试存档无法直接进入战斗界面，怀疑存档、剧情文本污染或战斗系统问题。

## 根因结论

- 启动页：旧 `TitleScreen` 根据本地 store 是否有角色进度直接显示“继续冒险”，没有会话级连通验证门槛。
- 本地存档：标题页没有轻量文件导入入口，用户必须先进游戏再打开存档面板，流程不适合直接测试专项存档。
- 战斗专项存档：近期保存/加载 finalizer 为了清理旧叙事候选，曾无条件写入 `squadCombatState: null` 与 `battleVisualQueue: []`，导致小队战专项存档加载后战斗状态被清空，只剩普通剧情。

## 已完成改动

- `src/components/title/TitleScreen.tsx`
  - 新增 `connectionVerified`、`loadStatus` 与隐藏文件 input。
  - 初始只显示 API Key 输入、清除、显示/隐藏、测试连通。
  - `测试连通` 成功后显示三入口：
    - `直接进入`：进入新局时间线/角色创建流程。
    - `继续冒险`：仅有本地浏览器进度时可用；无进度时禁用并提示。
    - `载入存档`：读取本地 JSON，复用 `loadFromFile`，成功后进入 `game_play`。
  - 测试失败时不显示三入口。

- `src/store/index.ts`
  - 新增加载期战斗运行时校验：
    - 合法 `duelState` 必须有 `phase/player/enemy`。
    - 合法 `squadCombatState` 必须有 `phase/members/enemies`。
    - 合法 `battleVisualQueue` 作为数组保留。
  - `buildLoadedStoreState` 只清理 `transientCombatConstraint`。
  - 若存档同时存在单人战与小队战，优先保留小队战，清空单人战，并在 `flags._loadBattleStateConflict` 写入诊断。
  - 普通叙事存档不会凭空生成战斗状态。

- `src/store/save-load-narrative.test.ts`
  - 新增小队战专项存档加载保留 `squadCombatState` 与 `battleVisualQueue`。
  - 新增单人战专项存档加载保留 `duelState`。
  - 新增普通叙事存档不生成战斗状态。

- `tests/e2e/v070-long.spec.ts`
  - 新增 DeepSeek 连通 mock。
  - 新增标题页测试：测试连通前没有三入口；成功后出现三入口；无本地进度时继续冒险禁用；载入中文小队战存档后直接进入 `game_play` 且 `squadCombatPhase` 非空。
  - 扩展长测场景：英文小队战专项和中文 `09-小队战-三王山遭遇.json`、`10-小队战-蛊仙期遭遇.json` 都要求加载后保留小队战状态。

- `指导大纲/v0.7.0/codex/07-发布收束/v0.7.0-发布收束核对表.md`
  - 追加本轮根因、源码修复、测试补强和验证结果。

## 剩余风险与下一步

- 本轮只修“战斗专项存档加载后被清空”的问题，不等于重做普通剧情中的战斗触发节奏。普通叙事反复弹战斗候选仍按 v0.7.1 polish 的战斗遭遇循环修复计划继续处理或在浏览器走查中复验。
- 闪图/地形专项存档目前主要验证资源/地形状态，不一定都保存 active `duelState`；如果后续希望“闪图存档一加载就进战斗界面”，需要重新生成这些专项存档，而不是在 finalizer 中凭空造战斗。
- 进入 v0.8 前仍建议用户做一次人工浏览器走查：标题页测试连通、三入口、载入战斗专项存档、七出身开局、行动/空窍/炼蛊/战斗/移动端 smoke。
