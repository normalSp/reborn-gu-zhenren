## 本轮目标

把 P2 已完成的调息、修行、突破、野外行动做成清晰独立 UI 面板，并改善单人战斗表现：更清楚展示压制、资源与战斗状态，为核心流派杀招/仙蛊补 CSS fallback 闪图特效。

## 已完成源码改动

- `src/components/game/ActionPanel.tsx`
  - 新增独立“行动”面板。
  - 显示 AP、时段、真元/仙元、修行进度。
  - 提供调息、专注修行、尝试突破、侦察、采集、陷阱排查、撤离准备入口。
  - 所有行动调用现有 store action，不绕过 AP 与 engine 结算。
- `src/components/game/GameScreen.tsx`
  - 新增 `actions` 侧栏入口，并懒加载 `ActionPanel`。
- `src/engine/activity-panel.ts`
  - 新增派生预览：AP、成功率、风险、资源消耗、modifier 来源。
  - 不新增持久化字段。
- `src/canon/path-visual-profiles.json`
  - 新增核心流派视觉 profile：力道、血道、宙道、宇道、魂道、梦道、炎道、水道、冰道、金道、木道、智道、运道。
- `src/engine/path-visual-profiles.ts`
  - 校验 visual profile 必须来自 `path-registry.json` 的 `runtimeAllowed=true` 流派。
  - 提供流派 fallback 解析。
- `src/engine/killmove-bridge.ts`
  - `DuelMove` 写入杀招 path，用于视觉反馈。
- `src/engine/battle-visual-effects.ts`
  - 根据 DuelMove path / trigger tags 选择流派色、motif、强度。
- `src/components/game/BattleFlashOverlay.tsx`
  - 无真实图片时显示流派 fallback 特效。
- `src/components/game/CombatOverlay.tsx`
  - 单人战 overlay 改为更宽的三段式顶部状态区。
- `src/index.css`
  - 新增流派战斗闪图 CSS：impact、blood、time、space、soul、dream、flame、water、frost、metal、wood、wisdom、luck。
- `src/types/index.ts`
  - `DuelMove` 增加可选 `path`。
  - `BattleVisualEffectEvent` 增加 `secondaryTint`、`pathId`、`motif`、`intensity`。
  - 新增 `PathVisualProfile`。

## 剩余建议

- 用户浏览器手动走查：
  - 行动面板是否比 StatusBar 的调息按钮更顺手。
  - 突破失败预览、野外行动成功率和风险是否容易理解。
  - 战斗闪图动效是否过强、过弱或遮挡信息。
  - 小队战触发杀招时，流派 fallback 是否与单人战一致。
- 如果继续打磨体验，建议下一步做“战斗前确认页”：显示敌我境界差、推荐杀招、逃跑窗口、战利品风险和是否适合硬打。
